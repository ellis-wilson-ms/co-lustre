== Multi-Rail LNet ==

Multi-Rail LNet allows Lustre nodes to use multiple network interfaces
simultaneously for bandwidth aggregation and improved fault tolerance.
Available since Lustre 2.10.

This page covers configuration, health monitoring, and routing with
Multi-Rail. For general LNet overview, see
[[Lustre Networking (LNET) Overview]]. For LNet configuration basics,
see [[Dynamic LNet Configuration and lnetctl]].

=== Overview ===

Without Multi-Rail, each Lustre node uses a single network interface.
With Multi-Rail:

* Bandwidth aggregation: Traffic is distributed across multiple
  interfaces, increasing aggregate throughput.
* Fault tolerance: If one interface fails, traffic is automatically
  redirected to healthy interfaces.
* NUMA awareness: Interfaces can be selected based on NUMA proximity
  to the requesting CPU.

Multi-Rail works with all LNet network types (TCP, o2ib).

=== Configuring Multiple Interfaces ===

Add multiple interfaces to a network:

 lnetctl net add --net tcp --if eth0,eth1

Or for InfiniBand:

 lnetctl net add --net o2ib --if ib0,ib1

Verify:

 lnetctl net show -v

For persistent configuration in /etc/lnet.conf:

 net:
     - net type: tcp
       local NI:
         - interfaces:
               0: eth0
         - interfaces:
               0: eth1

Delete an interface:

 lnetctl net del --net tcp --if eth0

=== Configuring Multi-Rail Peers ===

LNet Discovery (enabled by default since Lustre 2.11) handles peer
interface discovery automatically. Manual configuration is needed
only if discovery is disabled.

Add a peer with multiple NIDs (first is primary):

 lnetctl peer add --prim_nid 192.168.1.30@tcp \
     --nid 192.168.1.30@tcp,192.168.2.30@tcp

Show peer details:

 lnetctl peer show -v

Delete a single NID from a peer:

 lnetctl peer del --prim_nid 192.168.1.30@tcp --nid 192.168.2.30@tcp

=== LNet Health ===

LNet Health (Lustre 2.12+, refined in 2.13+) monitors interface
health and automatically avoids unhealthy ones.

Each interface starts with health value 1000. On send failure, the
value is decremented by health_sensitivity (default: 100). The
Multi-Rail algorithm selects the interface with the highest health
value. Unhealthy interfaces are PINGed periodically; successful
PING increments health by 1.

Key parameters:

  health_sensitivity  100    Health decrement per failure. 0=disabled.
  recovery_interval   1s     PING frequency for unhealthy interfaces
  transaction_timeout  30s   Message timeout before abandoning
  retry_count          2     Maximum resend attempts. 0=disabled.

Set parameters:

 lnetctl set health_sensitivity 100
 lnetctl set recovery_interval 1
 lnetctl set transaction_timeout 10
 lnetctl set retry_count 3

Monitor health:

 lnetctl net show -v 3       # local interface health values
 lnetctl peer show -v 3      # remote peer health values
 lnetctl stats show           # global health statistics
 lnetctl global show          # current settings

=== Multi-Rail Routing (Lustre 2.13+) ===

In Lustre 2.13+, Multi-Rail and LNet Health are integrated with
routing:

* Only one route per gateway needed — MR handles interface selection.
* Gateway interface health monitored via LNet Discovery PUSH.
* Failed gateway interfaces automatically avoided.

Route configuration:

 # On clients:
 lnetctl route add --net o2ib1 --gateway rtr1@o2ib

 # On servers:
 lnetctl route add --net o2ib0 --gateway rtr1@o2ib1

 # On the router:
 lnetctl net add --net o2ib0 --if ib0,ib1
 lnetctl net add --net o2ib1 --if ib2,ib3
 lnetctl set routing 1

Routing parameters:

  avoid_asym_router_failure     1     Require healthy gateway on remote net
  alive_router_check_interval   60s   Discovery interval for gateways
  router_ping_timeout           50s   Down after timeout+interval with no traffic
  router_sensitivity_percentage 100   Failure sensitivity (100=any failure)

Pre-2.13 routing required separate routes per gateway interface:

 lnetctl route add --net o2ib1 --gateway rtrA-nid1@o2ib
 lnetctl route add --net o2ib1 --gateway rtrA-nid2@o2ib

=== Recommended Configuration ===

Starting point for /etc/lnet.conf on a Multi-Rail node:

 global:
     numa_range: 0
     max_intf: 200
     discovery: 1
     retry_count: 3
     transaction_timeout: 10
     health_sensitivity: 100
     recovery_interval: 1
 net:
     - net type: o2ib
       local NI:
         - interfaces:
               0: ib0
         - interfaces:
               0: ib1

=== Important Notes ===

* Multi-Rail is for bandwidth, not HA: If both interfaces go through
  the same switch, a switch failure takes down all rails. For true
  redundancy, use independent network paths.
* NUMA awareness: Set lnet_numa_range to control interface selection
  distance from CPU's NUMA node. Default 0 means any distance.
* Interface selection priority: (1) health value, (2) NUMA proximity,
  (3) least loaded.
