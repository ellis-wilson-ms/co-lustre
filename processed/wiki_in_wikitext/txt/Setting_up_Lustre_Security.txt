Lustre™ security features include:

* ''Access control lists (ACLs)'' - Used to set permissions or access rights for users or groups to specific system objects, such as directories or files.

* ''Root squash'' - Controls super user access rights to a Lustre file system.

* ''Shared-Secret Key (SSK)'' - Provides network-level authentication and optional RPC encryption between Lustre nodes. See [[Shared Secret Key Authentication And Encryption]] and the [https://doc.lustre.org/lustre_manual.xhtml#managingsecurity SSK chapter] in the Lustre Operations Manual.

* ''Nodemap (UID/GID Mapping)'' - Maps client UIDs and GIDs to different server-side identities, enabling multi-tenant isolation. See [[UID/GID_Mapping|UID GID Mapping]] and the [https://doc.lustre.org/lustre_manual.xhtml#lustrenodemap Nodemap chapter] in the Lustre Operations Manual.

* ''Client-side encryption (fscrypt)'' - Per-file and per-directory encryption using the Linux fscrypt framework. Available since Lustre 2.14. See the [https://doc.lustre.org/lustre_manual.xhtml Lustre Operations Manual] for configuration details.

* ''SELinux policy enforcement'' - Basic SELinux labeling support allows Lustre clients to enforce SELinux security contexts on files and directories. Requires SELinux-aware client mount options.

For general information about Lustre security features, see the [https://doc.lustre.org/lustre_manual.xhtml#managingsecurity Managing Security chapter] in the [https://doc.lustre.org/lustre_manual.xhtml Lustre Operations Manual].
