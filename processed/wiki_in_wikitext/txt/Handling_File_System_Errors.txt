From time to time, usually due to catastrophic disk / RAID failures, it may be necessary to repair the backing file system of an OST or MDT to correct file system errors.  This is done using a special version of the ''e2fsck'' tool. In such cases, it may also be useful to run ''lfsck'', a Lustre™-specific ''fsck'' tool that checks the coherency of a running Lustre file system as a whole. 

A Lustre-specific version of ''e2fsprogs'' can be found at [http://downloads.lustre.org/public/tools/e2fsprogs/ http://downloads.lustre.org/public/tools/e2fsprogs/]. A quilt patchset of all changes to the vanilla e2fsprogs is available in ''e2fsprogs-{version}-patches.tgz''.

For information about: 

* Using e2fsck on a backing file system, see the ''Recovering from Errors or Corruption on a Backing File System'' section of the [http://doc.lustre.org/lustre_manual.xhtml ''Lustre Operations Manual''].
* Running e2fsck+lfsck on a corrupted Lustre file system, see the ''Recovering from Corruption in the Lustre File System'' section of the [http://doc.lustre.org/lustre_manual.xhtml ''Lustre Operations Manual''].
* Addressing orphaned objects, see the ''Working with Orphaned Objects'' section of the [http://doc.lustre.org/lustre_manual.xhtml ''Lustre Operations Manual''].

For more information about ''lfsck'', see the ''lfsck'' section of the [http://doc.lustre.org/lustre_manual.xhtml ''Lustre Operations Manual''].

'''Note:''' In modern Lustre versions (2.6+), the integrated LFSCK tool (started via <code>lctl lfsck_start</code>) is the recommended method for online filesystem consistency checks. LFSCK can verify and repair layout, namespace, and OI scrub inconsistencies while the filesystem is mounted and in use.
