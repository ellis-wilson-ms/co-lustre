Patch Set 7:

Shilong,
for future reference I would recommend to NOT retrigger the testing on this patch just for testing purposes, especially if this patch is fixing an important bug, since that test failed for some other reason (unrelated, LU-11183), and then we couldn't land it until the testing has passed.  Luckily, Oleg landed the patch before the test had failed. 

It would be better to checkout an exact copy of this patch using the "Download->Checkout" menu item in the top right corner of the patch window (do *not* use "Cherry Pick" as that would cause this patch to be rebased and lose all test results), then create a new trivial patch on top of this one (e.g. do space->tab conversion in a file), and add:

  Test-Parameters: trivial testlist=sanity
  Test-Parameters: trivial testlist=sanity
  Test-Parameters: trivial testlist=sanity
  Test-Parameters: trivial testlist=sanity

to the commit message (each line will run in a separate session).  This adds better testing, and also avoids changing the original patch, so it can be landed without problems.
