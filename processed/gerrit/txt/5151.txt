--- Commit Message ---
LU-2669 scripts: Fix module list in lnet script

The lnet start up script's module list was incorrect causing the
following command to fail:

   # /etc/init.d/lnet stop

This patch adds missing modules, and rearranges the order certain
modules, to match the current dependencies of Lustre modules.

In particular, the osc module was moved to match it's reliance on
lov, and the following modules were added to the list: ofd, mdt,
osd_zfs, osd_ldiskfs, osp, lod, mdd, fid, and fld.

--- Message (Andreas Dilger, 2013-01-25 08:48:58.000000000) ---
Patch Set 1:

Prakash,
the drawback of landing an "ok" fix instead of a "good" fix is that once the "ok" fix is landed there is little demand to develop and land the "good" fix later, but the problem remains in the code and will continue to break and need repair by a series of "ok" fixes each time.

Ideally, rather than copying unload_dep_module() from lustre_rmmod into the "lnet" script, it would be better to enhance lustre_rmmod to add an argument to remove the "upper" modules first, and then stop lnet, and then remove the remaining modules, something like:

 lustre_rmmod -a lnet  # remove only modules above lnet
 stop_lnet
 lustre_rmmod

That avoids duplication of code, and ensures that if lustre_rmmod is improved that it will also propagate to the lnet script automatically.
