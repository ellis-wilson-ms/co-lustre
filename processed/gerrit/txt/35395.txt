--- Message (Lai Siyao, 2019-07-03 01:46:37.000000000) ---
Patch Set 2:

Client will update statfs after it expires, by default it's 60 seconds. If test_413b finishes too soon, then the second part test which verifies subdirectories are created likely on MDT which has more free space may fail because statfs not updated, client still thought all MDTs have equal space.

By setting expire time to 1 sec, the issue can be averted. While in reality, QoS mkdir doesn't need to be quite accurate, so the default 60 seconds expiry can avoid frequent statfs.
