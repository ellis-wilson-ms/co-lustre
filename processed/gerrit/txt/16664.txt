(1 comment)

I'm not against this specific patch landing, but I think we could find other bugs and regressions if e2fsck and/or LFSCK was run regularly during testing.

The http://review.whamcloud.com/16237 patch could be improved to include running e2fsck on the target filesystems before running lfsck, so that we can check this during testing.  Running once after every test script is useful, but it might also be useful to have an option run it after every (or specified) subtests, like:

 FSCK=y
 SANITY_FSCK=y
 CONF_SANITY_FSCK="83 84 85"

to mean "run fsck after every test", or "run fsck after sanity.sh" or "run e2fsck after conf-sanity test_83, test_84, and test_85".
