LU-1347 build: remove the vim/emacs modelines

Remove the expandtabs directive from vim modelines from test scripts
and autoconf files as well.  In some cases where a very large test
script has been using 4-space indents for a long time, add directives
for 4-space tabstops so that the formatting is not too ugly.

Andreas, just as a suggestion, I think it would be good if people make comments like the following when uploading patches.  For example, one's patch set 1 got a "-1".  Instead of addressing review comments and rebasing to a new parent in _one_ new patch set, she could do _two_ pushes and say:

  - Patch set 2 addresses review comments---no rebases.
  - Patch set 3 is purely a rebase---no fixes.

So that the reviewer, which has a high probability to be you ;-), could set "Old Version History" to "patch set 1" and review patch set 2.  Also pure rebases could be identified and ack'ed much easier.
