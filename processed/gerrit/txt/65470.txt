LU-18876 build: exclude more files for dpkg-buildpkg

Running "make debs" generates a source tarball via "dpkg-buildpkg"
rather than "make dist" or similar, but this includes everything
in the tree.  It includes all of the previous binaries (.o, .ko),
previous tarballs from "make debs", etc.  This is very slow.

Generate the dpkg-buildpkg exclude list from the .gitignore file,
so that it is kept relatively uptodate.  This appears to work well,
but keeps a static exclude list as a fallback in case the build is
being done outside of a git checkout (e.g. from a tarball itself).

As a starting point, it only uses the top-level .gitignore file, but
could potentially be taught to use the subdirectory .gitignore files
to exclude files in those subdirectories.
