LU-12668 target: extend BRW_READ_BULK fail_loc bitmask

Extend OBD_FAIL_OST_BRW_READ_BULK (0x20f) to support
failing multiple OSTs simultaneously via a bitmask
encoding in fail_val:

  fail_val = 0         fail all OSTs (unchanged)
  fail_val = 1..0xffff fail single OST at index
                        fail_val-1 (unchanged)
  fail_val > 0xffff    bitmask mode: bits 0-15 select
                        which OST indices to fail

This is needed for EC degraded read testing. EC
recovery requires multiple simultaneous OST read
failures to exercise the reconstruction path. The
previous single-OST targeting could only fail one
stripe at a time.

The bitmask encoding is backwards compatible -- no
existing fail_loc user passes fail_val > 0xffff for
single-OST targeting since OST indices are 16-bit.

EC recovery tests use this with CFS_FAIL_SOME and
osc.*.resend_count=1 to inject persistent read errors
that propagate to the LOV layer without causing
infinite BRW retries.
