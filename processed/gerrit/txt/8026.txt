LU-2456 lnet: DLC user space Configuration utility

This is the eighth patch of a set of patches that enables DLC.

This patch adds an lnet configuration utility, lnetctl,
which provides a way for sys admins to configure LNET
parameters.  This utility has been added to the /lnet/utils
directory

The utility is built only if libyaml is installed on the
system.

For SLES libyaml is not available, so if DLC is to be
built libyaml has to be installed manually.

--- Message (Amir Shehata, 2013-11-08 16:46:51.000000000) ---
Patch Set 9:

I'll have to see which license I need to add the initial thought was to make all the new code LGPL, but I'm not a guru, so I'll need to clarify that.

I used getopt_long() to get the command lines arguments for the initial startup of the utility.  However, the design here is to use YAML to describe the syntax, and create a library which uses the YAML file to recognize the syntax of the command lines.  This way, the library can be used without any change for more than one set of commands.  All you'll need to do is write a YAML file describing the new syntax and everything remains the same.  Of course you'll need to add a new library to handle the new commands, and pass that to the utility (or add to the existing library).  Also you can modify the command names, help lines, add extra parameters, without having to recompile, and it can be done at run time if need be.  There is an option to reload the command file.
