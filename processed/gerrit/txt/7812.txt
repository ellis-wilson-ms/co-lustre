LU-3336 lfsck: user space tools for orphan handling

Offer some user space tools to allow the administrator to process
the orphan under /lost+found.

In former LFSCK patches, the LFSCK will create the lost MDT-object
under /lost+found. The LFSCK cannot further process the MDT-object
under /lost+found becuase it has limited knowledge about where the
original file located. Then it is the administrator's duty to help
the recovery.

1) list_orphans
   List orphans on the specified MDT.
   usage: lfs list_orphans <mount_point> <MDT_index>

2) stat_orphan
   Stat the specified orphan on the specified MDT.
   usage: lfs stat_orphan <mount_point> <MDT_index> <orphan_name>

3) move_orphan
   Move the specified orphan to normal visible namespace.
   Do NOT allow to replace an existing file with the orphan.
   usage: lfs move_orphan <mount_point> <MDT_index> <orphan_name>
          <target>
