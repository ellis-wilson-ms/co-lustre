LU-0000 lov: fix designated mirror reads

The recent immediate mirror write implementation incorrectly
overrode designated mirror logic for reads, breaking the
'lfs mirror read -N<num>' functionality.

The issue was in lov_io_mirror_init() and lov_io_iter_init()
where immediate mirror logic took precedence over designated
mirror selection, causing mirror reads to always use the
immediate mirror write index instead of the user-specified
mirror ID.

This patch reorders the logic to respect proper priority:

1. Designated mirror (ci_designated_mirror > 0) - highest priority
   for 'lfs mirror read -N<num>' operations
2. Immediate mirror writes (ci_immediate_mirror && ci_write_mirror)
   for immediate mirror write operations
3. Normal mirror selection - fallback for regular operations

Changes:
- Move immediate mirror logic after designated mirror logic
  in lov_io_mirror_init()
- Add ci_designated_mirror == 0 check before applying immediate
  mirror logic in lov_io_iter_init()

Testing confirms:
- Immediate mirror writes: Both OSTs receive writes
- Immediate mirror reads: Both OSTs can be read from
- Normal mirror operations: Preserved existing behavior
- Data integrity: Checksums match between mirrors
