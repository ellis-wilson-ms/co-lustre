--- Message (Andreas Dilger, 2012-08-22 20:49:56.000000000) ---
Patch Set 4: I would prefer that you didn't submit this

(2 inline comments)

Even if this isn't implementing a test case for 2.1->2.3+ interoperability, it makes sense to reserve the OBD_FAIL and test numbers in this patch for 2.3+ so that conflicting values are not assigned later on compared to b2_1.

Without this, some later test may run on 2.3+ clients with 2.1 servers and re-use the same OBD_FAIL value, causing different problems.  Similarly, if this new b2_1 test fails, tracking the bug will be confusing since 2.3 would have a different test at that number.
