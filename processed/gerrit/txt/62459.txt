--- Commit Message ---
LU-19545 ec: tight binding parity mirror to data mirror

Parity mirrors need to be tightly coupled to the specific data mirror
they were created to match, so that we can find the data mirror
corresponding to the parity mirror.

To accomplish this, we store the mirror ID of the data mirror in each
of the corresponding parity components and vice versa. The existing
64-bit lcme_timestamp field in lov_comp_md_entry_v1 is repurposed as a
union (lcme_time_and_id) that packs the timestamp in the lower 48 bits
and a new lcme_mirror_link_id in the upper 16 bits, preserving the
same offset and size on disk and on the wire. A new LCME_FL_IS_LINK_ID
component flag indicates when the link ID is valid.

Added llapi_layout_test tests 62-64. Also added comp verifications for
various llapi_layout_test tests to verify that data and parity
components were bound correctly by the MDS.
