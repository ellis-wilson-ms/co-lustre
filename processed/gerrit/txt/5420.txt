--- Message (Jinshan Xiong, 2013-03-01 14:00:37.000000000) ---
Patch Set 2:

"First, we test file size before comparison, so how there could be EOF?"

Because we're comparing the file content between file1 and ref2. From the failure, the size of ref1 is 47M, and ref2 is 30M. Dd copied 42M when swapping happened so copied=42M. After files are swapped, size of file2 will be 42M, and file1 will be 47M. Also the file content of file1 is: {30M ref2, sparse, 5M from ref1}.

After file are swapped, the size of file2 is fixed which is exactly how many bytes it copied. Reducing dd block size definitely makes copying slower so I'm not sure I understand your comment.
