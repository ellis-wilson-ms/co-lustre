--- Message (Andreas Dilger, 2012-03-15 10:05:16.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

I've added Liang as an inspector for this patch, since he is our networking Guru and should be inspecting this patch.

As with other patch comments, removing support for clients newer than 2.6.18 is undesirable, so parts of this patch should not be landed.

With OFED there is an added complexity that some users compile against out-of-tree OFED releases for a variety of reasons, so just because specific features were back-ported to RHEL5 does not necessarily mean that the out-of-tree OFED code has those features.

That means (5) and (6) are probably off the list, and probably (2) as well.
