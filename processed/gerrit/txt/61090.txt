--- Commit Message ---
LU-19157 nodemap: make nodemap name optional for banlist

Make optional the --name option to lctl nodemap_banlist_add and
nodemap_banlist_del. When omitted, the nodemap to use is inferred from
the existing NID ranges that include the provided ban range. If none
is found, the default nodemap is used.
This makes it easier for admins to ban clients, by not requiring them
to identify first in which nodemap they are.

Enhance sanity-sec test_81a to exercise this.
