LU-15441 wbc: write hang with O_DSYNC flag

When benchmark with filebench oltp workload, the writer hanged.
After investigated, we found the following dead lock in WBC code:

static ssize_t memfs_file_write_iter(..)
{
	down_read(&wbci->wbci_rw_sem);
repeat:
	if (wbc_inode_data_caching(wbci)) {
		rc = generic_file_write_iter(iocb, iter);
		...
	}
	up_read(&wbci->wbci_rw_sem);
	...
}

generic_file_write_iter() -> generic_write_sync()
static inline ssize_t generic_write_sync(...)
{
	if (iocb->ki_flags & IOCB_DSYNC) {
		int ret = vfs_fsync_range(...)
		...
	}
	...
}

static int memfs_fsync(struct file *file, loff_t start,
		       loff_t end, int datasync)
{
	...

	if (S_ISREG(inode->i_mode)) {
		down_write(&wbci->wbci_rw_sem); /* dead lock */
		rc = wbcfs_commit_cache_pages(inode);
		up_write(&wbci->wbci_rw_sem);
	}
	...
}

In this patch, we move @generic_write_sync() out of the rw
@wbci_rw_semaphore to avoid the deadlock.
