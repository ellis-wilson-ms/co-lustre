LU-1606 api: Define empty LASSERT (and variants) for user space

We need to have SOME definition of LASSERT and its variants
for user space, because they have crept into places like
lustre_idl.h.  Since a library intended to be used by normal
users should not be triggering assertiions in the first place,
it seems somewhat reasonable to make these no-ops.
