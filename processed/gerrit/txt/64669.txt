--- Commit Message ---
LU-20010 osd-zfs: modularise FatZAP default blockshift

Lustre directories on ZFS always use FatZAP format because each
directory entry stores both the ZFS dnode ID and a Lustre FID,
which exceeds the MicroZAP single uint64 value limitation.

The FatZAP leaf block size is controlled by leaf_blockshift, which
was previously hardcoded to 14 (16K blocks). On dRAID or RAIDZ pools
dsize is around 90-110K per empty directory due to stripe alignment
and parity overhead.

Lower blockshift has shown around ~30% reduction in directory dsize
and improved mkdir performance.

This patch introduces the sysfs tunable osd-zfs.*.fzap_blockshift to
control the FatZAP leaf block size for individual OSDs.
