--- Commit Message ---
LU-18111 ptlrpc: don't drop expired cancel request

There is no need to drop expired cancel request by
a server because the client will resend the same content.
Even if the server is heavy loaded cancel request processing
helps to release ldlm resources and avoids spending time
on processing of the same resends.

Add extra check to prevent same cookie for another client.

--- Message (Andreas Dilger, 2024-08-24 01:57:12.000000000) ---
Patch Set 2:

Oleg, presumably this would still cancel the lock earlier, and allow other clients to acquire the lock, rather than dropping the cancel and waiting for the client to resend it again?  Also, by cancelling the lock sooner, it reduces the risk of the client being evicted due to not cancelling it sooner.  Even though that _shouldn't_ happen, if the client has not cancelled and has some delay in resending the next cancel the server may consider it non-responsive.
