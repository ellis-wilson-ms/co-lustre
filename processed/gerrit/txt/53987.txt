--- Commit Message ---
LU-17518 gss: do not trust supp groups from client with krb

Thanks to Kerberos, Lustre does not have to trust clients anymore,
but relies on keytabs and tickets, cryptographically validated, to
recognize clients and users.
RPC provided supplementary groups should not be trusted, but checked
thanks to identity upcall and the trusted UID from the ticket.

Add sanity-krb5 test_9 to exercise this.

So to avoid modifications in all these places, I preferred to add a new field in 'struct lu_ucred', that can be used to later check if the supplementary groups need to be taken into account. This is easier, because there are only 2 places in the code where 'struct lu_ucred' is computed. Also it is more robust and less error prone to future code changes. If new code is written that sets uc_suppgids, then there is some chance that it is done without taking care of the GSS aspect.
