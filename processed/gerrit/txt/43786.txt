Please do not use the LU ticket for the patch itself as a reason for retesting the patch.  If LU-14707 is the reason this test failed, then the patch should be fixed rather than retested.  If there is some other reason (unrelated to this patch) that is causing the failure, then *that* ticket should be used (or created, if it is clearly not related to this patch) so that it can be tracked properly.

What we don't want is marginal patches being retested several times until they pass, then landing a new regression into the code.  Yes, it happens sometimes, but it helps all of us if test failures are properly tracked (and ideally fixed), or we all suffer the consequences of constant test failures.
