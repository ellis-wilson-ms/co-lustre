LU-13982 tests: fix infinite loop in sanity test_184c

If the dd in test_184c fails to create the file, for example due to
ENOSPC, the subsequent "while" loops indefinitely.

So add a loop-count to ensure it stops eventually.

Also change the test from "-f" to "-s" so we wait for the file to be
non-empty.

Yes, this is a problem that I've actually hit several times.  I'm trying to get all the tests to pass and I very often get ENOSPC errors (which is weird!)
Sometimes the 'dd' fails to create the file due to ENOSPC.  Times when I've checked, one of the two OSTs is (nearly) full, the other has plenty of space.

I think the purpose of the while is to wait for the file to have been created so that "lfs swap_layouts" doesn't try to open the file before it has been created.  My patch preserves this with risking indefinite wait.

It may make sense to add a small sleep to ensure the file isn't empty when swap_layouts starts, but that doesn't seem to be the purpose of the current code.
