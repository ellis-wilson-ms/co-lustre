LU-10553 tests: create and cleanup test specific working dir

We have seen a variety of tests fail because they are not
to clean up a file or directory left behind by a previous
test suite.  We need to make and, when the test suite is
done, remove a specific working directory for each test
suite.  Thus, this can insulate each test suite from each
other and reduce these failures.

To do this work, there is related clean up for the DIR*
variables that can be done:
* sanity-flr makes use of DIR3. So, add DIR3 to assert_DIR().
* assert_DIR() is called in run_test() and run_test() is
called for every test in a test suite. This seems excessive.
Let’s remove the call to assert_DIR() in run_test() and
make sure each test suite calls assert_DIR() and checks the
return code.
* assert_DIR() exits with an error code. Let’s change the
exit() to a return() so that callers to assert_DIR() can
check the return code and choose to exit or not.  We should
check the return code and call error if the return code is
not true(0) for all scripts that use DIR*.
* We export all the DIR* variables in sanitize_parameters()
which is called in check_and_setup_lustre().  Thus, we can
remove export of DIR2 in init_test_env().
* Add DIR3 and MOUNT3 parameters to sanitize_parameters().
* The MOUNTED variable is not used in check_and_setup_lustre()
and should be removed.
* Grab the first input parameter to check_and_cleanup_lustre()
and, if it exists, use this as the directories to remove at
the end of the test suite.  If it does not exist, use the
existing expression “rm -rf $DIR/[Rdfs][0-9]*”.
