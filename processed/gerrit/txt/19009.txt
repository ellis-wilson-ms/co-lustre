LU-7885 build: Add option for extra Module.symvers

Lustre would not install properly on a particular platform because it
wasn't built with the gni Module.symvers. While this only causes a
warning on SLES, it stops the installation on CentOS.

Add option to build with an out-of-tree Module.symvers.

Patch Set 4: Code-Review-1

I think a clearer explanation of the problem is needed.  Lustre can build against ZFS and other externally-built kernel modules without requiring this kind of special handling.

It sounds to me like you have an environment where the gni prerequisite hasn't been properly packaged and installed.  If it was, this problem wouldn't exist.  I would argue then that the core code should not be an enabler of broken processes.  If others want to have a weird broken process, they can carry the burden of carrying weird hacks to Lustre support that rather than making it a community burden.

If I'm missing something, please do elaborate on why the build environment is not a broken one.
