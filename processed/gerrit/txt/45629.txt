The exception was actually for ppc64 and not 64KB PAGE_SIZE, so it may be that there is some other issue here.  It looks like there have been a small number of ppc64le sanityn test runs, but they have all failed (even 2.14.57, which includes the LU-15122 patch).


Also, looking at the test itself, it seems clear that it won't work properly on a 64KB PAGE_SIZE system:

         # write data this way: hole - data - hole - data
        dd if=/dev/urandom of=$DIR1/$tfile bs=40K seek=1 count=1
        dd if=/dev/urandom of=$DIR1/$tfile bs=40K seek=3 count=1

because the "hole" is from 80KB-120KB, and with 64KB PAGE_SIZE it will actually dirty the 64KB-128KB page...

The test needs to be updated to use seek=0 for the first dd and it should pass.
