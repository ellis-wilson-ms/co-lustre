LU-10026 sec: additional memory page for an object

This page can:
1. Solve the problem when LZ4 access memory outside the
   allocated buffer during the decompress
2. Adds additional tail page poisoning to debug any
   pool misuse

These means are disabled in this patch, define:
POOL_ADD_PAGE to add a page to every allocated object
POOL_PAGE_POISON to poison and check such an additional page

EX-7536 csdc: fix POOLS_ADD_PAGE and POOL_PAGE_POISON

POOL_ADD_PAGE macross enable additional page for debug
purpose. POOL_PAGE_POISON macros enable poisoning and
poison check.

This patch adds POOLS_ADD_PAGE and POOL_PAGE_POISON
suppourt in case when only one page is allocated
from the pool. This useful for 64k page aarch64 clients.
