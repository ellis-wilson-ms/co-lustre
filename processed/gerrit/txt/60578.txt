--- Commit Message ---
LU-19245 dne: update lmm_oi in dir migrate

A dir migrate doesn't update lov lmm->lmm_oi during
migrate so LFSCK starts to complain about "bad lmm_oi".
Also adding a sane error code to lfsck_layout_master_exec_oit()
so it reports -EINVAL in dryrun mode instead of buffer size
from the previous function call.

Fix lfsck_layout_master_exec_oit() to be aware of
LOV_MAGIC_FOREIGN layout while checking and updating lmm_oi.
