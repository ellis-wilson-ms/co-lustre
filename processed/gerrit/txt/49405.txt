LU-16314 obdclass: Migrate LASSERTF %p to %px

This change covers lustre/obdclass through lustre/target and
converts LASSERTF statements to explicitly use %px.

Use %px to explicitly report the non-hashed pointer value
messages printed when a kernel panic is imminent. When
analyzing a crash dump the associated kernel address can
be used to determine the system state that lead to the
system crash.

As crash dumps can and are provided by customers from
production systems the use of the kernel command line
parameter:
    no_hash_pointers
is not always possible.

Ref: Documentation/core-api/printk-formats.rst

Patch Set 1: Code-Review-1

(7 comments)

Thanks for starting to look at this.  I would recommend that after you have done the mechanical replacement of the "%p" pointers that you look at the actual error/debug messages to see if they are useful in their current form.

For CDEBUG() and LASSERT() it seems OK to print the raw pointers (if that is actually useful) because they could be extracted from a crashdump.

However, for CERROR() messages that may only be seen hours or days later it seems less useful to print a raw pointer.  Not only does that expose kernel internals (which was the original reason for this %p change), but in many cases there could be much more useful "persistent" information printed about the system that could be used for post-mortem debugging.

Since we are taking time to look at every one of these error messages, we may as well improve them to be more useful.
