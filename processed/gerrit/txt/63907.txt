--- Commit Message ---
LU-16742 osd: allow multiple LUFIDs

Lustre Metadata Redundancy can benefit from storing
more than one LUFID in the same dirdata field. Currently,
there are no explicit restrictions preventing this:
multiple LUFIDs can be stored sequentially, with the only
limitation being the string size (255 bytes).

This fix removes checks that assume only a single LUFID
is present (while keeping size sanity checks intact) and
adds a test case that inserts three LUFIDs using fault
injection.
