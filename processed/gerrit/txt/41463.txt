LU-12815 socklnd: allow dynamic setting of conns_per_peer

Modify lnetctl and associated code to allow dynamic setting
of conns_per_peer lnd parameter per ni.

The parameter can be set for a specific active nid:
        lnetctl net set --nid 192.168.122.10@tcp --conns-per-peer=4

Or when adding a new net, taking effect on the new nid:
        lnetctl net add --net tcp --if eth0 --conns-per-peer=1

By default, conns_per_peer value specified as the module parameter
shall be used.

Serguei, thanks for the update to this patch, as I think it is important to get this one landed. Two high-level comments about this patch:
- there hasn't really been any explanation why the socklnd tuning is in the lnet code?  Even if the tunable needs to go through lnet, the heuristics for selecting the number of connections should be in the lnd itself.
- it would be better to limit this patch to allowing the tunable to be set dynamically and move the heuristics for selecting the number of connections to a separate patch. I think the dynamic tuning is useful in itself, and will allow more extensive testing for good default numbers (without the need to remount between each test), and be useful input for the patch that adds the heuristics.
