LU-19444 lnet: add IPv6 support for ip2nets configuration

Add support for IPv6 addresses and CIDR netmasks in ip2nets YAML
configuration, allowing specification of single IPv6 addresses or
IPv6 networks.

Extended lustre_lnet_ip_range_descr structure to support both IPv4
and IPv6, with separate fields for IPv6 addresses, netmasks, and
network addresses. Modified lustre_lnet_add_ip_range() to detect
and parse IPv6 addresses (by presence of ':') and handle optional
CIDR prefix notation (e.g., /64, /125), defaulting to /128 for
single addresses.

lustre_lnet_match_ip_to_intf() modified to iterate over all addresses
on an interface to check for matches against the supplied ip ranges.

Added lustre_lnet_match_ipv6_netmask() helper function to perform
IPv6 netmask matching. Exported libcfs_ip_str2addr_size() from
libcfs for use by liblnetconfig.

Example ip2nets yaml with ipv6:
ip2nets:
  - net-spec: tcp1
    interfaces:
         0: eth0
    ip-range:
         0: 2601:8c1:c180:2000:1000:1000:1000:cbdd
  - net-spec: tcp2
    interfaces:
         0: eth1
    ip-range:
         0: 2601:8c1:c180:2000:1000:1000:1000:cbd8/125
