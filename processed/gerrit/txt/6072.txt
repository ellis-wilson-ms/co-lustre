LU-3138 osd: ignore unlink inode in index delete

We should ignore unlink inode during index delete, otherwise
inode will be become bad inode, which cause unclean delete.

--- Message (wangdi, 2013-04-18 00:51:02.000000000) ---
Patch Set 1:

oh, if you do dt_ref_del first, then it will become an zero nlink file. Then in dt_delete(osd_object_destroy), it will do osd_iget (for trying to delete agent inode, if any), and with 0-link, osd_iget will make it become bad inode.  Then any following operation with bad inode will be affected.  

For this bug,  some parts of ldiskfs_delete_inode will be skipped, which cause the problem

--- Message (Alex Zhuravlev, 2013-04-18 16:02:56.000000000) ---
