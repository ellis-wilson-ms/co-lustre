--- Message (Andreas Dilger, 2016-06-08 04:24:21.000000000) ---
Patch Set 4:

The use of Obsoletes is consistent with the kernel.spec file:

 Obsoletes: kernel-headers
 Provides: kernel-headers = %{version}

and also the RHEL-6/7 e2fsprogs.spec file:

 Obsoletes: e4fsprogs < %{version}-%{release}
 Provides: e4fsprogs = %{version}-%{release}

I think it makes sense to limit this to Obsolete lustre-client of the current or earlier releases so that it would not cause problems with future packages:

 Obsoletes: lustre-client <= %{version}-%{release}
 Provides: lustre-client = %{version}-%{release}

though if/when we split into separate lustre and lustre-client packages the "Obsoletes:" part would be removed.

Running rpmlint on the current lustre.spec file complains about "unversioned-explicit-obsoletes" and "unversioned-explicit-provides", so this definitely seems like something that the RPM .spec is intended to contain.

I'll fix those issues and a few more that rpmlint is complaining about and re-push the patch.
