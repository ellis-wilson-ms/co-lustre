LU-12867 dne: place temp file on MDT where target is

Map temporary file created by rsync or mpifileutils dstripe on
the same MDT where target is located, so later rename won't
make target a remote entry.

And the same for backup file ends with .bak, .sav, .orig or ~.

Lai, I understand the motivation, and I admit I cannot think of a better way to handle a tool like rsync that does this.  However there are two reasons I think this is a mistake:
1. Highly fragile - rsync or mpitools might change the naming convention.
2. It also will affect other applications, creating different behaviors (unexpected by the user) from file to file.
