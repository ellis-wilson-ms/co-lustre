I see you are coming in late for the review. The reason for the new lnet_peer_primary_nid_locked() and lots conversion from nid to nid4 changes is that lnet_nid_t and struct lnet_nid are completely different. 
When you change a single lnet_nid_t variable to struct lnet_nid it still interacts with other code that are using lnet_nid_t for things like comparison. This approach avoids a super huge patch that would convert everything to struct lnet_nid. 

Just try changing one field in the LNet code lnet_nid_t to struct lnet_nid and see what happens.
