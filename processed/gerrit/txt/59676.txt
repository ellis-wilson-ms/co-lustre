LU-13643 flr: implement automatic FLR state transitions

Implement automatic server-driven FLR state transitions for
immediate mirror files using ACTIVE_WRITER locks. This enables
seamless RDONLY to WRITE_PENDING transitions based on active
writer lock lifecycle, providing the foundation for concurrent
write functionality.

Key Features:
- Automatic RDONLY to WRITE_PENDING transition when first
  ACTIVE_WRITER lock is acquired
- Automatic WRITE_PENDING to RDONLY transition when last
  ACTIVE_WRITER lock is released
- Server-side active writer counter tracking per mdt_object
- Workqueue-based FLR transitions for safe execution from
  blocking AST context
- Complete integration with existing FLR infrastructure

Client-side changes:
- Add immediate mirror detection in ll_file_io_generic()
- Implement ACTIVE_WRITER lock acquisition for immediate
  mirrored files during write operations
- Add ll_get_mirror_info() helper for layout analysis
- Integrate with existing write path in ll_file_write_iter()

Server-side changes:
- Add IT_ACTIVE_WRITER intent handling in mdt_intent_opc()
- Implement mdt_intent_active_writer() handler function
- Add MDS_INODELOCK_ACTIVE_WRITER lock bit (0x000000080)
- Add mot_active_writer_count atomic counter to mdt_object
- Implement mdt_active_writer_inc/dec() counter management
- Add custom blocking AST handling for ACTIVE_WRITER locks

FLR transition infrastructure:
- Add MD_LAYOUT_WRITE_DONE operation for server-driven
  transitions
- Add LAYOUT_INTENT_WRITE_DONE for intent-based transitions
- Implement workqueue (mdt_flr_wq) for safe FLR transitions
- Add mdt_flr_state_transition_to_rdonly() for WRITE_PENDING
  to RDONLY transitions with mirror un-staling
- Integrate with existing mo_layout_change() infrastructure

LOD layer enhancements:
- Add MD_LAYOUT_WRITE_DONE handling in
  lod_declare_update_write_pending()
- Implement mirror un-staling logic for WRITE_PENDING to
  RDONLY transitions
- Preserve existing FLR state machine behavior

Wire protocol updates:
- Add LAYOUT_INTENT_WRITE_DONE = 10 constant
- Update wiretest.c assertions for new intent
- Maintain backward compatibility

The implementation follows Lustre's existing patterns:
- Uses standard LDLM lock infrastructure
- Integrates with existing FLR state machine
- Follows kernel-style error handling
- Uses proper transaction handling for layout changes
- Includes comprehensive debug logging

Testing shows successful operation:
- Immediate mirror files correctly acquire ACTIVE_WRITER locks
- FLR state transitions occur automatically (RDONLY to
  WRITE_PENDING to RDONLY)
- Layout generation increments confirm state changes
- Active writer counter tracks lock lifecycle accurately
- Workqueue safely handles transitions from blocking AST

This provides the foundation for concurrent write functionality
where multiple clients can write to different mirrors
simultaneously while maintaining FLR consistency.
