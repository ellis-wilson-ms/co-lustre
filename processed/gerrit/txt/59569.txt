LU-0000 flr: add immediate mirror flag for FLR

Add LCM_FL_IMMEDIATE_MIRROR flag to support immediate
mirroring functionality in Lustre File Level Replication
(FLR). This flag indicates that mirrors should be created
and synchronized immediately rather than using the default
lazy synchronization behavior.

The flag uses value 0x4 which fits within the existing
4-bit ldo_flr_state bitfield, following the same pattern
as LCM_FL_PCC_RDONLY. This avoids expanding kernel data
structures while providing the needed functionality.

Changes include:
- Add LCM_FL_IMMEDIATE_MIRROR flag definition (0x4)
- Update LCM_FL_FLR_MASK to include new flag (0xF)
- Add --immediate option to lfs mirror create command
- Display "im" flag in lfs getstripe output for immediate
  mirrors
- Preserve immediate flag in kernel during mirror creation
- Update wire protocol constants in both wiretest.c files

The immediate flag can only be specified during mirror
creation and is preserved alongside the readonly flag in
the kernel's FLR state.
