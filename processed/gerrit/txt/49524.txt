--- Commit Message ---
LU-7579 osd-zfs: set orphan flag on the cached object

osd_attr_set() (zfs) write on disk the LMAI_ORPHAN flag on
"trusted.lma" xattr but does it not update the on cache object with
LUSTRE_ORPHAN_FL flag.
So after unlinking a directory, a client can still create a child
in it because DEAD_OBJ is not set (mdd_la_get()).

On the master branch this is corrected by :
28be311 ("LU-12275 sec: enable client side encryption")
