I realize my added test case is not doing what I wanted, because OBD_FAIL_LOV_INVALID_OSTIDX is only corrupting the in-memory LOV EA sent to the client, and not the actual layout on disk.  Even if the LOV EA was bad on the MDS, the test would fail because it is the first mirror that is corrupted, so the objects would be leaked.

Alex, do you know what happens on the MDS when an actually broken file is deleted?  Does the MDS just report an error, or does it crash/LASSERT because of the bad layout?  If deleting the broken file causes the MDS to crash, then it isn't really safe to give this patch to the customer yet, but if it just returns an error and leaks the object then it is probably OK.
