LU-18463 ptlrpc: removing cfs_flush_fput idle

mdtest performance degradations were observed for directory removes
of up to 28%. The cause is LU-16973.
The fix removes flushing from a ptlrpc path, and introduce a
work for osd ldiskfs. Let's flush files base on count. Also it
adds the sysfs file to set a descriptors count for flushing.
By default 5k files.

 echo 1000 > /sys/fs/lustre/osd-ldiskfs/flush_descriptors_cnt
 cat /sys/fs/lustre/osd-ldiskfs/flush_descriptors_cnt
 1000

(9 comments)

Thanks for pushing this patch.  I think it is mostly good, but a few fixes needed:
- 100k pending file handles is too many, probably 1000-5000 is reasonable, since the amount of work to clean them up is _mostly_ the same whether done in large batches or small ones, as long as the batch size is not _too_ small
- the "minutes" tunable is strange, and there is no good reason to not use "seconds" for this tunable.  You could have a minimum value like 5s if you think that would be a problem if it ran too often.

If you have already deployed this patch in production and need to maintain compatibility with the "minutes" parameter, you can always have a small patch that adds the "minutes" parameter and converts internally to seconds and prints a deprecation warning if used.
