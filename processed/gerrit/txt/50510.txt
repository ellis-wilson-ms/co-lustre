--- Commit Message ---
LU-16671 osc: a fix for unstable pages accounting

1) disable unstable accounting for short io since
   there's no bulk and no page cache pages are pinned
2) account unstable pages as writeback with writeback
   control
3) the proper zone stat name is NR_ZONE_WRITE_PENDING
4) don't wait in client unmount until unstable pages
   are commited - the server can be unavailable and
   we still need to umount, the statistics will be
   updated when ptlrpc requests are freed

This patch allows to avoid OOM with the unified memory
cgroup with writing to a file on Lustre.

The compat code here seems very confused.

But this is looking more important - the lowmem_sync approach has a problem at scale with cgroups.  Specifically, cgroups seems to only be freeing *just enough* memory so it can do the next allocation, rather than freeing all of the no-longer-unstable-pages.  So we stay stuck in memory allocation even though there is freeable memory.  I'm not sure we can fix that within Lustre - I'm still reflecting on it.

So I'd like to test this approach instead but I'm not sure what kernels this is going to work on or how.
