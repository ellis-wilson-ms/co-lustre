LU-10994 clio: remove cpo_prep and cpo_make_ready

Remove the cpo_prep and cpo_make_ready methods from struct
cl_page_operations. These methods were only implemented by the vvp
layer and so they can be easily inlined into cl_page_prep() and
cl_page_make_ready().

--- Message (John Hammond, 2022-05-20 12:46:10.000000000) ---
Patch Set 1:

> Are you also targeting removing cpo_completion?  It looks like it could go as well now.

Removing vvp_page_completion_read() runs into a similar readahead accounting problem as we saw in vvp_page_discard(). We may need to call ll_ra_count_put(ll_i2sbi(inode), 1). One way to address this would be to move the readahead stats and counters to struct cl_client_cache which is accessible from all three layers.

In vvp_page_completion_write() we need ll_dirty_page_discard_warn() but I think this is not hard to address.

--- Message (Patrick Farrell, 2022-05-20 15:23:24.000000000) ---
Patch Set 1:

I'm fine with that idea - I don't care if the readahead stats live in the SBI.  The client cache struct seems reasonable.
