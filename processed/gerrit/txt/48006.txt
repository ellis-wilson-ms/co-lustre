LU-16012 llite: null secctx name when selinux disabled

When touch a file on rhel 9.0 client with SELinux disabled, the
following LBUG occurred:
"mdc_lib.c:139:mdc_file_secctx_pack()) ASSERTION( buf_size ==
strlen(secctx_name) + 1 ) failed"
After enabling SELinux on rhel 9.0 client touching a file passed.

The reason is that in @ll_lookup_it(), we set @op_file_secctx_name
with XATTR_NAME_SELINUX in the function @ll_dentry_init_security,
but when get name of security xattr in @ll_listsecurity(), the
return code is 0 as the SELinux is disabled, but we do not reset
@op_file_secctx_name with NULL.
After reset @op_file_secctx_name, touching a file passed when
SELinux is disabled.
