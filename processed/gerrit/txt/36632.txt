Patch Set 1: Code-Review+2

Patch seems correct, but as stated in LU-12856 it isn't clear if this is going to fix the problem.  The flags field is zeroed at allocation time, so it is unlikely that the OBD_FL_SHORT_IO bit would be set randomly without OBD_MD_FLFLAGS being set, but I'm open to being wrong about that (e.g. if the buffer is re-used or something).
