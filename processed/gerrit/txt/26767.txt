LU-9379 tests: Improve get_version to handle empty return value

replay-vbr fails when get_version() returns empty value to
either pre/post variable. This patch modifies replay-vbr
get_version() by returning '-1' in case getobjversion fails.

This patch also adds chk_get_version(). Which is a wrapper
to get_version(). chk_get_version() helps in verifying
return code of get_version().

Arshad – I apologize for the delay. 

I’ve looked at this patch several times and had a couple of issues with it. I’m uncomfortable with the modification to get_version() that looks specifically for the file name in test 0a and takes specific action based on that name. I also don’t think it’s the job of get_version() to check if there is a valid output and return an error, but this point could be debated. 

I’m not totally against this patch, I think the purpose of test 0a is still intact and the changes you made to get_version() are the least intrusive compared to modifying all calls to get_version() to check the return status. Why can’t we add a return code to get_version() is there is no output and the subtest can choose to check the return code or not?

Please let me know what you think.
