> There are t-f helpers to do this, but maybe this is OK for an ldiskfs-only test.

Do you mean lctl set_param -n osd*.*.force_sync 1?
Unfortunately, that is not enough for case when mds1 is a regular file associated with loop device. "lctl force_sync" forces loop device transactions to commit, while the regular file does not get synced. As long as debugfs runs for the regular file (which is retuned by mdsdevname), debugfs does not see the changes.

> "sync" on the MDS doesn't necessarily do anything? 

It does commits for the loop device and the regular file.
