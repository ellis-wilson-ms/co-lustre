Patch Set 1: Code-Review-2

I don't understand this patch?

Is this cherry picking a patch *from* master *to* master, because this branch is so old that it doesn't contain the el8.7 patch series that landed 6 months ago?  This patch should have been marked "fortestonly" or similar, because it seems like it should never be landed. 

It would make sense to go through this whole patch series to rebase it on a current master version, and also merge the fix patches later in the series back into the original patches that introduced the problem, and generally take this opportunity to see if the code can be reorganized to make the patches easier to review and land. 

The focus should definitely be on the early parts of the series that provide the base WBC functionality, recovery, testing, etc. and leave the more complex enhancements (eg. PCC, "createahead", etc.) for a later time, unless they require wire protocol/data structure changes that need to be included in the earlier patches to avoid breaking compatibility later on. 

This will allow the initial patch series to be tested, reviewed and landed in time for 2.17, but I think the whole series is too large and complex to land all at once.
