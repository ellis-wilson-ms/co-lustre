--- Message (Olaf Faaland, 2016-12-21 01:14:31.000000000) ---
Patch Set 1:

Nathaniel and Alex:

Thanks for looking at this.

osd_object_ref_add/delete update the link count, via sa_update...SA_ZPL_LINKS().  They do not update the size, that would be sa_update...SA_ZPL_SIZE().

Lustre updates the size of non-ZAP objects via SA_ZPL_SIZE.  It does not appear to update the size of ZAP objects.

This came up because we have encountered issues testing DNE phase 2 where one or more llogs in update_log_dir are corrupted.  Mounting the dataset via ZPL, one cannot rmdir update_log_dir, even if it is empty, because the size is set incorrectly and ZPL checks the size as part of the rmdir code.

So the issue here is not about link count, but rather about size.

This patch may well be the wrong approach.  Suggestions are welcome.
