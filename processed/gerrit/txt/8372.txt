LU-4243 mgs: use proper uuid in mgs_write_log_failnids()

This patch revert the changes to mgs_write_log_failnids() in the
commit d9d27cad, which introduced a regression of that when multiple
'--servicenode' or '--failnode' option specified, the uuid from
first option will be wrongly used for all other connections.

--- Message (Erich Focht, 2013-12-03 10:18:29.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

The patch changes the old behavior and contradicts the comment "We don't know the failover node name, so just use the first nid as the uuid".

The original way was: if a failnode has multiple NIDs, select the first one as the failnodeuuid, add all its NIDs to it (as possible connections).

The behavior in this patch is to set each NID of a failnode as separate failnodeuuid, and only add itself as connection.

If each node should have just one failnodeuuid (and that's what the variable name suggests), then this must be set in the inner while() if unset, and reset right after the inner while() block.
