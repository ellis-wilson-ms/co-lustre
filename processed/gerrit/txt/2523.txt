LU-56 libcfs: NUMA allocator and code cleanup

- Implementation of NUMA allocaters based on concept of CPU
  partition
- Add macros of NUMA allocators for both libcfs and OBD
- cleanup OBD allocaters
- A minor fix for llite, it's using GFP flag which should be
  CFS_ALLOC flag

Patch Set 10: I would prefer that you didn't submit this

(13 inline comments)

I have no serious objection to the patch itself, but there are a lot of changes going on at the same time (moving CFS_ALLOC_ZERO flag to the callsites, adding the CPT allocators, renaming all of the cfs_page_{alloc,free}() functions) and this introduces a lot of room for getting something slightly wrong (e.g. missing CFS_ALLOC_ZERO somewhere, or getting the CFS_ALLOC_* flags wrong).

Also, I suspect that with the huge numbers of files/lines changed that there will be conflicts with some other patch repeatedly, making it difficult to land after testing and inspections have completed.

It might make sense to split this patch into a few smaller parts (e.g. add CFS_ALLOC_ZERO to callers maybe in a few patches, change primitives to use zalloc(), add CPT functions, etc), so that it is more likely to be able to land without conflicts.
