LU-10199 lfs: Fix file name comparison in lfs_migrate

The sanity test 56xb revealed that the file name comparison between
the previously migrated files and the current file was incorrect.
This bug could cause a hard-linked file to get migrated more
than once.  The comparison now uses the absolute path to ensure
that an already migrated file is correctly detected.

The sanity test is reenabled now that the name comparison has
been corrected.
