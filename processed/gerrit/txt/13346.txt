LU-6112 ldlm: wakeup ldlm waiters when error happened

When n process use the same file, and share osc_lock for it,
there is situation when one process is sleeping at ldlm_lock_match,
second doing cl_lock_enqueue. Lock request was sent, and process is
killed by signal. The second process wakeup from cl_lock_state_wait()
detach osc lock from ldlm and fail with -EINPROGRESS. Client recive
completion and grant ldlm lock, but ldlm_handle_cp_callback could not
wakeup the first process because osc_ldlm_completion_ast() end with
ELDLM_NO_LOCK_DATA because no osc_lock is attached  to ldlm.
After timeout server evict client and client has error:
  ldlm_resource_complain()): refcount nonzero (1) after lock cleanup;

This patch adds wakeup at error handle path for ldlm waiters.
