LU-12616 obclass: fix MDS start/stop race

The MDS unload happen when type of MDT has no reference.
The MDT drop it during obd_cleanup. So race window located
between obd_cleanup and server_stop_servers.
Lustre can lost MDS obd_device during server_start_targets
between MDS checking and taking the type reference, if another MDT
stops.

The patch takes one more reference for a MDT type at
server_start_targets, and put it at server_stop_servers.

This patch adds sanity test 278. It reproduces the next race
   started cleanup of MDT01
   started cleanup of MDT00
   finished cleanup of MDT00
   started MDT00 mount, checked MDS exist
   finished cleanup of MDT01, and cleanup of MDS also
   asserted during MDT00 initialization

It probably isn't worth it now, but I think this patch would be easier to review if it separated the refactoring/cleanup from the functionality change.
You've refactored server_start_targets nicely - a clear improvement - but that obscures the important functional change.
I think the function change is:
 - in server_start_targets, call class_get_type() before dropping server_start_lock
 - in server_stop_serves, call class_put_type after checking the refcount
 - introduce lsi_server_started to ensure the class_put_type() is only called if class_get_type was called.

As well as that change and the refactoring, there is the testing code.
So maybe next time, something like this could be three patches?

Anyway, I think it looks good now.
