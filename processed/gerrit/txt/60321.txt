--- Commit Message ---
LU-19202 llite: set lock owner for locked inode

NFSD could lock the inodes by fh_lock()/fh_lock_nested() before
calling VFS interface and Lustre is unaware of it, and further
call path could try to lock the inode again.

This patch set the ll_inode_info::lli_inode_lock_owner on the
entrance of VFS interface if that's the case.

Fixes: 37646c74bf ("LU-16958 llite: call truncate_inode_pages() in inode lock")
