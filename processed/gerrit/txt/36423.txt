There is no clear relationship between this patch and problem description in LU-12214. I would suggest you create a dedicated Jira ticket and explain why it is a problem to concatenate security context of temporary mount point. It would be good to see a use case for that, and add a new test in sanity-selinux.sh.

Or, is it only to get rid of a build dependency? In this case it would need a test to show that the change does not break anything regarding SELinux support.

Also, please include the following test parameters, so that we make sure the change does not introduce any regression in SELinux support.

   Test-Parameters: clientselinux mdtcount=4 testlist=sanity,recovery-small,sanity-sec,sanity-selinux
