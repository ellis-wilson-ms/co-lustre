Patch Set 2:

While stopping the test case from failing is good, we need to find some other way to handle this case. Otherwise, using foreign layouts for HSM files will be broken if we don't have any chance to hook the read and restore the file for zero-length files. 

I think two options exist:
- set the size of a foreign file to "1" or sizeof(foreign xattr)
- save the real size of the object in the foreign layout and return that 
- find some other way to hook into the VFS so that the read is passed to llite
