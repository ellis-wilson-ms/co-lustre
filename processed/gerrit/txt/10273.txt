--- Commit Message ---
LU-5011 lustre: don't export kernel types to userspace

Remove libcfs/posix/posix-types.h. Add a new header libcfs/types.h
which defines namespace safe fixed-width types lu_intXX_t and
lu_uintXX_t for use in packaged headers like lustreapi.h and
lustre_user.h. Rewrite those headers to use the new types. Including
these headers will no longer define the __{s,u}XX types unless Lustre
is being built. Remove the autoconf tests for the presence of
__{s,u}XX, and for the long-longiness of the primitive type used to
define __u64.

I realize that is but one step on a longer path towards user-kernel interface nirvana, and I am at peace with the general approach. I would prefer, though, if the new type names were based on the linux (__u64) types rather than the user space (uint64_t) types.  This keeps it clear that these are kernel data structures, and their use in user space should general be limited to low level libraries.
