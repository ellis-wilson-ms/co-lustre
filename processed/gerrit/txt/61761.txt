LU-0000 lod: implement write intent for EC files

Implement Phase 2 of FLR state management for EC files:
handle write intent to transition from RDONLY to
WRITE_PENDING state and mark parity components as stale.

When an EC file in RDONLY state is written to, the system
now:
1. Sends a write intent RPC to the MDT
2. Transitions the file from RDONLY to WRITE_PENDING state
3. Marks parity components as STALE (they need resync)
4. Allows the write to proceed

Changes:
- Update lod_declare_update_rdonly() to handle EC files
  by delegating to lod_declare_update_plain()
- Update lod_declare_update_plain() to accept
  WRITE_PENDING state for EC files
- Update client-side lov_io_mirror_write_intent() to
  skip mirror logic for EC files (which have no mirrors)
- Add test_1a to verify write intent handling
