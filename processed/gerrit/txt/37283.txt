LU-11025 mdt: normalize temp XATTR buffer usage

including HSM/SOM/PFID/LOV/LMV, expand it into an union for
convenience.

--- Message (Andreas Dilger, 2020-03-12 23:08:18.000000000) ---
Patch Set 1: Verified-1

It looks like this patch is failing a *lot* of tests.

Do you think the space savings for the xattr buffers is worth the complexity/risk of sharing those buffers for multiple uses?  Since the maximum xattr size is 64KB (and this would only be used if xattrs that large are found in the filesystem), and the maximum MDS thread count is around 2048 (default max 1024) it is unlikely that this extra 64KB allocation per thread would be significant on the MDS (typically they have 96GB or 192GB of RAM these days).

Having separate allocated buffers in mdt_thread_info is still a benefit, because it avoids the alloc/free when processing every request.

--- Message (Lai Siyao, 2020-03-13 03:14:55.000000000) ---
Patch Set 1:

The main goal is not to share space among different XATTRs, but provide the default buffer for LMV which can contain 9 stripes, if the real stripe count is larger than 9, it will switch to use big_lmm to store LMV. The problem is big_lmm is not initialized by default, so in every place to read LMV it needs to allocate memory for it, which is clumsy.

I'll switch to not share space.
