--- Message (Andreas Dilger, 2025-04-22 09:44:44.000000000) ---
Patch Set 2: Code-Review+1

When the "lfs quota -a" feature was first added, we discussed implementing a better handler in the kernel for this.  Instead of the current "get all quotas into one long list, then iterate the list and return it in small chunks to userspace" the userspace API was changed to allow the kernel implementation to become "get quotas in chunks as userspace requests them".

However, that patch to clean up the internal handling was never implemented.  I'm not *against* this fix landing, but I would prefer that it be followed by another patch that is cleaning up the kernel-side iteration to do it properly.
