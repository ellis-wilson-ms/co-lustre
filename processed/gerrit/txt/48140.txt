LU-16035 kfilnd: Fix setting transaction nob

The kfilnd transaction number of bytes is suppose to be the LNet
payload byte count. But, the tn_nob field was not being set for all
operations in addition to being set incorrectly. The end result of
this bug is the mlength being reported in LNet reply events was
incorrect. This prevented Lustre from running.

Correctly set tn_nob to the LNet payload size. In addition, report a
mlength of zero for LNet reply events if an error has occurred during
the transaction.
