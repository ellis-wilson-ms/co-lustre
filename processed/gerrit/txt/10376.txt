--- Commit Message ---
LU-4416 osd-ldiskfs: limit trans size to match reservations

Linux commit 8f7d89f36 (jbd2: transaction reservation support)
shrunk the limit of a transaction size to be
j_max_transaction_buffers / 2 to accomodate the reservations.
This can have a perforamce impact so we limit this change to
3.10+ kernels. The same commit that changed this is the same
one that changed the number of arguments to ext4_journal_start
function so we use that test for this. This patch adjusts the
limit in osd_trans_start to match depending on the kernel
version.

--- Message (Andreas Dilger, 2014-07-18 07:36:19.000000000) ---
Patch Set 7: Code-Review-1

There are two concerns I have with this patch:
- the smaller maximum transaction size is only applicable to kernel 3.10 and later, but is applied to all systems
- this has the potential to hurt performance for Lustre because with many active threads they can reserve a large number of blocks and run into the transaction size limit

On smaller test systems (with correspondingly smaller journals) and very busy large systems we have run into the previous transaction limit.

I think this patch should be changed to only affect the relevant kernels (i.e. configure check for  jbd2_journal_start_reserved() or similar).

That would give more time to see if this impacts performance at scale on new systems, and would not affect older systems that were typically formatted with a smaller journal size.
