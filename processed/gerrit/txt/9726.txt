> Fan Yong, would it be possible to allocate oti_idx_it for LFSCK purpose?

The LFSCK trace file uses IAM as the index file, which will uses oti_idx_it for lookup. During the LFSCK namespace traverse, it may operate on the trace file. So the oti_it_dentry and oti_idx_it may be used by the same thread at the same time, so they cannot share the same union.

On the other hand, IAM lookup will be triggered by any OI lookup, that is a critical path. It is better to pre-allocate it on the thread_info.
