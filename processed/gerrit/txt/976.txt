LU-432 file_operations->fsync no longer get 3 arguments

Starting from kernel 2.6.35 file_operations->fsync needs only 2 arguments,
leaving the dentry one out. This patch checks whether 3 arguments are
supported and if not uses the new 2 argument calls.
