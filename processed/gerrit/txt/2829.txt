LU-1346 libcfs: replace libcfs wrappers with kernel API

The libcfs kernel portability library had wrappers for many low-level
kernel functions (locking, bit operations, etc) that were simple
wrappers around Linux kernel functions.  This provides no value for
Linux clients and clients for other kernels are not under development.

Remove the cfs_ prefix from these simple wrapper functions.  For other
kernels, they will need to use the Linux kernel API for portability.

Affected primitives:
spinlock_t, spin_lock_init, spin_lock, spin_unlock, spin_lock_bh,

--- Message (Andreas Dilger, 2012-10-12 01:50:54.000000000) ---
Patch Set 2: I would prefer that you didn't submit this

(6 inline comments)

I would expect the libcfs_cleanup.sed script to be split into much smaller sections, and built up incrementally, along with a patch that changes all of the lines of the specific type that is being added (e.g. spinlocks, semaphores, mutex, etc).

I'm not quite sure what the best way to get such large patches landed would be, but I don't necessarily think it makes sense to create such a huge sed script and automatically apply it to the tree at build time, without it being attached to a patch that includes all of the resulting changes.  Otherwise, the next time someone updates and build that tree it will silently converting tens of thousands of lines of code in their tree.

Instead, I think it makes sense that the resulting changes to the tree be included with each change to the patch.  However, it is only worthwhile to do this right before each patch is ready to land (to minimize the conflicts), so it doesn't make sense to submit a large patch series, only one at a time.
