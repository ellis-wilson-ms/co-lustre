--- Commit Message ---
LU-20191 efalnd: fix module refcount leak on base startup

The base startup takes a module reference but the error paths never
release it, leaking the reference. This makes the module unremovable
after a failed base startup. Add a new error label that releases the
module reference on all error paths where it was successfully taken.
