Patch Set 6:

I think there is a problem to drop the mirror ID from the file completely if the second-to-last mirror is removed.  The mirror ID also makes up part of the component ID, and changing that while the file is referenced by any external application would make it hard to identify the remaining component. 

For example, if there is a user working on a file with "mirror #1" and "mirror #2", and then removes "mirror #1" from the file. If they want to add a new mirror there may be confusion about which mirror to use.
