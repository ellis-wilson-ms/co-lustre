--- Commit Message ---
LU-8130 ptlrpc: convert conn_hash to rhashtable

Linux has a resizeable hashtable implementation in lib,
so we should use that instead of having one in libcfs.

This patch converts the ptlrpc conn_hash to use rhashtable.
In the process we gain lockless lookup.

As connections are never deleted until the hash table is destroyed,
there is no need to count the reference in the hash table.  There
is also no need to enable automatic_shrinking.

--- Message (Andreas Dilger, 2018-11-08 08:10:39.000000000) ---
Patch Set 1: Code-Review-1

If we are having problems inserting into the rhashtable on Ihara's test system, with only 10 client nodes, how is this ever going to work when 10,000 clients are all trying to connect simultaneously?

Given that we are trying to make the 2.12 release in the next few weeks and we don't have resources to test this at extreme scales, and the rhashtable support is not critical for anything except code cleanup, I think I would prefer to keep the existing libcfs hash code for 2.12. That will give us more time to fix the issues with rhashtable, maybe fixing some bugs in the kernel code, and better test it at scale.  

I'm going to mark this -1 for now, to get it off the radar for 2.12, and we can look at it again when 2.13 opens. I'll move ahead with the revert of the rhashtable usage patch.

Patch Set 1:

Since this is new stuff it makes sense to delay this landing until 2.13. As for performance I seriously doubt libcfs hash can out perform rhashtable. The rhashtable is used for the net layer dealing with millions of messages per second and its lockless which libcfs is not. I do expect the configuration of the hash table is not optimal so I need to look into that.
