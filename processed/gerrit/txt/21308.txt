--- Commit Message ---
LU-8397 mgc: add comma-separated nids for primary MGS

In lustre_start_mgc(), if the primary MGS has multiple nids separated by
commas, then only the first nid is added, and the other nids are not added.
This patch fixes the above issue by adding the nids until the end of the
primary MGS nid list or hitting the first failover MGS nid.

--- Message (Zhenyu Xu, 2017-02-06 05:39:13.000000000) ---
Patch Set 3:

(1 comment)

',' is used to define multiple nids of a single mgs node, while ':' is used to separate different mgs node. So the ':' break in this patch would add multiple nids to associate with the primary mgs node's uuid; while the old code only associate the first one nid with the primary mgs, but add the remaining nids of the primary mgs to the failover mgs node in error.
