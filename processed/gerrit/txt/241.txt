I have two questions.
First, is not max_bytes a per-file property on ext3/4? Depends on extent use?

Second question, I assume ext3-based ldiskfs still retains the 2T limit?
If so then we have data-corruption looming for the case where somebody upgrades ext3 ldiskfs to ext4 ldiskfs I think, since old limit was in fact 2T-4k which means once the limit is lifted, some very large files at the edge of the old limit would get some extra zeroes inserted close to the end?

I was thinking of the case where there the object extend all the way to the current limit (2T-4k).
For simplicity, if we assume there are 2 objects and and stripe size is 64k, that means first object will extend all the way to 2T-4k, but that means last chunk would only be 60k.
