LU-6602 obdclass: variable llog chunk size

Do not use fix LLOG_CHUNK_SIZE (8192 bytes), and
it will get the llog_chunk_size from llog_log_hdr.
Accordingly llog header will be variable too, so
we can enlarge the bitmap in the header, then
have more records in each llog file.

--- Message (Andreas Dilger, 2015-05-29 07:44:42.000000000) ---
Patch Set 4:

The latest version of this patch does not change the actual llog record size from 8KB, only changes the code so that we might do that in the future of needed. We thought it would be necessary to change the log record size to fix the DNE2 atripe count limit, but then we worked out a different method of storing the records in the llog and didn't need to increase the limit, since it would have caused interoperability problems as you wrote, and still wouldn't be able to be large enough. . 

At least with this patch the limit can be increased in the future much more easily. The llog disk format was always planned to allow different record sizes, but the code didn't handle this case very well.
