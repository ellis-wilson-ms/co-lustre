LU-2934 lnet: Add LNet Router Priority parameter

This change adds a priority parameter to the route module settings.
This paramter can be >= 0. Like hops, the lower the prioirty number
the higher the priority.  So lower numbered priorities will be
selected over higher numbers.

--- Message (Doug Oucharek, 2013-04-11 17:08:31.000000000) ---
Patch Set 4:

Andreas: The one change I am aware of is how we interpret the priority number.  In Fujitsu's implementation, a priority of 0 is selected over a priority of 1 (lower number is a higher priority).  I believed that this might be confusing as users would expect the larger priority number would be chosen over the lower priority number.

--- Message (Andreas Dilger, 2013-04-12 00:51:18.000000000) ---
Patch Set 4: I would prefer that you didn't submit this

(2 inline comments)

I agree that priority "0" is a bit ambiguous, but in some cases (Bugzilla and Jira, for instance), priority "1" is ahead of priority "2" for bugs (first priority, second priority, etc).  In other cases, having a "higher priority" is literally a higher number.

I don't know that there is a clear precedence either way in the other Lustre code.  In LNET, a higher hop count for a route means it will not be selected, so I don't think it is unreasonable to have the priority behave the same way.  I think as long as this is documented clearly, IMHO it doesn't matter which way it goes so there is little benefit to breaking compatibility for this.

Is there a reason why the priority only has values 0 or 1?  Why not allow arbitrary values for the priority?  Given that the code is comparing integers, there isn't necessarily any reason to restrict the values to 0 and 1.

--- Message (Doug Oucharek, 2013-04-12 22:20:55.000000000) ---
Patch Set 4: (2 inline comments)

I will change the interpretation of the priority number to match Fujitsu's implementation.  I will also remove the bounds checking on the priority number so it can be any value (and is treated it as unsigned).

Documentation for this feature will be updated.
