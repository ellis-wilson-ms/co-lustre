LU-20026 lod: delete reference on update logs when destroyed

Add a missing reference deletion to update logs, so that the inode
link count drops to 0 when they are destroyed. This issue was found by
running e2fsck on the MDTs, which caused the inodes to be considered
unattached, as the names in update_log_dir had already been removed:

 # e2fsck -E no_optimize_extents -f -v -t -t -n <device node>
...
Pass 4: Checking reference counts
Unattached inode 141
Connect to /lost+found? no

Unattached inode 70665135
Connect to /lost+found? no
...
