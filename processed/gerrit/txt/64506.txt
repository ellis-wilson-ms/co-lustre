--- Commit Message ---
LU-18584 tests: speed up conf-sanity test_106

Running conf-sanity.sh test_106 takes a long time to create and
delete many files to fill up the changelog records.

The created files do not need to persist for any length of time,
so change createmany to also unlink the files immediately.  That
removes any requirements for the size of the MDT and OST to hold
the files, and also allows the creates to be run in parallel to
speed up the test.

This was previously timing out in Janitor testing after ~7300s,
and now can finish in 3200s (by necessity only on a single MDT).

Test-Parameters: trivial testlist=conf-sanity env=ONLY=106
