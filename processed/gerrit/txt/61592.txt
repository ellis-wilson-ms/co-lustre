LU-19386 lnet: use ip2nets handler for net sequence

Modify handle_ip2nets_sequence() (renamed to
handle_net_config_sequence()) to handle both the "ip2nets" and "net"
yaml sequences. This allows LNet to understand a net sequence where
more than one interface is specified.
i.e.

net:
-     net type: <net>
      local NI(s):
      -     interfaces:
                  0: <intf1>
                  1: <intf2>
                  ...
                  N: <intfN>

- Add validation to ensure ip2nets-specific fields like "ip-range"
  are only used with ip2nets sequences
- Add validation to ensure net-specific fields like "local NI(s)"
  are only used with net sequences
- Fix memory leak by properly freeing global_cpts in cleanup paths
- Remove cfs_expr_list_free(cpts) call from
  lnet_yaml_emit_cpt_sequence(). Callers are responsible for freeing
  this memory. This prevents some double-free and use-after-free bugs.
- Change "Unrecognizable key" in handle_lnd/cmn_tunable() to a
  warning. This aligns with how unknown keys are ignored by the
  kernel.
- Logic was missing from yaml_lnet_config_ni() to check whether
  struct lnet_dlc_intf_descr::cpt_expr was set, and to use it in place
  of the global CPT configuration. This is now done.
