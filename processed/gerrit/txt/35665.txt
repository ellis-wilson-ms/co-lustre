sanity test 802a and "rdonly_dev" mount option were introduced by patch https://review.whamcloud.com/24267. In addition, snapshot_mount_target() in lustre/utils/lsnapshot.c used "rdonly_dev" option to mount server target.

After removing the dev_read_only-*.patch, osd_mount() in lustre/osd-ldiskfs/osd_handler.c will return -EOPNOTSUPP:

        if (lmd_flags & LMD_FLG_DEV_RDONLY) {
                if (priv_dev_set_rdonly) {
                        // ......
                } else {
                        LCONSOLE_WARN("%s: not support dev_rdonly on this device",
                                      name);

                        GOTO(out_mnt, rc = -EOPNOTSUPP);
                }
        }

I'll change sanity test 802a to work only with ZFS.
