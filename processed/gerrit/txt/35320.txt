--- Message (Andreas Dilger, 2019-06-26 08:32:25.000000000) ---
Patch Set 2: Code-Review-1

(1 comment)

Thanks for the patch. Rather than using llapi_search_mounts(), which is inefficient, it would be better to use llapi_getname() instead, which only has one ioctl() to the kernel rather than several read calls and complex parsing of /etc/fstab. Bonus points if you add a man page for that function.

Please also add a sanity test case for this usage to verify that it is working correctly and does not break in the future.

--- Message (Dominique Martinet, 2019-06-26 08:53:31.000000000) ---
Patch Set 2:

(1 comment)

ok, I just looked at other similar functions that used llapi_search_mounts ; will make the switch.

That said cost-wise I'm not sure which really is more expensive - the ioctl needs to open the fs root directory, that means taking a lock if it's not already present and a few RPCs with the first MDS ; llapi_search_mount on the other hand is more complex userspace-wise but is 100% node-local so will probably end up being faster.

Anyway, pool_list is not a fast path operation, I don't think it really matters and will change that and add a test.
