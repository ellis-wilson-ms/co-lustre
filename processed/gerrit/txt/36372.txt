LU-1538 tests: standardize test script init – misc tests

Standardize the initial Lustre test script initialization for
clarity and consistency test suites are that are not normally
run during patch and branch testing.

The LUSTRE path is already normalized in init_test_env(), so this
doesn't need to be done in the caller.  Use $(...) subshells instead
of `...` in the affected lines.  Remove NAME, SRCDIR, PATH, MULTIOP,
SETUP, CLEANUP, CHECKSTAT, TMP, SAVE_PWD, variable initialization,
since it is already done in init_test_env() or not needed in the test
scripts.

Remove MACHINEFILE from dom-performance because it is now
included init_test_env().

Move all definitions of ALWAYS_EXCEPT and SLOW to after
init_test_env() and init_logging() and call build_test_filter()
immediately after the ALWAYS_EXCEPT and SLOW definitions.

Test-Parameters: trivial envdefinitions=SLOW=yes,ENABLE_QUOTA=yes \
clientcount=2 osscount=1 ostcount=7 mdscount=1 mdtcount=1 \
austeroptions=-R testlist=lfsck-performance

Test-Parameters: envdefinitions=SLOW=yes,ENABLE_QUOTA=yes \
clientcount=2 osscount=1 ostcount=7 mdscount=1 mdtcount=1 \
austeroptions=-R testlist=sgpdd-survey

Test-Parameters: envdefinitions=SLOW=yes,ENABLE_QUOTA=yes \
clientcount=2 osscount=1 ostcount=7 mdscount=1 mdtcount=1 \
austeroptions=-R clientselinux testlist=sanity-selinux

--- Message (Andreas Dilger, 2020-08-26 17:50:37.000000000) ---
Patch Set 11:

(4 comments)

There are a few places that lose the ability to set ALWAYS_EXCEPT from the caller, but the majority of scripts never allowed that. 

That said, it is obviously still possible to pass in a list of skipped tests via $EXCEPT $TESTNAME_EXCEPT, which is probably better in the long run to avoid ambiguity. I only point this out is case some other test script is depending on this behavior.

--- Message (James Nunez, 2020-08-26 22:34:35.000000000) ---
Patch Set 11:

(1 comment)

In other test scripts, we overwrite the ALWAYS_EXCEPT list at the start of the script and use “&<scriptname>_EXCEPT” as a way to let users not run tests for a specific script. For example, sanity.sh looks like

  41 ALWAYS_EXCEPT="$SANITY_EXCEPT "
  42 # bug number for skipped test: LU-9693 LU-6493 LU-9693
  43 ALWAYS_EXCEPT+="               42a     42b     42c "
  44 # bug number:    LU-8411 LU-9054 LU-13314
  45 ALWAYS_EXCEPT+=" 407     312     56ob"

I agree that this is changing the behavior of these test scripts. Yet, since we are standardizing the test scripts in this series of patches, maybe this is the appropriate place to standardize how we use/set ALWAYS_EXCEPT. 

If you feel differently, I'm willing to return this functionality to these test scripts.
