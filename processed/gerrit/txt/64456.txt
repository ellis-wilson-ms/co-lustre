LU-19874 misc: update module_init calls

... to use late_initcall() for LNet and late_initcall_sync()
for services dependent on LNet. Both LNet and Lustre need
networking to be available. So we use late_initcall_*().
And LNet must be started for Lustre - hence late_initcall_sync()
is used for Lustre to ensure that the LNet initialization has
been completed.

When Lustre is build as a module, these all evaluate to
module_init() regardless. So nothing changes in the typical
case.
