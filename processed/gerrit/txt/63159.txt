## Fix Summary

**Problem:** 
The compile error occurred because `__func__` is a `const char*` predefined identifier in C99, but it was being assigned to a `char *` array. The compiler flag `-Werror=discarded-qualifiers` treats discarding the `const` qualifier as an error.

**Root Cause:**
In `mount_lustre.c` line 429, the code had:
```c
char *targv[2] = { progname, __func__ };
```

Both `progname` and `__func__` are const strings, but they were being assigned to a non-const pointer array.

**Fix Applied:**

1. **lustre/utils/mount_lustre.c (line 429):** Changed `char *targv[2]` to `const char *targv[2]` to properly handle const string literals.

2. **lustre/utils/mount_utils.h (line 85):** Changed the struct member `char **mo_argv` to `const char **mo_argv` to accept const string pointers, since `mo_argv` is only used for reading (in fprintf calls).

3. **lustre/utils/mkfs_lustre.c (line 1094):** Added an explicit cast `(const char **)argv` when assigning `argv` to `mo_argv` to ensure type compatibility.

**Rationale:**
The `mo_argv` field is only used for logging purposes (reading and printing arguments), never for modification. Making it `const char **` is semantically correct and prevents the const qualifier discard warning. The explicit cast in `mkfs_lustre.c` is safe because the argv values are never modified through `mo_argv`.
