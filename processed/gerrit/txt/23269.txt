LU-9019 obd: remove s2dhms time function

Lustre created the s2dhms function to report a time
lapse in a days:hour:minute:seconds format. This is
over kill so remove s2dhms and just report the number
of seconds that has lapse.

I wouldn't be against removing s2dhms() either, since it isnt the kernel's job to convert the time format.  It was originally added over my objections as a way to be more "user friendly", but it isn't clear this is more important than any other timestamp we print:

 client# lctl get_param mdc.*.timeouts
 mdc.myth-MDT0000-mdc-ffff880075147800.timeouts=
 last reply : 1477473133, 0d0h00m06s ago
 network    : cur   1  worst   2 (at 1476410241, 12d7h14m58s ago)   1   1   1   1 
 portal 12  : cur   1  worst  31 (at 1476873635, 6d22h31m44s ago)   1   1   1   1 
 portal 30  : cur   1  worst   1 (at 1476332894, 13d4h44m05s ago)   1   0   0   0 
 portal 23  : cur   1  worst  31 (at 1476410241, 12d7h14m58s ago)   1   0   0   0 
 portal 17  : cur   1  worst  31 (at 1476873598, 6d22h32m21s ago)   1   0   0   0 

Also, it is easy to convert the seconds value into a real time if needed:

 client# date --date=@1476410241
 Thu Oct 13 19:57:21 MDT 2016
