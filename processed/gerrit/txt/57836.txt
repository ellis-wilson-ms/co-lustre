LU-13335 utils: allow using uid and gid for prjid

In order to avoid overloading the /etc/prjid(5) file with duplicate
entries for users and group there is now an opt-in by specifying
a username or group name in place of a project name like:
    'lfs project -p u:quota_usr /mnt/lustre/tfile'
    'lfs project -p g:quota_grp /mnt/lustre/tfile'

Updated sanity.sh test 190b.
Also now shows projname for lfs quota instead of prjid if available.

Modified uid2name and gid2name to use a buffer use getpwuid_r and
getgrgid to avoid unsafe calls with multi-threading.
