LU-18278 gss: link user keyring to thread keyring

Regular user keys used for GSS contexts are linked to the user
keyring, which is itself linked to the session keyring. This makes
this implemenation fragile to issues with the session keyring, such as
when it is revoked, unlinked, or unavailable. We depend on the kernel
implementation of request_key for that matter, so we need to implement
workarounds.
We choose to resort to the 'thread keyring' trick used elsewhere in
the kernel. This means we make the thread keyring point to the user
keyring when we are dealing with regular user keys. Because of the
implementation of request_key() and search_cred_keyrings_rcu() in the
kernel, we know this is going to be the first keyring being searched.

sanity-krb5 test_13 is added to check this is working as expected.
