--- Commit Message ---
LU-6391 llite: Add client mount opt to ignore suppress_pings

When Lustre servers enable 'suppress_pings', all clients will stop
pinging. However, some clients may not have external mechanism
to notify Lustre servers for node death and therefore need to
preserve the Lustre ping.

This patch provides a mount option 'always_ping' so that the
client will not stop pinging even if the server has enabled
'suppress_pings'.

It might make sense to allow a "ping=no" mount option so that pings can be suppressed on specific clients, rather than setting the global "suppress_pings" option.

--- Message (Wally Wang, 2015-12-22 18:53:33.000000000) ---
Patch Set 4:

If I understand correctly that server would assume client ping if there is no OBD_CONNECT_PINGLESS support for backward compatibility. That is where we take advantage of it with the always_ping implementation that we just negate the flag before connecting to the server.  

If we implement ping=no, we would need another connection flag to tell the server not to check ping and evict client regardless suppress_pings or not. Is this what we want? If so the options will be:

ping=yes (same as always-ping implemented in the patch so far)
ping=auto (client ping unless server suppress_pings, the existing code)
ping=no (client does not ping, regardless server suppress_pings or not)
