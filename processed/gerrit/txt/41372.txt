LU-14391 lnet: add config file support

Large network requires many config line for routes, but the module
param is limited to 1024 bytes. This patch resolves this limitation.
The routes and ip2nets module parameters accept absolute path names,
the old syntax is supported as well.

Example:
routes="/etc/lnet/routes.conf"

--- Message (Andreas Dilger, 2021-01-31 19:56:35.000000000) ---
Patch Set 1: Code-Review-1

This seems like a duplicate of patch https://review.whamcloud.com/2102 "LU-1071 lnet: add config file support for routes and ip2nets params" which was previously abandoned for a few different reasons. 

Instead of adding a new config file to emulate the module parameters, why not use the existing lnetctl YAML config file to configure the routes after startup?  That avoids adding any new lines to the code. If there is a gap in the lnetctl functionality it would be good to understand that and address it rather than going back to modprobe as a mechanism for configuration.
