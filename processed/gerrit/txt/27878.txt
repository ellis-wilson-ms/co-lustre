LU-9140 nrs: measure the runtime of dd directly

This patch changes the way to measure the runtime of "dd". Instead
of parsing the output of "dd", we use date command to calculate
the runtime of dd directly, avoiding the parsing failure caused
by changed output format of "dd".

(3 comments)

Just wondering why we are multiplying by the number of CPU partitions.

--- Message (Qian Yingjin, 2017-10-17 02:11:33.000000000) ---
Patch Set 7:

The reason is: In the current NRS framwork, each CPU partition has a seperate NRS scheduler with its own reqeust queue. When the server is configed with NRS TBF, each NRS TBF scheduler can only limit the RPC rate R in its own request queue. So when there are N CPU partition on the server, the upper limited RPC rate should be N * R.
