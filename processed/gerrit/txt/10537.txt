--- Commit Message ---
LU-4856 misc: Reduce exposure to overflow on page counters.

When the number of an object in use or circulation is tied to memory
size of the system, very large memory systems can overflow 32 bit
counters.  This patch addresses overflow on page counters in the osc LRU
and obd accounting.

Patch Set 3: Code-Review-1

(13 comments)

Did you actually see problems with the LDLM code, or is that only a result of the current defaults being a function of the RAM size?  It doesn't seem likely that there are any problems in this area, and given that an ldlm_lock + ldlm_resource are 906 bytes each it means with 2B locks this is consuming close to 2TB of RAM just for the lock structures even without any of the data.  Most of the LDLM API changes are only for lock cancellation, which can't grow so large, so it would probably be better to remove this part of the changes from the patch.
