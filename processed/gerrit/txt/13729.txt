Alex, my intention is to get 0.6.4 tagged and out the door as soon as possible (we're down to a dozen issue or so).  Once that's done I'd like to bring both the large block and ABD changes in to an 0.7.0 branch.  Both of these patches are fully functional but haven't seen as much real world run time as I'd like.  I want to get them in very early so we can shake out any issues which are introduced before the next tag.

On a related note it occurred to me the interfaces you need are almost exactly the same interfaces we need to support Direct IO in the ZPL.  For Direct IO we a get a 'struct iov_iter' from the VFS which maps pages in an applications address space for read/write.  If we added a dmu_write_iov()/dmu_read_iov() for Direct IO you could use that stable interface.  This way Lustre shouldn't be destabilized by any of the memory management work.  The interface itself would also be rigorously tested through the ZPL with xfstests.
