LU-20158 mdc: fix NULL ld_obd deref on device_alloc failure

In mdc_device_alloc() and osc_device_alloc(), d->ld_obd is not set
before mdc_setup()/osc_setup() is called. The framework (obd_setup()
in obd_class.h) only sets dev->ld_obd after ldto_device_alloc()
returns successfully. If setup fails (e.g. client specifies
network=kfi1 but the target NID is @o2ib, causing
client_import_add_conn() to fail with -ENOENT), the error path calls
mdc_device_free()/osc_device_free() with d->ld_obd == NULL.

mdc_device_free() immediately dereferences lu->ld_obd:

  struct obd_device *obd = lu->ld_obd;  /* NULL */
  LASSERT(cli->cl_mod_rpcs_in_flight == 0);  /* fault at 0x121a */

This causes a NULL pointer dereference (CR2=0x121a =
offsetof(struct obd_device, u.cli.cl_mod_rpcs_in_flight)).

Fix by setting d->ld_obd = obd immediately after obd->obd_lu_dev = d,
consistent with server-side device alloc functions (lod, ofd, osp).
