This looks like it is returning back to the original LU-16861 error in Janitor testing.  This is coming from commands in the obdfilter-survey script itself:

 # inspired from the lustre_build_version routine in test-framework.sh
get_lustre_version() {
        local host=${1:-${unique_hosts[0]}}
        local ver

        ver=$(remote_shell $host "$lctl get_param -n version 2>/dev/null ||
                                $lctl lustre_build_version 2>/dev/null ||
                                $lctl --version 2>/dev/null | cut -d' ' -f2")
        local lver=$(egrep -i "lustre: |version: " <<<$ver | head -n 1)
        [ -n "$lver" ] && ver="$lver"

        sed -e 's/.*: //' -e 's/^v//' -e 's/-.*//' -e 's/_/./g' <<<$ver |
                cut -d. -f1-4
 }

It may be related to patch https://review.whamcloud.com/53083 "LU-16827 obdfilter: Fix 'emfperf obdfilter-survey' error" causing problems with the "$lctl" setting:

 if [ -z "$lustre_root" ]; then       
        lctl=${LCTL:-lctl}
 else                                                                       
        lctl=${lustre_root}/utils/lctl                                   
 fi

I don't think we really need this version check for 2.3 servers anymore, so that could be removed.  However, if there is something wrong with the $lctl variable then the test would just fail later in the script.

Let me update this patch to see if I can see what is wrong.
