--- Commit Message ---
LU-17342 o2ib: build without Module.symvers

When building against an external kernel tree, the
configure script fails if there isn't a Module.symvers
available. This prevents us from using the
'modules_prepare' make target on the kernel tree.
ko2iblnd.ko can be build even without Module.symvers.
Hence, downgrade this message from an error to a
warning.

Also, don't fail if ko2iblnd can't be built. Just
emit a warning.

Yes I added a patch to enable /etc/sysconfig/lustre to pass flags to dkms configure.
The only problem is there is a 'guess' phase the tries to determine what LNDs should build and an install phase that *expects* to guess to have been correct.

The optional LNDs are for different interconnect hardware (mofed, OPA, slingshot, ...)

We could go the other direction and have DKMS to pass a flag to configure as the inverse of adding a flag for modules_prepare, that should be pretty straight forward.

Anyway not a reason to block this patch, imo.
