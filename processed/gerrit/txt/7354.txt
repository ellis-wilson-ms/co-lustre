LU-137 mount: Ioctl passthrough mechanism for OST/MDT mntpts

This patch implements an interface for sending IO Control (ioctl)
commands from userspace through the Lustre mount point to the
underlying ldiskfs filesystem to allow execution of filesystem-wide
ioctl() commands, such as resize.

--- Message (Andreas Dilger, 2013-09-25 05:39:08.000000000) ---
Patch Set 4:

Could you please clarify why lsattr/chattr do not work on the Lustre mountpoint?  Looking at the code it appears they are doing the obvious thing:

        fd = open (name, OPEN_FLAGS);
        r = ioctl(fd, EXT2_IOC_GETFLAGS, &f);
or
        r = ioctl(fd, EXT2_IOC_SETFLAGS, &f);

like your test program is.  The only difference I can see is that stat() is called on the device beforehand.  Maybe we need to add a basic "getattr" method for the mountpoint?

In any case, lsattr/chattr are only intended as a starting point to get the code into basic working order.  The real goal is to get resize2fs to be able to do online resizing.
