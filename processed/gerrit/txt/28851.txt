--- Commit Message ---
LU-9943 lnet: fix WR accounting for the FastReg mode

FastReg memory model need a two additional WR's per transfer.
First one for memory window registration, second one for the
memory window invalidate.

Lack accounting for these WR's may caused a bug
    mlx5_warn:mlx5_0:begin_wqe:4085:(pid 9590): work queue overflow
