--- Commit Message ---
LU-17856 tests: fix sanity/56xe stripe count check

The number of stripes in the last component is not a critical part of
test_56xe or test_56wa, they mainly check that PFL migration works.
Create the PFL file with fewer stripes in last component to avoid
occasional test failure if the MDS does not allocates OSTCOUNT stripes
due to load or an OST being temporarily unavailable for allocation.

Add test_56xe and test_56wa to sanity.ex so they are skipped during
interop tests with older clients to avoid spurious test failures.
Change tests over from deprecated lfs_migrate over to 'lfs migrate'.

Also add test_0d to sanity.ex so interop testing can pass.
