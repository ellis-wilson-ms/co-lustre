Since the checkpatch.pl script can used from the git commit hooks for any patch (contrib/git-hooks/prepare-commit-msg), this will complain about non-LU tickets for patches on any other branch, so I don't think this can land as-is.

It would be OK to have this check conditional for patches under fs/lustre-release, or possibly better to do this check in gerrit-checkpatch.pl so that we don't diverge checkpatch.pl too much from the upstream kernel version.
