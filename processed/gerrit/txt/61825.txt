--- Commit Message ---
LU-19476 lnet: Set local NI status based on health value

Automatically update the ni_status field of struct lnet_ni based
on health value transitions. When the health value transitions to 0,
set ni_status to LNET_NI_STATUS_DOWN. When the health value
transitions to LNET_MAX_HEALTH_VALUE (1000), set ni_status to
LNET_NI_STATUS_UP.

This provides more accurate NI status reporting that reflects the
actual health state of the interface. Previously, NI status was
only updated based on routing aliveness checks or manual interface
state changes. Now the status automatically reflects health
degradation from network failures.

New wrappers around NI health decrement/increment are added.
Wrappers used in every place that would previously increment/decrement
NI health.
- lnet_handle_local_failure(): Check if health reaches 0 after
  decrementing and update status to DOWN
- lnet_health_check(): Check if health reaches maximum after
  incrementing and update status to UP
- lnet_handle_recovery_reply(): Check if health reaches maximum
  after successful recovery and update status to UP

lnet_ni_set_healthv() modified to update status when manually setting
health to boundary values (0 or 1000).

Also modified lnet_ni_get_status_locked() to return ni_status
regardless of routing state, allowing health-based status to be
reported even when routing is disabled.

Removed the handle_local_health flag logic in lnet_health_check()
since local health should always be decremented for failures,
regardless of interface count. This ensures health and status
accurately reflect interface state. LNet health test cases in
sanity-lnet.sh updated to reflect this change.

Three new test cases added to sanity-lnet.sh:
1. Verify NI status changes with manual health value transitions.
2. Verify NI status DOWN when health reaches 0 after network failures,
   and NI status UP when health recovers to 1000.
3. Verify local NI status propagates to peer routers
