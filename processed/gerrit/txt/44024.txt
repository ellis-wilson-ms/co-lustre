--- Commit Message ---
LU-14677 sec: no encryption key migrate/extend/resync/split

Allow some layout operations on encrypted files, even when the
encryption key is not available:
- lfs migrate
- lfs mirror extend
- lfs mirror resync
- lfs mirror verify
- lfs mirror split
We allow these access patterns to applications that know what they are
doing, by using the specific flag O_FILE_ENC and O_DIRECT.

Also add sanity-sec test_59a,b,c to exercise these access patterns.

--- Message (Andreas Dilger, 2021-08-01 16:06:12.000000000) ---
Patch Set 14:

Files can be migrated by the file owner or root. Allowing or preventing migration by requiring the encryption key does not improve security, and would probably weaken it.   If admin tools that are managing data (eg. automatic resync of stale files, or tools that balance storage between OSTs) would need to have an key that could decrypt *every* file in the filesystem, this would be a huge security risk. 

The premise of all modern encryption is that the data should be secure even if the cyohertext is accessible, and the encryption algorithm is known. The point of on-disk encryption in Lustre is that the data is safe even against someone that has direct access to the OSTs (at least to the extent possible), so having a key available on the servers to decrypt the file, only in order to move the data around, is contrary to this goal.

The nonce is supposed to be unique within a communication, so an identical file the same nonce is just an extra copy of a message, which is obviously fine...

But if the file can be changed *separately*, as the mirrors can, then it's not quite right to re-use the nonce.  The implications are, as ever, complex, and probably a bit beyond me, but...

In general, a nonce is used to prevent replay attacks by guaranteeing the same data is not encrypted the same way in different messages.  In the case of mirrors, that's not a problem since we advertise whether or not they have the same contents.

Ah, but in the case of mirror split it is - because they are now different files and therefore must have different nonces.
