LU-12187 lfs: add EC setstripe support with k+m notation

Add support for Erasure Coding (EC) layouts in lfs
setstripe using simple "k+m" notation where k is data
stripe count and 'm' is parity stripe count
(must be <= min('k', 4)) that make up the RAID Set
geometry.

When --ec is specified, an EC parity mirror is
automatically created that matches the data mirror's
extent structure. The --ec option can appear anywhere
in the command line.

EC Inheritance Rules:
- Single --ec applies to all components in the parity
  mirror
- Multiple --ec values inherit forward: each applies to
  its component and subsequent components until replaced
  by another --ec
- The first --ec also applies to any earlier components
  (backward inheritance)

EC Stripe Count Adjustment:
- The EC parity data count (k) is capped at the data
  component's stripe count (-c). If the component's
  stripe count is less than k, the EC data count is
  automatically reduced to match.
- The component's -c value takes precedence; the --ec
  data count only sets the upper bound for parity.
- Example: -E 128M -c 4 -E -1 -c 16 --ec 8+2
  creates EC 4+2 for the first component (stripe_count
  4 < k=8, so k is reduced) and EC 8+2 for the second
  (stripe_count 16 > k=8, so k=8 is used as-is)

Usage examples:
  # Single component with EC(8+2)
  lfs setstripe -E -1 -c 8 --ec 8+2 /mnt/lustre/file

  # Order independent (--ec can appear anywhere)
  lfs setstripe --ec 8+2 -E -1 -c 8 /mnt/lustre/file

  # Multiple components, single EC spec (both get EC 8+2)
  lfs setstripe -E 1G -E -1 --ec 8+2 /mnt/lustre/file

  # Different EC for different components
  lfs setstripe -E 1G --ec 4+2 -E -1 --ec 8+2
    /mnt/lustre/file

  # EC inheritance with holes
  lfs setstripe -E 1G --ec 4+2 -E 512M -E -1 --ec 8+2
    /mnt/lustre/file

  # Long form option
  lfs setstripe --ec-stripe-count 4+2 -E -1
    /mnt/lustre/file

A new testlist sanity-ec is added to exercise the new
setstripe options.

Further, tombstones were added to guarantee
"llite.*.enable_erasure_coding" is enabled by default and
setting the environment variable "LFS_EC_OK=yes" is no
longer needed before 2.18.
