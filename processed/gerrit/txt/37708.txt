--- Commit Message ---
LU-13096 llite: limit default max lru pages

It is possible that client could have more than 24TiB memory,
which will have 12TiB as default max lru pages, this could
potentially cause write pressure if administrators want
to reduce the value later.

This patch try to limit max default value as 65536MiB
which is a practical value on some customer sites.

(1 comment)

Mark this -1 until I better understand what other issues there are here.

Right now this seems like changing the tunable to be a fixed value is specific to one site, which could just as easily be done with "lctl set_param llite.*.max-cached_mb=64G" on that node without changing the default value.

However, maybe there are other issues that cause problems that are not described in the commit message?  AFAIK, this limit could only affect _clean_ pages that could be easily dropped from RAM, since there is a separate max_dirty_mb limit to prevent too many dirty pages accumulating on the client.
