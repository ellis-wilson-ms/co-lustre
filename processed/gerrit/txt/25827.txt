(1 comment)

As a follow-on patch, it would be possible to pass "flags" to the MDS to have it check if it can do RENAME_EXCHANGE locally (same MDS is easier than cross-MDS, just pass the flags down to underlying OSD), or refuse the rename if RENAME_NOREPLACE is set. I haven't looked into the details much, but there are several fields in struct mds_reint_rec that could hold such flags: rn_bias, maybe rn_mode, and the (currently-unused rr_flags).

I don't mind this patch as a starting point to be able to allow servers to build on newer kernels, but I don't think we should consider this work complete.
