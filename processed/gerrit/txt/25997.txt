Patch Set 1: Code-Review-1

I don't think that this is the proper use of the package name "lustre-devel".  Granted, there was a "lustre-devel" in the past, but it wasn't doing the right thing either.

If we were to have a lustre-devel package, it should only contain things like lustreapi.h, liblustreapi.so, etc.  It should only contain those interfaces that we've explicitly decided are external programming interfaces.  Exposing lustre's internal-use-only header files is not the correct thing to include in lustre-devel.
