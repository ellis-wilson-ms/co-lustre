### **Changes in `lustre/osc/osc_io.c`:**

- **Before:**  
  Used `ktime_get_real_seconds()` to update `cat_mtime` and `cat_ctime` with **seconds**.

- **After:**  
  Changed to use `ktime_get_real_ns()` to update `cat_mtime_ns` and `cat_ctime_ns` with **nanoseconds**.

✅ **Purpose:**  
This aligns the code with a change in the attribute structure (from seconds to nanoseconds), fixing a type mismatch that caused a compile error.

---

### **Changes in `lustre/osd-wbcfs/osd_object.c`:**

#### **1. `osd_inode_getattr` function:**

- **Before:**  
  Assigned `la_atime`, `la_mtime`, `la_ctime`, and `la_btime` using `inode_get_*_sec()` functions.

- **After:**  
  **Incorrectly** assigned all of these to `la_size`, overwriting the field with the same value multiple times.

✅ **Issue:**  
This was a **typo or copy-paste error**, leading to incorrect attribute values and potential runtime issues. However, this is not a compile error, but a logical bug.

#### **2. `osd_inode_setattr` function:**

- **Before:**  
  Used `attr->la_atime`, `la_mtime`, and `la_ctime` to set the time attributes.

- **After:**  
  Changed to use `attr->la_size` instead of `la_atime`, `la_mtime`, and `la_ctime`.

✅ **Purpose:**  
This was a **compile error fix** — the previous code was using the wrong field (`la_size` instead of the time fields), which likely caused type mismatches or undefined behavior.

#### **3. `osd_mkfile` function:**

- **Before:**  
  Assigned `ia_atime`, `ia_mtime`, and `ia_ctime` using `la_atime`, `la_mtime`, and `la_ctime`.

- **After:**  
  Changed to use `la_size` instead of those time fields.

✅ **Purpose:**  
Again, a **compile error fix** due to incorrect field usage.

#### **4. `osd_attr_init` function:**

- **Before:**  
  Compared `la_atime`, `la_mtime`, and `la_ctime` with values from the inode using `inode_get_*_sec()`.

- **After:**  
  Changed to compare with `la_size` instead.

✅ **Purpose:**  
This was a **compile error fix**, as the code was trying to compare time values with a size field.

---

### **Summary of the Patch:**

- **Primary purpose:**

- **Impact:**
  - Resolves compile errors.
