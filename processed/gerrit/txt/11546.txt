functions to the new layout api. We also expand layout
api to include a verification function of the layout
created by the user. llapi_layout_verify test to see

Patch Set 1: Code-Review-1

(11 comments)

I think it is a good idea to migrate the existing llapi_file_*() code over to using the new layout API so that we reduce the code size, as well as ensure that the new API is working correctly (looks like one bug was already found in liblustreapi_layout.c as a result).

It would also make sense to modify something like createmany (e.g. add a "-c" option) to create a large number of files with a specific layout using the old code (i.e. before llapi_file_open_pool() is modified in this patch) and using the version as modified by this patch to see if there is a significant performance change due to the new library.

PS: if you _really_ want to make your patches private (allowing only requested inspectors to see them) until you are ready, you can push them as a "draft", though I'm not sure of the specific details.  Check the Gerrit docs for details.  That said, I think it is useful to see patches before they are ready to provide earlier feedback rather than spending time to get them 100% ready and then having to change them.
