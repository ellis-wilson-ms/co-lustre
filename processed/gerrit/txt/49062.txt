> this can be any character, so using e.g. "%" will avoid the need to escape it

According to the GNU sed manual - https://www.gnu.org/software/sed/manual/sed.html#Addresses

/regexp/
This will select any line which matches the regular expression regexp. If regexp itself includes any / characters, each must be escaped by a backslash (\).

Note that this does not apply to `s` commands, just to address specifiers.

For example:
$ echo 123 | sed '%123%d'
sed: -e expression #1, char 1: unknown command: `%'

$ echo 123 | sed '\%123%d'

> (which can be particularly tricky depending on how many functions process this string)

in this case it will work correctly because we are passing a string with a path without escaping characters - /path/to/mount/point.

Thank you!
