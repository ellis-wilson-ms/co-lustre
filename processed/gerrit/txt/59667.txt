LU-0000 llite: implement immediate mirror writes for DIO

This patch implements immediate mirror writes for Direct I/O
operations in Lustre's File Level Replication (FLR) system.
Previously, DIO writes to immediate mirror files would only
write to the primary mirror, leaving secondary mirrors
uninitialized and requiring manual resync operations.

The implementation includes three key components:

1. Mirror information detection and transfer in llite layer:
   - Enhanced ll_file_io_generic() to detect immediate mirror
     files using ll_get_mirror_info()
   - Transfer mirror metadata (ci_immediate_mirror and
     ci_mirror_count) from llite to cl_io structure

2. Mirror information inheritance in LOV layer:
   - Modified lov_io_sub_init() to inherit mirror information
     from parent IO to sub-IOs
   - Enhanced lov_io_mirror_init() to preserve immediate
     mirror information set by llite layer

3. Mirror iteration logic in cl_io layer:
   - Added detection of immediate mirror DIO operations in
     cl_io_loop()
   - Implemented mirror iteration that repeats the IO
     operation for each mirror
   - Added proper state reset between mirror iterations to
     ensure consistent IO parameters

The solution ensures that DIO writes to immediate mirror files
are automatically replicated to all mirrors simultaneously,
maintaining data consistency without requiring manual resync
operations. Both mirrors receive identical data and are marked
as initialized (init flag set).

Testing shows successful immediate mirror writes with:
- Both mirrors marked as 'init' after DIO writes
- Identical checksums across all mirrors
- No system crashes or stability issues
- Proper debug logging for troubleshooting
