> This is how the package would look like:
 > % rpm -qi kmod-lustre-client
 > Name        : kmod-lustre-client

 > Release     : 5.5.14.0_570.19.1.el9_6.x86_64

Will including the kernel version into the package name mean that "x86_64" and "el9_6" is included into the package filename twice?  Currently, "rpm -qi" shows:

 Name        : kmod-lustre-client
 Version     : 2.16.1
 Release     : 1.el9
 Architecture: x86_64

and the "rpm -q kmod-lustre-client" output is:

 kmod-lustre-client-2.16.1-1.el9.x86_64

so if we now have "Release: 5.5.14.0_570.19.1.el9_6.x86_64" instead of "Release: 1.el9" then the full package version would be:

 kmod-lustre-client-2.16.1-5.5.14.0_570.19.1.el9_6.x86_64.x86_64

which is going to be ugly...  I think we had this style of versioning in the past, and it was removed because of this.

http://review.whamcloud.com/19954 ("LU-7643 build: Remove Linux version string from RPM release field")

So I think that the removal of the duplicate "x86_64" from the version would be a minimum.
