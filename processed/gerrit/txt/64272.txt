LU-19344 llite: defer ktime_get past fast read path

Defer ktime_get() in do_file_read_iter to just before
the CLIO path, after ll_do_fast_read.  When fast read
succeeds (page cache hit), the function returns without
calling ktime_get at all.

Guard ll_stats_ops_tally with if (kstart) so the fast
read path avoids both ktime_get calls entirely.

On a KVM guest this eliminates ~20% overhead from
pvclock_clocksource_read in the fast read hot path,
improving small cached reads from ~549k to ~4.1M
reads/sec combined with the parent patch (+653%).
