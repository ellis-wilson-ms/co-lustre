--- Commit Message ---
LU-18624 zfs: disable compression by default

By default ZFS is enabling compression which is causing test
failures for us. It should the administrators choice to use
ZFS with compression turned on or off. Change mount.lustre
zfs backend to handle compression=val mount option. By default
we will turn it off.

For the test suite we can use <facet_type>_FS_MKFS_OPTS to
turn on compression for testing.

Disable lots of conf-sanity test which are broken with ZFS 2.X.

--- Message (James Simmons, 2025-02-20 14:50:55.000000000) ---
Patch Set 5:

The reason for turning off compression by default was that 1) we couldn't revert without breaking RHEL9.5 builds. 2) no patches could land at the time. It would of taken many weeks to resolve all the test failures.

Now the patch is written so you can pass the compression setting to mkfs.lustre so we can turn compression back on. In that case we can then go over every test failure and resolve it one at a time without impacting other work flows. We still have a few test failing even with compression off so once those are fixed we can look at turning compression on. Compression is very expensive which I noticed in Oleg's test harness caused my test to additonally fail with timeouts so we have to be care in such a case.
