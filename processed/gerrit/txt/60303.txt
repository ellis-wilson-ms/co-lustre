--- Commit Message ---
LU-18756 ofd: include resource id repair into id check

This patch reorganizes the resource id check on the OST and to include
the id repair into the id check, allowing the repair to be triggered
when the id check returns -EAGAIN, indicating that the OST object has
unset resource ids.

In all cases, the object's attributes are only retrieved once for the
ID check and/or ID repair.

Further, "struct ofd_object" is now explicitly passed to id check and
repair functions instead of relying on "ofd_info(env)->fti_obj".

This patch does not include the ID repair into the code path yet.
