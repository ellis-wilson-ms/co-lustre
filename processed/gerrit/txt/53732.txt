--- Message (James Simmons, 2024-01-22 21:47:19.000000000) ---
Patch Set 2:

I don't think the parsing of an integer from the Netlink packet sent from user land works with different types. The problem with libyaml is that only one type of integer exist (YAML_INT_TAG). So by default we always send a S64 since its not possible to know what large a value will be at anytime in the future. You could examine the value for size but then the packet sent from user land would end up being variable. So even if a S64 is the largest value that value at one time could be 128K or 256 making packet data of different size. That is a lot of complexity to handle.
