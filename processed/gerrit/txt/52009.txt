Patch Set 1:

added this follow-up patch to run some interop testing.

Firstly, I'm not even sure that "mdsversion" and "ossversion" are valid interop test parameters (instead of "serverversion"), but we'll see.

The test of interest is ossversion=2.15.3, which does not handle the new quota revoke functionality.  We don't usually allow sites to run with major differences in server versions, but this can happen for some time during an upgrade, so it can't totally explode even if the new revoke functionality is not working yet.

I expect that the mdsversion=2.15.3 test will fail because the new subtest doesn't have a version interop check, but I wanted to see if there are other reasons why the testing would fail in this case.
