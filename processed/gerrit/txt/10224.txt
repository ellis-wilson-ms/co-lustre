--- Commit Message ---
LU-3639 osd: check real OI files for local FID lookup

For the case of upgrading to Lustre-2.x (x >= 4),
there will be local FID in LMA and OI mapping for
the local object, and then when downgrade back to
Lustre-2.y (y <= 3), the osd_fid_lookup will use
local FID to (NOT the original IGIF FID) call the
osd_oi_lookup(), then we need to check the real OI
files to avoid to return -ENOENT.
