--- Commit Message ---
LU-18728 tests: use urandom to really consume ZFS space

It appears newer ZFS is using data compression by default, so reading
from /dev/zero results in files not consuming the expected amount of
space.  Instead, read from /dev/urandom for ZFS to write files in
sanity and conf-sanity to ensure they fill the OSTs, or the image
to be used for target creation, as expected.

Test-Parameters: testgroup=review-zfs env=ZFS_MKFS_OPTS="compression=on"

Hi Bruno, thanks for the patch.

You can use the "Test-Parameters:" keyword to change the testing for a patch.

https://wiki.whamcloud.com/display/PUB/Changing+Test+Parameters+with+Gerrit+Commit+Messages

The "trivial" keyword can be used to reduce testing for patches that do not affect code functionality, and then add in specific ZFS test sessions for the modified subtests:

 Test-Parameters: trivial
 Test-Parameters: testlist=conf-sanity env=ONLY=32,FSTYPE=zfs
 Test-Parameters: testlist=sanity env=ONLY=413,FSTYPE=zfs

Note that WC autotest and Gerrit Janitor will already run extra sessions for subtests that are modified, so in some cases it is enough to only specify "trivial" for patches that are modifying a particular subtest, as can be seen by the Maloo comment posted when testing is started:

  The following sessions will be run for build 110898 patch 6
  :
  * review-dne-subtest-change on el8.10-x86_64/ldiskfs servers and clients
