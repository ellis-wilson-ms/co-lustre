LU-0000 llite: improve ACTIVE_WRITER lock handling

This patch improves ACTIVE_WRITER lock handling for immediate
mirror files with three key changes:

1. Change ACTIVE_WRITER locks from LCK_EX to LCK_CW mode to
   enable concurrent write access, which is appropriate for
   immediate mirror scenarios where multiple writers can
   coexist.

2. Use ldlm_lock_decref_and_cancel() instead of
   ldlm_lock_decref() to ensure locks are properly cancelled
   and send cancellation RPCs to the server, allowing the
   server-side active writer count to be decremented.

3. Add enhanced debug logging to track blocking AST
   assignment for better debugging of lock lifecycle.

These changes ensure proper lock cancellation and server-side
counter management for ACTIVE_WRITER locks.
