--- Commit Message ---
LU-969 debug: reduce stack usage

various cases to reduce stack usage:
 - libcfs_debug_vmsg2 to accept libcfs_debug_msg_data structure
   to replace SUBSYSTEM, __FILE__, __FUNCTION__, __LINE__ and cdls
   on the stack
 - libcfs_debug_vmsg1 for regular users like CDEBUG() which don't
   need 2 format strings
 - CDEBUG, CHECK_STACK, DEBUG_CAPA to use static libcfs_debug_msg_data
 - RETURN/GOTO use return code 3 times, libcfs_log_return and
   libcfs_log_goto are introduced which accept return code once.
 - in ptlrpc_main, move struct lu_env to heap to save stack

In addition to the comments, the entire idea of static libcfs_debug_msg_data, that is not really fully static, but there is own copy for a lot of shared occurences, like every RETURN call and __CDEBUG and so on, this just strikes me as super memory wastage.

Also additionally, some places share this variable, and I can well imagine how different CPUs could pass by and fill in different details which would lead to a race on the content of the static var.
