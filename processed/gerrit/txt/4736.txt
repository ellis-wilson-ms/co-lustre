Patch Set 5: I would prefer that you didn't submit this

(30 inline comments)

Firstly, thank you for the clear and well-commented code - that makes this patch much easier to understand.  Also thanks for the update to the wiretest.c files to use tabs.

My main objections to this patch relate to the structures used in the wire protocol.  I'd like for those to be properly aligned and sized, so that there are no problems on architectures like IA64 (or others) that don't handle misaligned __u64 fields well.

It is definitely desirable that the llapi functions be under the LGPL, and it is a good start for you to add the new HSM methods with LGPL.  There was some discussion already about rewriting other parts of liblustreapi in order to improve the APIs, and make that under LGPL as well.
