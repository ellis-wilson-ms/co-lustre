--- Commit Message ---
LU-4653 fid: only compare non-IDIF seq in lu_fid_diff

Because it will pack OST index into IDIF FID in 2.6, and
in 2.4/2.5, the OST index in IDIF FID is always 0. So we
only compare non-IDIF sequence in lu_fid_diff.

(4 comments)

I see now that lu_fid_diff() is only used in the osp/osp_precreate.c file, and this function is not safe for comparing FIDs sent remotely.  It might be OK for use inside OSP since it knows that the FIDs are only from the one OST.

In this case, I'd suggest to move lu_fid_diff() into osp_precreate.c so that it isn't accidentally used in the OFD code where it would cause a DOS.
