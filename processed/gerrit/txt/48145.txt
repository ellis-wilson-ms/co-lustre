LU-16035 kfilnd: Fix targeting RX contexts

Prior to this commit, only RX context 0 would be targeted by untagged
messages. Untagged messages are used for sending the LNet payload for
an immediate transaction and the bulk request message for a bulk
transaction. The kfi header for these messages contains a prefer_rx
field. The prefer_rx field tells the remote target which RX context it
should use when sending untagged messages to the local peer. While the
prefer_rx field was being processed and the peer structure rx_context
field being updated accordingly, the rx_context  was never used to
construct a kfi address (kfi_addr_t) to target a specific RX context.
This patch fixes this by using kfilnd_peer_get_kfi_addr() to generate
a kfi address which targets a specific RX context.
