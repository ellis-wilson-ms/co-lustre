LU-17288 osd: HAVE_LDISKFSFS_GETHASH_INODE_ARG gone

With patch "ldiskfs: Parallel configure tests for ldiskfs"
aarch64 Ubuntu 20.04 build fails with message

in definition of macro ‘e_ldiskfsfs_dirhash’
expected ‘const struct inode *’ but argument is of type ‘const char *’

HAVE_LDISKFSFS_GETHASH_INODE_ARG macros have been removed, but still
used in osd_iam_lvar.c. This patch changes it to the
HAVE_LDISKFSFS_DIRHASH_WITH_DIR

Fixes: 5579168("LU-13485 ldiskfs: Parallel configure tests for ldiskfs")
