(3 comments)

We should avoid writing pairs of functions like hur_len_okay() and hur_len() where we need to call the first to see if it's OK to call the second. Can the overflow checks be moved into hur_len() and converted to error returns?

There is no reason for hur_len() to return int. size_t is the correct return type.

Also both functions should take a const struct hsm_user_request *hur.
