--- Commit Message ---
LU-19013 lnet: only send GDS traffic on capable interfaces

Only send GDS p2pdma traffic on interfaces that support
it. LNDs that implement lnd_get_dev_prio() support GDS,
hence filter NIs in lnet_get_best_ni() based on whether
this callback is implemented.

If GDS traffic is unintendedly sent over an incompatible
interface, the LND could associate the message with
garbage data and corrupt the file being written to.

This can most easily be replicated by using UDSP
to send gdsio traffic over an incompatible interface.
