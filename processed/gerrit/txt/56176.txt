the good news is: we do not create empty xattr even via nfs just like I thought, ext4 handles it for us:

[root@oleg95-server tests]# mount localhost:/mnt/lustre /mnt/nfs -t nfs
[root@oleg95-server tests]# touch /tmp/testfile
[root@oleg95-server tests]# chmod 644 /tmp/testfile 
[root@oleg95-server tests]# cp -p /tmp/testfile /mnt/nfs/  
[root@oleg95-server tests]# stat -c %a /mnt/nfs/testfile 
644
[root@oleg95-server tests]# getfattr -d -m system.posix_acl_access -e hex /mnt/nfs/testfile 
[root@oleg95-server tests]# getfattr -d -m system.posix_acl_access -e hex /mnt/lustre/testfile 
[root@oleg95-server tests]#

Unfortunately error code is 0 so we cannot use that to determine if the attribute is there or not though.. Should we just wc -l the output or some such?

[root@oleg95-server tests]# getfattr -d -m system.posix_acl_access -e hex /mnt/nfs/testfile   
[root@oleg95-server tests]# echo $?
0
