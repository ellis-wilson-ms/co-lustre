Patch Set 1: Code-Review-1

(1 comment)

Thanks for cleaning up this patch. I think this won't work correctly for any of the later states, because OBD_STATFS flags are not contiguous. This also implies there is a bug in "lfs df" printing of flags. The state_names[] array needs to be changed to "DRPxxSI" or similar to cover the gap in the flag values. The flags cannot be changed because they are part of the network protocol. 

Can you please add a test case to verify the new proc tunable, and set each flag with a fail_loc to verify it is correct. I think there are already some FAIL_VAL for most of them.
