LU-13005 lnet: remove lib-eq.

There is nothing of value in left in lib-eq.  An 'lnet_eq' just holds
a function pointer.
So change all 'struct lnet_eq' pointers to function pointers, and
bypass the "event queues".

--- Message (Amir Shehata, 2019-12-05 01:11:48.000000000) ---
Patch Set 5:

Originally, when we link an MD, we'd keep track of that via refcount on then eq. This way we know when we're shutting down that all the MDs were handled. With this change that is no longer possible. Having made some changes around this area, the ability to find out if we unlinked all the MDs we expected to unlink is useful. It ensures that we've properly handled outstanding MDs.

Another thing I'm thinking about is:
LNetNIInit()->fail->lnet_unprepare()->lnet_res_containers_destroy()
This deletes the MDs
At the same time you could be processing an:
lnet_finalize()->lnet_msg_detach_md()
Which access the MD.

I'm still thinking through this, but do you think this could open up potential memory access after free?
