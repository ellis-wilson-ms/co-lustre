--- Commit Message ---
LU-20164 lnet: Return correct error on alloc_ordered_workqueue failure

Successful libcfs_setup() will overwrite rc to 0. Then,
if call to alloc_ordered_workqueue() fails, it error path
returns rc unmodified, ie value of 0.

This patch resets rc to -ENOMEM before returning on
alloc_ordered_workqueue failure.

Reported by cppcheck.
