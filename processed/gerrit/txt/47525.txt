LU-15909 lnet: Add peer NI send lists

We can optimize the peer NI selection algorithm at the cost of some
extra memory by subdiving the peer NIs on a peer net into three lists:
1. The subset of peer NIs that have a priority defined
  - This is the "priority list"
2. The subset of peer NIs that do not have a priority defined
  - This is the "non-priority list"
3. The subset of peer NIs that have degraded health
  - This is the "degraded list"

By dividing the peer NIs in this manner the selection algorithm can
then consider only those attributes of each NI that are relevant when
comparing NIs on a list.

For example, NIs on the priority list are, by definition, completely
healthy so the selection algorithm doesn't need to perform the
atomic_read() to get the health value of each peer NI on the list.
We can optimize the selection further by sorting this list by the
priority value of each NI. Thus, if the selection algorithm encounters
a peer NI with a lower priority than the current "best" peer NI then
it can stop inspecting any more peer NIs as it has already found the
one with highest priority.

Similarly, peer NIs on the non-priority list are also completely
healthy, and they do not have a priority value set, so the selection
algorithm does not need to consider either of those values when
selecting the best peer NI.

Peer NIs on the degraded list may have varying health values and
selection priorities, so each of these criteria are considered when
selecting a peer NI from this list.

A read-write lock is used to synchronize any updates to these lists,
and a spinlock in the lnet_peer_ni struct is used to prevent
concurrent updates.

lnet_peer_ni_set_send_list() places an lnet_peer_ni on the appropriate
list based on the peer NI's priority and health value. As such, it is
called any place those values are modified, and whenever an
lnet_peer_ni is attached to a lnet_peer_net and lnet_peer.

lnet_peer_is_pref_nid_locked() now checks for LNET_NID_IS_ANY so it
can be passed NULL as a nid argument.

lnet_select_peer_ni() took dst_nid as an argument, but only used it
in an internal error message. This argument was removed.
