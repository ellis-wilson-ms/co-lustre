LU-12668 ec: add support to resync stale ec comps

Add support for syncing stale parity mirrors.
This is done by the command:
  lfs mirror resync ...
just like resyncing data mirrors.

If an extent has both stale data mirrors and also
stale parity mirrors, the data mirrors will be synced
first and the parity mirrors afterwards. Note, if the
data mirror has LCME_FL_NOSYNC set, resyncing the
corresponding parity mirror is skipped, regardless
of whether the parity mirror has no sync set.

Document the stripe layout and the relation between
stripe offsets in the data comp and the associated
parity stripes in the ec comp.

Change llapi_mirror_find_stale() to only return stale comps for
normal mirror and create a new llapi_ec_find_stale() to find
all stale comps that are used for ec.

llapi_mirror_find() is used to find a non-stale mirror
containing file data for the purpose of syncing to stale
mirrors. Skip any parity mirrors as they do not hold file
data to sync from.

sanity-ec/6b is re-enabled to exercise resync.
