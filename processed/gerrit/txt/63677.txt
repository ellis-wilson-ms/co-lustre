--- Commit Message ---
LU-12187 llapi: support parity mirror binding on layout merge

Layouts that are merged on the llapi intially use the same mirror and
mirror link ids which would conflict after the merge. Therefore, the
mirror ids and mirror link ids need to be offset for the layout that
is merged into the existing layout.

This patch implements this change and adds llapi layout test 65.

## Fix Explanation

**The Problem:**
The patch introduced a local variable `mirror_id` at line 3577 inside the `list_for_each_entry` loop in the `llapi_layout_merge()` function, but this variable was never actually used anywhere in the code. Since the code is compiled with `-Werror=unused-variable`, this unused variable declaration causes a compilation error.

**The Fix:**
I removed the unused variable declaration `uint32_t mirror_id;` from line 3577.

**Rationale:**
Looking at the patch, it appears the variable `mirror_id` was likely added during development but then the implementation changed to use `mirror_id_offset` (calculated before the loop) instead. The variable serves no purpose in the final code - all mirror ID operations use either:
- `comp->llc_mirror_id` (the source component's mirror ID)
- `mirror_id_offset` (calculated before the loop)
- `new->llc_mirror_id` (the new component's mirror ID after offsetting)

Therefore, removing this unused variable declaration is the correct fix.
