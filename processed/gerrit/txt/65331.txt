LU-20119 selinux: add lustre_helpers SELinux policy module

RHEL/Rocky 10+ dropped kernel_generic_helper_t from the base
selinux-policy (selinux-policy-42.1.7), causing SELinux to deny
every call_usermodehelper() spawn under enforcing mode. This
breaks l_getsepol (mount aborts after 30s), lctl (root_squash
not applied), l_getidentity, and l_getauth.

Introduce a dedicated lustre_helper_t domain with a
lustre_helper_exec_t file type for the Lustre helper binaries.
A type_transition rule transitions kernel_t to lustre_helper_t
on exec of those binaries, confining all permissions to Lustre
helpers only -- no other call_usermodehelper() callers are
affected.

On RHEL/Rocky 9 the base policy's kernel_t ->
kernel_generic_helper_t transition covers all UMH helpers and
Lustre works without this module. On RHEL/Rocky 10 that
transition was removed together with kernel_generic_helper_t
itself, so kernel_t execs the helper binary directly; the
kernel_t -> lustre_helper_t transition handles that case.

Files added/modified:

  lustre/utils/selinux/lustre_helpers.te  - policy source
  lustre/utils/selinux/lustre_helpers.fc  - file context labels
  lustre/utils/selinux/Makefile.am        - build and install rules

Build-time detection in config/lustre-core.m4 via the new
LC_CONFIG_SELINUX_POLICY macro auto-enables the module when
PLATFORM_ID is platform:el10 or later; can be overridden with
--with/--without-selinux-policy. Requires checkmodule
(checkpolicy) and semodule_package/semodule (policycoreutils).

For direct source installs (DESTDIR unset), make install runs
semodule -i and restorecon immediately. For RPM installs,
lustre.spec.in runs semodule -i in %post and semodule -r in
%preun.

The RPM bcond (lustre_selinux_policy) is enabled by default on
RHEL 10+ (%rhel >= 10) and disabled on all other distros.

RHEL 9.7 (selinux-policy-38.1.65) is unaffected as
kernel_generic_helper_t is fully defined there. Tested on
Rocky 10.1 (selinux-policy-42.1.7) with SELinux enforcing.
