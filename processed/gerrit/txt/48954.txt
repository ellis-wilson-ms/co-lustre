LU-14361 statahead: add support for mdtest shared dir workload

This patch adds statahead support for shared dir stat() workload
with fname patteren like mdtest shared dir stat() access.

The performance imporvements are shown as follows:
IO500 (KIOPS)		w/o patch	w/ path
mdtest-easy-stat	740.01		1276.31
mdtest-hard-stat	514.36		1105.33

That is good.  However, code tends to break if it is not tested  regularly.

Having a simple functional sanity test that can confirm that shared dir statahead is still working is more efficient than having to run mdtest and manually review the results.  Performance tests are hard to pass consistently in VMs that run on many different types of hardware and contend with other VMs on the host.  Having a simple functional test is more helpful in this case.
