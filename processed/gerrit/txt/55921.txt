LU-14288 libcfs: Support large NIDs in nidranges

Add support for large NIDs to the nidrange/nidlist code.

New member added to struct cfs_expr_list to store the base of
the linked range expressions. The base is also stored in each linked
range expression. These are to faciliate conversion of the nidlist
string into appropriate integers, as well as to ensure correct
formatting when the nidlist is converted back into a string.

cfs_ip_addr_parse() detects a large NID by searching for a ':'
character.

A new function is added to struct netstrfns to perform matching of
large addresses against a nidlist.

conf-sanity.sh/test43a is modified to generate a range expression that
contains the client's NID and verify correct handling when
nosquash_nids is set to the range expression.

Repeating my comment from the ticket here:

I've started plugging away at this. It is not a huge lift to modify the existing nid range code to support large NIDs. While it may not be practical to actually use this to specify lots of IPv6 NIDs, I think it is useful to have this support so that code can be simplified for cases where just a couple NIDs are specified (like various test cases). i.e. callers can pass any kind of nid to cfs_parse_nidlist and it "just works". I had originally had a few lines of code that would reject large nids that contained a range expression, but it seemed silly to prevent that when the rest of the code can handle them just fine.

In any case, these patches are not done, but they are fairly well baked. I'd appreciate feedback as I continue to polish them.
