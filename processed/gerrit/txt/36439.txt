Patch Set 1: Code-Review+2

Petros, the "*many" commands aren't really tests in themselves (at least not since the very early days of Lustre development), just helped programs to quickly create and remove test files. They are much faster than calling "touch" in a loop or "rm -r" to clean up. 

That said, I think there has been an over-proliferation of these tools and they duplicate a lot of code as a result. Some of them can be removed easily, like this one, since it is unused. Others, like unlinkmany need a bit more work but IMHO should be cleaned up.
