LU-14974 hsm: improve the repartition of requests between copytools

This patch changes the way the requests are put in the hsm_scan_data
structure. Instead of allocating one new HAL when a new archive_id
is encountered or when there are no available HALs with enough space,
we allocate one HAL for each copytool that supports this archive_id.
This group of HALs is called a pack.

We then try to balance the load between the copytools by putting new
requests in the least used HAL in the pack.

This solves an issue that happens when the coordinator receives a few
"big" requests (say 10 requests of archiving 10GB files). In such case,
every request will likely go to the same copytools and create a huge
load on it while leaving the other copytools idle.
