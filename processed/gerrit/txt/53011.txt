LU-17269 obdclass: serialize obddev creation

lookup and obd device creation in class_register_dev() must be atomic.
this patch is a simple workaround - a mutex is used to make lookup/create
atomic.

--- Message (Timothy Day, 2023-12-14 22:25:25.000000000) ---
Patch Set 2:

I think we should use obd_device_lock() here rather than the mutex. We can convert xa_alloc() to the lockless __xa_alloc(), and make a lockless class_name2dev_nolock(). Then, have obd_device_lock() cover the entire class_register_device(). I think that would be more robust than using a new mutex, since it would also make class_register_device() atomic with readers of the Xarray.

That should resolve the test failure you're seeing. I can work on making the ioctl() less racy another time.
