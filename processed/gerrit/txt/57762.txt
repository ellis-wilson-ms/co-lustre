LU-18615 mdt: support lfs hide [path]

This implements "lfs hide [path]". With this cmd,
[path] will be hidden from directory listing "ls -a". Similar to
.lustre.

We also give tunable(s) to turn this feature on/off. When this
feature is off, lfs hide cmd can still succeed, and mark the [path]
as hidden. However, ls -a still returns it. The tunable is one per
MDD.

The lfs hide cmd will triger an IOCTL call. The IOCTL call pass
the req to lmv_ioctl, then mdc_ioctl. mdc will create a ptlrpc
req to the MDT. The MDT will mark the object as hidden in its
trusted.lma.

When building a dir page (ls -a), mdd will check
1. if the tunable is on
2. then if the entry starts with "." (i.e. regularly hidden)
3. then if the entry is marked as hidden.
If all checks are passed, this entry will not be returned to
client.
