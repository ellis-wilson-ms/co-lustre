LU-18687 compat: move xarray to lustre_compat

Migrate the backported xarray code to lustre_compat.
Along the way, create the needed build infrastructure
for lustre_compat. Currently, lustre_compat is built
into libcfs.ko.

Eventually, all of the Lustre/LNet compatability code
will live in lustre_compat - maintaining a clear
separation from the functional code in Lustre and LNet.

Patch Set 3:

Ping reviewers. I'm interested in thoughts on the lustre_compat model. lustre_compat would mirror the actual kernel tree - pieces of backported code could be found at a similar path as the kernel. I think there's a few benefits:

1) Easier to reason about the diff between new kernels and old vendor kernels.
2) Headers would only differ by "lustre_compat/", so automated translation is easier.
3) All compat code lives in the same top-level directory
