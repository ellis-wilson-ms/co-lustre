LU-0000 llite: COPY safely duplicate iov_iter

COPY FOR PAF

When copying an iov_iter the underlying iovec/bvec also
needs to be sized duplicated.
Note that discard and xarray iters can be skipped here,
however the duplicated vector needs to be tracked and freed.

This borrows heavily from dup_iter()

The iter argument to ll_dio_user_copy() to no longer used
and can be removed.

Also addeded tests for aligned and unaligned readv/writev
and fixes the use of iov_iter_get_pages_alloc2() which
advances user backed iters where iov_iter_get_pages_alloc()
did not.

Updated ll_get_user_pages to revert the advanced iter in
the case of user backed iters.

Linux kernel v5.19-10287-gfcb14cb1bdac
  new iov_iter flavour - ITER_UBUF

This introduces user_backed_iter() and provides user_backed_iter
for older kernels
