LU-394: LND failure casued by discontiguous KIOV

This issue was imported by bug 18881 where I moved the urgent
pages to front of lop_pending to fix a deadlock issue.
I reverted bug 18881 in this patch and came up with a new solution:
cl_page_gang_lookup() only blocks on the first page. This is also
for deadlock avoid since we should never grab multiple pages' lock
without try method.

--- Message (Niu Yawei, 2011-06-10 10:43:38.000000000) ---
Patch Set 2: I would prefer that you didn't submit this

The term 'quota' is quite confusing, I think you meant 'grant' but not user/group quota, right?

Anyway, I think our io code will never build any partial page in the middle of an RPC, see osc_send_oap_rpc(), and this piece of code runs for years, It's hard to believe that there is such bug in osc_send_oap_rpc().

So, I think the reason should be CLIO doesn't calculate the page off correctly somewhere. After I looked into the code, I found that we doesn't set ASYNC_COUNT_STABLE flag for sync write in 2.x code, so osc_send_oap_rpc() will call ap_refresh_count() for sync page mistakenly, which will return a wrong oap_count for sync page, and break the logic of osc_send_oap_rpc(). 

I think the proper fix shoud be adding the ASYNC_COUNT_STABLE for sync write pages.

--- Message (Oleg Drokin, 2011-06-14 00:13:17.000000000) ---
Patch Set 2:

Niu, I found this hard to believe at first too, but there is a log attached in the Jira that you can take a look at to see for yourself.

We never hit this with b1_8 because there failure to put a page into cache (be it lack of quota or grant) caused "synchronous write" that sent just that one page and nothing else.

In CLIO when sync write happens it tries to see what other cached pages for this object are there and tries to combine them into the same RPC. Then of course we hit this hole because previously last incomplete page is still not extended (we don't know if the issue causing us to do sync write would prevent that page from being written on the server or not) add this leads to a hole in the kiov as the result.
