--- Commit Message ---
LU-19157 nodemap: forward ban status to client

Use new pb_flags value MSG_CLIENT_BANNED to foward ban status to the
client in the repmsg buffer. The client is not nodemap aware, but can
print a console warning that it was banned by the server. This message
is limited to one occurence every 6 hours to avoid flooding.

sanity-sec test_81a is enhanced to check the message is printed on the
client console.

(3 comments)

It would be good to add a test case for this (with version check for MDS support) that the message is printed on the client console. Please add a comment by the message that it is checked by sanity-sec test_NNN so that it is not changed unnecessarily and break the test. 

Note that because review-subtest-change runs in a loop, then the test case must handle this properly. Normally the "correct" thing to do for such tests is to limit dmesg searching to only the current subtest output, but that would fail due to rate limiting the output. Either it needs to allow checking all dmesg output (which kind of makes repeating the test pointless) or it should unset ONLY_REPEAT and ONLY_MINUTES if set, since repeating the test is meaningless.
