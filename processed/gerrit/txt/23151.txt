Patch Set 1: Code-Review-1

Checking the actual code shows that ilookup5_nowait() is actually returning NULL from find_inode() in at least 3.14 kernels. I don't find ilookup5_nowait() in 4.8 kernels, so this may be an OK fix for upstream (I didn't verify that myself), but not in code that also needs to run on older kernels.  

We could add a comment with the kernel version when this changed, so that it will be removed at some later date, or we can just assume that eventually the upstream kernel client will grow to replace the out-of-tree client and we don't need to fix this trivial issue in master.
