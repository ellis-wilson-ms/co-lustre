--- Commit Message ---
LU-1762 tests: get correct MMP update and check intervals

This patch fixes the get_mmp_update_interval() and
get_mmp_check_interval() in mmp.sh to get the correct
MMP update and check intervals from both the old and
new outputs of debugfs.

The patch also improves test_8() to increase the running
time of e2fsck to allow mount operation to be started
before e2fsck operation stops.

It would be best if the get_mmp_update_interval() code could work with both the old and new e2fsprogs output. This would avoid breakage for test environments that do not have the latest version installed (e.g. developer or external testing).

Something like the following should work:

    awk '/update.interval/i { print $NF }'

But I'm not at a system where I can verify the exact syntax. IIRC, $NF is the number of fields, so it should print the last item in the line, and /i will make a case-insensitive search.

--- Message (Jian Yu, 2012-08-21 07:42:19.000000000) ---
Patch Set 1:

> But I'm not at a system where I can verify the exact syntax. IIRC, $NF is the number of fields, so it should print the last > item in the line, and /i will make a case-insensitive search.

Thanks for the suggestions.

I just verified the command with e2fsprogs version 1.42.3.wc3 and 1.41.90.wc3 separately. $NF is correct, but /i is not an awk syntax. Per the link of http://www.gnu.org/software/gawk/manual/html_node/Case_002dsensitivity.html , I changed the command as follows:

awk 'tolower($0) ~ /update.interval/ { print $NF }'

It works correctly on both versions.
