LU-30 ldlm: improve & cleanup ldlm_handle_enqueue0

This patch includes a few things:
- ldlm_namespace::ns_policy is replaced by a new function table
  ldlm_it_policy_ops, which has two methos:
  ipo_check: check if intent policy should be executed
  ipo_handle: intent handler

- intent handler is moved from ldlm_lock_enqueue() to a dedicated
  function ldlm_lock_intend()

- ldlm_handle_enqueue0 will create ldlm_lock for ns_policy only if
  it's necessary

- ldlm_handle_enqueue0 will not call ldlm_reprocess_all() anymore
  even when ELDLM_OK is returned by intent policy

- code cleanup
