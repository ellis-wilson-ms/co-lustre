LU-4593 ptlrpc: req could loop forever in sending list

In ptlrpc_check_set(), when a request runs into
RQ_PHASE_UNREGISTERING phase, if the reply is successfully unlinked,
the unregistered flag should be set accordingly, otherwise the request
could possibly run into interpret section and repeat the reply
unregister.
