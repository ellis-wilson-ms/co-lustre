--- Commit Message ---
LU-11200 ldiskfs: add support for RHEL7.5 alt kernel

For Power8 and ARM systems running CentOS/RHEL to work they need
to run a special 4.14 based kernel. Expand lustre server support
using ldiskfs to this platform. With the lastest RHEL7.6 alt
versions it is now pretty equivalent to Ubuntu18 LTS.

Once thing to be aware is that for platforms with a page size
not equal to the block size file system encryption will not
work. This can only really be determined at run time so we
need to trust that ext4 will do the right thing and disable
encryption.

I see the mistake. The name lookup_one_len_unlocked() is very misleading. In this case it doesn't take the parent lock but the child dentry can take the lock. In fact it takes the child dentry lock when it it fails to find the dentry in lookup_dcache() and ends up calling lookup_slow(). lookup_slow() is where the lock is taken.

In the case of lookup_one_len() it assumes you have the inode lock so it calls the unprotected function __lookup_slow(). If you call lookup_one_len() it will lock because it itself takes the lock.

Now my thinking was lookup_slow() uses inode_lock_shared. iterate_dir() also takes inode_lock_shared() as well when iterate_shared() is provided so it should be safe. Have to think about this
