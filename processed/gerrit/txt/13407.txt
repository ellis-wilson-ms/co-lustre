Patch Set 1:

> we don't need the "-p" option
This is incorrect statement. When test_mkdir() is called without -p option we have :
test_mkdir() {

        case $# in
                1) path=$1;;   # path=/mnt/lustre/d27i.sanity

        child=$(basename $path) # child=d27i.sanity
        parent=$(dirname $path) # parent=/mnt/lustre

        if [ ! -d ${parent} ]; then # parent /mnt/lustre exist
                if [ "$p_option" == "-p" ]; then
                        mkdir -p ${parent}
                else
                        return 1
                fi
       fi
i.e. if parent=/mnt/lustre exists -- "mkdir" is not called at all and the test fails because of $tdir is not created.
James, this is easy reproducible by:
# sh llmount.sh
# ONLY=27i sh sanity.sh
