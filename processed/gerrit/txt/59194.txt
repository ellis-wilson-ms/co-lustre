tgt_is_multimodrpcs_client evaluating to true. This feature was

(1 comment)

If this is a question of whether the MGS should be updating the XID and replayable requests, then this should be covered by a flag in the export (similar to "imp_replayable") rather than strcmp() of the device name. 

That said, we might also consider the overhead of checking this branch for every RPC sent on the very busy targets vs. an occasional RPC on the mostly-idle MGS target. While I'm in favor of avoiding overhead and useless code avoidance, this adds a hard-to-find special case for the MGS, and if we ever decide to allow the MGS to accept modification requests from clients (eg. "store this config option for me", or whatever) then it might be hard to trac down an issue with why those rarely applied config settings fail only after (equally rare) recovery events.
