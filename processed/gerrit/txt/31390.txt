--- Message (Li Dongyang, 2018-03-01 06:05:45.000000000) ---
Patch Set 2:

> This is hack, you shouldn't do it this way.
 > 
 > According to your description in the ticket, this file is created
 > by open(O_CREAT) instead of create(), the dentry should have been
 > revalidated, why lookup is called?

Well yes, the file is created by open(O_CREAT), the thing is the antivirus software is doing scanning using fanotify FAN_OPEN_PERM.
The thread doing the open(O_CREAT) will be blocked inside the open syscall until the scanning thread thinks the open should be granted.
The scanning thread is doing the lookup, trying to scan the file for threats of course.
so the open thread is holding i_mutex of the parent, waiting for the decision from 
scanning thread to finish the open syscall. The scanning thread is trying to get the i_mutex so it can lookup the file and do the scan. Deadlock
