Patch Set 1:

Note that in order for any of these patches to get merged, they should be submitted against the "master" branch and not the "b2_1" branch.  All new features go into "master", and only serious bug fixes would go into "b2_1".


I think I'm beginning to soften up on enabling this feature on upgrade, because it would ensure that link EAs and FID-in-dirent will get created for new files.  My one concern is that there may be problems with doing a 1.8->2.x upgrade, use FID-in-dirent, then a 2.x->1.8 downgrade.

The 1.8 MDS will delete the LMA and FID for any files that it is accessing (because it doesn't know about FIDs) and revert to using the IGIF value.  However, it doesn't do anything with the FID-in-dirent at all, which is the main reason this isn't enabled automatically.  If there is an upgrade again the FID-in-dirent data will contain the old FID, and I suspect the upgrade tool will generate a different FID for it and cause confusion.

That is one reason why the admin needs to enable this feature when they are sure that they will not downgrade to 1.8 again.
