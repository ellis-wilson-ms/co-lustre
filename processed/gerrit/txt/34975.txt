LU-12140 lnet: adds checking msg len

The LNET can't handle a msg with len larger than LNET_MTU.
The next error occured for DOM 1MB
 LNetError: 3137:0:(lib-move.c:4143:lnet_parse()) 192.168.8.1@tcp,
 src 192.168.8.1@tcp: bad PUT payload 1051832 (1048576 max expected)

The patch adds fragment size check.

that doesn't look right. 1MB DOM stripe size shouldn't cause lnet problems and reducing it to 64K is not a fix but the hiding of problem if it exists. DOM read-on-open feature can transfer data but not more than reply buffer size, if it is bigger than LNET_MTU then probably something is wrong with RPC parameters and they should be fixes instead?

Without that patch disables reply buffer grow, read-on-open could ask for reply buffer to be up to:
max_reply_len = req->rq_rqbd->rqbd_svcpt->scp_service->srv_max_reply_size;
