--- Message (Jian Yu, 2011-11-15 08:26:27.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

(2 inline comments)

The original large-lun.sh was written without using the lustre/tests/test-framework.sh, so all of the variables/functions were home-made. To add the script into auster test suite, we need change the script to use the common variables and functions provided by test-framework.sh. For example:

1) replacing add_output_marker() with log()

2) replacing mount_target()/unmount_target() with start()/stop(), etc.

3) using/improving the run_llverdev(), run_llverfs(), run_e2fsck(), etc. in the t-f.sh

4) replacing dumpe2fs with $DUMPE2FS

5) ......

I think we need remove the PV/VG/LV creation codes from the script, just use the OSTDEV/OSTDEV$n and MDSDEV/MDSDEV$n like other tests. OSTSIZE and MDSSIZE could be used to control the large LUN size.

We also need replace the run_acc_sm() function with a function just running mdsrate program to create large number of files using up the inodes on OST. Please refer to mdsrate-create.sh. 

The mmp codes could be removed.

The whole script could be composed of several sub-tests:

1) run llverdev on the raw LUN

2) run llverfs the OST ldiskfs filesystem

3) use up the free inodes on the OST filesystem

4) run llverfs on the Lustre filesystem

The new script should be added into lustre/tests/Makefile.am.
