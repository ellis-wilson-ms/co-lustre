LU-9119 lnet: Fix discovery threading

Discovery shouldn't block the thread which is currently processing
a message. This thread is an LND scheduler thread, which also
services transmits. If discovery blocks it then it opens the window
for the scenario where a ping initiated by discovery also placed
on the blocked LND scheduler thread queue. The LND scheduler
will never wake up to service that ping and therefore discovery
will never complete.

Added a non-blocking mode for discovery exercised from
lnet_select_pathway(). In this mode discovery request is queued
on the discovery thread and then returns right away. The message
is then queued on a pending queue for the peer. When discovery
completes the pending queue is visited and if discovery was
successful then messages are sent out to their targets, otherwise
lnet_finalize() is called with the returned error.
