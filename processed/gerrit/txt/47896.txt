LU-16025 llite: adjust read count as file got truncated

File read will not notice the file size truncate by another node,
and continue to read 0 filled pages beyond the new file size.

This patch add a confinement in the read to prevent the issue and
add a test case verifying the fix.

Patch Set 6:

This patch has a problem of changing the DLM getting behavior in read IO, it calls cl_glimpse_lock() which gets extent DLM locks from OSS covering the whole file, and in the process of the read, the read ahead path in osc_io_read_ahead() will find the read ahead pages always have dlm lock covered, so that even though the read call range does not come to this far, the read ahead will read these pages unnecessarily.
