--- Commit Message ---
LU-1199 build: Cleanup ldiskfs kernel config defines

Make it more clear what the purpose of the "CONFIG_LDISKFS_*"
defines are with a comment, and separate them from options
that do not necessarily originate in the kernel source. Also,
remove these settings from Lustre's build file, since Lustre
does not (directly) use them.

However, Lustre does use ldiskfs headers.  One of these options,
CONFIG_EXT4_FS_XATTR, is set explicitly by a new ldiskfs patch
so that the value is not undefined when Lustre includes the
header.

--- Message (Andreas Dilger, 2013-01-16 22:42:11.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

(1 inline comment)

Ah, I think I understand the problem here.  The Lustre code is including ldiskfs/ldiskfs/ldiskfs.h, and there is one dependency on CONFIG_LDISKFS_FS_XATTR for a field in struct ldiskfs_inode_info.  If this is not set, then the xattr_sem is missing, and the osd-ldiskfs code will be accessing/modifying fields in this structure incorrectly.

It is probably easiet to keep the CONFIG_LDISKFS_FS_XATTR in lustre-build-ldiskfs.m4 for now.  The conditional check (which is always true for ldiskfs and every vendor kernel) has been removed in the 3.7 kernel already.  Probably worth a comment that this should be removed after 3.7, or earlier if the supported vendor kernels backport the removal of this CONFIG option.
