Patch Set 1:

I didn't open a separate Jira (maybe I should have...). The original patch introduced a change to changelog_rec_snamelen() function which uses strlen to retrieve length of source name of rename changelog. That change is not working as expected as it requires changelog_rec_sname() to end with "/0" which appears not the case in current code. We saw this cause RENAME changelog corruptions on real filesystems.

I might be able to write a sanity test for this, but I need to look into it more.

I'll add all of this context to the commit message when I refresh.
