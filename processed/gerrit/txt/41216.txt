--- Message (Andreas Dilger, 2021-01-13 17:55:23.000000000) ---
Patch Set 1:

> I've got two separate tests verifying data read/write consistency
 > for unwritten exetents, but that needs to be improved to work with
 > any OST

The sanityn test_16() fsx test is actually very good for this, since it is making interleaved fallocate(), write(), truncate(), mmap(), O_DIRECT, etc. calls from multiple mount points with data verification. 

For this patch, it should be enough to create a file, write 100MiB of data from /dev/urandom, truncate to zero, then "fallocate -l 100M" and verify that md5sum of the file is "2f282b84e7e608d5852449ed940bfc51" (that is for 100MiB of zeroes).

The one thing we should add to the testing (in the next patch) is to verify that fallocate() reports unwritten extents in filefrag when falloc_zero_blocks=0.
