--- Commit Message ---
LU-5299 obdclass: avoid race during Server device mount

Ensure bdev is opened exclusive to ensure atomicity of all
operations during mount process and avoid any side effects
during potential race (dual mount, ...).

But 2.6.38 Kernel introduced some API changes, and
open_bdev_exclusive()/close_bdev_exclusive() need to be replaced
by blkdev_get_by_path()/blkdev_put() calls with explicit setting
of FMODE_EXCL mode.

This is good, but my understanding is that open_bdev_exclusive() is the old API and blkdev_get_by_path(..., O_EXCL) is the new API?  It would be better to use the new API in the code, so that the compat macros and configure checks can eventually be deleted without further changes to the code.

That said, it does appear that wrapping blkdev_get_by_path() inside open_bdev_exclusive() is easier to implement properly, so if the reverse isn't possible to do then I'm ok to keep the current patch.
