--- Commit Message ---
LU-1257 utils: add --clear-virgin for tunefs.lustre

Sometimes, server registration could be successful on MGS,
but the registration RPC returned error because of the MGS
revokeing config locks timeout or networking problems, that
will cause the server can never be registered properly
without erasing all server configs.

This patch adds "--clear-virgin" for the tunefs.lustre,
then user can use it to resolve above problem by clearing
the first time registration flags in the local mountdata.

--- Message (Andreas Dilger, 2012-04-06 06:06:26.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

The problem with the approach in this patch is that if, for some reason, a duplicate or incorrect index is specified for an OST then there would be two OSTs fighting for the single index value.

I don't think it is right to just print a message on the console and ignore this error, since this could lead to serious filesystem corruption (e.g. incorrect MDS orphan recovery deleting many thousands of objects on the wrong OST of the same index, or objects being created on both OSTs and then being unreadable afterward).

A better solution would be to add some kind of "--force-index" flag to mkfs.lustre/tunefs.lustre that sends an extra flag to the MGS, and only then will it ignore (possibly with only a small LCONSOLE_INFO(), not an error) the -EADDRINUSE error.
