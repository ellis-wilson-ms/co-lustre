LU-3270 statahead: refactor statahead code

* statahead thread should wait for async stat RPCs to finish
  before quit.
* simplify sai refcounting:
  - a refcount is taken in allocation, and will be put after
    starting statahead thread.
  - statahead thread holds one refcount.
  - AGL thread holds one refcount.
  - 'stat' process takes refcount before revalidating dentry against
    statahead cache.
* sa_entry is not refcounted because only scanner can remove it
  from statahead cache.
* add 'lli_sa_enabled' to indicate whether statahead is enabled for
  a directory, it will be cleared upon failure or directory close.
* many smallfixes and cleanups.

This is a backport to 2.5, because other changes between master
and 2.5 cause lots of conflicts, 5 statahead patches are backported
as one:
