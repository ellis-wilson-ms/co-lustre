LU-20046 sptlrpc: expose send_sepol via lctl get/set_param

send_sepol is currently only settable via sysfs
(/sys/module/ptlrpc/parameters/send_sepol) or modprobe.d, both of
which are node-local.  This makes cluster-wide SELinux enforcement
via nodemaps unreliable: a single unconfigured client silently
bypasses enforcement without any error.

Expose send_sepol as a proper lctl parameter under the sptlrpc
namespace so that it can be read and written via:

  lctl get_param sptlrpc.send_sepol
  lctl set_param sptlrpc.send_sepol=N
  lctl set_param -P sptlrpc.send_sepol=N  # persist via MGS config log

Implementation follows the enable_pmqos pattern: remove the static
qualifier from send_sepol in sec.c, add an extern declaration in
ptlrpc_internal.h, and register show/store kobject attributes on
sptlrpc_kobj in sec_lproc.c via sysfs_create_group().
