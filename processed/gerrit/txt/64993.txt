LU-20055 llite: Break file lease and increase iversion for NFS

NFS clients serve stale data when Lustre is NFSv4 re-exported
because of the following reasons:
1) nfsd grants NFSv4 delegation (based on file lease) on Lustre
   files so that NFS clients trust their data cache and never
   check change_attr.
2) NFS clients don't invalidate data cache due to the matched
   iverson even if NFSv4 delegation has been recalled.

Fix the issue by breaking file lease and increasing iversion
during Lustre's LDLM lock cancellation.
