LU-4199 libcfs: remove assertion of spin_is_locked()

spin_is_locked() is always false when the platform is
uniprocessor and CONFIG_DEBUG_SPINLOCK is not enabled.
This patch replaces its assertion by assert_spin_locked().

Patch Set 2:

Hi James,

On a uniprocessor platform which do not have CONFIG_DEBUG_SPINLOCK enabled (which is true for Raspberry Pi kernel), the spin_is_locked() will always be false. The related codes are in kernel/include/spinlock_up.h. I don't have any other good idea to fix this. And it doesn't seems right to assume that the kernel enables CONFIG_DEBUG_SPINLOCK either.

Do you have any good idea?

Thanks!
Li Xi

--- Message (James Simmons, 2013-11-05 15:44:46.000000000) ---
Patch Set 2:

Looking at this problem I believe the best solution to be assert_raw_spin_locked(). If it is not defined for your kernel version we could backport it. Look at the latest kernel source tree spinlock_*.h files to see what I mean.

--- Message (Li Xi, 2013-11-06 01:01:47.000000000) ---
Patch Set 2:

Hi James,

The kernel of Raspberry Pi (3.6.11+) defines assert_raw_spin_locked() and assert_spin_locked() correctly. However the Lustre is not using this API, it calls spin_is_locked() instead, which is false all the time on my kernel. And LASSERT(spin_is_locked(...)) will always produce a LBUG. Yeah, as another solution, we could change the assertions in Lustre to assert_spin_locked() using the APIs befined by kernel. But it needs change in a lot of places anyway, right?
