--- Commit Message ---
LU-2744 build: fix race issues thanks to oap_lock

Fix 'data race condition' defects found by Coverity version
6.5.0:
Data race condition (MISSING_LOCK)
Accessing variable without holding lock. Elsewhere,
this variable is accessed with lock held.

This patch is dedicated to code fragments involving oap_lock.

Patch Set 1: I would prefer that you didn't submit this

Getting and dropping the spinlock for every page in the RPC can add a lot of overhead. Better to grab it once at the start of the section and hold it until after the loop. The number of pages in the RPC is limited, so it shouldn't be held for too long.
