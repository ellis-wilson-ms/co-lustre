LU-19597 tbf: Fix using this_cpu_ptr() in preemptible context

nrs_tbf_cli2str() can be called in preemptible context. This triggers
the following message with kernel compiled with CONFIG_DEBUG_PREEMPT:

BUG: using smp_processor_id() in preemptible [00000000] code:
ll_ost_io00_009/12480
caller is __nrs_tbf_cli_debug.constprop.18+0x117/0x290  [ptlrpc]
Call Trace:
 <TASK>
 dump_stack+0x6e/0xa0
 check_preemption_disabled.cold.0+0x46/0x51
 __nrs_tbf_cli_debug.constprop.18+0x117/0x290 [ptlrpc]
 ? rcu_read_lock_held_common+0x9/0x30
 nrs_tbf_res_get+0xf06/0xfd0 [ptlrpc]
 nrs_resource_get+0x51/0xb0 [ptlrpc]
 nrs_resource_get_safe+0x7e/0x100 [ptlrpc]
 ptlrpc_nrs_req_initialize+0x2e/0x80 [ptlrpc]
 ptlrpc_main+0x22d3/0x34e0 [ptlrpc]
 ? __kthread_parkme+0x47/0xa0
 ? _raw_spin_unlock_irqrestore+0x5d/0x70
 ? ptlrpc_wait_event+0x3a0/0x3a0 [ptlrpc]
 kthread+0x16e/0x1a0
 ? set_kthread_struct+0x40/0x40
 ret_from_fork+0x24/0x30
 </TASK>

this_cpu_ptr() calls smp_processor_id() with CONFIG_DEBUG_PREEMPT to
dereference the per-CPU debug buffer.
This patch replaces this_cpu_ptr() with raw_cpu_ptr() to silent this
warning. Disabling the preemption should not be necessary for this use
case (racy print to display a TBF class in Lustre debug).
