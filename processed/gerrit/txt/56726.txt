--- Message (Andreas Dilger, 2024-10-20 18:49:14.000000000) ---
Patch Set 5:

> do you mean to skip the test if free space less than 50% ?

It should skip calling wait_delete_completed() if free space > 50%.  In most cases (normal test run, or first few iterations of looping) then it won't add any overhead.   If there is less free space than 50% (eg. deletes accumulating), it is probably enough to call wait_delete_completed() once and then the free space will grow again.

actually wait_delete_completed() is not that dumb :)
it checks for pending updates and returns immediately if can't find such

	# find MDS with pending deletions
	for node in $(mdts_nodes); do
		changes=$(do_node $node "$LCTL get_param -n osc.*MDT*.sync_*" \
			2>/dev/null | calc_sum)
		if [[ $changes -eq 0 ]]; then
			continue
		fi
		mds2sync="$mds2sync $node"
	done
	if [ -z "$mds2sync" ]; then
		wait_zfs_commit $SINGLEMDS
		return 0
	fi
