LU-18635 ptlrpc: add max_mb_per_rpc_read/write parameters

Added "max_mb_per_rpc_read" and "max_mb_per_rpc_write" parameters to
get and set RPC size in terms of megabytes instead of number of pages
with "max_pages_per_rpc" parameter.

New parameters will allow the user to pass decimal values
(e.g.: 0.00390625G, 3.5M, etc.).

Parameter "max_pages_per_rpc" can now accept size values smaller
than 1M (e.g.: 768K, 0.5M, etc.).

Fix longstanding bug in sanity test_49 trying to get OST size
using 'lfs df' from OSS instead onlf client.  Instead, use new
'lfs df' options to get size directly.

Fixes: 9fe4b52ad2 ("LU-1030 osc: new IO engine implementation")

Case max_mb_per_rpc_show() function:
    E.g: PAGE_SIZE is 4096 and number of pages 384
    PAGES_TO_Mib(): 1M ((384) >> (20 - 12))
    Wrong M size given by PAGES_TO_Mib(), values in lesser 20 bits were discarded

Case max_mb_per_rpc_store() function:
    E.g. recieved value: 1.5MiB (1572864 Bytes)
    Convert it to MiB: 1M (1572864 >> 20), lesser 20 bits were discarded
    MiB_TO_PAGES() will give wrong number of pages

PAGES_TO_MiB() and MiB_TO_PAGES() were added to round up the values to the nearest MiB (https://review.whamcloud.com/c/fs/lustre-release/+/34317). Here our purpose is to give the user flexibility of setting and getting values in decimal.
