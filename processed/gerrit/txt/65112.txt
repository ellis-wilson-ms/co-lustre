LU-19901 nodemap: support incremental RBAC role updates

The lctl nodemap_modify command now supports incremental RBAC updates:
  lctl nodemap_modify --name <nm> --property rbac --value +<role>
  lctl nodemap_modify --name <nm> --property rbac --value -<role>

Multiple roles can be updated in a single command:
  lctl nodemap_modify --name <nm> --property rbac=+<role1>,-<role2>

This allows modifying roles without specifying the full set.

## Fix Applied

**What was fixed:**
Changed line 362 in `libcfs/libcfs/debug.c` from:
```c
newmask = oldmask;
```
to:
```c
newmask = *oldmask;
```

**Rationale:**
The compile error occurred because the patch added new code that handled the "NONE" special value in the `cfs_str2mask()` function. When the operator is '-' and the value is "NONE", the intent is to preserve the existing mask value. However, the code incorrectly assigned the pointer `oldmask` directly to the `u64` variable `newmask`, which caused a type mismatch error ("makes integer from pointer without a cast").

The fix dereferences the pointer using `*oldmask` to get the actual `u64` value that `oldmask` points to, which is the correct behavior and matches the pattern used throughout the rest of the function (see line 304 where `newmask` is initialized with `*oldmask`).
