LU-6319 tests: Remove tests from the ALWAYS_EXCEPT list

Individual tests on the ALWAYS_EXCEPT list are excluded from being
executed. Normally, the list is used to temporarily exclude a test
that is failing frequently and interfering with test completion. The
intention is to exclude the test from running while the problem is
diagnosed and fixed. In many cases, tests are added to the
ALWAYS_EXCEPT list and left on the list even after the root problem
was corrected.

The following tests will be removed from the ALWAYS_EXCEPT list:
recovery-small tests 24a, 24b, 52, 60
replay-single test 73b
sanity tests 42a, 42c, 42d, 45, 51d, 65ic, 68b, 78, 79, 80, 180, 184c
sanityn tests 14b, 18, 29, 35, 36, 51a

The ALWAYS_EXCEPT line in sanity-krb5sh is corrected to include the
environment variable 'SANITY_KRB5_EXCEPT'.
