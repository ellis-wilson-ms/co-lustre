--- Commit Message ---
LU-7139 test: Fixing file size calculation test_78 in sanity

When using opaque calculation of file size, the test used to fail in
lack of disk space, more robust file size calculation should be used.
So using probing technique for calculating the file size.
First write operation is done using dd and file size equal to
returned bytes by dd is set to do direct I/O. It makes more
sense because direct I/O is well defined for overwrite case.
