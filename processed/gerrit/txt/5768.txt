LU-2850 compat: posix_acl_{to,from}_xattr take user_namespace

Commit 5f3a4a28 (v3.7) added a struct user_namespace argument to
posix_acl_{to,from}_xattr. Pretty much every caller just uses
&init_user_ns. We mask it off in user mode so we don't need to
create a fake structure only to ignore it.

Patch Set 2: Looks good to me, but someone else must approve

The previous version built ok with SLES11 SP2 but not with 3.7. It was missing definition of &init_user_ns and HAVE_POSIXACL_USER_NS only matters for the linux version. The darwin and liblustre versions should always mask out the first argument. It turns out init_user_ns is declared in a few includes, so we use <linux/utsname.h> since it'll always be there even if the the init_user_ns declaration isn't. Saves us another #ifdef.
