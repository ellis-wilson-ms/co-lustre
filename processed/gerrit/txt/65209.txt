LU-20111 scrub: avoid O/1/LAST_ID LMA xattr FID corruption

osd_scan_lastid_seq() and osd_scan_O_seq() use kstrtoull(), which
expects NUL-terminated strings, but the names that are read from disk
using do_osd_ldiskfs_filldir() are not NUL-terminated and use a name
length parameter, instead.

OI scrub uses osd_scan_dir() multiple times
via osd_scan_O_main(), osd_scan_ml_file() and osd_scan_last_id_main()
and reuses osd_thread_info::oti_it_ea_buf without clearing the buffer
between calls, which can leave stale data in the buffer. When the O/1
entry is read, it can be placed at a point in the buffer that
contained an O/ directory named by a sequence that has '0' in its
second byte; the buffer iterator is then incremented to the next
8-byte boundary in do_osd_ldiskfs_filldir() and the next direntry is
copied into the buffer. The first field in struct osd_it_ea_dirent is
the FID and because that is zeroed for all directory names in O/ on
OSTs, it effectively NUL-terminates the name of the previous directory
entry in the buffer, which was the last field to be written in
struct osd_it_ea_dirent. This can shorten the previous name buffer
into "10", which is later passed into osd_scan_lastid_seq(), allowing
the !fid_seq_is_local_storage() check to fail; the corrupted sequence
is then checked against the existing LMA xattr self FID sequence of
the O/1/LAST_ID file, they are found to be different and so OI scrub
attempts to repair the LMA xattr's sequence by overwriting it with the
sequence from the corrupted name in the buffer, which actually
corrupts the LMA xattr FID. This can prevent subsequent mounts of the
OST, as osd_check_lma() fails when called via
local_oid_storage_init(), at MGC setup time.

Use a temporary buffer to copy the direntry name, NUL-terminate it and
use the temporary buffer in the kstrtoull() calls, to avoid this
issue.
