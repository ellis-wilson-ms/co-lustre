--- Commit Message ---
LU-16931 ptlrpc: allow disable import deactivation

If "lctl set_param fail_loc=0x537" is set, it will not be possible to
deactivate an import via "lctl set_param mdc.*.active=0" or similar
commands. It will print a warning with process name, pid, parent pid.

There was a rogue root-level system management script that was setting
active=0 and causing the client to be evicted from the server.  This
allowed debugging the issue and preventing the client eviction until
the root cause could be found and fixed.
