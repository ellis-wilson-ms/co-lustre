LU-16357 general: collective trace dump

This patch adds support to dump debug log on multiple nodes by
command or code: a virtual config called 'trace' is added, and
MDTs/OSTs add it to watch list by default, while clients not, but
it can be added/removed by proc, then upon config lock revoke, all
registed nodes will dump debug logs.

To turn on/off collective trace on client:
lctl set_param llite.*.collective_trace=1/0

To trigger collective trace dump by command:
run 'lctl dk --collective' on MGS.

To trigger in the code:
call libcfs_debug_cltv_dumplog() in any stack above libcfs, but it
only takes effect after MGC is started.

Add sanity 60k 60l and 60m.
