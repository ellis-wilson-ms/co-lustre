Patch Set 2: Code-Review-1

I think we're losing sight of whatever the goals are. lustre_idl.h is the wire protocol header for Lustre. With that being said, what is the problem with keeping this function in that header?

--- Message (James Simmons, 2016-08-05 17:27:12.000000000) ---
Patch Set 2:

lustre_idl.h is a UAPI header. As such the rule is anything in the header must be used by both user land and kernel space. If not it has to be removed so if you want to keep it find some use in user land.

The next question  is how is this function used to generate on the wire information?
It appears to me to only be used by LFSCK to validate xattr storing the MDS striping info. I'm I missing something?

--- Message (Andreas Dilger, 2016-08-06 02:51:07.000000000) ---
Patch Set 2:

The UAPI header is lustre_user.h, while lustre_idl.h describes structures for the protocol between clients and servers or between two servers. In some cases these same structures are also used in userspace, but primarily it is intended to be for the network protocol.
