--- Commit Message ---
LU-15234 lnet: add mechanism for dumping lnd peer debug info

Add ability to dump lnd peer debug info:
	lnetctl debug peer --nid=<nid>

The debug info is dumped to the log as D_CONSOLE by the respective
lnd and can be retrieved with "lctl dk" or seen in syslog.
This mechanism has been added for socklnd and o2iblnd peers.

Patch Set 2: Code-Review-1

I'm not sure it's a good idea to have this print to console. At scale, Lustre servers will have thousands of peer entries. Printing that much information to the kernel log buffer can be problematic.

I really think this should just use D_NET. If an admin wants this information to be printed to the kerne log then they can add net to the printk mask prior to running the command:

  lctl set_param printk=+net
  lnetctl debug peer --nid=<nid>
  lctl set_param printk=-net
