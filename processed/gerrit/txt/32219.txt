--- Message (Andreas Dilger, 2018-05-01 23:01:58.000000000) ---
Patch Set 2: Code-Review+2

Converting the CLASSERT()s is thankfully pretty straight forward, since any errors will be caught at build time (assuming the code is not #ifdef'd out).

Converting LASSERT() and LASSERTF() to BUG_ON() will be more complex, and we'll lose useful information.  I always disliked the kernel BUG_ON(), since it only prints the line number and doesn't give any opportunity like LASSERTF() to provide additional useful data like what the actual values were.
