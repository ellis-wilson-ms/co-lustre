--- Commit Message ---
LU-18939 llite: fix readahead pct to use totalram

This patch allows readahead percentage to be calculated
using total system RAM instead of the readahead limit.

It also makes setting the parameters more robust and
consistent by truncating the values to the maximum limit
exceeded rather than returning an error to the user and
have them work out what the actual limit is.

It also updates test_854 according to new behaviour
and actually runs this subtest.
