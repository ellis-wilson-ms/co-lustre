Patch Set 6: Code-Review-1

Charlie, it would be useful to update the "Test Parameters" wiki page with the limits for mdscount (or mdscount+osscount) that can be run without intervention, and how many by special request (as seems to be the case here since it looks like the test was held until you released it).

It seems that 10 MDTs/MDS (14MDS) is the safe limit for our current VM config.  With 11 MDTs/MDS (12MDS) one of them OOM at startup, and with 4 MDTs/MDS (32MDS) there were problems with SSH/mcmd connecting to the VMs.

However, it looks like there are some issues in test-framework when there are more than 10 MDTs specified that need to be fixed before this will actually test properly.
