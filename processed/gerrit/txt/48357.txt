LU-16112 build: ki_complete removed unused argument

Linux commit v5.15-rc6-145-g6b19b766e8f0
   fs: get rid of the res2 iocb->ki_complete argument

Prior to 4.1 Linux provided an aio_complete(iocb, res, res2)
which propagated res2 to io_event.res2. This functionality
migrated to iocb->ki_complete().

Provide a wrapper around iocb->ki_complete() to use
aio_complete() or iocb->ki_complete() as appropriate.

I am torn by how to handle this change. ki_complete() was modified in the kernel to remove the requirement of res2 which is largely unused and usually passed a 0. The same applies with the Lustre code. In the couple of instances where I see aio_complete() used, we pass a 0 for res2. I want to open the question up to others but did we want to only modify ki_complete() or did we also want to also apply the change to remove res2 in its parent function aio_complete() and in the case where HAVE_KIOCB_COMPLETE_2ARGS is not defined within the body, just pass a 0?
