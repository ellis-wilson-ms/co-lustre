LU-14760 llite: restart clio for AIO if necessary

If the clio needs to be restarted from where it left off,
do it for AIO as well, so we don't end up with short IO.
Limit thr number of retries to 1000, to avoid potential
issues if the loop is stuck forever.

--- Message (Patrick Farrell, 2021-07-19 15:46:26.000000000) ---
Patch Set 2:

There is not.

It only restarts in a few limited cases, I believe all related to layout - either changes or not having the layout lock.

The question is how many restarts is reasonable?  Under significant contention, we could probably get a lot of restarts before succeeding.

I don't think a limit is unreasonable as a way to avoid getting stuck, but I think it should be very high because this isn't a condition I've ever seen and it converts what is *currently* an expected retry to a hard failure.

--- Message (Patrick Farrell, 2021-07-19 15:50:16.000000000) ---
Patch Set 2:

Since I should offer a specific opinion...

Something like 1000 restarts?  This is a way to avoid hanging in the face of some unexpected behavior, where we can never acquire the layout lock or where the server never makes the layout change we need.  It should never happen, and the behavior today would be to loop forever.

For example, this would occur if the server did not instantiate new layout when required, and just sent back the original layout.  (I caused this sort of bug when developing self extending layouts, which is why I know.)
