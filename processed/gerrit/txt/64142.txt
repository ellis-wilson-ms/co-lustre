--- Commit Message ---
LU-19826 lov: increase maximum mirror count to 256

Add LUSTRE_MIRROR_COUNT_DEF=16 and grow LUSTRE_MIRROR_COUNT_MAX=256.

Add a server-side tunable lod.*.mirror_count_max (defaulting to
LUSTRE_MIRROR_COUNT_DEF=16) so the mirror limit can be increased
dynamically without needing to update all clients.  This maintains
compatibility with existing clients that may have issues processing
layouts with many mirrors, while allowing the limit to be raised
when all clients support it.

The client-side lfs uses LUSTRE_MIRROR_COUNT_MAX as a sanity check
and lets the server enforce the real limit via the tunable.

EC can double the number of mirrors (data + parity components), so
expanding this limit is needed for EC support.

Add a man page for the new mirror_count_max parameter and sanity-flr
test (test_28) that verifies creating a file with 256 mirrors.
