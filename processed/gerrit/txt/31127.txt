--- Commit Message ---
LU-10570 obd: use ktime_t for statfs handling

The comment 'use the value of cfs_time_current + HZ' for
obd_statfs() and obd_statfs_async() gave the impression
that stats were refreshed when more than one second passed
thus making it safe to use time64_t. This is not true and
we need to use finer time resolution. Migrate to ktime_t
instead. Remove the outdated comment as well.
