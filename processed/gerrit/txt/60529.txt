LU-19234 lst: Fix leak under emitter/parser delete (1)

This is first series of patch with fixes memory
leak which is leaked when yaml_emitter_delete()
and yaml_parser_delete() is called. All calls to
yaml_emitter_delete() will be replaced with
yaml_emitter_cleanup() and yaml_parser_delete()
will be replaced with yaml_parser_cleanup() which
is introduced in this patch.

This patch gets the reference back then release
memory after call to yaml_emitter_delete(). To
do this both calls to delete emiiter and memory
held is placed under a single call to
yaml_emitter_cleanup().

Similarly, it gets the reference back then release
memory after call to yaml_parser_delete(). To
do this both calls to delete emiiter and memory
held is placed under a single call to
yaml_emitter_cleanup().
