if (S_ISREG(mdd_object_type(sobj)))

1. The code on lines 3912-3913 (`fid_to_lmm_oi(...)` and `lmm_oi_cpu_to_le(...)`) was originally inside the if block
2. There was still a closing brace `}` on line 3914 from the original if block
3. Without the opening brace, the compiler interpreted lines 3912-3914 as being outside any function scope, leading to all the compilation errors about variables being undeclared and statements appearing at file scope

**The Fix:**
I added back the missing opening brace `{` on line 3909, so the corrected code is:
```c
if (S_ISREG(mdd_object_type(sobj))) {
    LASSERT(tobj != NULL);

    fid_to_lmm_oi(mdd_object_fid(tobj), &oi);
    lmm_oi_cpu_to_le(&oi, &oi);
}
```

**Rationale:**
The logic requires that the `fid_to_lmm_oi()` and `lmm_oi_cpu_to_le()` calls only execute when the source object is a regular file (`S_ISREG`), which is consistent with the original intent of the code. The patch correctly changed the condition from checking `attr->la_mode` to `mdd_object_type(sobj)`, but simply needed the proper brace structure to maintain the conditional execution of the code block.

This fix resolves all the compilation errors by ensuring the code has proper C syntax and that all statements are within appropriate function scope.
