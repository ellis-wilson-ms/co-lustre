> Uploaded patch set 2.

Just added a test.
Here is my analysis of modifications required in liblustreapi.c to gather correct btime.
btime is gathered by the get_lmd_info_fd() function from MDS.
However, when obtaining the updated stats from the OSTs, 'stat' calls are used. The stat calls don't have a btime field. Therefore they need to be replaced with 'statx' calls. 
Also convert_lmd_statx() need to modified to add a btime assignment to lmd_v2 btime.
Another modification required is the find_time_check() performed after convert_lmd_statx() does not include btime verification (on line 6029). So that needs to be added in find_time_check(), so that it rechecks the btime conditions as well.
Like suggested before, I plan to add a wrapper function to statx (something like, get_updated_stats()) that use statx calls if statx is supported, otherwise defaults to stat calls.
I had a question: I see there is statx_supported function in test-framework.sh which is what I used in my test to check for statx support. Is it a good way to use this to check statx support in my wrapper function or is there a better way?
