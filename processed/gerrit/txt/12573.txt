--- Message (Andreas Dilger, 2014-11-06 02:08:22.000000000) ---
Patch Set 1: Code-Review-1

The D_HA Flag is enabled by default in production, and this isn't really related to recovery in any case.

It would be better to log this using an appropriate mask like D_CACHE and then enable this flag in the tests that you are trying to debug instead of adding overhead for all production users of Lustre.

--- Message (Fan Yong, 2014-11-06 02:22:08.000000000) ---
Patch Set 1:

> The D_HA Flag is enabled by default in production, and this isn't really related to recovery in any case.
> It would be better to log this using an appropriate mask like D_CACHE and then enable this flag in the tests that you are trying to debug instead of adding overhead for all production users of Lustre.

We have used "D_HA" for osd_sync(), it is found in the LFSCK test log. On the other hand, dt_sync() is not often triggered, even though the log for dt_sync() uses "D_HA", it should not cause a lot of debug message.

If we replace the "D_HA" with "D_CACHE" here and enable "D_CACHE" in test, then the potential trouble will be:
1) The log size in test, because we have more other use cases for "D_CACHE" in Lustre.
2) The performance in VM environment, we have met similar trouble before.
