LU-14361 statahead: fname statahead pattern w/o opendir

The current statahead mechanism works as follows:
1. A user opens a directory via opendir(). It will authorize the
   statahead.
2. The user calls readdir() to obtain the name and inode number
   for the children file within the directory.
3. The user does stat() call on the dentries one by one.
4. Kernel Lustre detects the stat() entry is the first entry in
   hash order of the file name or matched the fname statahead
   pattern. The client will lanch a thread to do readdir() and
   getattr()/AGL asynchronous.
5. The user closes the directory and deauthorizes the statahead
   processing. It will notify the statahead to quit and destroy.

However, for mdtest stat(), i.e. mdtest.$i,  it will do stat()
without calling opendir() on the children files with sequential
file names.

This patch adds the support to do statahead for these kinds of
applications, which can be optimized, but without opendir()/
close() to start/stop statahead thread explicitly.

Instead, the statahead thread will stop and quit when found
that there was no acitivy for more than a certain time period
(i.e. 30 seconds).
