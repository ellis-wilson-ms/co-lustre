LU-0000 flr: add LCME_FL_INCONSISTENT component flag

Add new component-level flag LCME_FL_INCONSISTENT for FLR
mirror state management. This flag indicates mirrors with
data integrity issues that should be excluded from write
operations until explicitly resolved.

Infrastructure changes:
- Add LCME_FL_INCONSISTENT (0x800) to lov_comp_md_entry_flags
- Update LCME_KNOWN_FLAGS to include new flag
- Add "inconsistent" string name to comp_flags_table
- Include flag in wiretest and wirecheck validation

Behavior implementation:
- Skip writes to inconsistent mirrors in DIO operations
- Clear INCONSISTENT flag during resync completion
- Preserve STALE flag when INCONSISTENT is also set during
  WRITE_PENDING to RDONLY transitions

This provides a mechanism for marking problematic mirrors
that require administrator intervention or explicit resync
operations before being restored to active service.
