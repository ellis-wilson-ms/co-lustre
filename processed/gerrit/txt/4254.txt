Patch Set 5: (11 inline comments)

I was also thinking, that the way it works currently is the output is formed and printed in the kernel.  I'm thinking it might be better to form a cYAML block (which is a c representation of a YAML block) pass that up to the user space and let the user space use a common library to print out the YAML block.  This way this cYAML block can be stored/queried/printed, etc by the application.  For example IML can do that, by calling a utility which returns the cYAML block and can store it and parse it later on.  Instead of doing text parsing again in user space.  Also this gives the advantage of not doing any text manipulation inside kernel space (which as far as I understand is desirable)
