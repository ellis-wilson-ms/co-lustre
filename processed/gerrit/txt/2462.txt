--- Message (Andreas Dilger, 2014-02-26 22:40:17.000000000) ---
Patch Set 3: Code-Review-1

(4 comments)

The problem I see with this patch, as Keith pointed out, is that this would completely serialize object allocation on the MDS.  It probably makes more sense to opportunistically skip a number of OSCs based on the requested stripe count (and assume they are all available) and then drop the spinlock again.  If it turns out that some of the stripes are not available then the allocation of replacements can be handled as an exception as it is done already.
