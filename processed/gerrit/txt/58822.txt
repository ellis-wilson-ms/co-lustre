LU-16767 mdt: Allow jobID fields widths

Modify the function jobid_interpret_string in order to allow admin to
specify the widths of parameter printed by
lctl get_param mdt.*.job_stats.
By specifying the parameter jobid_name, the admin can truncate the
fields.
For exemaple, the format "%3e.%u.%6h" will print in job_stats
the 3 first characters of the executable name, a dot, the whole uid, and
the 6 first characters of the hostname.
If no digit is passed before the letter, it will print the whole field.

Andreas, I would prefer directly the printf format to truncate a string (e.g: "%.5s").
This can confuse the admin ("%5s" means padding of 5 chars).

Do we need to support truncate from the end? Is it to get the node number (e.g: "server123" with jobid_name="%-3H" would give "123")?

If not, we can directly get the prefix and use snprintf to parse it:

        if (*jobfmt != '%')                            
                goto cont;                     
                                                    
        sscanf(jobfmt, "%%.%*[-0-9]%*[hHje]%n", &sfmt);                           
        if (!sfmt)                                           
             sfmt = 2;                                  
        fmt = strndup(jobfmt, sfmt);                   
        if (!fmt)                                   
                return -ENOMEM;                           
                                                    
        switch (fmt[sfmt - 1]) {                    
        case 'e':                                   
                fmt[sfmt - 1] = 's';                
                l = snprintf(jobid, joblen, sfmt, 
                            jobid_print_current_comm(jobid, joblen));               
                break;
