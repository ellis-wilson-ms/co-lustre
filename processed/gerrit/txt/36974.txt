>  Do we even need LNET_MD_KIOV anymore since its the only thing LNet handles now?

Yes we do.  'struct lnet_md' can be either a list of page/offset/size (LNET_MD_KIOV) or a single virtual memory buffer.
lnet_md_build converts these into a 'struct lnet_libmd', which has only a single format.
