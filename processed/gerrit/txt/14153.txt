This patch introduces the maximum modify RPCs in flight variable of
a mdc client obd device. Its value is set from connection flag and
and connection data. It can later be tuned through the
max_mod_rpcs_in_flight procfs file.

--- Message (Gregoire Pichon, 2015-04-27 12:43:46.000000000) ---
Patch Set 5:

The code ensures max_mod_rpcs_in_flight is strictly less than max_rpcs_in_flight.

This is done:
- at initialization: OBD_MAX_RIF_DEFAULT - 1
- in /proc write handler routines obd_set_max_rpcs_in_flight() and obd_set_max_mod_rpcs_in_flight()

This is required to ensure the extra close request will be able to have a slot from rpcs_in_flight.

For example:
- the mdc sends max_mod_rpcs_in_flight non-close RPCs in parallel. it means these RPCs consums the same number of rpcs_in_flight slots.
- the mdc wants to send an additional close RPC in parallel
- the code allows the max_mod_rcps_in_flight limit to be exceeded for one close RPC
- the mdc need to have 1 rpcs_in_flight slot available for this close RPC. This requires to have max_mod_rpcs_in_flight < max_rpcs_in_flight.

Alex, I hope this answer your questions.
