LU-19839 llite: add persistent nohybrid I/O flag

Add a persistent flag to prevent hybrid I/O switching
for specific files. This uses the NOTAIL flag (0x00008000)
which is not used by Lustre/ext4 and unlikely to be used
in the future.

The flag is set via ladvise interface for discoverability
but uses FS_IOC_SETFLAGS internally for implementation.
When set, hybrid I/O will not switch the file from
buffered to direct I/O regardless of I/O size.

Changes:
- Define LUSTRE_NOTAIL_FL and LUSTRE_NOHYBRID_FL
- Add LUSTRE_NOTAIL_FL to LUSTRE_FL_USER_MODIFIABLE
- Implement LU_LADVISE_NOHYBRID using FS_IOC_SETFLAGS
- Update ll_update_inode_flags to handle NOTAIL flag
- Add test_119k to verify nohybrid flag functionality

**The Problem:**
The patch renamed the static function `fileattr_set()` to `ll_fileattr_set_flags()` and made it non-static (exported) so it could be called from `lustre/llite/file.c` for the new nohybrid functionality. However, there was a call site in the `ll_iocontrol()` function (line 3437) that was not updated to use the new function name.

**The Fix:**
Changed line 3437 from:
```c
RETURN(fileattr_set(file->f_inode, flags));
```

to:
```c
RETURN(ll_fileattr_set_flags(file->f_inode, flags));
```

**Rationale:**
This is a straightforward function rename fix. The patch author renamed `fileattr_set` to `ll_fileattr_set_flags` to:
1. Make it accessible from other files (removed `static` keyword)
2. Follow Lustre naming conventions (prefix with `ll_` for llite subsystem functions)
3. Make the function name more descriptive

The call in `ll_iocontrol()` within the `#ifndef HAVE_FILEATTR_GET` block handles the `FS_IOC_SETFLAGS` ioctl for older kernels that don't have the `fileattr_get/fileattr_set` VFS interface. This code path needs to use the same renamed function to maintain consistency and fix the compilation error.
