--- Commit Message ---
LU-14739 quota: nodemap squashed root cannot bypass quota

When root on client is squashed via a nodemap's squash_uid/squash_gid,
its IOs must not bypass quota enforcement as it normally does without
squashing.
So on client side, do not set OBD_BRW_FROM_GRANT for every page being
used by root. And on server side, check if root is squashed via a
nodemap and remove OBD_BRW_NOQUOTA.

My guess, though I do not know the details, is that the "always set for root" is because we don't want to discard pages that are in cache but delayed to be written and flushed by some daemon thread on the client. 

That said, I agree that it doesn't make sense to set "from grant" for all root pages, that would likely cause grant accounting errors (there are several bugs open about this), and possibly cause non-root processes to run out of space. It also invalidates much of the grant testing we do, if everything is running as root.

Oleg, this is not strange, this is done on purpose :)

If all servers (OSS, MDS, MGS) run on the same node, it is impossible to tell if they get synced with the MGS regarding nodemap configuration. So wait_nm_sync() (moved to test-framework.sh) is modified to just wait an arbitrary 30 seconds in this particular configuration. It was a miracle that the tests were passing without that, or at least they were not exercising what they were meant to (because nodemap configuration was not actually the expected one).

Maybe lowering wait time from 30 seconds to 20 would be enough?
