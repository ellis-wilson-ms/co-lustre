LU-19157 nodemap: introduce range type

Introduce a type for nodemap ranges. The regular type is for usual
nodemap ranges, and a new type is used for nodemap ranges that define
NID ban lists.
NM_RANGE_FL_REG = 0x0
NM_RANGE_FL_BAN = 0x1
The first 4 bits of nk_range_id are now reserved for the range type.
This is fine for upgraded systems that are not aware of the nodemap
range type, as the value for the regular nodemap range type is 0.
And this leaves us with 28 bits to store the range id, for a max range
id of 268435455.
