LU-18813 osd-wbcfs: refactor osd_read()/write()

Currently, osd_read()/write() are implemented using
cfs_kernel_read()/write(). This introduces a requirement
to periodically flush orphaned file descriptors created
by osd_alloc_file_pseudo().

To avoid this, we can access the page cache pages directly
(similar to the bulk read/write path). This allows us to
remove osd_alloc_file_pseudo() and the related delayed
fput() flushing code.
