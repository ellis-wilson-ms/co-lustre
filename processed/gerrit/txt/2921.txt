--- Message (Isaac Huang, 2012-06-06 23:57:02.000000000) ---
Patch Set 1:

Liang, the patch looks good. But, it doesn't make much sense to me to have the lnet_*_free_locked() functions. E.g.:
lnet_drop_delayed_put()
......
    lnet_msg_free_locked(msg);
    LNET_UNLOCK();
}

Here it's unnecessary to hold the lock while freeing the msg. I'd rather UNLOCK() earlier:
    LNET_UNLOCK();
    lnet_msg_free(msg);
}

There's quite some places like this. I don't like calling LIBCFS_FREE() with LNET_LOCK() - LIBCFS_FREE() is not cheap. I'd suggest to:
1. Remove all lnet_*_free_locked() functions.
2. Replace their call sites with: UNLOCK(); lnet_*_free(); In some places the changes aren't that straightforward, but I think it's worthwhile.
3.
lnet_xx_free()
{
/* Never call me with lock held */
#if FREE_LIST_STUFF
   lock_some();
   free_list_free(xx);
   unlock_some();
#else
   LIBCFS_FREE(xx);
#fi
}

So that locking is only used when LNET_USE_LIB_FREELIST defined, which is never defined for KERNEL, and for KERNEL all unnecessary LOCK are avoided.
