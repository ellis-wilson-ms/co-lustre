--- Commit Message ---
LU-18983 nodemap: support obdfilter-survey with nodemap

When running obdfilter-survey case=disk, no nodemap is associated with
the export, as the echo client is setup local to the server and the
ofd is accessed directly. So allow no nodemap for a ECHO_UUID client.
When running obdfilter-survey case=network, the echo client setup on
remote node (usually client) connects to the server via the echo obd.
So the echo obd needs to add the remote nid to the nodemap.

Patch Set 2:

There are 3 cases for obdfilter-survey:
* case=disk
The echo client is setup local to the server and the ofd is accessed directly. No NID is associated with the echo client, so we need to allow no nodemap for a ECHO_UUID client.
* case=network
The echo client setup on remote node (usually client) connects to the server via the echo obd. So the echo obd needs to add the remote nid to the nodemap in echo_connect(), called from target_handle_connect()->obd_connect().
* case=netdisk
Support for this case has been removed by patch "LU-10994 test: remove netdisk from obdfilter-survey" https://review.whamcloud.com/47239 3 years ago.
