--- Commit Message ---
LU-19008 hsm: add locking for coordinator thread stop

There is no locking around thread stop, which can race between
mdt_coordinator() and mdt_hsm_cdt_stop() and with use-after-free
during unmount.  Add locking to avoid this.

Fixes: 4512347d6c ("LU-16356 hsm: add running ref to the coordinator")

The coordinator failed to start and exit because the mdd took too much time to start. Then, kthread_stop() failed because the thread have already exited (the task has been freed).

The fix looks good. But I would prefer the get_task_struct()/kthread_stop_put() approach to take an additional ref on the task, so only mdt_hsm_cdt_start()/mdt_hsm_cdt_stop() handle cdt_task.
