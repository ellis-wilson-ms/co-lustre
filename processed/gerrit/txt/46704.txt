--- Commit Message ---
LU-13343 gss: no sec flavor on loopback connection

When using a local client, i.e. a client mounted on a server node,
there is no benefit from a security standpoint to enforce an SSK or
KRB flavor, since the data does not go over the network.
So force the 'null' security flavor for connections on 0@lo,
independently of the currently defined srpc flavor.

I checked on my Kerberos test system, by enabling the mdt2ost security flavor and mounting an MDT on an OSS node. And it worked without the need to have the lustre_mds credentials installed on the OSS. So the flavor was correctly forced to null for the loopback connection between the MDT and the OSTs. It was also confirmed when looking at the output of "lctl get_param *.*.srpc_info".
The patch as it stands today is able to force server-to-server loopback connections to null because security flavors apply to mdc, osc and osp only. So there is no need to handle other components like lwp for instance.

To sum up, this patch works for any kind of local connection, whether it is between client and server or server to server.
