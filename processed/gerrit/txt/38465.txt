LU-13510 lnet: Correct the default LND timeout

Default LND timeout is currently too low. To allow for
lnet_retry_count resend attempts within a single
lnet_transaction_timeout window, the LND timeout needs to be less
than lnet_transaction_timeout / lnet_retry_count. If the retry
count is 0, we still want LND timeout to be less than the LNet
transaction timeout.

However, if a user specifies a value for the lnet_lnd_timeout
parameter, then we should honor their request. To that end, this
change creates an lnet_default_lnd_timeout variable that is updated
when LNet is initialized, and whenever the lnet_transaction_timeout
or lnet_retry_count are set. lnet_get_lnd_timeout() is modified to
return this the default value if the lnet_lnd_timeout parameter is
not set, otherwise it returns the lnet_lnd_timeout value.

This potentially creates a little confusion regarding the value
of 'lnd_timeout' from lnetctl global show. If 'lnd_timeout' is 0 then
we're using the default value. The default value can be read via the
corresponding /sys entry.
