LU-12273 lod: Trivial metadata overstriping

With this change, setting this fail loc is sufficient to
allow running mdtest with metadata overstriping.

Example:
lctl set_param fail_loc=0x1703
lfs setdirstripe -D -i 0 -c 12 newdir

This sets a default layout which will create a twelve
stripe directory like this (4 MDTs):

newdir/newdir2/
lmv_stripe_count: 12 lmv_stripe_offset: 0 lmv_hash_type: fnv_1a_64
mdtidx           FID[seq:oid:ver]
     0           [0x200000400:0x5:0x0]
     1           [0x240000403:0x5:0x0]
     2           [0x280000403:0x5:0x0]
     3           [0x2c0000403:0x5:0x0]
     0           [0x200000400:0x6:0x0]
     1           [0x240000403:0x6:0x0]
     2           [0x280000403:0x6:0x0]
     3           [0x2c0000403:0x6:0x0]
     0           [0x200000400:0x7:0x0]
     1           [0x240000403:0x7:0x0]
     2           [0x280000403:0x7:0x0]
     3           [0x2c0000403:0x7:0x0]

This is sufficient for testing.

Note the 5 directories per MDT limit still applies!  More
than that will cause panics.

test-parameters: fortestonly
