--- Commit Message ---
LU-9859 libcfs: migrate libcfs_mem.c to lnet/lib-mem.c

Move the libcfs_mem.c code to the LNet core. The prototypes are declared in libcfs_cpu.h
but we don't move them yet since the CPT code depends on the libcfs_mem.c work. This can
end up in a modular cyclic dependency if we move the CPT work right away so limit what is 
changed at this point.

> Patch Set 3:

> I plan to when I move libcfs_cpu.c next. One step at a time :-)

"One step at a time" is good, but each step should be forwards.  Having declarations and code in unrelated places is a step backwards.
Sometimes it is optimal to have intermediate steps that are not pure improvements, but in those cases the commit message must spell out why there is a temporary regression.
But it don't think it is needed in this case.  Just create the new files in lnet, and move the desired content out of libcfs into those files.
It would make sense to delay renaming function to remove the "libcfs" from the name because that touches a different set of files.  But I don't think baby steps help.
But if you *really* want the baby-steps, please explain the plan in the commit message.
