LU-12669 lov: zero-fill pages beyond EOF in EC recovery

lov_ec_read_data_pages() only checked whether a page
offset exceeded the recovery group boundary (eoff)
before zero-filling. It did not check whether the
page was beyond the file's actual EOF.

When a file does not fill the entire last stripe
width, parity is computed with zeros for the
positions beyond EOF. During recovery, those same
positions must be zero-filled to match what the
parity encoder saw, otherwise the XOR/Reed-Solomon
reconstruction produces wrong data for the stripes
that do contain real file data.

Add an i_size_read(inode) check so pages beyond EOF
use the zero buffer instead of attempting to read
non-existent data from OSTs.

Resolves test_42a and test_42b failures with
partial file sizes: pages beyond EOF contained
stale data, causing wrong EC decode.
