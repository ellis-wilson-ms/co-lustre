LU-5896 hsm: update data version if layout is changed

lfs hsm_archive file
lfs migrate -i1 tfile
lfs hsm_release file

This actions lead to "Cannot send HSM request ...
Device or resource busy" error. What happens because of
data version mismatch. The error originally returned by
the MDS is EPERM (in mdt_hsm_release()) but is translated
into EBUSY by ll_close_inode_openhandle().

Layout swap should check to see if the 'old' version matches the
archived version in the HSM xattr and update the HSM xattr.

migrate_nonblock() doesn't expect object version to be changed, so
no need sending objects versions to a server.

When
swap_layouts o1 o2，
we must consider the following cases:
1. migrate
we know that the data content in o1 and o2 are same,
then it needs to indicate it via a flags SWAP_LAYOUTS_DATA_MIGRATE，
then when MDT know the swapping layouts is for data migration, it will do as following:

if o1 has been archived, check whether whether the archive is dirty (HS_DIRTY).
if so, do nothing to the HSM attrs for o1;
otherwise, update the data version in HSM attrs of o1 with the data version of o2.
if o2 has been archived, check whether whether the archive is dirty (HS_DIRTY).
if so, do nothing to the HSM attrs for o2;
otherwise, update the data version in HSM attrs of o2 with the data version of o1.
2. normal layout swap:

lfs swap_layouts o1 o2
As we don't know whether the content of o1 and o2 are same, thus on MDT:
if o1 has been archived, check whether whether the archive is dirty (HS_DIRTY).
if so, do nothing to the HSM attrs for o1;
otherwise, update the state with HS_DIRTY for HSM attrs of o1.
if o2 has been archived, check whether whether the archive is dirty (HS_DIRTY).
if so, do nothing to the HSM attrs for o2;
otherwise, update the state with HS_DIRTY for HSM attrs of o1.
3. data restore
MDT will swap the layouts after finished HSM data restore.
SWAP_LAYOUTS_MDS_HSM
do as it what does in Lustre currently.

All these modification must be in MDD layout in a transaction. Otherwise, it may result in an inconsistence.
