LU-7964 llite: add a deprecation console message

Lustre liblustreapi does not use the regular file's getstripe
interface, and this patch adds a deprecation console warning message
asking user to use "lfs getstripe" instead and to report its usage in
the ticket.

--- Message (Robert Read, 2016-04-01 14:23:46.000000000) ---
Patch Set 2: Code-Review-1

Why is this ioctl (and not OBD_IOC_GETFILESTRIPE) being deprecated?  We should be providing a reliable set of user space ioctl for applications to use. 

If an application is currently using this ioctl directly, then the developer is clearly not going to want to switch to using "lfs getstripe".   If we are going to deprecate this then we need to replace it with another method that is better for applications to use, not worse.
