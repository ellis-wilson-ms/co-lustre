LU-8969 utils: avoid use of O_DIRECT in lfs_migrate

Precise failure behavior on O_DIRECT reads at file EOF differs
on newer linux versions.
Stop using O_DIRECT to avoid the problem entirely.

I was going to complain about this patch because the bufffred IO would add two data copies for every page, but I see it is now abandoned. 

One thing I did notice is that the buffer size for the O_DIRECT IO is probably too small to get good performance anyway. It is stripe_size, or 1MB if that can't be found. I'd expect at least 32MB is needed for O_DIRECT to get good performance.
