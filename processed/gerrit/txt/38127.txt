LU-12275 sec: add llcrypt as file encryption library

Include fscrypt sources from Linux 5.4 into libcfs kernel module.
All fscrypt_ prefixes have been replaced with llcrypt_ to avoid
collision. For the exact transformations carried out, please see
script contrib/scripts/fscrypt_inclusion.sh, and patches under
contrib/scripts/crypto_patches/.

This llcrypt library will be built if kernel provides minimum
encryption support, ie IS_ENCRYPTED and S_ENCRYPTED.
If kernel provides support for encryption policies v2 (included in
Linux 5.4), then llcrypt library will not be built, and in-kernel
fscrypt will be used instead.

To be independent as much as possible from the kernel, llcrypt relies
on a new 'lsi_cop' field added to struct lustre_sb_info, to point to
struct llcrypt_operations, and makes use of the new 'lsi_master_keys'
field to store file system keys. llcrypt also uses i_private field on
struct inode to store file-specific enryption info.

Sorry, it is worth some explanations.

llcrypt code was included with the following method (I tried to explain in script contrib/scripts/fscrypt_inclusion.sh).

All C files from $LINUX/fs/crypto/ were put under libcfs/libcfs/crypto/. File $LINUX/include/linux/fscrypt.h was put in libcfs/include/libcfs/crypto/. File $LINUX/include/uapi/linux/fscrypt.h was placed under libcfs/include/uapi/linux/.

Then files were renamed if necessary to replace fscrypt with llcrypt. Then a number of sed scripts were run to replace fscrypt occurrences in the code with llcrypt, in order to avoid name conflicts. There is also a few other name changes for the same reasons.

After that, the patches under contrib/scripts/crypto_patches/ were applied. These patches are necessary for 2 reasons mostly:
1. llcrypt is part of the libcfs kernel module, whereas fscrypt is directly embedded into the kernel.
2. llcrypt needs to work on different kernel versions, that lack features on which fscrypt extracted from Linux 5.4 relies.

So what is put under libcfs already has all the necessary transformations. The contrib/scripts/fscrypt_inclusion.sh script and the patches under contrib/scripts/crypto_patches/ are just here to show the method used, in the hope it will be useful if we need to refresh llcrypt code from a more recent fscrypt kernel version (in case of interesting feature, bug or security fix, etc).
