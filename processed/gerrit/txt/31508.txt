--- Commit Message ---
LU-10761 osd-ldiskfs: not create REMOTE_PARENT_DIR on OST

The REMOTE_PARENT_DIR is used to link remote object which parent
resides on remote MDT to the global namespace. It is only useful
for MDT. So it is unnecessary to create such directory on OST.

--- Message (Fan Yong, 2018-03-15 08:05:40.000000000) ---
Patch Set 1:

> The addition of is_remote_parent_inode() is a good cleanup. One question is how much this change improves code efficiency beyond saving one inode in the filesystem?  On the flip side, it adds a bit of complexity into the code path as well, so it should be balanced by some optimization.

Honestly, for most of the case, the one inode optimization is nothing for the system. I open the ticket when handling DELL-439. In such ticket, the custom wanted to updated a totally full system. The new release needs to create more backend objects, then the upgrade failed because of exhausted inodes.

On the other hand, for ZFS backend, we never create REMOTE_PARENT_DIR on OST. But for ldiskfs backend, the REMOTE_PARENT_DIR exists since DNE period. I should remove it when made ZFS OI scrub patch, but I did not notice at that time.
