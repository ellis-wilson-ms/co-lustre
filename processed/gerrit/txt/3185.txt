One note about a patch like this - it is much more valuable to make a simple commit at the end of your current patch series instead of a large patch that duplicates the existing patches. That ensures the code being tested matches the code that was submitted.

If it is combining multiple branches, then it could be a merge or two that combines the branches. Any earlier merge patches in such a series can be marked in the commit message to exclude testing:

 Test-Parameters: fortestonly testlist=""

(please see Test-Parameters wiki to confirm syntax).
