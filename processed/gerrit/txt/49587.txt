>  pdsh@trevis-98vm1: trevis-98vm2: ssh exited with exit code 1
>  sanity-pcc test_101a: @@@@@@ FAIL: could not map uid 500 to root in namespace

According to newuidmap man page, it requires that the caller is the owner of the process indicated by the pid. But here, the following message shows it is not the case:

   newuidmap: Target process 15777 is owned by a different user: uid:500 pw_uid:500 st_uid:0, gid:500 pw_gid:500 st_gid:0

Looking at newuidmap's code, printed value for st_uid is effective UID, and st_gid is effective GID. So it looks like runas should be called with "-u 500 -g 500 -v 0 -j 0".
