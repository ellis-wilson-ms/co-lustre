LU-11361 tests: check for lctl

some conf parameters (like llite.max_read_ahead_whole_mb) are set
via udev & /usr/sbin/lctl. if udev rule and /usr/sbin/lctl aren't
installed, then number of tests (e.g. conf-sanity/30a) fail and
the root cause is not obvious. makes sense to verify external
components in test-framework.
