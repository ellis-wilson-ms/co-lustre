--- Message (John Hammond, 2021-03-13 16:03:44.000000000) ---
Patch Set 3:

> For mirror deleting, mirror split is used.

OK, right. So here is the current situation as I understand it: Assume that F is an FLR file with 2 mirrors. Mirror 1 is uptodate. Mirror 2 is stale.

A: 'lfs mirror split --mirror-id=1 F' does no checks and leaves F with only stale mirrors. This is bad but I don't know if anybody uses this.

B: 'lfs mirror split --delete --mirror-id=1 F' checks to see if we are removing the last non stale mirror. But the check is in lfs (not in lod) and the check is outside of critical section protected by the  lease. So this has a TOCTTOU race. And 'lfs mirror split --delete --mirror-id=1 F' is used.

So: 

1. I think we should have a race free check in lod that makes A and B both fail.

2. Removing the last non stale mirror is data corruption. Rather than handling this situation we should be preventing it. Can we use lod_last_non_stale_mirror() in lod_declare_layout_split() to prevent it?

3. If we really want to support removing the last uptodate mirror then there needs to be a 'force' flag/bit to ensure that we are not doing it accidentally. But really I think that should be managed by some component-flags-set type operation that sets certain mirror(s) uptodate and certain mirror(s) stale. Then the mirror we wanted to remove before can be removed without a special flag.
