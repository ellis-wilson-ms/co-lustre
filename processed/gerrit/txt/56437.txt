--- Commit Message ---
LU-18076 lfs: get client UUID by "lfs getname -u"

This patch is to get client UUID from the mountpoint by command
"lfs getname -u" directly. Functions llapi_file_get_type_uuid()
and llapi_file_fget_type_uuid() are added, and the man page
lfs-getname.1 is updated as well.
Also, function get_client_uuid() in test-framework.sh is modified
to verify this patch.

James, you are mixing up two unrelated things. The "lfs getname -u" command is returning the *client* UUID that is used on the server to distinguish between clients. This is randomly generated and only exists for the lifetime of the mount. AFAIK, the kernel FS_IOC_GETFSUUID ioctl (unfortunately there is a typo in the commit summary) is expected to return the "persistent" UUID of the *filesystem* which can be used to identify the filesystem persistently.
