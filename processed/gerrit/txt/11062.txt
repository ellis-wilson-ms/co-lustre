--- Message (John Hammond, 2014-07-16 13:56:00.000000000) ---
Patch Set 3:

Why revalidate for all opens?

Can you explain more about why this is important? This will really slow down repetitive small file workloads and executables on lustre.

--- Message (Oleg Drokin, 2014-07-25 18:39:39.000000000) ---
Patch Set 3:

This will not slow down repetitive small opens or anything.

You see - all opens go to MDS anyway to get the open handle (if we don't have one cached, which we normally don't).
So the fact that we needed to do it from lookup vs from revalidate does not even register on the general scale of things.

--- Message (Oleg Drokin, 2014-07-25 18:41:29.000000000) ---
Patch Set 3: Code-Review+2

I should add, that when we confirm the revalidate, then the path we take in ll_file_open forces the server to issue an open lock which is frequently undesired because we amass a lot of them and it's kind of heavy to get rid of them (hence the lu4367 - slowdown during unlinks). Additionally we cannot properly use them from revlaidate anyway, as opposed to lookup, where we can do a single step lookup-open and return a file handle right away.
