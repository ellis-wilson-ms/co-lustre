--- Commit Message ---
LU-13417 mdd: set default LMV on ROOT

To balance MDT usage, set default LMV on ROOT if it's not set. The
default stripe offset is "-1", and default stripe count is "1". Then
directory created by "mkdir" under ROOT will be scattered on all MDTs
by usage.

Add sanity 0e.

--- Message (Andreas Dilger, 2021-06-21 10:54:51.000000000) ---

Yes, something like that.  I'm not 100% sure if we can just make it take over the whole filesystem directory layout, or if there needs to be an extra "really make this the filesystem-wide default layout" flag.  I guess that would be the same as "--max-inherit=-1" to inherit this layout (virtually) across all subdirectories in the filesystem (it would not make an actual copy of the default layout in this case).

The main question is whether users/admins would be upset if suddenly *all* of their subdirectories would become remote on existing filesystems where MDT0000 is much more full than other MDTs?  Some admins may want to set "-D -c 1 -i -1" on the root to create new top-level subdirectories on new MDTs, but not affect leaf directories...   I guess if "--max-inherit=N" then that would mean "don't implicitly inherit the root layout for leaf directories that don't have it set on their parent".
