LU-10765 hsm: fix error handling for source_fd

The source_fd field in struct hsm_copyaction_private is defined as
__u32 (unsigned), but the error handling code was checking if it's
less than zero, which is impossible for an unsigned value. This
check will always be false, potentially leading to file descriptor
leaks.

Fix the error handling code to properly check if source_fd is valid
by comparing it with -1 (which is the value it's initialized to).
