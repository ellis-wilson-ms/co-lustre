This isn't really enough since the FIFO is opened O_NONBLOCK. Using fprintf() can issue small writes the first of which may succeed. It would be better to ensure a single write() to the FIFO:

 
char buf[PIPE_BUF];
 
FILE *file = fmemopen(buf, sizeof(buf), "w");
rc = llapi_json_write_list(event, file);
if (rc < 0)
        /* Bail. */;
 
fclose(file);
 
write(fd, buf, strlen(buf));

OK. I like your approach with fmemopen(); very clever. Question, though: If we are writing < PIPE_BUF, then do we even need the mutex?

http://man7.org/linux/man-pages/man7/pipe.7.html
POSIX.1-2001 says that write(2)s of less than PIPE_BUF bytes must be
       atomic: the output data is written to the pipe as a contiguous
       sequence.
