LU-16049 llite: add sync_on_close option

Add a mount option and tunable llite.*.sync_on_close to give Lustre
clients a more NFS-like experience, with data flush on file close.

To avoid significant distributed IO performance issues with this
option enabled, the fdatasync operation is only called on inodes
that actually wrote data, not on every close, and not if fsync()
was called on them already.

This could probably be optimized further, but is a useful waypoint.

Handle sync_on_close in subtests that expect delayed write behaviour:
- sanity test_45: checks both sync_on_close=0/1 are working properly
- sanity test_64f: always set sync_on_close=0, if enabled
- replay-single test_135: always set sync_on_close=0, if enabled
