--- Commit Message ---
LU-14301 client: use EOPNOTSUPP instead of ENOTSUPP

Don't return NFS-specific error code ENOTSUPP back to userspace,
instead use EOPNOTSUPP.  ENOTSUPP does not print a useful error
message from strerror() if it is hit by an application.

That "NFS specific error code" is also used by afs, btrfs, smb, squashfs, ubifs and a huge amount of code in drivers/
So I think the "is not a standard error code and should be avoided in new patches." in checkpatch.pl is trying to close the gate after the horse has bolted.

But if you are certain that no external use-space code depends on ENOTSUPP (and it would seem unlikely that any code does), then I guess we may as well make the change.  It might ease the upstreaming path.
