 ldlm_export_flock_hash(struct cfs_hash hs, const void key, unsigned mask)

Patch Set 1:

Timothy,

Based on comments for kernel hash functions.
from linux/hash.h
>>
 * This hash multiplies the input by a large odd number and takes the
 * high bits.  Since multiplication propagates changes to the most
 * significant end only, it is essential that the high bits of the
 * product be used for the hash value.
and
/* High bits are more random, so use them. */
>>
lower bits ('&' operation) should be don't applied to the hash result and ">>" (shift) used instead.
This bug live in lustre code for long time.
