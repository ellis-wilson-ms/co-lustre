Hi, LiXi

Could you share the design document? I think following concerns needs be decribed in the design doc:

- How to enable/disable user/group quota for specific pool?
- How is qsd_instace/qmt_pool allocated and prepared?
- When default pool (mdt/ost) quota and specific pool quota are both enabled, how is the quota enforced?
- Interoperability with old client. (I see protocol changes in the code)
- Upgrade problem: how to deal with old system which doesn't have pool id in EA? I'm not sure adding pool id in EA is a good idea, which looks redundant, since there is already pool name in EA.
- When there are many pools enforced quota, pool count * 2 threads will be started for quota recovery, I'm wondering if we need to change the reintegration logic bit to reduce thread count?
- Test cases.
