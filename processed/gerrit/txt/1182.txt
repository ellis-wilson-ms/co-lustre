--- Commit Message ---
LU-559 Keep the o_seq unchanged for 1.8 client

ost_validate_obdo() arbitrarily change the o_seq to 0 for the requests
from 1.8 client, which will make the 1.8 echo client use the 0 group
mistakenly. In fact, I think we needn't do anything special for the 1.8
client here.

--- Message (Andreas Dilger, 2011-08-05 01:26:57.000000000) ---
Patch Set 1:

Please look back through git/cvs history for the reason that this change was made in the first place. I believe it was because there were some 1.8.x clients that would not set o_gr (o_seq) = 0 in all code paths. Fixing this problem in 1.8.7 would be helpful, but not a complete solution, since not all customers using 2.1 will also be using 1.8.7 (when it is released).

If no 1.8.x clients set OBD_MD_FLGROUP but forget to set o_gr, then this change is safe. Otherwise this patch may not be correct.
