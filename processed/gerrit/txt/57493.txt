--- Commit Message ---
LU-18444 lnet: Remove per-peer health sensitivity

struct lnet_peer::lp_health_sensitivity was originally
added so that routers would always have their health incremented
or decremented even when lnet_health_sensitivity was set to 0. This
was to ensure that routes would go down when there were TX failures
even if the cached NI status was UP (see 00a2932b0a "LU-11297 lnet:
handle router health off"). Since health no longer determines route
aliveness we do not need the per-peer sensitivity value.

lustre_lnet_route_common() will still set a sensitivity value in the
ioctl data so that the new user-space tools continue to work with
older kernels that expect per-peer sensitivity.
