(1 comment)

Unless there's a specific RCU related issue with the debug macro (and maybe even then, since if there is I'm guessing it's rare "allocating a string under memory pressure" type thing that might be acceptable in a debug on scenario...), this way of matching up GOTO & EXIT actually isn't standard Lustre practice.

(FWIW, my instinct has always been to do it like you're doing it here, but doing it the other way does maybe make the difference between GOTO and a normal EXIT more obvious.)
