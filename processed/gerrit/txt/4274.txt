--- Commit Message ---
LU-2100 ofd: small batched precreation on a small system

on a small system (like NAME=local configuration) ldiskfs
transactions can not accept many changes. with batched
precreation in OFD, the default value of 128 objects may
lead to warnings and transaction overflow. to prevent this
OFD checks fs and if it's small (OFD_PRECREATE_SMALL_FS
= 1GB), then initial number of objects to precreate in a
single transaction is set to OFD_PRECREATE_BATCH_SMALL=8.

--- Message (Andreas Dilger, 2012-11-21 18:35:12.000000000) ---
Patch Set 3: I would prefer that you didn't submit this

Adding the (diff > OBD_MAX_PRECREATE) check complicates this patch unnecessarily, and will likely break some use cases. In particular, if an OST is restored from backup, or for some reason LAST_ID is lost and recreated to match the highest objid but is considerably lower than the MDS lov_objids value, then this OST can no longer be used by the MDS.

There is an old patch by Lai about handling these cases properly at mount time (reconstructing LAST_ID, and skipping object precreation if too many objects are requested after some recovery).

In any case, we shouldn't be adding this complexity into this patch (which should just be for reducing the precreate count for small OSTs).

Sorry I didn't catch this on the previous inspection.
