--- Commit Message ---
LU-9063 hsm: protect cdt_state with mutex lock

There is a possible race condition between mdt_hsm_cdt_start()
and mdt_device_fini() which makes cdt_state inconsistent and
leads to LBUG in hsm_cdt_procfs_fini(). So mdt_hsm_cdt_stop()
and hsm_cdt_procfs_fini() are protected under cdt_state_lock.

--- Message (Vinayak, 2017-02-06 07:25:32.000000000) ---
Patch Set 1:

Yes. Possibility of mem corruption is there. My bad. I could have tried with reproducer. 

The parameter which is leading to LBUG is "mdt->mdt_opts.mo_coordinator" flag. 

This flag is set in mdt_hsm_cdt_start() and mdt_fini() check for this parameter. If this flag is set then it will call mdt_hsm_cdt_stop().

But the possibility of race exists between mdt_hsm_cdt_start() and mdt_fini(). 

If mdt_hsm_cdt_start() is on the way and the flag is not yet set. During this time if mdt_fini() have reached and checked for "m->mdt_opts.mo_coordinator". At this point flag is not yet set and moved ahead without calling mdt_hsm_cdt_stop(). This will lead to LBUG
