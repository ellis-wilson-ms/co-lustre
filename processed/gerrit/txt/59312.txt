--- Commit Message ---
LU-14712 ldiskfs: keep EXT4_BG_TRIMMED flag in memory

Keep the EXT4_BG_TRIMMED flag in memory for the trimmed block groups
so that the filesystem without track_trim superblock bit(e.g. existing
filesystem created with earlier version of e2fsprogs) can still
skip trimmed block groups during fstrim as long as it's mounted.

For persistent trimmed block group tracking we should turn on track_trim
with tune2fs.

It would be useful to have a conf-sanity test case for this feature, to avoid regressions in the future:
- check if any MDT/OST devices are non-rotational:

        [[ $(lctl get_param -n osd-ldiskfs.*.nonrotational) =~ "1" ]] ||
                skip "no devices non-rotational for fstrim"

- find one of the targets that is nonrotational, check MDS/OST version has this patch
- run fstrim on target mountpoint, print output
- run fstrim again, check for "(0 bytes) trimmed" or error
- stop/start target
- run fstrim again, check for "(0 bytes) trimmed" or error

It would also be nice to submit a patch upstream that allowed "fstrim" to at least accept the block device as the argument, even if it converted this to a mountpoint internally.  I will file a bug and see if it gets me anywhere.
