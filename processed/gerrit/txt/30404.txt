LU-10337: Allow open of open orphans

Open by handle requires that we be able to open unlinked
files.  This currently only works if the file is already
open on the same node.

When an open file is unlinked, we put make it on orphan.
These files can be recognized by checking their open count
(mod_count).  It's enough to just make opening these files
possible, because the client cannot look them up to do an
open except when using a file handle.
