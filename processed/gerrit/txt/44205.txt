LU-14838 osc: Remove client contention support

Lockless buffered i/o and contention detection don't work,
lockless bufferd i/o is unfixable and contention detection
is broken enough that it will have to be rewritten.

Let's remove both.  This patch starts the removal by
pulling the client side support.

just a note that while contention detection is long not used and likely broken from the start (hence mostly disabled), lockless IO does work in several ways supposedly. there's direct-io with all its limitations and then there's kinda-directio-io without any of the actual directio limitatiosn that still works the same underneath (i.e. shipped out from user pages directly and does not return until server confirms it's committed.

Additional problem for lockless is going to be races with O_APPEND and truncate I suspect so those areas need to be closely examined.
