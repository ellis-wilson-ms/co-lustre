LU-3319 procfs: update shared server side core proc handling to seq_files

Several of the server side abstact layers such as mdt,mgs
etc share several common proc handling routines. This patch
adds the seq_file version so that the stack can gradually
be ported over to these new methods.

This build and the next one is failing on RHEL6 with:

 WARNING: lustre-2.5.52/lustre/ptlrpc/gss/ptlrpc_gss.o(.init.text+0x2f5):
   Section mismatch in reference from the function gss_init_lproc()
   to the function .exit.text:gss_exit_lproc()
 The function __init gss_init_lproc() references
 a function __exit gss_exit_lproc().
 This is often seen when error handling in the init function
 uses functionality in the exit path.
 The fix is often to remove the __exit annotation of
 gss_exit_lproc() so it may be used outside an exit section.
