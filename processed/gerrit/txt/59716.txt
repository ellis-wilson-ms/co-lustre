LU-0000 flr: implement immediate mirror writes for DIO

This patch implements immediate mirror write functionality for
Lustre File Level Replication (FLR) files with Direct I/O (DIO).
When a file is created with the --immediate flag, DIO writes are
automatically sent to all mirrors simultaneously instead of just
the primary mirror, providing immediate data protection and
eliminating the need for mirror resync operations.

Key features implemented:

1. Server-side FLR state management:
   - Automatic FLR state transitions: RDONLY -> WRITE_PENDING
     on first concurrent write lock, WRITE_PENDING -> RDONLY
     when last lock is released
   - ACTIVE_WRITER lock management with reference counting
     to track concurrent writers on immediate mirror files
   - Automatic mirror un-staling during WRITE_PENDING to
     RDONLY transitions

2. Client-side mirror information detection and inheritance:
   - Enhanced ll_file_io_generic() to detect immediate mirror
     files using ll_get_mirror_info() and transfer metadata
     to cl_io structure (ci_immediate_mirror, ci_mirror_count)
   - Modified lov_io_sub_init() to inherit mirror information
     from parent IO to sub-IOs, ensuring mirror metadata
     survives the LOV layer processing

3. Mirror iteration logic for DIO operations:
   - Added detection of immediate mirror DIO writes in
     cl_io_loop() based on operation type, immediate flag,
     mirror count, and DIO presence
   - Implemented mirror iteration loop that repeats the IO
     operation for each mirror with proper state reset
     between iterations (position, bytes, mirror indices)
   - Handles async IO completion codes (-EIOCBQUEUED) correctly
     during mirror iteration

4. Proper iov_iter state management in the VVP layer using
   iov_iter_revert() to reset the iterator state for subsequent
   mirror iterations, ensuring each mirror receives the complete
   write data.

5. Designated mirror support:
   - Maintains proper priority order for mirror selection:
     designated mirrors (for lfs mirror read -N<num>) take
     precedence over immediate mirror logic, which takes
     precedence over normal mirror selection
   - Ensures lfs mirror read operations work correctly with
     immediate mirror files

The implementation works by:
- Detecting immediate mirror files during write operations
- Determining the number of mirrors from the layout
- Iterating through each mirror with proper state reset
- Using existing LOV layer mirror selection logic
- Ensuring each mirror receives identical write data
- Preserving designated mirror functionality for reads

This provides significant benefits:
- Immediate data protection across all mirrors
- Improved read performance (all mirrors have current data)
- Simplified mirror management (no manual resync needed)
- Better fault tolerance for critical data

Testing shows successful simultaneous writes to multiple OSTs
with proper data integrity, correct mirror read functionality,
and no performance degradation.
