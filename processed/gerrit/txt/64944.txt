--- Commit Message ---
LU-20050 sptlrpc: fix positive errno in sptlrpc_sepol_get()

sptlrpc_sepol_get() wraps the l_getsepol return code directly with
ERR_PTR(rc). On RHEL 10.1, l_getsepol fails with rc=25 (positive
ENOTTY). IS_ERR() only detects values in [-4095, -1], so the positive
value passes undetected and the caller in
ptlrpc_connect_import_locked() dereferences a garbage pointer,
causing a NULL dereference and kernel panic.

Fix by negating positive error codes before passing to ERR_PTR():

  RETURN(ERR_PTR(rc > 0 ? -rc : rc));
