--- Message (Andreas Dilger, 2022-01-25 04:28:54.000000000) ---
Patch Set 2:

Does this hurt performance more than needed if we flush the client page cache every time the layout lock is cancelled?  I think this would have less impact if the object OST DLM locks are cancelled only when the components are makred stale.

No clients should be using those objects, and this doesn't impact normal usage of the non-stale objects, and doesn't add overhead on the clients for checking the layout state for each page.

That isn't to say I'm against it, if this is how we can fix the problem right now, but I think it could be better.  We should test the performance impact of this patch, if any, and maybe it isn't as much impact as I think.
