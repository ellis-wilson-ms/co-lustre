--- Commit Message ---
LU-18672 lmv: update stale tgt statfs if uninitialized

If LMV finds an MDT with a statfs timestamp that is over
1h old, then it will forcefully refresh this MDT's stats.
However, the 1h interval is too long for some workloads.

Always try to update the statfs cache if it is uninitiailized

Fixes: e262e0ffbe79 ("LU-17087 lmv: update stale tgt statfs every 1 hour")

Is ltd_statfs_age being initialized before the fields are actually set?  That might cause a problem if ltd_statfs_age allowed the statfs to be skipped.

I would think that the ltd_statfs_age field should be initialzed to 0 at startup, so it should always be > 1h old until it is first filled in?  Oh, I see it is using ktime_get_seconds() instead of ktime_get_real_seconds(), so the system would have to be mounted for at least 1h for this issue to be visible, but it should be clear if the stats have *ever* been updated by checking for a non-zero timestamp...
