LU-2600 osd-zfs: batched object accounting

using dsl_sync_task_do_nowait() we can schedule the updates
to be executed in a closing txg. this way object accounting
can be done in memory and then updated in ZAPs once per txg.

unfortunately the patch doesn't work with debugging enebled
in ZFS. that's because the internals (e.g. dbuf_dirty()) do
expect only the special objects can be modified at syncing.
and the special objects are those which dnode# < 0, which is
not so yet for the objects storing accounting.
