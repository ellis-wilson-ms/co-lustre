I agree with James, kallsyms_lookup does not seems to be a proper solution.

SElinux being defined as an LSM, it could be possible to access the 'enabled' field of the struct lsm_info. For instance by looping on all LSMs as done in function ordered_lsm_parse:


   struct lsm_info *lsm;
   [...]
   for (lsm = __start_lsm_info; lsm < __end_lsm_info; lsm++) {
   [...]


If it does not work, maybe accessing this info from userspace at Lustre load might be necessary. That would do the trick, as enabling or disabling SELinux is not possible without rebooting IIUC.
