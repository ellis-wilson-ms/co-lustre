LU-19768 obd: add obd_params/brw_stats netlink

Add netlink handlers for scalar OBD parameters (statfs data,
export counts, dirty/granted/pending bytes) and BRW histogram
statistics. The brw_stats handler reads histograms from
dt_device.dd_brw_stats, which is populated by osd-ldiskfs and
osd-zfs during device init.

## Fix Summary

**Problem:** 
The compile error occurred because the function `lnet_counters_get_common()` was being called in `lustre/obdclass/kernelcomm.c` at line 1216 (in the `lustre_health_dump` function), but the function declaration was not visible to the compiler.

**Root Cause:**
The function `lnet_counters_get_common()` is declared in the header file `lnet/include/lnet/lib-lnet.h` (at line 864), but this header was not included in `lustre/obdclass/kernelcomm.c`.

**Solution:**
Added the missing include directive `#include <lnet/lib-lnet.h>` to the file `lustre/obdclass/kernelcomm.c` at line 26, before the other Lustre-specific includes.

**Rationale:**
The patch added new netlink handlers for health monitoring that call LNet counter functions. The `lustre_health_dump()` function needs to retrieve LNet statistics using `lnet_counters_get_common()`, which requires the LNet library header to be included. This is a standard cross-module dependency in the Lustre codebase where OBD class code needs to access LNet functionality for reporting purposes.
