LU-20054 mgs: more checks for incoming buffers

Check for TARGET_NIDLIST buffer size to make sure
that NIDs were packed in it but not in mti buffer.
This can be in case of initial registration when MGC
is not yet connected, so export connection flags
are empty and buffers were packed in compatible manner

Test conf-sanity 73f is added to simulate that situation

Patch also handles IPv6 NID on target by assuming that MGS
has large NID support if there are IPv6 NIDs on target.
So if MGC is not yet connected but there are IPv6 NIDs
then target will pack them in registration request, and
they will not be packed only if MGS is connected and
reports no support for large NIDs explicitly
