--- Commit Message ---
LU-1644 recovery: to not check client version for mne swap

data->ocd_version in obd_connect_data can be overwritten by server
so that it'll be wrong if client is reconnected after server recovery.
Wrong clienv version caused wrong exp_need_mne_swab was set and then
has interop issue between 2.2 clients and 2.3 server.

In this patch, the following fixes are made:
1. do not check the client version on the MGS;
2. reset client version on each import connect;
3. client side fix will be removed after lustre 3.2

--- Message (Andreas Dilger, 2012-09-07 02:08:04.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

Removing the client version check breaks the ability to remove this workaround in the future.  Without this, the server will do swabbing for all clients that do not connect with OBD_CONNECT_MNE_SWAB, and if this flag stops being sent by the client in the future (e.g. 2.7.0 client vs. 2.6.0 server) then the bug will be reintroduced.

That is why there is a check for _both_ MNE_SWAB missing, and client version < 2.2.55 before doing the extra swabbing, so that when the workaround is removed in 2.7.0 this code will NOT do the extra swabbing.  It only adds the workaround for unpatched 2.2 clients.

--- Message (Andreas Dilger, 2012-09-07 06:26:33.000000000) ---
Patch Set 1:

No, using paddingX means that we would be corrupting some other part of the obd_connect_data later on.

The original code saves mne_swab_client_ver early on before obd_connect_data is overwritten with the server version, so the problem is really on the client.  The client should reset the ocd_version in client_connect_import() before reconnecting.

However, we can't change existing clients to do this retroactively, it is enough to set exp_mne_need_swab only the first time the client connects.  The client cannot possibly change versions (i.e. upgrade) but continue to use the same export on the server.

It should be enough to add another exp_have_mne_swab:1 flag that is set the first time exp_need_mne_swab is set, and then it prevents the flag from being changed on that export again in target_handle_connect():

        if (!export->exp_have_mne_swab) {
                export->exp_need_mne_swab =
                        !(data->ocd_connect_flags & OBD_CONNECT_MNE_SWAB) &&
                        mne_swab_client_ver && !ptlrpc_req_need_swab(req);
                export->exp_have_mne_swab = 1;
        }

--- Message (Andreas Dilger, 2012-09-08 17:23:17.000000000) ---
Patch Set 1:

I think it is still possible to handle this mostly on the server side, but we also should fix the clients to reset their ocd_version field at reconnect time to avoid this problem in the future.

What needs to happen is that the client version checks should be removed as in this patch.

In addition, the clients will need to continue to send the OBD_CONNECT_MNE_SWAB flag for a longer time than the servers are checking for it. After about 2.6.50, the servers can assume that clients no longer need swabbing, while the clients should continue to send MNE_SWAB until about 3.2.50 so that they can interoperate with older servers.  After that version the clients can stop sending MNE_SWAB as well.
