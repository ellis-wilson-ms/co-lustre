LU-12893 lnet: fix peer_ni selection

When selecting a peer-ni we must use the same peer NID
through all the messages which belong to the same RPC.
This is necessary in order to ensure we do the RDMA over
the optimal interface.

--- Message (Amir Shehata, 2019-10-22 19:32:50.000000000) ---
Patch Set 1:

Regarding Routing, didn't Alex make a change to use the same router? If the router has multiple interfaces currently we don't restrict it to a specific interface. There is no requirement to use the same gateway interface. The reason for this change is due to the NUMA distance criteria. We determine the interface early on when we setup the buffers. After that we need to stick with that interface to eventually do the RDMA over the correct interface. 
I don't think there is such a case for routers.
However, with the UDSP feature there will be a way to prioritize the router interfaces.
The question is, do you see a use case where we need to use the same router interface?
There is a work around: turn off Discovery for the gateway and configure each interface of the gateway as a different route.

--- Message (Amir Shehata, 2019-10-22 19:34:37.000000000) ---
Patch Set 1:

Just to add to the point above: after you configure each gateway interface on a separate route, then you can use the priority parameter of the route to make it more preferred.
