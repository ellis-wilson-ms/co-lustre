LU-19436 lnet: Correct ip2nets yaml parsing

The ip2nets parsing in handle_ip2nets_sequence() had several issues
that prevented it from adhering to documented behavior:

- Rules that didn't match local interfaces were incorrectly treated
  as errors instead of being silently skipped as documented
- Duplicate networks could be configured multiple times in a sequence
- Error messages were generic and didn't help diagnose configuration
  issues

This patch fixes the parsing logic to:
- Properly handle LUSTRE_CFG_RC_NO_MATCH as a valid state for
  ip2nets rules that don't apply to the current node
  - To accomodate this change, lustre_lnet_kfi_intf2nid() was changed
    to return RC_BAD_PARAM rather than RC_NO_MATCH on error.
- Track configured networks to prevent duplicates within sequences
- Provide specific error messages for different failure scenarios
  (interface not found, interface down, IP pattern mismatch, etc.)

The changes ensure that ip2nets behaves as documented: each node
identifies locally available networks based on IP address pattern
matching, with non-matching rules being silently ignored rather
than causing configuration failures.
