--- Commit Message ---
LU-13005 lnet: drop refcounting from event queues.

The refcount is no longer incremented or decremented
as there is no interesting state and no long term
references.  Nothing checks if LNetEQFree returns -EBUSY
any more - though a few calls LASSERT() that the return
value is 0.

So discard the refcounting, change LNetEQFree to return
'void', and remove all the LASSERTs.

I see this as protection against MD leaks. For example, if you have an EQ which is being used, and there is a bug in the code where it doesn't free the MDs before it frees the EQ, then by asserting at that point that there are still MDs on the queue, it'll point out where the problem is. yes, I was adding comments as I go along. So I see that the last patch in the series removes the eq all together.
The good thing about the eq is that it provided a hierarchical organization of the MDs. You'd create different EQs and you'd link the MDs against them.
