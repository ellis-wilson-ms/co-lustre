(1 comment)

I checked history of changes and it seems that this is not such regression as I thought. Meanwhile I found one case that looks not safe. Waiting for is_bl_done() is polling that bl_done flag, which is set in ldlm_cancel_callback() only if !ldlm_is_cancel(). At the same time in ldlm_handle_bl_callback() we have:

if (ldlm_is_cancel_on_block(lock))
	ldlm_set_cancel(lock)

Therefore later ldlm_cancel_callback() will not set ldlm_bl_done because ldlm_is_cancel() is set and will wait for is_bl_done until timeout.
Isn't it? Or that case with cancel_on_block is handled somehow?
