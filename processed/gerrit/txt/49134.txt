> Ubuntu 20 running 5.8, 5.11 and 5.13 kernels.

Strange.  In each of these kernels EXT4_IGET_SPECIAL is defined, but EXT4_GET_BLOCKS_KEEP_SIZE isn't.
So HAVE_LDISKFS_GET_BLOCKS_KEEP_SIZE will be defined.
Code in osd-ldiskfs/osd_io.c tests #ifndef HAVE_LDISKFS_GET_BLOCK_KEEP_SIZE which fails, so it doesn't even try to compile code that uses LDISKFS_GET_BLOCK_KEEP_SIZE (which is obviously backwards).
So you don't get an error.... so why do I get an error?
Ah-ha.  I was building 2.15.1 which doesn't have support for newer kernels so I added some patches including Shaun's "LU-15543 ldiskfs: Linux kernel 5.14.21 server support".  This flips the #ifndef to #ifdef.  The #ifndef is "obviously" wrong, but now that wrongness doesn't cancel out the wrongness of the testing of EXT4_IGET_SPECIAL when defining HAVE_LDISKFS_GET_BLOCK_KEEP_SIZE.
So your kernels build correctly for the wrong reasons - two mistakes effectively cancel each other out.
