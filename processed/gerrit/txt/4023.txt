Patch Set 4: I would prefer that you didn't submit this

(10 inline comments)

ldlm changes are wrong as indicated and need to be taken out.

Additionally, I see little point in adding assertions for NULL pointers where there were not any, (which was especially bad in pack_generic.c that I think needs to be greatly scaled down, no need to check every argument of every function there, these LASSERTs are not free in both CPU and stack usage terms).
After all if the assertion is triggered, kernel dies, if there's no assertion and we trip on a NULL pointer, kernel dies too - same result, much less effort.

So while I am fine with moving things around to have existing assertions actually cover stuff (or perhaps it's a better idea to just kill those clearly redundant assertions instead?), no new assertions where there were not any, please.
