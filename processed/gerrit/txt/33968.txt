LU-11838 socklnd: use ksocknal_enumerate_interfaces for individual interfaces.

When individual interfaces are listed, such a via a module parameter,
it is easy to use ksocknal_enumerate_interfaces() to find them, and
save duplicating code.

The iname passed into ksocknal_enumerate_interfaces could be a
primary interface (eth0) or a secondary interface (eth0:1). This
means we can't directly compare the net device name to iname. We
can take advantage that the primary interface is apart of the
secondary interface name. So test if dev->name is a substring of
iname. We also have to make eth0:1 does not equal eth0 with
strmcp(ifa_label, name).

We can take advantage that we find all the net devices in
ksocknal_enumerate() and from those discoveries set the ni_cpt.
We can additonally warn if ethernet interfaces are being grouped
together that are not in the CPT.

The way the count of interfaces is done is very error prone.
Instead ksocknal_enumerate_interfaces() can handling incrementing
ksnn_ninterfaces directly. The function then just returns 0 on
success or errno on failures like normal functions.

Harden ksocknal_enumerate_interfaces() to avoid duplicate entries.
This enables potential use for dynamic addition or removal of
NI interfaces in the future.
