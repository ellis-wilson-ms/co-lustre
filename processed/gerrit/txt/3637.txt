LU-858 build: check for generated autoconf.h header first

Fedora apparently provides a zero length autoconf.h in the standard
location (for programs that might be testing for its presence?).
However this causes the build scripts pre-include instead since it
finds that header file first.  Instead, check generated directory
first for autoconf.h
