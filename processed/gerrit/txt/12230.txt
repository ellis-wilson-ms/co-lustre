--- Commit Message ---
LU-5077 ptlrpc: simplify checsum calculation for reply

It's not necessary to call external crypto API to calculate chksum
for the small ptlrpc body on reply, instead, call crc32_le() directly
could avoid memory allocation.

This patch could fix the problem of OOM in crytpo_create_tfm(), or
make the OOM happen in other code path, then we can collect more
information for further analysis.
