--- Commit Message ---
LU-11053 llite: check loop device file size before read

ll_file_open() does not fill lustre's inode->i_size. That makes a
problem for losetup which avoid stat(). loop block device file gets
zero file size from lustre's inode, that ends with failure on
umounting the loop device at BUG_ON(!buffer_mapped(bh)) in
submit_bh().

The proposed fix is to check loop device file size during reading loop
device and to refuse to read if it mismatches lustre file size.

sanity.sh:test_54c() is modified to illistrate how mounting loop
device file fails without having lustre backing file stat-ed before
losetup and how it succeeds when stat precedes losetup.
