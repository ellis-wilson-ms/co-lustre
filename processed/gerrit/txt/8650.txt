Patch Set 2: Code-Review-1

I thought the whole point of the insanity tests was that there were two services failed at the same time and the servers need to be able to recover, even if the clients get errors?

It makes me wonder if this patch is making the test pass, but it is leaving a problem behind that will hit users in real usage?  Why is the OST local log not updated?  Is it only because the second MDT is added after the OST has first mounted?

I would have suggested to submit this fix separately from the patch to disable the test, so that we could land that one even if there is a problem with this patch.

If this test is being skipped by autotest, then it will need an explicit Test-Parameters line to run the test in DNE mode to verify it is now working.
