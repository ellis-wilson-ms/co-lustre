I spent some time finding the divergence of Lustre checkpatch.pl from upstream checkpatch.pl. 

I've submitted two patches: one which up-dates Lustre checkpatch.pl to 6.8-rc5 checkpatch.pl and another (this one) which applies the Lustre specific changes. This should make it easy to visualize how far we differ from upstream. It'll also help us understand what changes (if any) checkpatch.pl will need when Lustre is submitted.

Based on my testing, the number of warnings we get it noticeably lower. So this should save us some effort and false positives.

Testing/feedback/concerns would be appreciated.

Tim, thanks for working on this.  It is definitely an improvement over the first version of the patch, and allows at least side-by-side comparison of the different versions of the patches to the checkpatch script.

I was originally thinking that this patch would introduce only a single "contrib/scripts/checkpatch-lustre.patch" file that was generated from the current script changes (what you currently have in "patch-v4.12-11743-g96d0d83"), and that file would be updated to reflect the changes applied to the new checkpatch.pl script when it is updated (i.e. change "checkpatch-lustre.patch" to what is now "patch-v6.8-rc5-gb401b62175" as part of the next patch).

That would simplify the comparison of changes to the Lustre-specific patch from one version to the next, independent of the changes to the checkpatch.pl script itself.
