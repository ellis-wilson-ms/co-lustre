Patch Set 1: Code-Review-1

(4 comments)

To be honest, I think switch the position of rename_sanity and lu_object_find is just temporary solution, I think the real problem is that subdir check is not protected by ldlm lock.

It seems to me that the deadlock described in LU-4725 can be avoided by only checking the direct relationship between src parent and dst parent, i.e. it does not need to use is_subdir, which will go all the way up to root or until remote parent is found.  Instead you can get linkea of src, and just compare the parent FID its dst parent? So

1. global rename lock
2. Lock the src parent, check whether its parent fid = dst_parent, and sort out the lock order. unlock the src parent.
3. ldlm lock of all rename objs.
4. do rename.
5. unlock objs.

Please correct me if I miss sth.
