--- Commit Message ---
LU-3126 osd: remove fld lookup during configuration

Remove fld lookup during configuration, so to avoid accessing
FLDB on MDT0 before MDT0 is setup.

1. add od_is_ost to check whether the FID is on OST, instead
of lookup in FLDB.

2. add oic_device in OI cache, so in OI cache lookup, it will
try to match device to avoid the ino of agent inode is being
retrieved from OI cache. Then osd_remote_fid, which will trigger
fld lookup, can be removed from osd index lookup.

--- Message (Andreas Dilger, 2013-08-08 18:33:30.000000000) ---
Patch Set 3:

Two notes on this patch:
- in LFSCK phase 2 it is starting to flag the OST objects separately on disk in the LMA so that it can tell if this is a named (MDT object in OI) or unnamed (OST in O/{seq}) object during recovery. It flags this in the object in memory and it makes sense to use the same flag here.
- the use of the FLDB in the OSD is only to distinguish between the OST and MDT objects and IMHO should be removed entirely. Each OST knows which sequences belong to it already because it can check O/* for the list of sequences, and the OSD shouldn't need a network connection to work properly. This patch is a step in the right direction, but if like to see this finished off.
