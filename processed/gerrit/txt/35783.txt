--- Message (Andreas Dilger, 2019-08-28 20:01:48.000000000) ---
Patch Set 4:

> > I don't think it is necessary to pass in a variable to get the size:
> 
>       if (m->lm_bufcount == 0 || m->lm_bufcount > 
>                sizeof(((struct ptlrpc_request *)0)->rq_req_swab_mask) * 8)

Actually, it might be better to have a #define for this in lustre_net.h and use it in the few places where the size is checked:

 #define PTLRPC_MAX_BUFCOUNT \
        (sizeof(((struct ptlrpc_request *)0)->rq_req_swab_mask) * 8)

I think it is OK to assume that the number of buffers will be the same for request and reply, or at least we can ensure this is true for these swab_masks.

We could also add another definition for the maximum buffer size like:

 #define MD_MAX_BUFLEN     ((MDS_REG_MAXREQSIZE > OUT_MAXREQSIZE ? \
                             MDS_REG_MAXREQSIZE : OUT_MAXREQSIZE)
 #define PTLRPC_MAX_BUFLEN (OST_IO_MAXREQSIZE > MD_MAX_BUFLEN ? \
                            OST_IO_MAXREQSIZE : MD_MAX_BUFLEN)

which should be resolved at compile time instead of runtime, since it is all constant values.  Then PTLRPC_MAX_BUFLEN can be used instead of INT_MAX in the checks.
