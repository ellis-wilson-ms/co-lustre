LU-16053 zfs: fix dmu_offset_next() check

Use LB_CHECK_COMPILE() to detect if the dmu_offset_next() function
has been exported by ZFS.  This is consistent with the rest of the
checks and more reliable than checking the source which is normally
unavailable at build time.

Fixing the check now exercises SEEK_HOLE and SEEK_DATA operations.
Provide a sparse file tool that creates a sparse file which is
friendly to lustre zfs-osd.

Testing suggests that dmu_offset_next() never returns EBUSY.
Update SEEK_HOLE and SEEK_DATA operations to explicitly
sync before attempting dmu_offset_next().
