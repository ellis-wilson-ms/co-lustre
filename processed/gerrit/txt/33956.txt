--- Commit Message ---
LU-11739 lod: don't inherit default layout from root directory

There is no need to inherit the default directory layout from
the root directory when subdirectories are created therein.
This consumes xattr space on the subdirectories, and makes it
more complex to change the filesystem default layout in the future.

This patch fixes the above issue in lod_ah_init() to check if
the parent directory is the root directory and not copy
the default layout xattr to the new subdirectory.

Lustre debug log on MDS showed that:

 (lod_object.c:5164:lod_declare_create()) Process leaving via out (rc=18446744073709551594 : -22 : 0xffffffffffffffea)

The "-22" error code came from the following codes in lod_declare_create():

                                cfs_foreach_bit(ltd->ltd_tgt_bitmap, i) {
                                        tgt = LTD_TGT(ltd, i);
                                        if (tgt->ltd_index ==
                                                lo->ldo_dir_stripe_offset) {
                                                found_mdt = true;
                                                break;
                                        }
                                }


                                /* If the MDT indicated by stripe_offset can be
                                 * found, then tell client to resend the create
                                 * request to the correct MDT, otherwise return

                                if (found_mdt)

                                        GOTO(out, rc = -EINVAL);

It means the MDT indicated by lo->ldo_dir_stripe_offset can not be found. This review-ldiskfs test session has only one MDT, but in its debug log:
