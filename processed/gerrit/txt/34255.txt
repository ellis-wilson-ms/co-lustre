LU-11964 ldlm: Don't match kms_ignore DOM locks

DOM uses the kms_ignore flag to indicate the DOM bit is
being cancelled, so if a lock with this bit set matches a
DOM lock request, LDLM rejects it and creates a new lock.

The problem is that a lock with kms_ignore set can remain
the first lock in the queue, so all new requests match it
and then reject it.  This creates a lock for every request.

The solution is to not match this lock in lock_matches,
then we will continue and find other locks which allow our
request, rather than creating a new lock.
