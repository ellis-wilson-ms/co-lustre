LU-4878 osd-ldiskfs: don't assert on possible upgrade

There are some cases possible during upgrade to the Lustre 2.5
from older versions which need workaround instead assertion.
More generaly, this concerns any upgrade to a version that
include Gerrit change #5049 from LU-2059:
Lustre-change: http://review.whamcloud.com/5049
Lustre-commit: I137e38abb0e93c12406c6610c0922f4294e67290

The osd_object_ref_del() shouldn't assert on nlink == 0 if
object is local file but produce error message.

The underlying ext4_unlink() code doesn't treat decref on an
inode with nlink == 0 as a fatal error either, so only print
an error in the non-upgrade case for regular objects.

This patch is the b2_4 back-port of original/b2_5 version
from Mike, in LU-3915 with Gerrit-Id #7673:
Lustre-change: http://review.whamcloud.com/7673
Lustre-commit: I5aa1cc3d9a6d6210e8ce9f55193a4f25507b0986.
