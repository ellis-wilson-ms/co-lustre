LU-19626 tests: multiop running out of single-letter commands

The multiop test utility uses single-letter commands (a-z,
A-Z) to perform file operations. We've used up most of the
alphabet - only 4 letters remain unused (b, h, k, q). We
need a way to expand beyond the current 52-command limit
without breaking existing tests.

Implement two-character prefix commands using 'X' as the
prefix letter. The prefix letter 'X' is followed by a second
character to specify the command (e.g., 'Xe', 'Xf'). This
gives us ~50 new commands, and can expand if needed by using
X as a sub-prefix, eg, 'XXe'.

Implementation:
- Use 'X' as the prefix letter (semantically means "extended")
- Add 'Xe[num]' for lseek(SEEK_END) with optional offset
  to demonstrate
- Move existing 'X' command (get OST layout version) to 'Xx'
  subcommand
- Reserve 'XX' for future nested prefixes
- Alphabetize help text with lowercase before uppercase for
  consistency

Also fix a bug where commands with numeric arguments would
lose their return value due to the main loop resetting rc=0
at each iteration.

Add test_860 to sanity.sh to verify the new functionality.
