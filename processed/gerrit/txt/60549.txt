--- Commit Message ---
LU-19229 mgc: client to handle IR with unknown nets

When client processes config log or IR information it
assumes that server LNet configuration is the same, e.g.
list of networks. If LNet configuration is not the same,
e.g. server has many nets but client just single one, it
fails to mount over that network in most cases.

Patch makes client more flexible and able to mount in
such environments.

- don't fail when initial connection is not established
  based on config file, enable dynamic_nids for target
  and let IR to fill the gap
- when using IR NID list, don't stick to first NID while
  setting/checking initial connection but try all until
  one on valid network is found
- LNetAddPeer() to don't add peers on missing networks
  when peer discovery is off
- add conf-sanity test 73e for dynamic NIDs feature
