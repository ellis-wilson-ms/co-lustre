--- Commit Message ---
LU-1199 build: Rename ldiskfs rpm ldiskfs

The ldiskfs rpm package has been named lustre-ldiskfs for some
time, which seems to imply that it is created as part of the
lustre.spec file.  In reality, it was designed to be its own
stand-alone package with its own build system and separate spec
file.  This naming is less surprising.

Patch Set 1:

Chris, while it may be true that there is a small semantic issue that the lustre-ldiskfs package is not actually built from the lustre.spec file, but I think this change will confuse a larger number of end users that expect to be able to do "rpm -Fvh lustre-*" and get all of their lustre packages upgraded.  True, the spec files are not related, but ldiskfs is definitely part of Lustre and to me it makes sense that the name reflect this. 

I've added James and John as inspectors to get some more opinions on what is more important to end users.

--- Message (John Hammond, 2013-04-04 17:06:27.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

I agree that lustre and ldiskfs build should be as loosely coupled as possible but I do not see how this achieves that or that the benefits outweigh the cost of change.

The simplicity of prefixed names is worth preserving: "rpm -Fvh lustre-*" should work.

Moreover, as you know, ldiskfs is not intended as a "real" stand alone filesystem for Linux, or at least not one that we support as such.  Therefore it belongs in the Lustre suite of rpms, albeit as an optional member.
