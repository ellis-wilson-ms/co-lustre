LU-2641 mgs: fix cleanup of MGSSELF_NAME proc file

Commit 12bac0d9 (v2.6.33) added a WARN_ON for trying to remove
nonexistent proc entries.

mgs_new_fsdb only calls lproc_mgs_add_live if the fsname isn't "_mgs",
however mgs_free_fsdb calls lproc_mgs_del_live unconditionally.

This triggers the warning.

We check the FSDB_MGS_SELF bit which is only set when
fsname == "_mgs", so we know to skip the proc entry removal and avoid
the warning.
