--- Commit Message ---
LU-19388 osd: remove calls to alloc_file_pseudo()

fput() which we need to call after alloc_file_pseudo() doesn't release
file struct right away and postponed jobs to release it may create
a massive load to system interrupting regular Lustre processing.

The reason for using alloc_file_pseudo() was due to the removal
of kallsyms_lookup_name(). The function iterate_dir() will crash
unless security_file_alloc() and security_file_free() are not
called. Now that Lustre uses kprobes to export functions we
can unwind the work of LU-13783.

## Summary

The compilation error was caused by a missing header file `lustre_compat/linux/security.h` that was being included in the patch but didn't exist in the codebase. 

**The fix I implemented:**

1. **Created the missing header file** `include/lustre_compat/linux/security.h` with:
   - Proper include guards
   - Includes for `<linux/fs.h>` and `<linux/security.h>`
   - Function declarations for `compat_security_file_alloc` and `compat_security_file_free` when `CONFIG_SECURITY` is defined
   - Inline stub functions that return 0/do nothing when `CONFIG_SECURITY` is not defined

2. **Cleaned up redundant code** in `lustre_compat/symbols.c` by removing the static inline functions that were duplicated in the `.c` file (which wouldn't be accessible to other compilation units anyway)

**Rationale for the fix:**

The patch introduced new compatibility functions `compat_security_file_alloc` and `compat_security_file_free` to replace the use of `alloc_file_pseudo()` and `fput()`. These functions provide a compatibility layer that:

- When `CONFIG_SECURITY` is enabled: Uses kprobes to call the kernel's `security_file_alloc` and `security_file_free` functions
- When `CONFIG_SECURITY` is disabled: Provides no-op inline functions

The header file is essential because:
1. It provides the function declarations needed by the files that use these functions
2. It handles the conditional compilation properly based on `CONFIG_SECURITY`
3. It follows the existing pattern in the Lustre codebase for compatibility headers

This fix ensures that the security context is properly managed for file operations while avoiding the performance issues that were caused by the delayed `fput()` mechanism in the original `alloc_file_pseudo()` approach.
