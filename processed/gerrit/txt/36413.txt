LU-12832 ldlm: avoid to search all PR extents

PR locks could be overlapped, but trying to enqueue
an PR extent which is a sub-extent in the tree will
be returned with existed extent.

For following cases:
 [5, 10]->enqueue [1, 2] -> [1, 2] [ 5, 10]
 [5, 10]->enqueue [1, 6] -> [1 10]
 [5, 10]->enqueue [1, 11]-> [1,11]* we expand high end always
 [5, 10]->enqueue [5, 9] -> [5,10]

Todo: verify above assumptions
