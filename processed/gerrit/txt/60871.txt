LU-19293 lnet: list_group wth wrong group name crashes kernel

If 'list_group' is called with group name which
was not yet created by 'add_group' this would lead
to kenrel crash.

lstcon_group_find() finds group by name. When the
group is not found lggp_grp is left dangling/NULL.
This is not checked under lst_groups_show_dump()
which was causing a bad dereference.

This patch fixes the crash by adding an extra
'lggp_grp' check under lst_groups_show_dump()
to skip if group name is not found.

lst list_group call with 'bad group' is added
under lnet-selftest.sh/smoke for verification
