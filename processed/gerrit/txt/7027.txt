LU-3565 mdt: a new param to allocate sequences

sequence controller (aka master sequence server) exports
FLDB in proc/fs/lustre/seq/ctl-*/fldb. the same entry
can be used to modify FLDB using available sequences:
lctl set_param
  seq.ctl*.fldb="[0x0000000280000400-0x0000000280000600):0:mdt"

--- Message (Andreas Dilger, 2013-07-22 08:06:25.000000000) ---
Patch Set 1:

In my inspection comments, I'd prefer to make the input and output formats symmetrical, so it would be possible (for example) to dump the output of the fldb and then use it to restore the fldb later:

 # lctl get_param fld.srv-testfs-MDT0000.fldb
 [0x0000000200000007-0x0000000200000008):0:mdt
 [0x000000000000000c-0x0000000100000000):0:mdt
 [0x0000000200000002-0x0000000200000003):0:mdt
 [0x0000000200000400-0x0000000240000400):0:mdt

the above exists today, but it appears that this is not the master sequence server, only the MDT0000 sequence server.  To set a sequence, the same "[start-end):idx:type" format could be used.  While "start" is redundant, it would also avoid skipping large gaps if the userspace was not in-sync with the actual FLDB content:

 # lctl set_param fld.srv-$fsname-master.fldb=[0x280000400-0x2c0000400):1:mdt

--- Message (Alex Zhuravlev, 2013-07-22 18:32:12.000000000) ---
Patch Set 1:

there is an issue with adding records via *.fldb. in current implementation the entry point is the sequence server which grabs some sequences from own space and inserts them into FLDB. I think we can find proper sequence server via fldb->object->site->..., but fldb and sequence server are implemented in different modules. and such an implementation would introduce cyclic dependency making impossible to load fldb module.

I can see benefits in having the same format for listing and altering FLDB, but not sure how to implement this properly...
