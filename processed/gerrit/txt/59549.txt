LU-16138 osd-ldiskfs: remove block integrity kernel patches

Rewrite osd_integrity.c to remove the need for block integrity kernel
patches.

Now we do the equivalent of bio_integrity_prep() ourselves in
osd_bio_integrity_handle(), without setting BIP_BLOCK_INTEGRITY flag
on integrity payload to preserve the metadata buffer.

For a WRITE we fill the metadata buffer with guard tags from
ptlrpc if there's any, then submit the bio.
For a READ we need to take over the data integrity
verification when the I/O is completed, during verification we can reuse
the guard tags for ptlrpc, this is done on a per-OSD workqueue.

With the above changes we can finally remove the custom block integrity
kernel patches, and still be able to reuse the guard tags for ptlrpc.

Tested on rhel7.9, rhel8.10 and rocky9.X
