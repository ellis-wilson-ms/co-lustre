This could start testing that the EC generation code is working properly, by writing a synthetic data pattern to the file, and then using "lfs mirror read" to dump the parity component and verifying in userspace that the parity was computed correctly.

Often for EC libraries there are known patterns that can be used to verify that the parity generation is correct.  Alternately, there could be a userspace test program that computes the parity of the file data and compare it against what is stored in the parity component.
