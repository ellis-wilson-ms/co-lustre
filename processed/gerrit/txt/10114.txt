(1 comment)

Note that OBD_IOC_CREATE is only used by Lustre testing tools (lctl and loadgen and maybe mds_survey) that build and ship as part of Lustre itself, so there shouldn't be any problem to change the specification of this ioctl. Even better, it should be given a proper IOC number that is using the "size" parameter correctly to determine the argument size, which would be a benefit because it would distinguish the new IOC number from the old one so that new tools and old kernels (and vice versa) do not get confused.

That said, I don't know that anyone has used the ability to create striped files via lctl/OBD_IOC_CREATE in a long time, so maybe there is no loss to remove this complexity entirely?  If that is needed, then it could easily pass lov_mds_md_v[13] instead?
