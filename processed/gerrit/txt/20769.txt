LU-8223 lnet: Fix use of NETERR logging

NETERR logs are not sent to console (syslog) by default because
customers complained they are "too noisy".  The main reason is
when nodes are taken down for maintenance, the other Lustre
nodes keep trying to reconnect creating a lot of NETERR logs.

This patch does the following:

- turn sending NETERR logs to console ON by default but
  still allow customers to turn them off.
- change all uses of CERROR in LNet/LND code to use NETERR
  except in cases where we fail to allocate memory or when
  the call to CERROR is followed by an LBUG.
- add code to o2iblnd and socklnd to suppress NETERRs
  when trying to connect to a node on a network we already
  have failed connections to. This should stop the flood of
  connection failures when a node is taken down for maintenance.
- clear the above suppression flag once we succeed to connect
  to a node on the given network.

Test-Parameters: trivial
