Patch Set 3: Code-Review+2

Strictly speaking, the DEGRADED OST should only be skipped if there are enough other non-DEGRADED OSTs available to meet the stripe requirements.  However, since lod_alloc_qos() will fall back to lod_alloc_rr() and this correctly includes DEGRADED OSTs when there aren't enough other OSTs, this is OK.

Ideally, someone would work on LU-9 and make a single round-robin allocator that takes space balance into account, instead of jumping to random QOS allocation...
