LU-19148 ldlm: update best_ptlrpc conn in import_set_conn

Currently if we have several consecutive calls to import_set_conn
to the same node, only the first one will "take effect". i.e. if
say the second call finds a better ptlrpc_conn for a connection to
the same "node", it won't update to use the best ptlrpc_conn.
