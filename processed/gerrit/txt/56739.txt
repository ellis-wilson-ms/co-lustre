LU-18386 utils: add lfs quota --delimiter option

Add a new --delimiter option to the lfs quota command, allowing
users to specify a custom delimiter for the output. This enhances
the flexibility of the quota display, making it easier to parse
the output for scripts and other automated tools.
The default delimiter is a space.

$ lfs quota --delimiter="," /mnt/lustre
Disk quotas for usr root (uid 0):
      Filesystem, kbytes, bquota, blimit, bgrace, ..., igrace
     /mnt/lustre,   4360,      0,      0,      -, ...,      -
Disk quotas for grp root (gid 0):
      Filesystem, kbytes, bquota, blimit, bgrace, ..., igrace
     /mnt/lustre,   4360,      0,      0,      -, ...,      -

Sorry for my last comments, this was a bit messy.

For me the purpose of having a "--delimiter" option is to be easily parsable (e.g: CSV, or cut/awk command).

So if a delimiter is set, I think we should: 
- Not print column padding. The output does not require to be human readable. The output should be interpretable by a dumb CSV parser (e.g: "2,, ," means col0 = "2", col1 = "", col2 = " ").
- Not print over state "*". It can confuse a parser (it would expect a number and get a string). And this state can be re-compute.
