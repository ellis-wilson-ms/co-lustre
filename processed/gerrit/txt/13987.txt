--- Message (Patrick Farrell, 2015-03-05 20:57:53.000000000) ---
Patch Set 1: Code-Review-1

I'd like the commit message to also explain what the patch changes to fix the problem, not just describe the symptom.  (I'm glad it does describe the symptom, though.)

Something like: "When there is no fid-in-dirent on the '..' dentry, the previous patch would not get the ea for the '..' dentry unless the parent was a remote directory."

I am also a bit concerned about why we don't see this problem in el6 testing.
Adding fid-in-dirent to the ".." dentry of an existing directory without fid-in-dirent is not possible except when the directory is moved (and even that is special cased, see LU-5626 for details)...
But my understanding is that for a new file system formatted with dirdata on from the beginning, all the ".." dentries should have 'fid-in-dirent' from when they are created.

Do we understand why this isn't the case for ldiskfs on el7 but not el6?
