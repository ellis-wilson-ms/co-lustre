LU-14713 llite: add memory barriier to the trunc_sem.

There is no implicit memory barrier before wake_up_var(), so we need
an explicit release() unless using a RMW atomic operation which
returns a value.

No explicit matching barrier is needed in the wait_var_event()
condition as the included set_current_state() provides one.
