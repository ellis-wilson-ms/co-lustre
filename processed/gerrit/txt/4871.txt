LU-2511 procfs: Disallow secretly (un)writable proc entries

Add an assert to lprocfs_seq_create() to ensure that proc entries
implementing write have user writable modes.  Fix a few callers to
pass this assertion.

--- Message (John Hammond, 2013-01-05 02:26:16.000000000) ---
Patch Set 1:

We only add write permissions for root and only for the stats files.  It makes sense to allow write for these files since writing to them clears the relevant counters.  Technically we are not increasing functionality since root could open these files for writing regardless of their modes.  The point is to use the mode to give a hint to admins.  The only place where I remove write permission in in the case of srpc_rules which does not implement a write method.
