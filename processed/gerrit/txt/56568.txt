LU-18096 enc: ll_get_symlink overlay function

ll_get_symlink() overlay function is necessary to properly encode for
presentation the symlink target when the encryption key is not
available, in a way that is compatible with the overlay function
ll_setup_filename(), so that further readlink without the encryption
key works properly.

> Is there a way to test this properly?  Something like creating a symlink to an encrypted filename, and then comparing the symlink with/without the key to confirm it references the same file (e.g. by FID or inode number).

Well, this test already exists, this is sanity-sec test_47. And this is thanks to this test that this regression with recent kernels was detected.

Note that as explained in the kernel's fscrypt documentation at https://www.kernel.org/doc/html/latest/filesystems/fscrypt.html#without-the-key :
"Symlink targets may be read and followed, but they will be presented in encrypted form, similar to filenames in directories. Hence, they are unlikely to point to anywhere useful."
Indeed, without the key, a symlink target is just encoded for presentation. But there is no chance the "encoded for presentation" form of the symlink target corresponds to the "encoded for presentation" form of the actual target, as these names are not encrypted with the same per-file key.
