LU-12668 ec: basic ec tests

This patch adds a number of tests for erasure coding and requires 8
OSTs:
- Number of parity stripes used cannot exceed the number of stripes in
  the data mirror or else the parity data will not fit in the extent.
  Add tests that we can not create ec mirrors that violate this.
- Resyncing computes the parities correctly.
- The "parity" of a single stripe data component is just a copy of the
  data.
- The stripe size for the ec component to match the stripe size for
  the data component is set.
- EC layout operations (migration, split, truncate, fallocate)
- Error rejection for invalid EC component changes
- lfs find with parity flags
- Multi-stripe data integrity
- PFL lazy parity instantiation
- Failure injection scenarios (OST stop/restart recovery
- Write failures during parity resync, exceeding parity tolerance)

Test-Parameters: trivial
Test-Parameters: testlist=sanity-ec
Test-Parameters: testlist=sanity-ec ostcount=8
