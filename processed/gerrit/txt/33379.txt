LU-11526 rpc: support maximum 32MB I/O RPC

In the new SFA, the block driver allows 32768 max_hw_sector_kb
(32MB) from vOSS (SFA embeded stack) to SFA below. To maximize
driver efficiency, Lustre should also have bump up maximum I/O
RPC size to 32MB.

--- Message (Andreas Dilger, 2018-10-23 05:53:16.000000000) ---
Patch Set 2:

I think this patch is reasonable, and Ihara has talked about increasing the maximum RPC size as high as 64MB.

Please look through the code for the use of PTLRPC_MAX_BRW_* (and values based on this like DT_MAX_BRW_SIZE, OST_IO_MAXREQSIZE, MAX_DIO_SIZE) to see what else needs to be changed to handle this change.  We don't want clients or servers making decisions or allocations based on the maximum RPC size, but rather the current RPC size or the largest RPC size that a server has seen from a client.

This includes values like cl_max_pages_per_rpc, ocd_brw_size, cl_max_extent_pages, ras_rpc_size, ll_readpage(), ll_write_end(), osc_max_write_chunks() on the client.  For the client, I'd prefer not to increase the number of discontiguous fragments in a single RPC more than it already is, as that will increase the amount of memory that the client and OSS need to allocate for large random RPCs, even though that typically will not increase performance.
