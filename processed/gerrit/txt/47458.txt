LU-15835 llite: remove lld_nfs_dentry flag

The kernels knfsd for NFSv3 constantly opens and closes files
for each access which can result in a continuous stream of
open+close RPCs being sent to the MDS. To avoid this Lustre
always enables the MDS_OPEN_LOCK. The fhandles API uses the
same exportfs API as NFS so the MDS_OPEN_LOCK was also always
taken. This has the side affect of not doing a normal close
and introduces a race with a rapid open happens after the
close.

WARNING: CPU: 1 PID: 722753 at fs/exportfs/expfs.c:96
reconnect_path+0x22b/0x2c0

The solution is not to take the MDS_OPEN_LOCK automatically
with the exportfs code. This way fhandle is treated just
like normal file opens. We still need to handle the export
of Lustre over NFSv3 so add a special mount option to always
enable MDS_OPEN_LOCK. Lustre already needs a special option
for NFSv4 (32bitapi) so its not a large ask for a very
uncommon setup. This does show issues exist for
opencache_threshold_count that will be explored in another
patch.
