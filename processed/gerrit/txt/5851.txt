LU-3038 mgs: clear mti in the end of marker

The target information should be cleared when it met
the end marker of each mdc/osc, otherwise, the information
left over in mti will be brought into the config log of
next target, even worse some target might be identfied
wrongly as failover pair.

This llog steal process will walk through all of the config record client log, and convert each osc/mdc records into the osc/mdc records inside the MDT config,  and tmti will be used to collect the current mdc/osc information, which marked by "add mdc" marker pairs,  so once the end of the marker is reached, we need clear current tmti information, and be ready to collect next osc/mdc information,  otherwise, the leftover information from last osc/mdc will confuse the convertion process, for example mti_nids, etc.
