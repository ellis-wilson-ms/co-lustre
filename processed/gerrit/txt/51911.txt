--- Commit Message ---
LU-10329 obdclass: add LINKEA_FL_PACK_INDEX flag

Change link_ea_header.leh_padding to leh_flags, and add one flag:
LINKEA_FL_PACK_INDEX: append parent object MDT index after name in
	link_ea_entry.

Add dt_device.dd_linkea_pack_index to indicate whether the parent
object MDT index should be packed into linkea. It can be set by:
	lctl set_param osd-ldiskfs.<MDTDEV>.linkea_pack_index=[0|1].

This is preparation work for REMOTE_PARENT_DIR scalability: put remote
object agent entries under per MDT subdirs.

> I don't understand the need for mdt_index to be stored in linkEA and the format change.

OSD can't access FLD, so it can't know which MDT pfid is on. Last time you suggest move remote agent entry handling to MDD layer, but it's complicated, and will introduce extra RPCs. So in this change, MDD fill MDT index info in linkea, and pass to OSD, then OSD can handle all this in old way, and put remote object entries in obj map to improve scalability.
