--- Commit Message ---
LU-17431 nodemap: allow deleting an id map from a dynamic nm

Deleting an id map from a dynamic nodemap requires specifying the '-d'
option to 'lctl nodemap_del_idmap'. This in turn sends the appropriate
ioctl to the MDS or OSS device.
On kernel side, we need to allow deleting an idmap to a nodemap even
if the nodemap config file is not accessible (i.e. we are not on the
MGS). This dynamic nodemap is not stored on disk.
