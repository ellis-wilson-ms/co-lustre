--- Message (Andreas Dilger, 2016-08-13 03:18:48.000000000) ---
Patch Set 1:

What is the justification for the change from EINTR to ERESTARTSYS?  I thought the former is what should be returned to userspace when a process is interrupted before completing any IO, while (AFAIK, haven't verified) that ERESTARTSYS is caught in the kernel and actually restarts the syscall?

--- Message (Patrick Farrell, 2016-08-13 15:05:56.000000000) ---
Patch Set 1:

Yes, mostly, but -ERESTARTSYS doesn't *force* a restart, it makes one possible if userspace has asked for it by setting the SA_RESTART flag on a sigaction.  If the flag is not set, ERESTARTSYS is translated to EINTR.

We've made this change elsewhere, in LU-3020 and LU-3581 - I wrote up a slightly long-winded explanation of it here:
https://jira.hpdd.intel.com/browse/LU-3020

Sorry to reference my own previous comments, it's surprisingly hard to find references explaining this behavior...

I originally assumed that Lustre was using -EINTR for good reason. Since it sounds like there's no particular reason, I'll see about making the larger change from -EINTR to -ERESTARTSYS for this patch.  Now's a good time to start a patch like that for 2.10.
