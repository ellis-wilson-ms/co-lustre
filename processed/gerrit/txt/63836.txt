--- Commit Message ---
LU-8453 obdclass: fix use-after-free in class_manual_cleanup()

class_manual_cleanup() accesses obd->obd_name after calling
class_process_config() for LCFG_CLEANUP and LCFG_DETACH.
These operations may drop the last reference on the OBD device
via class_decref(), freeing the obd structure. The subsequent
CERROR calls then dereference freed memory, causing a kernel
page fault in strnlen() during debug message formatting.

Save obd_name to a local buffer before the cleanup/detach
operations and use the saved copy in all subsequent references.
