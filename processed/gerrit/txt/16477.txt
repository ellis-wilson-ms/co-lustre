LFSCK is online tool. The system must be mounted as "lustre" to find out the orphan OST-objects. So should we keep the offline mode ll_recover_lost_found_objs tools in the case of the system cannot be mounted up (as Lustre)?

--- Message (Andreas Dilger, 2015-10-26 17:46:32.000000000) ---
Patch Set 4:

Fan Yong, are there really cases where ll_recover_lost_found_objs will fix a problem that prevents the server from mounting? Shouldn't early OI Scrub fix the problems that would prevent it from mounting, at least in the context of files in lost+found.

--- Message (Fan Yong, 2015-10-27 04:16:41.000000000) ---
Patch Set 4:

> Fan Yong, are there really cases where ll_recover_lost_found_objs will fix a problem that prevents the server from mounting? Shouldn't early OI Scrub fix the problems that would prevent it from mounting, at least in the context of files in lost+found.

In theory, most of the orphan OST-objects can be put under backend lost+found by e2fsck, especially some key files, such as the OI framework related directories objects become orphans. For those bad cases, we have to use some user space tools to recover that.

Unfortunately, current ll_recover_lost_found_objs will not do more than the online LFSCK can do. Since then, we can totally replace it.

--- Message (Fan Yong, 2015-10-27 07:36:10.000000000) ---
Patch Set 4:

Andreas, sorry for the confused comment. All its functionalities can be covered by the online LFSCK. So I agree with you to remove it.

--- Message (Fan Yong, 2015-10-27 07:47:19.000000000) ---
Patch Set 4:

> Is it possible to handle the cleanup of lost+found early enough in the mount to fix the problems that can be fixed running this tool offline?

For the LFSCK, it can handle most of the cases in the /lost+found, but if the /O structure itself is moved to /lost+found by e2fsck, then it is not easy to handle. Because the initial OI scrub cannot use a lot of time during the mount process to find out and recover the lost /O or some of its children in order. It is more convenient for the user space tools to analysis and recover. Unfortunately, our current ll_recover_lost_found_objs does not support that yet.
