in lustre_cfg.c. Sometimes errno may change intermittently
and results in wrong error message to be displayed.

--- Message (Vinayak, 2016-02-27 03:34:17.000000000) ---
Patch Set 1:

Hello Dimitry,

There is a small window of time between storing the error code into errno when the system call returns and printing of this value by program during which the errno can be changed for example any thread in execution in the same process may change it. That's the reason why we store the "errno" into "rc". So same "rc" should be used in fprintf.

In rare cases, fprintf can also set errno. So its safe to use stored value "rc" while reporting the error string in fprintf.

--- Message (Christopher J. Morrone, 2016-02-29 21:03:13.000000000) ---
Patch Set 1:

Actually, that is not accurate.  errno is explicitly a thread local variable.  It pretty much has to be, otherwise the entire standard C library error handling would be unsafe.  Your "fix" here would not fix that race condition either, only make it a little smaller.

Additionally, errno in these case is passed by value to printf, not passed by reference.  While errno can potentially change under printf, it has already been passed by value so there is no risk of that errno change modifying the output of the print statement.
