--- Commit Message ---
LU-0000 utils: fix EC setstripe validation timing

Fix validation logic for EC (erasure coding) setstripe
commands that was incorrectly rejecting valid EC layouts.

The issue was that EC validation occurred before the final
component was processed, causing false positives when
checking if at least one EC component was created.

This change moves the EC component count validation to
after component processing is complete, while keeping
basic validation (conflicts with -c, setstripe-only)
earlier in the flow.

Before this fix:
- lfs setstripe --ec 4+2 -E EOF file would fail with
  "--ec requires explicit component end with -E"
- Tests 3a and 3c in sanity-ec.sh would skip

After this fix:
- Single and multi-component EC layouts work correctly
- All sanity-ec tests pass except 0d (needs >= 4 OSTs)
- EC file creation and IO operations work end-to-end
