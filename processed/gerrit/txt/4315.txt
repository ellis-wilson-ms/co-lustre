LU-1732 utils: allow ldiskfs wide striping with ea_inode

Format the MDT filesystem with the "ea_inode" option by default
so that files can have more than 160 stripes, and large xattrs
over one filesystem block in size (normally 4096 bytes).

--- Message (Andreas Dilger, 2018-08-18 00:07:17.000000000) ---
Patch Set 5: Code-Review-1

Marking this -1 for now, as it was causing repeated test failures due to OOM on the client.  I suspect the 1MB max_ea_size (also referenced in places as "max_easize") is causing too much memory to be pinned on the client for RPC reply buffers or similar.

Either we need to shrink the RPC reply buffers after the initial send to match the actual reply size, or allocate smaller buffers by default and only expand them when needed, or a combination of the two.
