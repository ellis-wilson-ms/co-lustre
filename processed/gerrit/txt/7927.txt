LU-4002 hsm: add hsm_restore_item

Divide the action field of hsm_action_item into a enum part and a
flags part. Do this so that the CDT can send the attributes needed for
restore to the copytool. The goal is to revert
http://review.whamcloud.com/#/c/7792/ without allowing the deadlock
described in LU-4002.

*** Not intended to land as is. For external use only. ***

--- Message (Jinshan Xiong, 2013-10-30 17:10:11.000000000) ---
Patch Set 1:

"John, this patch is adding too much complexity to hai management IMO. It is bug prone, not especially this patch but other people having to patch this later could miss this. Moreover, copytools also have to take care. I do not like the polymorphism aspect of hai/hri."

It's generally bad to take another lock while you've taken one. There is a discussion on LU-3601.

In our case here, we've already held LAYOUT lock for restore. However in copy tool, it fetches state of released file which will grab LOOKUP, UPDATE, and PERM lock. This is really deadlock prune. So a rule of thumb for copy tool implementation would be to not take any lock on restore.

Copy tool is a dedicated thread to perform special job, I don't think transferring all information it needs in the action RPC is a bad thing.
