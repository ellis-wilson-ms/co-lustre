LU-13417 test: add mkdir_on_mdt0()

Once default LMV is set on ROOT, and default stripe offset is "-1",
mkdir may not create directory on MDT0, but it's a premise for many
tests. Add a function mkdir_on_mdt0() to create directory on MDT0
by "lfs mkdir -i 0".

Replace mkdir with mkdir_on_mdt0() for such tests in sanity.sh and
sanityn.sh.

Test-Parameters: trivial testlist=sanityn

(4 comments)

The -1 is because sanity.sh includes the new test for checking that the default LMV is on the root directory, but that doesn't happen until the next patch now...

It would be better to split this patch into two or more separate patches.  One (or more) for the test scripts that are run regularly in review testing and are known to pass, and one for each other test script that is currently failing regularly.

This is just a large patch, and the chance of some defect (typo, test bug, etc) existing in one of the changes is relatively high.  Also, because the Gerrit Janitor testing will run *each* modified subtest 20x on both ldiskfs and ZFS (***even if they are normally skipped by ALWAYS_EXCEPT***) means that the chance of some random subtest failure is high.

Having smaller patches will allow us to land the important parts of this change more quickly, and the other parts can land when they pass a test run and/or those test scripts are fixed.
