LU-19592 lraft: change config by raft

The MGS config change should be replicated to all MGS's in the
system by Raft FSM(finite state machine): the MGS config is changed
by ioctl, and there are a number of ioctls, to avoid digging deep
into the details, the obd_ioctl_data is appended to raft log.

Because the config change command is run on MGS0 (in case it's
separate MGS), it's quite possible that it's not raft leader, the
obd_ioctl_data is sent to the raft leader.

A new RPC RAFT_SET_INFO is introduced to send obd_ioctl_data to raft
leader node.

A new test sanity-raft.sh test_3 is added.

CAVEATS:
* normally the config change command is run on MGS0 node, but if
  separate MGS is deployed, the config change command should be run
  twice: one is run on MGS0 (separate MGS), and another on a Raft
  node that started MGS service.
