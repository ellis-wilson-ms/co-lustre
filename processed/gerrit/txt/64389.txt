LU-19848 lnet: add p2pdma support

Currently, Lustre supports GPUDirect Storage for NVIDIA
devices via the out-of-tree GDS driver [3], that interfaces
with the out-of-tree NVIDIA display driver. Recent versions
of cuFile and CUDA support p2pdma [1] as an alternative to
the out-of-tree GDS driver [2].

Lustre should adopt p2pdma as the primary method to support
GPUDirect Storage. This enables us to drop the dependency
on the out-of-tree GDS driver - which is a requirement for
upstreaming Lustre into the Linux kernel. Furthermore, it
allows us to increase device support beyond only NVIDIA [4].

Linux kernel dma functions should natively understand
ZONE_DEVICE pages. Lustre needs only to correctly map the
pages to the correct network device. We leverage the kernels
native distance calculation to replace the hints the GDS
driver previously provided. We also implement a debugfs
file to display the distances in a table format (for debugging).
Also, we provide a new set of counters to track p2pdma
send/recv.

[1] https://docs.kernel.org/driver-api/pci/p2pdma.html
[2] https://docs.nvidia.com/gpudirect-storage/troubleshooting-guide/index.html#troubleshooting-and-faq-for-nvme-support-using-linux-pci-p2pdma
[3] https://github.com/NVIDIA/gds-nvidia-fs
[4] https://github.com/ROCm/hipFile/README.md
