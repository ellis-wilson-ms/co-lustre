I wish there was a standard macro of some kind (either in Linux or in the C standard). It seems like a fairly common operation. I think omitting the macro puts the logic closer to the comment discussing the code.

If we used a macro, I think we should aim for something that could be used more generally, and put it in some common header with the aim to get it adopted more broadly in the kernel once the client is upstream. I tried to find if anyone was doing that already, but I didn't see anything. Do we have a header for generic macros? In libcfs maybe?

Using max seems a little opaque, imo.

--- Message (Neil Brown, 2023-08-24 05:56:49.000000000) ---
Patch Set 1:

I don't think that a saturating add is a common operation.
The only example I could think of is in refcount.h
It does
  if (old < 0 || old + increment < 0)
        handle_saturation
  else
       var = old + increment

or similar.
I thought of using min() because what we really want is;

   a = min(a+b, MAX_INT)

except that we need a+b to not overflow, which of course it can.
So I subtract b, and add it back:

   a = min(a, MAX_INT - b) + b;

But I agree it isn't completely transparent.
I generally like the cond ? a : b construct, but for some reason it didn't seem particularly clear in this case - maybe because the calculation needed is a bit unusual..
The comment explaining what it needed makes it easy enough to interpret the code.  I'm not exactly against it - I'm just not convinced that it is an big enough improvement to justify the change.  Others might disagree and I won't try to block it.
But if anyone can think of a different approach that IS a big improvement - that would be great.
