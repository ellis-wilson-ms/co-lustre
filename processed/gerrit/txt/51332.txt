--- Commit Message ---
LU-16890 obd: OBD_FREE_PRE() to ignore NULL pointers

This patch modifies OBD_FREE_PRE() to not LASSERT
when null pointer is passed as the kfree() function
accepts NULL as no-op.

This is first set of patch that modifies the definitions.
Subsequent set will modify the callers to accept this
case.

As as example:

This caller will now be change to:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
char *test;
OBD_ALLOC(test, 32);
...
OBD_FREE(test, 32);

Previously the caller used to be:
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
char *test;
OBD_ALLOC(test, 32);
...
if (test)
	OBD_FREE(test, 32);

Patch Set 1: Code-Review-1

My thought was rather that OBD_FREE() and friends should just accept NULL as a normal case. The kernel kfree() code simply accepts NULL as a no-op, which simplifies cleanup in case of allocation errors or other issues during setup.  That brings the Lustre code more inline with the kernel coding style, so that it doesn't need to check for NULL before calling OBD_FREE_*().
