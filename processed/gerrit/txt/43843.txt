--- Commit Message ---
LU-14709 osp: invalidate upon .release

Normally remote objects are invalidated upon unlock, but sometimes
remote objects are accessed without LDLM lock, e.g. fid2path,
is_subdir check and lfsck code, since these code path are rarely
called, which means such object refcount is often zero after use, it's
better to invalidate remote object upon last refcount put than
call .invalidate in every places like this.
