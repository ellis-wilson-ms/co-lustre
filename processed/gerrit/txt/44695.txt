--- Message (James Simmons, 2021-08-20 14:51:20.000000000) ---
Patch Set 2: Code-Review+1

Looking at the native patch for refcount_t the old assert is still their. When porting this I have the option to use refcount_inc_not_zero() which will report just a warning instead of crashing the node. Should I do that or keep the assert in your opinion.

--- Message (Patrick Farrell, 2021-08-20 15:22:56.000000000) ---
Patch Set 2:

Keep the assert.  This should crash the node - it's a reference counting error, so if this is ever wrong, the result will be user-after-free or other random memory access.  Very bad - random crashes, possible data corruption, etc, etc.  Definitely better to crash intentionally.  (Especially because it will help catch any bugs in testing before they get to a customer.)
