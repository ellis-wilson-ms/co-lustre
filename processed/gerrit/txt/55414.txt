to either '/etc/lustre/mount.client.params' or
'/etc/lustre/mount.FSNAME.params' to be set upon

Tim, the reason for this is a bit complex, and an artifact of how Lustre developer testing is done.  When running tests directly from the build tree (e.g. "make; cd lustre/tests; bash sanity.sh") we are still required to run the mount.lustre helper from /sbin when using "mount -t lustre ..." because that is the only place that mount will look for it (for security reasons if non-root users are able to trigger filesystem mounts).

In order for this one binary to be runnable from /sbin it is more complex to depend on a shared library that is also not installed in the system, so static linking of lustre_param.c was used to ensure mounting still worked in the various test environments (note 37 versions of this patch) and they could run "set_param" as needed.

To avoid dragging in a bunch of *other* files, a few stub functions were used to avoid dependencies for functionality that is not actually needed.  This shouldn't be needed elsewhere since most applications would link to liblustreapi.
