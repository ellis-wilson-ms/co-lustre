LU-0000 llite: fix RESYNC_DONE with component IDs

Fix kernel bug preventing RESYNC_DONE with
component IDs from working for plain EC files.

Client-side struct mismatch: The code used struct
ll_ioc_lease * instead of struct ll_ioc_lease_id *
when processing RESYNC_DONE. This caused incorrect
memory layout when copying data from userspace,
reading component IDs from wrong memory offsets.

Server-side validation: For plain EC files in
RDONLY state, parity components are not stale. The
code only marked component IDs as processed if they
were stale, causing validation to fail with EINVAL.

Changed ll_close_inode_openhandle() and
ll_file_unlock_lease() to use struct
ll_ioc_lease_id * for RESYNC_DONE operations.
Modified lod_declare_update_sync_pending() to mark
parity component IDs as processed even when not
stale, allowing RESYNC_DONE to work for both RDONLY
and WRITE_PENDING states. Updated ec_lease_test.c
to use correct struct type and set lil_mirror_id = 0
for plain EC files.

Added placeholder tests for LL_LEASE_LAYOUT_MERGE
and LL_LEASE_LAYOUT_SPLIT lease operations on EC
mirror files. These tests currently skip since EC
mirror support is not yet fully implemented.
