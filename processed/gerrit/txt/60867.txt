LU-18878 ptlrpc: evictor/prolong timeout formula fix

Let's stay with the main rule - cluster is supposed to survive just
one (the worst ever) failure, not a set of failures. The current
formula covers:

- an RPC timeout;
- the 1st reconnect time out;
- the 2nd reconnect to the failover pair returns an error;
- the 3rd reconnect to the original node succeeds;

i.e. 2 failures. At the same time, The 2nd (reconnect timeout) failure
was the result of the LNET problems, which were fixed separately.

Besides that, a fix for the ping evictor timeout, a normal RPC case,
where the timeout of an ping itself was not accounted in the formula.

Besides that, a fix for the extra timeout used for obd_eviction_timer,
where net latencies for the accounted RPCs were taken not correctly.
