LU-0000 llite: check inode locking on truncate

We need to actually have the inode locked for calling this,
let's do a little testing and see where it isn't locked
and/or where it hangs.

For now, we assert when it's !locked, which will catch some
cases.

Once those have been identified and sorted out (hopefully),
we'll use that to define when we DON'T need to take the
lock, and then flip around to locking it in all other cases
and see what hangs come up.

This two-directions approach can hopefully find all the
cases we're missing.
