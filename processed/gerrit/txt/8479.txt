--- Commit Message ---
LU-1778 libcfs: add a service that prints a nidlist

The libcfs already provides services to parse a string into a nidlist
and to match a nid into a nidlist. This patch implements a service
that prints a nidlist into a buffer.

This is required for instance to print the nosquash_nids parameter
of the MDT procfs component.

Additionally, this patch fixes a bug in return code of
parse_addrange() routine, so that parsing of nids including
a * character works fine ('*@elan' for instance).

Liang,

I am sorry but I don't understand your point.

In my opinion, the code is safe. Buffer overflow is not possible because:
- cfs_snprinf(buffer, size, ...) write at most _size_ bytes (including the
  trailing null byte) and returns the actual number of bytes (not including
  the trailing null byte) written to _buffer_
- all print subroutines called by cfs_print_nidlist() in the stack down to
  cfs_snprintf() have the same semantic than cfs_snprintf()
- calls are all made like this:
     i += xxx_print(buffer + i, count - i, ...);
     i being the offset in the buffer where next characters can be written

It means _i_ is always strictly less than _count_. There is no need to
have additional check 'i < count'.

The entry points cfs_expr_list_print() and cfs_print_nidlist(), which are
exported symbols, have an additional check to ensure count parameter is
strictly positive.

During the nidlist printing, once i equals count-1, cfs_snprintf() does
not write any additional character and returns 0. There is no buffer overflow.
