Patch Set 2:

What I mean is that in the normal test case, the "skip_env" will not be hit, the test will PASS, and it may appear that this patch is OK.

If the "skip_env" check is hit in developer manual interop testing, it will report SKIP and not cause a test failure, but if there is official release interop testing done then "skip_env" will cause the test (and hence the whole build) to FAIL instead of SKIP.  Looking at the code:

 skip_env () {
     $FAIL_ON_SKIP_ENV && error false $@ || skip $@
 }

My understanding is that FAIL_ON_SKIP_ENV=true for release builds, or at least that was the case at CFS/Sun/Oracle.  The intent is that "skip_env" is to ensure tests which might legitimately be difficult for developers to run (e.g. MPI tests), but should not be skipped during automated testing or there will be a hole in our test coverage.

So, because this test cannot possibly pass if it is run against a server that does not have support for these stats, the test should just use the normal "skip" instead of "skip_env", as I had originally suggested in my comment.
