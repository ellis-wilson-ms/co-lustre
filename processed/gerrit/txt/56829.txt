This test session is added "for our own protection" so that tests that do not pass consistently (at least 95% of runs) cannot be landed and introduce intermittent failures.

However, for old subtests that have not run with "ONLY_REPEAT" before, there may be latent bugs if the test is run multiple times.  In this case, it looks like the subtest failed after 34 iterations, so it looks like this subtest is just flakey (LU-17712)?

If we can't get this subtest to pass when running in a loop, then possibly it needs to exit after some number of iterations.  It would also be useful to add some extra debugging to the subtest to help understand why it is failing.
