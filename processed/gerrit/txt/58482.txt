> LNet and libcfs need a separate value. The reason is that the native client version is always lower than the OpenSFS version. This ensures that if Opensfs modules are installed then get loaded instead of the native client.

That's not the correct way to do that, IMO. We should use a targeted depmod config file. We used to use a global config file (i.e. https://review.whamcloud.com/c/fs/lustre-release/+/54025) which interfered with other modules. But if we make one that lists each Lustre module by name, we can get the desired effect without making the version strings wrong.
