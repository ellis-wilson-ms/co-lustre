LU-18996: Fix race condition in obd_device_list_seq_show

When calling obd_device_list_seq_show, if a
class_unregister_device is happening at same
time, the race condition can cause unnecessary
LBUG when asserting obd->obd_minor == num.

In this case no need to LBUG, just return NULL
and behave like the obd device does not exist.

This won't fix the race. The real bug is that obd can be freed while obd_device_list_seq_show is still using it. This can still happen even with this new check.

If you look at the latest code in master [1], we solve this by holding a reference to the obd. This ensure the reference to obd is valid until we drop the reference.
