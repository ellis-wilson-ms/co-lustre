--- Commit Message ---
LU-1337 llite: 3.3 changes super_operations/inode_operations

1. Kernel 3.3 switchs super_operations to use dentry as parameter
   (kernel commit 34c80b1d93e6e20ca9dea0baf583a5b5510d92d4).
   Add LC_SUPEROPS_USE_DENTRY/HAVE_SUPEROPS_USE_DENTRY to check it.
2. Kernel 3.3 switchs inode_operations to use umode_t as parameter
   (kernel commit 1a67aafb5f72a436ca044293309fa7e6351d6a35).
   Add LC_INODEOPS_USE_UMODE_T/HAVE_INODEOPS_USE_UMODE_T to check it.

--- Message (Andreas Dilger, 2012-08-12 16:55:15.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

Better to define an ll_umode_t macro that maps to the correct argument type for the kernel. That way for upstream submission it is easy to use sed to replace ll_umode_t -> umode_t in the code, instead of having to edit several places in the code.

Also, it would be better if the 3.3 patches were before the 3.4 patches in the series so that they can land first and we at least get 3.3 working instead of half of each kernel. Ideally, each patch would be submitted independently, but they all conflict in the lustre-core.m4 file. It should at least be possible to have a separate 3.3 stream and a 3.4 stream of patches.
