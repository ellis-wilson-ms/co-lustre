LU-20104 lnet: fix selftest session teardown and BRW bugs

Fix multiple issues in lnet-selftest that caused kernel
deadlocks, crashes, and stale sessions.

In srpc_handle_rpc(), call srpc_server_rpc_done() when a
server RPC workitem re-enters in SWI_STATE_DONE after an
async LNetMDUnlink event, preventing orphaned RPCs on
scd_rpc_active.

In sfw_remove_session(), stop and drain client-side batches
before deactivating the session, then drain server-side
RPCs with re-abort on each iteration.

In lstcon_session_end(), stop all active batches before
sending end_session RPCs.

Add lst_wait_until_timeout() and use it in
lstcon_rpc_cleanup_wait() to bound waits for console RPCs,
preventing infinite spins while holding ses_mutex.

Defer zombie session destruction from lst_test_wq to
lst_serial_wq via sfw_destroy_session_deferred(), avoiding
cancel_work_sync() deadlock on the same workqueue.

Fix BRW offset handling: mask offset with ~PAGE_MASK in
brw_client_prep_rpc(), allocate bulk with off+len in
brw_client_init(), and validate off+len <= LNET_MTU.

Add lnet-selftest.sh tests for session teardown stress
and BRW offset edge cases.
