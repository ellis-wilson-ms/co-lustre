LU-16741 ptlrpc: remove unnecessary asserts

ptlrpc_free_committed is at no risk of being called with
either a null import or the import lock unlocked.  These
don't have a significant cost, but they're not adding much
either.

Removed 'trivial' from this patch to ensure the LU-16741
refactoring series gets at least one full run.

There is not an LU, and I didn't open one because I'm not game for building what's required...

So lockdep is a relatively simple (but capable) lock dependency checker, and it requires you create a descriptive model of your locking.  The kernel docs are the best resource I'm aware of (but I've never actually used it):

https://www.kernel.org/doc/Documentation/locking/lockdep-design.txt

It would be a fair bit of work to do.  If someone wants to take it on, I'd be happy to try to support it.  Obviously you'd want to start with doing it in one specific area of code first.
