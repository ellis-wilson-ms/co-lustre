Patch Set 15: Code-Review-1

Thank you, Chris, for this patch. I think this is a move forward with making sure LNet is tested. 

I do have one major issue that I would like to see worked out. From what I can see, there is an implicit belief that test 0a will be run before 0b, but this isn't necessarily true. Some people/Lustre providers run each test individually to speed things up and we've worked to combine tests if necessary or place conditions in tests if there is a file or something setup that is expected to be there so the test will run correctly. 

So, we would like 0b to be able to be run on a separate machine/VM than 0a. When we do this we get an error:

# NAME=ncli ./auster -k -v sanity-lnet --only 0b
...
== sanity-lnet test 0b: Bring up the LNet Network Interface system =================================== 23:07:38 (1574377658)
/usr/sbin/lnetctl lnet configure
configure:
    - lnet:
          errno: -22
          descr: "LNet configure error: Invalid argument"
 sanity-lnet test_0b: @@@@@@ FAIL: test_0b failed with 234 
  Trace dump:
  = /usr/lib64/lustre/tests/test-framework.sh:6108:error()
  = /usr/lib64/lustre/tests/test-framework.sh:6410:run_one()

For this test suite, we've broken the promise that you can run each test individually. 

So, should some/all the 0a, 0b, ... tests be combined?
