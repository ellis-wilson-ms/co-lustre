LU-17514 target: add expected_clients param and max client tracking

Add a new 'expected_clients' module parameter to hint services the
expected number of client connections, along with automatic tracking
of maximum clients ever connected.

This allows code to have an idea of the system scale during startup
so that data structures, tunable values, etc. can be selected with
some idea of future growth.  This doess not impose a limit on the
number of clients that may mount the filesystem.

## Summary

**The Fix:**
Added the missing debug mask parameter `D_WARNING` to the `CDEBUG` macro call on line 51 of `lustre/target/tgt_lastrcvd.c`.

**Rationale:**
The `CDEBUG` macro in Lustre requires two mandatory parameters:
1. **Debug mask** - An unsigned int flag that controls when the debug message is printed (e.g., `D_INFO`, `D_WARNING`, `D_HSM`, etc.)
2. **Format string** - The printf-style format string

The patch incorrectly called `CDEBUG` with only the format string, omitting the debug mask. This caused a compile error because:
- The compiler tried to pass the format string (a `char *`) as the debug mask parameter (which expects `unsigned int`)
- This resulted in "makes integer from pointer without a cast" errors

The fix adds `D_WARNING` as the first argument, which is appropriate since this `CDEBUG` call is in an error handling path (when `rc != 0`). This matches the pattern used throughout the Lustre codebase where warning-level messages use `D_WARNING` or the `CWARN` macro (which is equivalent to `CDEBUG(D_WARNING, ...)`).
