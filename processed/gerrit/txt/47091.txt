LU-15748 osc: Correctly pass obdo(o_valid) flags

When punch mode to fallocate(cb037f305c64c) was landed.
The new check for 'o_valid' flag was added under ofd.
However, the under client(osc) it was not getting
populated. This was resulting in fallocate call being
terminated with EOPNOTSUPP.

This patch populates the o_valid with correct flags
before sending it to server

It isn't really possible to retroactively fix older releases in this way. Even if this patch landed on b2_14 it would not help the sites that are running 2.14.0. 

Do you know if 2.14.0 correctly populated these fields, but just didn't set the flags?  In that case I would recommend that we disable the new flag check on master *before* 2.15.0 is released, and put it under

 #ifdef LUSTRE_BUILD_VERSION > OBD_OCD_VERSION(2, 18, 52, 0)

or check if the client version is >= 2.15.0 before enforcing the check. Alternately, it could be checked by an OBD_CONNECT2_ flag already added in 2.15.0, but I'm a bit reluctant to burn a flag for this minor issue.
