--- Message (Shaun Tancheff, 2023-07-01 05:45:04.000000000) ---
Patch Set 4: Code-Review-1

> Patch Set 4:
> 
> On kernel prior to 4.8 we used to use NR_UNSTABLE_NFS.  After this patch we will use NR_WRITEBACK.
> 
> That change should be mentioned (and justified) in the commit message.

In older kernels with NR_UNSTABLE_NFS support a background writeback will be
issued when NR_WRITEBACK is under the dirty page limit and adding NR_UNSABLE_NFS
exceeds the dirty pages limit in balance dirty pages.

Currently this does neither help nor hurt lustre clients. However if/when
("LU-16713 llite: writeback/commit pages under memory pressure")
https://review.whamcloud.com/c/fs/lustre-release/+/50544 lands then
NR_UNSTABLE_NFS may improve client I/O in some cases.

So it seems better to keep the NR_UNSTABLE_NFS hinting for the older kernels.
