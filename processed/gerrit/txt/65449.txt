LU-20176 osd-ldiskfs: trigger full OI scrub by RPC

Partial OI scrub triggered by RPC may not fix inconsistencies
because scrubbing inconsistent mappings won't fix anything, and the only
way is to iterate all inodes and fix the wrong mappings.

After OI scrub finish, move those inconsistent mappings to stale
mappings, so the subsequent access to them will return -ESTALE
immediately.

Remove partial scrub code, since it complicates scrub state machine and
tends to cause issues.

Update sanity-scrub.sh to reflect the change.

## Fix Summary

The compile errors were caused by the patch removing the `os_in_prior` field from `struct lustre_scrub` in the ldiskfs OSD implementation, but the ZFS OSD implementation (`lustre/osd-zfs/osd_scrub.c`) still contained references to this removed field.

### What was fixed:

1. **Line 151-153** - Removed the check for `scrub->os_in_prior` and the code that retrieved the `oii` (inconsistent item) from the list based on this condition. The partial scrub functionality that this supported has been removed.

2. **Line 216-218** - Removed the conditional logic that incremented different counters (`sf->sf_items_updated_prior` vs `sf->sf_items_updated`) based on `os_in_prior`. Now always increments `sf->sf_items_updated`.

3. **Line 352-359** - Removed the assignment `scrub->os_in_prior = 1` when processing items from the inconsistent items list.

4. **Line 414-429** - Simplified the `osd_scrub_exec()` function by removing the conditional block that checked `os_in_prior`. The code now always runs the prefetch logic when not in full speed mode, eliminating the else-branch that was resetting `os_in_prior`.

### Rationale:

The patch (LU-20176) removes partial OI scrub functionality triggered by RPC because it was problematic and complicated the scrub state machine. The partial scrub would only scan inconsistent mappings rather than all inodes, which could fail to fix actual inconsistencies. The `os_in_prior` field was used to track whether the scrub was processing items from the priority list (inconsistent items found by RPC), but with partial scrub removed, this tracking is no longer needed. The ZFS OSD code needed the same changes that were applied to the ldiskfs OSD to remove references to this deleted field.
