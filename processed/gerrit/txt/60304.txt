LU-18756 ofd: enable resource id repair for ost rpcs

This patch enables and includes the resource id repair to the
corresponding OST RPCs:
- OST_READ: In addition to the id check, the OST object is now
  repaired if its attributes are unset.
- OST_GETATTR: The object is only repaired if its attributes are
  unset.  Note, similar to MDS_GETATTR, we do not include the id check
  because it would break stat-like operations.

In summary, the following OST RPCs include the id check, id repair, or
both:

ID check only. ID repair is unnecessary for these RPCs since they
implicitly set new attributes on the OST object:
- OST_SETATTR
- OST_WRITE
- OST_PUNCH
- OST_FALLOCATE

ID repair only:
- OST_GETATTR

ID check and repair:
- OST_READ

Test sanity-sec/75b was added to test id repair. Because there seem to
be no operations left that leave an object untagged, an "old" untagged
OST object that is missing IDs is simulated by modifying the OST
object metadata through debugfs.
