--- Commit Message ---
LU-2642 osd-ldiskfs: Unlock inode before iput

Commit 250df6ed added a WARN_ON for I_NEW being set in iput_final
(v2.6.38).

osd_oi_index_create_one calls calls both iget_locked and
ext4_create_inode, both of which will return an inode with I_NEW set.

With the check in iput_final, we'll hit a WARN_ON every time this
function is called.

We unlock the new inode before the iput as other file systems already
do.
