LU-15444 tests: assertion for llog_hdr->lrh_tail

This patch adds an assertion for the OUT write, to check that
the llog_header->lrh_tail.lrt_index is increasing. This will prevent
llog record reordering and inconsistencies in the llog.

For test only.

Parallels transactions do not have a strict order of llog
records. For example, transaction 1 may write to llog index 5,
transaction 2 may write to index 2, and transaction 3 may
write to index 3. During the llog write, bitmap and tail are updated.
After commiting transaction 2, the tail will point to index 2
in the llog, and the bitmap will have zero for index 5.
So this update log contains incorrect data, as the first transaction
is missing at the remote MDT. Any MDT recovery process based on this
log will lead to an incorrect FS state. In a situation where
transactions udpate different u32 words in the bitmap, if there is an
error for an OUT transaction, it can lead to another error when
accessing the llog because the tail is smaller than the last llog
record.
