(1 comment)

Sorry for the delay in inspecting this. If you find your patches are not being inspected or landed in a timely manner, please ping people in the patch or in Skype to get them moving. 

I think the reason they were being excluded is because there are separate stats for individual MDS_REINT operations (e.g. mds_rename, mds_setattr, etc) that were added. Similarly, individual DLM lock types were added (e.g. ldlm_ibits_lock).

See /proc/fs/lustre/mdt/MDS/mds/stats for example. I think it would be more useful to also account for the individual operations in the MDC and OSC stats rather than just MDS_REINT and LDLM_ENQUEUE. With the current patch this would add double accounting on the server side. 


It might also be useful to have a test if these RPCs are being accounted for. There are already some tests for the server side stats, but I'm not sure about the client stats.
