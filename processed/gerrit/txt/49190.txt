LU-16321 osd: Allow fiemap on kernel buffers

Linux commit v5.17-rc3-19-g967747bbc084
  uaccess: remove CONFIG_SET_FS

When KERNEL_DS gone lustre needs an alternative for fiemap to
copy extents to kernel space memory.

Direct in-kernel calls to inode->f_ops->fiemap() can utilize
an otherwise unused flag on fiemap_extent_info fi_flags
to indicate the fiemap extent buffer is allocated in kernel space.

Include ldiskfs patches for ldiskfs_fiemap() to
define EXT4_FIEMAP_FLAG_MEMCPY and utilize it.

(6 comments)

It's too bad the abstraction here is so lousy that we need to duplicate the whole call stack to make this change, instead of just adding a replacement function into the ext4_iomap_report_ops or similar.  It's also sad that the whole set_fs() interface went away, but not much can be done about that.

In any case, I think this can be trimmed down some more, and the flag naming needs to be more consistent/less confusing.

We might also think about implementing this fiemap replacement directly in osd-ldiskfs rather than as a patch to the ext4 code?  That would reduce the ongoing burden of patching the kernel each time, and the only caller of this alternate path is osd-ldiskfs...
