--- Commit Message ---
LU-16597 tests: sanityn/101a add check for 'ost_write'

For DoM files, the writes within first component
should not go to OST. This means that the 'ost_write'
statistic should not be incremented.

This patch adds a check to verify 'ost_write'
statistics immediately after the IO to MDT.

Test-Parameters: trivial fstype=ldiskfs testlist=sanityn env=ONLY=101a
Test-Parameters: trivial fstype=zfs testlist=sanityn env=ONLY=101a

I am not sure about idea behind this patch, why do you need that extra check exactly? The patch commit message describes wrong expectations, so I'd say patch is wrong. But test changes are harmless and checks what test expects actually. There should be no writes neither to MDT nor to OST because test disables writeback on client to keep data in cache until it is discarded by 'rm'. So in that sense patch is correct, just commit message need to be updated. Could you explain why this changes were added at all?
