(3 comments)

Overall, I think allowing users to set and (more importantly) to get information about a private structure is not the best thing. It constrains the implementation of the changelog api. If the api were to stop using a buffer, I don't think there would be an easy/non-breaking way to reimplement this function.

I think giving hints to the llapi would be a less intrusive method. It would allow applications to say: "I can process more changelogs if you can give me more", or "I am a slow application anyway, no need to go above and beyond for me". Applications would not have strict expectations, and the operations could easily be no-ops if they do not make sense in the future.
