Patch Set 2: Code-Review-1

Artem, the last time I looked at this tool, there was no way that it could work because it depended on ioctls in ldiskfs to create the IAM file, and those ioctls were removed many years ago.  See my comments on patch https://review.whamcloud.com/46049 about this.

Secondly, using this tool to recreate an OI file or FLDB would still require a manual effort from a knowledgeable user/support person.

My preference would be to be able to rebuild the FLDB file using "lctl set_param", as is (mostly) done by patch https://review.whamcloud.com/46048 "LU-15437 fld: extend procfs to allow fldb changes".  Ideally, each MDT and OST would keep a full copy of the FLDB file locally, and then it would be possible to just copy it from another target (maybe manually at first, then automatically via RPCs from any MDT later).
