--- Commit Message ---
LU-2622 obdclass: Use per CPU cl_env cache lists

Instead of serializing access to the cl_env cache structure using a
single global lock, this patch attempts to create per CPU caches for the
cl_env structures. This allows a thread to obtain a cl_env from it's CPU
cache with virtually no locking overhead (i.e. spin_lock/spin_unlock on
each per CPU lock will never wait).
