LU-8865 tests: add fs_test test

Patch adds parallel-scale fs_test test.

Formerly it is MPI-IO test and it provides I/O performance
results for Effective bandwidths. We used fs_test to generate
a maximum IO write and read bandwidth scenario where a lot of
data is written to amortize away the overhead of open/sync/close
operations.

> It would be useful to add some description of what this test is doing and why it is useful to include besides other tests already running.
Andreas, our customers run fs_test. We add it to lustre/tests to have a flexible way for issues reproduction on the automated tests system and avoid the manual efforts.

> What I mean is that "skip_env" is too strong a check, since it will cause parallel-scale.sh to fail in our testing if the fs_test.x command is not installed

skip_env () has been added to track that the correct tests env. were provided. It seems that Intel runs the tests with FAIL_ON_SKIP_ENV=false.
Currently we have test_mib skipped if MIB not found :
$ grep skip_env functions.sh | grep "not found"
        { skip_env "compilebench not found" && return; }
        { skip_env "metabench not found" && return; }
        { skip_env "simul not found" && return; }
        { skip_env "mdtest not found" && return; }
        { skip_env "connectathon dir not found" && return; }
        { skip_env "IOR not found" && return; }
        { skip_env "MIB not found" && return; }
        { skip_env "cascading_rw not found" && return; }
        skip_env "write_append_truncate not found"
        { skip_env "write_disjoint not found" && return; }
        { skip_env "mdsrate not found" && return; }
And mallo shows that test_mib is always SKIPPED (not failed) :
https://testing.hpdd.intel.com/sub_tests/query?utf8=✓&test_set%5Btest_set_script_id%5D=b10ed7ea-55b4-11e0-bb3d-52540025f9af&sub_test%5Bsub_test_script_id%5D=af769ea8-fab5-11e0-bbc0-52540025f9af&query_bugs=&warn%5Bnotice%5D=true&hosts=&test_session%5Buser_id%5D=&test_session%5Bquery_recent_period%5D=&test_session%5Bstart_date%5D=&test_session%5Bend_date%5D=&test_node%5Bos_type_id%5D=&test_node%5Bdistribution_type_id%5D=&test_node%5Barchitecture_type_id%5D=&test_node%5Bfile_system_type_id%5D=&test_node%5Blustre_branch_id%5D=&test_node_network%5Bnetwork_type_id%5D=&commit=Update+results
