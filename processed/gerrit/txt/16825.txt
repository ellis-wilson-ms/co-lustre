--- Commit Message ---
LU-5041 ofd: precreate to use LOC_F_NEW

the idea is to skip OI lookup at object create declaration,
instead issue prefetch request to ZFS. so having few creates
declared we'd get few I/Os in fligh. this should improve
creation rate as the precreation thread doesn't need to lookup
OI mappings one by one issuing syncrhnous I/Os.

OSDs should be able to deal with the case when an object
exists, initialize the object using IO and return -EEXIST.

the patch isn't complete yet.
