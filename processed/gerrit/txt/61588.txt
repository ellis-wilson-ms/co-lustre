LU-14590 obdclass: rename duplicate health_check param

Rename the duplicate /sys/kernel/debug/lustre/health_check file
to /sys/kernel/debug/lustre/health_check_devices, since running
"lctl get_param health_check" is now printing duplicate entries
when fetching the "health_check" parameter.

The debugfs "health_check" file was printing which device was
unhealthy, but only 'healthy' or 'LBUG' or 'NOT HEALTHY' is used
by scripts/tests I'm aware of.  The unhealthy device message is
printed to the console/syslog for reference.

The new 'health_check_devices' parameter will print the specific
device names that have encountered problems.
