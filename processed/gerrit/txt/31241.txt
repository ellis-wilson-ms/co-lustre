--- Commit Message ---
LU-10615 osd: stop OI scrub before FLDB closed

OI scrub may check FLDB when scans the device. During umount
the device, we need to stop OI scrub before closing the FLDB
to void invalid RAM accessing.

Some code optimization and cleanup.

--- Message (Fan Yong, 2018-02-22 02:12:28.000000000) ---
Patch Set 1:

> I think we should be able to run OSD w/o FLDB for number of reasons. so I'd suggest to detect missing OSD instead.

Currently, OI scrub can work even if FLDB is NOT opened. But it does means the OI scrub will not access FLDB if it is accessible. The issue for this ticket is that when the umount closing the FLDB, the OI scrub may be in-using FLDB by race. Because lacking of concurrent control,  the OI scrub will access invalid RAM that is just freed by umount.

On the other hand, closing FLDB is the event when umount the device, it is reasonable to stop OI scrub when umount.
