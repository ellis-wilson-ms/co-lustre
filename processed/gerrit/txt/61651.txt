LU-0000 lod: trivial EC cstripe/dstripe support

Add basic support for storing and retrieving EC stripe
counts (dstripe_count and cstripe_count) without using
them for actual EC functionality yet.

Server-side changes (lod/):
- Add llc_dstripe_count and llc_cstripe_count fields to
  lod_layout_component struct
- Save EC parameters when packing layouts in
  lod_generate_lovea()
- Load EC parameters when unpacking layouts in
  lod_parse_striping()
- Preserve EC parameters in lod_declare_layout_add(),
  lod_get_default_lov_striping(),
  lod_use_defined_striping(), and lod_qos_parse_config()
- Add CDEBUG statements to track EC parameter flow

Client-side changes:
- No changes needed (already present from previous
  commits)

API changes:
- Add VERBOSE_EC_COUNT flag to lustreapi.h for
  displaying EC stripe counts
- Update VERBOSE_DEFAULT to include VERBOSE_EC_COUNT
- Add EC count display in lov_dump_comp_v1_entry() in
  liblustreapi.c
- Save EC parameters when packing layouts in
  llapi_layout_to_lum()

Test changes:
- Add test55 to llapi_layout_test.c to verify EC stripe
  counts are correctly stored and retrieved
- Fix debug format in pack_generic.c from %#x to %u for
  EC stripe counts

This patch enables correct storage and retrieval of EC
stripe counts, laying the groundwork for full EC
implementation.
