LU-13791 sec: enable FS capabilities

FS capabilities are not effective because they are dropped for
non-root users for historical reason: they are used to be enforced
before operations, but now they are checked in MDD layer only (see
mdd_fix_attr()).

Add sanity-sec 51.

Patch Set 2: Code-Review-1

(5 comments)

IIUC, this patch completely removes the code that is dropping FS capabilities for non-root users. As I understand it should be safe because it is handled in the MDD layer (commit comment should mention which functions in the mdd layer take care of this), I realize there are no tests in the whole test suite that exercise FS capabilities, apart from the one you are introducing with this patch that just tests CAP_CHOWN and CAP_FOWNER.

So I think we are missing tests in the test suite to make sure that now that we do not drop FS capabilities for non-root users anymore, the MDD layer handles them all properly.

This patch looks globally fine, but -1 until this point regarding test coverage is clarified.
