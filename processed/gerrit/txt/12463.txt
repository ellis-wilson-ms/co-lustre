--- Message (Andreas Dilger, 2014-11-10 01:43:48.000000000) ---
Patch Set 1:

Matt,
See my comments in LU-5822 about using OBD_FAIL_CHECK() to inject an error into the statfs code path. This would be in the ldiskfs osd_statfs() code that is being modified by this patch. 

To trigger this code, a sanity.sh test would inject the failure into the kernel via:

        do_facet ost1 "lctl set_param fail_loc=0xnnn fail_val=mmm"

Where 0xnnnn is the hex value of the newly added OBD_FAIL_OSD_READONLY value, and mmm is the OST index that should see this problem. Then, the test should verify that the OST is read only by checking health_check on the OST again.
