--- Commit Message ---
LU-12187 lod: initialize EC files with RDONLY FLR state

Erasure coded (EC) files should follow the same FLR
(File Level Redundancy) lifecycle as mirrored files.
Parity components in EC files are conceptually
identical to mirror copies - they're redundant data
that can become stale and need resynchronization.

This patch implements Phase 1: Initialize EC files
with LCM_FL_RDONLY state instead of LCM_FL_NONE, so
they enter the FLR state machine from the beginning.

Changes:
- Add ldo_ec_count field to track parity component
  count
- Add lod_comp_parity() helper to identify parity
  components
- Count parity components during layout parsing in
  lod_qos_parse_config() and lod_use_defined_striping()
- Set EC files to RDONLY state when loading from disk
  if they have parity components
- Write FLR state to disk for EC files in
  lod_generate_and_set_lovea()
- Allow EC files to have FLR state without multiple
  mirrors in lov_init_composite()
- Keep EC files in RDONLY state after resync in
  mdd_changelog_data_store_by_fid()
- Add EC FLR tests to sanity-ec.sh (test_5a through
  test_5i) to verify EC file creation, initial RDONLY
  state, write intent handling, and basic operations

Test-Parameters: trivial
