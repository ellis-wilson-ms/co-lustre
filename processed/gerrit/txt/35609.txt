Hmm, so some digging and fiddling showed me that we do indeed require that all pages in an extent be adjacent...  So, write 4K, seek 4K, write 4K, creates two extents.  It also creates two frags.

So this means each extent is generally only going to be one frag...  And if extents are adjacent, they're merged, unless they're too big, in which case there will be very few because the maximum extent size is = max_pages_per_rpc.

All of that is to say that basically the existing max_extents code should limit the frags exactly like we want, in this case to 256 per RPC.

I realized that after I'd completed modifying the frags checking code and noticed it's now, well, the same as the max_extents stuff.

So...  Perhaps 256 is just too high there...?
