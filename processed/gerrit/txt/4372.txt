Patch Set 1: Looks good to me, but someone else must approve

(1 inline comment)

I'm fine with either this or my patch which uses modprobe from mount.lustre.

My one concern about using cfs_request_module() from within the kernel is the deadlock reported in LU-1279 that makes kernel-driven module loading slow if there are multiple concurrent mounts.  The hope is that by doing this in userspace is that it avoids the "load a module while another module is being loaded" problem seen there.
