Arshad, *my* request was to keep the compatibility for the old function name for at least several tags, if not a longer time, otherwise this breaks every other patch for very little benefit IMHO. Also, given the size of this patch it will always have a large number of conflicts. 

The main objection AFAICS is to apply a time-limited compat macro before all of the code has been converted, so I propose the following:
- first patch renames function and adds unlimited compat macro and spelling.txt to start warnings
- several independent patches to do conversions of existing code
- final patch to be applied later to remove compat macro and fix any remaining users
