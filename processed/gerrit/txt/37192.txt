--- Commit Message ---
LU-12133 osd-zfs: set blocksize to 8K for llog objects

with ZFS-0.8+ default blocksize is 512 bytes. as many llog
operations use 8K chunks it turns into 16 dbuf lookups
which is quite expensive.

for example, sanity/60a takes 104s with blocksize=512 and
90s with blocksize=8K

Patch Set 3:

> Setting the block size to 8k to match to llog size makes good
 > sense.  But I'm a little confused as to why it would have been 512b
 > previously?  I'd have expected these objects to use the default
 > maximum record size.  Or where they specially created elsewhere

It could be that the llog files are being written sparsely (header block, then first log block is discontiguous? Other than that, I'm not sure why the blocksize would be 512 either.  We definitely don't want it to be 1MB, since the header block is updated repeatedly to show which records are in use, and a 1MB write for a few bytes delta is a lot of inflation.
