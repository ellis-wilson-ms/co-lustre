(1 comment)

Hi Doug, a issue of this patch is, it is very likely to end up with two connections  if there is connection race:
* winner will accept incoming connection request on the second race
* winner's outgoing connection request will be accept by the remote side unconditionally.

Although it is always possible to have multiple connections for the same peer, but without this patch, the window of it is quite small and very unlikely. With this patch, it could be quite possible and consume more memory.

Do you get a chance to check this patch? http://review.whamcloud.com/#/c/17876 
It provides another solution for this issue, idea behind is the same, but it needs negotiation between both ends of connection.
