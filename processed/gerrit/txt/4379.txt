LU-2221 ptlrpc: kerberos support for kernel>=2.6.24

Since kernel 2.6.24 the scatterlist struct has no field "page".
Then related Lustre kerberos code cannot work anymore.

So do not access scatterlist::page directly, instead, use the
scatterlist functions sg_set_page and sg_assign_page.

--- Message (Andreas Dilger, 2012-10-25 01:05:49.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

Hi Thomas,
thanks for your patch.

In the Lustre/kernel coding style the use of #ifdef in the code is avoided, because it can make the code difficult to read.

Secondly, checking the kernel version to decide if a particular API is available is not robust, because vendors backport kernel APIs from newer kernels frequently, and just depending on the kernel version number can cause compilation failures for a variety of reasons.

Instead, please look at http://review.whamcloud.com/3709 to see how sg_init_table() is defined after running a configure check to determine if that function is available, and then the code uses either the "real" function or the replacement macro, but both look the same so the code is not messy.  Also note that a configure check for sg_set_page() was already added to the configure script, so this could just be used in the Kerberos code.

Finally, the 2.3.0 release was made just a few days ago, so it would be better to submit this patch to the "master" branch so that it can be included for the next release.
