--- Commit Message ---
LU-18664 target: set proper kthread state

The function distrubte_txn_commit_batchid_update() is called in
an atomic context from distribute_txn_commit() while it is in an
idle state. As is with other cases in this code, set kthread to
TASK_RUNNING before calling distrubte_txn_commit_bacthid_update().
This should fix the might_sleep() issues that are reported.
