--- Commit Message ---
LU-7140 llite: too few 4 MiB RPCs from readahead

Currently, readahead step is set to 1 MiB.

In a setup with 4 MiB multibulk BRW RPCs enabled,
sequential reads generate a mix of 1 MiB and 2 MiB RPCs.
First class RPCs result from the readahead window being
moved by 1 MiB. Second class RPCs result from the readahead
window being moved by 1 MiB and at the same time expanded
by 1 MiB.

The only exception is RPCs generated in the first read
syscall when the readahead window is expanded to cover
the whole requested chunk - these can be 4 MiB RPCs.

This patch calculates the increase step as the largest
value, between OSCs, of the largest RPC size.

This setting is specifically added for configurations with all OSCs set to max_pages_per_rpc=1024 (i.e. 4 MiB). And even then it requires administrative action in order to apply changes, so nothing will break or change automatically.

If some readahead code rewrite handles this case in the same way, it can simply remove the setting. However, currently Lustre read performance is below values that can be reached since 4 MB BRW introduction (our performance team reported some 24-33% throughput increase with their tests).
