--- Commit Message ---
LU-16730 build: Change block comment handling

This patch changes checkpatch handling for block
comments. Such that the start of comment '/*' and
end of comment '*/' if on separate lines will
throw an warning. Additionally it will not throw
and warning if they are on same comment line as it
used to do earlier.

I think if we changed block comment handling, we should disallow 'net' style block comments in Lustre:
 /* This is a comment
  * that I don't like
  * the style of
  */
Lustre should use only:
 /*
  * This is a comment
  * that I like the style
  * of
  */
I think we should still disallow:
 /* This is a comment
  * that I don't like the
  * style of */
since Linux doesn't like it.

https://www.kernel.org/doc/html/v4.10/process/coding-style.html#commenting
https://wiki.lustre.org/Lustre_Coding_Style_Guidelines#Function_comment_blocks

The present checkpatch.pl handles this what you have mentioned above.
The change in this patch is to _not_ complain on style as below. This 
is disallowed and not liked by Linux, however this style does _save_ a line when we are not
forced to put the terminating comment marker in a new line.

 I think we should still disallow:
  /* This is a comment
   * that I don't like the
   * style of */

On the other hand, the current patch does complain on styles as below to discourage putting terminating marker in new line.

  /* This is a comment
   * that I don't like
   * the style of
   */

and...

  /*
   * This is a comment
   * that I like the style
   * of
   */

> https://www.kernel.org/doc/html/v4.10/process/coding-style.html#commenting
> https://wiki.lustre.org/Lustre_Coding_Style_Guidelines#Function_comment_blocks
