LU-12847 docs: add man pages for data_version handling

Split man page for the lfs-data_version(1) command from lfs(1).
Add a man page to describe the llapi_get_data_version(3) function.
Add long options to the "lfs data_version" command while updating
its man page, since the short option meaningss were not very clear.

related and was missing its own man page.

Add a proper example for llapi_fsync(3), instead of referencing lfs.c,
since it is implemented via llapi_get_data_version() internally.

## Fix Explanation

**The Problem:**
The compile error occurred on line 11917 in `lustre/utils/lfs.c`. The error message indicated "request for member 'has_arg' in something not a structure or union", which is a typical C syntax error when struct member initialization is malformed.

**The Root Cause:**
A missing comma (`,`) after `"write-lock"` in the struct option initialization. The line read:
```c
{ .val = 'w',	.name = "write-lock"	.has_arg = no_argument },
```

Without the comma between `.name = "write-lock"` and `.has_arg = no_argument`, the C compiler couldn't parse this as two separate member initializations and instead tried to interpret it as a single malformed expression.

**The Fix:**
Added the missing comma after `"write-lock"`:
```c
{ .val = 'w',	.name = "write-lock",	.has_arg = no_argument },
```

This is a simple typo that was introduced in the patch when adding long options to the `lfs_data_version` function. All other entries in the `long_opts` array have commas separating their member initializations, but this one was missing it.
