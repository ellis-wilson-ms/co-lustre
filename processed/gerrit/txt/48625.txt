--- Commit Message ---
LU-14361 statahead: add statahead advise IOCTL

This patch reuse ioctl(LL_IOC_LADVISE2) for statahead advise.
This allows userspace programs to advise the kernel statahead
of the order that they will be traversing a directory, so that
the client can prefetch inode attributes from the MDT, similar
to what posix_fadvise(POSIX_FADV_SEQUENTIAL) does for file data.

After patched mdtest via adding this statahead IOCTL hint, it
can support mdtest benchmark with regularized file naming format:
	mdtest.$rank.$i
The usage of this statahead advise IOCTL could be as follows:
	open(dir);
	ioctl(dir_fd, IOC_LADVISE2, ...);
	stat mdtest.0.0;
	stat mdtest.0.1;
	stat mdtest.0.2;
	stat mdtest.0.3;
	...
	clsoedir(dir);

--- Message (Qian Yingjin, 2022-09-23 08:47:55.000000000) ---
Patch Set 1:

Lustre has readahead, but it also has implemented fadvise (DONTNEED, WILLREAD) for more precise read-ahead control from the user hint.
This patch can be also seemed as a precise stat-ahead from user hint via IOCTL.

Detecting the access pattern for stat() calls is desirable, but it is much more complex to implement, especially for access mode like mutest shared dir stat() workload (stat mdtest.$rank.$I, each rank is a separate process to do stat() call series with regularized naming format.).
Moreover, it is hard to predict how many entries should be stat-aheaded, and when to stop the kernel stat ahead thread...
