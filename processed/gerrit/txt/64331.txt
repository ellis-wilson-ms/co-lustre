LU-19536 obdclass: fix double-free in DIO buffer

When obd_pool_get_pages_array() fails during
ll_allocate_dio_buffer(), the error path calls
ll_free_dio_buffer() to clean up the pages array.
However, cl_sub_dio_end() also calls
ll_free_dio_buffer() when the sub-DIO completes,
causing a double-free of cdp->cdp_pages.

With kmalloc (kvzalloc), this corrupts the SLUB
free list and triggers 'kernel BUG at mm/slub.c'
in set_freepointer.  This is reliably reproduced
by large unaligned DIO writes that exhaust the OBD
page pool.

Make ll_free_dio_buffer() null-safe: check for
NULL cdp_pages on entry and set it to NULL after
freeing, following the kfree/vfree convention.
This eliminates all caller-side NULL checks.

Add OBD_FAIL_LLITE_DIO_BUFFER_ALLOC (0x1438) to
inject allocation failure and test_398t to verify
the cleanup path doesn't crash.
