LU-16035 kfilnd: Fix setting of transaction buffer

The kfabric data transfer APIs do not allow for an offset to be
applied to buffer. Instead, the kfabric API expects the buffer to have
any user defined offset factored into it. Kfilnd was not taking this
into account. When LNet was passing kfilnd an IOV, this IOV was being
passed into the kfabric API without taking into consider the offset
and length of the IOV. This resulted in kfabric RMA operations with an
incorrect size.

To resolve this issue, kfilnd_tn_set_buf() is defined. This function
will take the LNet IOV, apply the LNet offset and length, and store
the resulting IOV within the transaction structure. This new IOV will
be used for kfabric memory registration and data transfers.
