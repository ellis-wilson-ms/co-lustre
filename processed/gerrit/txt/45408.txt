LU-15177 ldlm: do not check patch version

Do not do patch version check for MDT-MDT interoperability.
The check of abs(patch - LUSTRE_PATCH) > 3 looks absolutely
arbitrary and does not allow a live update across patch version
change > 3.

 > nothing in the protocol that says it cannot work.  Let's allow it. 
 > That said, major.minor should be ok too, but this should be enough.

On the flip side, there is absolutely never interop testing between different versions installed on different MDTs, and this is rarely considered an important factor for patch implementation as a result.

The "patch < 3" check is intended to allow very minor updates that are unlikely to affect the protocol (e.g. some bugfix for an issue only affecting one MDT at a time), but I personally wouldn't want to be running a system with 2.12.0 and 2.12.7 MDTs in the same cluster just because of the wide range of unknowns.

You are of course free to change this version check in any way you want, but I think for the general community users it is safest to only allow MDS versions that are exactly the same to be running at one time.

Even so, I'm a bit confused about your process that would upgrade only a single MDS in a cluster with such a large version difference?
