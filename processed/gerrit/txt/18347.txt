> Is this the best approach to this problem?

It is a reasonable approach, in my opinion. What I like about it is that it doesn't require setting the value separately for each interface. (Being able to set these kinds of values per interface is a very nice feature to have. Being required to do so is not.)

One concern I have is that divide-by-two can give you values that are almost half of the highest reasonable value. Would a more gradual decrease be feasible? (For example: if the original is > 2k, remove 1k and try again, if the original > 256, remove 128 and try again, if the original > 16, remove 8 and try again.)
