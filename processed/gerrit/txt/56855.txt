LU-18033 llite: enable hybrid IO by default

Enable hybrid IO by default.  This is now passing all
tests.

Includes three small test fixes, one for sanityn 108a:
The purpose of test 108a is to confirm for async IO
(buffered writes) the write does not need to go to disk
for the file size to be updated, but for hybrid/DIO the
write does not complete until it goes to disk.

We disable hybrid here so this definitely tests buffered
IO.

One for replay-single test 89:
We need to ensure we have the dlmlock from the OST before
we take the OST offline, so add an ls -la.  Previously the
write took care of this, but hybrid IO does not use locks
when in DIO mode.

And one for sanity test 806:
ZFS does not allocate blocks immediately after a write, so
hybrid IO/direct IO cannot reliably give correct LSOM
blocks.  Disable the blocks test for ZFS and HIO.

Test-Parameters: fstype=zfs testlist=sanity

Andreas,

Question for you - If ZFS isn't assigning blocks until a tx commit, then why does buffered IO block counting seem to work?  Perhaps some difference between O_SYNC and hybrid/O_DIRECT (confirmed O_DIRECT fails, fyi) and maybe we're losing something in switching to hybrid...

--- Message (Andreas Dilger, 2025-05-09 02:02:59.000000000) ---
Patch Set 18:

> Andreas,
 > 
 > Question for you - If ZFS isn't assigning blocks until a tx commit,
 > then why does buffered IO block counting seem to work?  Perhaps
 > some difference between O_SYNC and hybrid/O_DIRECT (confirmed
 > O_DIRECT fails, fyi) and maybe we're losing something in switching
 > to hybrid...

The RPCs continually return a stream of size/blocks to the client in the write reply, and AFAIR ZFS is committing the TXG every second or so (or if it gets too large).  For larger files or busy servers the blocks counts should be updated within a second or so of writing.

It is also possible that we are estimating block counts on the client at some point, to avoid "stat()" returning blocks=0 to userspace which will then think the file is totally sparse and not try to read it.
