--- Commit Message ---
LU-12668 ec: try map ec mirror to data mirror during creation

When creating a layout and allocating OSTs for the
different components, if creating a parity component
try to find the matching data component so we know how
many stripes are in the data component.
We need this in order to compute how many raidsets
there will be so that we can request to allocate one
OST per parity per raidset for the parity component.

Also fix parity EOF check in vvp_prep_size() for
buffered I/O. For parity mirrors, the logical EOF
is smaller than the data mirror size, so we must
check against ci_parity_eof before checking against
kms. Without this fix, buffered reads from parity
mirrors could read past the parity EOF when kms was
still valid and larger than the read position.
