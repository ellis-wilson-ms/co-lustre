operations to use the new cfs_get_paths() function which

Have been testing this patch with the upstream client and it found a bug. force_lbug hangs when running lctl get_param and writing to one of the files in /sys/kernel/debug/lnet causes the lustre client to crash hard. Sanity test 401 fails due to duplicate directory in the upstream client. Both /sys/kernel/lustre and /sys/fs/lustre contain almost the same exact trees. Because of this the find command will repot some directories twice. So the find results need to stripped of $proc_dir at the start of the returned strings and then only the unique values need to be reported. So much more bash magic is still needed.
