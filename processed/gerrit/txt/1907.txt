If we are going to start building iokit RPMs the implication is that they will get distributed eventually, either officially or unofficially.

To that end, I would suggest that the Release: value remain untouched by us.  It's typically for downstream packagers anyway and should only be incremented when there are changes that only affect the build within a given version (that should be rare for us once this has all been implemented).

I would further suggest that a "bugfix" be added to the Version: and that given the iokit's relative stability that we be religious about incrementing the bugfix or the minor version component of the version for each landing depending on whether it's an enhancement of some sort or a bug fix.

Patch Set 7: I would prefer that you didn't submit this

(1 inline comment)

When fixing (trailing, at least) whitespace errors, it's usually a good idea to just fix the whole file at once instead of just the few lines that have been pointed out within gerrit's context.

If you use vim for example:

:g/[ ^I][ ^I]*$/s///g

will strip the trailing spaces from every line.

Note that the ^I in that expression above is a "tab", not a caret and an `I'. You get that in vim by typing control-v then hitting tab.
