LU-18756 ofd: add sysfs tunables for id repair

This patch adds two new "lctl set_param" commands to (1) enable and
disable resource id repair of OST objects, and (2) to modify the id
repair queue count per ofd device.

Enable/disable id repair is controlled through (1 by default):
lctl set_param obdfilter.*.enable_resource_id_repair={0,1}

The id repair queue count is controlled through (1024 by default):
lctl set_param obdfilter.*.resource_id_repair_queue_count=N

The soft limit of the queue size is set at "65536". If the queue size
is exceeded a potential OST object is skipped and attempted to be
repaired upon the next access.

ID repair is not yet included into the code path.
