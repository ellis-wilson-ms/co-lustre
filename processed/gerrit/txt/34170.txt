LU-9859 libcfs: replace cfs_srand() calls with add_device_randomness().

The only places that cfs_srand is called, the random bits are
mixed with bits from get_random_bytes().  So it is equally effective
to add entropy to either pool.
So we can replace calls to cfs_srand() with calls that add the
entropy with add_device_randomness().  That function adds time-based
entropy, so we can discard the ktime_get_ts64 calls.

One location in lustre_handles.c only adds time based
entropy. This cannot improve the entropy provided by
get_random_bytes(), so just discard that call.

> Ouch. This crashed my nodes.

That is why typecasting should be avoided if at all possible.  You were casting the NID into an address, which is almost certainly bad.  

If you'd correctly used "&ni->ni_nid" without the typecast, GCC would know that this is a pointer that is being passed to a "void *" pointer argument and all would be well.

If you incorrectly used "ni->ni_nid" without the typecast, GCC would see you are passing an integer type to a "void *" pointer argument and  it would rightfully complain.

Seeing as the typecast hid a node-crashing bug (I didn't see this before) I'll give this -1 and instead prefer that the patch is fixed.
