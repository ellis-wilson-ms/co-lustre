LU-5271 lnet: debug patch for o2iblnd

If ib_poll_cq returned +ve even without initialising ib_wc,
then o2iblnd will run into unpredictable situation because
o2iblnd unconditionally trust ib_wc::wr_id as user defined data
which should always be pointer of tx/rx.

This patch can verify this conjecture. If this is true, the LASSERTF
should be changed to CWARN.
