--- Commit Message ---
LU-16106 lnet: allow direct messages regardless of peer NI status

If check_routers_before_use is enabled, the router needs to
be pinged before it is used, which is not possible because
its NIs are assumed to be down at start-up. Don't prevent
discovery of the router in this case.

This change allows non-routed traffic to peer NIs with "down"
status.

> (1 comment)

If we come to this point in the function, it means our node is a router. If check_routers_before_use is enabled, we do want to consider the peer NI of the router to be alive if we are checking it - so we can ping it to proactively check on it. 
We can probably assume that the message our router initiates to the other router is to check if it is alive. So how about the following:
        
if (!list_empty(&lpni->lpni_peer_net->lpn_peer->lp_routes) &&
    !msg->msg_routing &&
    lpni->lpni_ns_status == LNET_NI_STATUS_DOWN &&
    lnet_peers_start_down())
           return true;

This narrows it down so that non-router nodes are not affected.
