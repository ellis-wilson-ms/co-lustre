--- Message (James Simmons, 2015-06-10 22:26:47.000000000) ---
Patch Set 7: Code-Review-1

It is a fine patch in itself but it is just too big. I tried it but it collided with the DNE2 work. Plus if it lands and then later external patches that have OBD_FREE_LARGE land after it will cause bad breakage.

Here is what I recommend. Break this patch up into smaller patches. The core root patch will have the OBD_FREE_LARGE and OBD_ALLOC_* changes. Make OBD_FREE_LARGE a alias to OBD_FREE to avoid breakage for future patches landing. In that patch also add to checkpatch.pl a  check for OBD_FREE_LARGE so when people refresh their patches HPDD will alter them to not use this function. Using the core patch as a base then submit a bunch of smaller patches
for various modular components i.e ptlrpc, lov etc. Some will collide with other things but you have a greater chance of landing bits and pieces. Once done we can then remove the OBD_FREE_LARGE macro.

--- Message (Andreas Dilger, 2015-06-11 02:02:09.000000000) ---
Patch Set 7:

I was going to say the same thing as James. Leave the old macros in place and add checks to checkpatch.pl (which will warn about new code using these functions, but will not automatically replace them) and then submit the last patch that removes the old macros. We can land the last patch when they are no more users in the tree and when people have a chance to rebase their in-progress patches.
