LU-18555 lnet: Track successful and failed resends

Add a count of successful and failed resends to the LNet health
stats. This will allow us to more easily determine the effectiveness
of this feature.

In the current code, lnet_incr_hstats() only uses cpt 0. Callers can
now pass a cpt argument (they must hold net lock for that cpt), and
lnet_incr_hstats() will update the corresponding per-cpt counters. In
lnet_health_check() we will now lock either the msg_rx_cpt or
msg_tx_cpt in the same manner as lnet_finalize(), and
lnet_incr_hstats() will update the successful resend count.

On the message resend path, the msg_tx_cpt was already being taken so
we use this to update the failed resends counter. Failed resends is
updated there rather than in lnet_incr_hstats() to avoid duplicating
the check for whether the message retry count has exceeded the max
retries.

Modify sanity-lnet health tests to check that failed_resends is
incremented correctly.

Add new test to check that successful_resends is incremented
correctly.

Modify the behavior of lnet_drop_rule_add() when the specified rate
is 2 so that the first matched message is always dropped. This allows
the new test to work correctly.

cleanup_health_test() would overwrite the return value from
lnetctl net del. This is fixed.

Some style fixes were made in functions touched by this patch.
