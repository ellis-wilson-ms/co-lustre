LU-3302 llog: Do not use ostid swab for llogid

Since logid still use id/seq format in the request,
it will be swabbed by its own swab func, instead of
using ostid swab, which might see logid as FID incorrectly.

There are still some spots where ostid_id() is being applied to lgl_oi, and similar combinations. Have you checked those? If I understand this code correctly it may be useful to add some assertions/trace that:
