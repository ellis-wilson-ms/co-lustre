--- Message (Timothy Day, 2024-10-03 19:13:06.000000000) ---
Patch Set 4: Code-Review-1

I don't really understand the logic here.

The original compat patch https://review.whamcloud.com/c/fs/lustre-release/+/51126 implements OBD_CONNECT2_UNALIGNED_DIO. In the current implementation, if the server reports this flag the client will send it unaligned DIO. For old unpatched ZFS servers (that don't report OBD_CONNECT2_UNALIGNED_DIO), the client will not send unaligned DIO. But for unpatched ldiskfs servers, the client will ignore OBD_CONNECT2_UNALIGNED_DIO and send unaligned DIO regardless.

Now we're adding an explicit version check for 2.14. This seems to nullify all of the ZFS check, which means old servers will never get unaligned DIO. But isn't that the point of OBD_CONNECT2_UNALIGNED_DIO? Why not remove all of the complex logic and just respect OBD_CONNECT2_UNALIGNED_DIO?
