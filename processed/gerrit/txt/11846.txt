--- Commit Message ---
LU-5519 lfsck: repair bad name hash for striped directory

If the name hash of some name entry under the striped directory
does not match the shard of the striped directory, then the LFSCK
will repair the inconsistency. Ideally, the LFSCK should migrate
the name entry from the current MDT to the right MDT (another one),
but before the async commit finished, the LFSCK will change the
striped directory's hash type as LMV_HASH_TYPE_UNKNOWN and mark
the lmv flags as LMV_HASH_FLAG_BAD_TYPE.

--- Message (John Hammond, 2014-09-10 15:28:47.000000000) ---
Patch Set 1: Code-Review-1

This series contains a few instances of the following:

 
char *name2 = lfsck_env_info(env)->lti_tmpbuf2;
 
rc = snprintf(name2, LFSCK_TMPBUF_LEN - 1, DFID":%u",
                      PFID(fid), index);
name2[rc] = '\0';

There are a few things wrong here:

1) Please sizeof(lfsck_env_info(env)->lti_tmpbuf2), ...) rather than LFSCK_TMPBUF_LEN - 1.

2) There is no need for the -1. Pass the size of the buffer to snprintf().

3) If the buffer size is positive then snprintf() will add a terminal '\0' regardless of whether or not the string fit in the buffer.

4) snpritnf() returns the number of characters that *would* have been written if the buffer was large enough. So if the buffer was too small then setting name2[rc] = '\0' is an out of bounds array access.

Please use rc = snprintf(name2, sizeof(...), ...) and skip the manual '\0' termination.
