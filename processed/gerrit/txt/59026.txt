### 1. Commit Summary

This commit modifies the Lustre nodemap functionality to support identifying different filesets within the nodemap's Index Attribute Manager (IAM) storage. It introduces the concept of a `fileset_id` and modifies functions related to adding, updating, deleting, and processing fileset information in the IAM.

Specifically, it:
*   Adds a `fileset_id` parameter to `nodemap_idx_fileset_add`, `nodemap_idx_fileset_update`, and `nodemap_idx_fileset_del` functions and their internal counterparts.
*   Introduces `nodemap_fileset_get_subid` to calculate the starting subid within the IAM record space based on a `fileset_id`. This reserves a range of subids (`LUSTRE_NODEMAP_FILESET_SUBID_RANGE`) for each fileset.
*   Introduces `nodemap_fileset_get_id` to reverse this calculation, determining the `fileset_id` from a given subid.
*   Refactors `nodemap_cluster_fileset_helper` to use `nodemap_fileset_get_id` and introduces `nodemap_cluster_rec_fileset_fragment` to handle applying a fragment to the correct fileset buffer (currently only the primary fileset, ID 0).
*   Updates call sites in `nodemap_handler.c` and `nodemap_storage.c` to pass the `fileset_id` (currently hardcoded to `0` as only the primary fileset is handled).
*   Updates a comment in the UAPI header `lustre_disk.h`.

The commit message indicates this is preparatory work for handling multiple filesets per nodemap, but this specific change only implements the logic for the primary fileset (ID 0).
