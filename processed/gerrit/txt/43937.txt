I will try, but this is not as simple as removing the capability on server side for squashed root. AFAIU, it works like this: on client side, when the capability is set, a number of flags are set on pages, and these flags are interpreted on server side to bypass quota check. So the idea would be to detect root squash on server side, and remove the pages' flags accordingly. The most tricky part being the OBD_BRW_FROM_GRANT flag, indifferently set on all pages as soon as root is doing IOs.
