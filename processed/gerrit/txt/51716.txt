--- Commit Message ---
LU-16969 build: check that pkg-config is installed

PKG_CHECK_MODULES macro fails in very annoying to debug ways.
Often, this will fail with:

 syntax error near unexpected token `LIBNL3,'
 `	PKG_CHECK_MODULES(LIBNL3, libnl-genl-3.0 >= 3.1)'

and provide no indication that the real error is that
pkg-config is not installed. Adding an explicit check
for pkg-config will make the error more self-evident.

--- Message (Timothy Day, 2023-07-19 18:19:44.000000000) ---
Patch Set 1:

I could add a fixes line. But, the change is so small that it probably not worth back-porting. The functionality is identical. It's a developer QoL fix.

Every couple months I run into this warning, and spend a lot of time debugging before realize pkg-config is missing. The issue can't be fixed using PKG_CHECK_MODULES, since pkg-config provides that macro. If you don't have it, you get arcane errors when autotools explodes.

This post (https://stackoverflow.com/questions/10220946/pkg-check-modules-considered-harmful) helped me understand why this strange behavior happens. I'm open to other ideas, as long as configure outputs sane errors.
