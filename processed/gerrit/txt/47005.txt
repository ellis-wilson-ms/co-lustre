LU-15724 osp: wakeup all precreate threads

Number of threads could sleep at osp_precreate_reserve() and
wait objects from OST. When MDT stops Lustre should wakeup
all threads. When opd_pre_recovering is set any wakeup of
opd_pre_user_waitq is useless. Failover of MDT does not produce
disconnect event, only inactive, so osp_precreate_cleanup_orphans()
can not be awakened.

--- Message (Alexander Boyko, 2022-04-18 08:24:20.000000000) ---
Patch Set 4:

It is not only for MDT stopping, for regular processing too. But fix included a specific part for MDT stopping - disconnect event, it was broken. A threads wakeup failure is visible and reproducible by test with MDT stopping. During MDT umount, it could not wakeup osp_precreate_cleanup_orphans thread(disconnect event was missing). Orphan thread could not wakeup osp_precreate_reserve because sleeping was based on opd_prerecovering flag. For general processing of precreated objects only single precreate thread was waked up, but sleeping queue could have a numbers of threads.
