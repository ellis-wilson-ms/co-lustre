LU-744 osc: add lru pages management - new RPC

Add a cache management at osc layer, this way we can control how much
memory can be used to cache lustre pages and avoid complex solution
as what we did in b1_8.

Right now, admins can set how much memory an OSC can cache lustre
pages. This is not perfect because an application may only access
limited number of OSCs in a burst read. A self-adapative algorithm
will be worked out to solve this issue.
