LU-2264 test: Create memory load for testing

It can be useful to add extra memory pressure for testing.
Client nodes are likely to use extra resources.

rewrite memhog.c to include an automatic mode. ./memhog -a will
consume all but 160-128mb of Ram and Swap. The mode consumes
then exits at this time. Original behavior is maintained with
the exception of moving the chunk size to 32mb (down from 128m)

--- Message (Keith Mannthey, 2012-11-21 21:00:41.000000000) ---
Patch Set 1:

I looked at memhog a little and it didn't seem ideal.  Right now is seem to be a "Tell me how much memory you want me to eat" program.  The issue with that is as you approach but not cross over the OOM threshold in the time it takes to allocate the memory the Target amount can change and you can wind up crossing into OOM.  Putting a box into OOM is not really useful,  making allocation requests block a bit more than normal is.   Exhausting swap can take a while and the system is live.  

The basic code for both of these could can be easily factored,  Perhaps making memhog to have a safe eat everything mode would be good. 

"memhog -a" for automatic perhaps. How about memhog -a runs an endless allocation/deallocation loop. ctrl -c kill it?
