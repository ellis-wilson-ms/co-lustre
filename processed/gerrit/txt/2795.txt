LU-1411 obdclass: compatibility for kernels < 2.6.18

lustre_get_sb() conditionally calls get_sb_nodev() with a struct
lustre_mount_data2 * for kernels 2.6.18 and newer, and a void * data
pointer for older kernels.  However, lustre_fill_super()
unconditionally casts its void *data argument to a struct
lustre_mount_data2 *, which causes a crash when we deference into it
on older kernels.  To fix this, add conditional code to
lustre_fill_super() to support older kernels.

> is LLNL still using clients with kernels < 2.6.18?

Yes, our BG/P system runs 2.6.16 on the IO nodes.  We decided to start testing the 2.1 client on BG/P mainly due to LU-1378 and our desire to discontinue local maintenance of our 1.8 branch.  I realize that continued support for these kernels is not something we can expect in future Lustre releases.  But we'll probably be running and supporting 2.1 at least until we retire our BG/P systems, so landing fixes in this branch is much less burdensome for us.
