LU-0000 flr: implement immediate mirror writes for FLR files

This patch implements immediate mirror write functionality for
Lustre File Level Replication (FLR) files. When a file is
created with the --immediate flag, writes are automatically
sent to all mirrors simultaneously instead of just the primary
mirror, providing immediate data protection and eliminating
the need for mirror resync operations.

Key features implemented:

1. Mirror iteration logic in cl_io_loop() that detects immediate
   mirror files and iterates through all mirrors for each write
   operation.

2. Proper iov_iter state management in the VVP layer using
   iov_iter_revert() to reset the iterator state for subsequent
   mirror iterations, ensuring each mirror receives the complete
   write data.

3. Mirror index tracking through ci_write_mirror field that
   directs each iteration to write to a specific mirror.

4. Integration with existing FLR infrastructure and DIO (Direct
   I/O) operations.

The implementation works by:
- Detecting immediate mirror files during write operations
- Determining the number of mirrors from the layout
- Iterating through each mirror with proper state reset
- Using existing LOV layer mirror selection logic
- Ensuring each mirror receives identical write data

This provides significant benefits:
- Immediate data protection across all mirrors
- Improved read performance (all mirrors have current data)
- Simplified mirror management (no manual resync needed)
- Better fault tolerance for critical data

Testing shows successful simultaneous writes to multiple OSTs
with proper data integrity and no performance degradation.
