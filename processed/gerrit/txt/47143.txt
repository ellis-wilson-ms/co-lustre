LU-13106 wbc: add hardlink support

This patch implements the hardlink support for WBC.

To simplify the implementation for the hardlink() operation,
when hardlink a source file with a target file under a directory,
the client will deroot the source file and the target directory
first from the root WBC directory object.
After that, the source file and the target directory are both
normal Lustre files which were no longer charged by WBC memory FS.
And then the client can perform the hardlink as a normal Lustre
hardlink operation (via @ll_link()).

Another design choice may decomplete the parent directory of the
source file and the parent directory of the target file (if any
of them are in a complete state first and then synchronize to the
metadata server to execute the hardlink() operation.
But this design may introduce the complex DLM locking protocol
changes on the server.

Add test case sanity-wbc test_110.
