(1 comment)

There is a statx parameter which reports the DIO alignment requirements - STATX_DIOALIGN and the fields in the struct:

        __u32   stx_dio_mem_align;      /* Memory buffer alignment for direct I/O */
        __u32   stx_dio_offset_align;   /* File offset alignment for direct I/O */
        /* 0xa0 */
        __u64   __spare3[12];   /* Spare space for future expansion */

but I don't see it set anywhere in these patches.  It makes sense to add these fields to our internal "struct statx" definition in lustre_user.h, and check for this and conditionally (or unconditionally?) fill in the fields for applications that are checking this where we fill in struct statx (IOC_MDC_GETFILEINFO_V2 and ll_getattr_dentry):

 #ifdef STATX_DIOALIGN
         stat->stx_dio_mem_align = 1;
         stat->stx_dio_offset_align = 1;
 #endif
