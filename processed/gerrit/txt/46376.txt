LU-15499 wbc: avoid deadlock for parallel buffered write

When benchmark with IOR or mpiFileUtils/dcp, we found the
following deadlock:

memfs_file_write_iter()
{
	...
	down_read(&wbci->wbci_rw_sem); // lock A
	if (wbc_inode_data_caching(wbci)) {
		inode_lock(inode);
		rc = generic_write_checks(iocb, iter);
		if (rc > 0)
			rc = __generic_file_write_iter(...);
		inode_unlock(inode);
		...
	}
	up_read(&wbci->wbci_rw_sem);
	...
}
memfs_write_begin(...)
{
	rc = memfs_write_getpage(inode, index, pagep);
	if (rc == -ENOSPC) {
		...
		up_read(&wbci->wbci_rw_sem);
		rc2 = wbc_make_data_commit(file->f_path.dentry);
		down_read(&wbci->wbci_rw_sem);
		...
	}
	...
}

There is a deadlock due to the order of locking between inode lock
and @wbci->wbci_rw_sem.
This patch avoids this deadlock by keeping the lock order in
@memfs_write_begin().
