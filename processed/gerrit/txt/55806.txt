LU-16897 socklnd: handle sparse file

Hole information present in md is read to build kiov without hole
pages in lnet_extract_kiov. Thus, the pages that are holes are
skipped, and only useful data is sent. To have all data, the client
needs to know which pages are holes.

In that purpose, the server sends hole bitmap header after lnet
header. To achieve that, SOCKNAL_RX_LNET_SPARSE_HEADER state has been
added.

Sparse header is added only if sparse data is detected and, if it is
the case, the server will use an LNET_MSG_PUT_SPARSE message.

When a client receives an LNET_MSG_PUT_SPARSE messages, it will know
there will be a sparse header. Once the sparse header received it will
zero the right pages and build the right kiov to be able to receive
data at the right location.
