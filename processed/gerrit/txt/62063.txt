--- Commit Message ---
LU-19527 lod: prevent parity mirrors from being write targets

Parity mirrors store erasure coding redundancy information and
should never be selected as write targets. They can only be
updated through resync operations from their corresponding data
mirrors.

This patch is the MDS-side guard: it adds checks in LOD mirror
selection logic so the MDS will not pick a parity mirror as the
primary mirror for a write. It is complementary to the client-side
guard in https://review.whamcloud.com/62539 ("LU-19481 lov: skip
parity mirrors for normal read/write IOs"), which prevents the
client from issuing normal read/write I/O to parity mirrors. Both
are needed: the client-side patch blocks user-driven I/O to parity
mirrors, while this patch prevents the MDS from accidentally
choosing a parity mirror as a write target (e.g. as a fallback
when no prefer flags apply, since prefer flags are blocked on
parity mirrors).

This patch adds:
- Check in lod_primary_pick() to skip parity mirrors when
  selecting a primary mirror for writes
- Defensive checks in lod_declare_update_write_pending() to error
  out if a non-stale parity mirror is found
