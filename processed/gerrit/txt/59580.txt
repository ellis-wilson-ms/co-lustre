LU-0000 llite: acquire concurrent write lease for setattr

For setattr operations on mirrored files that modify OST
attributes (size, timestamps), acquire concurrent write
leases to ensure proper FLR state management and mirror
synchronization.

This extends the existing concurrent write lease mechanism
to cover setattr operations that communicate with OSTs,
specifically:
- ATTR_SIZE (truncate operations)
- ATTR_ATIME, ATTR_MTIME, ATTR_CTIME (timestamp updates)

The implementation:
- Detects mirrored files using cl_object_layout_get()
- Correctly handles lcm_mirror_count encoding (actual
  mirrors minus 1)
- Only acquires leases for OST-bound operations
- Provides graceful degradation if lease acquisition fails
- Integrates with existing server-side FLR state transitions

This ensures that setattr operations on mirrored files
trigger the same server-side state management as write
operations, maintaining consistency across all mirrors.
