--- Commit Message ---
LU-17000 obd: check for NULL ptr in OBD_FREE_LARGE

Add a check for NULL ptr in OBD_FREE_LARGE and treat
a NULL ptr as a non-vmalloc address. If a NULL ptr is
considered a vmalloc address, this could lead to a
NULL ptr dereference.

CoverityID: 440591 ("Dereference after null check")

Test-Parameters: trivial

Patch Set 3: Code-Review-1

I think this makes no sense.

is_vmalloc_address cannot return true for NULL because the check is addr >= VMALLOC_START so if you set it to 0, ALL addresses would be vmalloced

moreover of course both vfree and vfree_atomic have hardcoded

 if (!addr)
    return;

unconditionally.

As such I propose to just mark this coverity report as invalid and move on.
