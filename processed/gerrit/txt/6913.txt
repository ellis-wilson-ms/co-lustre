Hello JC,

While looking into sanity-hsm.sh, I found that in copytool_setup() only one copytool was started and the host of it (aka agent) was combined with the local Lustre client node.

In LU-2064, the document of HSM-16.lyx described that we need introduce AGTCOUNT, AGTDEV{n}, SINGLEAGT, agt{n} and maybe agt{n}_HOST, etc. to support multiple remote agents. I was assigned the task to add those stuff and more test cases into sanity-hsm.sh to validate multiple copytools serving different archive back-ends or the same archive back-end under $SHARED_DIRECTORY (which is visible by all the agents). 

I'm still learning and have a basic question now:

In a test cluster which has multiple Lustre client nodes, if I started copytools on remote clients (not on the local client), will the "lfs hsm_*" operations on the local client work correctly? In other words, must "lfs hsm_*" operations be run on a node which has copytool started?

Thanks.
