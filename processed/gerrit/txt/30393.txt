LU-10026: WIP: client-side compression (write path)

Data is compressed by the client and decompressed by the server.

This also adds build system changes for compression:
- The configure parameter --enable-compression can be used to enable compression.
- If lz4 is not available in the kernel (< 3.11), we build a private module for it.

A quick thought or two starting out...  This should eventually get a proc switch to enable/disabled it, and I'll let others weigh in, but I would think we'd want to default to building it.  It could be off by default.

Also, given that the kernel version of LZ4 varies a lot, I would personally be happier if we just used our own all the time.  I know that's bad, etc, etc, but a lot of the kernels we currently support have some pretty out of date versions of LZ4, I think.
