This patch makes the assumption that the interfaces in use have not changed, which someone needs to check. In addition, this patch requires that ZFSOnLinux and Lustre be upgraded in lock-step and if the previous assumption is correct, then doing something like the following would avoid that:

+       handle_libzfs = dlopen("libzfs.so.2", RTLD_LAZY);
+       if (handle_libzfs == NULL)
                handle_libzfs = dlopen("libzfs.so.1", RTLD_LAZY);
        if (handle_libzfs == NULL)
                return EINVAL;

By the way, there is an existing issue in jira that I used for a comment, but I will repeat what I said there here:

"ZFSOnLinux imported a change to its internal library API from Illumos, which required a SONAME bump. It would be advisable to evaluate a switch to the new libzfs_core library, which is meant to provide a public library API with a stable interface."

https://jira.hpdd.intel.com/browse/LU-4606#comment-76996
