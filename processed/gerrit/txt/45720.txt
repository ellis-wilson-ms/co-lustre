LU-15274 llite: True whole file readahead

The current whole file readahead code blows up when
faced with a reverse read - it tries to read the file
forward from the point where it's accessed, which for
reverse reads is exactly the wrong thing, missing every
time.

There's a simple fix: Make it start the read from the
beginning of the file.  The code looks like this was
probably the original intention - note the setting of
window_start_idx to 0 - but no one noticed.

NB: Also, repeatedly attempting to read the file forward
with whole file readahead also breaks any other form of
readahead, by continually resetting the state to attempt
full file.  This is understandable - full file takes
precedence - but it's an extra reason to force full file
readahead to start from the beginning.
