LU-7312 test: With set -e exit status in sanity-hsm was not valid

The exit status which was obtained using $? in copytool_cleanup
function wasn't returning a valid value thereby leading to
improper verification and thus exiting from the script. The fix
does the appropriate error handling caused due to set -e and
does the proper cleanup at the end.
