--- Commit Message ---
LU-13411 llog: allow delete of zero size llog

1) all plain logs belonging to catalog should have flag
LLOG_F_ZAP_WHEN_EMPTY base on llog_cat_new_log(). When
llog_cat_process_common processing plain log with zero file size,
this flag is not set during llog_cat_id2handle LLOG_EMPTY, so these
plain llogs are not canceled/destroyed. They appeared during cross
MDT updates. Fix adds flag LLOG_F_ZAP_WHEN_EMPTY for any plain llog
at catalog.

>do you have an idea what sequence(s) lead to holes?
Alex,
well, for now, we have found a r/c for llog corruptions. The reproducer  is a racer.sh at DNE2b environment, stripe directory etc. It is a bit complicated. 3 cross mdt operation with llog modification.
1) object destroy, our includes llog object at invalidate list
2) xattr_set/attr_set doesn't include llog object at invalidate list cause 1) operation already take it. 2) failed at remote MDT with -2, interpret_cb could not invalidate llog because of empty list
3) some modification do llog write outside of the end of llog file cause local dt object doesn't fit with a remote llog
This exactly llog corruption equal to recX,000...000,recX+2. But I think the same scenario could lead to recX, recX+2.
