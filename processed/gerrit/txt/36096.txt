LU-9855 obd: Replace LPROCFS_CLIMP_[CHECK/EXIT]

Static inline functions are preferred over macros. The function is
placed in obd_class.h instead lprocfs_status.h because obd_class.h
includes header obd.h which contains definition of struct obd_device
and in that way avoids build error: Dereferencing pointer to incomplete
type. Also remove macro LPROCFS_CLIMP_CHECK since it's no longer used.

Linux-commit: a15b2225dd504d6fb5eb4b5a1b29b71cdc9b1f06

In all call sites type of argument that macro LPROCFS_CLIMP_EXIT
use is the same, so replace it with up_read function.
Also remove the macro since it's no longer used.

Linux-commit: e5b6b5d48fa26dd5bb1ec8d6829d318ef4c8585b

I agree, this code is rather ugly.

I imagine something like this:

#define with_obd_cl_sem(__rc, __obd)
  for ( down_read(&(__obd)->u.cli.cl_sem), 
        rc = (__obd)->u.cli.cl_import ? -1-MAX_ERRNO : -ENODEV;
        rc == -1-MAX_ERRNO ? (rc=0. 1) : (up_read(&(__obd)->u.cli.cl_sem), 0);
        )

(with \ at the end of each line)
Then could would be

with_obd_cl_sem(rc, obd) {
     do stuff with obd, possibly set rc
}
return rc;

and the "do stuff..." would run with the sem held and cl_import non-NULL,
or else rc would be -ENODEV.

Interestingly, in the upstream client 'active_show' in lproc_mdc.c doesn't have the climp_check protection.  I wonder how many other places it is missing, and why.
