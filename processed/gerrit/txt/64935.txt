LU-20045 utils: l_getsepol: fix policy version on RHEL 10.1

On RHEL/Rocky 10.1, security_policyvers() reports version 33 but the
only policy file on disk is policy.35. The original code decremented
policyver in a while loop looking for a matching file, reaching 0
without finding one, and failing with ENOENT. This caused l_getsepol
to exit with a non-zero status, which in turn caused ptlrpc to abort
the mount with an error.

Fix by trying security_policyvers() first, then falling back to a
directory scan that picks the highest-versioned policy file present.
Extract this fallback into find_policy_with_highest_ver() to keep
main() readable.

The scan uses fstatat() to avoid a separate stat() round trip, and
uses basename()/dirname() on a copy of the path to avoid modifying
selinux_binary_policy_path(). The getenforce check is moved before
the policy file search so a disabled-SELinux system exits early
without touching the policy directory.

The policy file is always read from disk (not from
/sys/fs/selinux/policy) to ensure all l_getsepol clients in a
nodemap produce an identical hash. The in-kernel re-serialization
of policydb may differ from the on-disk file.
