--- Message (Andreas Dilger, 2016-06-07 23:31:45.000000000) ---
Patch Set 3:

No, because this doesn't permanently block/drop writes to the device, only temporarily while the IO is flushed and the journal committed.  If the device was permanently frozen, then all the MDS/OSS threads would be blocked and potentially client threads would become blocked when their max_rpcs_in_flight limit was reached.  Many of the recovery tests would not work properly because they expect a reply from the server before the server is restarted.
