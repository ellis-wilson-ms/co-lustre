LU-12550 llite: Automatic lockahead

This is a quick draft of a patch to do lockahead
automatically, rather than only at the direction of a
library.

This uses server side contention detection and client side
write pattern tracking to decide when to do lockahead,
doing it only when the server reports lock contention and
the client detects a strided pattern which could benefit.

The main open items are:
1. Total lack of tests
2. Figuring out exactly how to decide this isn't working
and disable it

Known bug:
Attempts to request locks even when writes are not page
aligned, which is harmful, because LDLM locks are aligned
to page boundaries so such requests are useless

test-parameters: fortestonly
