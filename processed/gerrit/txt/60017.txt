LU-19143 tcu: return the original UID/GID/PROJID for getattr()

When a user tries to get the attributes of a file or directory
moved into Trash Can, MDT should return the original UID/GID/
PROJID of the object, which stores in "trusted.unrm" XATTR,
instead of the real ones stored on the file inode: @trash_can_uid,
@trash_can_gid, @trash_can_projid.

By this way, it can ensure the access permission of the file moved
into Trash can with its original UID, GID or PROJID.
