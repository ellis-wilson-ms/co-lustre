LU-18687 lib: move libcfs userspace library

Move the libcfs userspace library to the top-level
lib directory. Eventually, all of the userspace
tools and libraries will live alongside it.

We move *.c files:
from: libcfs/libcfs/util/*.c
to: lib/libcfs/*.c

We move *.h files:
from: libcfs/include/libcfs/util/*.h
to: include/libcfs/util/*.h

git grep __KERNEL__ -- ':(exclude)*.h'

Patch Set 11:

Andreas, about making the libcfs userspace consume the in-kernel NID string definitions:

After thinking about it, I think we should be aiming to decouple userspace and kernel more. Eventually, sites could use the latest up-to-date userspace tools on any version of Lustre. But consuming the in-kernel NID string definitions would add a build time dependency on the kernel where it is not needed.

Personally I like to see libcfs_netstrfns go away. Instead LND drivers should properly register with LNet and use exported functions like libcfs_ip_addr2str() instead. This
