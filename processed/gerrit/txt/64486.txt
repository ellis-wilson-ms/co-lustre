LU-19806 tests: fix sanity-quota FSNAME usage

Use actual OST target name instead of hard-coding "lustre-OST0000"
in the test.  This appears to work coincidentally because 'getquota()'
is using only the last digits as the index number, but this avoids
potential issues in the future.

Fix test-framework.sh::getquota() to properly strip the "_UUID" suffix
from the target name, which is a problem in sanity-quota.sh test_7e.
Also change the MDT/OST index selection to correctly use hex digits.

Fix test script style issues in modified subtests.
- lowercase local variable names
- use ((...)) for numeric comparisons
- use [[...]] for string comparisons
- use "check || action" to avoid spurious error states
