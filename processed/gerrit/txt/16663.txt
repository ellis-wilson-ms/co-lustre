Hi Andreas,

For lfsck, t32_test() contains the following tests:

        if [ "$ff_convert" != "no" -a $(facet_fstype ost1) == "ldiskfs" ]; then
                $r $LCTL lfsck_start -M $fsname-OST0000 || {
                        error_noexit "Start OI scrub on OST0"
                        return 1
                }

                # The oi_scrub should be on ost1, but for test_32(),
                # all on the SINGLEMDS.
                wait_update_facet $SINGLEMDS "$LCTL get_param -n \
                        osd-ldiskfs.$fsname-OST0000.oi_scrub |
                        awk '/^status/ { print \\\$2 }'" "completed" 30 || {
                        error_noexit "Failed to get the expected 'completed'"
                        return 1
                }

                local UPDATED=$($r $LCTL get_param -n \
                                osd-ldiskfs.$fsname-OST0000.oi_scrub |
                                awk '/^updated/ { print $2 }')
                [ $UPDATED -ge 1 ] || {
                        error_noexit "Only $UPDATED objects have been converted"
                        return 1
                }
        fi


For lfs migrate, t32_test() contains the following tests:

                # migrate files/dirs to remote MDT, then move them back
                if [ $(lustre_version_code mds1) -ge $(version_code 2.7.50) -a \
                     $dne_upgrade != "no" ]; then
                        $r $LCTL set_param -n   \
                                mdt.${fsname}*.enable_remote_dir=1 2>/dev/null

                        echo "test migration"
                        pushd $tmp/mnt/lustre
                        for dir in $(find ! -name .lustre ! -name . -type d); do
                                mdt_index=$($LFS getdirstripe -i $dir)
                                stripe_cnt=$($LFS getdirstripe -c $dir)
                                if [ $mdt_index = 0 -a $stripe_cnt -le 1 ]; then
                                        $LFS mv -M 1 $dir || {
                                        popd
                                        error_noexit "migrate MDT1 failed"
                                        return 1
                                }
                                fi
                        done

                        for dir in $(find ! -name . ! -name .lustre -type d); do
                                mdt_index=$($LFS getdirstripe -i $dir)
                                stripe_cnt=$($LFS getdirstripe -c $dir)
                                if [ $mdt_index = 1 -a $stripe_cnt -le 1 ]; then
                                        $LFS mv -M 0 $dir || {
                                        popd
                                        error_noexit "migrate MDT0 failed"
                                        return 1
                                }
                                fi
                        done
                        popd
                fi

Rather than testing these images manually, it makes more sense to investigate why they are not being tested automatically.  Otherwise, they will not be tested automatically in the future and much of the value will be lost.

Maybe /tmp is a ramdisk and this is too small to hold all of the images?  Would it be better to use /var/tmp, or possibly uncompress the images into OSTDEV1, OSTDEV2, MDSDEV1, etc. so that we are sure there is enough space to hold them? 

As for testing LFSCK, it makes sense to test it for ZFS these days also.  I suspect LFSCK testing was restricted to ldiskfs during development, before ZFS support was added in 2.5.58.
