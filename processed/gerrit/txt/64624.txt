LU-19999 tcu: non-root user can access and restore Trash Can

The mode of a directory moved into the Trash Can should not be
changed.

Non-root user should be able to access the Trash Can via ".Trash".

The file in Trash Can is in read-only mode. The attributes such
as UID/GID/PorjID can only be changed when clear LUSTRE_UNRM_FL
flag and restore the file.
It should skip the permission check when restore the file's
original UID/GID/ProjID when recover the files by the non-root
user which is the file owner.

Add sanityn/test_118q to verify that the dir mode does not changed
when a file is moved into Trash Can.
Add sanityn/test_118r to verify that non-root user can access the
Trash Can via ".Trash" and be able to restore the file.
