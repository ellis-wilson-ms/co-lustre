It looks like the upstream code is at https://github.com/intel/isa-l and the most recent release is 2.29, so we should stick with a known tag.

One option that I've seen with other projects is to have this Git repo checked out as part of e.g. autogen.sh so that it is available for building immediately.  That would allow us to link the library directly into "lfs mirror resync" and llite.ko without depending on an external package.  I couldn't find any RPMs or even a .spec file for this library, so it doesn't seem like it is installed separately very often.

James, could you please elaborate?  Is that an RPM thing, autoconf, Git, something else?  AFAICS, there isn't an (S)RPM package for isa-l anywhere, maybe because it is mostly used "under the covers" rather than something that users install?

That means either we need to make our own RPM package(s), check it out from Git directly for the developers/builders (making this automatic would be good), or possibly include the relevant parts of the source directly into Lustre (I think there is a *lot* that we don't need).  

The latter would be easier from a usability POV, but would need ongoing maintenance.

As it is, I think all of the later patches are failing because this library is not on the builders, but the code is not conditional on its presence.  Even if that were fixed (and it should be), then none of the EC functionality would actually exist for testing.
