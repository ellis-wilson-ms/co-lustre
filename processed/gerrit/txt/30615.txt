This was commented though:

> > This is too complicated given that it already does something handled by request timeouts.

> The part that fixes the "purge" mechanism is needed though.
Currently, if a client tries to access a released file and an admin uses:

> lctl set_param mdt.*.hsm_control purge

> The RESTORE request is canceled but locks are not released properly and the client hangs (any new client that tries to accces the same released file will hang too). The workaround is quite simple (stop and restart the coordinator) but a clean fix is probably better.
