LU-17325 o2iblnd: CM_EVENT_UNREACHABLE on established conn

There were examples in the field with RoCE setups which demonstrate
that CM_EVENT_UNREACHABLE may be received when connection is already
in ESTABLISHED state. This causes an assert in kiblnd_cm_callback to
fail.

Handle this in a more gracious manner: report the event as unexpected
and allow the flow to continue. If there are indeed issues on
the connection, it is expected to report transaction errors later
and get cleaned up without crashing the whole system.

> Is it reported with host down? if yes - should connection to be

Sorry, missed your question. I don't know for sure if it was reported with peer host down, perhaps Shuichi Ihara can confirm. My understanding was that the peer host wasn't actually down, but the local link went down. It is true that we could query link status and ignore the event if the link is down, but I was concerned about race conditions involved, so decided it was easier to just ignore the event and let the connection close because of the tx timeout later on.
