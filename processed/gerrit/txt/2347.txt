LU-957 scrub: general framework for new LFSCK

LFSCK contains several components, different components may
perform checking/repairing in different layers. This patch
defines common iterfaces and functions, which can be shared
among these different components.

1) Scrub head
For each level (layer), there is a single scrub head, for the
versions, common parameters, and etc.

2) Scrub unit
For each LFSCK component, there is a scrub unit, to trace its
processing: how many objects have been checked/updated/failed,
the latest position, the first failure position, and so on.

3) Checkpoint file
For each level (layer), there is a single checkpoint file to
record both the scrub head and the scrub units in the level.
The checkpoint file is only visible inside its level.

4) scrub_prep()
Each LFSCK component can register an prep() function to prepare
something before related LFSCK component start.

5) scrub_exec()
Each LFSCK component must register an exec() function to perform
corresponding system component checking/repairing.

6) scrub_post()
Each LFSCK component can register an post() function to cleanup
something before related LFSCK component stop.
