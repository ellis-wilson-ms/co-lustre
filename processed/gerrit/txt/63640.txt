## Summary

**The Fix:**
Changed `res->lqe_glbl_data` to `lqe->lqe_glbl_data` on line 231 of `lustre/quota/qmt_lock.c`.

**Rationale:**
The original patch attempted to change the error message to print `lqe_glbl_data` instead of `lr_lvb_data`, which makes sense given the context (the code is checking if `lqe->lqe_glbl_data` is held by the work queue). However, the patch incorrectly tried to access this field from the `res` variable (which is a `struct ldlm_resource`), when it should have accessed it from the `lqe` variable (which is a `struct lquota_entry`).

The `struct ldlm_resource` only has the field `lr_lvb_data`, while `struct lquota_entry` has the field `lqe_glbl_data`. Since the code is checking `lqe->lqe_glbl_data` on line 220 and the error message is meant to print the pointer value of this field when it's unexpectedly held, the correct fix is to use `lqe->lqe_glbl_data` instead of `res->lqe_glbl_data`.
