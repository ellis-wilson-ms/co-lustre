--- Message (Bob Glossman, 2014-05-29 17:36:20.000000000) ---
Patch Set 6: Code-Review-1

It makes me very uncomfortable to leave out any of the triple set of SLES kernel rpms; kernel-default, kernel-default-base, kernel-default-devel.  I ALWAYS deal with that set of rpms as a unit figuring if I don't need them all for one flavor of build I will soon need them for another.  It seems to me to be cheap insurance to keep them all in our build environments.

--- Message (Brian J. Murrell, 2014-05-29 19:00:01.000000000) ---
Patch Set 6:

Bob,

kernel-default is the binary kernel RPM.  There is absolutely no need for it when building Lustre (as much as there is no need for the RHEL "kernel" package when building on RHEL) so it is a waste to download it, a waste to cache it, a waste to unpack it for the build and adds more error surface to build process since it adds one more thing that could be missing, fail to download, etc.

And clearly the Jenkins builds that were successfully done to give this patch a +1 from Jenkins proves that it's not needed.

--- Message (Bob Glossman, 2014-05-29 20:08:28.000000000) ---
Patch Set 6: Code-Review+1

Brian, yes I can see that Jenkins can do both server and client builds OK without. Based on that I'm not willing to fight about it so I will give in and +review.  I still think it's a bad idea to cut down on the population of our build environments even though not needed right now.
