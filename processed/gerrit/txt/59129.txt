## 1. Commit Summary

This commit (LU-18357) introduces foundational changes to the Lustre nodemap's IAM (Index Attribute Map) handling to support future work on multiple filesets per nodemap.

The key modifications include:
*   Introducing a `fileset_id` parameter to functions that add, update, or delete fileset information within the nodemap IAM (`nodemap_idx_fileset_add`, `nodemap_idx_fileset_update`, `nodemap_idx_fileset_del`). For this commit, this `fileset_id` is typically hardcoded to `0` in the calling functions (e.g., in `nodemap_handler.c`), indicating that it currently supports the primary (single) fileset.
*   Adding a new helper function `nodemap_fileset_get_subid(fileset_id)` in `nodemap_storage.c`. This function calculates the base sub-identifier (subid) in the IAM for a given fileset, enabling different filesets to occupy distinct subid ranges. The formula used is `NODEMAP_FILESET + (fileset_id * LUSTRE_NODEMAP_FILESET_SUBID_RANGE)`.
*   Refactoring the existing `nodemap_cluster_fileset_helper` function in `nodemap_storage.c`. The original logic for applying a fileset fragment is extracted into `nodemap_cluster_rec_fileset_fragment`. The new `nodemap_cluster_fileset_helper` now uses the incoming IAM record's subid to determine the `fileset_id` and then calls `nodemap_cluster_rec_fileset_fragment` for the appropriate fileset (currently only the primary fileset if `fileset_id` is 0).
*   Updating function signatures in `nodemap_internal.h` to reflect the new `fileset_id` parameter.
*   A minor modification to a comment in the UAPI header `lustre_disk.h` related to padding in `struct nodemap_fileset_rec`.

Overall, these changes lay the groundwork for associating IAM entries with specific filesets by using a `fileset_id` to segregate their respective configuration fragments within the nodemap's IAM cluster index space.
