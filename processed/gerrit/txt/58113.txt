LU-18502 utils: Support YAML for get_param parameter names

"lctl get_param -y" does not print a valid YAML for the parameter
names even if the values are YAML-compliant. This patch extends the
"-y" parameter to also affect the parameter names, such that the '='
separator is replaced by ': '. For example, the get_param default
output for nodemap 'id' and 'idmap' are:

nodemap.c0.id=2
nodemap.c0.idmap=[
 { idtype: uid, client_id: 1003, fs_id: 11000 },
 { idtype: gid, client_id: 1003, fs_id: 11000 }
]

With "lctl get_param -y" this becomes YAML-compliant:

nodemap.c0.id: 2
nodemap.c0.idmap: [
 { idtype: uid, client_id: 1003, fs_id: 11000 },
 { idtype: gid, client_id: 1003, fs_id: 11000 }
]

In the context of this patch, the empty line was removed when printing
multi-line parameters with "lctl get_param -H" to be consistent with
the other get_param modes. Further, the tabs when printing filesets
through "lctl get_param" were replaced with spaces since tabs are not
yaml-compliant. The corresponding "awk" changes in fileset-testing
utility functions are compatible with previous tab format.

Sanity-sec test_25c was added to test this new argument and nodemap
yaml-compliance for all present and future properties.
