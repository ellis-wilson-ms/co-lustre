LU-12188 ec: handle user land EC layout setup

Make the user land tools aware of the new parity components.
Changes done to support EC are:

1) Updating liblustreapi code to dump lmm containing EC information

2) Changes to add a new parity component to an existing layout

3) Changes to create a new layout containing parity components

4) Parity components are immutable so don't allow a component
   setting change with the flags

Parity components are always a specific setup, meaning we setup
individual OST objects. This is because we need to know which
components contain the data that we calculate the parity values
with.

(3 comments)

I did that approach at first but to do say a RAID5 setup would require a really complicated setup on the user end. Especially if they want to shift the location of the parity to avoid the single OSS failing with all the parity components. I moved that complex behind the scene from the user using the OST objects. A RAID5 setup would be (approximately)

lfs setstripe -E 1M -c 3 -E -L ec --ec_data_count 3 --ec_parity_count 1 -E 1M -c 2 -E -L ec --ec_data_count 3 --ec_parity_count 1 -E 1M -c 1 -E 1M -c 1 -E 1M -c 1 -L ec --ec_count_data 3 --ec_parity_count 1 -E 1M -c 2

Even then you can't do this since that would require supporting multiple extents in the EC component. Only one extent that covers the entire range. So with the component approach you really can do a RAID4 type setup.

Sorry, I don't understand this (even after re-aligning to see what it is doing).  Why are there are all of the components ending at 1M?  I'm _guessing_ that the two "-E 1M -c 1" components are a mirror for the first 1MB of the file (1-stripe), but not sure about the rest.

I was thinking something like the following for single parity (with abbreviated syntax):

 lfs setstripe -E 32M -c 1:1+1 -E 1G -c 8:8+1 -E eof -c 64:8+1

The ":D+P" is shorthand for "-L ec --ec_count_data D --ec_parity_count P".  It is a little tricky to specify the first (mirror) component, but that would be handled internally to create a regular mirror component I think, and/or the EC code would understand D<=P means "mirror component".
