LU-15909 lnet: Add peer net send lists

We can optimize the selection of peer nets in a similar manner as we
do for peer NIs. Divide peer nets into three lists:
1. The subset of peer nets that have a priority defined
  - This it he "priority list"
2. The subset of peer nets that do not have a priority defined
  - This is the "non-priority list"
3. The subset of peer nets that have degraded health
  - This is the "degraded list"

By diving the peer nets in this manner the selection algorithm can
then consider only those attributes of each net that are relevant when
comparing nets on a given list.

Peer nets on the priority list are, by definition, completely healthy,
so the selection algorithm doesn't need to compare their health
values. This list is sorted by priority, so if we encounter a peer
net with a lower priority than the current "best" peer net then we can
stop inspecting any more peer nets as we've already found the one with
highest priority.

Peer nets on the non-priority list are also completely healthy, and
they do not have a priority value set, so we needn't consider either
of those criteria when selecting the best peer net.

Peer nets on the degraded list may have varying health values and
selection priorities, so all of these critera are considered when
selecting a peer net from this list.

A read-write lock is used to synchronize any updates to these lists,
and a spinlock in the lnet_peer struct is used to prevent concurrent
updates.

lnet_select_best_lpn() is introduced to provide a common function for
selecting a peer net. The selected peer net is assigned to the
lnet_send_data::sd_best_lpn field. This function takes an additional
argument that tells it whether the send is to a local peer or a
remote peer. For a local peer, the lnet_send_data:sd_best_net field
is also populated with the lnet_net object that corresponds to the
selected peer net. For a remote peer, the lnet_send_data::sd_best_rnet
field is populated with the lnet_remotenet object that correponds to
the selected peer net. Both of these lnet_send_data fields are
newly added with this patch.
