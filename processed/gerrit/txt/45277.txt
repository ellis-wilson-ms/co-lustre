--- Commit Message ---
LU-15047 gss: gss integrity check with multi-rail

With multi-rail, a primary NID is used as node identifier, but LNet
decides which NID is actually used for sending/receiving data, on a
per request basis.
For the integrity check mechanism implemented as part of GSS, the
primary NID must be used in order to compute HMAC with the correct
key, independently of the actual NID for the current request.

--- Message (Chris Horn, 2021-10-28 21:49:26.000000000) ---
Patch Set 1:

> James, Jeremy, the "primary NID" is used by Lustre and assigned to
 > the node when it is first connected, and does not change for the
 > lifetime of the system (AFAIK), even if the actual NIDs change.

The primary NID of an LNet peer can certainly change. It can be manually deleted by an admin, or a peer's interface could fail to initalize on a reboot which would result in a different primary NID being defined. There is work in LU-14668 to allow Lustre to "fix" the primary NID in LNet, but that work has not been completed.
