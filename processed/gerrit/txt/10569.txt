LU-5582  mgc: Client fails to mount the filesystem

lustre_start_mgc() creates MGC device, if error happens later on
ll_fill_super(), this device is still attached, leaving a stale entry.
Subsequent lustre client mount attempts, fail, with EEXIST (-17).

This change, makes sure, such scenarios are taken care of with
mgc_precleanup() remvoving llog subsystem with any stale entry.

Thus the subsequent mount attempts succeed without any errors.

mgc_precleanup asks the mgc_requeue_thread to do cleanup/shutdown
not only for last MGC entry but for every MGC being shutdown.

When mgc_precleanup() kicks the requeue thread - it does not start
immediately but _waits_ for some amount of time magically calculated
with 'lwi'. As long as RQ_STOP is not set before the kick we added
only expired timeout makes mgc_requeue_thread() to continue.

The flag RQ_MGC_PRECLEANUP is introduced to reduce this slowness.
