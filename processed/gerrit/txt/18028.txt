--- Commit Message ---
LU-6838 llog: limit file size of plain logs

on small filesystems plain log can grow dramatically. especially
given large record sizes produced by DNE and extended chunksize.
I saw >50% of space consumed by a single llog file which was still
in use. this leads to test failures (sanityn, etc).
the patch introduces additional limit on plain llog size, which
is calculated as <free space>/64 (128MB at most) at llog creation
time.

--- Message (wangdi, 2016-02-02 19:26:15.000000000) ---
Patch Set 7:

This patch actually expose a problem of current llog implementation.
See llog_cat_current_log(), if the current plain log is full, it will switch the current plain log to next plain log (chd_next_log), then set chd_next_log = NULL.  This is not safe for multiple threads.

Because once chd_next_log is set to NULL, if there are enough following threads fill up this plain log again (because of this patch), then there are no next plain log to be backup here.  i.e. probably panic in llog_cat_current_log() because chd_next_log is NULL. (Note: the next llog will be only be re-created in declare phase.)
