LU-7991 quota: project quota against ZFS backend

This patch enables the project quota feature against ZFS
backend. The project quota patch for ZFS itself will be
landed via https://github.com/zfsonlinux/zfs/pull/6290.
Such patch may be released via zfs-0.7.x (x >= 5).

This patch also cleanup the quota logic for ZFS backend
and sanity-quota.sh

Patch Set 7: Code-Review-1

(17 comments)

This patch could be simplified in several ways:
- don't rename OSD_QID_* to QID_*
- check for ZFS_PROJINHERIT instead of HAVE_PROJECT_QUOTA_ZFS, then no need for configure check
- don't rename HAVE_PROJECT_QUOTA for ldiskfs

then no need to change the osd-ldiskfs code at all.

This shouldn't depend on the other patches, since there is no reason that that the ZFS quota patch depends on the client freeing the reply buffer early, or lfsck, or ldiskfs changes.

The code should check if project quota is enabled for the dataset at runtime, rather than assuming it is set.
