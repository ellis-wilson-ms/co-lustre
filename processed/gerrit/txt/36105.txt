We need it to be part of the packages, since many users do not build their own binaries.

 > honestly got a bit lost with the automake files.
 > I think I've found something usable here https://www.swansontec.com/bash-completion.html
 > and will monkey something into lustre-build.m4 in a v2 here today

I don't think you really need to do anything with autoconf, since we want to (IMHO) always install the completion file.  The only thing that is needed is the update lustre/scripts/Makefile.am (I don't think we need the "completions" subdirectory) to install lctl.bash like:

 completiondir = /usr/share/bash-completion/completions
 completion_SCRIPTS = lctl.bash

and add it to lustre.spec.in for the utils package:

 %files -f lustre.files
 :
 :
 %{_datadir}/bash-completion/completions/*

Not totally sure about how to do it with Debian, but once it is in RHEL then someone will also add it there.
