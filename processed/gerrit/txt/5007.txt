LU-28 fileset: add fileset mount support

This patch enables client to mount subdirectory as fileset.

usage: mount -t lustre mgsname:/fsname/subdir /mount/point

* mdt lookup fileset fid and return to client during mount.
* `lfs fid2path` support for fileset.

Test-Parameters: envdefinitions=FILESET=/somedir testlist=sanity

Patch Set 2: I would prefer that you didn't submit this

It would have been better to resubmit against the existing patch in http://review.whamcloud.com/165 instead of submitting a patch as a new change.  That would keep Lai as the original author of the patch, and would make it easier to compare the new patch against the old one.

I think this could be done in a much simpler manner using a "--bind" mount at the client.  Essentially:

- mount the filesystem in some temporary location
- mount --bind the subdirectory to the final mountpoint

That said, it is a bit messy to have a temporary mountpoint in the middle of mounting each subdirectory (e.g. in case of automount mounting a subdirectory for each user).  The benefit is that there is only a small amount of complexity inside mount.lustre instead of a larger amount of complexity in the Lustre kernel code.

For fid2path, it would be cleaner to fix this in the kernel instead of returning all the way to the "filesystem root" and then chopping off the "subdirectory" part of the path. The kernel could stop traversing the path upwards when it hits the vfsmount mnt_root dentry, or possibly the directory on which the ioctl is being run.  That would ensure that fid2path also works as expected if a subdirectory is mounted using --bind (whether done by mount.lustre or not) by some user or chroot or as the root filesystem of some client node.
