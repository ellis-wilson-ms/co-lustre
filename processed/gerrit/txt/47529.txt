LU-15909 lnet: Use net/peer net credits for net selection

Add tracking of tx credits to lnet_net and lnet_peer_net objects. This
allows us to consider the resources available in a net/peer net when
selecting the net/peer net to be used for a send. This provides more
efficient utilization of available network resources when there are
multiple network types (i.e. fast IB/o2ib vs. slow ethernet/tcp).

With this change we no longer consider sequence numbers when selecting
a net, peer net, local NI or peer NI. The sequence numbers were used
to provide round robin behavior between nets/NIs when other selection
criteria were otherwise equal. However, there are pathological cases
where the round robin behavior can be broken which leads to poor
utilization of available network resources (e.g. sequence numbers wrap
such that some net/NI's is reset to 0 while others have very large
sequence numbers). Furthermore, it is unlikely that when a system is
under load that this round robin behavior provides any benefit because
the available resources are unlikely to be equal.
