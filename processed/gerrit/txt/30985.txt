In test-framework::load_modules_local(), change the order in which
modules are loaded, by inverting mdd and mdt.
This is necessary to load mdt module before mdd because mdd module
uses mdt_get_client_nid() exported by the mdt module.

--- Message (Andreas Dilger, 2018-01-24 08:09:27.000000000) ---
Patch Set 1:

It seems that this is a bug added by the very first patch in the series, https://review.whamcloud.com/28213 but it isn't clear why this problem is only showing up here?

Inverting the module loading order is only hiding the problem, and may cause other problems in the future. The MDT device is logically stacked on top of the MDD, but for some operations (eg. mdd-survey, or something like echo_client used by obdfilter-survey, that may not be true, and calling an mdt_* method on the env May access invalid data, so it is bad that there is a reverse dependency on the modules.

It would be better to fix this by adding the client NID for this request down to the ChangeLog writer. This can be done via uc_cred by adding a new uc_nid field in the same way as uc_jobid.
