--- Message (Andreas Dilger, 2016-06-17 04:07:11.000000000) ---
Patch Set 1:

I'm totally fine with this change in principle, but I think it is too big and changing too many things to be easily landed.  Trying to convert every lnet/ typedef in a single patch just makes the whole thing too large and very likely to hit conflicts just because the patch is so large.

What makes patches like this more easily created and landed without conflicts is to add the struct name to all of the typedef'd structs, but NOT remove the typedef in the same patch.  The typedefs should be marked deprecated in checkpatch.pl in the first patch so that new patches do not add more of them (though there may be some patches in flight that are still using them).

Then, a later patch can go through and remove the typedefs a few weeks later, both at the struct definition as well as in the code where any patches that were landed while this one was being reviewed.

That avoids a flag day that breaks any in-flight patches, and also allows this huge patch to be split into smaller chunks since the original typedef is not being removed.
