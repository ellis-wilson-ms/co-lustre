LU-12597 contrib: move comma_list rules for checkpatch

"checkpatch.pl" uses "userspace_spelling.txt" for files in
lustre/utils/, lustre/tests/, lnet/utils/, libcfs/libcfs/util/. For
all other files, "spelling.txt" is used. Therefore,
"comma_list.*osts_nodes" is currently not checked for lustre/tests/.

This patch fixes this, so the bash rules are applied correctly to
lustre/tests/ instead of kernel files.
