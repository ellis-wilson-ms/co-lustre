--- Message (Patrick Farrell, 2019-05-31 14:17:38.000000000) ---
Patch Set 2: Code-Review-1

This looks correct, but I think there is another issue as well, which I alluded to in the original ticket (probably wasn't very clear).

The assert hit is at the top of ptlrpc_set_add_req:
        LASSERT(req->rq_import->imp_state != LUSTRE_IMP_IDLE);

And imp_inflight is incremented in:
                ptlrpc_send_new_req(req);

Which is called at the end of ptlrpc_set_add_req.

So using imp_inflight leaves a race, since we assert !IDLE before we increment it.

Would it be easier to just use the import refcount, like we do when deciding if a connection is ready to idle?  (Or we could just use the same check entirely, with an adjustment for having the extra ref for the disconnect?)
