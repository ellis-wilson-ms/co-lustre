LU-19074 ec: optimize encode with lookup tables

While building some sanity tests for the EC code,
Claude suggested this notable optimization.
Correctness will be tested in an upcoming test to be
added to sanity, but has been verified.

Replace expensive gf_mul() Galois Field multiplication
with fast lookup table operations in ec_encode_data().
The current O(log n) computation creates a performance
bottleneck for erasure coding operations.

The optimization splits each byte into 4-bit nibbles
and uses pre-computed lookup tables:
coeff = v[offset + lower] ^ v[offset + 16 + upper].

Testing with standalone benchmark across multiple
scenarios shows consistent 15-25% performance
improvements in encoding:
- Small blocks (4KB, 4+2 EC): 1.20x speedup,
  395->476 MB/s
- Medium blocks (64KB, 4+2 EC): 1.20x speedup,
  384->460 MB/s
- Large blocks (1MB, 8+4 EC): 1.18x speedup,
  77->91 MB/s
- Best case (256KB, 6+3 EC): 1.32x speedup,
  242->320 MB/s

The optimization provides better CPU cache utilization
and is particularly beneficial for high-frequency
operations and complex EC configurations.
