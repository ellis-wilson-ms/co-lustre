doesn't go over s_maxbytes.
We should do the same in osd_punch() before calling
i_size_write() and ldiskfs_truncate().

(3 comments)

I see that cl_object_maxbytes() and lov_object_maxbytes() are also using LLONG_MAX instead of MAX_LFS_FILESIZE. While they are the same value on 64-bit systems, it is more clear to use MAX_LFS_FILESIZE to see what it means and where it is used.

I also just found patch https://review.whamcloud.com/41365 "LU-14384 tests: write and read last object stripe" (which is also adding test_34i) that is testing the file size limits.

I want to give this patch +1, but I think we need to have a "large size files still work" test included (maybe from 41365, or update that patch and rebase this on top of it?) so that we aren't accidentally shooting ourselves in the foot.
