Patch Set 8: Code-Review-1

(53 comments)

I'm not sure why all of the mapping functions changed from returning the qtype or similar, and now take the return value as an argument and return void?  That makes it difficult to stack these functions or return an error for unknown quota types.

It seems there are still too many qtype2*() and *2qtype() mapping functions, and it seems at least some of them could be removed.

The usage of MAXQUOTAS in the Lustre code should all be replaced with LL_MAXQUOTAS so that it is independent of what the kernel defines MAXQUOTAS to be, since I don't think the Lustre code interacts directly with the kernel quota code at all.
