The original patch 54fe9796ec837698 ("LU-2216 mdt: remove obsolete DNE code") in 2012 removed all caller which set MAY_UNLINK. mdd_may_unlink() was only called if MAY_UNLINK is set. So this made it so that mdd_may_unlink() is never called in practice. Then 55a61af0d96581 ("LU-6450 mdd: Remove unused MAY_ constants") in 2015 removed the only code checking for MAY_UNLINK. So this made it so that mdd_may_unlink() in never called in the code either. But it failed to remove mdd_may_unlink().

So it was disabled for so long, I think it was intended. Is there anyway we can verify with a test that everything is working as expected?
