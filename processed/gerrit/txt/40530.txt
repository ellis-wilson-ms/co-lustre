--- Message (Andreas Dilger, 2020-11-25 08:43:28.000000000) ---
Patch Set 2:

I don't think we can *require* fallocate(PUNCH_HOLE) support on the target filesystem.  To work around this, we can do two things:
- if the target file size is not larger than the start of the hole, no action is needed
- if the target file *is* larger than the start of the hole, try to use fallocate(PUNCH_HOLE) to create a hole of the correct size
- if fallocate() fails, then truncate the target file to the start of the first hole, then continue writing the rest of the file (no more truncates are needed)
