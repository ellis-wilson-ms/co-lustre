LU-17431 nodemap: modify all properties of a dynamic nm

All properties of a dynamic nodemap must be taken into account. So
move the check for dynamic in nodemap_idx_* functions, except for
nodemap_activate which is independent of any nodemap.
Update sanity-sec test_72 to correctly exercise this.

One comment here: Instead of having to remember adding "if (nodemap->nm_dyn) ..." in nodemap_handler.c to handle dynamic nodemaps, I think it could be easier to let nodemap_storage handle this transparently. 

We already have this in some functions like "nodemap_idx_cluster_add_update()", which simply exits successfully for dynamic nodemaps. I'm not 100% certain if this works in all cases, but I can imagine it would avoid future bugs due to some corner cases. 

What do you think of adding this shortcut to all relevant functions (nodemap_idx_*) in this patch instead of the nodemap_handler mods?
