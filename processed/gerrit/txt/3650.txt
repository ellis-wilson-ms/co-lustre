Patch Set 1: (9 inline comments)

First-pass comments on a partly-completed inspection on an earlier version of the patch.  I suspect these are still needed for the new code.

I haven't really looked at the core functionality yet, but the idea of "mount" handling the osd internal mount, and the server stack being built on top of this is a lot nicer than if "mount" is mounting the top-level MDT or OST device, and having to pass all the config arguments down through the stack.  This will also help with multiple top-level devices sharing the same OSD.

That could simplify device setup considerably, since the device passed from mount can be used directly for mounting inside the kernel, instead of the convoluted mess that it is in today, and similarly the user-visible mountpoint can be attached more directly to the underlying filesystem, so things like ioctl passthrough are less complex.
