Patch Set 1: Code-Review-1

(6 comments)

I think this patch needs good description of what 'hint' is for and how/when to use it properly. My understanding is that it is optimization to be used when lock is canceled/converted to don't reprocess everything but just canceled bits. Meanwhile there is also several calls when 'hint' is used from just granted locks which I don't understand yet. We are reprocessing queues again for granted lock bits inside ldlm_reprocess_inodebits_queue() so what is the point to call reprocessing for just granted lock externally?
