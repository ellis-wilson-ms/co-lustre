--- Commit Message ---
LU-537 o2iblnd: support aliased interface

Current kiblnd_create_dev() doesn't allow to dev_get_by_name() to
return NULL (we added this for supporting failover of ib-bonding),
but this can happen for aliased interface, we can fix this by simply
remove the limitation and set kib_dev_t::ibd_can_failover to 0 for
aliased interface.

If o2iblnd needs to support aliased interfaces, then this patch alone is not enough. Aliased interfaces can fail in kiblnd_startup():                                                                       

2874         if (strlen(ifname) >= sizeof(ibdev->ibd_ifname)) {                          
2875                 CERROR("IPoIB interface name too long: %s\n", ifname);              
2876                 goto failed;                                                        
2877         }

Linux allows alias names up to IFALIASZ, which is larger than sizeof(ibdev->ibd_ifname).
