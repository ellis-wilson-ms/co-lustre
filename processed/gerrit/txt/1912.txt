--- Message (Andreas Dilger, 2012-01-05 00:26:48.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

(1 inline comment)

The main objection is that this reduces the minimum debug amount below 10MB on a system with few cores.

That said, I wouldn't object to fixing this in a more robust manner, such as reducing the kernel-side minimum limit to some fixed number (e.g. max(2MB, CFS_PAGE_SIZE * cfs_num_possible_cpus()).  On a system with a large number of cores (e.g. some SGI monster with 4096 cores needing 8GB of RAM just for debug logs) this might consume a lot of memory even though only a limited number of cores might be doing Lustre IO, and those cores might not even have that much RAM themselves.
