LU-19884 mdt: add 'enable_foreign_dir' parameter

The "lfs setdirstripe" command for remote directories and DAOS
foreign layouts represent similar directory operations but differ
significantly in how they affect the filesystem.

For Lustre remote directories, it moves potentially a large
number of new files and subdirectories over to the remote MDT.
The mdt.*.enable_remote_dir* parameters control whether users can
explicitly create subdirectories on MDTs other than parent's.

For DAOS, a foreign directory is object that references a remote
"directory tree" stored in an container external to Lustre, and
does not affect Lustre MDT usage at all beyond the initial create.
This is currently also controlled by mdt.*.enable_remote_dir*,
which defaults to disabled for non-root users, even though DAOS
foreign directories are no more overhead than regular dirirectories.

Introduce new tunable parameters "mdt.*.enable_foreign_dir" and
"mdt.*.enable_foreign_dir_gid" on the MDTs to separate control of
foreign directories from DNE remote directory control.  They have
the same semantics as mdt.*.enable_remote_dir* parameters, but apply
only when creating new foreign directories and default to enabled.

Add new nodemap RBAC 'foreign_ops' role to control this per nodemap.

Add mdt.enable_foreign_dir.4, mdt.enable_foreign_dir_gid.4 man pages.

Add sanity.sh test_27Ke to test enable_foreign_dir parameter settings.
Add sanity-sec.sh test_64k to test nodemap foreign_ops RBAC settings.

Update runas to allow specifying supplementary groups by name.
Fix usage in parse_foreign_dir test util and improve error messages.

gid_t gid;  // New variable to hold the numeric GID
