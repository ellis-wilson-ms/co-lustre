--- Commit Message ---
LU-12668 lov: proactive dead-OST detection for degraded reads

Instead of letting I/O proceed to an inactive OST and failing deep
in the BRW/lock path, detect dead imports early and route to the EC
recovery path immediately.

- Check for deactivated/invalid imports in lov_io_mirror_init() and
  switch to CIT_EC_RD before starting normal read I/O.
- Mark dead stripes as errored upfront in lov_ecio_add_data_sub()
  so the recovery loop knows which stripes to reconstruct without
  attempting I/O to them.
- Force buffered reads in vvp_io_ec_rd_end() so that after EC
  recovery populates the page cache, direct I/O does not bypass it
  and re-issue reads to the dead OST.

Created with the aid of claude. This patch enables the ability to
run the EC failure orchestrator which will be in a patch following
this one.
