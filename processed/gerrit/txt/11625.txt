Patch Set 9:

The location of the dynamic library isn't known, since it isn't installed. So libtool uses a mechanism to create a new binary with the hardcoded location of the library the first time the executable is ran.

For instance lustre/utils/lctl is a script, that will use lustre/utils/.libs/lctl to create lustre/utils/.libs/lt-lctl. Every time lctl is run, it will exec .libs/lt-lctl. The problem here is that it's failing to create the lt-lctl binary.

This all goes away when the software is installed, and /usr/bin/lctl is the real binary.

I don't know how your dev setup is. A possible workaround is to run each binary individually after they are built on the build machine (like lctl --version), which will trigger the creation of the lt-* binaries. That will only work if the paths are the same on the build and test machines.
