--- Message (Andreas Dilger, 2016-09-16 07:29:52.000000000) ---
Patch Set 1:

Is there a compatibility issue with this change, since all previous osd-zfs filesystems created the quota files in MASTER_NODE_OBJ?  I think the reason the quota files were created outside the namespace was to avoid them being deleted by users if the filesystem was mounted locally.

Also, since the ZFS dnode accounting feature is coming soon, it might not make sense to introduce changes to these files when they will soon become obsolete, and eventually removed once osd-zfs is using a newer version of ZFS that includes dnode accounting and there is no chance of downgrading to an older version.
