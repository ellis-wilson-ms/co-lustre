LU-19496 kfilnd: Fix error handling in DMA mapping

Refactor kfilnd_tn_set_sgl_buf() to improve error handling and code
clarity:

- Add function documentation describing parameters, behavior, and
  return values
- Include network interface name in all error messages for better
  debugging context
- Add error handling for DMA mapping failures (both GPU and non-GPU
  paths)
- tn_sgt_mapped is now only set when the sg_table is successfully
  mapped. kfilnd_tn_set_sgl_buf() will free the sg_table in the case
  of error.
