LU-17463 osc: add option to disable page cache shrinker

The pages mapped into VM_LOCKED [mlocked()ed] VMAs are unevictable
pages. Those pages are marked with PG_mlocked.
However, page cache shrinker in Lustre treats all cached pages
equally even some of them are unevictable. It may evict mlocked
pages by mlock() or mlockall() calls wrongly.

This patch adds an tunable option to enable or disable page cache
shrinker:
- osc.*.enable_page_cache_shrink
It is enabled by default.

--- Message (Andreas Dilger, 2024-01-24 16:30:52.000000000) ---
Patch Set 1:

While I like the idea of having a parameter that can disable the shrinker, to allow debugging, I don't think this is fixing the problem properly. A second patch is needed to only skip the mlock pages when checking the page cache limits. 

Most sites will not know about this parameter, so they would only enable it after opening a ticket. Also, it is a global setting only accessible to root, so there may be many times that it should not be enabled, but a few times that it should be enabled on the same system (different jobs/users).

We need to decide what the correct behaviour is in this case:
- exclude mlock pages from the cache tracking entirely?  That would mean the number of cached pages will exceed the limit. Does that cause any issues with writeback finding the dirty pages?
- include them in the cache calculation but never evict them?  That would shrink the space for other pages but stay within the limits (at least until the number of mlock pages exceeds the total limit).

Patrick, any thoughts on which one is better for real usage?
