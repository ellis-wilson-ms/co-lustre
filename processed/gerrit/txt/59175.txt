LU-18993 osc: Add IO RPC latency statistics by size

Add a new 'osc.*.io_latency_stats' stats that tracks RPC latency
histograms separately for each RPC size (in KiB). This provides
more detailed performance metrics for diagnosing Lustre client-server
communication issues by showing how RPC latency varies with size.

The implementation:
1. Adds new histogram arrays to client_obd for read and write RPCs
2. Updates brw_interpret() to record latency in the appropriate
   histogram based on RPC size (aa->aa_page_count)
3. Adds a new proc entry 'rpc_latency_stats' that displays the
   histograms in a human-readable format
4. Skips printing zero-valued entries for better readability
5. Adds test_127e to verify the functionality

Example output format:
  io_latency_by_size:
  snapshot_time: 1756402276.152694808
  start_time:    14.921641461
  elapsed_time:  1756402261.231053347
  rd_4K: { 256us: 5, 512us: 5, }
  rd_64K: { 512us: 10, }
  rd_1024K: { 1024us: 5, 2048us: 4, 4096us: 1, }
  rd_4096K: { 4096us: 9, 8192us: 1, }
  wr_4K: { 2048us: 8, 4096us: 1, 8192us: 1, }
  wr_64K: { 2048us: 8, 4096us: 2, }
  wr_1024K: { 4096us: 10, }
  wr_4096K: { 8192us: 9, 16384us: 1, }
