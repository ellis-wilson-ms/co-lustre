LU-8964 llite: tunable ptasks number with optimized IO size

The tunable llite.*.max_read_ptasks can be used to reduce the number
of ptasks used in ll_readpages() used for async IO. Lower numbers than
the number of CPU cores are increasing performance with ZFS OSTs.

The number of pages handled by each ptask is now rounded to a multiple
of 1MB, if possible, such that at most one of the async threads has
an RPC which is not optimal. This increases chances for a nice stream
of requests which are multiples of 1MB away from each other, thus
helping prefetch to see good patterns in sequential streams.
