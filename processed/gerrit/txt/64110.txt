LU-19886 utils: fix buffer overflow in lshowmount nidlist

nl_strxcat() passes max_len (total buffer size) as the size
argument to the first snprintf() call, but the write position
is s + strlen(s), so the available space is only
max_len - strlen(s). With FORTIFY_SOURCE=3 (default on el10),
glibc detects that the size argument exceeds the remaining
object size and aborts with "buffer overflow detected".

Similarly, nl_xstring() passes len (total buffer size) as the
count argument to strncat(), which should be the maximum number
of characters to append, not the total buffer size. The correct
value is len - strlen(s) - 1 (remaining space minus null
terminator).

Fix all four instances to use the remaining buffer space rather
than the total allocation size.
