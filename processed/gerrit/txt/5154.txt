--- Commit Message ---
LU-2634 ldiskfs: clear extent flag for fast symlink

Fast symlinks on MDT filesystems formatted with the "extents" feature
are created with EXT4_EXTENTS_FL. e2fsck considers this a corruption
and deletes the symlink.

The patch is to clear LDISKFS_INODE_EXTENTS flag in
osd_ldiskfs_writelink() just as ext4_symlink() does.

Signed-off-by: Liu Ying <emoly.liu@intel.com>

--- Message (Andreas Dilger, 2013-01-24 18:57:14.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

The job of fixing the bad symlinks can be done by e2fsck, since there can be problems trying to do this in the kernel as you have done (doing a write during a read operation can cause problems, or possibly exceed the transaction credits for a nested transaction.

The main other question I have is why this problem is not hit with normal ext4 filesytems?  That is what makes me think that there is some problem in osd-ldiskfs, or possibly our existing ldiskfs patches. It doesn't seem right that fixing this needs a new ldiskfs patch, since no similar problem is hit with ext4, which is presumably using the same code.
