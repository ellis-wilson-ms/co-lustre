LU-6135 lustreapi: allow specific-OST with llapi_layout

Add support to llapi_layout_ost_index_set() for specifying
specific OST indices for a newly created files. Using
ost_index=LLAPI_LAYOUT_DEFAULT to stripe 0 will reset the
specified stripes back to the default.

The barrier to supporting the layout api is the xattr handling. I remember a special hack in xattr handling in llite to make tests pass with tar and cp due to xattr used for striping info being stored and then restore improperly. Also be aware that Ned views the layout code as his kitchen and as he stated "to many cooks in the kitchen" is a bad thing.

--- Message (Andreas Dilger, 2015-12-23 00:11:05.000000000) ---

Well, you can't land code to the main Lustre repo and then not maintain it but not allow anyone else to try and fix it either.  I'd be equally happy if Ned takes over this patch and gets it into shape for landing, but I haven't seen that in the 13 months since I first pushed it.

At this point I'd consider llapi_layout_* to be fair game for anyone that wants to work on it, within reason of course, and I'll try to make sure that Ned is added as a reviewer for any patches affecting this code.

Patch Set 2:

I think the major shortcoming of llapi_layout_* is that there isn't actually any code using it today, in particular "lfs setstripe" and friends all need to be converted over to using this API instead of continuing to maintain (and enhance, see http://review.whamcloud.com/13242) the older API.  Otherwise, we'll have two non-overlapping sets of functionality that will need another major effort to reconcile.
