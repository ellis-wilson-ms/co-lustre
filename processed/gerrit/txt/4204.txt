--- Message (Andreas Dilger, 2012-10-06 19:25:56.000000000) ---
Patch Set 1: Do not submit

I'm not sure what the intent of this change is, but it is incorrect.  The value of the OBD_CONNECT_LIGHTWEIGHT is 0x1000000000000ULL, as defined in lustre_idl.h, and this file is verifying it correctly. I was surprised and concerned when I saw this patch, since errors in the OBD_CONNECT flags could cause serious interoperability problems. 

The wirecheck.c file is used to verify the protocol constants do not change, since this is not easy to test with a single version of Lustre, as both the client and the server could have the same (wrong) value for a constant. Such changes would only show up in interop testing, which we don't do for every build.

In any case, two other comments about this patch:
- there is no such thing as a completely trivial patch, as this demonstrates. 
- "trivial fix" in no way explains what this patch does, so even if it were a valid patch, it wouldn't be accepted because there is no explanation of why it is changing, and why it was broken before, (and in exceptionally good commit comments) which patch introduced the problem originally

Lest I be over reacting to what might be a test patch submitted for some reason that is not apparent here, if this IS a test patch it is a very good idea to also put that in the commit comment, and potentially mark it with:

 Test-Parameters: fortestonly

So that it doesn't burn testing cycles needlessly.
