--- Commit Message ---
LU-12092 llite: Allow EX lock for layout refresh

when a client sends a write intent to the server to instantiate a new
layout component, the server handles the write intent and grants an EX
lock on the layout to the client.

Currently, the client ignores this lock when refreshing the
layout, and so it requests a new lock from the server,
requiring several round trips to cancel the existing lock.

The client can just use the EX lock to refresh the layout.

In a simple VM benchmark of time to do 1K writes to new PFL
components, this improves performance by ~12%.
