--- Message (Andreas Dilger, 2025-03-22 17:58:00.000000000) ---
Patch Set 1:

I was thinking about this some more, and to have useful semantics for the "undelete" flag shown by "lsattr" it would make sense that if the global "llite.*.enable_trash_can=1" is set, then it will virtually OR "FS_UNRM_FL" to every file|dir attribute on the client. 

If users explicitly set "chattr -u" it will set the NOTRASH flag on the inode, which will no longer show the "FS_UNRM_FL" for that file. 

The main question is whether we should take over the ldiskfs "FS_UNRM_FL" to actually mean "FS_NOTRASH_FL" (ie. the opposite of the documented meaning?  Maybe with a superblock flag to mean "the FS_UNRM_FL bit has the opposite meaning"?
