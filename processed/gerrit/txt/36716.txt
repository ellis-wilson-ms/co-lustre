--- Commit Message ---
LU-12036 ofd: add "no_precreate" mount option

Add a mount option to disallow object creation on the OST.  That
allows an OST to be mounted by the administrator without it being
immediately available for use by clients/applications.  This may
be useful if the OST needs to be added to a specific pool first,
or if it is being debugged or similar.

Mount option can be disabled with the obdfilter.*.no_precreate
tunable parameter.

I'm having second thoughts about naming this option "no_precreate", because I want to allow the same option to be added to the MDT mount, so that the administrator has the option to mount an MDT but it cannot be used until the administrator enables it.

Strictly speaking, MDTs don't "precreate" anything on the peer MDTs, so that is a bit confusing.  It wouldn't be terrible to have two mount options that mean the same thing "no_create" and "no_precreate"?
