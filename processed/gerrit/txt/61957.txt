LU-19015 llog: logic for skipping a zeroed record

For ENOSPC errors during dt_write() and threads races, the changelog
could have a sparse file with zeros inside. The current processing
logic skips records for the next chunk.
The patch adds the ability to skip only zeros in the buffer and start
from a valid record.

## Summary

**Fix Applied:** Removed the unused variable declaration `bool refresh_idx = false;` from the `llog_process_thread` function in `lustre/obdclass/llog.c`.

**Rationale:** 
The applied patch (commit 1c0881ab318d3cc8a5d92b8f81db8a47e1627bee) modified the error handling logic in the `llog_process_thread` function. The original code used a `refresh_idx` variable to control flow when encountering invalid records, but the new implementation replaced this logic with a more sophisticated approach using `llog_skip_gap()` function and direct manipulation of `cur_offset` and `index` variables.

The patch removed all the code that used `refresh_idx` but forgot to remove the variable declaration itself. Since the compiler is configured to treat warnings as errors (`-Werror=unused-variable`), this caused a compilation failure.

The fix simply removes the unused variable declaration, which resolves the compilation error without affecting the functionality of the new logic introduced by the patch.
