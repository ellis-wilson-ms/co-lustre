--- Message (Andreas Dilger, 2016-10-18 02:00:16.000000000) ---
Patch Set 1:

Alex, the root driver for this patch is because we are doing RPCs to the FLD server in the middle of an open transaction, which can cause deadlocks. Di created a patch at my request (http://review.whamcloud.com/23111 "LU-8250 osd: add journal info check") to catch cases where an RPC is sent with an open transaction, and it caught a few places doing this.

My thought is that we could do the FLD lookups during declare in advance of starting the transaction, and that should avoid any deadlock?  Even if we don't know the exact FID, we should at least know the right SEQ numbers. In extremely rare cases when the SEQ wraps and the next SEQ is on a different server, we might send an RPC with the transaction open, but as long as this doesn't trigger an LASSERT() we should be fine.
