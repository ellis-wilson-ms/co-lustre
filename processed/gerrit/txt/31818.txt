LU-10822 llapi: correct snprintf() buffer usage

The current code allocates a large buffer for fp_lmd, and
then uses the address of lmd_lmm inside fp_lmd to snprintf
in to this buffer in get_lmd_info(), cb_getstripe(), and
ct_md_getattr().

This causes the new GCC fortify source protections in glibc to
report a buffer overflow, because the actual size of lmd_lmm is
not large enough for all filenames (up to 255 chars.  The solution
is to make explicit that we are writing to a buffer (of unknown
size) rather than just use the address of a defined struct member.
