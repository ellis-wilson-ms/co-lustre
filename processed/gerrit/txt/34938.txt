--- Commit Message ---
LU-12045 tests: honor EXCEPT tests when using ONLY list

The Lustre test framework allows a user to specify a subset
of tests to run using the ONLY parameter or --only flag.
The test framwork also allows the user to specify a list of
tests to skip using the EXCEPT or ALWAYS_EXCEPT parameters.
By default, if the ONLY parameter or --only flag is used,
the EXCEPT and ALWAYS_EXCEPT lists are ignored.

Add a flag to auster, -H, and an environment variable,
HONOR_EXCEPT, to skip the tests on the ALWAYS_EXCEPT,
EXCEPT and SLOW lists when using the ONLY/--only parameter.

Test-Parameters: trivial
Test-Parameters: envdefinitions=ONLY="40-43" testlist=sanity
Test-Parameters: envdefinitions=ONLY="40-43" austeroptions=-H testlist=sanity
Test-Parameters: envdefinitions=SLOW="no",ONLY="27" testlist=sanity
Test-Parameters: envdefinitions=SLOW="no",ONLY="27" austeroptions=-H testlist=sanity
Test-Parameters: envdefinitions=SLOW="yes",ONLY="27" testlist=sanity
Test-Parameters: envdefinitions=SLOW="yes",ONLY="27" austeroptions=-H testlist=sanity

I think this patch works correctly and is ready for review. Here’s what I think the testing shows:
Test-Parameters: envdefinitions=ONLY="40-43" testlist=sanity 
Using ONLY list that includes a test on the ALWAYS_EXCEPT list is executed. This is the existing behavior and is expected. Results at https://testing.whamcloud.com/test_sets/28c71b64-8e1c-11e9-abe3-52540065bddc

Test-Parameters: envdefinitions=ONLY="40-43" austeroptions=-H testlist=sanity 
Using ONLY list that includes a test on the ALWAYS_EXCEPT list and specifying the auster –H flag now does not run the test that is on the ONLY list and on the ALWAYS_EXCEPT list. This is the desired new functionality. Results at https://testing.whamcloud.com/test_sets/28d4082e-8e1c-11e9-a77a-52540065bddc

Test-Parameters: envdefinitions=SLOW="no",ONLY="27" testlist=sanity 
sanity test 27m is on the SLOW list. When using the ONLY list, the SLOW list is ignored and sanity 27m is run. The test is actually skipped due to a skip_env() condition. This is the expected default behavior. Results at https://testing.whamcloud.com/test_sets/272724dc-8e1e-11e9-8730-52540065bddc

Test-Parameters: envdefinitions=SLOW="no",ONLY="27" austeroptions=-H testlist=sanity 
Using the auster –H flag, the SLOW list is now honored using the –H auster flag which is the desired behavior. Results at https://testing.whamcloud.com/test_sets/dee6da28-8e1d-11e9-a77a-52540065bddc

The last two results show that the –H flag does not change the behavior when we ask for SLOW tests to be run (SLOW=yes) which is the desired action.
