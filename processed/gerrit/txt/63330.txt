## Summary of the Fix

### Problem
The patch introduced compile errors in `lustre/tests/llapi_nodemap_test.c`:
1. **Unused variable warnings**: Three static variables (`mgs_device`, `mds_device`, `oss_device`) were defined but not actually used
2. **Compiler flag error**: `-Wno-gnu` flag not recognized (this is a separate build system issue, but the unused variables were being caught because warnings are treated as errors)

### Root Cause
The test file defined three static variables at file scope:
```c
static int mgs_device = -1;
static int mds_device = -1;
static int oss_device = -1;
```

However, these variables were never actually used:
- The `*_device_accessible()` helper functions call `llapi_get_mgs_device()`, `llapi_get_mds_device()`, and `llapi_get_oss_device()` which take **no parameters** (they manage device IDs internally with their own static variables)
- Only `test1()` referenced `mgs_device` when calling `llapi_name2dev(__func__, "MGS", mgs_device)`, but since `mgs_device` was always `-1` and never modified, it was effectively just a constant

### Solution
1. **Removed the three unused static variables** (lines 23-25)
2. **Updated the call in test1()** to pass `-1` directly instead of the removed `mgs_device` variable

### Rationale
The fix is correct because:
- The `llapi_get_*_device()` functions maintain their own internal static device ID cache (as seen in the implementation in `liblustreapi_ioctl.c`)
- The test file doesn't need to track device IDs separately
- Passing `-1` to `llapi_name2dev()` is the correct way to indicate "use default device for ioctl" according to the implementation
- This eliminates the unused variable warnings while maintaining the same functionality
