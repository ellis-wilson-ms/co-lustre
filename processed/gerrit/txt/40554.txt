LU-14116 autoconf: check if DES3 enctype is supported

krb5 releases 1.18 and later completely remove support for
all DES3 enctypes (des3-cbc-raw, des3-hmac-sha1, des3-cbc-sha1-kd).

This patch adds HAVE_DES3_SUPPORT to check if DES3 enctype
is supported.

Patch Set 2: Code-Review+1

I would be happy to remove DES support from the client completely at this point, because DES encryption is no longer secure at all. I had wanted to do this previously, but was convinced several years ago to keep it for compatibility reasons. However, now that the libraries are also removing it, and the CPUs have built-in AES instructions, I don't see any value to keeping this compatibility at all.

+1 because it is partly removing this support, but +2 if is removed entirely.
