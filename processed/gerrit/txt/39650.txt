LU-13883 lnet: Refactor lnet_discover_peer_locked

Callers of lnet_discover_peer_locked() assume that the peer NI
provided as an argument cannot change. However, it is possible that
discovery will delete the peer NI for some NID and create a new one
(see lnet_peer_add_nid()). Rather than have every user of discovery
handle this situation separately we instead refactor
lnet_discover_peer_locked() to deal with this situation for users of
discovery.

Changes to lnet_discover_peer_locked():
 - Now takes an lnet_nid_t that we wish to discover and it returns an
   lpni for the specified NID.
 - Now takes an additional argument that specifies whether it should
   add the LNET_PEER_FORCE_PING and LNET_PEER_FORCE_PUSH prior to
   queuing the peer for discovery.
 - On each iteration of the discovery loop we lookup the peer NI for
   the specified NID after we've been woken by the discovery thread.
   The next loop iteration will work on the (possibly new) peer NI
   and (possibly new) peer.
 - When we exit the discovery loop we lookup the peer NI for the
   specified NID and check whether the peer has changed. If so, we
   repeat the discovery loop to see if the new peer needs to be
   discovered.
 - Returns the lpni from the last lookup or ERR_PTR on error.

Callers are adjusted as follows:
 - lnet_initiate_peer_discovery()
   - If the peer NI returned by lnet_discover_peer_locked() is
     different than the one specified by our caller then queue the
     message on the new peer's lp_dc_pendq and add the new peer to
     the discovery request queue. A new function,
     lnet_peer_queue_message() is introduced to accomplish that task.
     If the new peer is already up to date then the discovery thread
     should simply re-send the queued message.
 - lnet_discover()
   - Lookup existing peer for specified NID. If one is found then set
     FORCE_PING flag to ensure we ping it. Otherwise, just call
     lnet_discover_peer_locked() since we know it will trigger
     discovery (we don't need to create the peer first or clear
     LNET_PEER_NIDS_UPTODATE flag on existing peer).
 - lnet_check_routers()
   - Leverage lnet_discover_peer_locked() to perform the lpni lookup
     (plus discovery). We pass 'true' for the 'force' flag to
     lnet_discover_peer_locked().
Misc:
 - lnet_initiate_peer_discovery()
   - Reordered the pre-condition checks for discovery so that the one
     most likely to be true (lnet_peer_is_uptodate()) is checked
     first.
   - Refactor lnet_peer_gw_discovery() into
     lnet_peer_needs_discovery() to check all relevant lp_state
     flags without needing to acquire the lp_lock spin lock multiple
     times.
 - lnet_find_peer()
   - lnet_find_peer_locked() introduced to provide callers that are
     already holding a net lock (lnet_discovery()) a way to lookup
     peer directly without first releasing their net lock.
