--- Commit Message ---
LU-15873 osd: Make mounting with "-o rdonly_dev" work

When "rdonly_dev" flag is passed at mount time to
MDS/OSS. This allows MDS/OSS to be mounted read-only.
This read-only is granted by dm-flakey's "drop writes"
switch. That is, it is not "pure" read only but only
discards the writes to the device. For ldiskfs-osd
this was always returning "EOPNOTSUPP". This patch
removes the check where EOPNOTSUPP is returned and
allows the mount to continue.

Before Patch:
~~~~~~~~~~~~~
$ mount.lustre /dev/mapper/mds1_flakey /mnt/lustre-mds1 -o rdonly_dev
mount.lustre: mount /dev/mapper/mds1_flakey at /mnt/lustre-mds1
failed: Operation not supported

After Patch:
~~~~~~~~~~~~
$ mount.lustre /dev/mapper/mds1_flakey /mnt/lustre-mds1 -o rdonly_dev
$ mount | grep lustre | grep mds
/dev/mapper/mds1_flakey on /mnt/lustre-mds1 type lustre
(ro,svname=lustre-MDT0000,rdonly_dev,mgs,osd=osd-ldiskfs,user_xattr,
errors=remount-ro)

(1 comment)

I don't think we should revive the dev_rdonly mount option, as it is *not* the same as s normal "ro" mount, because "dev_rdonly" pretends that writes are succeeding, but throws away the data. I don't think that should be mixed with "ro", since that might lead to data loss if someone is accidentally mounting with "-o ro" but not getting an "-EROFS" error back.
