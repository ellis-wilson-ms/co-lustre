LU-3717 llite: check whether parent is NULL in ll_encode_fh()

Since Linux kernel 2.6.39, the open by handle system calls
have been added. In fs/fhandle.c, exportfs_encode_fh() is
called with connectable=0, which makes parent=NULL passed
to ll_encode_fh(). This patch checks the value of parent
in ll_encode_fh() before running ll_inode2fid(parent)
so as to avoid the ASSERTION( inode != ((void *)0) ) failure.
