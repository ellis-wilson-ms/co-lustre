--- Commit Message ---
LU-19343 utls: appropriate error message based on the quota type

The lfs_setquota function where all quota types (user, group, and
project) were using the same error handling path that always
displayed "invalid project id" regardless of the actual quota type
being processed.

This is fixed to shows the appropriate error message based on the
quota type:

"invalid user id" for -u (user quota)
"invalid group id" for -g (group quota)
"invalid project id" for -p (project quota)
