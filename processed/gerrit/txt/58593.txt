--- Commit Message ---
LU-13726 build: fix checkpatch fix matching

When the spelling.txt suspect string is a regexp, then the matched
$typo string is different from the regexp, and is not a valid key
in the $spelling_fix associative array. This generates an error:

    WARNING: 'foo' may be misspelled - perhaps 'CHECKPATCH ERROR'

because $typo is not found in $spelling_fix and $typo_fix is empty.

If $typo_fix is empty after lookup, iterate the $spelling_fix keys
to find the one matching $typo so that it can be used as $typo_fix.

Avoid length warnings from DNAME usage, improve spelling suggestion.

Yes, it is the "%.*s" entry, which is matching anything with "%s" in it. It should be "%\.\*s" to match the literal '.' and '*' for strings that may not be NUL-terminated. 

I've fixed that in the spelling.txt in this patch, but the problem with the regexp matching still remains. The issue is that "$typo" is the matched string, but not the regexp that is needed to find the $typo_fix in the spelling_fix array. 

What is still needed is some Perl-fu to iterate the spelling_suspects array, find the regexp that matched the $typo and then use that to look up the typo_fix.
