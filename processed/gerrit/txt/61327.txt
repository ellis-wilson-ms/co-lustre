--- Commit Message ---
LU-19370 lfsck: don't use ptlrpc_thread for LFSCK

Lfsck may start multple threads: the master engine thread, and the
assistant threads for every LFSCK component. Use native kthread
instead of ptlrpc_thread to manage start/stop of these threads.

- kthread_stop() and kthread_should_stop() are used for stopping
  thread, and wake_up_var()/wait_var_event() ared used for thread
  communication.
- change CFS_FAIL_TIMEOUT() in lfsck loop to wait_var_event_timeout(),
  because the latter checks kthread_should_stop() to avoid timeout on
  stopping.
- add LAD_STOPPED flags because some places need to wait for
  assistant thread stopped
- add lfsck_instance.li_master_ready to indicate master engine has
  called lfsck_prep(), so subsequent slave join won't fail (e.g.
  rbtree is not initialized yet)
- add kthread_should_stop() check in more loops to avoid stopping
  timeout
