--- Commit Message ---
LU-11276 ldlm: rework ll_md_have_lock()

- move main cycle of bits matching from ll_have_md_lock() to
  the ldlm_resource_match_bits() which does everything in one
  pass.
- use LDLM_FL_MATCH_LOCK in m_lock_match() to distinguish bits
  matching through all resource locks and a single lock search
  in mdc_lock_match() and lock_matches()
- don't stop on the lock we are matching against but continue
  search to check all locks in IBITS queue.

Patrick, the main user of ll_have_md_lock() is ll_lock_cancel_bits() which is part of blocking AST processing. For each bit set it does some canceling job, but before doing that it checks if there is an other lock with the same bit set, so canceling job can be skipped. So purpose of that is to make sure we are the last user of that particular lock bit. So its task just to drop bits duplicated in some other lock.
