LU-7880 ptlrpc: expand obd_statfs size by 48 bytes

Expand struct obd_statfs size by 48 bytes.

To return useful OSD IO statistics to the client,
8 new fields are needed in obd_statfs (prefer some 64-bit),
while there are only 7 32-bit fields are available.

Add reserved fields to obd_statfs to expand the size of
this struct and run interop tests to verify operation.

Also reserve the OS_STATFS_IOSTATS = 0x400 flag value.

Test-Parameters: testlist=sanity serverversion=2.17 env=SANITY_EXCEPT="27D 27X 127f"
Test-Parameters: testlist=sanity clientversion=2.17 env=SANITY_EXCEPT="60b 77k"

Patch Set 2:

I added in the OS_STATFS_IOSTATS flag here as well, so that this can be filled in by the server when the stats fields are valid, and the client can check for this.

The client should send OS_STATFS_IOSTATS when it is requesting the added stats, just like OS_STATFS_SUM, so that servers don't spend time fetching this info for 99.9% of the normal STATFS RPCs that are sent.

--- Message (Andreas Dilger, 2025-06-12 05:06:46.000000000) ---
Patch Set 2:

We don't use version number checks for any client/server interoperability, since that is fragile and prone to errors if patches are backported to maintenance releases. 

To access these extra fields, the server should check the OS_STATFS_IOSTATS flag in the request, and also set it in the reply. There is a helper function to shrink the reply buffer, with a useful name like "ptlrpc_shrink_reply_buf()" or "ptltpc_reply_buf_shrink()" or similar, but I can't check it right now since I'm just on my phone. It should be possible to find it in lustre/ptlrpc/ or usage in the lustre/mdt/ code.
