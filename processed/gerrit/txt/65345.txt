LU-20087 tests: add test_27ad for nodemap range offset

Add test_27ad to sanity-sec.sh to verify that a file created
through a nodemap with an ID range offset has its UID/GID/PROJID
stored in the filesystem with the offset applied.

The test activates nodemap if needed, creates an offset nodemap
for the client (offset_start=100000), creates a file and sets its
projid as UID=ID0 (500) through the nodemap so the server stores
FS ID = ID0+100000 = 100500, then removes the nodemap and verifies
the client sees the raw FS IDs confirming the offset was applied.

The stack_trap for nodemap_del uses "|| true" (not "; true") so
that set -e in the EXIT trap does not fire when nodemap_del returns
non-zero because the nodemap was already removed in the test body.
