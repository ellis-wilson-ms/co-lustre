--- Commit Message ---
LU-17323 lib: l_wait_event occasionally clears sigpending

l_wait_event() calls recalc_sigpending() which can clear
TIF_SIGPENDING if a signal was processed in a parallel thread
and is no longer pending.

If this l_wait_event() is called from the fork cleanup part,
it's possible that fork() is going to return ERESTARTNOINTR,
however, if TIF_SIGPENDING is cleared, this errno can leak
to userspace while a syscall restart is expected.

Make sure TIF_SIGPENDING is restored if it was set on entry
to l_wait_event().
