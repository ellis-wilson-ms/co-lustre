--- Message (Andreas Dilger, 2013-12-16 08:20:06.000000000) ---
Patch Set 1:

Niu, I think it makes sense to have a limited significant digits for larger numbers, so I think "%5.4g" is just right:

 int main(int argc, char *argv[])
 {
	printf("%5.4g\n", (double)1.4404);
 	printf("%5.4g\n", (double)10.444);
	printf("%5.4g\n", (double)710.444);
	printf("%5.4g\n", (double)1030.444);
 }

Gives:

  1.44
 10.44
 710.4
  1030

Which I think is just right, and avoids adding complexity to the code.  This can be used to print values between 1.0 and 1024 MB/GB/TB easily.
