LU-19469 utils: tool to create or analyze a sparse file

Develop a tool to create a sparse file with specified sparity
and block size, or analyze a given sparse file to display the
hole distribution and sparsity ratio.

It can output the total hole count, total hole bytes, sparsity
ratio, min/max hole size, and min/max hole range and hole size
histogram.

Add sanity/test_857a to verify the sparsity tool.

## Summary

I have successfully fixed all the compile errors in the `sparsity.c` file. Here's what the fix was and the rationale:

### **The Problem:**
The compile errors were caused by format string mismatches between the printf format specifiers and the actual data types being passed:

1. **`__u64` vs format specifiers**: The code used `__u64` types (which are `long long unsigned int`) but tried to print them with `%lu` (expecting `long unsigned int`) or with `PRIu64` macro without proper casting.

2. **Double vs integer format**: One case tried to print a `double` value using `%d` (integer format).

### **The Fix:**
I made the following changes:

1. **Cast `__u64` values to `uint64_t`**: Added explicit casts `(uint64_t)` before all `__u64` variables when using `PRIu64` format specifier. This ensures the format macro matches the actual data type.

2. **Fixed double format**: Changed `printf("%d\n", (double)...)` to `printf("%.2f\n", (double)...)` to properly format the floating-point sparsity ratio.

3. **Consistent format usage**: Used `PRIu64` consistently for all 64-bit unsigned integer values with proper casting.

### **Rationale:**
- **Type safety**: The `PRIu64` macro is designed to work with `uint64_t` types, not with kernel-specific `__u64` types. By casting to `uint64_t`, we ensure compatibility.
- **Portability**: This approach works across different architectures where `long` and `long long` might have different sizes.
- **Compiler compliance**: The fix eliminates the `-Werror=format` warnings by ensuring exact type matching between format specifiers and arguments.

The changes maintain the original functionality while fixing all format string compilation errors. The tool will now compile successfully and produce the same output as intended.
