--- Commit Message ---
LU-1411 obdclass: compatibility for kernels < 2.6.18

lustre_get_sb() conditionally calls get_sb_nodev() with a struct
lustre_mount_data2 * for kernels 2.6.18 and newer, and a void * data
pointer for older kernels.  However, lustre_fill_super()
unconditionally casts its void *data argument to a struct
lustre_mount_data2 *, which causes a crash when we deference into it
on older kenels. To fix this, add conditional code to support older
kernels.

--- Message (Andreas Dilger, 2012-05-17 05:08:18.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

This check is only needed for kernels _older_ than 2.6.18, so this patch shouldn't be landing to master (where we are actively removing support for kernels < 2.6.18).

I verified in upstream kernel git that the change to 5-arg get_sb() was present in vanilla 2.6.18, and exists all the way to 2.6.38.  In 2.6.39 the get_sb() method disappears completely, and is replaced with "->mount()", but that is something to fix once we port to a newer kernel.  There may already be patches for this in Gerrit, since people are already working on 3.1 and 3.3 kernel support.

I think this patch should be abandoned.
