--- Commit Message ---
LU-15425 utils: 'lfs migrate' can accept FIDs

Enhance 'lfs migrate' command to accept FIDs as argument.

If --lustre-dir=LUSTRE_MOUNT_POINT and --fid are specified at
the same time, then FIDs are specified on command line, or from stdin,
or by --files-from=LIST_FILE argument instead of file paths.

If there is only one mounted Lustre file system,
option --lustre-dir=LUSTRE_MOUNT_POINT is not necessary.

I think this is generally good.  Though I *wanted* to suggest a number of optimizations for handling FIDs and lustre_dir_fd more directly, that should be done in a later patch.

The main issue I saw was changing the "copy" operation to use "lustre_dir" instead of "fname", which I think will be subtly broken if the file does not have the same layout as the parent directory.

The second issue is that processing should print an error and continue processing files if one file has an error (like "tar", "rsync", "cp", "rm", etc.).  Otherwise it would be necessary to edit the input file to remove all entries before the failed one, then restart.
