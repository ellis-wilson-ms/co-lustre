--- Commit Message ---
LU-0000 ldlm: transfer LVB data for Active Writer locks

Active Writer locks with mirror error information need to
transfer their LVB data to the server during cancellation
so the server can update its active writer count and access
the mirror error information.

This patch implements LVB transfer for Active Writer locks
by:

1. Adding ldlm_should_use_cancel_lvb() to detect Active
   Writer locks that need LVB transfer
2. Modifying ldlm_cli_cancel() to use ldlm_cli_cancel_req_lvb()
   for Active Writer locks instead of regular cancellation
3. Adding server-side support in mdt_reint_open() to extract
   and process LVB data from Active Writer lock cancellations

The LVB transfer ensures that mirror error information
attached to Active Writer locks is properly communicated
to the server during lock cancellation, enabling correct
active writer count management and error handling.
