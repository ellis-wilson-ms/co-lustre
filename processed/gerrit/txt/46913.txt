LU-15658 lod: revert "LU-14480 pool: wrong usage with ost list"

This reverts commit b384ea39e593cda1ac4d6fb8b955d0c7d1a1f67b.

This patch creates inconsistency between "lfs setstripe -i N" and
"lfs setstripe -o M,N" when operating in a directory that has a 
default pool set, but specified OST index is not in the pool. The
patch allows the "-o M,N" case to be accepted, but "-i N" is not.

Also, it unconditionally removes the pool name from the layout,
even if the specified OST index is in the pool.
