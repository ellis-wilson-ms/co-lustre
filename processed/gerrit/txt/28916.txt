LU-9964 ldlm: Prevent multiple group locks

There are two ways in which we can get multiple group locks
on the same resource (which is invalid because it breaks
various requirements for page writeback):

1. Multiple requests can go to the server at the same time
and the server will grant them (as they are compatible)
Solution: Wait in OSC layer for compatible group lock
requests to be granted
2. When being destroyed, group locks must eventually stop
matching new i/o so they can complete writeout and
destruction.  When this happens (by setting cbpending in
ldlm_cli_cancel_local), another request can be sent to the
server and be granted before the lock finishes writing out
dirty pages.
Solution: Wait (in ldlm_lock_match) for any group locks
with no references to be done (LDLM_FL_BL_DONE), so
cancellation is complete and only freeing remains to do.
At this point, pages are written out and the lock is no
longer associated with a resource.  This makes it safe to
request a new lock.

More details about this (including why it can't be solved
server side) are in LU-9964.
