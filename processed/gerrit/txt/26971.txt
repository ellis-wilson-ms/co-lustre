--- Message (Christopher J. Morrone, 2017-05-24 18:46:11.000000000) ---
Patch Set 6:

Amir, I don't think that is the kind of approach that we should be recommending to Lustre users.  It means that "lnetctl export" would need to be _very_ careful to only print values that where actually manually set, and not include any values that are system defaults.  From the examples that you've shown, the export command seems to be very verbose, not working that way at all.

If users adopt your approach, they will wind up locking in Lustre  default values in their configuration file that they otherwise would have left up to Lustre.  We _want_ users to allow Lustre to change values with Lustre upgrades in all of the cases where the user doesn't explicitly need to set their own value.  But this will break that normal pattern.
