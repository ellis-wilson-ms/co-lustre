--- Commit Message ---
LU-5304: no need to zero out the allocated memory when GFP_ZERO is set

The kernel will zero out the memory allocated when GFP_ZERO is set,
so no need to zero it out again.

It is possible the original intent was to memset the allocated memory
if GFP_ZERO was not set. However the only two existing callers of this
function already set GFP_ZERO.

--- Message (Liang Zhen, 2014-07-10 04:00:13.000000000) ---
Patch Set 2:

the reason we have this bug is because old libcfs didn't use GFP flags, instead it had self-defined CFS_ALLOC_* flags, then translated it to GFP flags, however, GFP_ZERO was not translated for some historical reason, and GFP_ZERO was never passed into kmalloc. 

After we removed  all CFS_ALLOC_* and replaced them with GFP flags, GFP_ZERO can be passed into kmalloc, so the memset should be removed.
