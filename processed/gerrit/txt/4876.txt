LU-1431 ptlrpc: PTLRPC_BRW_MAX_SIZE usage cleanup.

Clean-up the layering in current code by eliminating direct
usage of PTLRPC_BRW_MAX_SIZE macro outside of the ptlrpc
module. This should help us acheive "floating" max brw size
value across the cluster, which in turn should help with
4MB IO task.

Patch Set 7: Looks good to me, approved

Sergii, you are right that 32MB default LOV_DEFAULT_STRIPE_SIZE is probably too large...  I was thinking the default would be based on the cl_max_pages_per_rpc value, but that is a bit more tricky to implement.

Since this patch is not the one actually changing the PTLRPC_MAX_BRW_SIZE, it is not harmful to land this patch as-is.  The default stripe size should be changed in the following patch.

It probably makes sense to limit this to min(ocd_brw_size) returned from the OSTs or if that is difficult to implement then some fixed value like 1MB might be OK.  Since this value is easily changed at runtime, it probably makes sense to stick with the 1MB default for now.
