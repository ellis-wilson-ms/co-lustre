--- Commit Message ---
LU-18770 lfs: mirror extend '-N=' get correct mirror number

It does not count the existing mirror number correctly in mirror
extend with "-N=<total_mirror_count>" option, since the raw layout
format stores the actual (mirror_count - 1) in its mirror count field,
so that plain or non-mirrored PFL file which has 0 in the mirror count
field logically has 1 mirror in it.

This patch fixes this error.

Fixes: a796b10fa2 ("LU-16446 utils: specify total count for mirror extend")
