LU-10391 mdt: extend downcall to handle large nids

A new version of the downcall structure is created - with a new magic
number - which can hold large nids.

If all the nids used are nid4, then we use the original structure for
max backward compatibility.  Kernel accepts both formats.

Patch Set 7:

As the userland upcall and the corresponding kernel code all run on the same node, I am not sure it is worth maintaining compatibility with the old format. I mean, maintaining compatibility between clients and servers is already a full time job, do we want to also provide compatibility for nodes that are half-way updated (in this particular case, new kernel code running with old user-space l_getidentity, or the other way around)?

I think introducing a new MAGIC, and returning an error as soon as we get something else would be enough.

And there is also the question of the relevance of the "perm_downcall_data" related code, as raised by Andreas.
