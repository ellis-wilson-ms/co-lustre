LU-19888 utils: Filter & Print IOCTL request along with full command

Currently, IOCTL debug just prints full command integer number.
This patch filters out IOCTL integer number to IOCTL string and
adds to the the debug log for easy debug log reading.

This patch introduces no functional changes to IOCTL path.

Before:
$ lfs osts

  cmd=800866a8 arg=7ffce147942c

After:
$ lfs osts

  cmd=800866a8 (_IOR('f', 168, 8)) arg=7fff3073ab4c

## Summary

**Fix Applied:** Added `#include <linux/ctype.h>` to `include/linux/libcfs/libcfs_debug.h`

**Rationale:**
The patch introduced a new inline function `cmd2ioctl()` in `libcfs_debug.h` that uses the `isprint()` function to check if a character is printable (line 328). However, the header file didn't include the necessary header that provides this function.

In Linux kernel code, the `isprint()` function is declared in `<linux/ctype.h>`. Without this include, the compiler generates an "implicit declaration of function" error because it doesn't know about `isprint()`.

The fix adds `#include <linux/ctype.h>` to the header file's include section, making the `isprint()` function available to the `cmd2ioctl()` implementation. This is a standard pattern in kernel development where character classification functions like `isprint()`, `isdigit()`, `isalpha()`, etc., are provided by `linux/ctype.h`.
