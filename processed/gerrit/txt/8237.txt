LU-3974 llite: fix mkdir endless loop

Running on 3.11-rc4 kernel, I got below endless loop. It
turns to be that Lustre always saves the first page of a
dir inode mapping at index ~0UL. And after commit 5a720394
(mm: teach truncate_inode_pages_range() to handle non page
aligned ranges), truncate_inode_pages_range() _NO LONGER_
truncates the page that is sitting at index ~0UL.

Patch Set 1: I would prefer that you didn't submit this

(4 inline comments)

James, thanks for taking up this work of back porting the upstream kernel patches. I don't want to put roadblocks in your way, but I also want to make sure that we get all of the upstream patches in the Lustre tree. That means the commit comments need to have some extra info to allow tracking against the upstream kernel. It also makes sense to have some systematic tracking of which patches have been backported (wiki page with a list of upstream originated patches?), so that they are not missed, and there isn't effort spent on recreating the same solution.  

Bob, it isn't possible to use Linux kernel patches directly with the Lustre git repo because the pathnames are different, and some of the conditional code in the Lustre repo is not in the upstream code anymore. That said, I'd prefer to keep the patches as similar as possible. Any improvements in the ported patch would likely be lost, because the patch will not be re-ported to the upstream kernel.

It is better to make improvements in a separate follow-on patch and that will eventually be merged upstream as well.
