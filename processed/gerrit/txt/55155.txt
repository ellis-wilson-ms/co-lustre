LU-17000 utils: Capture MAJOR return value in int

ofd_access_log_reader IOCTL to retrive device major
presently returns an 'unsigned int'. However, MAJOR()
macro returns integer. This patch use int instead of
'unsigned int' to capture return from MAJOR() as it can
return a negative value.

Patch Set 3: Code-Review-1

I'm against storing this same value in two different variables.  This would appease the code checker, but doesn't actually fix the issue that the kernel is using one type and userspace another.

Instead, the kernel should just be changed to use "int" instead of "unsigned int".  AFAIK, this should not affect the ioctl number used (which is based on the variable size only), and I don't think there will be device numbers >= 2^31, so I don't think there is a functional problem either.
