--- Message (Neil Brown, 2022-05-16 01:00:35.000000000) ---
Patch Set 2:

I don't think there is any real consistency in views on comparing with NULL.  Some people definitely don't like it - it is extra noise.  Other people definitely do.  I doubt there are many people who would be upset by it in code that they themselves aren't responsible for.
My own view isn't entirely consistent.  With something like
  if (foo != NULL && foo->member > 12)
I would drop the "!= NULL" because it adds unnecessary noise.
When testing if an alloc failed, I would probably us "if (!foo)"
If walking a NULL terminated list, I would explicit test for NULL.
If there is any risk that the "!" might not be seen - maybe a fairly busy expression - I would prefer an explicit "== NULL".

A lot of the things that checkpatch reports are worth fixing - consistency is good.  But I think comparing with NULL should always be a judgement call.  Sometimes it is best to keep the NULL, sometimes is it only adds to the noise.
