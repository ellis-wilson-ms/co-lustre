--- Commit Message ---
LU-13004 osp: break read request into pages.

Rather than breaking up a read request into arbitrarily
sized (4K) pieces of memory in virtual address space,
break it up into pages (which might be 64K) and
use a kiov rather than kvec to manage them.

This is a step towards removing kvec suport and
standardizing on kiov.

I didn't think this change would affect the wire protocol at all.

For RDMA, I guess it is possible (simply because I don't know much about RDMA, so anything is possible), but I'd be surprised.

But did I miss something important?  Would these changes affect the bytes sent over the wire?

--- Message (Andreas Dilger, 2019-12-03 23:57:32.000000000) ---
Patch Set 4:

My main concern is that there had to be a reason why we introduced OUT_BULK_BUFFER_SIZE in the first place.  This code is only a few years old (Lustre 2.8), and we definitely know what PAGE_SIZE is, so my assumption is that it was there to isolate the peers from the different PAGE_SIZE.

That isn't to say your cleanup is necessarily wrong, but it is easy to miss changes like this if we are only testing on x86 clients/servers, or if the constant is changed on both the client and server at the same time, so I'd rather delay landing this a bit until we know what the impact is.
