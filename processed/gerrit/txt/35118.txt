LU-12410 lnet: Fix off-by-one in lustre_lnet_parse_nids

This function is allocating memory, via calloc, in a loop for each
entry in an array. If the calloc fails we need to free the memory
that we've allocated thus far. The number of entries that have been
allocated when we perform this check is equal to the value of the
loop iterator variable.

Also, all callers pass 'size >= 0' so we can simplify some of the
related code here.

There's a segmentation fault when '--nid' arguments do not contain
an '@' symbol. This is fixed by getting rid of
get_next_delimiter_in_nid and using strchr to find commas directly.

Finally, the while loop is a do-while in disguise. Reveal it for what
it is!

Test-parameters: trivial
