Neil, James,
I was looking at refreshing this patch, but (vaguely) recall that there is work underway in the upstream kernel to remove the use of PAGE_SHIFT and PAGE_MASK and PAGE_SIZE from the filesystem code.

If that is the case, it would make sense to directly replace cl_page_*() with the replacement functions from the kernel rather than landing this patch to introduce the PAGE_* constants, only to need a second patch that replaces them all with functions again.

--- Message (Neil Brown, 2021-09-14 02:03:50.000000000) ---
Patch Set 2:

I'm not aware of any attempt to remove PAGE_SIZE etc from fs code, and cannot easily find anything in email.

PAGE_CACHE_SIZE was removed in 2016, so I doubt you are thinking of that.
The "folio" work from Matthew Wilcox replaces 'struct page' with 'struct folio' in some contexts, but doesn't seem to be remove PAGE_SIZE etc from fs code.

So I don't know what might be to source of this vague recollection.

--- Message (Neil Brown, 2021-09-28 06:24:41.000000000) ---
Patch Set 2:

Re: the removal of PAGE_SIZE etc from filesystems.

This might be related to the introduction the "iomap" interfaces  - include/linux/iomap.h

iomap and netfs (which is a vaguely similar net library focussing on network filesystems) are mentioned here: https://lwn.net/Articles/869942/

We probably need to think about how these can be used by lustre at some stage, but probably not yet.
