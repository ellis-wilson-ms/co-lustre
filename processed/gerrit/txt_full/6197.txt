--- Message (Andreas Dilger, 2013-04-29 15:33:04.000000000) ---
Patch Set 2: I would prefer that you didn't submit this

I don't think this is quite correct.

The label parsing is a bit unpleasant, since it is possible to have a dash in the fsname itself... That is why all the scanning is done with strrchr() instead of strchr(). Also, the fsname/target separator may be either '-' or ':' depending on whether the target is registered or not.

It probably makes more sense to search for the end of the target (e.g. NUL string or whitespace) and then continue to use strrchr() to find the separator. 

It would also be good to add all of this to a comment block in the code so it is more clear.
