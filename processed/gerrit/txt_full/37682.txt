LU-13288 llite: avoid symbol_get() in ll_account_page_dirtied()

ll_account_page_dirtied() is hot path for buffer write and symbol_get()
is very heavy operation as it need preempt disabled or module_mutex which
hurts performance, see following benchmark numbers:

$ ior -w -t 1m -b 192g -e -o /scratch/testdir/file

c38ab030d0 LU-12904 build: account_page_dirtied is not exported

Max Write: 802.32 MiB/sec (841.30 MB/sec)

998a494fa9 LU-12861 libcfs: provide an scnprintf and start using it

Max Write: 2253.32 MiB/sec (2362.78 MB/sec)

We have a helper LB_CHECK_EXPORT to check wheather a function is
exported or not, which means symbol_get() could be avoided.

Notice, cgroup writeback support for newer kernel could be broken,
but that is not problem of this patch, we could address it in
seperated patch.
