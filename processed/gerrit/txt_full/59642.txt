LU-13643 llite: implement concurrent write lease for mirrors

This patch implements concurrent write lease functionality to
enable multiple concurrent writers on mirrored files for
immediate write operations to all mirrors.

Key changes:
- Add LL_LEASE_CWLCK lease mode for concurrent write access
- Implement server-side support for concurrent write leases
- Add automatic lease acquisition for mirrored files in
  ll_file_io_generic()
- Update userspace API to support new lease type

The concurrent write lease allows multiple clients to write
to different mirrors simultaneously, enabling immediate
mirror writes without the need for exclusive access.

Server-side changes:
- Add MDS_OPEN_CONCURRENT_WRITE flag handling
- Implement lease compatibility checks for concurrent writes
- Allow multiple concurrent write leases on same file

Client-side changes:
- Automatic detection of mirrored files during write operations
- Transparent acquisition of concurrent write leases
- Proper lease cleanup on file close

Userspace changes:
- Add LL_LEASE_CWLCK to lease mode enumeration
- Update lease acquisition/release functions
- Add support in liblustreapi
