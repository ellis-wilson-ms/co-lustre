LU-19179 o2iblnd: properly handle IB_DEVICE_SG_GAPS_REG

supported by Lustre we haven't been setting up
IBLND_DEV_CAPS_FASTREG_GAPS_SUPPORT. What was missed all
this time was IB_DEVICE_SG_GAPS_REG was renamed so update
the ko2iblnd to use the new name IBK_SG_GAPS_REG.
