LU-2246 mdt: fix issues related to SOM

1, if there is no LSM, ll_som_update won't need to update SOM
   attributes on MDS (see subtest 206 in sanity.sh)
2, if the file isn't a regular one, ll_setattr_raw should not
   open the file for truncation epoch (MF_EPOCH_OPEN), for it
   doesn't release the setattr request and the obd_import
   and obd_device won't be released (MDC with ST state will
   show up in 'lctl device_list')
3, mdt_som_au_close could be called during eviction, then there
   is no corresponding ptlrpc_request.
4, during closing a file(mdt_close), MDT should not require
   the client to send AU if the file has been unlinked.
