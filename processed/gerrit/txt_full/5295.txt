LU-2614 build: Modified lbuild script to build SR-IOV enabled OFED
This change will enable the compilation of Lustre using the
Mellanox OFED stack providing SR-IOV capability for:

  - RHEL/CentOS 6.5 and Linux kernel 2.6.32-431.20.5
  - Mellanox Connect-X2 and Connect-X3
  - HCA flashed with firmware version 2.9.1000
  - Virtual machines set-up with kvm-qemu >= 0.12.1.2
  - Fabric using subnet-manager version '4.1.5'

The SR-IOV feature allows the usage of virtualized functions
(VF) within an infiniband HCA, and to assign these VF to VMs.
At the moment the 'vanilla' OFED (www.openfabrics.org) doesn't
support this feature.

The change rely on Mellanox's OFED version 2.2.1.
It is necessary to edit the rpm specfile of the kernel-ib source
RPM, repackage the source RPM and recreate the OFED tar archive
to make the compiled content (kernel-ib, kernel-ib-devel)
fit into the Jenkins build environment. This change
handled by a separte ticket.

The former changes to lustre-lnet.m4 and o2iblnd.h became
superfluous since compat header files have been include
in the LNET source code now.
