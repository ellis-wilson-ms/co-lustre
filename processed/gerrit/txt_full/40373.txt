Patch Set 3: Code-Review+1

Neil, what is your plan for using/maintaining these ldiskfs patch series for mainline kernels?  Typically, we don't keep separate patch series for *every* kernel, just because it ends up with too many different patches that need to be updated when there is a bug found, or some new patch is added to the series.

If you are just doing these incrementally so that you can build/test the server code, one reasonable (IMHO) proposal would be to add the new kernel series, then delete the (next-)older series and any unused patches as you are moving to a newer kernel.  It might make sense to keep the series for LTS mainline kernels, since it is more likely those will be adopted by distro kernels.

> Neil, what is your plan for using/maintaining these ldiskfs patch series for mainline kernels? 

I had no concrete plan, but your proposal makes lots of sense.   I somewhat lost interest in these patches as they weren't getting any review and I had other things to work on, but if/when the support for newer kernels lands I'm happy to look at removing the support for earlier kernels.

Relatedly:  SUSE always builds KMP (kernel module packages) against the SP release, never against a maintenance update.  So it would suit me if, for SUSE at least, we didn't discard the SP kernel patches when adding support for a maintenance update.  No need for multiple maint update support through.
