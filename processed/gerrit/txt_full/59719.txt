--- Commit Message ---
LU-13643 ldlm: implement special aging for ACTIVE_WRITER locks

Active writer locks should not be cancelled immediately, so
clients can reuse them, but they should be cancelled very
quickly because their presence prevents immediate mirror
files from becoming fully redundant.

Track them specially and run LRU pruning aggressively when
they're present, so we can respect short age limits.

Default of 1 second.
