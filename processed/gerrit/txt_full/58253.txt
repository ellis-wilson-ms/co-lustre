(11 comments)

The test needs to confirm that the --no-resync mirror is marked 'stale' so that it cannot be used by clients.

I would also like there to be a second option (maybe in a second patch, if it is complex) that _tries_ to resync the file, but if the resync fails it will still attach the mirror as a stale copy as if --no-resync was used (LU-17531).  Maybe "--allow-stale" or similar?  That would be useful in case of trying to mirror a file that is constantly being modified, so that at least _some_ mirror can be created on the file, even if it is slightly old.
