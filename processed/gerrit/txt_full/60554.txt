LU-19239 tcu: keep oldest version when repeated deletion

If the same filename is repeatedly created and deleted within the
same parent directory, then the deleted files will have conflicts
when moved into the "pFID" directory in the Trash Can.

In this patch, we implement tunable option to select the repeated
deletion of same filename via:
"mdd.*.trash_can_delete_policy"
- "oldest": Keep the oldest version and destory the current one.
  This policy is only used for regular files or previous empty
  directory.
- "timestamp": The conflicting filenames are disambiguated by
  appending a timestamp to the filename;

And in this patch, we have implemented the simpest "oldest"
repeated deletion policy of the same filename for the regular
files or an empty dir.

Add sanityn/test_118Ba.
