LU-19536 osc: clean up dead and confusing grant code

Since osc_queue_dio_pages was split out (LU-13814), DIO
pages no longer flow through osc_queue_sync_pages.  The
DIO-specific grant reservation, dirty page accounting,
sync_io anchor setup, and ci_parallel_dio fallback in
osc_queue_sync_pages are all dead code.

Remove the dead DIO handling and add an LASSERT to catch
any future regression where DIO pages accidentally reach
this function.  Also simplify the oe_dio check in the
extent priority logic since oe_dio is never set here.

Inline __osc_unreserve_grant into its sole caller
osc_unreserve_grant_nolock - having two lockless
near-identically named functions for the same operation
is needlessly confusing.
