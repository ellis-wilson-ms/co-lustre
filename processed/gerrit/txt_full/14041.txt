--- Commit Message ---
LU-6356 tgt: handle sec context requests properly

When dealing with security context initialization requests,
no export is associated with the request yet, because these
requests are sent before *_CONNECT requests.
So the normal *_common_slice handlers will not be called,
given there is no reference to the target.

So add a special case for handling of SEC_CTX_INIT, SEC_CTX_INIT_CONT
and SEC_CTX_FINI requests in tgt_handle_request0() and
tgt_request_handle(), in order to directly send the reply.

Also remove tgt_sec_ctx_handlers because they become useless.

I tried to add tgt_sec_ctx_handlers to ofd_common_slice like in
0001-SNT-148-unsorted-fixes.patch attached to LU-6020, but I did
not manage to make it work.
The problem is that when dealing with requests with opc == SEC_CTX_INIT,
there is no export associated yet, because these requests are sent
before *_CONNECT requests (so req->rq_export is NULL).
As a consequence, we have no reference to the target, and thus we
are not able to retrieve handlers from slice.
I also tried to retrieve the target information by other means
than the export, but I was unsuccessful due to the fact that the
SEC_CTX request format does not contain the target uuid.

Unless there is another way to retrieve request handler, I would
tend to stick with my initial proposal.
What do you think?
