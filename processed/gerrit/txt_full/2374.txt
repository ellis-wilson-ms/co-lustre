LU-1191 nfsd4: Remove check for a 32-bit cookie

Fan Yong <yong.fan@whamcloud.com> noticed setting
FMODE_32bithash wouldn't work with nfsd v4, as
nfsd4_readdir() checks for 32 bit cookies. However, according to
RFC 3530 cookies have a 64 bit type and cookies are also defined
as u64 in 'struct nfsd4_readdir'. So remove the test for >32-bit
values.

In fact, such patch has been picked by new kernels. What we can do is to supply a solution to the Lustre user who want to re-export Lustre through NFSv4 against old kernel.

Since re-export Lustre through NFS is relative seldom case compared with using native Lustre client, it is reasonable to ask the Lustre users to patch the NFSv4 server for such purpose.

So my suggestion is that: only add the patch under our patch directory (do not add it to serise), and add changelog entry to tell Lustre users which patch(s) should be used if they want to re-export Lustre through NFS. Then the Lustre users can patch the kernel as needed by themselves.
