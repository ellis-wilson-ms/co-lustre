While this fixes the immediate problem, it does not fix similar problems in the future. It would be better if the DKMS.conf file was automatically regenerated at build time so that it included the actual list of modules to be built. Otherwise, since we do not build DKMS RPMs for every patch I suspect that you will have an ongoing series of problems building DKMS as modules are added and removed.
