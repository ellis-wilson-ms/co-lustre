--- Commit Message ---
LU-11204 obdclass: remove cl_object_put_last()

There is possible use-after-free in lu_object_put() right
after atomic_dec_and_lock() when reference is dropped and
no lock is taken but lu_object_header is being accessed.
The related code was added to solve bz22520 problem which
looks outdated now.

This patch removes whole problematic check along with
waitqueue wakeup and cl_object_put_last() assuming that
bz22520 problem doesn't exists anymore.
