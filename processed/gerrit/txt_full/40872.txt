LU-14172 mds: disable GETATTR_PFID feature

Remove the OBD_CONNECT2_GETATTR_PFID feature flag from the list
of supported MDS features so that clients do not try to send the
parent FID on each lookup.  This caused a performance regression
in directory "stat" operations in mdtest for striped directories:

    # lfs setdirstripe -c 4 /ai400x/mdt_stripe
    # lfs setdirstripe -c 4 -D /ai400x/mdt_stripe
    # salloc -p 40n -N 40 --ntasks-per-node=16 --allow-run-as-root
      mpirun mdtest -i 3 -p 10 -n 1500 -u -D -d /ai400x/mdt_stripe
