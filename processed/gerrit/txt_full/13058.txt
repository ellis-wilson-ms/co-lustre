LU-5969 lustreapi: change license from GPL to LGPL

This is an patch to update the copyright license of the liblustreapi
library from GPL to LGPL. This will simplify the use of liblustreapi
by third party applications (e.g. non-GPL HSM copytools, Lustre-aware
user applications or libraries, etc) and distribution of same in both
source and binary form.  In particular, liblustreapi.c, lustreapi.h
and supporting files and headers libcfs_kernelcomm.h, l_ioctl.c,
lustre_user.h, and lustre_ioctl.h will have their license changed.

Other files in the library (liblustreapi_hsm.c, liblustreapi_json.c,
liblustreapi_layout.c, liblustreapi_lease.c, lustreapi_internal.h)
already have explicit LGPL headers.

This patch requires signoff from all organizations that have copyright
on these files by virtue of making changes to the code, as recorded by
the commit records after the Lustre 2.0 release.

Copyright assignment was previously required for all contributions to
Cluster File Systems, Sun Microsystems, and Oracle for Lustre 2.0 and
earlier.  The Oracle Lustre copyright of the liblustreapi.c code was
later assumed by Xyratex/Seagate, so a signoff by Seagate is enough
to cover all earlier contributions.
