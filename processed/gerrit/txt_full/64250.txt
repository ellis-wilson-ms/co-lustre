--- Commit Message ---
LU-19932 osc: restore interruptible waits on client

Some interruptible wait conditions were lost in translation to
kernel wait primitives.  Change some wait_event_idle() calls back
to l_wait_event_abortable() so that user processes can be killed
if the server is unresponsive.

Simplify ldlm_completion_ast() to only call l_wait_event_abortable()
once instead of in two separate branches to avoid code duplication.

Simplify osc_extent::oe_rc usage to only hold 0 (success) or a
negative errno.  Storing the number of pages there isn't used for
anything, and it complicates its usage elsewhere.

Fixes: fa9c4d0fee ("LU-10467 lustre: use wait_event_idle() where appropriate.")
Fixes: d6ce546eb7 ("LU-10467 obdclass: convert waiting in cl_sync_io_wait().")
