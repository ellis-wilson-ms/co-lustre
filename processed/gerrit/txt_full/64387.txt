--- Commit Message ---
LU-19848 llite: don't use iov_iter_is_aligned()

Filesystems should probably not use iov_iter_is_aligned().
This function was originally only exported to be used in the
block layer. Once the only user was dropped, it was removed.
No filesystems ever adopted it. When testing on a 6.18
kernel with `gdsio`, this function incorrectly rejected
directIO that was previously accepted in Lustre 2.15. Hence,
remove this check. The other checks should be sufficient for
our needs.
