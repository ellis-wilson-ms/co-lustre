--- Commit Message ---
LU-11893 lnet: respect 'tcp_bonding' parameter.

Today Multi-Rail configurations treats each ethernet interface
as an unique LNet interface. You can bind several ethernet
interfaces by having them belong to the same LNet net. Before
Multi-Rail only a one to one relationship existed between LNet
nets and a network interface. The way socklnd worked around this
was with tcp bonding. Today this approach is not recommended
but can be enabled with the 'tcp_bonding' module parameter.
In that case each ni_interface[X] is used for this tcp bonding
mode. The problem is this configuration can be used independent
of the 'tcp_bonding' setting. Only allow this ksocklnd setup
when 'tcp_bonding' is enabled.

Test-Parameters: trivial testlist=sanity-lnet
