--- Commit Message ---
LU-10360 tests: test dynamic NIDs feature

Add five LUTF test cases to test the following:
1. Enabling/Disabling dynamic_nids module parameter.
2. Allow clients to continue using servers which have changed
   their IP address during a boot cycle.
3. Verify feature is disabled if dynamic_nids module parameter
   is not set.
4. Verify feature is disabled if the dynamic_nids module parameter
   is asymmetrically set.

(16 comments)

Having the comment block at the top to explain the test is definitely useful, though it remains to be seen how well people do at adding useful explanations to every subtest.

>>> The intent here is to avoid having to write a separate test plan. The comment block would be filled out with the test description. The LUTF provides a facility to export the description into a csv file. This can be imported into the wiki (or added to a doc file). This way we don't have multiple places which reference the test description. I think this would be easier to maintain. We can go back and update the test scripts and then the plan can be generated out of it. Of course that implies people have to update the test description in the script itself. Nothing comes for free :)

Is there some mechanism in LUTF by which subtests can be skipped for interop testing between different client/server versions?  That would be needed for both LNet and Lustre testing, since these tests will eventually run against 2.12 servers that do not have the dynamic_nids feature at all.

>>> Yes there is a similar mechanism as what currently exists. You can specify the list of tests you don't want to run and it'll exclude those. Currently it's specific tests that can be excluded. But LUTF can be modified to specify entire suites. Which could be useful. For exmaple b2_12 doesn't have UDSP, so we can exclude the UDSP completely.
