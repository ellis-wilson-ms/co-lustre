LU-6952 mount: Mount options parsing problem

The code which parse the mount option expect that the default
mount options come first and then the user given options. But
actually the default options are appended after the user given
option. Because of this user given options are cleared.
Following code fix this bug.

Added a conf-sanity test 104 to verify this fix.

The only way to test whether this patch works or not is to actually induce errors in fs. 
We cannot rely on the output of "mount": 
Following is the output of mount in two cases
1. "mount" without patch.
  [root@myserver tests]# mount -t lustre -o  errors=panic /dev/sdd /mnt/SDD/ 
  [root@myserver tests]# mount -t lustre
   /dev/sdd on /mnt/SDD type lustre (rw,errors=panic)

2. "mount" with patch
   [root@myserver tests]# mount -t lustre -o errors=panic /dev/sdd /mnt/SDD/ 
  [root@myserver tests]# mount -t lustre
   /dev/sdd on /mnt/SDD type lustre (rw,errors=panic)

In both the above cases it shows lustre is mounted with "errors=panic" but it is only with second case (i.e. with patch) the kernel actually gives panic when induced erros. In the first case, when errors are induced, fs will be just remounted with "ro" without panic.

For the "errors" mount provides three options:
1. remount-ro: This is the default option. We cannot use it to test whether user given optins are respected.
2. continue: This will simply ignore the error. 
3. panic: This will generate a kernel panic. I found only a third option (i.e. panic) which provides testable scenario but as it generates panic we may    not use it.
 Can you please suggest me any way I can test this without generating a kernel panic?
