LU-7816 quota: add default quota setting support

Similar function which is motivated by GPFS which
is friendly feature for cluster administrators to manage
quota.

Lazy Quota default setting support, here is basic idea:

Default quota setting is global quota setting for
user, group, project quotas, if default quota set
for one quota type,newer created users/groups/projects
will inherit this setting automatically, since Lustre
itself don't have ideas when new users created, they
could only know when these users trying to acquire space
from Lustre.

So we try to implement lazy quota setting inherit,
Slave firstly check if there exists default quota
setting, if exists, it will force slave to acquire
quota from master, and master will detect default
quota is set, then it will set this quota and also
return proper grant space to slave.

To implement this and reused existed quota objects
API, we try to introduce another type of
quota: default quota, this type of quota don't have
accounting object, so no edquota for it, clients Side
write should not be affected by this new types.

Lazy quota Limitations:
if default quota changes, previous inherited
quota setting won't be changed,they have been
persistent in the backend storage

Use lfs setquota -U/-G/-P to set default quota.
Use lfs quota -U/-G/-P <mnt> to show default quota
