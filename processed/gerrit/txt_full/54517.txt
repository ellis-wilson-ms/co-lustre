LU-17431 nodemap: allow nodemap_test_id on non-MGS nodes

Testing nodemap IDs on non-MGS nodes requires specifying the '-d'
option to 'nodemap_test_id'. This in turn sends the appropriate
ioctl to the MDS or OSS device.
On kernel side, we need to allow testing ids even if the nodemap
config file is not accessible (i.e. we are not on the MGS). This
dynamic nodemap is not stored on disk.

Test-Parameters: trivial
Test-Parameters: testlist=sanity-sec env=ONLY=71 mdscount=2 mdtcount=4 osscount=1 ostcount=8 clientcount=2
