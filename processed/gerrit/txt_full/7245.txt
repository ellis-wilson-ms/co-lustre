--- Message (Ned Bass, 2013-08-06 17:52:56.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

This new function is at odds with the llapi_layout API patch at http://review.whamcloud.com/5302.  In particular, the llapi_layout API was specifically designed to abstract away details like the size of a layout.  I'd prefer we focus on API improvements along those lines, rather than exposing more internal details through new public interfaces.  Otherwise, can we at least choose a different prefix for this function, and reserve llapi_layout for properly abstracted functions as in change 5302?
