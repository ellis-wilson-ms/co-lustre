--- Message (Andreas Dilger, 2014-07-10 15:28:51.000000000) ---
Patch Set 3: Code-Review+1

Note that if you use wait_for_completion_interruptible() the process can also be interrupted by <ctrl-z> or sigalarm or other signals. I don't know that anyone would want their IO to fail when they hit <ctrl-z>. 

That is why we created l_wait_event() - so that the process could not be interrupted while there was still a chance to complete the initial RPC, but the process became interruptible after the client had timed out. 

If CLIO will handle the resend of the write if it is interrupted, then wait_for_completion_interruptible() is fine, otherwise the current patch with wait_for_completion_killable() is safer - the user cannot expect that the IO is finished if the program is killed.
