LU-8418 libcfs: remove lnet upcall code

Removing lnet upcall infrastructure completely
as nobody uses it anymore. The upcall causes a delay
before calling BUG() and might even cause a hang
making getting a crash dump unreliable or containing
outdated info.

--- Message (Andreas Dilger, 2016-07-21 06:58:04.000000000) ---
Patch Set 1: Code-Review-1

(2 comments)

This patch does two different things, only one of which is described in the commit message.

The first is to check for the "NONE" keyword to mean no upcall is used, which is fine, though from your description in Jira it might also make sense to skip the upcall of it is unset, or if it references a non-existent file (not sure if that is already done in the usermodehelper() call or not). 

The second is to change the default upcall to "NONE", but that is not described in the commit comment, and can potentially break existing setups that are using the upcall. Before a change like this is made, it would be worthwhile to post to lustre-discuss and lustre-devel to see if there are existing users. If there are a few, then they could safely change their current configs to set the upcall path explicitly to match the current path. If there are many (which I doubt) then we might not change this default at all, but I think this is unlikely.
