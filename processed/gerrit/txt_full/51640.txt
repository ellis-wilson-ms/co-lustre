LU-16374 enc: rename O_FILE_ENC to O_CIPHERTEXT

Rename O_FILE_ENC to O_CIPHERTEXT as per discussion in linux-fscrypt
mailing-list.
Also change the flag combination to be:
O_NOCTTY | O_NDELAY | O_DSYNC
to avoid the risk of accidental issues with tar that already opens
files with the 'O_NOCTTY | O_NDELAY' combination.

O_DSYNC does not make much sense for O_RDONLY files, but will force
writes on encrypted restore to be synchronous. With O_DIRECT and large
enough writes (32MB?) that might be OK, but not ideal for small files.

Fixes: fdbf2ffd41 ("LU-14677 sec: no encryption key migrate/extend/resync/split")

(2 comments)

I'm generally OK with this. Preventing multiple writers is unfortunate, but potentially more problematic is that O_APPEND would cause the file to be silently corrupted if there are two writers to the file, rather that overwriting the file, or writing at a specific offset in the file as expected. 

In this case we should probably return an error from fseek() to an offset that is not i_size on an fd opened with these flags, or any write with pwrite() that does not match i_size.  It would also prevent the use of AIO/DIO to speed up the restore of large files.
