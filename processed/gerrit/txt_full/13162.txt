LU-6060 lnet: set downis to 1 if there's no NI for remote net

lnet_route_t::lr_downis is marked as zero even if there is no NI to
target network, this is wrong and breaks logic of ARF.
This patch fixes this problem, and also removes obsolete code for
ptllnd.

Is this valid for multi-hop configurations?

Say you have a client on gni0, a router with gni0 and o2ib0, a router with o2ib0 and tcp0, and a server with tcp0.  Since the client's first router doesn't have a tcp0 interface, will this mark the route unusable?

Admittedly, I think multi-hop configs are rare.  I'm not aware of us using it at ORNL. Maybe it make sense to have a config for this (multi-hop off by default, turning it on disables this check).
