LU-9295 test: skip sanity-lfsck 9 less than 2.8

to see if the test should be run. Unfortunately, the
server_version variable is never initialized and the tests
are skipped.

For sanity-lfsck tests 9a and 9b, get the server version
using the lustre_version_code routine.

Patch Set 3:

Andreas – I understand and agree that the sanity-lfsck test suite is skipped if run with clients and servers with different versions of Lustre. Right now, running sanity-lfsck in any Lustre configuration (interop or otherwise), tests 9a and 9b are being skipped due to the version check at the beginning of the tests because we never initialize the server_version variable:
[[ $server_version -ge $(version_code 2.7.50) ]] ||
		{ skip "Need MDS version >= 2.7.50"; return; }

The purpose of this patch is to remove the server check at the beginning of test 9a and 9b to allow the tests to run.
