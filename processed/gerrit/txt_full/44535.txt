LU-14915 osd: add flags for project files

Currently, the directories and files owned by some project ID can be
limited by quota, this patch adds flags to implement more controls on
it, such as Read-Only access, Append-Only access etc.

It reuses the framework of project default quota to save these flags
and sync between MDTs and OSTs.

These new flags (such as LUSTRE_IMMUTABLE_FL) are added to the inodes
according to the project ID at OSD when the attribute of the inode is
retrieved, then it won’t need to iterate all the related directories
and files to set or clear some flag.
