LU-0000 tests: add immediate mirror lease tests

Add comprehensive tests for the concurrent write lease
functionality with immediate mirror files. The tests
verify:

- Automatic lease acquisition for write operations on
  immediate mirror files
- Lease acquisition for setattr operations affecting OSTs
- Selective lease acquisition (only for immediate mirrors,
  not regular mirrors or single files)
- Compatibility with existing FLR operations
- Multiple write operations and mirror count preservation

Tests added to sanity-flr.sh:
- test_213a: automatic concurrent write lease acquisition
- test_213b: lease acquisition only for immediate mirrors
- test_213c: setattr operations with lease acquisition
- test_213d: multiple writes with lease acquisition
- test_213e: compatibility with explicit mirror operations
- test_213f: selective lease acquisition verification

Also includes a standalone test script for basic
functionality verification.

Note: These tests verify the lease acquisition mechanism
and basic functionality. Actual write duplication to
multiple mirrors is not yet implemented, so tests focus
on the lease infrastructure and immediate flag handling.
