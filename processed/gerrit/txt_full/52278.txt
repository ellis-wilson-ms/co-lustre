LU-16510 uapi: fix __read_overflow2() error for __QCTL_COPY

With the introduction of fortify-string.h the __QCTL_COPY macro
can now fail with an __read_overflow2() error. The reason is that
the new fortify memcpy sees the flexible array at the end of the
quota structures as zero in size. Much newer kernels have new
handling for this case but this doesn't help with the much older
kernel and gcc version Lustre encounters. The best solution for
all cases is to change the qc_poolname to be a real fix size.
We can also move the QCTL macros to obd.h since no user land
code uses this macro.
