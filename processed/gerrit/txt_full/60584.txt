LU-19211 utils: skip testing for non-functional patches

changed.py is used by Autotest to analyze patches for additional
testing.  The current output of the script lists changed subtests
and affected files.  For example:

    {
      "updated_tests": {
        "sanity-hsm": ["test_11d"]
      },
      "affected_files": {
        "m": ["lustre/mdd/mdd_object.c", "lustre/tests/sanity-hsm.sh"],
        "a": [],
        "d": []
      }
    }

There is also a skip_testing feature which tells Autotest to send
+1 and not run any testing for patches with only non-functional
changes (i.e. Documentation/, contrib/, LICENSES/, etc.)
