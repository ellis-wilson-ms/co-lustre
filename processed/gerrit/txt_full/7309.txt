--- Commit Message ---
LU-107 build: Add abitilty to remove failover startup scripts

This patch will allow users/vendors who do not want to remove the
high availability support(failover) from the source but want to
remove its startup scripts from the RPM build package.
The command line option "--disable-HA" given to the configure script
will remove the startup scritps. Default option is to enable the high
availabity support by installing these startup scripts but will
prevent the old or modified one to be overwritten.

lustre.spec.in - remove the startup scripts from the RPM build
                 or add them without overwriting the old modified one

lustre/autoconf/lustre-core.m4 - add new option --disable-HA.

lustre/scripts/Makefile.am - remove/add the startup scripts from/to
                              variable init_SCRIPTS.
