LU-20090 nrs: use allowed classification types for bucket lookup

When the number of rules is large and the adminstrator configures
the allowed classification types for the dynamic classification
strategy, the TBF scheduler will use the allowed classification
types to generate the key of a request for class bucket lookup.

As all rules are managed in a list, the time complexity of rule
match is O(N). It's not efficient when the number of rules is
large.
After define the allowed classification types, the time complexity
of request classification can be O(1) X the number of the allowed
classification types.

It needs special handling for the default rule. It must be set
with the finest grained classification type which containing all
allowed type bit mask. Otherwise, the request will be classified
into the wrong class bucket (the one of the default rule),
resulting in incorrect rate control.

Add sanityn/test_77td to verify the allowed classification types
configuration works as expected.
