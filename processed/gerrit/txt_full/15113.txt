LU-6677 build: Add RHEL style kmod packages

Provide a Redhat specific lustre-kmod.spec file which uses the old style
kmods (not kmods2) packaging.  By using the provided kmodtool script
packages can be built which support weak modules.  This allows for the
kernel to be updated without having to rebuild the Lustre kernel modules.

Given spl-kmod, spl, zfs-kmod, zfs, lustre-kmod, and lustre source
RPMs it is possible to use the 'mockchain' command to construct a
Lustre yum repository.  The packages are built in the order listed
by mock and the resulting binary packages are immediately added to
the yum repository.  This ensures the mock will be able to satisfy
from its local yum repository the specified build requirements.

@ Construct the yum repository from source RPMs using mockchain.
@ Make sure to add the base debuginfo repo to the mock configuration.
$ cat /etc/mock/epel-6-x86_64.cfg
...
[base-debuginfo]
name=BaseOS - Debuginfo
baseurl=http://debuginfo.centos.org/6/$basearch/
gpgcheck=1
gpgkey=file:///etc/pki/rpm-gpg/RPM-GPG-KEY-CentOS-Debug-6
enabled=1
...

@ Build the mock packages
$ mockchain \
    -r epel-6-x86_64 -l /var/tmp/mock-repo \
    spl-kmod-*.src.rpm spl-*.src.rpm \
    zfs-kmod-*.src.rpm zfs-*.src.rpm \
    lustre-kmod-*.src.rpm lustre-2.5.3-*.src.rpm

@ Add the mock repository to yum
$ cat /etc/yum.repos.d/mock.repo
[mock]
name=Mock Repository
baseurl=file:///var/tmp/mock-repo/results/epel-6-x86_64/
enabled=1
