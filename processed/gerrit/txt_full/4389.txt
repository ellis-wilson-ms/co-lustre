--- Commit Message ---
LU-2225 tests: skip data writes in few tests

few tests need to fill filesystem or just generate a lot
writes. this usually take quite long. this patch introduces
OBD_FAIL_OST_SKIP_BIO. when it's set osd_submit_bio() skips
writes. also, fast_dd() is added to t-f which sets this
fail_loc and drops D_TRACE as the latter decreases CLIO
performance a lot.

sanity/64b on a local setup:
(dgb=full)			- 223s
(dbg=-trace,-dlmtrace)		- 76s
(dbg=0)				- 61s
(skipio,dbg=-trace,-dlmtrace)	- 46s
(skipio,dbg=0)			- 27s

I like the idea of this patch, but it needs a few improvements before it can land.  The main improvement would be to make this OBD_FAIL check also work for ZFS, since I bet it is even slower than ldiskfs.  It may be that using 1MB writes instead of 1kB writes will already speed up these tests noticeably for ZFS.  In Jinshan's patch http://review.whamcloud.com/5164 it appears that this is OSD neutral, but is a lot more complex than this patch.

If this patch isn't actually dependent on http://review.whamcloud.com/4388 (which it doesn't appear to be) then it should be submitted from a separate branch so that these two patches can land independently.
