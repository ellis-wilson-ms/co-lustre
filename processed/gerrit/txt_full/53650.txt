Patch Set 2: Code-Review-1

I think this is the wrong fix. libcfs_init() should run first and only do what is required to make its services available.
libcfs_setup() should run afterwards, and ideally only when the services in libcfs are about to be used. 
If libcfs_init() needs debugging to be enabled, then it should enable debugging directly rather than having that happen in  libcfs_setup().  libcfs_setup() might then check if debugging was successfully enabled.
