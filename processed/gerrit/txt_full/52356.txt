> Andreas, should count be reseted by some timer or lfsck start? With
 > a high uptime stack would not be repeated even after month.

It could be changed from just a one-time check to "print once a day" or similar, if you think that is important:

        static time_t print_next;

        if (ktime_get_seconds() > print_next) {
                print_next = ktime_get_seconds() + 24 * 3600;
                dump_stack();
        }

Note that this is only affecting the stack trace, the error message itself is rate limited but would still be printed every 10 minutes.
