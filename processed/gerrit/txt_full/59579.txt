LU-0000 llite: server-side lease-driven FLR state transitions

Implement automatic FLR state transitions driven by
concurrent write lease acquisition and release on the
server side.

This adds:
- Reference counting for concurrent write leases per file
- Automatic RDONLY -> WRITE_PENDING transition when first
  concurrent write lease is acquired
- Automatic WRITE_PENDING -> RDONLY transition when last
  concurrent write lease is released
- Mirror staling on lease acquisition to maintain FLR
  consistency

The server-side state management ensures that mirrored
files transition to the appropriate FLR state based on
active concurrent write leases, enabling proper mirror
synchronization and consistency checking.
