LU-5170 lfs: Standardize error messages in command parser

Error messages for the command parser are updated to a standard
format.  The message is prefixed with the name of the utility
and the command that caused the error.  User-provided values are
delimited with single quotes.

When incorrect options or parameters are provided to a command,
the full help text is no longer printed.  Instead, a brief message
is displayed asking the user to consult the help.

When help is requested for a command, the help for any associated
sub-commands is displayed along with it.
