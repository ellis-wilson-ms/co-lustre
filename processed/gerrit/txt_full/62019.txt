--- Commit Message ---
LU-19505 lfsck: don't set LAD_TO_POST if <= 0

lad_post_generic() shouldn't set LAD_TO_POST if *result <= 0, which
means assistant thread failed or is stopped, otherwise it may trigger
assertion in lfsck_assistant_engine() because the assistant thread
shouldn't do post work upon failure.

Fixes: 2b5df830d ("LU-19370 lfsck: don't use ptlrpc_thread for LFSCK")
