Patch Set 3:

Since this patch is only affecting conf-sanity, you can cut out almost all of the other testing (which doesn't provide any value for this patch) via:

 Test-Parameters: trivial testlist=conf-sanity

Without the "trivial" keyword, any change to the commit message would cause a new build and trigger another full round of testing, in addition to an extra conf-sanity run, which isn't what you want.

With the "trivial" keyword, it only does a single "review-ldiskfs" test run, which is sanity.sh and lnet-selftest.sh to verify basic functionality, and then whatever additional tests are requested via "testlist=".
