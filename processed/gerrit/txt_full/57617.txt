--- Commit Message ---
LU-10499 utils: use libyaml for pin xattr handling

Change the format of trusted.pin xattr to a YAML flow sequence.
This allows us to store far more data in the xattr

trusted.pin="hsm: [ 2, 50, 100 ]"

To use this new format move from using cYAML which doesn't support
flow sequence to libyaml. The move to libyaml enables using the
Netlink API as well if we ever need far more data.

Update the libyaml configure test to PKG_CHECK_MODULES. This gives
us the proper CFLAGS and LIBS flags for libyaml.

- user defined format, something like "hsm:2,3,4". Most simple and easy to be parsed by string functions, but not flexible.

Lastly JSON format won.
