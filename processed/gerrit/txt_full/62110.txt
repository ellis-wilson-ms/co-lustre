LU-18838 llite: fix race condition at putting sax

In ll_statahead_thread() and ll_deauthorize_statahead_advise(),
after ll_sai_put() drops the last sai reference and sets
lli->lli_sai = NULL, the subsequent ll_sax_put(dir, lli->lli_sax)
reads lli->lli_sax which may have already been freed or set to
NULL by a concurrent thread that dropped the last sax reference.
This causes a panic in atomic_dec_and_lock() on the stale pointer.

Add a sai_sax field to struct ll_statahead_info so that each sai
holds a stable pointer to its associated statahead context. Save
the ctx pointer from sai->sai_sax into a local variable before
calling ll_sai_put(), ensuring the pointer remains valid for the
subsequent ll_sax_put() call. Also update sa_alloc() and sa_kill()
to use sai->sai_sax instead of lli->lli_sax for consistency.
