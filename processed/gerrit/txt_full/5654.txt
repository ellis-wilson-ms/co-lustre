--- Commit Message ---
LU-951 tests: wait import state in fail()

There still has a rare chance that the request meets
an invalid import after fail() returns. So we should wait
import restored to a certain state before doing the next
operation.

The fail() function can only be used in the situation
that before failing a server target, all of the Lustre
clients have $MOUNT mounted as Lustre filesystem.

For the case that not all of the Lustre clients are mounted
on $MOUNT before failing a server target, we need use
fail_nodf() to fail the server target, and then run
wait_clients_import_state() and client_up() to check the
previously live clients.

Test-Parameters: envdefinitions=SLOW=yes,ENABLE_QUOTA=yes \
testlist=replay-dual,replay-vbr,recovery-small
