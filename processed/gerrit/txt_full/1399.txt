--- Message (Andreas Dilger, 2011-09-23 17:07:05.000000000) ---
Patch Set 4: Looks good to me, but someone else must approve

I'm not totally against calling the compat functions ll_delete_from_page_cache() and ll_remove_from_page_cache(), but usually the ll_ prefix is used in the case where a function does not exactly match the kernel version (e.g. different args or something). Using the actual function name for the compat macro means there is no code change needed when the old compat macro is removed and only the actual function is left. 

The other note about this code is that truncate_complete_page() seems to never be exported from vanilla kernels, and is unlikely to do so. It would be better to use truncate_inode_page_range(), which is exported, and is also more efficient. This might need some more serious surgery in CLIO, so I don't suggest to prevent this simple patch from landing, but that is the better direction to go in the long run.
