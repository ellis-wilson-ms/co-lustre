LU-12187 lod: calculate parity stripe count from data stripes

Implement automatic parity stripe count calculation for EC
(Erasure Coding) layouts based on the actual data component
stripe count.

When a parity component is instantiated, the server (LOD
layer) calculates the required number of parity stripes
using the formula:

  parity_stripes = ceil(data_stripes / k) * p

where k = dstripe_count (data stripes per RAID set)
      p = cstripe_count (parity stripes per RAID set)

The calculation handles cases where data_stripes is not
evenly divisible by k by rounding up, ensuring all data
stripes are covered by parity. The parity stripe count is
also limited by the number of available OSTs.

Example: With 10 data stripes and 6+2 EC:
  RAID sets = ceil(10 / 6) = 2
  Parity stripes = 2 * 2 = 4

Changes:
- Block setting stripe count directly on parity components
  in llapi_layout_stripe_count_set()
- Set parity component stripe count to 0 in LOV blob so
  server calculates the correct value
- Add RAID set calculation logic in lod_qos_prep_create()
- Add tests to verify parity stripe count calculation for
  different EC configurations (6+2, 4+2, 8+3, 2+1)
- Add API test to verify stripe count cannot be set on
  parity components
