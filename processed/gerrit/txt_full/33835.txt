LU-11763 lnet: Add warning message in "lctl network configure"

Currently, if the network interface is not specified
in the modprobe.conf file, then the "lctl network configure"
command configures network using default interface values for
socklnd and o2iblnd.

Starting Lustre 2.13 release, it will be necessary to
specify network interface to configure network using
"lctl network configure" command.

Oleg was commenting that this patch results in a warning message for every test in our test session.  If this is going to be a warning, and then become an error, this patch also needs to fix the test configuration to properly configure the nodes, presumably requiring a line in /etc/modprobe.d/lustre.conf like:

    options lnet networks="tcp0(eth0)"
 
I was looking at my home filesystem as well, and see that the clients do not specify anything for LNet, because they are using the default tcp0 network.  To me that means we should add the above line into the default lustre.conf file, to maintain compatibility with current behavior, and quiet the warnings that Oleg sees.
