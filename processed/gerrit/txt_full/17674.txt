LU-6142 lnet: handle NULL and zero comparison correctly

For the linux kernel the test if (blah == NULL) and
if (rc == 0) are considered bad style. This patch
converts it to the if (blah) style that is required.

Looking at the upstream kernel, I see over 30k cases of "if (foo == NULL)" or "if (foo != NULL)" and over 58k cases of "if (bar == 0)" or "if (bar != 0)", so I'm less inclined to believe that this is a requirement for moving out of staging.

While I'm ready to accept patches that have already been landed upstream for the sake of code unification, I'd rather avoid wholesale changes like this driven by us that IMHO actually make the code worse.

I'd rather focus on productive changes like removing wrappers that obscure callpaths like TGT_RPC_HANDLER() and OCD_HAS_FLAG() and LDLM_HAVE_MASK() that actually make the code harder to follow.

Not sure what Oleg thinks of this.
