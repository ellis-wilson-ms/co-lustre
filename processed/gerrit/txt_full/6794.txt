I think I am confused about something here so Aurelien, if you can explain it to me, that would be great.

In my mental picture of HM, there's the copy phase after which the file is "clean", and after some time the file might be released.

Now, before the release happens somebody does some writes and has dirty pages - we no longer can have a release because the dirty data is not replicated onto hsm and by releasing the objects (deleting them, really) we would introduce a data corruption because restore would pull in an older version of the file, right?
I have some sort of a memory about problems about the dirty flag being discussed in the past.

Also if nobody has any dirty pages, then at most there could be cached pages only, those should go away when the objects are destroyed (because this must be done under a lock already anyway, kind of like object destroy I hope?) and that will nuke all cached pages anyway.

So why do we need this patch then?
