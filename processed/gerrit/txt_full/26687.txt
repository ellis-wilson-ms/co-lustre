--- Message (Andreas Dilger, 2017-04-19 17:56:48.000000000) ---
Patch Set 2:

Is there a reason why all of these patches removed:

 Test-Parameters: trivial

which would limit them to lnet-selftest and sanity testing?  I saw this on the previous versions of these patches (~20 of which were pushed just yesterday) but these ones are missing it.

This backlogs the build and test queue significantly (currently at 64 patches) with very little benefit for LNet-specific patches for the ~28h of added test time per patch.

That said PLEASE do not resubmit these patches just to add the Test-Parameter line again, since that will only make the situation worse.

Instead, we need to look into improving lnet-selftest, possibly with a SLOW=yes check, that can be used to test LNet-specific patches more thoroughly without spending so much time running Lustre tests that aren't going to find LNet bugs but are much more likely to fail with intermittent Lustre bugs that will also prevent your patches from landing.
