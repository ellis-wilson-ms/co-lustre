*why* doesn't direct I/O work on 1 byte reads?  This is a bizarre problem that doesn't seem like it should be unfixable, if it works on larger unaligned reads?

And why 512 bytes?  Lustre doesn't care about this old definition of sector size - it uses page_size for DIO - and the bug isn't at 512 bytes either.  The choice is arbitrary.

We should fix this bug not hack around it.
