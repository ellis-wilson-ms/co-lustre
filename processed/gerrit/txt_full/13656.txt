--- Commit Message ---
LU-6195 osd-zfs: dmu_tx_hold_zap() called with wrong keys

The osd_declare_object_destroy() calls dmu_tx_hold_zap() with buf
that holds string representation of the FID for the inode accounting
ZAPs. This is wrong as the keys for these ZAPs are not the FID. Since
the code will be obsoleted by ZFS object accounting feature, just
replace buf with NULL, and zap_count_write() will use worst-case
estimate. Same for osd_declare_object_create().
