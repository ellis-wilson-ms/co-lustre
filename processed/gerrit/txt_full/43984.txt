LU-14757 wbc: MemFS lookup fallback to Lustre lookup

In WBC lock drop flush mode, when enter into the code path
@memfs_lookup during the new file creation, and the dir (parent
inode) is flushed (WBC state becomes WBC_STATE_FL_NONE), at this
time, it will fall back into Lustre lookup path @ll_lookup_nd to
perform lookup() operation from MDT.
The old Lustre lookup path with LOOKUP_CREATE | LOOKUP_OPEN will
return ERR_PTR(-ENOENT) wrongly.

In this patch, when fallback lookup from MDT returns -ENOENT to
indicate that the name path is not existed, it will return NULL
instead of ERR_PTR(-ENOENT). Otherwise, the creation application
will report -ENOENT error.
