--- Commit Message ---
LU-17144 mdt: set dmv by setxattr

Client side: convert setxattr("trusted.dmv") to "setdirstripe -D", as
will help restore directory default LMV from backup.

Server side: add a tunable to enable setxattr("trusted.dmv"), it can
be turned on by "lctl set_param -n mdt.*.enable_dmv_xattr=1". It's
off by default. Since empty buffer can be set by setxattr, add check
in server code to avoid crash.

Add sanity 413j.

Patch Set 1: Code-Review-1

Lai,
can you please add a tunable parameter "mdt.*.enable_dmv_xattr"  or similar to enable/disable this functionality.  That would allow it to be enabled for systems/users that depend on this functionality, but disable it by default for users that need setxattr to work (e.g. backup/restore where they want to preserve striped directories).

I think it makes sense that setxattr("trusted.dmv") is intercepted at the client and call ll_dir_setdirstripe() internally to send the standard MDS_SETATTR instead of sending MDS_SETXATTR.  That would fix the case of new clients with old MDS. 

Also, changing mdt_reint_setxattr() to allow the "trusted.dmv" xattr without crashing to handle the case of old clients with new MDS, which can also happen in the field.

It also makes sense to not save the layout if it is the same as the default:

 lmv_stripe_count: 1 lmv_stripe_offset: -1 lmv_hash_type: none

to avoid hard-coding "default" into every directory if used with rsync/tar.
