There is one remaining possible problem - "#include <../fs/mount.h>".

I think the main idea behind hiding vfsmount guts (move from
include/linux/mount.h to fs/mount.h) seems to be it is not suggestted to access them above or within VFS layer.

There are some other kernel code inside "fs" directory also include fs/mount.h. So I think possibly it is not a big issue for lustre to include that file, as lustre will also be inside "fs" directory in the future.
