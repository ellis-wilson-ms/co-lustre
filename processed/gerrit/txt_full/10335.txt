Patch Set 4: Code-Review-1

(18 comments)

John, on the one hand, you were objecting in patch #1 to expanding the comments to provide further explanation (e.g. what they are used for), yet you also object to the terse comments that you asked for.

I agree that a comment "Setup LWP device" for "lwp_setup()" is not very useful.  Having the description of the function arguments (e.g. what "nidstring" is actually describing) adds more value to the comment blocks.  

For wrapper functions that are only a few lines long I don't think we need to have comment blocks for them.  There is some value even for sparse comment blocks on large functions, firstly to have a place to accumulate comments in the future, and secondly as a good practice to get people in the habit of adding comments to new code and this is easier to enforce if we are consistent about it.

The -1 is for other issues I found when reviewing the comments.
