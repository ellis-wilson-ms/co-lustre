--- Commit Message ---
LU-13904 tests: don't assume echo_client is a module.

The upstream lustre client can be compiled with all code
in the kernel - no modules.
So testing if the module is loaded gets incorrect results.
So add a test to see if the functionalty is already present,
and don't load a module if none is needed.

--- Message (Neil Brown, 2020-08-12 06:54:27.000000000) ---

> I wouldn't want echo_client to be compiled permanently into the Lustre kernel module. 

No question that it should be possible to build as a module (or not at all), and that should be the default.
But would it matter if someone were to choose a configuration that isn't your personal first choice?
I do all (almost) all my kernel testing by building a monolithic kernel (no modules) and running "qemu -kernel /path/bzImage -sda root-image ...."  It's a lot easier to set up than having to copy modules around, and making sure the modules match the booting kernel.

Note that I'm not suggesting echo_client be part of the lustre module.  They would either be separate modules, or both part of the kernel (or lustre be in-kernel, but echo_client be a module).
