--- Commit Message ---
LU-20018 obdclass: fix the race between lu_object

If one lu_object is not active and put on the LRU list,
then there could be two lu_object linking to the same
backend filesytem object if the "lu_site_purge_objects"
is trying to delete these unused lu_object, and causes
this issue, which both lu_object are cleaning the same
object of the backend filesystem.
This patch also fixes an issue of using freed session
"lu_context" in "distribute_txn_replay_handle".
