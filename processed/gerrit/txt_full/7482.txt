LU-3027 clio: only match original lock request

By matching only originally requested region in sublock we
are making sure to avoid the race where we match the expanded region,
but the uderlying ldlm lock gets cancelled by the time we have
a chance to reference it and then we end up reenqueuing the wrong lock.

This does have a downside of potentially creating more sublocks
as a rolling write will not be able to match previous sublocks,
but this should not result in any extra RPCS because underlying ldlm lock
would still be matched (Assuming it's still there at the match time).
