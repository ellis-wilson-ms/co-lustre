--- Commit Message ---
LU-2321 debug: Quiet uninitialized target connects

When a target is not fully initialized and receives a connection
attempt, it will refuse the connection and print a message to the
console stating this fact. This is "normal operation", so rather
than using LCONSOLE_WARN to convey this information, this patch
modifies it to use CDEBUG.

--- Message (Oleg Drokin, 2012-12-13 00:44:54.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

This is another one of those annoying messages  that you and other big sites really hate it due to the volume,
yet it is very valuable at a lot of new and less sophisticated deployments to tell people their config is broken and their clients are only connecting to a wrong server.

I suspect the real proper way here would be to have a new message level that would be toggable for displaying on console, and it should be on by default too, but you will be able to turn it off.
Other messages that you do not like would be able to be added into that loglevel too.
