--- Commit Message ---
LU-18847 mdt: version layout checking

During migration, it's possible for a client to have an outdated
stripe and directory entry cached. While holding a CWD reference,
the client may create a directory using the stale layout, resulting
in duplicate entries with the same name - one on the new stripe and
one on the old (demonstrated by sanity test 230B).

This patch introduces layout version tracking between client and
server. The client packs the layout version during create operations,
and the server validates it. Since the OUT layer bypasses
MDT/MDD/LOD during XATTR modifications, stale layout version data
can occur. The osd_check_stale() part detects LMV updates to ensure
the version check uses current data.
