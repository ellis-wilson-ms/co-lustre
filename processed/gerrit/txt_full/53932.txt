--- Commit Message ---
LU-11085 ldlm: Use separate list of whole-file extents

It is likely that there can be many more whole-file extent locks than
any other extent locks.  This could potentially swamp the rbtree used
to hold extents and increase lookup time.

Previously all locks with the same extent only occurred once in the
extent tree but a recent simplification change that.  This patch
re-introduces the optimisation just for whole-file locks.  That is the
only extent like to occur a large number of times.

ldlm_extent_search() is enhanced to search both the list and the tree.
The micro-optimisation of skipping the call to ldlm_extent_search()
when the tree is empty is removed as it is unlikely to help and
clutters the code.
