LU-2886 scrub: non-fixed FIDs for some local files

For old lustre-2.x (x < 4), the local files PENDING/lfsck_bookmark
used fixed/reserved FIDs when created. The cases have been changed
to use variable local FIDs since lustre-2.4. So adjust OI scrub to
match the changes.

--- Message (Mikhail Pershin, 2013-05-09 20:15:41.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

this is not needed because local storage doesn't use sequence in which old files were created. FID_SEQ_LOCAL_FILE is used only for nameless objects created by dt_object_find_or_create() and names for them are created by OSD compat code. The simplest way to check that this sequence will not be used occasionally in local storage would be just assertion checking that sequence is not FID_SEQ_LOCAL_FILE.

--- Message (Fan Yong, 2013-05-10 01:22:09.000000000) ---
Patch Set 1:

The trouble is that, for OI scrub, it thought that some local files (such as lfsck_bookmark, lfsck_namespace, PENDING) should use fixed/reserve FIDs with "seq = FID_SEQ_LOCAL_FILE", but the real FIDs assigned to them are with "seq = FID_SEQ_LOCAL_NAME". Such difference will cause OI scrub confused.

So either we make OI scrub to know above changes, or adjust the create logic to assign fixed FIDs to them.
