--- Commit Message ---
LU-16313 pcc: use two bits to indicate pcc type for attach

PCC currenty supports two types: readwrite and readonly.
The attach data structure @lu_pcc_attach is using 32 bit value to
indicate the PCC type:
	struct lu_pcc_attach {
		__u32 pcca_type;
		__u32 pcca_id;
	};

In this patch, it changes to use 2 bits to represent the PCC type.
The left bits in @pcca_type can be used as flags for attach such
as a flag to indicate using the asynchronous attach via the
command "lfs pcc attach -A" for PCCRO.

ah, Yes, we need to add mask in:
llapi_pcc_attach_fid()
However, Personally, I don't think we need to mask with LU_PCC_TYPE_MASK in:
pcc_dataset_get()
pcc_try_dataset_attach()

As the "LU_PCC_FLAGS_MASK" is only used to pass the flags into the kernel.
After known the flags, we can drop these flags and directly use the LU_PCC_READWRITE or LU_PCC_READONLY to store in @pcci_type.
Moreover, we already have flags for each file in PCC: @lli_pcc_state, add extra flags is not needed, I think.
