--- Message (Andreas Dilger, 2015-02-14 00:29:23.000000000) ---
Patch Set 4:

James, the only place that the "extents" _mount_ option is used anymore is when the static mount options are read from the "mountdata" file inside the MDT/OST filesystem, and then passed as options for the internal Lustre mount.

If the internal Lustre mount is taught to ignore these obsolete mount options (depending on the EXT4_FEATURE_INCOMPAT_EXTENTS flag in the superblock instead) then we don't need to patch ldiskfs to silently ignore them.  The INCOMPAT_EXTENTS option will always be set on existing OSTs and not be set on MDTs, and is set at format time on new OSTs/MDTs, so it will do the right thing.

That keeps the Lustre-specific details (and warts) inside Lustre instead of having to add complexity into every version of ldiskfs that is built.
