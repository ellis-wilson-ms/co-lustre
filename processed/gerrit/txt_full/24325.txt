--- Commit Message ---
LU-6401 uapi: turn lustre_param.h into a proper UAPI header

Move all the kernel specific function prototypes from
lustre_param.h into obd_config.h which is a kernel only
header. The inline functions lustre_is_*_valid are used
only by user land so we can remove them. Remove the user
land error checking with lustre_is_*_valid() since its
the job of the kernel to validate the date passed in.
The libcfs.h header shouldn't be exposed to user land
so remove it.

(2 comments)

(defect) mkfs.lustre should not depend on lustreapi.

(defect) The llapi_is_{fs,pool}name() functions should not be exposed externally. Any uses of these functions by external code is likely to be wrong (to see why, note that half of the internal uses are wrong or unnecessary). The utilities should try to do what the user asked and the kernel should reject what's invalid.

Please create an internal header in the utils directory (if needed) for these functions. Let's not make a mess in user space headers just to make the kernel headers conform to some upstream aesthetics.

(defect) LUSTRE_MAXFSNAME should be moved to lustre_user.h (and several of the hard coded '8's should be using LUSTRE_MAXFSNAME).
