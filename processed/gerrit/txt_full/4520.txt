LU-1187 tests: Fixes in test-framework for DNE

1. Add dev parameter to mkfs_opts, so it can check if it needs
--mgs for dev. combined_mgs_mdt is only useful for mdt0.
2. Add multi-node facet failover support, so it can fail multiple
targets at the same time.
3. Remove mdsn_HOST/failover_HOSTs in local.sh and add them to
facet_host.
4. replace MDSDEV1 and MGSDEV with mdsdevname and mgsdevname, because
these vars are not always defined.

Patch Set 5: I would prefer that you didn't submit this

In conf-sanity.sh test_56() and test_60() it uses mds1 for the MDT name even though there are potentially multiple MDTs being added. This might work for the code today before DNE is landed, but will probably break for DNE. 

In test-framework.sh facet_failover() and facet_stop(), please indent the whole function with tabs. Maybe they are, and it is just the way it is displayed by Gerrit?

Also, in facet_failover() I still don't agree with the inclusion of sync before triggering failover (which is apparently from LU-924). That is not how failover works in real life and I think it will hide bugs during testing. Also, a change like that does not belong in a patch which describes only changes for DNE testing. Per inspections in the LU-924 patch (which is where it should be landed), there are AFAIK only a very small number of tests that may be affected by this problem (basically only after a client first mounts or reconnects after eviction and then does nothing in the few seconds before another eviction) so I'd rather add explicit sync to those tests than for every test, or add some mechanism to only force sync updates to last_rcvd for connections during testing. 

Finally, it might be possible for insanity.sh to use the multi-facet failover change added in this patch. It might be worthwhile to do that in a separate patch.
