LU-9341 lod: Add special O_APPEND striping

Files opened with O_APPEND are almost always log files,
which generally stay small and do not benefit from being
striped widely.  Additionally, PFL files accessed with
O_APPEND are fully instantiated, meaning that because the
files usually stay small, these objects usually wasted.

This patch adds special striping for files created with
O_APPEND.  This is controlled on the MDS by two new proc
variables:
mdd_append_stripe_count
mdd_append_pool

If the stripe count is set to 0 and the pool is not set,
this functionality is disabled and files created with
O_APPEND will be striped like any other file.

Nathan's suggestion is a little trickier than I thought - It's tricky to validate a pool name when it's provided at the mdd layer.

Andreas,

Any thoughts on that?  I guess if a pool is removed we're going to end up in the "non-existent pool" situation regardless, but it seems a little unfriendly to let us set a pool with an incorrect name.

Any thoughts?  If users adding a pool with an incorrect name is acceptable, then this gets easy again.

--- Message (Andreas Dilger, 2019-08-09 06:00:07.000000000) ---
Patch Set 2:

> (1 comment)
 > 
 > Nathan's suggestion is a little trickier than I thought - It's
 > tricky to validate a pool name when it's provided at the mdd layer.
 > 
 > Any thoughts on that?  I guess if a pool is removed we're going to
 > end up in the "non-existent pool" situation regardless, but it
 > seems a little unfriendly to let us set a pool with an incorrect
 > name.
 > 
 > Any thoughts?  If users adding a pool with an incorrect name is

It is already possible to create a file with an incorrect pool.  For example, if a default directory layout is saved with "poolX" and then that pool is removed, the MDS doesn't scan the whole filesystem to remove that pool from the layout (maybe it will even be added back with a different config).

Instead, the LOD layer should detect and ignore such cases, and treat files with an invalid pool as if there was no pool specified.
