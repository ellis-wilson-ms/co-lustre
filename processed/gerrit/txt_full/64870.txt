LU-20038 utils: find_param should not read non-matching params

lctl find_param reads every parameter file even when the
parameter name doesn't match the search pattern. This is
because the regex filter is only applied during printing,
after read_param() has already opened and read every file
in the parameter tree.

Move the regex check earlier in do_param_op() so that
non-matching regular files are skipped before read_param()
is called. Directories are still traversed for recursion.

This avoids unnecessary overhead from opening and reading
params like brw_stats and job_stats that don't match the
search pattern. Reading job_stats has the side effect of
clearing old job statistics, and reading kbytesfree or
filesfree triggers OBD_STATFS RPCs.

Add sanity test 401gd to verify with strace that
non-matching params are not opened.
