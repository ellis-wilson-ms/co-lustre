LU-2827 ldlm: wrong handling of ldlm resends (ORNL version)

If a successful reply is lost for an intent
lock request, MDS will not correctly recover
from this situation on resend. Instead, a new
lock is created.

This patch also adds handling of resend during layout-lock.

This patch is the merge + b2_4 back-port of the following
2 patches to the special ORNL 2.4 server tree. This is done
to see what regression are in the LU-2827 patch that has
been modified so it can be applied to the ORNL lustre code
base :
