LU-16035 kfilnd: Fix immediate buffer ref count

The current implementation has a race condition where two threads
could repost the same immediate buffer. To resolve this issue,
the ref count value is now checked atomically when decremented. Once
the ref count is zero, the immediate buffer will be reposted.
