For lfs migrate, t32_test() contains the following tests:

                # migrate files/dirs to remote MDT, then move them back
                if [ $(lustre_version_code mds1) -ge $(version_code 2.7.50) -a \
                     $dne_upgrade != "no" ]; then
                        $r $LCTL set_param -n   \
                                mdt.${fsname}*.enable_remote_dir=1 2>/dev/null

                        echo "test migration"
                        pushd $tmp/mnt/lustre
                        for dir in $(find ! -name .lustre ! -name . -type d); do
                                mdt_index=$($LFS getdirstripe -i $dir)
                                stripe_cnt=$($LFS getdirstripe -c $dir)
                                if [ $mdt_index = 0 -a $stripe_cnt -le 1 ]; then
                                        $LFS mv -M 1 $dir || {
                                        popd
                                        error_noexit "migrate MDT1 failed"
                                        return 1
                                }
                                fi
                        done

                        for dir in $(find ! -name . ! -name .lustre -type d); do
                                mdt_index=$($LFS getdirstripe -i $dir)

Rather than testing these images manually, it makes more sense to investigate why they are not being tested automatically.  Otherwise, they will not be tested automatically in the future and much of the value will be lost.

Maybe /tmp is a ramdisk and this is too small to hold all of the images?  Would it be better to use /var/tmp, or possibly uncompress the images into OSTDEV1, OSTDEV2, MDSDEV1, etc. so that we are sure there is enough space to hold them?
