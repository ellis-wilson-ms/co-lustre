Just checking - you do know that in recent kernels, %p doesn't report the actual address, but rather a hash of the address.  So you can know if two addresses are the same, but you cannot use the address to help examine a crashdump.
Given that - are you still happy with %p ??
%px will print the actual address, but it is descouraged as it can be used to defeat address-space-randomization.

Based on some chatting in slack you may want to change %p to %px or %pK
%px -> print the address (not a hash)
%pK -> print the address conditionally per /proc/sys/kernel/kptr_restrict sysctl

https://lwn.net/Articles/420403/
