--- Commit Message ---
LU-20000 lnetctl: bad CPTs/tunables during import

This patch fixes only old API. There will be another patch
for netlink.

When an LNet configuration is imported, parsing of CPTs and tunables
was done wrong. This patch fixes the following issues:
- Wrong CPTs if different between two NIs of same net
- lnd_tunables cannot be different between NIs
- lnd_tunables not read if no tunables
- Wrong tunables (lnd_tunables) read from NI if first is empty
- Tunables can be set directly in the net
- Documentation had malformed yaml example
- Show warning if we try to set different tunables (not lnd_tunables)
  for NIs of a same net
