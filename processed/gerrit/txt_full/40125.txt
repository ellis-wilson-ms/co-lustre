LU-12682 llite: make undeletable property generic

As part of Gerrit change #35856 foreign symlink files/dirs
had the property of being undeletable using unlink/rmdir, in
order to avoid losing reference to external objects by
mistake.

This patch makes this property generic and available to any Lustre
file/dir.

Patch Set 1: Code-Review-1

(12 comments)

Bruno, thanks for the patch.

I would really prefer to avoid using the XATTR interfaces, since those are from XFS and we have absolutely no control over them.  Instead, use the ext4 FS_IOC_{SETATTR,GETATTR} interfaces that we already use for other flags.

Also, I would prefer not to have a separate command for clearing the "undeletable" flag from the file.  It would be better to add e.g. "lfs chattr" and "lfs lsattr" to match the e2fsprogs "chattr" and "lsattr" commands, or alternately a separate "lfs unremovable" command that only sets/clears this one flag.

Unfortunately, there is already an EXT4_UNRM_FL that is described as "undeletable" in the chattr(1)/lsattr(1) man page, but it means the opposite - that the file can be "undeleted" after it is deleted.  That is not implemented anywhere, but I think it has a confusingly similar name.

Alternatives would be "keep" and use "+/-k", which is clear and unique.  I also thought about "(un)removable" and use "+/-r" as the attribute, but "-r" is also confusingly close with "--recursive" and "--read".
