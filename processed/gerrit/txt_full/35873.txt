LU-12685 llite: fix check for mem-alloc failure.

A change to allocation of op_data in ll_statahead_thread()
appears to have assumed that OBD_ALLOC_PTR() would set the
pointer to ERR_PTR(-ENOMEM) on failure.  It actually sets
it to NULL, so the test needs to be changed accordingly.

One note about pushing a single large patch series like this to Gerrit is that ALL of the earlier patches in the series have to pass testing before a given patch can land.

If you are using default testing (which should definitely be the case for patches that change code functionality) as opposed to "trivial" patches that only affect building (such as dead code removal) then there is a much higher likelihood that later patches can be delayed landing because of intermittent test failures.

Also, if an earlier patch in the series needs to be refreshed, either extra care is needed to push only the one modified patch, or the review/verified flags of all patches in the series may be reset.

If the patches are independent (as appears to be the case for many of them) it is more efficient to push them as independent patches (each on its own branch in your local repo) so that they can be reviewed, refreshed, and landed separately rather than all being dependent on each other.
