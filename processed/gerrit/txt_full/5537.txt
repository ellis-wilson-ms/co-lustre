--- Commit Message ---
LU-2869 llite: extended attribute cache

This patch implements an extended attribute cache for
a Lustre client. It is organized as a write-through
cache: reads are performed from cache, updates are sent
synchronously to the MDS. An additional inode bit
MDS_INODELOCK_XATTR is added to protect the cache.

The reason of using a complicated locking scheme is that blocking AST needs to avoid deadlocks with the enqueue code. So, we cannot take a single lock in BL AST when we are already holding it in the enqueue code which was the cause of that blocking AST.

BL AST code needs to handle non-atomic structures like lists, so it needs to lock its access anyway.
