LU-12669 lov: handle unavailable parity in EC recovery

Two related fixes for EC recovery when parity stripe
OSTs are deactivated:

1) Make parity sub-IO setup and page read resilient.
lov_ecio_add_parity_sub() and lov_ec_read_parity_pages()
previously aborted with -EIO when any parity stripe
was unavailable (NULL lo_sub or lov_sub_get failure).
With p parity stripes we only need err_nr of them,
so skip unavailable stripes and only fail if fewer
than err_nr were successfully added/read.

2) Track which parity stripe was actually read and fix
the encoding matrix accordingly.  When parity stripe 0
is unavailable and stripe 1 is used instead, the
Galois Field coefficients in the decoding matrix must
come from encode_matrix row (dcount + 1), not row
(dcount + 0).  Add a parity_used[] array that records
the actual stripe index for each parity slot read,
and use it in lov_recover_data() to select the correct
encoding matrix rows before inversion.

Without fix 2, fix 1 silently produces wrong data
because the matrix inversion uses coefficients for a
parity stripe that was never read.

Tests 42a-46c pass.  Test 46d (which deactivates
a parity stripe's OST) passes with these fixes.

Resolves test_46d failure where deactivated
parity OSTs caused recovery to abort (bead
2-xcl).
