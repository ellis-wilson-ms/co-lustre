LU-0000 llite: restrict concurrent write leases to immediate

Modify concurrent write lease acquisition to only acquire
leases for mirrored files that have the immediate flag
(LCM_FL_IMMEDIATE_MIRROR) set. This ensures that the
concurrent write lease mechanism is only used for files
that are configured for immediate mirror writes.

Changes:
- Add LCM_FL_IMMEDIATE_MIRROR flag check in
  ll_file_io_generic() for write operations
- Add LCM_FL_IMMEDIATE_MIRROR flag check in ll_setattr_raw()
  for setattr operations affecting OST attributes
- Update comments and debug messages to reflect the
  immediate flag requirement

The lease acquisition now requires both conditions:
1. File is mirrored (lcm_mirror_count > 0)
2. File has immediate flag set (lcm_flags &
   LCM_FL_IMMEDIATE_MIRROR)

This provides more precise control over when concurrent
write leases are used, limiting them to files specifically
configured for immediate mirror write functionality.
