LU-4783 mgs: start params log during mgs setup(fortestonly)

Params llog was introduced by LU-3155 to set parameters permanently
with lctl "set_param -P" instead of "conf_param". But in a newly
mounted filesystem, it won't be created automatically until "lctl
set_param -P" is called.
This patch is to start params llog during mgs initialization so that
the server can make a local copy successfully during its setup, and
the confusing console error message "LustreError: 13a-8: Failed to get
MGS log params and no local copy." can be quieted.
