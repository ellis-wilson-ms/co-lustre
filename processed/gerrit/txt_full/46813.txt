I don't know exactly the mechanism, but I just saw this in EX-3205 and I recall seeing a similar case where there are timeouts or some other error being returned (-EINVAL or whatever) and the code just keeps looping forever (not even 2^32 limit).  The default for "lfs df" was changed to not print errors because sites configure with gaps in the OST index, so it is too noisy. 

So this isn't fixing the root cause, but at least it eventually unblocks and returns an error code to the caller.
