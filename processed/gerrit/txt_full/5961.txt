Andreas, yes, the change in error() is _mostly_ a no-op change.  But bash will raise an exception on the original code, whereas with my version of the code it will not.

It is the difference between explicit control flow ("if" statement) and using a more implicit control flow with the && operator.  When you don't use an explicit "if" statement, you are telling bash: X must be true AND Y must be true.  If X isn't true, it raises an error (because you guys have that option turned on).
