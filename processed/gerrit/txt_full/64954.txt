LU-19921 mgc: change sec flavor for local mount

Improve the mgc sptlrpc config log process to support security
flavor change for local mount.

In previous implementation, the code would check if there is
cld already exist, then it will skip the sptlrpc log process.

In local mount(llmount.sh), MGC will be persistent across client
unmounts and remounts, since MGC are shared by both client and
server. and also the cld and their associated lock handles will
be kept, so the security flavor between client and server will be
changed by only umount/remount client.

The update explicitly calls mgc_process_log(obd, cld->cld_sptlrpc)
for every mount and also check the cld lock, in case it already
cached locally.
