LU-15069 llite: Remove unnecessary ras_align

There is an extra call to ras_align() in readahead_pages,
which attempts to align the readahead indexes to RPC
boundaries.

This seems fine, but is actually wrong in a few ways:
1. The readahead window is already aligned to stripe
boundaries by the cl_io_read_ahead code, and since RPCs are
divided by stripe, rounding the end of the window to
"pages per RPC" pages has no bearing on whether or not
good RPCs are formed.
2. We're just rounding the *end* of the window - with no
attention to where it started, so no attention to how many
total pages are in it.
3. We already grow the window by "pages per RPC" pages.

The alignment to stripes avoids the main case of concern,
which is that a large readahead window ends with a small
RPC when a few pages slip in to the next stripe.  And in
any case, this code didn't help with that.

So this code to align the window end to RPC count pages
isn't causing problems, but it's unnecessary and it makes
the readahead code even more complicated.
