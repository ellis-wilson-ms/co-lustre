LU-10460 rpm: Skip packaging lustre init for systemd

Don't install the lustre init script on systemd systems.  For
instance, starting in RHEL7, SysV is obsolete, replaced by systemd.
We choose not to add a systemd script at this time, instead favoring
an OCF Resource Agent script for starting lustre servers through
Pacemaker.

Installing the init script would result in SystemD generating a
service for it.  We would then need to ensure that the service is
never enabled, in production, so that it does not conflict with
Pacemaker.  We would also need to maintain the script in addition to
the Resource Agent; and we would need to ensure admins, who manage
both RHEL6 and RHEL7 systems, never start the service out of habit.

These considerations do not exist for some use cases, such as file
systems running on nodes without shared storage (e.g. for testing).

This patch was written for use at LLNL and likely is not sufficient
for general inclusion.
