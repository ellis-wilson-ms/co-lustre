--- Commit Message ---
LU-2079 llog: dump attributes in error case

this is a strange situation when non-zero size is returned
on an empty llog. the patch should dump attributes so we
can check what's wrong. at the moment the idea is that for
a reason llog was created with the wrong type of directory
(supposed to be a regular file) and for directories we do
not return size stored in zfs as it's equal number of names,
instead we calculate it as number of blocks multiplied by
blocksize.
