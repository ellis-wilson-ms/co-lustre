(1 comment)

A patch touching this many files is bound to have many conflicts and/or break in-flight patches.

At a minimum, the ptlrpc_req_finished() -> ptlrpc_req_put() renaming should be added to spelling.txt to warn people about this, since it is used fairly often.  Luckily, the Gerrit checkpatch spelling.txt is being updated from master daily, so it should be picked up pretty quickly *after* this patch is landed.

It seems reasonable to add a short-lived compat macro to allow in-flight patches to not break outright, then remove it in a later patch:

 #if LUSTRE_VERSION_CODE < OBD_OCD_VERSION(2, 15, 57)
 #define ptlrpc_req_finished(rq) ptlrpc_req_done(rq)
 #endif

possibly the patch to remove even the conditional code could be landed in a couple of months.

Other than the staging issues, I definitely agree with the name change.
