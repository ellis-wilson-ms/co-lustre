--- Message (Andreas Dilger, 2013-10-01 23:56:40.000000000) ---
Patch Set 1:

What is the intended use of this option?  The configured system that is left over after testing finishes depends on the test being run, in particular conf-sanity will change and reformat the filesystem several times. Instead of using "auster -K" you could just run the test directly and/or run llmount to get a configured filesystem?

--- Message (Nathaniel Clark, 2013-10-04 13:12:02.000000000) ---
Patch Set 1:

The purpose of this patch is to allow auster to leave a setup in place after running a test (for example after it run with --only)  This is just a way to be lazy and not need to run llmount before running auster.
