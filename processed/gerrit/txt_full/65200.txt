LU-20110 tests: fix slow recovery-small test_136

The check added to debug intermittent test failures was checking
if a server had restarted was itself timing out while waiting for
the restarted server.

It now wraps the uptime_info print attempt in a timeout call to
avoid it hanging if the server is down.

  env=SLOW=yes,FAILURE_MODE=HARD,ONLY=136 \

  fstype=ldiskfs testlist=recovery-small
