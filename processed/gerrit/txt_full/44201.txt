LU-13799 llite: Simplify sync_io references

The reference counting for sync i/o/aio involves careful
adding of an extra reference to make sure things aren't
freed early.

We can make this less weird by making it more like normal
reference counting:
The init and destroy functions hold one extra reference.

This simplifies things a bit.  We still have to free
memory differently for AIO and DIO, but this is still
a bit simpler.
