LU-19195 lnet: Generic LND tunable handling

Introduce generic handling of LND tunable parameters.

struct tunable_spec describes an LND tunable parameter.

lnd_tunables is an array of struct tunable_spec that describes every
LND tunable parameter.

set_lnd_tunable() and get_lnd_tunable() are functions for setting and
retrieving parameter values.

init_lnd_tunables() will initialize a struct lnet_lnd_tunables so
that each parameter is set to the special value which tells the LND
to use its internal default. The string "null" is used to specify a
string parameter that has not been defined by the user.

User-space functions that send or retrieve tunables to or from the
kernel, both the old ioctl and the newer netlink implementations,
leverage these new functions. When configuring an NI, the tunables
struct is always initalized with init_lnd_tunables(), and it is
always sent to the kernel regardless of whether the user has specified
any params. kfilnd_nl_set() is updated to understand (and ignore) the
traffic_class_num parameter.

With these changes, the lnetctl import command can now correctly set
all LND parameters.

The lnet_yaml_*_mapping helpers are used in the code paths related to
configure network interfaces. This is to reduce code duplication.

A typo in kfilnd_tunables_setup() is fixed where we were checking the
value of lnd_prov_major_version twice instead of checking it and
lnd_prov_minor_version.
