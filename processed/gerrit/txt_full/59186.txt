--- Commit Message ---
LU-18958 osd: improve quota declarations in setattr

Change osd_declare_attr_set() to avoid declaring transaction
credits for quota IDs that are not changing.  It is still
necessary to reserve transaction credits even if bspace=0,
because the inode accounting is still changed in this case.

This also quiets a bogus static analysis warning about 'bspace'
being uninitialized, even though the condition to enter both
code blocks is the same.
