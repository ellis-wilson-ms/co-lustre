LU-19502 scripts: add lnet_legacy2yaml conversion utility

Introduces the lnet_legacy2yaml script and accompanying man page to
facilitate migration from LNet kernel module parameter syntax
(ip2nets=, routes=, and networks=) to the YAML configuration format
used by lnetctl import (Lustre 2.10+).

The script supports multiple input modes:
- Direct command-line legacy strings
- Separate ip2nets, routes, and networks files (-i/-r/-n options)
- Modprobe configuration files (-m option)
- Mixed inputs (files + inline strings accumulate)
- Standard input when no positional argument provided

Key features:
- Preserves rule and interface ordering from legacy config
- Supports both legacy and alternative route formats:
  - Legacy: <network> [<hop>] <gateway>[@<exit>][:<priority>]
  - Alt: <network>: { gateway: <gw>@<exit> [hop: N] [priority: N] }
- Supports networks parameter format:
  - <nettype>[<number>][(<iface-list>)]
- Optional default hopcount assignment (--default-hop)
- Handles IP range patterns with wildcards and bracket expressions
- Auto-quotes YAML tokens that would be misinterpreted (e.g., '*')
- Supports modprobe file line continuations
- Validates syntax and reports errors for missing delimiters
