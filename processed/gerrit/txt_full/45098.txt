--- Commit Message ---
LU-15021 quota: protect lqe_glbl_data in lqe

The lqe_glbl_data in "struct lquota_entry" is allocated in
qmt_lvbo_init and freed in qmt_lvbo_free, it could be freed
during qmt_seed_glbe called by qmt_set_id_notify, and cause
panic because of using freed memory.

Patch Set 1: Code-Review-1

(5 comments)

There is also another approach to avoid using mutex. You can also add atomic references to protect lqe_glbl_data. Just increment/decrement the reference together or even directly in lqe_getref/lqe_putref. This way you need just to put lqe_glbl_data reference in qmt_vlbo_free.

lqe_gdata_putref(struct lquota_entry *lqe)
{
    if (atomic_dec_and_test(&lqe->lqe_glbl_data->lqeg_ref)) {
         lqe->lqe_glbl_data = NULL;
         qmt_free_lqe_gd(lgd);
     }
}
I see only one problem in such approach - we need to do something in qmt_lvbo_init if lqe_glbl_data != NULL: a) wait until it becomes NULL b) continue to use old lqe_glbl_data instead of allocating a new one.
