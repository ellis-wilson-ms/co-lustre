Looking at the removed LNDs, I'd think the only one of concern might be the ptllnd, which would be needed for some old Cray client to mount a newer server (the server would need to map the @ptl NID in its routing table to a specific network type).  I don't think there would be any old _servers_ directly attached to a cluster but exported via @ptl NIDs that a new client would connect to.

Since I expect this is a rare case, and I've added Patrick to the review list to see if there are any sites that may need this, I'm OK with the current change, and I'll add a comment to the commit comment to that effect.
