Patch Set 1: Code-Review-1

I understand a few of the failures that this patch encountered with the custom interop test session:
1. In sanity test 27D, the variable to skip some tests is not properly set. See https://review.whamcloud.com/#/c/30725/. 

2. The failures in test 77c are due to, we test with old servers, we exit the test early and this exit hits the trap set and calls the cleanup routine cleanup_77c(). cleanup_77c() uses local to tes 77c variables. When cleanup_77c is called at the end of the test, the local variables are in scope. When cleanup_77c is called as part of a trap, they are not in scope. I need to open a ticket for this.

I haven't reviewed the other sanity failures yet.
