LU-19475 kfilnd: GDS support for kfilnd

Add GDS support to kfilnd.

Add config check to see if kfabric supports the KFI_SGL IOV type.

When kfabric supports KFI_SGL:
 - struct kfilnd_transaction will contain a struct sg_table and
   associated fields to leverage the LNet or Linux dma APIs:
   - lnet_rdma_map_sg_attrs()/lnet_rdma_unmap_sg(), for GPU pages
   - dma_map_sgtable()/dma_unmap_sgtable(), for CPU pages
 - kfilnd_tn_set_sgl_buf() maps the struct bio_vec provided by LNet
   to the scatterlist and then calls the appropriate "map" API.
 - The appropriate "unmap" API is called by kfilnd_tn_finalize() when
   the transaction has completed.
For older kfabric that does not support KFI_SGL we continue to use
the KFI_BVEC IOV.

Replace calls to kfilnd_tn_set_kiov_buf() with kfilnd_tn_set_buf().
kfilnd_tn_set_buf() will call either kfilnd_tn_set_sgl_buf() or
kfilnd_tn_set_kiov_buf() depending on whether kfabric has KFI_SGL
support.

For immediate messages, kfilnd_tn_set_kiov_buf() is essentially a
noop. It's only functional purpose was setting the tn_num_iovec and
tn_nob fields of the struct kfilnd_transaction. This was so they
could be used later in kfilnd_tn_pack_immed_msg() as arguments to
lnet_copy_kiov2flat(). lnet_copy_kiov2flat() is now called by
kfilnd_send() so that kfilnd_tn_set_sgl_buf() needn't bother setting
those fields in struct kfilnd_transaction.
