Patch Set 1: Looks good to me, approved

Too be honest, there is no reason why this needs to be specific to ZFS. This test is related to evicting patches from the client cache, not from disk, so it could be async for any filesystem.  I guess the only question is whether the pages are currently part of a write RPC, which might be why the sync was added to ensure the pages are not locked.

If there is only a single page written then it shouldn't be undergoing write if the eviction happens immediately afterward, unless the eviction is so slow that it allows the page to start being flushed.
