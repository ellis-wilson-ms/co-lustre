LU-20197 lod: fix bogus memcpy in lod_layout_repeat_comp()

When an SEL component with an explicit OST list is repeated into
a new component, llc_ostlist.op_array is deep-copied so the new
component owns its own buffer.  The memcpy passes the address of
the op_array pointer rather than the pointer itself, so the new
buffer holds pointer bytes instead of OST indices.

The corruption is masked from user space by lod_sel_handler()'s
sd_force unwinding in the common case, but garbage bytes that
happen to fall in [0, OSTCOUNT-1] would let the new component be
created with stripes outside the user's -o list.

Fixes: 76ca884398ca ("LU-10070 lod: SEL: Repeated components")
