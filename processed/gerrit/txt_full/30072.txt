Patch Set 2:

> This looks like it should actually be MDS_ATTR_ATTR_FLAG,
 > since not all of the ATTR_* flags match with the MDS_ATTR_* flags.

That comment turns out to be incorrect, but now in the opposite direction.  op_attr is struct attr, so ATTR_* flags are correct.  The problem is that there is no "ATTR_ATTR_PROJID" in that namespace, so there is a risk of collision with other "ATTR_* flags (the 0x8000 flag is currently used by "ATTR_TIMES_SET", as is seen in ll_setattr()).

We can't change the value of MDS_ATTR_PROJID, since this is already used over the network.  Instead, we should declare an ATTR_PROJID flag and map it properly in attr_pack(), like with ATTR_CTIME_SET, that avoids conflicting with other flags.  Since this is in-memory only (if handled correctly) it doesn't matter if it needs to be changed in the future, which is the whole point of keeping it separate from MDS_ATTR_* flags that need to stay the same forever.
