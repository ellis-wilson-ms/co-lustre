--- Commit Message ---
LU-6684 lfsck: set the lfsck notify as interruptable

If the LFSCK engine is notifying the remote LFSCK engine about some
LFSCK event, such as LE_PHASE1_DONE, but if the remote server (MDT
or OST) is offline, then such notification RPC will be blocked until
the remote server is online. At that time, if someone wants to stop
the LFSCK, he/she has to wait.

To avoid such trouble, we will make the LFSCK notification RPC to
be interruptable. Then even if some remote server is offline, the
running LFSCK still can be stopped.

Patch Set 3: Code-Review+1

shouldn't we also make control RPC non-resendable so that unavailable target doesn't block running LFSCK ?

--- Message (Fan Yong, 2016-01-27 01:02:43.000000000) ---
Patch Set 3:

For a large cluster, it will be normal that some MDT/OST become unavailable temporarily during the LFSCK because of network trouble. From the LFSCK view, it is NOT easy to distinguish such case from the server down. If we set the control RPC as non-resectable, that will increase the possibility of scanning the system partially. Although we can rescan the system when next LFSCK run, but next time, there may be other OST/MDT unavailable temporarily. Finally, there almost will be no completely LFSCK scanning. That is my worry about.
