--- Commit Message ---
LU-3485 tests: Add resize2fs testing for OST/MDT filesystems

Add testing of the MST and OST filesystems with resize2fs.
Currently, only offline resizing is possible until LU-137
is finished. Thus, each file system is unmounted and
e2fsck is run before and after the resize to check for
errors.

We are restricting the number of OSTs since large-lun
test 2 run time is based on the number of OSTs and can
time out if there are 16 OSTs as in the last large-lun
test results. We've also increased the verbose flags
to llverfs to track where all the time is being spent.

Test-Parameters: testlist=large-lun ostcount=4 \
mdsfilesystemtype=ldiskfs mdtfilesystemtype=ldiskfs \
ostfilesystemtype=ldiskfs

Patch Set 1: I would prefer that you didn't submit this

(11 inline comments)

Tests look generally good, though some cleanups are needed. 

One thing that came to mind while inspecting this is that resize of a filesystem with data in it to a smaller one may result in inodes being moved during the rename in order to be able to free the itable blocks in the last group(s) being removed. This may result in an inconsistency in the OI mapping for these inodes.

A question for Fan Yong is whether OI FID-to-inode lookups verify the found inode with the LMA FID each time, and trigger an OI scrub if not?
