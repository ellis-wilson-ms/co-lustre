LU-9119 lnet: fix peer discovery shutdown

The code in lnet_peer_discovery() that handles shutdown of
discovery doesn't properly free resources and uses
lnet_for_each_entry() in an unsafe manner.

When shutting down, do the following:

1) Walk the working queue and call lnet_peer_discovery_timeout()
   to stop any Ping or Push in progress on the peer. All peers
   are moved to the expired queue.

2) Wait for the event handler to run. This will move all peers
   from the expired queue to the request queue.

3) Clear the request queue by calling lnet_peer_discovery_error()
   and lnet_peer_discovery_complete() on each peer. This removes
   the peers from the request queue.

At this point the discovry queues are all empty and peer discovery
shutdown is complete.
