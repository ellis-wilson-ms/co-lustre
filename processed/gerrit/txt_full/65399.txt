LU-17158 ptlrpc: Per-rule per-CPT TBF statistics

Per-rule per-CPT statistics are added to nrs_tbf_rule:
 - enqueue: Total requests matched to this rule
 - dispatch: Total cost of all dispatched requests
 - exhausted: Total times bucket ran dry
 - min_used: Times minrate floor was used
 - elapsed_ms: time since stats were reset / rule created

dispatch and elapased_ms enables user to calculate achieved rate over
some time interval:
 rate = dispatch * 1000 / elapsed_ms

All are plain __u64 updated under scp_req_lock. Stats are only shown
when yaml format is requested. e.g.:
lctl get_param --yaml ost.OSS.ost_io.nrs_tbf_rule
- cpt: 0
  rules:
    - name: example
      conditions: "jobid={jobid}"
      rate: 1000
      ref: 0
      enqueue: 0
      dispatch: 0
      exhausted: 0
      min_used: 0
      elapsed_ms: 2075

Status for all rules can be reset with, for example:
lctl set_param ost.OSS.ost_io.nrs_tbf_rule="reset_stat"

Or reset a specific rule:
lctl set_param ost.OSS.ost_io.nrs_tbf_rule="reset_stat rule_name"
