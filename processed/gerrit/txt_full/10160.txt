--- Commit Message ---
LU-3953 build: fix paths for generated headers

Headers that are generated during kernel build and placed
in various include/generated directories are always in
the kernel build dir. There are several places in which the kernel
source dir is used instead. This causes configure/build failures
on systems using separate build and source directories.

This patch cleans those paths up.

I can't comment on the patch itself, but I've resubmitted the patch for testing due to LU-5097. 

Jeff,
I appreciate you sending in patches for these issues, since they keep us on top of newer kernels better than we can ourselves. That said, if you need to submit a series if patches repeatedly in the future, it would be great if you could contact me or Oleg or Bon so that we can cancel the builds for the older versions of the patches. As it was, the 50 or so patches you submitted in quick succession kept our build system busy for a good part of last week because they also triggered full kernel builds (which can take 2h or more) due to the autoconf and include changes precluding the use of previously built kernels. 

Also note that it is now possible to edit commit comments directly in Gerrit. This will still generate a new build (until that is fixed or the build is cancelled), but at least it doesn't cause all the dependent patches to be rebuilt also, unlike re-pushing the whole series each time.
