LU-10756 ptlrpc: send uevents when import state changes

When the import for a client changes state such as when the client
is evicted send a uevent to make user land so it can be aware that
the import state has changed. This allows the use of udev rules to
perform site specific actions.

This is the general idea, and having a Lustre subsystem in general is very useful.

In this case, which nodes could a Udev rule be set on to handle client state changes?

--- Message (James Simmons, 2018-03-02 16:27:24.000000000) ---
Patch Set 3:

I looked on the server side to see what events are generated. For this patch it generates uevents for import state changes. So on clients the udev rules detect when the remote target state has chanegs i.e lustre-MDT0000_UUID. Now on the server the import state change is also detected. In that case we see server to server interactions.

In order to detect if clients disappear on the server side we have to
report the "export" state change. We can discuss what is needed on ticket LU-10756. Adding this is usually pretty easy to do. Don't know if it will make it for 2.11 tho. This can make it already if I get one more review. Hint Hint :-)
