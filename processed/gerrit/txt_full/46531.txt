LU-15549 lod: layout version shouldn't be zero

OFD doesn't check layout version if OBD_MD_LAYOUT_VERSION flag is
not set in OST_WRITE. in turn the client doesn't set this flag if
layout version is 0. All regular layouts start with layout version
equal 0. Thus if a client, caching such a layout with version = 0,
gets evicted from MDS it can keep sending OST_WRITE RPCs and OFD
won't be checking layout version silently execting RPCs while the
layout may have changed by this moment. Data can get corrupted.
