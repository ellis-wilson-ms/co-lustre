LU-5657 doc: allow the use of rst2man to build man pages

The man page sources can now be written in reStructuredText (rst), and
the man pages will automatically be generated with rst2man.

Added a build dependency on rst2man and the package python-docutils.

Converted lustreapi.7 to ReST to validate the solution.

--- Message (Andreas Dilger, 2014-09-25 18:32:40.000000000) ---
Patch Set 3: Code-Review-1

Adding new dependencies to external tools significantly increases the complexity of building Lustre. We need to build on a growing number of distros, and the fewer dependencies, the better.

I really so appreciate your efforts to write these man pages, but I'd prefer that they stick with the standard nroff formatting that other man pages use.

--- Message (Frank Zago, 2014-09-25 19:30:25.000000000) ---
Patch Set 3:

python-docutils is not an unreasonable dependency. It's present on RHEL 6/7 and derivative, Suse, Debian/Ubuntu, and a few others like Arch, Mint or Mageia.



The upside is that one can now write a man page in a readable format. Writing man pages in nroff is just another reason not to write them (and there's quite a few missing in Lustre).



Something that could work is to have a cache of generated man pages. If rst2man doesn't exists, then the cached copy is used instead. Would that be ok with you?

--- Message (Andreas Dilger, 2014-09-25 19:58:42.000000000) ---
Patch Set 3:

I personally don't think that the .rst format is significantly more readable than nroff, though I may be biased by having used nroff for a long time. Both are markup languages that can be edited in a relatively simple manner.  I don't think the actual markup format is the reason these man pages don't exist, but rather oversight on the part of the developers. 

The problem that I see with having both .rst and .N files in the tree is that it is entirely likely that there will be version skew between the two files. In the past we also had Perldoc format files and needed "pod2man" to convert them to nroff format.  However, this resulted in some developers updating only the nroff files and caused added maintenance needed to update the .pod files.

--- Message (Robert Read, 2014-09-25 20:51:15.000000000) ---
Patch Set 3: Code-Review+1

I believe there is a significant improvement in readability in the RST files over troff, and it does  make it substantially easier to write, maintain, and even read our man pages. It is also format we could use for other documentation, as well. This is a fairly standard tool in the Python community and should be available in all modern distros.  The problem of developers editing the wrong file is easy to solve - don't commit the generated man files.
