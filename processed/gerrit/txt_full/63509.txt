--- Commit Message ---
LU-12187 tests: sanity-lfsck/20[ab] should allow 'hole'

The "lfs getstripe" pattern output for a layout with a 'hole' in it
(missing OST objects) is being changed in the EC patch series.
Previously it just printed a numeric layout pattern "40000001", but
now it will print "raid0,hole", which is much more useful for users.

Update sanity-lfsck test_20a and test_20b to accept both formats.

Test-Parameters: trivial
