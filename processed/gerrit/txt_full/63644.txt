--- Commit Message ---
LU-18838 lli: Fix sax use race condition in ll_statahead_thread

At the end of the thread it's unsafe to use a cached lli sax value
because ll_sai_put could lead to dentry/inode being freed already.

Cache the sai context before ll_sai_put call to be used with
ll_sax_put and make sure it's not NULL as well as there are a few
places in the code tha could actually free it, the most common likely
being the statahead deauthorize.

Refactor ll_sax_put to not require inode argument because a suitable
inode is already referenced from inside the sax itself, so just use
that.
