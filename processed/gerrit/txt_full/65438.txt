LU-20151 nodemap: skip dynamic nodemap clear when none exist

To ensure dynamic nodemaps are properly removed if the MGS is
co-located with an MDT/OST, dynamic nodemaps are cleared on every
ioctl that involves updating the nodemap config. This is important
because on non-co-located targets, changing the nodemap wipes the
config including dynamic nodemaps. Clearing dynamic nodemaps involves
iterating the full nodemap hash even when no dynamic nodemaps exist,
making bulk operations O(n^2). As a result, sanity-sec test 26
regressed from ~400s to ~2000s.

This patch tracks the number of dynamic nodemaps in the
nodemap_config, so the MGS ioctl path can skip clearning dyn. nodemaps
when none exist. As a result, test 26 completes in similar completion times as before.
