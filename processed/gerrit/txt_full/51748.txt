LU-16975 lnet: setup all available interfaces

Currently, setting up LNet with socklnd or o2iblnd
only automatically sets up the first network
interface. Unless a user knows to manually setup
the remaining interfaces, their node will experience
subpar network performance. This patch changes the
default behaviour to automatically setup all
available interfaces.

Unfortunately, there does not appear to be a way to
cleanly do this at the LND layer. NI are passed down
from LNet to the LND via lnd_startup to be setup
one-by-one. Hence, this change is in LNet itself.

As for the remaining LNDs, filnd appears to support
only a single interface. gnilnd looks like it would
need a different mechanism to grab and setup all
interfaces. Hence, both are outside the scope of
this patch.
