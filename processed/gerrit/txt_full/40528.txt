--- Commit Message ---
LU-14111 obdclass: count eviction per obd_device

Add a new 'obd_eviction_count' counter to obd_device which
is increased every time a client is evicted, which means
every time we call `class_fail_export()`.

Expose this counter through `lctl get_param *.*.eviction_count`
for every target.

I'm not against listing the count in multiple places. 

As I see it, the problem with the "one value per file" paradigm is that it requires monitoring tools to open and read hundreds or thousands of different files on a regular basis, and that is very inefficient. The upstream kernel folks are just starting to come to this realization, and trying to do crazy things like adding a syscall to read from an array of fd's at the same time, but it is just more efficient to have all of the values in a single file that is formatted for easy parsing (YAML).

So, if you want, adding a separate file for the count is OK, but I'd like it in an aggregate file also for efficient monitoring.
