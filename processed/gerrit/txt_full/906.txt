--- Message (Andreas Dilger, 2011-06-07 17:31:56.000000000) ---
Patch Set 2: Looks good to me, approved

I know that there are a lot more such messages in the Lustre source that should be changed as well.

Rather than hard-code them in the source, however, I think it best to create something like one or both of:

#define LUSTRE_BUGREPORT_URL "http://bugs.whamcloud.com/
#define LUSTRE_SUPPORT_URL   "http://wiki.whamcloud.com/

and then use those in the source so that they can be updated more easily.  For example lustre/lvfs/fsfilt_ext3.c, lustre/obdclass/class_obd.c, lustre/obdfilter/filter.c, lustre/obdfilter/filter_io_26.c, lustre/osc/osc_request.c, and lustre/utils/mkfs_lustre.c all have "contact" URLs.

There are also a few "quaint" error messages still left in the code that could be replaced at the same time:

llu_inode_revalidate()
                CERROR("REPORT THIS LINE TO PETER\n");

__ll_inode_revalidate_it()
                CERROR("REPORT THIS LINE TO PETER\n");
ll_lookup_it()
                CERROR("Tell Peter, lookup on mtpt, it %s\n", LL_IT2STR(it));
ll_unhash_aliases()
                CERROR("unexpected NULL inode, tell phil\n");


Note also that the URL for the GPLv2.pdf file in the header does not work anymore, so we might consider replacing it with a URL that does work, but that should be done in a separate patch.
