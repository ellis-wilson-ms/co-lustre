LU-5011 lustre: don't export kernel types to userspace

THIS IS A DRAFT PATCH FOR DISCUSSION I DID NOT TRY TO FIX ALL OF
THE TAB ALIGNMENT OR WHITESPACE ISSUES. PLEASE DO NOT CLUTTER THE
DISCUSSION HERE ABOUT HOW I CAN FIX THIS OR THAT HERE OR THERE.
THIS MEANS YOU ANDREAS!

Remove libcfs/posix/posix-types.h. Add a new header libcfs/types.h
which imports the fixed-width types intXX_t and uintXX_t for use in
packaged headers like lustreapi.h and lustre_user.h and in userspace
components. Rewrite those headers to use the new types. Including
these headers will no longer define the __{s,u}XX types unless Lustre
is being built. Remove the autoconf tests for the presence of
__{s,u}XX, and for the long-longiness of the primitive type used to
define __u64.
