--- Commit Message ---
LU-4249 quota: fix race on setting quota lock data

In qsd_intent_lock(), we set the lock data as lqe (or qqi) when
creating lock, and take the lqe (or qqi) reference after lock
created, which introduced a small race window which can lead to
extra reference drop on lqe (or qqi):

- qsd_intent_lock() calls ldlm_cli_enqueue()->ldlm_lock_create();
- ldlm_lock_create() created lock and set lock data succefully;
- OST disconnect with MDT, cleanup_resource() clears all quota
  locks, that'll trigger blocking ast on the newly create lock;
- qsd_id_blocking_ast() drops lqe refcount;
- ldlm_lock_create() continue and fail on lvb allocation;
- qsd_intent_lock() fail out without taking lqe reference;
