--- Message (Andreas Dilger, 2013-09-19 15:31:00.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

I don't think brw_stats makes any sense for LOD devices. LOD is the abstraction layer for multiple OSPs on the MDS, so it isn't even clear which OSD the statistics would be for.  Also, in the LLNL case they are using ZFS and not ldiskfs, so no brw_stats file even exists to symlink to. 

Per my second comment in LU-3300, what is needed is the subset of the brw_stats data that can be generated by osd-zfs.  Then, the current mechanism to symlink this into ofd at startup time.
