LU-0000 mdt: fix ACTIVE_WRITER lock cancellation

ACTIVE_WRITER locks created for immediate mirror files were
not sending cancellation RPCs to the server when cancelled
on the client side. This caused the server-side active
writer count to remain elevated and not get decremented to 0.

The issue was in mdt_intent_lock_replace() where ACTIVE_WRITER
locks were not properly configured for client-server
communication:

1. l_conn_export was NULL, preventing cancellation RPCs from
   being sent to the server
2. LDLM_FL_LOCAL_ONLY flag was not being removed, causing
   client-side local cancellation instead of server RPCs

This fix ensures that when ACTIVE_WRITER locks are sent to
the client, they have the proper export connection and flags
for distributed cancellation, allowing the server-side active
writer count to be properly decremented when locks are
cancelled.
