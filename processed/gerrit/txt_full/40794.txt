I agree there is a possible problem here.  I don't think this is the best solution.
This is an rhl table, meaning that there can be multiple entries with the same key.  These entries will all be linked together.  rhltable_lookup() returns the list and rhl_for_each_entry_rcu() can extract the entries.
There will still be a need to restart the walk each time an entry is deleted (I think), but we don't need to rewalk the entire table, just the short list.

I'll look at writing a patch.
