--- Commit Message ---
LU-4998 lustre: unpackage several headers

Remove lustre_idl.h, libiam.h and lustre_lfsck_user.h from the list of
packaged headers. Move conf-sanity 74a (checking that Lustre client
api program can compile and link) to sanity 400a. Add sanity 400b to
check that the headers installed in /usr/include/lustre/ (with the
exception liblustreapi.h) can be compiled without warning.

I feel like test 400a in sanity always failing on developer systems is bad.
This is something people are supposedly run all the time. The failure also has pretty long output which is annoying.

We should at least check that /usr/include/lustre is there and skip the test if not. I kind of dislike how it ignores other potential package locations, but inventing a special autodetect logic just for this is overkill, so just fixed locations is probably ok as long as we make sure it's there.

It would not be bad to test that $CC actually is there and works too before drawing any conclusions.
