Is ffff for kernel pointers x86 specific?  I thought ARM was the same, with kernel pointers in just the upper part of the address space?  (Perhaps Shaun knows for sure?)  Obviously that *is* possibly architecture specific, I just think it might be true on both archs of interest.  (But I've never really wrangled an ARM kernel dump, so...)

My thought was that maybe this is failing when something is a little strange in the config, like an OST is offline - because ost1 should almost always be OST0000.  It's not that using OST0000 rather than the import name is correct - it's obviously not - but it's a little hard to see what would cause it to fail.
