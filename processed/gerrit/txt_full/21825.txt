--- Message (Andreas Dilger, 2016-08-11 19:37:46.000000000) ---
Patch Set 2: Code-Review-1

(7 comments)

I don't think this patch is correct, since it pushes the fix to the client instead of on the server where it belongs.  Consider the case of a server with ZFS recordsize=16MB and an old client using ocd_brw_size=1MB (e.g. unpatched 2.5.x).  In that case the client would get back ocd_brw_size=16MB and do something bad.

This also needs a conf-sanity test case which sets the server brw_size=4MB and the client max_pages_per_rpc to some odd value (e.g. 7*128KB and 11*128KB) and ensures that it doesn't trigger the LASSERTF() logic.

I agree that there does need to be a fix for the client to prevent it from hitting the LASSERTF() with an older server, but the first fix should be that the OFD sets ofd_brw_size = max(ONE_MB_BRW_SIZE, os->os_bsize) in ofd_init0().
