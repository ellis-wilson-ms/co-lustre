--- Commit Message ---
LU-20076 tests: speed up sanity test_123a

Sanity test_123a can run too long in Janitor testing while trying
to create a very large number of files for the test.  Exit the
test loop at the end of the iteration if creates take too long.

Use "find | xargs unlink" to delete the files instead of "rm -r"
to avoid taking forever for "rm" to stat all of the files first.

This subtest was timing out on Janitor after 60 minutes, and can
now finish in about 30 minutes.
