LU-5937 llite: merge attr for striped dir in dir_ioctl

In ll_dir_ioctl (LL_IOC_MDC_GETINFO), if the inode is
a striped directory, it should merge the attributes
from all of stripes. So lfs find --size can get correct
result.

Remove some unnecesary merging in ll_revalidate_slave(), and
the attributes will be stored in each stripe, only
merging them if required.

There is a specific reason to do ioctl(GETFILEINFO) using { parent, name } instead of ioctl() on each file separately - to avoid opening, locking, and closing each file on the client if that is not needed. The GETFILEINFO ioctl can be done on the patent dir with a single open for all files in that directory, and it also doesn't pollute the client dcache with millions of dentries when traversing a large filesystem.

I see that there are several issues other than LU-5937 that are addressed by this patch. I'm sorry to hold up their fixes but they should have been put in a different change. While this seems to be correct, it's the wrong fix. For a regular file we do not assume that these ioctls return fully valid attributes. Instead if the size (or other attribute) is really needed then we call IOC_LOV_GETINFO later on. I suggest that directories be handled in the same way but by calling fstat() instead of an ioctl.

In this case we also do not need to exclude the use of IOC_MDC_GETFILEINFO for directories. Which is good because we are also using this ioctl when we have dentries with d_type == DT_UNKNOWN.
