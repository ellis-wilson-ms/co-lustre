--- Commit Message ---
LU-957 scrub: support MDT file-level backup/restore

Lustre-2.x uses Object Index files for mapping global FID to MDT local
identifier, which is different from Lustre-1.6/1.8. Such OI file wills
become invalid after MDT file-level backup/restore, because of objects
re-allocated on MDT.

The patch rebuilds the OI files after MDT file-level backup/restore to
make it workable as Lustre-1.6/1.8 series do.
