LU-11025 dne: refactor dir migration

Tidy up directory migration code to make it cleaner to add
directory split/merge, also some small fixes:
* add MDS_MIGRATE_NSONLY flag to migrate dirent only, but not inode.
  It's reserved for directory split/merge: don't migrate subdirs,
  only remove dirent from stripe before layout change, and add dirent
  into stripe after change.
* handle migration differently for different case:
  1. migration inode:
    a. create target, if it's striped directory, migrate stripes to
       target.
    b. migrate xattrs.
    c. if it's non-directory, update namespace for links since FID
       changes.
    d. destroy source if it's not a plain directory.
    e. update namespace.
  2. update namespace only if no need to create target, there are
     three cases:
    a. file happens to be located on MDT where target parent stripe
       is, this happens when file is a remote object.
    b. file has link on its MDT.
    c. 'lfs migrate -m -r' is run, in this case, subdir doesn't
       change its layout, norally it will become remote directory
       after migration.
* in lod_xattr_set(), use LU_XATTR_REPLACE to update directory LMV,
  add LU_XATTR_APPEND to append source stripes after those of target,
  add LU_XATTR_DELETE to delete source stripes after migration.
* __lmv_name_to_stripe_index() takes care of current layout is old
  or new, and lmv_name_to_stripe_index() is mapping on new layout,
  while lmv_name_to_stripe_index_old() is mapping on old layout.
* add LPROC_MDT_MIGRATE in md_stats to count files migrated.
* other cleanups.
