--- Commit Message ---
LU-6098 tests: start debug_daemon before server

Alter test-framework.sh:setupall() to start the debug_daemon
before mounting the servers, instead of waiting to start
it just before mounting the client.

This makes examining the server startup process easier.

De-duplicate the host list in do_nodes() to
eliminate the potential race condition created when
commands are run multiple times on a single host.

Remove unnecessary duplicate *mount_client* calls,
no functional change.
