--- Commit Message ---
LU-5422 obdclass: Fix null pointer derefs in lu_cache_shrink()

When lu_cache_shrink() is called on a machine with no Lustre targets
mounted, the following null pointer dereference may happen:

  BUG: unable to handle kernel NULL pointer dereference at (null)
  IP: [<ffffffffa0aed076>] lu_cache_shrink+0x156/0x310 [obdclass]

This is a regression introduced recently by 8701e7e.  lu_sites must be
initialized before the lu_sites shrinker is registered.  And, it
cannot be re-initialized every time when an lu_site is initialized.

Patch Set 1:

The reason I moved from LIST_HEAD_INIT to INIT_HEAD_LIST was to make merging upstream easier and avoid conflicts on other potential platforms. Currently we have to do something like

static struct list_head lu_sites = LIST_HEAD_INIT(lu_sites);

to avoid the conflicts with the list handling in POSIX queue.h.The problem is the upstream maintainers would 
prefer if we did LIST_HEAD(lu_site) which is what would
conflict. So to avoid that case I just port it to INIT_HEAD_LIST which doesn't have these issues.
