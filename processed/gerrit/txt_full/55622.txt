--- Commit Message ---
LU-17993 llite: not crook unalignment in direct IO

To fix interoperability with old server which does not support
unaligned DIO, the unalignment is bend in direct IO path, and that
could cause the assertion:

"ASSERTION( i == pv->ldp_count ) failed" in ll_direct_rw_pages()

This patch helps the interop issue without crooking the unalignment by
not shortening the iov_iter for DIO so that the DIO size passing in
ll_direct_IO_impl() would be page aligned.

Fixes: a3a0cced41 ("LU-16025 llite: allow unaligned DIO reaching EOF")
