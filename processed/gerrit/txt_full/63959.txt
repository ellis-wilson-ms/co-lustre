LU-12669 lov: fix EC recovery skipping pages for aligned reads

When the read size is an exact multiple of stripe_size,
pg_seq_end wraps to 0 due to modulo arithmetic:

  pg_seq_end = (crw_bytes >> PAGE_SHIFT) % pgs_per_stripe;

For example, a 5MB file with 1MB stripes: 1280 % 256 = 0.
After adding pg_seq_start, pg_seq_end == pg_seq_start and
the page processing loop "for (j = start; j < end; j++)"
executes zero iterations.  EC recovery produces no data,
causing data corruption (md5sum mismatch).

Fix by checking for the modulo-wraps-to-zero case and
setting pg_seq_end to pgs_per_stripe (a full stripe)
before adding pg_seq_start.

Resolves data corruption in tests 42c and
46a through 46c where file size is an exact
multiple of stripe_size.
