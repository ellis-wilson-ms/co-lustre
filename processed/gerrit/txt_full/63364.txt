LU-11509 ldlm: fix lfru priv_ratio_limit overflow issue

Change the type of ns_lfru_priv_ratio_limit_256 from __s8 to __u8
to prevent potential overflow issues during ratio calculations.

Additionally, enforce a valid range for priv_ratio_limit between
10% and 70%. This ensures the LFRU privileged pool maintains a
reasonable size relative to the total cache.

## Fix Explanation

**The Problem:**
The compile error occurred because the `%` character in the format string `"10-70%:"` was being interpreted as the start of a format specifier by the C compiler. When the compiler encountered `%:`, it didn't recognize `:` as a valid conversion type character, resulting in the error:
```
error: unknown conversion type character ':' in format
```

**The Solution:**
In C format strings, to print a literal `%` character, you must escape it by using `%%`. I changed:
```c
CWARN("%s: ratio limit '%s' must be between 10-70%: rc = %d\n",
```
to:
```c
CWARN("%s: ratio limit '%s' must be between 10-70%%: rc = %d\n",
```

**Rationale:**
This is a standard C convention - `%%` in a format string produces a single `%` in the output. This fix follows the existing pattern in the Lustre codebase (as seen in `lustre/osc/osc_page.c` where `%%` is used to print percentages). The fix ensures the warning message will correctly display "10-70%" when printed, while satisfying the compiler's format string validation.
