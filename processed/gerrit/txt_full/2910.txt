--- Commit Message ---
LU-56 ptlrpc: clean up ptlrpc history

Each ptlrpc service is using service::srv_hist_lock to serialize
all operations on history requests. In current implementation all
requests will be enqueued to history list while incoming and dequeued
from history list on freeing, no matter RPC history is enabled or
disabled. This is unnecessary because RPC history is disabled by
default, user enables it for only for problem diagnosis.
This patch will bypass history list operations if RPC history is
disabled, it will avoid performance issues on partitioned ptlrpc
service in the future.
