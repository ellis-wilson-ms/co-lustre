--- Commit Message ---
LU-16429 dom: batch open|create and write for DoM files

This patch implements open|create-ahead for DoM-only files. And it
returns DOM|OPEN ibits lock with PW mode to the client, thus the
subsequent write can perform in buffered I/O mode quickly under
the protection of the [0, EOF] DOM lock.

Combined with the feature of batching massive small buffered
writes, it is expected to improve the performance of IO500
mdtest-hard-write.
