--- Commit Message ---
LU-15231 lod: make sure obd device is set up

In lod_add_device(), we could find a obd device which
failed the class_setup() and not yet cleaned up.
Using this kind of obd device could trigger the
assertion
LASSERT(obd->obd_lu_dev->ld_site == lod->lod_dt_dev.dd_lu_dev.ld_site);

Fix another issue where if an obd device failed class_setup,
currently there's no error handling and cleanup/tear down.
e.g. if osp_init0 fails during mount, as a result class_setup() fails
but the obd device remains and have an active reference on osp module,
making lustre_rmmod impossible and mount fails with EEXIST as the
obd device is already there.
