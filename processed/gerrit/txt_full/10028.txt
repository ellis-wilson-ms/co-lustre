(4 comments)

I agree with John that it would probably be better to allow userspace to specify the initial mode.  It looks like that should be easily possible through the data->ioc_flags field and the "flags" parameter for llapi_dir_create_pool(), which is initialized to zero currently.

Rename the llapi_dir_create_pool() "flags" parameter to "mode", add "mode" as an argument to ll_dir_setdirstripe() and extract it in ll_dir_ioctl(LL_IOC_LMV_SETSTRIPE):

 #if LUSTRE_VERSION_CODE < OBD_OCD_VERSION(2, 9, 50)
        mode = data->ioc_flags ?: 0755;
 #else
        mode = data->ioc_flags;
 #endif

Initially this wouldn't allow creating directories with mode 0000, but IMHO that is unlikely to happen, and is no worse than what is happening today, and allows forward compatibility for applications that are using llapi_dir_create_pool() without specifying the mode.
