LU-15805 wbc: avoid deadlock when uncache a striped dir

In LMV layer, for striped directory, we can not know parent stripe
FID without name when open by FID (via the flag MDS_OPEN_BY_FID),
thus we can set it to the child FID, and MDT will obtain it from
linkea in open.

In such case, we skip to early lock cancel for the parent stripe
FID which was set with the child FID. Otherwise, it will cause
deadlock when uncache a striped directory via "$LFS wbc uncache
$file" and reopen the directory under WBC:
	rwsem_down_write_slowpath+0x32a/0x5e0
	wbc_inode_lock_callback.cold.46+0x25/0x13d [lustre]
	ll_lock_cancel_bits+0x5b6/0x990 [lustre]
	mdc_resource_get_unused+0x43/0x60 [mdc]
	mdc_intent_open_pack.cold.25+0xb0/0x1492 [mdc]
	mdc_enqueue_base+0x5c0/0x23d0 [mdc]
	mdc_intent_lock+0x21e/0x580 [mdc]
	lmv_intent_lock+0x19c/0x390 [lmv]
	ll_intent_file_open.cold.68+0x25d/0x52b [lustre]
	ll_file_open+0x415/0xd40 [lustre]
	wbc_reopen_file_handler.isra.29+0xc3/0x300 [lustre]
