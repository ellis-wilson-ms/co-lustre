--- Commit Message ---
LU-9771 osd: make osd_write_lock to be recursive

It's really bad that there is no rule for LOD and MDD to take object
write lock.

For layout change operations, it really needs to take the lock from
declare stage to the completion of operation because it depends on
the decision that makes on declare stage. In this patch, it revises
the programming model on the MDT stack to hold object write lock
before starting transaction.
