Thank you for the comments, Aurelien.
By default, the right path is autodetected by pkg-config in lustre/autoconf/lustre-core.m4 while running configure command. This patch doesn't change that. It resolves the issue of specified "--with-systemdsystemunitdir" option not passing to the configure command in "make debs".
In addition, "lib/systemd/system" is hardcoded in debian/lustre-{client,server}-utils.install.in, which is incorrect on Ubuntu 24.04 and causes "dh_install -p lustre-client-utils" fail to find the path. So, this patch updates that path with the correct value specified by "--with-systemdsystemunitdir" option, the value of which is autodetected by pkg-config.
