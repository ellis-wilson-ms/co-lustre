--- Commit Message ---
LU-19253 target: get all NIDs at start in one call

Introduce LNetFetchNIDs() to fetch all local NIDs across
all networks in single call

Rework server_lsi2mti() to use LNetFetchNIDs() call to
get all NIDs in single call.

This provides the following benefits:
 - avoid multiple calls to LNetGetId() with traversing
   both networks and nids lists from the beginning for
   each NID, i.e. it is now O(n) instead of O(n^2)
 - less load on LNet in terms of lock contentions
 - consistent results as Lnet lists can't be changed
   during call
 - can specify single network to get NIDs from

Also filter target networks by 'network' parameter just at
registration time, but don't do that for notification to
MGS about target NIDs for IR NID table. While 'network'
parameter defines what network to use by target itself,
it doesn't mean that incoming requests should be filtered
by 'network'. IR still need to know about all configured
networks by which a target is available for clients.
