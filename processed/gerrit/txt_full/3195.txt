Patch Set 5:

Talking with Alex, I remember why that is bad to re-open failed opens again instead of reconstruct - consider there were several opens from different clients on single file and then one client does unlink. So originally one client failed to open file for some reason and got rc in last_rcvd, so unlink happened with some 'open_count'. If we allow resent to happen really instead of taking rc from last_rcvd then open_count changes and we can affect all other recovery flow.
So better to repeat every step as it was originally, so let's keep old checks and do reconstruct
