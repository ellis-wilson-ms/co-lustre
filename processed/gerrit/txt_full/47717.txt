LU-15896 gss: support OpenSSLv3

Lustre GSS code makes use of some OpenSSL API that has been
deprecated in v3, namely all the functions in the DH_* family.
So replace them with their EVP_PKEY_* counterparts if Lustre is
built on a system with OpenSSLv3.

Fixes: ee60c14360 ("LU-15896 gss: ignore OpenSSLv3 deprecated API")

Thanks for your suggestions Andreas. My intention was to prefer fewer but larger #ifdef blocks. The idea to use compat macros is going to simplify the code even more, although as you noticed the API is sometimes very different between OpenSSL v1 and v3.

At the moment, only Ubuntu 22 distro comes with OpenSSLv3. So I was wondering, is there a way to get Maloo to run tests on Ubuntu 22? Is there any chance for this to work?
   Test-Parameters: clientdistro=ubuntu2204 ...

One thing that is worth mentioning is the fact that they changed the way prime numbers are generated between OpenSSL v1 and v3. So a prime number generated by a Lustre client using OpenSSL v3 will fail sanity checks when received by a Lustre server using OpenSSL v1. At least version 1.1.1c and earlier do not accept v3 primes, but 1.1.1g does...
