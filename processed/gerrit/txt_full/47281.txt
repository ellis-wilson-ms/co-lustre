LU-10215 tests: remove disk2_4 disk2_5 images

Remove the old disk2_4-*.tar.bz2 and disk2_5-ldiskfs.tar.bz2
images from the Git repo.  The disk2_5 image was never included into
testing due to an oversight in Makefile.am, and adding it to testing
is unlikely to be of any practical value as these releases are both
more than 10 years old and very unlikely to have any users that would
actually want to upgrade their systems at this point.

Adding 2.5 upgrade testing might have been useful when I filed LU-10215 7 years ago, but I don't think it is worth the effort anymore to get this image upgrade testing working.

From what I saw in the failed disk2_5 test runs, it looks like the MDT had a different FID super sequence assigned to the files compared to what was recorded in the "list" file, so the inode numbers were different.  I don't think that was an upgrade problem, but rather an issue with how the image was created, since this issue was never hit with disk2_4 upgrade testing.
