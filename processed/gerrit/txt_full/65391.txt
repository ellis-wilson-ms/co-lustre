LU-20090 nrs: add static and dynamic classify conf for TBF policy

This patch adds static and dynamic classify setting for TBF
policy.
Under the static strategy, the class type of each rule can be
only the global classification type;
Under the dynamic strategy, the class type of each rule can be
automically detected according to the rule's conditions, i.e.
The global rule is NID which is configured via:
ost.OSS.ost_io.nrs_policies="tbf nid"
A newly adding rule:
nrs_tbf_rule="start r1 nid={10.0.0.1@tcp}&uid={50} rate=100"
will be with class type of "nid+opcode" instead of the global
class type "nid".

The classify strategy can be configured by sysfs entry "classify"
and "nrs_policies":
i.e.
- ost.OSS.ost_io.nrs_policies="tbf nid+opcode classify=dynamic
- ost.OSS.ost_io.classify="static"
- ost.OSS.ost_io.classify="dynamic"
- ost.OSS.ost_io.classify="reg_classify:dynamic"
- ost.OSS.ost_io.classify="hp_classify:static"

Add sanityn/test_77tab to test  the setting of the classify
strategies.
