LU-16306 tests: using multiop to simulate I/O call sequence

For the bug occured in a non-public environment or non-open-source
applications, the developer can only analyze the bug from the
provided Lustre debug log, which makes the debug processing
complicated between the customer and developer.

In this patch, we propose a solution to simulate the I/O call
sequences via multiop according to the parsing I/O call traces
from Lustre debug and give out a method that could reproduce the
bug happened on the customer environment.

Add sanity/907 to verify that I/O call trace simulation from
Lustre debug log via multiop is workable.

This patch also adds lseek SEEK_END support and fixes the number
range error (which should be type 'off_t' instead of 'int') for
SEEK_CUR and SEEK_SET operations.
