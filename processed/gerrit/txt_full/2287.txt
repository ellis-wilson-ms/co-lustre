LU-993 tests: remove redundant sanity test_51c

When the patch in change Id02e7d2ac3a7664a55566b1de783f0a73162339b
for handling nlink counting in osd-ldiskfs was landed, it added a
new sanity.sh test_51ba() which deleted all of the subdirs created
in test_51a().  However, test_51c() was also expecting to delete
those subdirs, and now fails on every SLOW=yes test run (it is
excluded from normal review test runs).

Rename test_51ba() to test_51c(), since it largely does the same
things, but with a bit more checking, and delete the old test_51c.
