LU-12669 utils: zero stripe buffers in EC parity

llapi_ec_compute_parities() reuses the same stripe
buffer array across raid set iterations.  When the
file does not fill the entire last stripe width,
pread() returns fewer bytes than stripe_size, leaving
stale data from the previous raid set in the tail of
the buffer.  The parity encoder then XORs against
this stale data instead of zeros, producing wrong
parity for the final raid set.

Fix by zeroing all data stripe buffers before each
raid set's pread() loop.  This matches the kernel
recovery code which zero-fills beyond-EOF positions,
ensuring the parity encoder sees the same input data
during both write and verify/recovery.

Resolves parity mismatch in tests 42a and
42b where partial last raid set had stale data
in stripe buffers.
