--- Commit Message ---
LU-7003 utils: must quote the value of the context option

As per mount(8) notes, the context value might contain commas,
in which case the  value has to be properly quoted, otherwise
mount(8) will interpret the comma as a separator between mount
options.

> > I think it would be better to use strlcat() instead of strscat(),
 > > in order to be able to detect truncation and return an error in
 > > that case.
 > 
 > I'm not sure to understand your point here, because strscat() does

 > exit. Are you considering some other error handling?

Well, it will print error messages and return, but as append_context_for_mount() returns void, we will end up with incorrect context being used anyway.
So I suggest to use strlcat in order to be able to detect truncation in append_context_for_mount(), and change this function's return type to int so that calling function can avoid using an incorrect context.
