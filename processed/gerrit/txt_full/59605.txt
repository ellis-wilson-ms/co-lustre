LU-13643 mdt: implement ACTIVE_WRITER lock support

Add support for active writer locks, which are taken by the
client while actively writing an immediately mirrored file.

This adds a new bit to the MDT IBITs lock, and creates & releases
the lock as appropriate from the client.  This is done with a new
type of intent lock - IT_ACTIVE_WRITER.

This only holds the lock during writes, which is insufficient, as
the client needs to hold the lock until all data is committed.

This also does not implement any of the server side layout
behavior - it just creates and can release the lock.

This lock is specialized so it will call the blocking callback
on cancellations from the client, and it counts the number of
active writer locks currently held so it can correctly manage
state transitions.
