--- Message (Andreas Dilger, 2013-02-19 16:58:59.000000000) ---
Patch Set 8:

Excuse my ignorance, but could you please detail how fid2path works for me?  I thought that MDT would return fragments of path back to client, and client will assemble it if the path crosses MDT boundaries?  It also makes sense for fid2path to take the "root" FID as an argument, so that processes can specify the top of the tree they are looking at, and path building will stop when the specified "root" FID is hit.

In either case, it doesn't seem to make sense to me that the remote MDT needs to know what the actual ROOT FID is. It can stop building the path when it hits the local proxy parent directory or any parent FID that is on a remote MDT, return the current path and parent FID to the client, and the client will call fid2path with the new parent on another MDT and repeat. The client will assemble the path fragments as needed, possibly even putting "//" between segments on MDTs so it is visible to the user where the MDT boundaries are.
