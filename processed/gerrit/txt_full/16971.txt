--- Message (Andreas Dilger, 2015-12-04 03:44:03.000000000) ---
Patch Set 4: Code-Review-1

(3 comments)

Unfortunately, I think there is still a bug in the patch, which I noticed when looking at the upstream kernel's version of this code.

The callers of read_mmp_block() will call brelse(bh) on the bh pointer, even though it was freed here on error.  That means this needs to set *bh = NULL before returning.

In the upstream kernel version of this patch, it probably makes sense to move the cleanup code to warn_exit: and then only do the cleanup in one place to avoid future bugs like this.  For coding style consistency, it might make sense to reverse the last (return 0) check so that it is checking for an error instead of success:

        submit_bh(READ_SYNC | REQ_META | REQ_PRIO, *bh);
        wait_on_buffer(*bh);
        if (!buffer_uptodate(*bh)) {
                ret = -EIO;
                goto err_brelse;
        }

        mmp = (struct mmp_struct *)((*bh)->b_data);
        if (le32_to_cpu(mmp->mmp_magic) != EXT4_MMP_MAGIC ||
            !ext4_mmp_csum_verify(sb, mmp)) {
                ret = -EINVAL;
                goto err_brelse;
        }

        return 0;

 err_brelse:
        brelse(*bh);
        *bh = NULL;

 warn_exit:
        ext4_warning(sb, "Error %d while reading MMP block %llu",
                     ret, mmp_block);

        return ret;
 }
