--- Commit Message ---
LU-790 obdclass: Removed hardcoded "kernel: patchless_client".

Currently patchless_client is hardcoded in the
obd_proc_read_version() function. patchless_client means client does
not use any VFS patches. The earlier clients use to report here what
kernel patch were applied. Since all lustre clients are patchless we
can remove the "kernel: patchless_client" string altogether.
Removed check_kernel_version() function from lustre/tests/sanity.sh
which uses "lctl get_param -n version" output to get the kernel
patch version. Now as all the clients are patchless the
check_kernel_version() would always return true so no need to keep
it around.
