EX-10133 nodemap: enforce prjquota by different GID

Draft patch to enforce the project quota based on different GID.

The patch introduces a flags for ID nodemap to control the behaviors
of the ID mapping, and defines NODEMAP_IDMAP_FLAG_ROOT_PRJ_QUOTA to
enforce the project quota for the given Group ID mapping.

Sebastien, it would be useful to get your input on how this should work, but I thought that this property would be for the whole nodemap and use a nodemap property like 'readonly_mount' added in patch https://review.whamcloud.com/54282 "LU-17609 sec: nodemap readonly_mount for remount" or an RBAC flag like 'fscrypt_admin' added in patch https://review.whamcloud.com/50184 "LU-16524 sec: add fscrypt_admin rbac role".

I also don't understand why this has anything to do with GID?  My expectation was that the root_prj_enable check in qsd_op_begin() would be changed to check if either the global qsd_root_prj_enable flag was set, or a per-nodemap nm_root_prj_enable flag was set, something like:

     if (local_flags && qi->lqi_id.qid_projid &&
         (qsd->qsd_root_prj_enable || nodemap->nm_root_prj_enable))

but I'm not sure if there is direct access to "nodemap" in this code, or if this flag needs to be passed in some other way (e.g. struct lquota_id_info)?
