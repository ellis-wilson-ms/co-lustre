LU-19295 utils: handle insufficient arguments to lst

When passing insufficient arguments to lst command it
was expecting to throw a usage message. Instead it was
segfaulting. The reason was that cfs_parser() did not
have an exit condition and was recursively calling
the same function and finally reaching the stack limit
and segfaulting. This behavior was seen across all the
lst commands.

Add an exit condition and pass the return value to the
calling function so it can bail out with a usage message.

------------
Before patch
------------
$ lst add_group
Segmentation fault (core dumped)

$ lst del_group
Segmentation fault (core dumped)

------------
After patch
------------
$ lst add_group
Usage: lst group NAME IDs [IDs]...

$ lst del_group
Usage: lst del_group NAME
