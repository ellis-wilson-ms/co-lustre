It's not a bad patch, but makes me wonder whether it is worthwhile to have an autoconf check for something that can be checked directly in the code via:

 #ifndef __REQ_WRITE /* before 2.6.35 */
 #define _REQ_WRITE BIO_RW
 #endif

or so?

The benefit of the autoconf check is that it will be clear when it is time to be cleaned up, but the test will also need to be deleted instead of just one place. In the end it isn't a huge deal either way.
