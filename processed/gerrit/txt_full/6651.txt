--- Commit Message ---
LU-684 kernel: replace dev_rdonly with fault injection

Lustre has over time developed a method to simulate disk failures.
This approach requires you to patch your distribution kernel. Today
the linux kernel has a fault injection to simulate many types of
failures. This patch moves us over to this api to simulate failed
disks. The current dev_x_rdonly patches are kept for backwards
compatibility but if both are enabled fault injection will be used
instead of dev_x_rdonly. Most distros don't enable fault injection
by default so keeping dev_x_rdonly helps ease this transition.

--- Message (Andreas Dilger, 2013-06-15 17:57:12.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

Rather than deleting these patches outright, for now please just change the comment at the top to say that they are not needed if CONFIG_FAIL_MAKE_REQUEST is enabled.  This will allow the code to continue testing if this option is not enabled in the kernel, and will allow the same kernel to be used for 2.1/2.4 Lustre to facilitate upgrades. 

 The main osd_ro() code should check:

 #ifdef CONFIG_FAIL_MAKE_REQUEST
         [stuff]
 #elif defined(HAVE_DEV_SET_RDONLY
         [other stuff]
 #else
         [complain]
 #endif

In some cases the "stuff" and "other stuff" can be the same. If the CONFIG_FAIL_MAKE_REQUEST code was wrapped in a little helper function like dev_fail_make_request() there could be a single #ifdef that wrapped the helper and selected whether to use the new or old code instead of putting all of the #ifdef inline in the code.

Since CONFIG_FAIL_MAKE_REQUEST is not enabled by default in vendor kernels yet, we can't go over to this totally, but making it optional will get us half of the way.
