--- Commit Message ---
LU-20019 lnet: adler32: fix __adler32_finup()

Since v6.15-rc1-160-g7650f826f7b2
crypto: shash - Handle partial blocks in API

crypto_shash_final() is not using ->final() anymore,
instead it calls ->finup() with NULL as buf and 0 as length.

This creates an issue for us as zlib_adler32() returns the
required initial value, which is 1 if the buf is NULL.

We should detect this and return the hash digest for this case.
