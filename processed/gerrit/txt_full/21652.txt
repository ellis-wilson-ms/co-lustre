LU-8464 llite: Lustre I/O hung waiting for page

A deadlock during lock_page()

Thread 1:
sleep_on_page
__wait_on_bit_lock
                        <<< lock 2nd page
__lock_page
vvp_page_make_ready
cl_page_make_ready
osc_extent_make_ready
osc_io_unplug0

Thread 2:
__wait_on_bit
wait_on_page_bit
shrink_inactive_list
shrink_list
shrink_zone
do_try_to_free_pages
try_to_free_mem_cgroup_pages
mem_cgroup_hierarchical_reclaim
__mem_cgroup_try_charge
mem_cgroup_prepare_migration
                        <<< lock 2nd page
migrate_pages
compact_zone
__compact_pgdat
compact_node
sysfs_compact_node

Thread 2 tries to migrate 2nd page from extent
and waits for 1st to complete writeback.

But these 2 pages are going to fit in one RPC.

Lock all extent pages and modify them after.

(1 comment)

The way I understand the bug: 
The ptlrpcd thread loops doing [lock page, set page writeback flag, unlock page]. The rpc that it's trying to send will clear the writeback flag when it completes.

The migrate pages thread wants to move page A to page B so it locks page B and then must wait for the writeback flag on page A to clear. Page A is not locked. (I may have this backwards; maybe it moves B to A).

So the ptlrpcd thread can lock A but not B so it doesn't send the rpc nor clear the page writeback flag. The proposed patch fixes the deadlock by locking all pages before updating any of the page flags.
