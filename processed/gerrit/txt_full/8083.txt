--- Commit Message ---
LU-4152 mdt: Don't enqueue two locks on the same resource

Due to mechanics of ldlm internals, enqueueing two different ibits
lock on the same resource is deadlock prone.
As such change mdt_object_open_lock to release open lock if it becomes
necessary to get exclusive layout lock (to create objects).
It's ok to release the open lock right away as it's never guaranteed to
be issued anyway.

--- Message (Oleg Drokin, 2013-12-18 03:53:04.000000000) ---
Patch Set 5:

Actually we must take the open lock even if there is no layout.
This is because the open lock taking will flush the existing conflicting open locks held by other clients.
If we don't do that, then the clients will continue to hold the locks and cached filehandles that might prevent us from opening a file for write or for exec.
See bugzilla bug 2443 for more details.
