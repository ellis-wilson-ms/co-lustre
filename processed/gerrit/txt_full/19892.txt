LU-8042 osd-zfs: set ashift=12 recordsize=1024k by default

When creating new OST zpools add the "ashift=12" option by default,
since it is typical that the underlying storage is using 4KB sectors.
Without this parameter, significant performance problems can be seen
on drives that emulate 512-byte sectors, and it may cause problems
replacing drives in the future if not specified at format time.

Set the recordsize=1024k option by default for ZFS OSTs to allow ZFS
blocksize up to 1MB to improve performance for most OST workloads.
This can be overridden by "mkfs.lustre --mkfsoptions=recordsize=128k"
at format time, or by "zfs inherit recordsize <filesystem>" to clear
it after the filesystem was formatted.
