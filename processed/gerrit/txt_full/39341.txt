--- Commit Message ---
LU-13777 lnet: select reachable peer ni for discovery

Ensure the selection of a reachable peer NI.
commit 71ca66bcd9 sends the PUSH for discovery out of the same
source NID. However, this change caused an issue where a peer NI
can be selected which is not reachable from that source NID.

This patches fixes this behavior. When selecting the destination NID,
take into consideration the source NID. For directly connected peers
select a peer ni on the same net as the source NID. For routed peers
select a peer ni which has a route from the source NID.

Fixes: 71ca66bcd9 ("LU-13471 lnet: use the same src nid for discovery")
Signed-off-by: Amir Shehata <ashehata@whamcloud.com>
