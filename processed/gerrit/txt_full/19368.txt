Patch Set 2:

> -       m->ofd_brw_size = ONE_MB_BRW_SIZE;
> +       m->ofd_brw_size = PTLRPC_MAX_BRW_SIZE;

I'm reluctant to enable 16MB RPC by default for all OSTs. Let's keep it as an advanced option for now.

The problem with st_blksize is that the preferred readahead RPC size can vary among different stripes and OSCs. Therefore, I would prefer to make it a parameter of readahead, just like other parameters like dlm lock coverage, available readahead grants, etc.
