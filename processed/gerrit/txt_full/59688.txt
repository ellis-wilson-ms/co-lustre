LU-13643 mdt: implement workqueue for FLR state transitions

This patch implements a workqueue-based solution for handling
FLR (File Level Replication) state transitions from WRITE_PENDING
to RDONLY when the last active writer lock is released.

The workqueue approach solves the problem of performing FLR state
transitions in contexts where proper transaction environments
may not be available, such as during lock cancellation in
blocking AST callbacks.

Key changes:
- Add mdt_flr_wq workqueue to mdt_device structure
- Implement mdt_flr_queue_rdonly_transition() to queue work items
- Implement mdt_flr_rdonly_work_fn() as the work function
- Initialize workqueue during MDT startup in mdt_init0()
- Cleanup workqueue during MDT shutdown in mdt_fini()
- Update mdt_active_writer_dec() to use workqueue when no proper
  environment is available

The workqueue ensures that FLR state transitions are performed
in a proper thread context with appropriate transaction handling,
avoiding potential deadlocks and environment issues that can
occur when called from blocking AST contexts.
