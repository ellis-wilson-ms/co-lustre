LU-8837 ptlrpc: only build service.c on the server

service.c contains no code needed on the client, so only build it when
the server is wanted.

Calls to ptlrpc_hr_init/fini() are only now need to be protected with
HAVE_SERVER_SUPPORT, and the declaration moved to ptlrpc_internal.h
which seems more appropriate.

A large section of ptlrpc_nrs_policy_register() is also server-only,
so it is protected accordingly.
