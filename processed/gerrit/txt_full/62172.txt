--- Commit Message ---
LU-19472 lod: alloc big enough bitmap for OST avoidance

We use a bitmap to collect used OST indices of other mirrors so that
they are avoided in allocating OST objects in a new mirror. While
it use the number of OSTs for the bitmap size, and when a filesystem
uses sparse OST indices then the OST index would out of the boundary
of the bitmap.

This patch fixes it by using the size of the lu_tgts array as the
OST avoidance bitmap, and this value guarantees to cover all used
OST index (referring to
lov_add_target()->ltd_add_tgt()->lu_tgt_desc_resize())
