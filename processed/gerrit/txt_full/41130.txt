sanity-sec test_56 still fails with ZFS. According to filefrag manpage, the error "FIBMAP unsupported" suggests that FIEMAP was attempted first, but because it was not supported filefrag tried to fall back to FIBMAP.

I looked for other uses of filefrag in the test suite. There is for instance sanity test_130a, but unfortunately it is on the ALWAYS_EXCEPT list for ZFS, because of LU-1941. It turns out FIEMAP support in ZFS is still not available, as the feature tracked under https://github.com/openzfs/zfs/pull/7545 is not landed yet. According to the latest comment from Brian Behlendorf last summer, there is still work to do.

With respect to sanity-sec test_56, I will update the patch to skip the test if FIEMAP is not supported.
