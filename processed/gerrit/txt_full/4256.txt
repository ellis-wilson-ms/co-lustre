--- Commit Message ---
LU-2150 ost: ost_brw_read() to ptlrpc_free_bulk_nopin()

since a643e38 (LU-2089) OST do not pin pages involved in
BULKs: this is done to prevent get/put on the pages which
were allocated as part of order N (>1) allocation with 0
refcounter. get/put on such a page leads to warning from
the kernel. in the original patch one code path was not
fixed, so this patch completes the change.

also, to prevent confusion, the patch removes couple macros:
ptlrpc_free_bulk() and ptlrpc_prep_bulk_page(). so now the
caller should specify whether ptlrpc should reference pages
or not.

Patch Set 1: I would prefer that you didn't submit this

Two things that should be fixed in this patch:
- the commit summary doesn't really describe anything beyond what the patch itself is doing. Why was this wrong, what patch introduced the problem, how does this change fix the problem?  You have to write commit summaries with the though that people besides you and I (who know already that this was missed in a previous patch) will be reading it at some point in the future and will need to know why this was done. Including the commit hash for the change that introduced this problem would be very useful, and explaining that this fixes the reference count on the page (since the OST does not manage this itself, but leaves it to the OSD) would go a long way to helping others understand this code. 
- keeping around the old wrapper for ptlrpc_prep_bulk_page() seems like a bug waiting to happen... Why not remove the old wrapper (which would have made this a clear compile error) and then the coder needs to decide which version is needed?
