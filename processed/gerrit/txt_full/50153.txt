--- Commit Message ---
LU-16429 recovery: resend recovery for batch intent creation

When the prepared reply size of a batched RPC on a client is
smaller than grown reply buffer size on the server, the client
will see this buffer overflow in the client's incoming reply
callback (@reply_in_callback), and then it will grow the reply
buffer size and resend the batched RPC.
This resend is easy to trigger when the prepared LOV EA buffer
size on client side is smaller than the actual LOV EA of a regular
file during the batch intent creations.

This patch adds the resend recovery support for batched intent
creation RPC. It forces to set disposition with DISP_OPEN_CREATE,
and call @mdt_open_by_fid() to reconstruct the sub reply message.

Add test case: sanityn/test_117.
