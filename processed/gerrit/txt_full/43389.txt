LU-13717 sec: filename encryption - server side

Server side, mainly the OSD layer, needs to know the encryption
status of files being processed.
If an object belongs to an encrypted file, the filename has been
base64 encoded by the client because it is binary, so it needs to
be base64 decoded before being handed over to the backend file
system layer.
And conversely, the filename of an encrypted file has to be base64
encoded before being sent over the wire.

Note this is osd-ldiskfs only for now.
