LU-17408 mdt: Fix the order of barrier calls

The barrier_entry and barrier_exit called during a setquota
are called in a reverse order with barrier_exit getting called
first and then a barrier_entry. This causes the barrier counter
to go negative if its called when a barrier freeze is enforced.
The check for barrier freeze is done only in the barrier_entry
and not done in barrier_exit. The change reverses it to the
correct order.

Also added a test case 801d to test the change. The test would
call a barrier freeze and then it would call the setquota. Once
the freeze is thawed, it would call a barrier freeze again to
check if a freeze can be placed. Without the fix, the test would
fail in the second freeze as the freeze would timeout.
