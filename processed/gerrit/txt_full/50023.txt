LU-12275 sec: remove bio functions in fscrypt compat

is not needed for client builds, and the use of stuct bio on the
client adds unnecessary complexity to portability.

In lite we implemented our own "zeroing" function, ll_io_zero_page(). The function defined in llcrypt is not used by us, so it can be disabled. Maybe something like "#ifndef CONFIG_LL_ENCRYPTION" would be more suitable? I mean, it feels weird to test for HAVE_SERVER_SUPPORT in code that is meant for client side. And as this code is part of the llcrypt library, testing for !CONFIG_LL_ENCRYPTION would always eval to false :)
