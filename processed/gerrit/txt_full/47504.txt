LU-14123 wbc: obtain (default) LMV EA from lsm

In LL_IOC_LVM_GETSTRIPE ioctl, the caller uses @magic to indicate
which LMV stripe the ioctl like to get. LMV_MAGIC_V1 is for normal
LMV stripe, LVM_USER_MAGIC is for default LMV stripe.

Under WBC, the client can allocate shard FID(s) for normal striped
dir and inherit the setting of the default stripe MD without
contract with the server (MDS).
As the directory may not flush to the MDT, thus the user may not
obtain the (default) LMV EA from the server.
However, we can prepare the (default) LMV EA from @lsm directly.

In this patch, it implements the functionality for directires
under WBC to get the (default) LMV EA from @lsm locally for the
command: lfs getdirstripe.
