Patch Set 1: Code-Review-1

This will cause more memory allocation for all of the histograms, even though it is not needed in the vast majority of cases. 

I'm already working on a patch to do dynamic allocation of the histogram structure (in chunks of 16 buckets at one time) so that it does not need to be allocated at the maximum size, and in fact will save done memory in the common cases. 

It will take me a little while to post it, since in traveling and the patch is still in the coding stage, but I think that is a better long-term solution.
