--- Commit Message ---
LU-9019 libcfs: discard cfs_time_seconds()

cfs_time_seconds() converts a number of seconds to the
matching number of jiffies. The standard way to do this
in Linux is "* HZ". So discard cfs_time_seconds() and
use "* HZ" instead.

With lustre now using proper 64 bit time we no longer
need the cfs_time_seconds() wrapper. Also using
msecs_to_jiffies() no longer works to get around the
variable jiffiess issue. With all uses of HZ being
to the local node it is safe to remove the wrapper.

--- Message (James Simmons, 2018-04-09 21:29:13.000000000) ---
Patch Set 1:

The reason it doesn't work anymore is that for newer kernels msecs_to_jiffies() is now an inline function in jiffies.h. This was done to deal with odd HZ values that don't divide evenly into milliseconds.  The real function is __msecs_to_jiffies but I see comments about depending on HZ not being greater than 1000. I don't trust that will be the case. I think the best approach is to move to nsecs_to_jiffies().
