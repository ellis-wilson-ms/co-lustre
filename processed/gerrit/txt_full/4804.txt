LU-2473 ldiskfs: Add ldiskfs support for RHEL 6.4

Add an ldiskfs kernel patch series to support the RHEL 6.4 kernel.

The ldiskfs series selection macro (LB_LDISKFS_SERIES) is fixed up
to use the AS_VERSION_COMPARE, which allows us to check if the kernel
version is greater than or equal to a specific number, rather than
just a simple pattern match.

Patch Set 4: Looks good to me, approved

I'm not against this reorganization, but I'm concerned about quickly increasing the number of patches for multiple different kernels.

Either in this patch or the next one (adding RHEL 6.4) please add a README that explains that patch series should re-use the patches from existing kernels instead of creating new patches for each new kernel, and that the separation into separate subdirectories is only for tracking and indication of which kernel the patch was first created for.

Since I see a few fixes needed in the RHEL6.4 patches, we can do it as part of that one.

--- Message (Andreas Dilger, 2013-01-10 18:35:33.000000000) ---
Patch Set 4: I would prefer that you didn't submit this

My bad, was commenting in wrong window. This IS the RHEL6.4 patch.

The check for the kernel version to select the patch series is fragile, and will break as soon as there is a new minor kernel version, with the result that it will fall back to the RHEL6.3 series instead of staying at the RHEL6.4 series. I'd prefer two changes:
- this conditional is only checked for $RHRL_KERNEL
- it checks the minor version and uses the RHEL6.4 series for any kernel newer that -343
