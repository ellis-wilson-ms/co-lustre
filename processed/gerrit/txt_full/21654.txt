So I read the ticket and commit description and I am not sure what's up with this patch other than we can actually track allocations for this bit separately.

Do we really allocate so many of these things all the time, I think it was one per service thread = relatively little? (also would mean we have number of threads of allocations here).

How is the allocation improved? The ticket also emntions vmalloc pressure - is the structure bigger than 4k? You do realize currently we always try regular malloc first before proceeding to do vmalloc on failure, right?
And if regular malloc failed - assuming this is the path here, then we'd just have allocation failure instead of a vmalloc attempt which does not look like any improvement in behavior.

I feel like better explanation is needed. If any profiling was performed to show additional benefits, please show it as well.
