LU-12931 misc: correctly use jiffies for mod_timer

In some places there were previously incorrect conversions from
seconds to jiffies in conjunction with mod_timer().  Instead of
using seconds to store the lock expiry timers, use the more natural
jiffies units in conjunction with kernel timers rather than trying
Affected variables have added a _jiffies suffix to make this clear:

   msg_delay_send     => msg_delay_send_jiffies
   dl_delay_time      => dl_delay_timeout_jiffies
   dl_time_base       => dl_delay_base_jiffies
   dl_msg_send        => dl_msg_send_jiffies
   l_callback_timeout => l_callback_timeout_jiffies

The dl_msg_send_jiffies is an absolute jiffies number.  It is used
to set a timer (via mod_timer()).  The "-1" special value previously
was used to indiciate the timer wasn't started, but since all jiffies
values are valid, and "-1" is actually a valid time 5 minutes after
boot, change it to "0" to indicate an unset timer.  Add the function
cfs_time_set() to ensure timer always has low bit set, to identify if
timer is unset.  Ensure timer is valid before checking if expired.

Improve comments for timeout values to clearly show units.

Fixes: b11be372c21d ("LU-9019 lnet: move ping and delay injection to time64_t")
Fixes: e920be681451 ("LU-9019 ldlm: migrate the rest of the code to 64 bit time")

Patch Set 6:

> It isn't clear from this comment, are you for or against having DL_MSG_SEND_UNSET?  

I'm against having a value with two different meanings, as that can lead to confusion.  As you say, maybe that doesn't matter in this particular code as the cost of confusion would be low.  But people copy code, so it is best to always aim for clarity.

So I'm against trying to store any arbitrary jiffie value as well as a "value is unset" flag in the same "unsigned long".  There are several alternative.
The simplest is to have a separate variable "timeout_is_set", but that is a little ugly.
Another option is to reserve a value ("0" or "-1" or whatever) and make sure it is never used as a "jiffies" value.  So when calculating  a jiffies value, if it happens to match the reserved value, add one.
Or, we could round the jiffies number up, and have the reserved value being an unrounded value.
This would probably be simplest to code.  Have the reserved value be "0", and whenever we calculate a jiffies value, use " | 1" to set the lsb, so the result is certain to not be the reserved value.

Writing code that you *know* can fail, even when unlikely, just feels unclean :-)
