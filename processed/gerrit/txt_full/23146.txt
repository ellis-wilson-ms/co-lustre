--- Commit Message ---
LU-4423 obd: use ktime_t for calculating elapsed time

process_param2_config() tries to print how much time has passed
across a call_usermodehelper() function, and uses struct timeval
for that.

We want to remove this structure, so this is better expressed
in terms of ktime_t and ktime_us_delta().

Patch Set 4:

I had a earlier discuss with Frank about the limitations of using jiffies. The problem is that changing jiffies to milliseconds is very machine specific since HZ is a constant that can vary between machines. So if Intel distribute prebuilt rpms built on a machine using HZ = 100 and those rpms are installed on a machine with a HZ of 1000 this means the millisecs you calculate will be off by a factor of 10. From what I can tell this problem doesn't exist with ktime so I'm leaning to moving away from jiffies to ktime in the future.
