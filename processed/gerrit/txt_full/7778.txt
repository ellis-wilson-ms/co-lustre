LU-2887 tests: disable SLOW for ZFS in parallel-scale

Because all zfs pools are on the same physical device in autotest
ZFS backed OSTs run VERY slowly so disable SLOW.  Decrease load on zfs
backed MDS.

Test-Parameters: envdefinitions=SLOW=yes,ENABLE_QUOTA=yes mdtfilesystemtype=zfs mdsfilesystemtype=zfs ostfilesystemtype=zfs testlist=parallel-scale

--- Message (Andreas Dilger, 2013-10-19 17:59:29.000000000) ---
Patch Set 6: Looks good to me, but someone else must approve

(1 inline comment)

I think that putting the OSTs into a single pool would help to some extent. It would avoid the extra overhead of 7 separate OST pools trying to do transaction commits on LVM devices that share the same physical disk. Having just 2 or 3 OSTs would probably improve things a bit more, since there would be less overhead from Lustre per OST. 

I'm ok with changing this in the short term until the test config can change.
