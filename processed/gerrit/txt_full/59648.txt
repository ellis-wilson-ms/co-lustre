LU-0000 mdt: add mirror un-staling to FLR transitions

Enhance the workqueue-based FLR transition system to automatically
un-stale mirrors when transitioning from WRITE_PENDING back to
RDONLY state, matching the behavior of mirror resync operations.

Problem:
The previous implementation correctly transitioned FLR files from
WRITE_PENDING to RDONLY state but left mirrors in stale condition,
requiring manual resync operations to restore mirror validity.
This created an inconsistent user experience compared to normal
mirror resync behavior.

Solution:
Modify mdt_flr_state_transition_to_rdonly() to automatically clear
the LCME_FL_STALE flag from all mirror components during the
WRITE_PENDING to RDONLY transition, similar to how mirror resync
completion works.

Implementation details:
- Iterate through all layout components in the composite layout
- Clear LCME_FL_STALE flag from any stale components
- Add debug logging for each component that gets un-staled
- Update commit message to reflect the new un-staling behavior
- Preserve all existing workqueue and environment handling logic

This enhancement ensures that after concurrent write operations
complete on immediate mirror files, all mirrors are automatically
restored to a valid (non-stale) state without requiring manual
intervention, providing a seamless user experience.

Verified functionality:
- ACTIVE_WRITER locks acquire/release correctly
- Workqueue executes FLR transitions in proper environment
- Files transition WRITE_PENDING to RDONLY successfully
- All mirror components are automatically un-staled
- No stale flags remain after transition completion
- Layout generation increments correctly
- System remains stable under all conditions
