--- Message (Andreas Dilger, 2016-12-02 20:41:22.000000000) ---
Patch Set 1:

The patch itself is fine, since lu_env clearly already contains a kiocb that can be used without any allocation, and it solves this problem. 

That said, the reason for having lu_env in the first place was to avoid problems with the 4KB stacks that were all the rage a few years ago. However, newer kernels allow larger stacks (3.15, maybe backported to RHEL7 3.10 kernels?). We should re-examine the use of lu_env and remove usage of it where possible and use on-stack structures where practical (in separate patches for 2.10). The use of lu_env makes the code harder to follow, and I think it is also abused in some places as a hidden way to pass arguments between functions.
