LU-12867 dne: place temp file on MDT where target is

Map temporary file created by rsync or mpifileutils dstripe on
the same MDT where target is located, so later rename won't
make target a remote entry.

And the same for backup file ends with .bak, .sav, .orig or ~.

Add sanity 33h.

> Lai, I understand the motivation, and I admit I cannot think of a

This is of course possible, but unlikely without motivation.  The worst outcome of such a change to those tools would be no worse than what exists today - using rsync would result in (N-1)/N of the files in an rsync'd directory being on a remote MDT vs. the filename hash.

 > 2. It also will affect other applications, creating different
 > behaviors (unexpected by the user) from file to file.

Sure, but if other applications also have a filename that is ".filename.XXXXXX" then it also seems possible that those files
