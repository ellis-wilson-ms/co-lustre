LU-10378 utils: add formatted printf to lfs find

Introduced two new options with lfs find utility:
1. '--yaml' (-y) that allows users to request lfs find to gather
all metadata related to the specified file/directory path.
2. '--printf' (-k) along with the format specifiers given below
that allow users to easily obtain metadata in formatted style.

How to use these options:
-------------------------
1. -y or --yaml: Required to print all the metadata information
in yaml format
2. -k or --printf : Specify this flag along with the format
specifiers to print selected information from the metadata.
Note: You still have to specify –y for this to work. Specify the
format specifiers without white spaces in between as shown in
the example command below.

"lfs find -y -mtime -1 -k %a%t%c%bt%b%s%U%G%p%P
/lustre/lustre/test_dir/

-----------------------------
Parameter  | Format Specifier
-----------------------------
atime      |   %a
mtime      |   %t
ctime      |   %c
btime      |   %bt
blocks     |   %b
size       |   %s
name       |   %p
uid        |   %U
gid        |   %G
project id |   %P
-----------------------------

Additionally users need to set any of these flags to
obtain all the metadata even if they set –y flag.
M: mtime, O: Obd or Ost, m: mdt-index, i: stripe-index, U: user,
G: group, B: btime, b: blocks, LFS_NEWERXY_OPT, A: atime,
c: stripe-count, s: size, LFS_POOL_OPT, S: stripe-size,
L: layout, LFS_COMP_COUNT_OPT, E: component-end, comp-end,
LFS_COMP_START_OPT, LFS_COMP_FLAGS_OPT, N: mirror-count,
LFS_LAYOUT_FOREIGN_OPT, LFS_MIRROR_STATE_OPT, z: extension-size,
LFS_PROJID_OPT, T: mdt-count, H: mdt-hash

--- Message (Andreas Dilger, 2021-10-06 00:26:23.000000000) ---
Patch Set 1:

"lfs getstripe" prints the stripe count/size/index of the last initialized component for these parameters on PFL files.  The presence of the earlier components (with fewer stripes, smaller stripe size, etc) is not very useful when migrating files and/or mirroring the layout.
