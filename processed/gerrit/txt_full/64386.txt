LU-19848 obdclass: fix directIO for p2pdma

iov_iter_get_pages_alloc2() does not like the pages from
the NVIDIA p2pdma stack. CUDA allocates ZONE_DEVICE pages
directly from the NVIDIA driver. It then passes these pages
to filesystem via the normal directIO path. The kernel does not
expect users pages to be ZONE_DEVICE pages. Hence,
iov_iter_get_pages_alloc2() can't try_grab_folio() since
FOLL_PCI_P2PDMA isn't set. One potential workaround is to use
pin_user_pages() directly.

## Summary of the Fix

The compile errors were caused by the patch attempting to use newer kernel APIs that don't exist in older kernel versions. Here's what was fixed:

### **Error 1: `iter->ubuf` field doesn't exist**
**Problem**: Line 148 tried to access `iter->ubuf`, but this field only exists in kernels with `HAVE_USER_BACKED_ITER` support (Linux v5.19+).

**Fix**: Added conditional compilation using `#ifdef HAVE_USER_BACKED_ITER` to only access `iter->ubuf` when available. For older kernels, we fall back to using `iter_iov(iter)` to get the iovec structure.

### **Error 2: `FOLL_PCI_P2PDMA` flag undeclared**
**Problem**: Line 164 used `FOLL_PCI_P2PDMA` flag which doesn't exist in older kernels.

**Fix**: 
- Introduced a `gup_flags` variable to build the flags incrementally
- Used `#ifdef FOLL_PCI_P2PDMA` to conditionally add this flag only when it's available
- For older kernels, the code will work without P2PDMA support (graceful degradation)

### **Error 3: `pin_user_pages()` has wrong number of arguments**
**Problem**: Line 163-165 called `pin_user_pages()` with 4 arguments, but older kernels require 5 arguments (including a `vma` parameter).

**Fix**: Used the existing `HAVE_GET_USER_PAGES_WITHOUT_VMA` compatibility flag to call the correct version:
- Newer kernels (v6.4+): `pin_user_pages(addr, nr_pages, gup_flags, pages)`
- Older kernels: `pin_user_pages(addr, nr_pages, gup_flags, pages, NULL)`

### **Error 4: C89 variable declaration compliance**
**Fix**: Declared all variables (`gup_flags` and `iov`) at the beginning of the `#if defined(HAVE_DIO_ITER)` block to conform to C89 standards.

### **Rationale**

The original patch was written for newer kernels that support:
1. **ITER_UBUF** (v5.19+): A new iov_iter type for single user buffers
2. **FOLL_PCI_P2PDMA** flag: For peer-to-peer DMA support with PCI devices (NVIDIA GPUs)
3. **Simplified pin_user_pages()**: Without the vma parameter (v6.4+)

The fix maintains backward compatibility by:
- Detecting kernel capabilities at compile time using existing autoconf macros
- Providing fallback code paths for older kernels
- Preserving the new P2PDMA functionality when available while gracefully degrading on older kernels

This approach ensures the code compiles on both old and new kernel versions while maintaining the intended P2PDMA functionality where supported.
