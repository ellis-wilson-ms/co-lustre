--- Message (Doug Oucharek, 2016-09-21 21:48:35.000000000) ---
Patch Set 2:

DLC was conceived as a replacement to module parameters (make them changeable dynamically) and not a replacement to procfs or sysfs.  DLC has evolved to include some information from procfs (i.e. routing info, peer info), but that is a bonus and not the primary purpose of DLC.  

DLC is not replacing previous config systems but is "adding" to them.  There are cases like MultiRail which is new and would be too painful to do with the older config system so it will be DLC-only.

We have existing procfs files which are writable for resetting stats (i.e. /proc/sys/lnet/stats).  That has not been added to DLC yet.  It does not make sense to make this change DLC-only.  If that decision is made, then a new ticket should be created to collectively update all the LNet procfs files in one go rather than try to do them piecemeal.
