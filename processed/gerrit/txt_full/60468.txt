LU-19196 tcu: handle orphan objects moving into Trash Can

For regular files and empty directories, upon the last unlink,
they are moved into Trash Can directly. In this case, even the
object is still opened, we do not mark the object as ORPHAN until
the object is deleted from Trash Can, we then move the object
into PENDING directory and mark it as orphan if it is still opened.

For an non-empty directory, the stub dir ("pFID") in Trash Can is
used as a replica of the original non-empty directory, thus the
handling for the object of the non-empty directory is same as
before. If the object is still having open count, move it to the
PENDING directory and mark it as orphan; Otherwise, destory it
directly.

Add test cases sanityn/test_118A{a, b}.
