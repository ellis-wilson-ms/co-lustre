This patch addresses compile errors that arose from a previous patch by correcting the format strings and arguments in `CERROR` messages in the `osd_quota_fmt.c` file. Specifically, the changes are as follows:

### Summary of Changes:

1. **Updated `CERROR` Format Strings:**
   - Changed the format string from `"%s"` to `"%s"` but now using `obj->oo_inode->i_sb->s_id` instead of `osd_obj2str(obj)` to get the filesystem identifier.
   - Changed the format specifier for the return code from `%d` to `%lld` to match the type of the return value (`loff_t` or `int`), which is typically a 64-bit value.

2. **Code Adjustments:**
   - In one place, a local variable `ref` was declared after `buf`, which could cause a compiler warning. The declaration was moved up to fix this issue.

### Key Modifications:

- **Before:**
  ```c
  CERROR("%s: cannot read quota tree block %u: rc = %d\n", osd_obj2str(obj), blk, ret);
  ```

- **After:**
  ```c
  CERROR("%s: cannot read quota tree block %u: rc = %lld\n", obj->oo_inode->i_sb->s_id, blk, ret);
  ```

### Purpose:
These changes ensure that the code compiles without errors and that the logging is more accurate and consistent, particularly in terms of data types and the information being logged.
