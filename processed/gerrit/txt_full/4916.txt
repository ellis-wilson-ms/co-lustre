LU-2544 lnet: SMP scaling api for Gemini driver

Increase the performance of the Gemini LND driver by
using the Lustre SMP api. Bind the threads to specific
cores to limit the cost of thread migration between
different cores. The second improvement is allocating
memory based on which core so we can keep the cache
warm to limit cache line misses. Lastly we reduce the
amount of memory allocated for the cksum pages from
all online cpus to due the cpus actually used by the
LND driver.

Test-Parameters: trivial

--- Message (James Shimek, 2013-11-17 05:00:45.000000000) ---
Patch Set 10: I would prefer that you didn't submit this

(2 inline comments)

This should not be landed yet, the way schedulers threads are bound to cpt's and how connections have there own cpt can cause an issue where the scheduler thread does not wake up to send messages when kgnilnd_send is called. If forward progress is to be made the connections need to have the same CPT as the one the scheduler thread/s is/are bound to.

kgnilnd_queue_tx calls kgnilnd_schedule_conn which calls kgnilnd_schedule_device which takes the conns->gnc_sched.. Since this sched is might be a a different cpt than the ones that the actual scheduler threads are bound to nothing will happen and forward progress on the send will be stalled.
