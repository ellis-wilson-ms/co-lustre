LU-14742 socklnd: detect link state to set fatal error on ni

To help avoid selecting lnet ni which corresponds to a downed
ethernet link for sending, add a mechanism for detecting link
events in socklnd. On link up/down events, find corresponding
ni and toggle ni_fatal_error_on flag, similar to o2iblnd way.

--- Message (Serguei Smirnov, 2021-06-16 00:17:00.000000000) ---
Patch Set 3:

> Have you guys looked at whether this sort of infrastructure, if it
 > were available for all LNDs, would be sufficient to determine local
 > NI health? i.e. could we get rid of the recovery pings for local
 > NIs and replace it completely with these link events?

I agree that would be cleaner. The problem is that with socklnd, for example, it is possible that the link comes back up with no IP assigned to it anymore: it that case trying to ping still makes sense. So these IP-level events would be separate from link-level events. I'm guessing it would be possible to subscribe for these too somehow, I just haven't explored it in enough detail.
