LU-14032 osd: create agent inode with proper GID

If some parent directory is set as S_ISGID, the agent inode for
the remote stripe directory in it will be created with its GID as
its parent GID during creation, but it is restore as current_fsgid
after the creation, which cause the group quota discrepancy.
