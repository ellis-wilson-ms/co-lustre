--- Message (Andreas Dilger, 2011-04-08 08:11:14.000000000) ---
Patch Set 1: Looks good to me, but someone else must approve

I think it is still important to work on the 2.x server-side fix for this problem as well.

We could use the lack of OBD_CONNECT_FULL20 flag to detect older 1.x clients, so that only 32-bit hash values are sent from the MDS.  However this would also mean in order to allow 1.8 clients with the full 64-bit hash value to use OBD_CONNECT_FULL20 they would also need to implement the fixed ptlrpc buffer hashing (commit 30dda5e6), and ensure that the obdo.o_gr field was always initialized to 0 (so ost_validate_obdo() works correctly with 1.8 clients).

Another option would be to use a new OBD_CONNECT_64BITHASH flag to distinguish these clients.  This might be OK for the 2.1 code since it hasn't really gone into production anywhere.
