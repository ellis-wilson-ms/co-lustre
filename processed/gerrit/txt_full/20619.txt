Okay I did some looking at the git history of the linux kernel. Since the introduction of bio_for_each_segment_all() for the 3.10 kernels it always has taken a integer for its last parameter. As for bio_for_each_segment() it took a integer parameter until the 3.14 kernel. Then it started taking the bvec_iter parameter instead. This means bio_for_each_segment_all() added to lustre to handle 3.9 or earlier kernels should work unless the distro does something really stupid. By stupid I mean they would have to back port all the new bvec_iter infrastructure which I doubt they will since that would be a lot of useless work.

Sorry for the late reply, I've been off the Internet for a while.
I should add "without this patch, 20478 breaks client build against 4.5" in the previous comment, I was showing the problem to Shilong.

I think we need to handle 3 cases here:
1. bio_for_each_segment_all is defined, which means we should use *bvec and int iter, depending on HAVE_BVEC_ITER, we use bio->bi_iter.bi_sector or bio->bi_sector
2. we only have bio_for_each_segment, and HAVE_BVEC_ITER is true.
we need bvec and bvec_iter for this case, plus bio->bi_iter.bi_sector.
3. we only have bio_for_each_segment, HAVE_BVEC_ITER is not there.
This means we should use int iter and *bvec, plus bio->bi_sector.

I'm thinking about something like this:
#if defined(HAVE_BVEC_ITER) && !defined(bio_for_each_segment_all)
        struct bvec_iter      iter;
        struct bio_vec        bvec;
#else
        int                   iter;
        struct bio_vec       *bvec;
#endif

and then
#if defined(HAVE_BVEC_ITER) && !defined(bio_for_each_segment_all)
                offset = (pgoff_t)(bio->bi_iter.bi_sector << 9) + lo->lo_offset;
                bio_for_each_segment(bvec, bio, iter) {
                        BUG_ON(bvec.bv_offset != 0);
                        BUG_ON(bvec.bv_len != PAGE_CACHE_SIZE);

                        pages[page_count] = bvec.bv_page;
                        offsets[page_count] = offset;
                        page_count++;
                        offset += bvec.bv_len;
#else
# ifdef HAVE_BVEC_ITER
                offset = (pgoff_t)(bio->bi_iter.bi_sector << 9) + lo->lo_offset;
# else
                offset = (pgoff_t)(bio->bi_sector << 9) + lo->lo_offset;
# endif

# ifdef bio_for_each_segment_all
                bio_for_each_segment_all(bvec, bio, iter) {
# else
                bio_for_each_segment(bvec, bio, iter) {
# endif
                        BUG_ON(bvec->bv_offset != 0);
                        BUG_ON(bvec->bv_len != PAGE_CACHE_SIZE);

                        pages[page_count] = bvec->bv_page;
                        offsets[page_count] = offset;
                        page_count++;
                        offset += bvec->bv_len;
#endif

which should work for both old kernel and new, but we need to touch lustre/include/lustre_compat.h and lustre/osd-ldiskfs/osd_io.c as well.
