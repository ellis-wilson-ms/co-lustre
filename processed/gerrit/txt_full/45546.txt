Andreas,
I agree this is more like a bandaid than fix.
I think there are several problems:
__ext4_xattr_set_credits() from upstream just enlarge the credit needed
no matter if ea_inode is actually needed or not.
a xattr create only needs 10 vs 20 if ea_inode is needed and the xattr  can fit in 1 block.

dt_find_or_create code path only declare create op,
but osd_create will set fid in inode ea, the credits were not reserved for the xattr set I think.

then in lustre the osd_dto_credits_noquota[DTO_XATTR_SET] is a fixed 14, should be consistent with ldiskfs, large when ea inode is used?(not related to this ticket)
