Patch Set 1: Code-Review-1

(1 comment)

1) I'm not against the first part of this patch, since I agree that checking the descriptors before journal replay isn't the best. That said, I don't think this would have avoided the "Block bitmap for group 192 not in group (block 18446744073709551615)!" problem since the bitmaps never change location and it is very likely that even after journal replay that this group descriptor would be corrupt. My second comment here is that this kind of fix should be sent upstream to the linux-ext4@vger.kernel.org list, so that it is merged into the upstream code and doesn't have to be updated for every new kernel forever into the future. 

2) the fixes for checking dev_read_only() cannot be sent upstream. They would be OK to land as an ldiskfs patch, but need to be fixed first. As previously mentioned in my review, the error handling should be fixed. Secondly, we are moving away from the dev_read_only() patch (see LU-684 which will be landed by 2.10) so that we don't need to patch the kernel to run tests. In that case, the dev_read_only() code needs to be under #ifdef since newer kernels will not have this function at all, and there will need to be some other way to detect this in the testing case.
