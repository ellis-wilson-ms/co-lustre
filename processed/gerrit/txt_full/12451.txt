--- Commit Message ---
LU-5718 o2iblnd: multiple sges for work request

In current protocol, lnet router cannot align buffer for rdma,
o2iblnd may run into "too fragmented RDMA" issue while routing
non-page-aligned IO larger than 512K, because each page will
be splited into two fragments by kiblnd_init_rdma().

With this patch, o2iblnd can have multiple sges for each work
request, and combine multiple remote fragments of the same page
into one work request to resovle the "too fragmented RDMA" issue.

(3 comments)

Shadow, this is really about router, because router will always set kiov_offset of router buffers to 0 despite of buffer offset from peer, which means if kiov_offset is not zero on remote peer, either on router or remote peer, most pages will be split into two fragments because of misalignment, and IO > 512K will exceed fragment limit of o2iblnd (256).

Of course we can change o2iblnd and send rdma descriptors to peer in PUT_REQ, but this is a protocol change that I'd like to avoid.

In this way, there is no protocol change, and it has zero overhead for aligned cases.

I think that fixing style issues in files should be addressed in Jira tickets specifically opened to address such issues.  Like what James Simmons is doing for the upstream staging area.  The problem I see with fixing style issues in bug fixes like this is it creates more change than is required to fix the bug.  That has two negative effects:

1- More stuff to be inspected.  That in turn starts a lot of patch churn as each inspector expresses different opinions on style rather than focusing on the bug fix.
2- Cherry picking the bug fix to other branches/repos carries a lot of extra changes which increases the chances of a collision.  Keeping bug fixes simple and minimal makes it much easier to port changes (something which happens very frequently with LNet).
