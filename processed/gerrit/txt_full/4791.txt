One question here is what is the effect of setting the OBD_CONNECT_FID flag?  It seems that even 2.1 clients (MDS OSCs and client OSCs and MGCs) connect to the OST with OBD_CONNECT_FID set.

Fortunately, it appears that they do nothing with this flag on the OST if it IS set, but the OST cannot depend on this to enable/disable features itself.  This will have to be handled by the MDT to decide that it is connecting to a FID-enabled OST to start requesting a FID sequence and allocating FIDs.  I _think_ this is how the DNE code is implemented, but just wanted to confirm...

Yes, OSP will check whether the OST support FID by 

static inline int osp_is_fid_client(struct osp_device *osp)
{       
        struct obd_import *imp = osp->opd_obd->u.cli.cl_import;
        
        return imp->imp_connect_data.ocd_connect_flags & OBD_CONNECT_FID;
}       

If the OST does not support FID, it will go with old protocol.
