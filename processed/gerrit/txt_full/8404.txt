--- Message (Christopher J. Morrone, 2013-11-27 21:24:06.000000000) ---
Patch Set 2:

Yes, that is basically what I was thinking.  At the very least, I see no reason to have both "all-sources" and "sources".  The only reason I can think of for "sources" is as a convenience, if folks want to manually run "make sources".

But they could just as soon run that under the ldiskfs directory...I guess if people think they still want that we could keep it.

I would imagine that we can eliminate the "lustre-sources" target without too much work in the near future.  It looks like the only thing it still does is generate the version number and really that should be done at configure time, not at build time.  If we move that into autoconf, we won't need that part in the makefiles.

I toyed with the idea of also generating the ldiskfs sources as part of configure, but that change was too large to incorporate in the ldiskfs build system patch I got landed in the recent past.  I tend to think that is really the right place to apply the patches.  If the kernel and the patches are not compatible, we would really want to learn that at configure time, not wait until build time.  We would just need to provide mechanism to make applying patches convenient for folks doing the development work, in addition to applying them at configure time.

But I digress...those are things for the future, not for this patch.
