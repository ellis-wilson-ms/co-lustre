--- Message (Andreas Dilger, 2023-02-02 20:34:54.000000000) ---
Patch Set 1:

It isn't clear why these are module parameters, instead of just using "lfs setdirstripe -D ... $ROOT" to set the desired layout?  I'm not totally against this, but it seems strange.

It is possible to return to the old "keep everything on MDT0000" behavior with "lfs setdirstripe -D -c 1 -i 0 $ROOT", though I'm not sure why that is better?  All files using only MDT0000 was one of the biggest support headaches for DNE systems (i.e. MDT0000 full, other MDTs empty), and the main reason why round-robin mkdir was enabled by default.
