--- Commit Message ---
LU-8688 tests: start client on MGS for sanity tests

When the Lustre file system has a separate MGS and MDS,
we need to start a client on the MGS when tests create
or manipulate OST pools.

Mount the client on the MGS for sanity tests that deal
with OST pools. Add a trap for all these tests to
umount the client from the MGS using stack_trap().

In order for stack_trap() to work in these cases, we
removed the call to "trap 0" in destroy_test_pools.

Test-Parameters: trivial
Test-Parameters: standalonemgs=true testlist=sanity,ost-pools
