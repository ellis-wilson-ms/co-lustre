--- Commit Message ---
LU-0000 tests: Determine sanity-hsm memory leak

In review-zfs, sanity-hsm is failing with a memory leak. This patch
is to determine what test is leaking memory.

Currently, 'complete $SECONDS' is called before
check_and_cleanup_lustre which is where the memory leak/hang occurs.
Move 'complete $SECONDS' after check_and_cleanup_lustre to see if
this will trigger a fail of the test suite.

Test-Parameters: fortestonly alwaysuploadlogs envdefinitions=SLOW=yes, mdtfilesystemtype=zfs mdsfilesystemtype=zfs ostfilesystemtype=zfs clientdistro=el6.6 ossdistro=el6.6 mdsdistro=el6.6 clientarch=x86_64 ossarch=x86_64 mdsarch=x86_64 testlist=sanity-hsm
