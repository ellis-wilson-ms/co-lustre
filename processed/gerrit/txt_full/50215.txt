--- Commit Message ---
LU-16429 mdt: add batch index into open file data on MDT

During the resend recovery for intent open, MDT uses RPC xid to
identify the open file handle. However, for batch intent open
(create), all sub open requests are using the same xid of the
batched RPC. In the old implementation, it always returns the open
file handle of the first sub request in the batched RPC wrongly.

To solve this problem, we add batch index in the open file data
(mfd) on MDT. Thus it can find the correct open file data
according to the xid and batch index during resend recovery.
