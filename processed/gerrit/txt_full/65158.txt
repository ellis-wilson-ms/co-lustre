--- Commit Message ---
LU-20095 osd-ldiskfs: clean up orphan list when truncate fails

If ldiskfs_truncate() fails for some reason, e.g. fs remounted read-only,
we should remove the inode from the in memory orphan list,
otherwise during umount we will trigger
J_ASSERT(list_empty(&sbi->s_orphan)) in ldiskfs_put_super().

Fixes: 1f3e4df2fb ("LU-18562 osd: llog write batch block allocation")
