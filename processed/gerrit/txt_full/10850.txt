--- Commit Message ---
LU-3660 acl: support mount option "noacl" for zfs

zfs parse mount option in zpl layer, but osd-zfs calls dmu
directly, since ldiskfs parse "noacl" at mount time, osd-zfs
should do the same in osd_mount().

Similar to ldiskfs, osd-zfs should check whether ACL is enabled
for ACL operations, if not -EOPNOTSUPP is returned.

add sanity test 103b for "noacl" mount option.

(3 comments)

I don't mind many of the fixes here, but I think there is a more serious problem with why the client doesn't disable ACL support when the MDT does not return OBD_CONNECT_ACL?  

According to mdt_init0() the MDT should only set mo_acl = 1 if MNTOPT_ACL is returned from the OSD.  In osd-ldiskfs::osd_conf_get() it returns MNTOPT_ACL only if the filesystem was mounted with "-o acl" (the default), but it seems that osd-zfs::osd_conf_get() it always returns MNTOPT_ACL.

It would be better to not return MNTOPT_ACL if ZFS is mounted with "-o noacl" so that the clients don't send ACL requests to the MDS at all.
