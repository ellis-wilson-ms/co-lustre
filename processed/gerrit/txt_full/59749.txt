--- Commit Message ---
LU-0000 ldlm: limit LDLM_CANCEL_LVB to one Active Writer lock

The LDLM_CANCEL_LVB protocol was designed to carry mirror
error data from client to server during Active Writer lock
cancellation. However, the current implementation allows
multiple locks to be packed into a single RPC, which causes
issues when multiple Active Writer locks are present.

The problem occurs because:
1. Client-side: ldlm_cli_cancel_list() uses LDLM_CANCEL_LVB
   if ANY lock in the batch is an Active Writer lock
2. Server-side: ldlm_handle_cancel_lvb() applies the same
   LVB data to ALL Active Writer locks in the request

This creates confusion when multiple Active Writer locks
are cancelled together, as they all receive the same mirror
error data regardless of their individual states.

This patch addresses the issue by:
- Limiting LDLM_CANCEL_LVB requests to only one lock per RPC
- Adding constraints in both ldlm_cli_cancel_list() and
  ldlm_cli_cancel_req_lvb() to enforce the single-lock limit
- Adding server-side warning if multiple Active Writer locks
  are found in an LDLM_CANCEL_LVB request

This ensures that each Active Writer lock gets its own
dedicated LDLM_CANCEL_LVB RPC with the correct mirror error
data, preventing cross-contamination between locks.
