--- Commit Message ---
LU-10026: Configuration for compression support and lz4 module backport

This patch adds build system changes for compression:
- If lz4 is not available in the kernel (< 3.11), we build a private module for it.
- Always build compression module except when ldiskfs is enabled.
- An option cl_compression, settable via lctl, enables or disables compression.

Patrick, thank you for reviews!
> Some small stuff, a few questions.
> One further question: LZ4 vs LZ4 FAST... What do we do if only LZ4 is found in the kernel?
Currently we don't do anything. We would want to add and use the fast-mode then, but it seems to be tricky.
I'm not sure if you can really split up the module to build only the fast-mode part and keep the old common lz4 compressor.
It also doesn't seem to be a good idea to build the complete module and have the common lz4 compressor twice.
So for kernel versions >3.10 and <4.11 we only have the old lz4. When using the functionality we currently would need kind of a querie which version is available and how the API looks like.

James, I understand you doubts, we just try to find the best way to move forward.

> The default lz4 has been port for RHEL7. I looked. The "fast" version looks to be missing.
Since we are a research group at a university, not a part of commercial industry, we and our students use the free CentOS, not RHEL.
CentOS does not have that speacial patch, so we need that backport. Whether only lz4 lib or together with crypto API, for both we need a backport, since both have been introduced in kernel 3.11.
I don't know how easy it could be to backport the cryto stuff together with its hardware support.

> I do want this project to consider the potential to use hardware acceleration which is why I brought up compression in crypto layer.
We keep that in mind. We have also had a look at Intel's QAT technology, but so far didn't have the capacity to take care of it.
You might look at these patches like at the minimal base for proof of concept purpose. Sure, there is a ton of things to be done..
