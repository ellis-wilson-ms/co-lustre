LU-16738 utils: fix auto mgsname= mount option

When mounting a client using a TCP hostname that does not contain
an "@nettype" component, the "@tcp0" nettype is assumed.  However,
this was not being handled properly when adding an "mgsnode="
mount option to capture the original hostname during mounting.

Only compare if the original mount device does not match the new
mount device before deciding whether to add an "mgsnode=" option.

Fix check for already-mounted filesystems to use "lustre_tgt"
instead of "lustre" when mounting server targets.

Simplify check_mtab_entry() and callers to avoid comparing devices
multiple times before deciding if the filesystem is mounted. Check
fstype and mountpoint before devname, and only pass mo_device once.

Update the mount.lustre.8 man page to better describe the options.
