--- Commit Message ---
LU-20114 ldlm: include resource and ibits_node in lock cost

ldlm_ratio2locknr() uses only sizeof(struct ldlm_lock) to
estimate per-lock memory, but each granted lock also has an
associated ldlm_resource (~1:1) and ldlm_ibits_node (server
IBITS). This underestimates true cost by ~1.5x, so reclaim
and enqueue rejection trigger too late under memory pressure.

Also fix the sysfs store functions for lock_reclaim_threshold_mb
and lock_limit_mb which had the same underestimate.

Expose lock_limit_count and lock_reclaim_threshold_count via
sysfs for debugging lock threshold calculations.

Add sanity test 134c to verify the accounting accuracy.
