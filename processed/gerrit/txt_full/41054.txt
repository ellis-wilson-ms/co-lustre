LU-14265 utils: allow 'lfs mkdir' without arguments

If no arguments are given to "lfs mkdir <newdir>" assume that
"-i -1 -c 1" were given, to create the specified <newdir> on
an MDT with above-average free space/inodes.

Note, when this is enabled, for directory created by 'lfs mkdir' without arguments, the default LMV on its parent will be ignored.

--- Message (Andreas Dilger, 2020-12-21 07:56:24.000000000) ---
Patch Set 1:

> Note, when this is enabled, for directory created by 'lfs mkdir'
 > without arguments, the default LMV on its parent will be ignored.

Hmm.  I'd guess that if the user wanted to follow the default LMV on the parent, they would just use "mkdir" directly?  The more common problem is that there *is* no default LMV on the parent, so all of the directories stay on MDT0000.

I was thinking about making this a bit smarter, to still keep the new subdirectory on the parent MDT when all of the MDTs are (mostly) evenly balanced (i.e. avoid a remote directory if it isn't needed), but for 2.14 that decision is made in LMV, and this patch is mainly intended to make "lfs mkdir" easier to use.

--- Message (Lai Siyao, 2020-12-21 09:26:49.000000000) ---
Patch Set 1:

Yes, directory created by 'mkdir' will honor parent default LMV.
