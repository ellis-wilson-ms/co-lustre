> I can easily test for TCP IPv6. IB is a bit more difficult. I have
 > an IB system but no IPv6 setup on it.

Not sure which procedure you're planning to use, but in the past I 
found the important steps to do when testing this feature are:

1) Test both "soft" and "hard" link disruption (i.e. "ip link set" vs. "pull cable out with your hand"). "Hard" disconnect may probably also be simulated by disabling a port on the switch.
2) Test what happens when originally assigned IP is deleted.
3) Test using a multi-port card with more than one interface used by LNet.
4) Test using bonded interfaces

The reason why I'm mentioning this is because I remember that skipping these steps for the new changes in the related area can turn into an issue.
I'll create a wiki which describes the procedure more coherently.
I'll also check if I can do anything with IPv6 IB on my setup.
