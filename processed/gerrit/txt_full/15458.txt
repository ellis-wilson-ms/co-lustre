--- Commit Message ---
LU-6780 ptlrpc: Do not resend req with allow_replay

If the request is allowed to be sent during recovery,
and it is not timeout yet, then we do not need to
resend it in the final stage of recovery.

Unnecessary resend will cause the bulk request to resend the
request with different mbit, but same xid, and on the remote
server side, it will drop such request with same xid, so it
will never get the bulk in this case.

re-enable the multiple MDT failover cases in replay-single.sh

--- Message (wangdi, 2015-07-01 15:43:22.000000000) ---
Patch Set 1:

"Di, please explain why this blocks recovery. I'd expect this to improve recovery time, but not block it totally."


So the situation should like this  MDT1 and MDT2 restarts at the same time,
1 MDT1 send bulk request (with rq_allow_replay, retrieving update log) to MDT2 during recovery, MDT2 get request and wait for the bulk.

2. And MDT2 just finish the replay at the moment, then MDT1 as a client, needs to resend all of request,

3. MDT1 resend the bulk request(in step 1) , and it will unregister the bulk, and resend the request with old xid  and new mbit.

4. And MDT2 still use the old mbit and old xid to wait the bulk. So the resend request with old xid will be dropped, and MDT2 will never get the bulk and timeout, which cause the recovery failed.
