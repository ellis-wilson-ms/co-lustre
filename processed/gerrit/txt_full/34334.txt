LU-11915 tests: clean up after conf-sanity 110/111/115

Clean up properly after conf-sanity test_110 and test_111 are
finished running by reformatting all targets, so that it doesn't
adversely affect tests running later in the session.

Use stack_trap() to clean up each test phase properly so that it
still cleans up the loop device on intermediate failures.

Remove the useless "reformat" wrapper and just call "formatall"
directly, to avoid the appearance that these two functions do
something different.
