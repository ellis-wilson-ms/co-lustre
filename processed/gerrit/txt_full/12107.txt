(2 comments)

I also wrote a similar patch http://review.whamcloud.com/12149 that I hadn't pushed until now.  It creates a temporary file in the directory and then uses:

        lfs getstripe -M $file

to determine the actual MDT that the file was created on.  That solves the problem of a potential default directory layout in the future, but doesn't solve the problem with test_mkdir() creating 2-stripe directories.  It probably makes sense to fix code using get_mds_idx() to use "test_mkdir -c 1" to avoid this problem.

My patch also does a bunch of other cleanups.  I'm not sure if it is better to actually create and delete a file for this test, or to use the FLDB method used in this patch.  If this one lands first, I can always rebase mine to use this method.
