--- Message (Andreas Dilger, 2020-04-06 19:09:07.000000000) ---
Patch Set 1:

> Because all version before RHEL7.6 versions are no longer tested.
 > Are we sure they are 100% okay for general use? Additionally
 > RHEL7.5 and before has mlx memory allocation issues and we have the
 > spectre issues with those earlier kernels as well.

We also don't test SLES or Ubuntu servers for every patch either, nor every client kernel version.  I think it is up to the users/admins to decide what kernel they are using, with the understanding that we are only testing the latest releases.  For some users, they may not care about Spectre vs. the ability to run their specific kernel without having to upgrade first.

I'm not against removing kernels when it cuts down on significant code complexity, but I prefer to be lenient if it doesn't make much difference.  At some point, either those old kernels won't work, and we will remove them, and/or people stop using them.

In summary, I'm not *dead* set against this, but it seems a bit too aggressive and may cause some users heartburn for relatively little benefit on our side.  Why the rush to remove the RHEL7.x series, but not e.g. ubuntu14+16 or sles12, which are *far* less widely used (if at all)?
