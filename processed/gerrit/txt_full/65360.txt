--- Commit Message ---
LU-20119 selinux: introduce lustre_helper_t domain

Replace the kernel_generic_helper_t-based rules with a dedicated
lustre_helper_t domain and lustre_helper_exec_t file type.

kernel_generic_helper_t is a shared type assigned by the kernel to
every process spawned via call_usermodehelper(), regardless of which
subsystem initiated the call. Granting permissions to it affects
all UMH callers, not just Lustre.

The new approach labels the Lustre helper binaries (l_getsepol,
lctl, l_getidentity, l_getauth) as lustre_helper_exec_t via the
new lustre_helpers.fc file context spec, and adds a type_transition
rule so that kernel_t transitions to lustre_helper_t on exec of
those binaries. All Lustre-specific permissions are confined to
lustre_helper_t.

On RHEL/Rocky 9 the base policy's kernel_t -> kernel_generic_helper_t
transition covers all UMH helpers and Lustre works without this
module. On RHEL/Rocky 10 that transition was removed together with
kernel_generic_helper_t itself, so kernel_t execs the helper binary
directly; the kernel_t -> lustre_helper_t transition handles that.

Makefile.am is updated to pass -f lustre_helpers.fc to
semodule_package and to run restorecon after module installation.

Tested on Rocky 10.1 (selinux-policy-42.1.7) with SELinux enforcing.
