--- Commit Message ---
LU-14477 lnet: handle possiblity of IPv6 being unavailable.

If CONFIG_IPV6 is not enabled, the attempt to create an IPv6 socket
for accepting new incoming connections will fail.  In that case
we need to creae an IPv4 socket instead.

Also ipv6_dev_get_saddr will not be available, so we must not include
the code that tries to use it.

Patch Set 2:

> Just to confirm - if the IPv6 socket *is* created, can it also accept connections from IPv4 clients?

Yes it can.  This is the default behaviour for IPv6 listening sockets, though it can be disabled by setting the IPV6_V6ONLY socket option to 1.  The option defaults to 0, and I explicitly set it to 0 in case the default were changed.

> Has this been interop tested with older clients/servers that don't have IPv6 support?

I don't think there is anything special to test.  An IPv4 connection is the same whether it comes from an old kernel or a new kernel.  Lustre cannot yet make IPv6 connections, so all connections will currently be IPv4.  If accepting an IPv4 connection on an IPv6 socket didn't work, lustre wouldn't be working at all.
