LU-19849 build: disable GSS if libkeyring missing

Lustre GSS support is enabled automatically if found in the kernel,
but configure fails if libkeyutils is not installed:

    checking whether to enable gss support... auto
    checking whether to enable gss keyring backend... yes
    checking if kernel was built with CONFIG_KEYS... yes
    checking for keyctl_search in -lkeyutils... no
    configure: WARNING: GSS keyring backend requires libkeyutils
    configure: error: Cannot enable gss_keyring. See above for details

Instead, if GSS support is auto-detected, then missing libkeyutils
should auto-disabled GSS again:

    checking whether to enable gss support... auto
    checking whether to enable gss keyring backend... auto
    checking if kernel was built with CONFIG_KEYS... yes
    checking for keyctl_search in -lkeyutils... no
    configure: WARNING: GSS keyring backend requires libkeyutils
    :

It is still an error if --enable-gss or --enable-gss-keyring are
specifically requested and libkeyutils cannot be found:

    checking whether to enable gss support... yes
    checking whether to enable gss keyring backend... yes
    checking if kernel was built with CONFIG_KEYS... yes
    checking for keyctl_search in -lkeyutils... no
    configure: WARNING: GSS keyring backend requires libkeyutils
    configure: error: Cannot enable gss_keyring. See above for details
