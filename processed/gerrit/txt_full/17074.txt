LU-3322 lnet: make connect parameters persistent

Store map-on-demand and peertx credits in the peer, since the peer
is persistent. Also made sure that when assigning the parameters
received on the connection to the peer structure through create,
that if another peer is added before grabbing the lock we assign
these parameters to it as well.

Patch Set 3: -Code-Review

In patch set #2 the ibp_queue_depth/ibp_max_frags are set in three places: 1. in the race found in kiblnd_passive_connect(), 2. kiblnd_reconnect(), and 3. in kiblnd_create_peer().   If it wasn't one of the first two conditions the peer accepted our own connection parameters which were the defaults now set in kiblnd_create_peer().  So we don't really have to update them.  The kiblnd_rejected->kiblnd_reconnect path will take care of the necessary updates due to a mismatch.
