--- Commit Message ---
LU-4945 mdc: resolve hash conflict with same hash offset

The original way to resolve hash conflicts is to keep entry
pointer during iteration, but it needs to hold the ldlm lock,
otherwise entry(op_ent) might refer to some random entris.

It will use op_same_hash_offset to remember entry offset among
the same hash entries, so mdc_read_entry will tell which entry
the caller needs.
