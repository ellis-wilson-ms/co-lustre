--- Commit Message ---
LU-20057 gss: do not use lookup_user_key() if cred overrided

In construct_get_dest_keyring(), lookup_user_key() must not be
called because if current thread has no session_keyring
set, then an atttempt to populate it will cause commit_creds()
to be called which will trigger a BUG() because we are already
running with overrided cred.

The fix is to call key_get() instead which is conform to what is
being done in Kernel's version of construct_get_dest_keyring()
and which was already done before e2156e69cc ("LU-18145 gss: fix
key unlink for regular user") has landed.
