LU-16766 llite: fix JobID for readahead threads

Properly assign the JobID for async readahead threads to be that
of the calling process, rather than "ll_sa", when using a JobID
that contains the process name, so these RPCs are tracked against
the correct application process.

Add a test to confirm ll_sa is not being used for the JobID.
