--- Commit Message ---
LU-5568 lnet: fix memory leaks and avoid double free

Problem is caused by following commit:
8fab48a8ef0bad6961c2ca1e2959726252ba43ac

lnet_startup_lndni() should free @ni since caller dosen't
do it, and if we have added this network into global list,
lnet_shutdown_lndni() will free @ni memory in that error path,
so we need not call lnet_ni_free() explictly, otherwise double
free happens.
