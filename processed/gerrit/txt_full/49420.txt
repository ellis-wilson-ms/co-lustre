LU-16227 utils: Add warning for lfs setdirstripe -D -i x,y,z

Adjust setdirstripe to be more user friendly. The
use of "-D -i x,y,z" now returns a clear error
that this is creating a default striped directory
layout and that this is a bad idea, if it is not
accompanied by "-c N" that matches the number of
index values given.

Thanks for the feedback!

The "unrecommend max-inherit" warnings only trigger when "max_inherit" is "LMV_INHERIT_UNLIMITED" or greater than "LMV_INHERIT_DEFAULT_STRIPED + 2" (lfs.c:6917). It doesn't seem like an error is thrown if "max_inherit" is "LAYOUT_INHERIT_UNSET".

A command like "lfs setdirstripe -D -i0,1,2 -c3 <directory>" fails with "Invalid argument" because "llapi_dir_set_default_lmv" returns an error when "param->lsp_is_specific" is set to true (liblustreapi.c:960). "param->lsp_is_specific" is always true when "lsa.lsa_nr_tgts > 1" and "llapi_dir_set_default_lmv" is always run when "-D" is specified. I think I'd still need to add another conditional to throw a more verbose error before this happens. Although, it seems strange that arguments like "-D -i 0,1" always cause a failure.

I'll submit another revision with my ideas + the other fixes you mentioned.
