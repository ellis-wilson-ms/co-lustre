I don't want to reinvent this wheel every time. See redhat readonly-root: https://access.redhat.com/solutions/2196881

Basically it all plugs into early boot, allows you to define random parts of fs as stateless and start either as empty or a copy of the preexisting r/o fs files.

So I override: separate /boot for every vm (so I can have per-vm kdump initrd as it cannot be generic), kdump.conf (same reason), /etc is copied as "mutable stateless" so Lustre tests can make whatever changes it needs but they don't persist across reboots.

Under the covers there's a bunch of other readonly-root enabling stuff happening of course (that sles might already have), but again I am trying not to reinvent any bicycles and also doing it separately means there are more chances of it breaking as infrastructure/distros get upgraded while my rhel recipe works the same from rhel6+ when I first started to this day with no changes and I can freely use the same VMs between all those different rootfses.
