LU-6488 tests: remove the check for ZFS to test large file

test_250 calls getstripe on a file before creating it and
can fail test as it doesn't exists.

This patch removes the check altogether thus enabling
the large file test to run irrespective of FS type,
e.g. LDISKFS or ZFS.

--- Message (Andreas Dilger, 2016-04-17 10:59:03.000000000) ---
Patch Set 1: Code-Review-1

The test being changed is expected to fail for writes above 16TB, so only for ldiskfs. This isn't a limit for ZFS and the test should not fail there. 

That said, the test does need to be fixed. Firstly, it should check that the FSTYPE is ldiskfs instead of checking that it isn't zfs. Your removal of getstripe is the right goal, but it could be done differently - check index 0, then force the file to be on OST0, or similar. Alternately, create the file on any OST first, then move the check after the file was created.
