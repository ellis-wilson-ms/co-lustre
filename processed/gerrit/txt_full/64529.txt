--- Commit Message ---
LU-19975 nodemap: add rbac property to default nodemap

It is useful to have the rbac property available on the default
nodemap, in order to control what can do clients that do not belong
to any other explicit nodemap.
All existing roles are available on the default nodemap, except
local_admin which would not make sense as there are no possible id
mappings or offsets on the default nodemap.

Tests in the sanity-sec test_64 family are enhanced to exercise roles
on the default nodemap.
