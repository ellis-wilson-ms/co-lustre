--- Message (Christopher J. Morrone, 2014-07-24 18:33:55.000000000) ---
Patch Set 1:

Oleg, you've clearly missed an important point.  Please read Nathan's comments in LU-5074 very carefully.

This is a feature that will _not_ land in the end.

This is an entirely proprietary feature that Xyratex has admitted they cannot share with the open source tree because they are contractually bound to _not_ share the code.  Unless Xyratex can find a way out of that contractual requirement, we have zero expectation that this code will ever see the light of day.

Since we _know_ that is the case, I think we need to see some actual code to believe that they have overcome their contractual limitations before we can reasonably expect that they are going to land code at some point.

Now, more generally speaking, yes, we could land a reservation for things that are going to be developed in the future.

We should, however, have some evidence that there will really be code provided in the future.  A set of design documents and/or prototype code would perhaps be the evidence needed to justify a reservation.
