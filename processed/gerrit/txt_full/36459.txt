LU-12734 misc: allow older bash_completion versions

Allow the "lctl" bash_completion to work on older versions which
which don't have _init_completions().  Check at runtime if this
function is available, and if not fall back to an older interface.

Has been manually tested with both bash-completion v1.3 and v2.1.

> but I wasn't sure what functionality would be missing

From what I can read on v2.9, _init_completion() completes variable names, filenames after redirections (and hides redirections when they are not the part of the line that is being completed), and it can split long options too, but that last one we don't use (it is an option: '-s').

So, nothing too important, but I guess it is nice to keep calling _init_completion() when it is defined.
