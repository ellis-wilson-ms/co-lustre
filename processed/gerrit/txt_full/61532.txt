--- Commit Message ---
EX-12873 lustre: Add IOCTL to compute ec parities.

This adds a new ioctl to generate the parities.

We will need access to the lustre EC kernel module from userspace
so that we can use the same code from both kernel and userspace.

Exposing the parity computations to userspace also makes it easy to
create tests to verify the correctness of the computations.

On some configs we are almost out of stack at this point so we need to
stuff as many of the local variables we can into a dynamically allocated
structure.
