--- Commit Message ---
LU-10843 mgs: allow snapshot after MGS remount

If the MGS is unmounted/mounted without the MDS reconnecting,
the fsdb FSNAME-barrier would not be created.

This change allows mgs_barrier_freeze (called from snapshot_create)
to attempt the create the required fsdb.

This adds a test to sanity-lsnapshot.sh for this issue.

Patch Set 1: Code-Review-1

(1 comment)

Setup the barrier fsdb during mgs_barrier_freeze() only can resolve the trouble for snapshot related usages, but the barrier related APIs, such as freeze, thaw, stat, rescan, can be called by others directly. So if the MGS is remounted, then before mgs_target_reg() called, the barrier fsdb will not be setup, as to direct barrier operations will get failure.
