--- Commit Message ---
LU-0000 tests: add file creation tests for EC layouts

Add file creation and removal steps to all EC layout tests
that pass validation. This provides a testing framework for
implementing EC layout support in the kernel.

Each test that creates a valid EC layout now attempts to
create a file with that layout using llapi_layout_file_create().
Currently all file creation attempts fail with "Inappropriate
ioctl for device" which is expected since kernel-side EC
layout support is not yet implemented.

Tests modified:
- test42: Valid partial EC component ordering
- test44: Complete EC layout with all components
- test45: Two mirrors - EC only first mirror
- test46: Two mirrors - EC both mirrors
- test47: Three mirrors with different extents
- test50: EC components can skip data components
- test52: EC parameter constraints are enforced
- test53: Duplicate EC components are rejected

This foundation will help identify missing kernel support
and validate end-to-end functionality as EC layout
implementation progresses.
