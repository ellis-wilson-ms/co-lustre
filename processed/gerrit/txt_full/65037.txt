LU-20074 ptlrpc: fix nrs_tbf refcount use-after-free

A prior LU-16796 patch replaced atomic_t with refcount_t in
NRS TBF, but incorrectly used refcount_inc() in nrs_tbf_hop_get().

When a TBF client hits 0 references, it moves to an LRU list
for deferred destruction. If a new RPC arrives before garbage
collection, nrs_tbf_hop_get() finds the object with a 0-count.
Calling refcount_inc() on a 0-count object violates kernel
hardening rules, triggering a panic.

Fix this by cleanly reviving the object via refcount_inc_not_zero()
and refcount_set(1).

Fixes: 7375cd6da6 ("LU-16796 ldlm: Change struct ldlm_resource to use refcount_t")
