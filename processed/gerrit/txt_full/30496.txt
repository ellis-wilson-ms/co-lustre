--- Commit Message ---
LU-10344 tests: fix atime in sanity-hsm/test_24a

test_24a requires a file for which atime < {mtime, ctime}

Previously, the test would create the file, sleep 1 second and
append an empty string to the file. This was unreliable and
sometimes, mtime and ctime would remain the same.

Now, the test uses "touch -ad yesterday $file" to rewind the atime.
