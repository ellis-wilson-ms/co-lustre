--- Message (Andreas Dilger, 2022-05-05 17:28:26.000000000) ---
Patch Set 1:

PS: the credits declared at the ext4/ldiskfs level are almost always ignored because the transaction is started by osd-ldiskfs with a larger number of credits to allow compound operations like {actual update + last_rcvd + LAST_ID + Changelog, etc} in a single transaction handle. Then the ldiskfs-level "start_this_handle()" calls just inherit the parent transaction credits.

So whatever the problem is here, it likely needs to be fixed at the osd-ldiskfs or mdd or ofd level to declare more credits.
