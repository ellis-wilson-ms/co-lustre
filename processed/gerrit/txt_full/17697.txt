Patch Set 1: Code-Review-1

It isn't clear to me why this is being changed from LGPL to GPL, since this is used by userland tools?  It is fine to include LGPL code into GPL programs like the kernel and non-GPL programs, but it isn't OK to include GPL code into non-GPL programs.

Also, please do not remove the Intel Copyright statement from the header.

--- Message (James Simmons, 2015-12-22 23:06:04.000000000) ---
Patch Set 1:

This header is also used by kernel space as well.

--- Message (Andreas Dilger, 2015-12-22 23:39:09.000000000) ---
Patch Set 1:

It is fine to use the LGPL v2.1 code in a GPL v2.0-only program.  LGPL v2.1 is compatible with GPL v2.0, see:

 http://www.gnu.org/licenses/gpl-faq.html#AllCompatibility
 http://www.gnu.org/licenses/gpl-faq.html#compat-matrix-footnote-7

What we can't do is change this header to be GPLv2.0 only and then use it in a non-GPL userspace program, so it needs to remain as LGPLv2.1.
