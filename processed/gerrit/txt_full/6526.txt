--- Commit Message ---
LU-1333 hsm: Add hsm_release feature.

HSM Release is one of the key feature of HSM. It removes file
LOV objects, if the file has been archived previously.

This patch is implementing all mechanisms, from client to MDS,
to do a release, modifying LOV EA, triggering actions and checking
the file is in a correct state.

Can you please tell me how a typical HSM release is happening? I saw the file ll_file_hsm_release() is the entry function of HSM release but there is nobody calling it.

After some consideration, I think it would be an HSM release open would happen as follows:
1. normal open the file;
2. ioctl to turn it into an exclusive open;
3. do what you want;
4. IT_RELEASE_CLOSE to close the file.

>After some consideration, I think it would be an HSM release open would happen as follows: 1. normal open the file; 2. ioctl to turn it into an exclusive open; 3. do what you want; 4. IT_RELEASE_CLOSE to close the file.

Does your idea fits with that? It seems this will require a couple of RPC?

By IT_RELEASE_CLOSE, what do you mean by that? What kind of RPC is it? Where do you this intent?

We can rename it but we need to take care of the release specific checks.
