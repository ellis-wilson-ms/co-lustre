Patch Set 1: Looks good to me, but someone else must approve

I've put a question out in LU-241 to see if anyone is in production with the existing checksum algorithm.  Since that patch was around for a long time (since 1.8.4 or so) it may be in use there already.

If that is the case, then this patch will need to be changed to include a new checksum algorithm type (deprecating the old crc32c algorithm) so that it doesn't break existing systems using it.

You may want to consider submitting a separate patch (separate Change-Id) which goes down that route, so that it can begin testing and inspection in parallel until we hear back from anyone in LU-241.
