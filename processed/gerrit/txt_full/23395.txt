--- Commit Message ---
LU-4939 utils: Add scripts to output MGS config parameters

Add config output utilities to lctl.  Print a list of steps to
re-create any config parameters, OST pools and quota info.

These utilities are helpful in migrating configurations, or backing
up and restoring configurations during a writeconf.

usage: lctl --device [device] config_print <fsname>
lctl pool_list -c <fsname>

I'm not clear at what the idea is for these scripts.

Are they just random scripts that you think other people might find handy, but shouldn't be packaged or installed on system?  Then the proper place to put these would be the contrib subdirectory.

Are these supposed to be installed and make Lustre's tool set more complete and consistent?  In that case they really need work.  These script names are really too vague for packaging and installation.  And perhaps they shouldn't be separate scripts, but improvements to lfs or lctl, or whatever.

In any event, I see no changes to an .am files, so I doubt these are properly packaged, even just with appropriate EXTRA_DIST (Note that the contrib directory is one big blanket EXTRA_DIST, more-or-less, so you wouldn't need to edit any makefiles if you put them there.)
