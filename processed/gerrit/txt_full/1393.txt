--- Commit Message ---
LU-682 test for tar patched for completely sparse files.

test 102l: FAIL if tar does not handle sparse files with
zero blocks in a optimized way.

The test works by creating a large file with zero blocks.
This file is then tar'ed using 'tar' from the path. If
a un-patched version of tar is called, the tar command
takes >0.01 seconds provided the local disk system is
slower than 400GB/sec. If a patched tar is present tar
completes in <0.01 sec.

If we back-port this patch to 2.1.x series, then we should need to use our own tar with that.
