Patch Set 1: Code-Review-1

Ah, I see now.

It looks like you were editing the same code and committing it, but not using "git commit --amend" to merge the refinements into the original patch.  Using:

 git commit --amend -av

will merge any new changes into the previous patch (presumably one that you are working on, you don't want to do that when the top patch is not yours), and leave you with a single patch containing all of the changes.

It is really these later patches that should be abandoned, and the first one could be un-abandoned to take the single patch.

It is typically preferable to keep all changes on a single Gerrit Change-Id rather than abandoning the patches and creating new Change-Ids for each revision, since that makes it harder to compare inter-patch differences when reviewing a patch multiple times.  In this case, there is a need to abandon the later patches since they are not really needed.
