LU-2059 llog: MGC to use OSD API for backup logs

MGC uses lvfs API to access local llogs blocking removal of old code

- MGS is converted to use OSD API for local llogs
- llog_is_empty() and llog_backup() are introduced
- initial OSD start initialize run ctxt so llog can use it too
- shrink dcache after initial scrub in osd-ldiskfs to avoid holding
  data of unlinked objects until umount.

Some questions:
1) is there already a test case for remounting a target (MDT and OST) if the MGS is down?
2) is this the last code using fsfilt, and can it be removed now?

Possibly the old llogid and record count in the header could be used to check if the backup and the new llog are the same?

The (2) and (3) items are for later patches, but (1) would be better to address in this patch, if it isn't already.
