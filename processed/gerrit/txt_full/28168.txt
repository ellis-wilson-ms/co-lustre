LU-9780 test: Testing Round-Robin allocation without LU-977

This patch is for testing that bug in LU-977(incorrect
RR object allocation) is reproducible without LU-977 fix.
This test checks the uneven file(object) allocation by using
a new reproducer, rr_alloc, which uses MPI to create files in
parallel, and checking the even file distribution over OSTs.

Distribution check formula is adjusted as per implementation,i.e
some factors like - exhaustion of pre-created objects and counter
‘lrq_start_count’(lod_qos.c) reseed are also taken care so that
object allocation is not affected by these factors.

It is essentially LU-9780 patch with LU-977 revert. LU-977 fix is
reverted in this patch so as to check that 'rr_alloc' test is able
to reproduce the the problem in LU-977.

Fixes: LU-9780" target="_blank">https://jira.hpdd.intel.com/browse/LU-9780

Test-Parameters: trivial osscount=3 clientcount=6 \
                 envdefinitions=ONLY=rr_alloc testlist=parallel-scale
