--- Commit Message ---
LU-2789 lod: initialize objects in a detached stripe array

Initializing the ldo_stripe member of lod_device in place exposes some
races with attribute setting so instead we use a detached array and
assign that to ldo_stripe once it's complete.

striping information is created during declaration only, at this time the device above doesn't usually grab any local locks. so i think it should be fine to re-use existing dt_write_lock().

also, lod_declare_attr_set() and lod_attr_set() can be optimized to ignore any changes except for uid/gid/size. with this change i was able to run your test for hours successfully (w/o that, few minutes then hit).
