LU-12368 obdclass: allow 'lfs df' to specify cache age

Allow "lfs df -a N" to specify maximum statfs cache age, up to 3600s.
To preserve API/ABI compatibility, pass the max_age in the first field
of struct obd_statfs, __u64 os_type.  Since it is a __u64 and to allow
future needs, pass this as a 64-bit nanoseconds field.

Consolidate the lov_iocontrol() and lmv_iocontrol() handling of
ioctl(IOC_OBD_STATFS) to reduce code duplication and centralize logic,
while the similar IOC_OBD_STATFS handling code in mdc_iocontrol() was
removed entirely since it hasn't been used since Lustre 1.8.  The
reduction in duplicate code compensates for lines added elsewhere.

This requires converting the obd_osfs_age to be in ktime_t rather than
in seconds, which changes a lot of lines, but is relatively simple.

Rename ll_obd_statfs() to ll_statfs_ioctl() to better reflect that
this function is used for handling ioctl(IOC_OBD_STATFS).
