--- Message (Isaac Huang, 2014-04-01 23:10:20.000000000) ---
Patch Set 1:

I tend to think this is similar to setting blocksize when formatting a local FS, e.g. mke2fs -b block-size. The admin knows better how the FS is going to be used and can hopefully make a better choice of block-size. Here zfs allows block-size to be set per object, and we know better how an object is going to be used, so it seems to me that the block-size should be set by the osd-zfs.

--- Message (Andreas Dilger, 2014-04-02 02:13:05.000000000) ---
Patch Set 1:

If ZFS is setting the blocksize to 512 it is a bug. It should be set using the ashift value, which will already be 4096 for modern AF drives. 

That said, I don't mind being able to specify the blocksize from within Lustre. In some cases this will be useful, such as when the client is already doing large writes.

The ZFS code would ideally increase the blocksize dynamically depending on the file size. I think the ZFS VFS code does this, but there is some reason why this does not work with Lustre.
