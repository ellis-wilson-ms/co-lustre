--- Commit Message ---
LU-653 i_version shouldn't be used for VBR

RHEL6 plus EXT4 combination cause i_version is to be used to store
Lustre versions instead of i_fs_version. Meanwhile this field is used
by ext4 internally and may be changed breaking Lustre recovery.

Patch makes only i_fs_version to be used for versions.

--- Message (Andreas Dilger, 2011-09-14 16:41:48.000000000) ---
Patch Set 4: I would prefer that you didn't submit this

(1 inline comment)

Per discussion with Mike, the commit comment is misleading, because it isn't ext4 that is changing the i_version at all.  Rather, fsfilt_ext3.c is updating i_version, but ldiskfs is patched to write i_fs_version to disk (which is untouched from the time it is initialized to 1).

Beyond that, the check for last_rcvd < inode_ver is potentially broken and would cause permanent recovery problems under some conditions (e.g. last_rcvd is lost, and transno is reset to 0), so I don't think it should land.
