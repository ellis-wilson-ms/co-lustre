--- Message (Andreas Dilger, 2017-04-20 15:31:32.000000000) ---
Patch Set 1: Code-Review-1

(3 comments)

The main reason for -1 is the bitfield location. I would be OK to land this patch if that was fixed. Moving to a separate lfs-getstripe.1 man page could be a separate patch. 

What is still missing (should probably be a separate patch) is to allow the YAML layout as input:

   lfs setstripe --yaml=/path/to/template.yml {other opts} /new/file

and a few tests to ensure this works properly. It should also be possible to pipe the stdout from "lfs getstripe --yaml" into "lfs setstripe --YAML". I believe there is a YAML parser already used for Lustre by lnetctl so that should be re-used. 

That said, I think that implementing "lfs setstripe --copy=/path/to/template /new/file" will be much easier to implement and for users to use, since it doesn't involve using a temporary file.
