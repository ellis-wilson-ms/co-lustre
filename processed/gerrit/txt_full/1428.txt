--- Commit Message ---
LU-723 Remove Linux 2.6 requirement to enable ldiskfs

The LB_LINUX_VERSION test now runs after LB_PATH_LDISKFS, but the
ldiskfs tests depended on the detected kernel version.  Thus
configure was disabling ldiskfs by default.  At this point we can
safely assume that Lustre is being built on Linux version 2.6 or
newer, so we can simply remove the kernel version dependency when
determining whether to enable ldiskfs.

FYI, if this patch is a functional stand-alone cleanup, I don't insist that all of the potential < 2.6.0 code is cleaned up in this patch.
