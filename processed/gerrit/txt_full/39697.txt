LU-13915 ldiskfs: Avoid atomic operation while bitmap prefetch

It is expensive since test_and_set_bit is a atomic operation. So
use test_bit while bitmap prefetch to avoid call it frequently.

Despite the fact the patch can be useful for filled and fragmented bitmap where _test_ operation mostly returns 1, it can make things slower for a quite empty partition, where (_test_+_set_) operation is faster then _test_ operation + _set_ operation.

--- Message (Andreas Dilger, 2020-08-27 02:01:05.000000000) ---
Patch Set 1: Code-Review+1

> Despite the fact the patch can be useful for filled and fragmented
 > bitmap where _test_ operation mostly returns 1, it can make things
 > slower for a quite empty partition, where (_test_+_set_) operation
 > is faster then _test_ operation + _set_ operation.

In the case of an empty group, then it seems that test_and_set() would not be called as often as test_bit() would be called when the bitmap is more full (the common case), so that seems like the right tradeoff?  That makes the uncommon-but-fast case a bit slower, while making the common-but-slower case a bit faster...
