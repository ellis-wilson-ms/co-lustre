--- Commit Message ---
LU-8130 nrs: convert NRS ORR/TRR to rhashtable

Move away from the cfs hash implementation to rhashtable
for NRS ORR handling. Since rhashtable is lockless it
should also increase performance. For the NRS TRR handling
we can use Xarray since its based on OST index which are
sequential which will give better performance than an hashtable.
TRR entries are added to the Xarray but not removed until the
Xarray is destroyed.

Patch Set 22: Code-Review-1

I suspect that the crashes Oleg is seeing are caused by the xarray code rather than this patch exactly.
The xarray code in lustre (which is used in older kernels like the 3.10 one being tested) assumes that the memory available from radix_tree_node_cachep matches the "struct xa_node" structure.
However in 3.10 it is "struct radix_tree_node" and in Redhat 3.10 that has a slightly different layout to struct xa_node.

We might need to have a distinct xa_node structure for each different kernel that we support.
