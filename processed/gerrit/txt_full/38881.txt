--- Commit Message ---
LU-12275 sec: introduce null algo for filename encryption

Introduce a "null" algorithm for client side filename encryption,
which is basically a no-op.
This is needed because first implementation step only supports
content encryption, and not names. So give the ability to support
encryption policies that have a 'filenames_encryption_mode' property
internally set to LLCRYPT_MODE_NULL.

Patch Set 2:

> The reason I'm asking is what happens with newer kernels in which
 > we use the native fscrypto.

Indeed, this solution works for embedded llcrypt only. If using a client built on kernel 5.4+, it will rely on native fscrypt from the kernel, and not have the ability to support NULL encryption for filenames.
I do not think this is really an issue, as 5.4+ kernels are not officially supported. We will have to clearly document that all files encrypted from such clients with 2.14 will be unusable (corrupted file names) when upgrading to future 2.15 that adds filename encryption support.
