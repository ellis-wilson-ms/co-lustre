LU-0000 llite: add EC read/write IOCTL stubs

Add stub implementations of EC (Erasure Coding)
read/write IOCTLs to support EC component I/O
operations.

This patch adds:
- LL_IOC_EC_SET_MIRROR IOCTL definition
- llapi_ec_set() and llapi_ec_clear() functions
- llapi_ec_read() and llapi_ec_write() stub
  functions
- Kernel-side IOCTL handler in llite/file.c

The EC IOCTL (LL_IOC_EC_SET_MIRROR) is modeled
after the existing FLR mirror IOCTL
(LL_IOC_FLR_SET_MIRROR) and sets the designated
mirror for EC component I/O operations.

The stub implementations verify that the file is
opened with O_DIRECT (required for EC I/O) and set
the designated mirror ID. Full EC I/O functionality
will be implemented in subsequent patches.

The lfs ec read and lfs ec write commands now call
these IOCTLs to verify the kernel interface is
working correctly.
