--- Commit Message ---
LU-6246 tests: split JSON_NUMBER into int and float

Integers are all encoded with a floating point number. But decoding
large integers will result in rounding errors, which we don't
want. Instead, use an integer whenever possible. A number will only be
decoded as a float if it is not an integer (ie. presence of a dot).
