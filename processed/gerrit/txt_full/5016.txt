--- Message (Andreas Dilger, 2013-01-14 20:39:46.000000000) ---
Patch Set 1: I would prefer that you didn't submit this

Nathaniel, notwithstanding the typo in "envdefinitions", my understanding is that using "envdefinitions=FSTYPE=zfs" is _not_ going to work properly with autotest, because the autotest configuration itself (creating LVM devices, etc) needs to be done before the tests are started.

You should instead use the proper Test-Parameters: settings for the filesystem types, per http://wiki.whamcloud.com/display/PUB/Changing+Test+Parameters+with+Gerrit+Commit+Messages:

 Test-Parameters: mdtfilesystemtype=zfs ostfilesystemtype=zfs

and this will internally set the appropriate *FSTYPE environment variables.  The comment in the wiki page for "envdefinitions" explicitly warns against using it for "FSTYPE".
