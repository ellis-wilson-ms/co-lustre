--- Message (Lai Siyao, 2022-07-12 08:28:18.000000000) ---
Patch Set 1:

The cause of 413g failure is: exp->exp_obd->u.cli.cl_default_mds_easize was not set for the MDT0 (where ROOT is located), therefore ll_fileset_default_lmv_fixup()->ll_dir_get_default_layout() won't fetch ROOT default LMV back because server found the reserved space size of default LMV on client is 0.

The reason why exp->exp_obd->u.cli.cl_default_mds_easize is not set for MDT0 is that this value is updated upon md_getattr/md_enqueue, while for subdir mount, these requests may haven't been sent to MDT0 yet, that's why 413g fails randomly.

I'll provide a fix later.
