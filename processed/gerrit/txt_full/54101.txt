Thanks for bringing these scripts in!

This is good as is. Mabybe couple of improvements for next version.

I would really like to have errors=nn/warnings=nn instead of cumulative number. The reason is that soon we should be near zero errors, however warnings would still be left. Warnings mostly is over 80 chars..., string which is split accross two lines and multi line comment not ending on new line and few others. Unfortunately, these warnings would still remain, specially "over 80 chars" as these are errors/messages which are printed and no way to change them. Having errors/warnings numbers would give correct snapshot. 

Second, git-checkpatch have it also take a folder as an input?
