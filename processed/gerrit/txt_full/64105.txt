LU-17901 tests: fix sanityn test_81a intermittent EIO

The Maloo test logs show that test_80b's aggressive
directory migration leaves the MDS in a state where
the immediately following test_81a gets EIO on mkdir.
The kill -9 of the migration process was not followed
by wait or any MDS sync, so the MDS could still be
processing the killed migration when test_81a starts.

Fix cleanup_80b to wait for the killed migration
process and call wait_delete_completed to let MDS
processing settle.

Also add error checking on mkdir, setdirstripe,
and other operations in test_81a that were missing
it, use absolute paths instead of cd with relative
paths, and fix a "filed" typo.
