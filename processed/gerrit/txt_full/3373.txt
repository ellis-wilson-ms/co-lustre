LU-1621 capa: disable Lustre capa by force

Current Lustre capa framework has some security holes, and cannot
work as expected. It also affects the performance because of the
overhead of calculating Lustre capa. So disable it by force until
patched or new Lustre capa implemented.

--- Message (Fan Yong, 2012-07-25 02:39:41.000000000) ---
Patch Set 1:

I do not think so.

Currently, capa is enabled by default. But it will distinguish client type to determine whether generate and verify capa. For local client, capa should be bypass. But it is just "should". We have not tested capa since Lustre-2.0 released. Just as described in LU-1621, we ignore capa process for new changes since Lustre-2.0 released, there may be some potential issues introduced by those patches.

On the other hand, capa should take effect for remote client. But as I known, nobody use remote client feature on Lustre-2.x yet, and we never test remote client since Lustre-2.0 released. So nobody can guarantee the right behavior.

In fact, current capa has some known security holds, which cause the capa cannot work as original expectation. We have already re-designed Lustre-capa in original SUN period. But it was never implemented.

https://bugzilla.lustre.org/show_bug.cgi?id=18519

So current capa does not bring security assurance for most use cases, and may introduce potencial bugs. So please consider to land this patch to disable Lustre-capa totally.

Thanks!
