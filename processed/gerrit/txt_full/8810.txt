--- Message (Andreas Dilger, 2014-01-17 08:29:51.000000000) ---
Patch Set 1:

Note that it would make sense to set the OBD_INCOMPAT_FID flag on the OST once it is starting to use FID_SEQ_NORMAL objects for DNE (or for regular usage in the future).  That would ensure that the OST is not downgraded to an older version of Lustre that doesn't understand the higher group numbers.

It probably also makes sense to add a patch for b2_4, b2_5 and master that adds OBD_INCOMPAT_FID to the OFD_INCOMPAT_SUPP mask, since those versions understand FID_SEQ_NORMAL objects added in DNE Phase 1.
