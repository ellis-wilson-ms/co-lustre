--- Commit Message ---
LU-12668 lov: fix concurrent EC read page cache race

When multiple threads read the same EC file concurrently
in degraded mode, the second reader could find an
uptodate page in the cache and return it before EC
recovery had filled it with reconstructed data.

Fix by checking whether the page was filled by recovery
(PagePrivate2) before returning it from the cache, and
re-initiating recovery if needed.
