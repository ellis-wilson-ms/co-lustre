LU-19440 tests: sanity 127e entries found in wrong buckets

On aarch64 systems, the page size is 64K and so when attemping to make
4K writes, they will end up as 64K writes. To avoid this, read/write
size are only added if they are larger or equal to the page size of
the system.

Additionally, only the 4K writes were being check to see if they
existed, this issue has been fixed.
