LU-17814 utils: Split llapi_semantic_traverse

Extract the logic for processing a single directory entry
into a separate handle_path_entry function. For now, this
is just a refactoring with no functional changes - the
returned -EISDIR is immediately handled by recursing,
maintaining the same behavior.

This split prepares for parallel find implementation, where
we'll  be able to reuse the entry handling logic but handle
directories differently (queueing them for parallel
processing instead of recursing).
