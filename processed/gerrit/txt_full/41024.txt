> actually I'm still a bit skeptical on the idea of setting prefer on
 > every replicated file. IMO, this is expensive and brings very
 > little value given that we want (I guess) to use fast OSTs by
 > default.

I agree, which is why I've also suggested to use the NONROT flag on the OSTs to prefer writes to select the flash mirror of a file on the MDS, and for reads to select the NONROT OSTs on the client. 

See LU-10158 for a list of possible improvements to the OST selection policy. It would be fine to start with automatically preferring NONROT OSTs for read and write. That would avoid the need to set {{prefer}} in all the layouts, especially since users are unlikely to get this right all the time.
