--- Message (Blake Caldwell, 2014-04-17 16:21:35.000000000) ---
Patch Set 1: Code-Review-1

The verbose mount option is infrequently used, so it doesn't go very far notifying the user that the block tunables were changed. Our preferred solution would be emitting a message to  syslog if max_sectors_kb is tuned below a threshold (1MB, 4MB, or max request size) advising that changes are made within udev.


A log message to syslog would be preferable, because if there is a problem, it would likely get seen.
