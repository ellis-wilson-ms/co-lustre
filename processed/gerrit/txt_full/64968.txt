LU-12668 lov: skip FLR backoff for proactive EC switch

The proactive dead-OST detection in lov_io_mirror_init()
was gated on ci_ndelay_tried == 0, so it only fired on
the first IO iteration. On subsequent iterations (each
stripe set restart), the code fell through to the FLR
mirror-retry backoff which does a 10ms schedule_timeout
before eventually reaching the same EC switch. This added
~170ms of unnecessary sleeping for a 64MB degraded read
(one 10ms nap per stripe set).

Remove the ci_ndelay_tried == 0 guard so the proactive
detection fires on every iteration, bypassing the FLR
backoff entirely when we already know the OST is dead.
