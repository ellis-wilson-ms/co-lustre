LU-18687 lib: move libcfs userspace library

Move the libcfs userspace library to the top-level
lib directory. Eventually, all of the userspace
tools and libraries will live alongside it.

We move *.c files:
from: libcfs/libcfs/util/*.c
to: lib/libcfs/*.c

We move *.h files:
from: libcfs/include/libcfs/util/*.h
to: include/libcfs/util/*.h

> Is there any provision for sharing code between kernel and userspace, without resorting to large static inline functions in headers?  A bunch of stuff like struct accessors, nid2str/str2nid, etc. could be shared again if there was a clean way to do so.

Running something like:

git grep __KERNEL__ -- ':(exclude)*.h'

on latest Linus tree shows a handful of examples. For struct accessors, uapi seems to be the best place. I doubt those are super large?

For stuff like nid2str, I think canonical version would live in the kernel and the userspace library would copy the file over and compile it into libcfs. libcfs would still live in lib/, so we can still get rid of the weird automake stuff from the kernel build.

Do you have a list of what's duplicated?
