LU-15928 osd: pass keylen for dt index operations

*** WIP TODO ***
- finish all function conversions
- test random encrypted filenames to hit corner cases

With fscrypt passing binary names to the underlying ldiskfs storage,
the lookup API should be passing the length of the key through the
DT index API instead of calling strlen() on the string, otherwise
this may incorrectly process keys with embedded NUL bytes.
