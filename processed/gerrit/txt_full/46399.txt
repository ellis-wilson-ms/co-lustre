--- Commit Message ---
LU-15505 libcfs: reduce stack overflow warnings

Make the stack overflow warning less agressive, by makign it 1KB
short of overflow, rather than 3/4 of stack depth, which can be
hit on PPC64 systems too often.

Add a module parameter to allow this threshold to be changed.

--- Message (Andreas Dilger, 2022-04-02 01:13:30.000000000) ---
Patch Set 1:

I'm not sure in the end whether this patch should land or not.  There was a site hitting this stack warning regularly with PPC clients, but it turned out they were using a kernel module that consumed a *lot* of stack and Lustre just happened to be the only one that was making any noise about that, so the warning was in fact valid.

--- Message (Andreas Dilger, 2022-04-02 01:16:11.000000000) ---
Patch Set 1:

I think 1KB for 16KB stacks might be "too late", given that it is likely the task struct is also larger on these systems (I don't have a good idea how large this is on PPC64).  Beyond just abandoning this patch, I've been thinking "1KB on 4KB stack systems (if those even exist anymore), and 2KB on 16KB+ systems".

--- Message (Neil Brown, 2022-04-05 06:20:14.000000000) ---
Patch Set 1:

Linux has had CONFIG_DEBUG_STACK_USAGE since 2.6.23.
If that is enabled, then this extra warning isn't really needed.

If it is NOT enabled (likely for enterprise kernels), then maybe it is best for lustre to just give a similar gentle warning.  i.e. just one "info" line, no stack trace.
Maybe have a config option (module parameter?) to provide the stack trace...
