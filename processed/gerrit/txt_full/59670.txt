LU-12345 ldlm: add LDLM_CANCEL_LVB infrastructure

This patch implements the foundational infrastructure for
LDLM_CANCEL_LVB, a new lock cancellation protocol that enables
client-to-server communication during Active Writer lock
cancellation for immediate mirror operations.

Key components implemented:

1. Wire Protocol Extensions:
   - Added LDLM_CANCEL_LVB opcode (108) to lustre_idl.h
   - Created RQF_LDLM_CANCEL_LVB request format in ptlrpc
   - Updated wiretest.c files for protocol validation

2. Client-Side Implementation:
   - ldlm_cli_cancel_req_lvb(): sends enhanced cancel requests
     with LVB data containing client state information
   - ldlm_prepare_cancel_lvb_data(): packages client state
     for transmission to server
   - ldlm_should_use_cancel_lvb(): detects when to use
     enhanced cancellation for Active Writer locks

3. Active Writer Integration:
   - Enhanced ll_file_io_generic() to detect immediate mirrors
     and acquire Active Writer locks appropriately
   - Added comprehensive debug logging for Active Writer
     lock lifecycle and immediate mirror detection

4. Server-Side Enhancements:
   - Extended mdt_blocking_ast() to handle Active Writer
     lock cancellation with enhanced logging
   - Improved active writer counting and FLR state transitions

The implementation is stable and crash-free. All immediate
mirror operations work correctly with proper Active Writer
lock acquisition and release. The infrastructure is ready
for integration with the actual protocol usage.

Testing shows successful immediate mirror creation, Active
Writer lock detection, and proper FLR state transitions
without system crashes.
