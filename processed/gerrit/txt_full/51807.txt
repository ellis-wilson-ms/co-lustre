--- Commit Message ---
LU-17000 lnet: stop abusing lo interface status

The lo interface status was being abused to
transmit a sequence number. This was being
hidden via a macro. Add a proper struct field
for seqno and remove the macro. Remove another
unused macro at the same time.

Incidently, this should resolve some static
analysis issues.

NI status should be under lock, but this wasn't
a real NI status. Hence the false positive.

Addresses-Coverity-ID: 397680 ("Data race condition")
