--- Message (Zhenyu Xu, 2023-03-13 15:20:21.000000000) ---
Patch Set 2:

> Patch Set 2:

> How does the converting flag escape from ldlm_cli_inodebits_convert()?
> 
> It looks like it's always set and unset there, and only there, so I don't see how it could be set when we're doing resource cleanup?

after setting converting flag, it unlock the resource and it goes to LDLM_CB_CANCELING part of the blocking ast, which executes the bits downgrade, then lock the resource again and clear the flag.

At the same time, the client could be evicted for some reason, and the resource cleanup could pick up the lock to cancel it and ldlm_cli_cancel() then trip the assertion.
