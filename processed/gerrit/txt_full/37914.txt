LU-12678 lnet: discard WIRE_ATTR

This macro adds nothing of value, and make the code harder to
read for new readers so it was remove for the Linux client.
We still want to keep track of what data structures are
transmitted over the wire and ensure the protocol does not get
broken. Move the wire protocol structures to their own header
files and add wire checking.

(2 comments)

To be honest, I think that WIRE_ATTR makes it totally clear that those structures are passed over the wire, and should not be changed. I don't think "packed" is nearly so clear to the reader. 

Unlike Lustre with lustre_idl.h and lustre_user.h being dedicated to structures used on the network and with userspace, the constants for the LNDs are lumped in with other in-memory data structures and function definitions in the same header file.  I also suspect (but am not positive) that these wire structures are not all checked in the LNet equivalent of wirecheck/wiretest, so there seems to be a real risk of accidental protocol breakage.

While that was not a huge deal in the past due to the relatively little change to the LNet protocol, this is a huge concern as we move ahead with IPv6 changes, so we need as much correctness checking as possible.

I'm of course always open to being proven wrong and there are lots of correctness checks for the LNet protocol, but let's make sure we have those in place before we remove this otherwise harmless macro.

--- Message (Andreas Dilger, 2020-03-15 22:04:38.000000000) ---
Patch Set 3:

At least socklnd.h looks like it is only holding wire structures, but gnilnd.h and o2iblnd.h have a mix of both.

I believe that there already *is* a version of wiretest for LNet, I just didn't recall the exact details off the top of my head.  It looks like the core LNet data structures are checked in lnet/lnet/api-ni.c::lnet_assert_wire_constants().

It looks like lnet/utils/wirecheck.c is supposed to generate the checks, but the majority of updates are just code cleanups (only one in 2017 related to ping messages), and it doesn't appear that there are corresponding checks for the wire structures/constants of the LNDs themselves.  The comment in lnet_assert_wire_constants() reports it was generated in 2004...

Separating the over-the-wire declarations into their own files (e.g. o2iblnd_idl.h) would improve things, as well as having each LND have a function to do its own wirecheck, like o2iblnd_assert_wire_constants(), so it is totally clear that this is being done properly.  Under normal circumstances, this function would be a no-op at module init time, since it is mostly/only BUILD_BUG_ON() that do not generate any code if correct.
