LU-17918 llite: protect layout in ll_delete_inode

Mirror operations could change layout during
ll_delete_inode()->cl_sync_file_range(), so we need to protect the
layout during the process, otherwise it could access NULL sub object
in LOV:

BUG: unable to handle kernel NULL pointer dereference at 0000000000006
IP: cl_object_top+0xe/0x130 [obdclass]
Call Trace:
    cl_io_sub_init+0x1f/0x70 [obdclass]
    lov_sub_get+0x52a/0x910 [lov]
    lov_io_iter_init+0x2da/0xab0 [lov]
    cl_io_iter_init+0x51/0x100 [obdclass]
    cl_io_loop+0x71/0x200 [obdclass]
    cl_sync_file_range+0x142/0x150 [lustre]
    cl_delete_inode+0x142/0x150 [lustre]
    evict+0xd2/0x1a0

While umounting the filesystem, we'd ignore the layout protection as
there is no other operations would change the layout during the
umount.
