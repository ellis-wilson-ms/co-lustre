LU-11860 lnet: support config of LNDs with numeric intf name

This patch adds support for the net configuration of LNDs that
have numeric interface name (PTL4LND, GNILND). The GNILND case has
already been treated with a specific fix (see patch a29eb587).

Maybe I'm missing something, but we definitely used to have interfaces of this type, such as qswlnd and gmlnd, that has numeric interface names. Rathe rhan hard-code the string-to-nid conversion, each LND type registered its own NID parsing method that was used. That allowed arbitrary parsing routines if needed, but allowed shared routines for the common IPv4 dotted-quad format.
