LU-7160 mgs: Skip processing .bak files on MGS

lctl replace_nids command saves previous version of
config files to file with original_name.bak file name.
This file should never be processed by MGS.

This patch adds code that skips file with .bak extention
from list to be processed by MGS.

Patch Set 6: Code-Review+1

I'm not against this patch since it is an improvement over the current, but I think it could be more robust. In addition to .bak files, I've personally made backup files of the form "myth-MDT0000-20160811" or similar before doing some major system change. 

It would be more robust if the replace NIDs code looked for specific filenames like "fsname-{OST,MDT}nnnn" and "fsname-client" and only changed those instead of changing every filename that it finds.
