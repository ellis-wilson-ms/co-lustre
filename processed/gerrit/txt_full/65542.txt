LU-20198 nodemap: do not inherit gss_identify from default

The gssonly_identification flag cannot be set on the default nodemap,
but nodemap_inherit_properties() copies it unconditionally from the
default to every newly created nodemap. If the on-disk default record
has this flag bit set (e.g. from stale data in a field that was
previously padded before being repurposed for NM_FL2_GSS_IDENTIFY),
every new nodemap inherits the unintended value, and there is no way
for an admin to clear it on the default.

Skip inheritance of nmf_gss_identify when the source is the default
nodemap. Also expose gssonly_identification read-only on the default
nodemap so admins can observe its underlying value for diagnostic
purposes.
