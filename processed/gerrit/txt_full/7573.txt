Patch Set 3: I would prefer that you didn't submit this

This functionality was not implemented for the new stack because we expected LFSCK Phase 2 would be ready by this point.  I'd prefer to put this code under a LUSTRE_VERSION_CODE check, so that the setea functionality is automatically disabled for 2.5.53+.  Otherwise, it will remain open to abuse from userspace tools other than lfsck but it will not be needed after LFSCK Phase 2 is finished (definitely for the 2.6 release).

It might not be a bad idea to add a check for CAP_DAC_OVERIDE as well, so that at least it is limited to root-only users.
