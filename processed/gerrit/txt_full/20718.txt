--- Message (Andreas Dilger, 2016-07-09 06:50:47.000000000) ---
Patch Set 2:

In general I also prefer not to have "-r" when deleting only files. That reduces the danger of mistakes if e.g. tfile="foo " then it tries to run:

 rm -rf $DIR/foo *

which might be bad, or if DIR="" and tfile="" then it becomes:

 rm -rf /*

which is much worse.

That said, the current patch is already much better in this regard than the older code, so I'm reluctant to block this from landing.
