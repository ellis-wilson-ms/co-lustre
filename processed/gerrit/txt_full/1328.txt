--- Message (Andreas Dilger, 2011-11-22 21:19:14.000000000) ---
Patch Set 6: Looks good to me, but someone else must approve

The update to the error message looks good.  Looking at the rest of this function, it seems that something is not quite right.

The question I have, based on your comment in LU-651 "adjacent pages disagreeing with OBD_BRW_NOQUOTA flag can not be merged", is whether these pages should be in the same BRW request at all?  It seems that can_merge_pages() is only checking whether the pages can go into the same niobuf.  On the server side the iobuf->dr_ignore_quota flag is for the whole IO request, so if any niobuf sent from the client has this flag set it will mark the whole IO as ignore_quota.  If the IO happens to be fragmented, then it may still be that some requests do not get NOQUOTA set, but it is not always correct.

If Johann is OK with this case, which amounts to one more way to overflow the quota, then it is OK with me.
