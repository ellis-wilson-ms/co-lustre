LU-12214 build: spec file cleanup

fix using a %{name} macro in spec file.

Provide an build requirements to make build happy with gss enabled.

lack of MPI library dependence will lost a mpi tests
from build

add a MOFED / OPA modules depences to produce a right
package.

add kabi package dependence and define to use it, to avoid warning
during build.

add ability to use an 'epoch' in kernel version
(to avoid confuse a kmodtool)

don't replace a requested kernel version with
kernel source provided to avoid epoch lost.

make configure --enable-dist to be happy without libyaml installed.

Remove concatenating of context for the temporary mount point
if selinux is enabled.
mount.zfs don't have that option, so revert it for consistency.
It can be added with -o option if needed.

fix build without lustre_utils. build failed with llog_test access.

spec file don't need to add a library as dependence, it covered
by autorequrement script.

add ability to specify an build number for the lustre package.
It needs to avoid package manger confussion in case OFED change
without any lustre update.
