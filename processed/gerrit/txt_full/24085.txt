LU-7734 lnet: cpt locking

When source nid is specified it is necessary to also
use the destination nid. Otherwise bulk transfer will end up
on a different interface than the nearest interface to the
memory. This has significant performance impact on NUMA
systems such as the SGI UV.

The CPT which the MD describing the bulk buffers belongs to
is not the same CPT of the actual pages of memory.
Therefore, it is necessary to communicate the CPT of the pages
to LNet, in order for LNet to select the nearest interface.

The MD which describes the pages of memory gets attached to
an ME, to be matched later on. The MD which describes the
message to be sent is different and this patch adds the
handle of the bulk MD into the MD which ends up being
accessible by lnet_select_pathway(). In that function
a new API, lnet_cpt_of_md_page(), is called which returns the
CPT of the buffers used for the bulk transfer.
lnet_select_pathway() proceeds to use this CPT to select
the nearest interface.

(7 comments)

I'm thinking it is rather unfortunate that typical code defining an MD initializes it by zeroing it, but correctly initializing the bulk_md_handle field requires an additional function call. This complicates the places in the code that use non-bulk MDs: they'd now have to be updated, and it is something all future users need to remember.

So I'm wondering if we should look at the way the max_size field is handled. There the field is only valid if LNET_MD_MAX_SIZE is set in the MD options. This allows for the default of zeroing the struct to have the expected result.

The corresponding change in this code would be to have an LNET_MD_BULK_HANDLE MD option flag which indicates that the lnet_md_t::bulk_md_handle / lnet_libmd_t::md_bulk_handle field is valid. That way no extra needs to be affected.

I'd then also be tempted to change the interface of lnet_cpt_of_md_page() once more, so that it doesn't take a handle, but rather a pointer to the lnet_libmd_t *. In lnet_select_pathway() we'd provide msg->msg_md to lnet_cpt_of_md(). In lnet_cpt_of_md() we can then check whether the MD has a valid bulk desciptor, and if so, base the CPT returned on that. If not, we can base the CPT on the MD's own buffer.

I've sketched the changes required below. What do you think?
