I've only briefly looked at this, and so far the patch looks OK, but I'd rather not completely disable all of the tracking.

Rather, I think we can do better and keep all of the tracking, but simply enable or disable the "limit" within lustre.

That would allow "reasonable" performance for the workload using few OSTs, but would still allow the statistics to be collected (which can be useful information regardless of throttling), and the kernel can still do it's throttling using "NFS_Unstable" (which will help prevent OOMs).

Does that sound like a better idea until we properly address the root of the issue?
