--- Commit Message ---
LU-17965 pcc: reopen PCC copy for fallback file handle

When a file is being copied from Lustre OSTs to PCC copy, the file
is in PCC ATTACHING state. During this phase, new opens and I/O on
the file will fallback to Lustre OSTs.

In the old implementation, even a client finished PCC attach on
the file, the new I/O for the fallback file handle is still
issuing to Lustre OSTs using the normal I/O path.

In this patch, we reopen the PCC copy for valid PCC cache and
direct I/O to PCC copy instead.

Add the test case sanity-pcc/test_110.
