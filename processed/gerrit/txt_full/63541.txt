LU-19791 quota: fix silent data loss for root project

When root_prj_enable is set, it is possible that the pages stored in
the cache after buffered IO could be dropped if root is out of quota
on a different client. Fix that by forcing to write all lnb coming
with OBD_BRW_ASYNC flag.

## Fix Explanation

**The Problem:**
The patch introduced a syntax error in `lustre/osd-zfs/osd_quota.c` at line 582-583. The `if` statement was missing the logical AND operator (`&&`) between two conditions:

```c
if (tmp_flags & QUOTA_FL_ROOT_PRJQUOTA
    !(osd_qid_declare_flags & OSD_QID_ASYNC)) {
```

**The Fix:**
Added the missing `&&` operator to properly combine the two boolean conditions:

```c
if (tmp_flags & QUOTA_FL_ROOT_PRJQUOTA &&
    !(osd_qid_declare_flags & OSD_QID_ASYNC)) {
```

**Rationale:**
The condition needs to check two things:
1. Whether `tmp_flags` has the `QUOTA_FL_ROOT_PRJQUOTA` flag set
2. Whether `osd_qid_declare_flags` does NOT have the `OSD_QID_ASYNC` flag set

Without the `&&` operator, the C compiler sees `QUOTA_FL_ROOT_PRJQUOTA` followed by `!(...)` which is invalid syntax - it expects either a closing parenthesis or a logical operator. This matches the pattern used in the ldiskfs version of the same code in `lustre/osd-ldiskfs/osd_quota.c` where the `&&` operator is correctly present.
