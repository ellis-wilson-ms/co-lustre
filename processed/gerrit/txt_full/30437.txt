--- Message (Andreas Dilger, 2017-12-07 21:46:33.000000000) ---
Patch Set 2:

(7 comments)

I think the main improvement to be done here is not to expand the macros that are doing checks (which are better than having the same boilerplate code repeated in every function), but rather to get rid of the whole obd_*() wrapper functions completely.

The main confusion that reviewers have is that e.g. llite calls obd_get_info(), but they don't know that it goes to lov_get_info() or lmv_get_info() depending on which export is used. I think it would be better to keep the EXP_CHECK_MD_OP() and friends macros, but remove these obd_*() functions and OBP() and have llite call lov_get_info() or lmv_get_info() directly as needed.

That does have the drawback that we will need an EXPORT_SYMBOL() for every method. Potentially, if we named the "ops" pointers correctly at the callsite, like "lov_obd_ops->o_get_info" to match "static struct obd_ops lov_obd_ops" it would be more clear what is happening.

That will make the high-level code flow more clear without introducing 30 lines of boilerplate checks before every call, which I think does not add to understanding code flow.
