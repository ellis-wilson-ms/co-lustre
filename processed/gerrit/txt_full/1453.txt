LU-687 clio: retry if fault page was truncated

In vvp_io_fault_start, if a page was truncated we should retry it in
ll_fault() instead of return -EFAULT because it will cause fake OOM.

Patch Set 3:

to niu: for nopage case, it will return a truncated page and kernel is supposed to catch this case when it locks the page and installs pte.

If we revise vvp_io_kernel_fault() to return a locked page, we will have trouble to unlock page after cl_page_find(), that's to say, for .fault case, we need to clear VM_FAULT_LOCKED bit and one extra '#ifndef HAVE_VM_OP_FAULT' will be needed.
