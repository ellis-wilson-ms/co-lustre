LU-13670 tests: fix sanity-quota/6+7 issues

Fix bug in test_7e where it incorrectly used the MDT UUID
instead of the MDT name when calling test-framework.sh::getquota(),
but the syntax errors produced would not cause the script to fail.

Fix interop check in test_7f, it used 2.16.5x instead of 2.17.5x.

Fix bash code style in test_6 and test_7:
- use lower-case variable names for local variables
- use ((...)) for numeric comparisons
- use [[...]] for string comparisons
- use "check || action" style to avoid lingering error return
