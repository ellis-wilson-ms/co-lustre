LU-12610 obd: delete OBD_ -> CFS_ redefinitions

With all consumers of the OBD macros fixed, finally
remove OBD macros that are simply redefinitions of CFS
macros.

Remove:
OBD_FAIL_PRECHECK(id)
OBD_FAIL_CHECK(id)
OBD_FAIL_CHECK_VALUE(id, value)
OBD_FAIL_CHECK_ORSET(id, value)
OBD_FAIL_CHECK_RESET(id, value)
OBD_FAIL_RETURN(id, ret)
OBD_FAIL_TIMEOUT(id, secs)
OBD_FAIL_TIMEOUT_MS(id, ms)
OBD_FAIL_TIMEOUT_ORSET(id, value, secs)
OBD_RACE(id)
OBD_FAIL_ONCE
OBD_FAILED

Avoid losing the unlikely optimization with OBD_FAIL_PRECHECK by
adding unlikely to CFS_FAIL_PRECHECK. For libcfs_private.h not
all callers of CFS_FAIL_PRECHECK had unlikely so this is fixed
as well.

Patch Set 3: Code-Review-1

A few things to note about this patch:
- it is huge and touches a lot of files == high chance of conflict.  It would probably be better to split it into a few patches to allow landing them independently.
2) it does not include an update to contrib/scripts/spelling.txt marking the OBD_FAIL_* macros as obsolete and suggesting CFS_FAIL_* replacements.
3) it deprecates and removes the OBD_FAIL_* macros in the same patch, which will break all patches in flight using the OBD_FAIL_* names.

These macros are used often enough that we don't want to break all of the unlanded patches in one shot.  It would be better to add the deprecation (via spelling.txt) in this patch, and then split out the removal of these interfaces (basically the changes in obd_support.h) into a follow-on patch.  The second patch can wait a month or two before landing, by which point all of the pending patches using this interface should have landed and/or gotten a warning about OBD_FAIL_* and updated to use CFS_FAIL_* already.
