LU-19258 tcu: add support for file deletion via rename()

Add support on TCU to handle the case where a file is deleted via
rename() like:
write new $FILENAME.tmp;
rename|mv $FILENAME.tmp $FILENAME;

This is a common pattern for tools like editors to avoid
corrupting $FILENAME when a new version is written, in case the
process crashes in the middle of writing out the new data. This
will move the old $FILENAME into Trash Can, with a version of
appending timestamp if this happens repeatedly.

Add a test case sanityn/test_118C.
