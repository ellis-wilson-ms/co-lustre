--- Message (Andreas Dilger, 2017-11-09 23:19:54.000000000) ---
Patch Set 1: Code-Review+1

This looks like it is done correctly to me - stop checking for the presence of IBITS flags in the protocol (assume all clients support and use this feature), but continue sending it from the client and accepting it at the server to maintain interoperability with old client/server combinations.

It might be useful to add an annotation like:

 #define OBD_CONNECT_IBITS               0x1000ULL /* not checked in 2.11+ */

so that it is easier to see that it is no longer being actively used.  It needs to stay in MDS_CONNECT_SUPPORTED for the foreseeable future.
