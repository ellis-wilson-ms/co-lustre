LU-8330 wiretest: Add IOCTL checking to wiretest

Add IOCTL checking (to ensure they don't change) to wiretest, but
don't add to ptlrpc ASSERTS.  Doing this, showed that the
OBD_IOC_GET_MNTOPT doesn't work in user-space as mntopt_t isn't
defined in user-space and throws an error if the appropriate header is
included.  This type is an enum, so should be the same as a long.

Patch Set 1: Code-Review-1

(15 comments)

Nathaniel, thanks for the patch.  As always when digging in crufty old corners of the code there are things that should be cleaned up as well.

I suspect that there is more than the one OBD_IOC_GET_MNTOPT ioctl that is either completely unused, or is not used directly from userspace.  If that is the case, we should look into removing or changing these ioctls rather than carving their values in stone.

There is also the separate problem that OBD_IOC_DATA_TYPE is used in the definition of many ioctls, even though it is completely useless and should be replaced with a proper structure argument.  It would be trivial to define duplicate ioctl constants for each of the ones using OBD_IOC_DATA_TYPE with the proper data structures, and add the new constants to the case statements where the ioctls are handled.  Then, in a few releases we can start using the new constants instead of the old ones, and finally in a few years delete the old constants.

Also, OBD_IOC_GETNAME->OBD_IOC_GETDTNAME needs to be replaced in llapi_file_fget_lov_uuid(), and probably many other cleanups that could be done (see https://bugzilla.lustre.org/show_bug.cgi?id=20731 for more details).  Definitely for separate patches, but something to keep in mind for next steps.
