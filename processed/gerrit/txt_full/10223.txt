LU-4345 osp: store valid bits in setattr record

We'd store LA_UID/LA_GID bit along with the UID/GID in the setattr
llog record, otherwise, osp could set a random uid/gid to the OST
object.

Patch Set 1: Code-Review-1

(1 comment)

It seems that mdt_{create,link,open}_unpack() is setting both LA_UID and LA_GID, so objects should always have both la_uid and la_gid set.  Where are we getting RPCs that are missing one of these flags?  If this is coming from the client during write requests, then the client needs to be fixed to set both of these fields so that quota works correctly.

I don't mind to fix the MDS to fix up RPCs from old/broken clients, but that should clearly be marked as a workaround (maybe inside LUSTRE_VERSION_CODE checks) and the root of the problem should be fixed.

It would also be good to get a test case for this.
