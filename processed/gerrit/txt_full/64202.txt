--- Commit Message ---
LU-19916 mgs: start MGS replicas on MDTs/OSTs

Start MGS replicas on MDTs/OSTs which are not located on the MGS0
primary and failover hosts.  The MGS replica is a readonly backup
of MGS0, any configuration change should be done on MGS0.

Once a MGS replica is stopped, wake up ll_cfg_requeue thread to start
MGS replica on another target if possible.

Register MGS replica as observer of MGC, so that it can be notified
when the config is changed.

Add an argument for obd_notify() to pass data.

Disable ioctl on MGS replica.

Add debugfs entry mgs/MGS/replica as indicator of  MGS replica, and
is_mgs() in liblustreapi should check this because MGS replica is
readonly, and doesn't handle ioctl.

If MGC import connection count is more than 2, it means the MGS nid
list are MGS replicas, shuffle connections so that clients are
distributed among all MGS replicas evenly.  Caveat: this doesn't
distribute load if there are 2 MGS replicas, because it can't tell
these 2 MGS's are replicas, or are primary and failover of MGS0, but
after some time (clients reconnect happened), clients will be
distributed evenly.
