LU-20183 hsm: handle short-read with KUC

It is possible for some coordinator workload to write
KUC messages larger than a page (ie: large 'data' buffer).
libcfs_ukuc_msg_get() is calling read() and is expecting this read
to get a full kuc message in 1 call. However, when message
is larger than a page, and the system is under load, it is possible
that the read() will return the 1st page first, and a second call
is needed to get the full page. This is causing message drop.

This patch is fixing this short-read ensuring partial reads
and other typical read errors are handled.

If a Changelog reader is registered with O_NONBLOCK, it will
still receive -EAGAIN if the pipe is empty while trying a read
a new message. However, if the pipe does not have enough bytes
to read a full message, while some bytes have already been read,
it will block until the full message is read. This keep
the main use case for O_NONBLOCK, while fixing the bug.
