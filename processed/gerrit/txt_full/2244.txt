LU-1160 tests: remove liblustre from default tests

The liblustre testing is currently failing. Since this functionality
is not being used by any (or at least not many?) users since it is
no longer used for the Cray Catamount compute clients, the liblustre
testing should be removed from automated test runs.

Until the test can be fixed this test can be re-enabled. As yet,
the liblustre code will not be removed from the tree.

I still use acc-sm for running my own tests, I'm not sure if others do as well.  It is also documented in many external locations as the default test suite to run for patches.

Running "auster" doesn't really provide any list of tests to run, nor is there a so I don't consider it a useful test script for most developers.  I would have no objection to acc-sm becoming a wrapper around "auster" that just supplied a list of tests for it to run, but that should be done in a separate patch, IMHO.

In any case, it is misleading for external developers (e.g. EMC) to include liblustre in acc-sm if it does not pass, and if there are only a few people using acc-sm today then there is also little harm in landing the patch.
