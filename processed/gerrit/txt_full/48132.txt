--- Commit Message ---
LU-16035 kfilnd: Fix transaction double free

On receiving an eager message, within kfilnd_tn_state_idle(),
lnet_parse() is called. If matched, lnet_parse() will call back into
kfilnd_recv() which may try to issue an RMA command. If the RMA
command fails, the transaction will be freed. Then as the stack is
unwound back to kfilnd_tn_state_idle(), the transaction is incorrectly
accessed and freed again. To resolve this issue, do not access the
transaction again within kfilnd_tn_state_idle() if lnet_parse() was
successfully.
