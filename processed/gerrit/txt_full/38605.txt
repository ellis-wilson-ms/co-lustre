--- Commit Message ---
LU-12475 libcfs: don't register crypto if it can cause a panic

When ther kernel is running with FIPS enabled (fips=1) the system
will panic if an unsigned module attempts to register a crypto
algorithm.  In Lustre this happens with the adler32 checksum is
registered in libcfs.  To prevent this from happening we will now
check these conditions and load without adler32 support.
