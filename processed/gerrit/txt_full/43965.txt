--- Commit Message ---
LU-11872 ldiskfs: do not inherit project id for links

There is a limitation that symlink inodes may have a project id,
but the project id attribute cannot be changed after symlink creation.
At creating time, the symlink inherits project id from parent dir,
if corresponded inode flag and attribute is set
(i.e. project inheritance flag and project id).
This leads to problem and wrong files quota accounting if file tree
is transferred from one project id to another one.
In this ticket I am attempting to disable project id inheritance
for symlinks.
