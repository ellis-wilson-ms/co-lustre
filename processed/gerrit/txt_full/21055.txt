The zfs_set_prop_str function returns a non-zero if the property wasn't set. zfs does not store property settings if they aren't supported. Setting those properties are best effort at file system creation time. If the property couldn't be set, then no harm is done. I also chose to use the zfs_set_prop_str approach because it wouldn't rely on any compile time definitions.
