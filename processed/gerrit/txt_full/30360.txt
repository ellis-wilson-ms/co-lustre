--- Commit Message ---
LU-10282 flr: comp-flags support when creating mirrors

This patch will allow flags to be set when creating mirrors.
The flags are set to individual components therefore it would be
flexible to flags based on the location of components. Also, 'stale'
and 'prefer' flags are allowed to set to individual components later
on.

This patch also revises component flags matching rules to allow
flags and inverted flags to be set at the same time in the command
lfs-find(1) and lfs-getstripe(1).

> "lfs mirror verify" reports that the mirror is not outofdate 

"lfs mirror verify" only checks valid components. The main purpose is to make sure the valid components actually tell the truth.

In order to clear 'stale' flag, we need to take lease lock and verify the contents of component are valid, which is pretty much what 'lfs mirror resync' is doing.

Setting 'stale' flag has side-effect too. If a component has been picked as primary, and users want to set stale flag to the component, the I/O afterward will fail.
