LU-17768 build: embed BTF in Lustre kernel modules

Out-of-tree Lustre kernel modules lack BTF sections because
kbuild skips BTF generation when vmlinux is not present in
the kernel build directory (kernel-devel does not include it).

Add a btf-prepare target to autoMakefile.am that runs before
the modules target.  It symlinks /sys/kernel/btf/vmlinux (BTF
of the running kernel) into the kernel build directory so
kbuild's Makefile.modfinal invokes pahole with --btf_base
and emits .BTF sections in the out-of-tree modules.  pahole
accepts a raw BTF file for --btf_base.  Placing the logic in
the Makefile covers `make`, `make rpms`, and `make debs`.

The symlink is gated on pahole being installed and
scripts/Makefile.btf existing.  Without either, BTF cannot
or must not be emitted (RHEL 8's modpost misparses the BTF
file as ELF).

The step skips silently without write access, so non-root
`make` succeeds without BTF.

Add --keep-section '.BTF' to rpm's debuginfo extractor so
.BTF survives in the shipped RPMs.  dwarves is not a
BuildRequires; builds succeed without it, just without BTF.
