Patch Set 1: Code-Review-1

(4 comments)

Please fix style issues and add a test.

That said, there should also be a default value set for obd_recovery_time_limit.  The question is what value should be used?

We could choose a long timeout like 3600s, which is long, but will be hit eventually.  It would be better if we could have a much shorter timeout (e.g. at_max) and just handle "late" MDT recovery better.  

That isn't something for this patch, since it will need a lot of testing, but being able to recover with a missing MDT can be useful for a number of reasons.
