LU-7245 socklnd: Bind peers to a specific CPT

Currently the socklnd driver doesn't support
CPT affinity for its peers. Binding peers to
a specific CPT and memory allocated to the
NUMA node belonging to the CPT should give a
performance boost.

(1 comment)

The code that places the peer datastructures on a specific CPT has
the issue that you ought to pick the CPT (and preferably the node,
when CPT != node) after you've picked the preferred interface to use
to talk to that peer, and know which node that interface lives in.
But that's another discussion.

What I do see this code doing is follow the same placement rules
that LNet uses for the socklnd-specific datastructures, which is
at least an improvement over the status quo.
