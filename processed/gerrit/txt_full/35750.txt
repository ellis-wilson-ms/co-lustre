LU-12652 ldiskfs: disable credits check in 4.18

4.18 has a credits check in xattr_set path with different math
resulting in false ENOSPC and subsequent failure to update LMA.
as a result - many tests fail on 4.18, especially with DNE.

Patch Set 7: Code-Review-1

I'm not a fan of patching ext4 in a way that cannot be pushed upstream, since it is an ongoing patch burden that we should avoid if possible.

I understand that we don't want to over-inflate the handle credits since Lustre stores multiple xattrs on each inode at create time, and each call to __ext4_xattr_set_credits() will double/triple account many of the blocks.

Things that I think that could go in upstream __ext4_xattr_set_credits():
- check ext4_test_inode_state(inode, EXT4_STATE_NEW) to skip credits for modifying existing xattrs, which reduces base credits from 7 to 4
- do not need to account indirect/index blocks for xattrs <= one block, which reduces extra credits from 6 to 3
- do not need to double count xattr data/bitmap blocks for ea_inode and the base 7 blocks, which reduces credits by 2
- __ext4_xattr_set_credits() does not need to modify an EA inode for new inodes, which removes another 38 credits

- osd_attr_set() should use XATTR_CREATE for xattrs stored at inode create time
