--- Commit Message ---
LU-5274 llite: fix ll_statahead_thread() problems on failure

When ll_prep_md_op_data() fails, ll_statahead_thread() does not
release reference of sai and parent dentry. It does not wake
up parent thread either. This patch fixes these problems.

Lai, It seems that by clearing lli_opendir_key we are preventing ll_stop_statahead() from dropping the last reference to sai. But it's impossible to really say because nowhere is it stated which references are counted. Does lli_sai count? If so why doesn't ll_stop_statahead() set lli_sai to NULL after calling ll_sai_put(lli->lli_sai)?

There are comments like: "Put the ref which was held when first statahead_enter. It maybe not the last ref for some statahead requests maybe inflight."

References should be clearly associated with pointers and not with control flow.
