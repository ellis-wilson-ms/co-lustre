--- Message (Andreas Dilger, 2020-05-27 18:01:20.000000000) ---
Patch Set 1:

So, there are two objections that I had with this patch:
- the LUSTRE_REVISION is hard-coding the /sys/ path, which is what "lctl get_param version" is reading in the background, *if it exists*, but the latter is much more portable across versions
- LUSTRE_REVISION will be exactly the same as LUSTRE_VERSION, which is itself using "lctl get_param version" in the background, but again in an inferior manner, so I question why LUSTRE_REVISION exists at all?

A better solution would be to put something more useful into LUSTRE_REVISION, like "git describe" or similar, but that would not be possible for installed packages, and the YAML output is unlikely to be generated from a developer build tree?
