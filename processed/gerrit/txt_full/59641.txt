LU-13643 llite: add mirror detection for concurrent write

This patch implements the foundation for concurrent write
functionality on immediate mirrored files by adding mirror
detection capabilities to the client.

Key changes:
- Add ll_get_mirror_info() function to detect mirrored files
  with immediate flag using cl_object_layout_get()
- Integrate mirror detection into ll_file_io_generic() for
  write operations
- Add server-side FLR state transition support in mdt_open.c
  for ACTIVE_WRITER intent handling
- Files correctly transition to WRITE_PENDING state on write
  operations to immediate mirrored files

The implementation uses a safe approach with proper buffer
allocation and error handling to avoid kernel crashes.
Server-side integration ensures FLR state transitions work
correctly with the existing mirroring infrastructure.

This establishes the foundation for concurrent write locks
that will allow multiple writers to immediate mirrored files
while maintaining data consistency through automatic mirror
staling mechanisms.
