--- Commit Message ---
LU-17922 utils: added idmap range functionality

Added the ability to a declare a range when adding idmaps to a
nodemap. The syntax is:
 <clientid_start>-<clientid_end>:<fsid_start>[-<fsid_end>]

The uid_end value is optional. In practice this looks like:

nodemap_add_idmap --name test --idtype uid --idmap 500-510:10000

It is also now possible to delete idmap ranges with the
nodemap_del_idmap command as well with the same syntax.

I appreciate this is a first version of the patch, so maybe this is not completely ready for review yet, and there is still some room for improvement.

Apart from the way the id range is handled between nodemap_parse_idmap() and nodemap_add_idmap_range()/nodemap_del_idmap_range(), I think the challenge is to avoid storing individually each idmap generated from a range. Otherwise adding a range of 1000 ids would lead to having 1000 entries stored on disk. It would be great if we could avoid that, see the discussion in LU-17922 for more information.
