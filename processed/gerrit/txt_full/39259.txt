LU-13783 libcfs: support removal of kernel_setsockopt()

Linux 5.8 removes kernel_setsockopt() and kernel_getsockopt(), and
provides some helper functions for some accesses that are
not trivial.

This patch adds those helpers to libcfs when they are not available,
and changes (nearly) all calls to kernel_[gs]etsockopt() to
either use direct access to a helper call.

->keepalive() is not available before v4.11-rc1~94^2~43^2~14
and there is no helper function, so for SO_KEEPALIVE we
need to have #ifdef code in the C file.

TCP_BACKOFF* setting are not converted as they are not available in
any upstream kernel, so no conversion is possible.

Also include some minor style fixes and change lnet_sock_setbuf() and
lnet_sock_getbuf() to be 'void' functions.

> Amir or Chris do we really need this functionality anymore?

James, by "this", what do you mean?  Setting all of the tunables, or a specific one?

My answer would be "yes", because we invested significant efforts to tune TCP performance before IB became popular, and I'd rather leave these setting as-is unless there is a comparably significant effort to benchmark and tune the performance with these setting removed or changed, not just because there would be fewer lines of code without them.  

Note that socklnd establishes three separate connections for performance reasons, because there are different types of network traffic, and each connection has quite different needs for the TCP stack. We don't want to congest one socket with large messages and then try to send small control messages, or read vs. write. Enabling or disabling Nagle should be different for small messages vs. large streaming read/write operations, window sizes, etc. I don't know the details of all of the settings, but I know enough to know that I *don't* know enough to change them. 

I've recently also discussed with Amir about establishzing multiple sockets per interface ("conns_per_peer=N", like ko2iblnd) in order to be able to drive 100GigE faster over WAN.  We'll need to make sure those extra connections get set up with the right parameters for the type of messages that will be sent over them.
