LU-14110 obdclass: Protect keys_fill instances

keys_fill could be call by different mount instances on the same
client for the same lu_context. This result to a race: we can
alcocate context memory several time for the same context and
increment each time "lu_context_key.lct_used".

When "rmmod lustre" is called, "lct_fini" functions will free the
memory at the address contains in "lct_value" and not the additional
memory allocate during the race, that's why lct_used will remain > 0
and "lu_context_key_degister" will hang.

This patch prevents several "keys_fill" instances to run at the same
time.
