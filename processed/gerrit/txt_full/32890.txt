--- Commit Message ---
LU-11185 mgc: config lock leak

Regression introduced by "LU-580: update mgc llog process code".
It takes additional cld reference to the lock, but lock cancel forget
during normal shutdown. So this lock holds cld on the list for a long
time. any config modification needs to cancel each lock separately.

Cray-bugid: LUS-6253
Fixes: 5538eee216a1 ("LU-580: update mgc llog process code")

Ok, your feedback was useful.
I actually found out why I was not reproducing.

To trigger the bug, the client should not be the only mount point on the system.
When unmounting this only client, the MGS locks are properly cleaned.
However, if you have other local mount or other MGC reference, some locks will be leaked and the bug appears.
