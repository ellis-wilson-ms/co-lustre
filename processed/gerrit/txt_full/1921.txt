--- Commit Message ---
LU-907 nfs: open only fetches OPEN lock

Open only fetches OPEN lock, because open(O_READONLY) will fetch
lock with mode LCK_CR, which doesn't make sense for LOOKUP or
UPDATE lock; and this can avoid open conflict with setattr/getattr.
In this way open lock cache is really effective for nfs export.

Patch Set 4:

I am not sure I understand the logic behind this change at all.

If we return just open lock from open, this open lock can never be reused unless there is already a lookup lock on the client somehow. But there are a lot of cases where there is no way to get this lookup lock. E.g. when there are just opens happening, the lookup lock won't appear at all.
Note that CR and PR lock modes don't conflict.

Can you explain this some more?

Note that setattr conflicting is important, because if the lock is for EXEC and you do setattr (truncate), you must flush the open first or the setattr will fail with file busy.
