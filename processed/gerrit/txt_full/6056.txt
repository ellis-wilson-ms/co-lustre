LU-3173 build: Incorrect detection of ldiskfs series to use

Extract correct upstream kernel release for RHEL kernels and
form it into RHEL_KERNEL_VERSION as
$(VERSION).$(PATCHLEVEL).$(SUBLEVEL)-$(RHEL_RELEASE)
Then use this string for correct ldiskfs series selection.

--- Message (Dmitry Eremin, 2013-04-17 20:36:08.000000000) ---
Patch Set 1:

So far we also use <linux/version.h> file for RedHat detection:

# check is redhat/suse kernels
AC_MSG_CHECKING([that RedHat kernel])
LB_LINUX_TRY_COMPILE([
                #include <linux/version.h>
        ],[
                #ifndef RHEL_RELEASE_CODE
                #error "not redhat kernel"
                #endif
        ],[
                RHEL_KERNEL="yes"
                AC_MSG_RESULT([yes])
        ],[
                AC_MSG_RESULT([no])
])

Fedora Core it's not exactly RedHat distributive. All checks will fail for it. I didn't see any RHEL kernels with new v3.8 kernel source structure. We will required to change the main check first because RHEL_RELEASE_CODE looked only if RHEL_KERNEL="yes".
