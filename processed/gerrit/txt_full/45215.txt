--- Message (Serguei Smirnov, 2021-10-25 19:46:30.000000000) ---
Patch Set 1:

In kiblnd_net_init_pools lnd_map_on_demand is used to determine whether to use FMR/FastReg or use global MR. If negotiation failed with IBLND_REJECT_RDMA_FRAGS, it can be because the number of fragments didn't match and FMR/FastReg is disabled. Why not notify that map_on_demand should be enabled in this case if it is disabled?

--- Message (Chris Horn, 2021-10-25 20:43:54.000000000) ---
Patch Set 1:

That's a fair point, but we can't actually detect that situation can we? It _may_ be the case that we need to enable map_on_demand, or it may be the case that we can already negotiate max frags.
