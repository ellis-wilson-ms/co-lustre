## Summary

**Fix Applied:**

I removed the duplicate definition of `yaml_get_limit_uid()` from `lustre/utils/lustre_cfg.c` (lines 624-678).

**Rationale:**

1. **Root Cause**: The compile error was a "multiple definition" linker error. The function `yaml_get_limit_uid()` was defined in two places:
   - `lustre/utils/liblustreapi_param.c` (line 1490) - part of `liblustreapi.a` library
   - `lustre/utils/lustre_cfg.c` (line 624) - compiled into `lctl` binary

2. **Why the duplicate existed**: Before the patch, `lustre/utils/obd.c` had a local function `lcfg_get_nm_offset_limit()` that called `yaml_get_limit_uid()`. The patch moved this functionality to `liblustreapi_nodemap.c` to use the library version.

3. **Why removing from lustre_cfg.c is correct**:
   - The function is already properly defined in `liblustreapi_param.c` as part of the library
   - It's declared in `lustreapi_internal.h` for internal library use
   - The `lustre_cfg.c` file doesn't actually call this function anywhere
   - The new code in `liblustreapi_nodemap.c` (added by the patch) uses the library version
   - When linking `lctl`, it links against `liblustreapi.a`, which already contains the symbol

4. **Result**: By removing the duplicate definition from `lustre_cfg.c`, the linker will only find one definition (from `liblustreapi.a`), resolving the multiple definition error.
