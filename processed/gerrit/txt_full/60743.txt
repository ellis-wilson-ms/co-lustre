--- Commit Message ---
LU-9680 utils: make cYAML support more types of booleans

Currently cYAML only supports all lower case "true" and "false"
but by default libyaml returns similar values with the first
letter capitalized. This can cause yaml config files created
with the latest LNet stack to fail on older lustre versions.
To resolve this add support for the default libyaml boolean
to cYAML handling.
