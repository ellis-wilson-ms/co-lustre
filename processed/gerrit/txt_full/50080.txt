LU-16567 ldlm: Add asserts about l_ast_data

There is only ever one OSC object at a time for each LDLM
lock.  OSC objects can go away, but when they do, the
l_ast_data in the dlmlock is cleared.

So for EXT locks, the l_ast_data either always matches the
current OSC object (since there cannot be more than one) or
it is NULL.

Assert this.

This is a work in progress - the osc_lock_set_data code
assumes this *isn't* true, but I'm pretty sure that code is
wrong.  Once I am sure about the assertion passing, I will
adjust that code rather than leave it to mislead.

NB: I am also suspicious of a bug with l_ast_data handling
and OSC objects, so I'm hoping this assertion may help
catch it.  Let's find out.
