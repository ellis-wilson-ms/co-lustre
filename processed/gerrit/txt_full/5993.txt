--- Commit Message ---
LU-2584 data: T10 DIF/DIX integrity support

This patch implements Lustre end-to-end integrity
support with T10 DIF/DIX:
1) on write, CRCs are calculated on the client,
transferred over wire, stored on disk;
2) on read, CRCs are read from disk,
transferred over wire, checked on the client.

With protocol A (inpill), CRCs are packed in RPC.
With protocol B (inbulk), CRCs are packed in bulk.

--- Message (unknown, 2013-04-11 13:29:57.000000000) ---
Patch Set 1:

Thank you for the review.

This patch does not cover checksumming itself, instead it uses the kernel implementation of crc_t10dif(). There is a separate patch (T10-CRC) uploaded to WC gerrit by Alexander Boyko which replaces crc_t10dif() with a faster implementation.

This patch deals with packing/unpacking/attaching data to page cache pages/attaching data to bios. The code is split between different layers, because it heavily depends on structures from these layers (brw pills/bulks, page cache storage in vvp and osd) even more than ost_checksum_* / osc_checksum_bulk, which themselves do not belong to the checksumming code.

It is possible to have different chunk sizes for different OSTs, because each page cache page belongs only to one OST and is processed on per-OSC basis. OST MDRAID on top of devices with different chunk sizes will not report itself as DIX-capable.

I will update the patch to include the comments you requested and to fix the style issues.
