--- Commit Message ---
LU-16207 build: add rpm-build BuildRequires for SLES15 SP3

SLES15 SP3 fails to build using rpm-build-4.14.1-29.46
from the main O/S repository with error message:

- Dependency tokens must begin with alpha-numeric,
  '_' or '/': BuildRequires: %kernel_module_package_buildreqs

Updating rpm-build to 4.14.3-150300.46.1 or higher
resolved the build issue.

Probably.  I changes a cryptic error message to a meaningful error telling
you that you need a new version of rpm-build.
The initial release of SLE-15-SP3 had rpm 4.14.1.  Presumably that has problems.
The latest update has 4.14.3 which apparently fixes the problem.

It isn't clear to me why this BuildRequires has to be specific to suse 15.3.
Wouldn't earlier version of rpm have the same problem on all platforms?
Maybe this is a suse-specific bug - I cannot find any evidence for that though.
