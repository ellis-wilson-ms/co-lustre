LU-13903 utils: separate out server code for wiretest

Both the kernel and userland utility wiretest is used by both
client and server to validate data being sent over the network.
Make userland  wiretest buildable on the native Linux client
which lacks server specific data structures. Use of the UAPI
values to hardern testing of user land data passed to the
kernel.

Patch Set 5: Code-Review-1

(6 comments)

I'm ok to exclude structures and constants that are totally unused by clients, but I think it is problematic to exclude individual values of a class of constants (eg. OBD_CONNECT_*) just because they happen not to be in use by the kernel client code. That should be addressed by adding that constant to the kernel client headers, rather than removing it from the checks here. 

The problem is that not having the constant defined in the kernel headers incorrectly gives the impression that value is not in use. If/when development shifts over to the kernel client, this could lead to mistakes being made by re-using the "unused" values, causing potential havoc in the protocol.  It is better to define all of the values in the header to avoid this.
