--- Message (Andreas Dilger, 2011-09-07 16:40:13.000000000) ---
Patch Set 8:

Rather than resubmitting the whole patch series to trigger a new build, it is possible to just restart the build.  

First follow the link to the broken build, if not already logged in then login to Jenkins, then select "retrigger" on the menu at the left to restart the build.

However, if there is a problem with the build system, blindly restarting the builds will not actually fix anything and will instead waste a lot of time rebuilding patches that will just fail again.

Having duplicate patches in Gerrit is bad for a number of reasons, such as causing old inspections to be invalidated, causing autotest to run multiple sets of tests unnecessarily.  Often the broken builds are automatically restarted once the problem that caused the failure is fixed, and the load on the test system has been needlessly increased by having several sets of identical patches.
