If I understand the code correctly, lcm->lcm_flags can be initialized from lo->ldo_flr_state

3    976  lustre/lod/lod_lov.c <<lod_generate_lovea>>
             lcm->lcm_flags = cpu_to_le16(lo->ldo_flr_state);

So I expect non-FLR layout flags should be preserved in ldo_flr_state too. I see a lot of places there "lo->ldo_flr_state = LCM_FL_NONE;" is set rather than clear FLR mask bits. In these functions:

lod_parse_striping()
lod_declare_layout_add()
lod_declare_layout_del()
lod_declare_update_extents()
lod_declare_update_plain()
lod_declare_pccro_set()
lod_declare_layout_change()

Also, I see in mdd_split_ea()
 comp_vic->lcm_flags = cpu_to_le16(LCM_FL_NONE);
