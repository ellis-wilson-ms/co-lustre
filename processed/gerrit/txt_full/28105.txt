--- Message (Andreas Dilger, 2019-10-05 03:01:53.000000000) ---
Abandoned

> Andriy - I agree that recovery-small test 101 needs to restore this
 > parameter, but I'd rather the test itself clean up after itself on
 > error. One way to do this would be to set a trap that restores the
 > parameter on error.

Using stack_trap() is the best way to handle conditions like this.  We don't want a remote shell command at the end of every test to fix up a parameter that is only changed by one or a few tests.  That would quickly lead to a lot of needless complexity and slowdown in the cleanup, if it were done this way for every test.

In any case, it looks like the stack_trap was added as part of patch https://review.whamcloud.com/33557 "LU-11601 ptlrpc: IR doesn't reconnect after EAGAIN", so this patch is no longer needed.
