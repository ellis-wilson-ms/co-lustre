Patch Set 1: Code-Review-1

(1 comment)

The main objection is that the extra wrapper function could just be avoided.

Separately (not a reason to reject this patch, but a recommendation) would be to refresh the patches I just linked into LU-12480 and submit them to the upstream kernel (but don't mention Lustre for political reasons).  The main benefit of adding projid to statx() is that it would allow getting the projid together with all of the other statx attributes in a single system call.  This would remove the need for the open+fget_projid+close calls in cb_find_init() (for "lfs find", and other tools), since it already has statx information for the inode.
