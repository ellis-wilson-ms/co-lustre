LU-18977 efalnd: Introduce AWS EFA Lustre network driver

This commit introduces a new Lustre network driver (LND) for AWS Elastic
Fabric Adapter (EFA), enabling high-performance, low-latency networking
for Lustre clusters on AWS infrastructure.

EFA is a network device that provides reliable communication and kernel
bypass capabilities, targeting more consistent latency and higher
throughput than traditional TCP-based communication.

Key components added:
- EFA LND core implementation with support for RDMA operations
- Dual NID support:
  * Small NIDs: Uses TCP NI for out-of-band communication to discover
    remote GID and CM QP data, this utilizing LNET ping operations.
  * Large NIDs: Leverages 16 bytes of available address space to store
    connection data directly, eliminating the need for out-of-band
    discovery.
- DebugFS interface for monitoring and troubleshooting
- Module parameter configuration for tuning performance
- Integration with existing LNET infrastructure

The LND already has active customers and has been thoroughly tested
using selftest, IOR, FIO, GDS, and multiple sanity test variations.
