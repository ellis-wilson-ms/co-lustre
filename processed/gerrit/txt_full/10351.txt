LU-5053 ptlrpc: fix for ptlrpcd soft lockup

This patch changed a few things:

- ptlrpcd can loop for long time without releasing CPU if upper
  layer are consecutively adding new requests to ptlrpcd because
  ptlrpcd_check() may always return true, this patch added
  cond_resched() to ptlrpcd_check() and allow ptlrpcd yield cpu
  when necessary

- ptlrpcd_check() always scan all requests on ptlrpc_request_set
  and try to finish completed requests, this is low efficience,
  this patch improved this by moving completed requests at the
  head of list in ptlrpc_check_set.

  With this change, ptlrpcd_check doesn't need to scan the request
  list for twice all the time.

- l_wait_event() always checks condition for a two times before
  sleeping and one more time after waking up, which means it will
  call ptlrpcd_check() for three times in each loop.

  This is very expensive because ptlrpcd_check will scan all requests
  for 1-2 times.

  This patch added "wakeup version" to ptlrpc_request_set, this
  version number increases whenever reqset is scheduled for any
  reason.

  ptlrpcd will check this version number, if it's different with the
  version that ptlrpcd has already checked, then ptlrpcd needs to
  save the new version number and call into ptlrpc_check_set(),
  otherwise ptlrpcd can skip scan.

  With this change, ptlrpcd_check() could be much cheaper if it's
  called for more than once in the same loop.
