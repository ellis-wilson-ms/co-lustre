LU-1866 lfsck: FID-in-dirent and linkEA consistency

In LFSCK 1.5, we have implemented the following functionalities:

1) Initial OI scrub to verify and rebuild OI mappings for server
   local objects when the server mounts.

2) Verify and rebuild FID-in-dirent, mainly for MDT file-level
   backup/resotre.

3) (Partly) verify and rebuild FID-in-LMA: if the FID-in-dirent
   is valid but FID-in-LMA is missed, then the later one will be
   repaired.

4) Verify and rebuild linkEA: find out and repair crashed linkEA
   entry; find out and remove invalid linkEA entry.

5) Handle IGIF object upgrading, generate IGIF-in-LMA, and add
   FID-in-dirent and generate IGIF-format linkEA.

6) Find out missing name entries and add them back to namespace.

7) (Partly) find out missed object and add it back to namespace:
   if name entry exists, but related inode nlink count is zero,
   then the inode nlink count will be repaired.

8) Verify and correct object's nlink attribute.
