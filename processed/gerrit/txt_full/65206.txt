LU-12668 llite: DIO EC recovery on read failure

When a DIO read fails on an EC file, the BRW error arrives
after cl_io_end via cl_sync_io_wait_recycle, so the
synchronous -EAGAIN handling in cl_io_loop cannot catch it.

Detect the failure in the DIO completion path and switch to
CIT_EC_RD, which reconstructs data from parity via the page
cache.  The key steps are:

 - Free the DIO aio anchor so the buffered EC_RD restart
   does not re-attach or re-wait on it.
 - Revert and re-expand the iov_iter, which was consumed by
   vvp_io_advance during the failed DIO iteration.
 - Rewind ki_pos, which was advanced by vvp_io_read_start.
 - Zero ci_bytes so the post-IO accounting does not advance
   ppos (via crw_pos) or inflate result for this failed DIO.

The EC_RD restart populates the page cache, then
vvp_io_ec_rd_end copies data to the user buffer via
generic_file_read_iter with IOCB_DIRECT temporarily cleared.
