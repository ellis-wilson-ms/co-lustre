LU-5319 ptlrpc: atomically assign XID and put on the sending list

it's required to have all assigned XIDs on the sending list so
we can learn the lowest active XID at any point. this is in turn
required to be able to report the highest replied XID to the server
so the server can release corresponding slots in last_rcvd.

use that value to calculate the highest XID the server won't get.
this value is sent to the server.

currently XIDs are generated way ahead of actual sending, so it's
easily possible that the sending list doesn't list all XIDs in flight
or to be sent.

Test-Parameters: fortestonly mdtfilesystemtype=ldiskfs mdsfilesystemtype=ldiskfs \
ostfilesystemtype=ldiskfs envdefinitions=SLOW=yes
