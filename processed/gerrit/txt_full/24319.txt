Patch Set 1: Code-Review-1

(4 comments)

Looks like the update to the lfs getdirstripe usage message does not match what is actually implemented for lfs_getstripe_internal().  Possibly there is some confusion between lfs_getdirstripe() and lfs_setdirstripe()?

That said, having the "--mdt-hash" option would definitely be useful (e.g. to locate directories that are in the middle of migration).

Running "lfs getdirstripe --mdt-hash" for a non-striped directory should print something useful like "none" or similar, instead of returning an error.  Also desirable would be making an array of LMV_HASH_TYPE_* -> name mappings and looping through those when parsing stripe_hash_opt in lfs_setdirstripe() instead of a large if/else block.

Please also verify that the '-t' option does not conflict with PFL.
