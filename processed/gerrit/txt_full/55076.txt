> Do version codes with git hashes work?

The version_code() function will use any "-._" as a separator, drop non-numeric characters, and build a 4-digit version code from that, and ignore the rest. For a version from a git commit hash, it will use eg. 2.15.4.27 and ignore the hash, so that gets us down to a one-patch version granularity, since the "-27" part is the number of commits on the branch after the tag.  

While the commit hash is ignored, I prefer to include the full "git describe" into the check so that it is easier to see exactly which commit is involved. Same with kernel version checks in autoconf that show exactly which kernel commit was the source of some API change, in case it is later of interest.
