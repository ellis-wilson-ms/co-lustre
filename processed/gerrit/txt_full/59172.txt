LU-18993 osc: Add separate read/write RPC latency histograms

Add separate read and write RPC latency statistics tracking to the
OSC layer, similar to the existing offset histograms. This tracks
the time between RPC submission and completion for both read and
write operations, providing valuable performance metrics for
diagnosing Lustre client-server communication issues.

The latency statistics are displayed in the rpc_stats file along
with the existing histograms, with separate columns for read and
write latency.

A test is added to verify that both read and write RPC latency
histograms are populated correctly.

Partially coded by Augment.
