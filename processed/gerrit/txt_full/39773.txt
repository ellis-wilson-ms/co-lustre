LU-13187 ldiskfs: Fix max_dir_size_kb for RHEL7

The patch ext4-max-dir-size patch creates a 'max_dir_size' and
'max_dir_size_kb' which are the exact same thing. This is wrong
which causes sanity test 129 to fail. Change 'max_dir_size_kb'
to cache the attribute value in KiB.

There are two reasons this is wrong.

Firstly, that patch already handles each case correctly.
sbi_ui_show and _shore are modified to treat "max_dir_size" specially and to convert between the stored size in KB and the apparent size in bytes.

Secondly, you patch changes the behaviour for max_dir_size_kb so that instead of just presenting the stored value, it now convert to (and from) bytes.
