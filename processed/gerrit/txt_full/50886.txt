--- Commit Message ---
LU-7495 utils: add --links option for lfs find

This adds a "--links" option for lfs find to filter files and
directories by the number of hard links. It also adds a printf format
'%n' to print the number of links for a file.

This commit also fixes '-l' as a short option for '--lazy' which was
added in 11aa7f8704c490b011f60f234c3ac9929ce76948 but the short option
did not work.

Patch Set 2: Code-Review-1

It looks like the test failure happens because the stat info from the MDS (from get_lmd_info_fd)  has a different link count than the link count from the regular stat syscall, at least when the directory is striped across multiple MDSs. I didn't realize that was the case.

I believe the expected behavior is to have --links use the value from the regular stat system call. The lmd->lmd_stx struct gets filled in with the regular stat values later in cb_find_init so this can be fixed by moving the link check after that. I'll upload a revision that does that. I'll also add test cases for the link count of directories with single and multiple stripes. (It looks like --printf already got the correct value)
