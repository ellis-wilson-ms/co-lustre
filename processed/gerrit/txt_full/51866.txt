LU-17015 sec: support large Kerberos tokens

On client side, increase default size allocated for token in GSS
context init request, from 1024 to 4096. This will consume more memory
only for SEC_CTX_INIT requests, and in all cases buffers are shrunk
before being sent over the wire.
And when packing context init request, do not assert if token is
larger than pre-allocated size. Return an error instead.

On server side, change the format of sunrpc cache requests/responses
exchanged between kernel and user-space. The protocol only imposes
that each request or response record should be printable ASCII with
precisely one newline character which should be at the end.
So instead of printing data to hexadecimal format, which basically
doubles size used in buffer, just base64url encode it. It has a 4/3
ratio, which is much more efficient.
This format change does not require any compat flag, as this is
server-side internal processing.
