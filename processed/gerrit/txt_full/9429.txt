Hi Lai, I'm copying my response to Jinshan's question.

> As part of the code cleanup project I've been trying to do some work with our headers with the main goal being that we don't expose so much to llapi/userspace. (This will make them less fragile in general and discourage inappropriate use.) But it gets very messy because headers include so many other headers but .c files often don't directly include what they need. So if I remove the inclusion of lustre_foo.h from lustre_bar.h then suddenly lustre/utils/baz.c won't compile.

> So my plan was to gradually move to having .c files include headers for what they used. It's easy to automate finding the missing and unneeded headers.

Moreover we still build and ship liblustre.so. So we should remove unused functions here so that we can better simplify the components used by liblustre.
