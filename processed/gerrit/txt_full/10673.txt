LU-5163 mdt: using mdo_lookup(..) in mdt_rename_parents_lock()

Using mdo_lookup() instead of mdo_is_subdir() to check the
parent/child relationship in mdt_rename_parents_lock(), because
we only need to know if source and target directory are direct
parent and child, otherwise they can not be locked in one
operation, except rename, but rename has already been serialized
by global rename lock.
Besides, in the original implementation, mdo_is_subdir() is not
protected by ldlm lock, which might bring some risks.
