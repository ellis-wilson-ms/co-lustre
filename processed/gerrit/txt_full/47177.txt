LU-15805 wbc: uncache a given file or directory from WBC

We implement a new lfs command for WBC:
	$LFS wbc uncache $file

This command will uncache the whole subtree from the topmost root
WBC directory to the given file under WBC.

Please note in aging keep flush mode, the command "sync $file" can flush the file into the persistent storage on the server, but it will not drop the root WBC EX lock protected on the $file
The new LFS command "$LFS wbc uncache $file" will drop the root WBC EX lock level by level from the topmost root WBC directory to the given $file under WBC.
After that, the given files or directories and its ancestor are all not protected by any WBC EX lock, the operations on these files or directories are just the normal Lustre operations.
