--- Commit Message ---
LU-7988 hsm: wake up cdt when requests are empty

The coordinator only runs once per second, we need a mechanism
to send more work when everything is done (cdt_request_count
goes to zero)

Without this, there is a hard limit of max_requests per sec
requests that can be processed, causing performance issues
with small files.

--- Message (Ben Evans, 2017-10-24 17:21:16.000000000) ---
Patch Set 1:

Frank, that's exactly what this patch is supposed to do.

If you mark 10k files for HSM to process, and the default max is 3, then the absolute fastest you can process the whole list is 55 minutes, regardless of file size, number of copy tools, bandwidth, etc.

This provides a way of signalling that all the requests have been processed, and more need to be handed out before the 1 sec interval.
