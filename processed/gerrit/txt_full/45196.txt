LU-0000 llite: Make ras align round up

ras_align rounds readahead *down* to the nearest RPC size,
which sometimes causes readahead to not read data that it
wants to read.

As noted previously in this patch series, we should *never*
decide not to read a page selected by the readahead
prediction algorithm.  We can *fail* or be *unable* to, but
we should always read every page it asks for if we can.

The only benefit to not doing this is increased complexity
and making it impossible to write an entirely reliable
prediction algorithm - when the lower layers decide they
don't think they should read a predicted page, it makes it
impossible for the readahead algorithm to do its job.
