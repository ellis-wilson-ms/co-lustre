--- Commit Message ---
LU-80 utils: Large stripe support

Currently a file can be stripped across OSTs up to a limit of
160 stripes due to the ldiskfs xattr size limit of 4096 bytes.
This limit will be increased to 2000 stripes or more by increasing
the maximum xattr size for the MDT.

During testing, issues emerged with clients that are not expecting
more than 160 stripes. This patch allows clients to interoperate with
servers that do not support large xattrs, and also servers with
support with at least 2000 stripes, though it is intended not to have
any hard upper limit.

Patch Set 2: I would prefer that you didn't submit this

(8 inline comments)

Several errors with this patch:
- it appears that it would not work as-is, because of the critical missing OBD_CONNECT_MAX_EASIZE flag at connect time.  Without this, the MDS has no idea that the client understands large xattrs, and it needs to assume it does not.
- the addition of the "max_lov_ea_size" flag is incorrect.
- if this was being submitted for adding client-only support to 2.1, there are quite a number of patches for the server-side that would not be needed (all md[ds]/*, lvfs/*, osd-ldiskfs/*, ptlrpc/* changes should be dropped).

That said, I'm reluctant to add client-only support to 2.1 because it makes it difficult to test the client code, and it opens up the chance of compatibility issues if the code that is finally tested and landed for 2.2 is different for some reason.
