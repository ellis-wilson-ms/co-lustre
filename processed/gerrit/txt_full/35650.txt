--- Commit Message ---
LU-11205 mdd: process mixing changelog records

A changelog subsystem assigns index and adds a new records
to the end of llog. There is no synchronization to combine
both operation. And a llog records order doesn't fit with an
index order, and it could be mixed. But a changelog logic
base on index order.

The patch changes changelog processing logic to work with
a mixing records. Also cr_index assignment is moved after
taking transaction handle, this guarantee that mixing
records beelong to a single transaction.

The patch adds 160j sanity, to check that reordered recordes
at changelog could be canceled.
