--- Commit Message ---
LU-169 lov: add basic infrastructure for layout lock

This patch adds some basic infrastructure to support the layout lock
in a near future. This includes defining a new inode lock bit to lock
the file layout (namely MDS_INODELOCK_LAYOUT) as well as a new lookup
intent (IT_LAYOUT).

--- Message (Andreas Dilger, 2012-01-07 23:59:01.000000000) ---
Patch Set 4: I would prefer that you didn't submit this

(1 inline comment)

I have no objection to landing the i_size_{read,write}() part of the patch, and #define MDS_INODELOCK_LAYOUT, but this goes beyond just defining constants.

It appears the clients will actually be getting MDS_INODELOCK_LAYOUT due to its inclusion in MDS_INODELOCK_FULL, but I don't know what effect that will have on the server thinking that the client will drop the layout if that lock is canceled.  If there is a good case for this (i.e. it will actually improve future interoperability, instead of (IMHO) possibly make it worse), it should be clearly explained in the commit comment what effect this part of the patch intends.
