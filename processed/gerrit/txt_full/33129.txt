--- Commit Message ---
LU-11276 ldlm: fix lock convert bl_ast races

A lock may be canceled being converted and DoM data
is not flushed as needed.

Patch changes ll_md_blocking_ast() code to don't rely on
'canceling' lock flag but checks cancel_bits instead and
do flush always if DoM bit is set.
For DOM lock the data flush should be done with DOM bit set
so do that before lock convert while that bit is set.
