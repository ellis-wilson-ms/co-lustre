--- Commit Message ---
LU-14142 lfs: don't ignore pool in migrate w/o other params

Attempting to migrate a file between pools with "lfs migrate" fails
when only the pool name is specified, without any other stripe
options.

E.g. this doesn't move the file to the specified pool:

lfs migrate -p pool2 /lustre/dir0/file0

but this does:

lfs migrate -c 1 -p pool2 /lustre/dir0/file0

Ensure the pool name is used when it is the only striping parameter
specified during file migration.
