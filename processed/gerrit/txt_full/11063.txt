LU-5315 utils: a new option to specify plugin modules dir

Instead of checking $LUSTRE environment variable set by lustre
test-framework.sh, this patch adds a new command line option
"--plugin-dir" to mkfs.lustre and option "plugin_dir" to mount.lustre
respectively to specify lustre plugin modules directory.
If this is not used, the relative path will be checked for the local
build modules, and then the default location for the installed ones.
The default location is defined at compile time by the automake
variable $(pkglibdir), e.g. /usr/lib64/lustre on RHEL6.

The usage of this option is like:
mkfs.lustre --mgs --mdt --index=0 --fsname=lustre \
--plugin-dir=../utils/.lib/ --reformat /dev/sda2
mount -t lustre -o loop,plugin_dir=/usr/lib/lustre/.libs/ \
/tmp/lustre-mdt1 /mnt/mds1
