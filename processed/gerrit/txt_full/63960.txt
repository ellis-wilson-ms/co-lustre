LU-12669 lov: fix parity offset calculation in EC recovery

The parity page offset (poff) incorrectly includes
lsme_p->lsme_extent.e_start (the PFL component's file-level
start offset):

  poff = lsme_p->lsme_extent.e_start +
         raid_set * lsme_p->lsme_stripe_size;

But poff is used as a sub-object offset for reading from
the parity OST.  Parity sub-objects are indexed from 0
within each component, not from the component's file
offset.  For the first PFL component (e_start == 0) this
is harmless, but for subsequent components the extra
offset causes parity reads to access the wrong data,
producing incorrect reconstruction.

Fix by removing the e_start term so poff is purely
sub-object-relative.

Resolves test_47a failure for multi-component
PFL layouts (bead 2-ka6).
