--- Commit Message ---
LU-14316 llite: quiet spurious ioctl warning

Calling "lfs setstripe" prints a suprious warning about using the old
ioctl(LL_IOC_LOV_GETSTRIPE) when that is not actually the case.

Move this warning up to ll_file_ioctl() where the ioctl is actually
being called, instead of deep inside lov_getstripe() where it cannot
know which ioctl is in use.

Change the LL_IOC_LOV_{GET,SET}STRIPE ioctl names to use _V1 and _V2
suffixes instead of _NEW, and make the _V2 size a fixed struct.  If
this ever changes to e.g. lov_mds_md_v3 a new IOC number will be used.

Remove the SETSTRIPE and GETSTRIPE lines from spelling.txt since they
generate only false checkpatch.pl warnings and are no longer useful.
Fix checkpatch.pl to consider RETURN in a switch as a case break.

Fixes: 364ec95f3688 ("LU-9367 llite: restore ll_file_getstripe in ll_lov_setstripe")
