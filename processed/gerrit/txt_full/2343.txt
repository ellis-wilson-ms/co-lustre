--- Commit Message ---
LU-1126 ldlm: ensure correct deref if lock mode changes

If a lock is granted in different mode other than it is
originally requested, it is possible to trigger LBUG because
of incorrect lock dereference.

e.g., LBUG code path:
ldlm_lock_create()->ldlm_lock_addref(PW)
server returns PR lock
ldlm_cli_enqueue_fini() sets lock->l_req_mode to PR
ldlm_grant_lock() sets lock->l_grant_mode to l_req_mode(PR)
ldlm_process_flock_lock() finds the lock and tries to destroy it
with ldlm_flock_destroy(lock->l_grant_mode)
