LU-37 create proper macro check for bdi interface

The original patch by Lai Siyao from bz 23064 was landed in Lustre 1.8.
This is a rebase against 2.1.

The wb_cnt field of struct backing_dev_info is removed by patches
backported to RHEL6 kernels from 2.6.34.  Replace Lustre autoconf
tests for wb_cnt with explicit checks for bdi_register() and the s_bdi
field since those symbols are what the code actually references.
