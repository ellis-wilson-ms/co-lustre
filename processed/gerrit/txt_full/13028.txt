LU-4647 tests: enable sanity-sec for DNE mode

Allow sanity-sec to run with multiple MDS nodes.  While this patch
does not necessarily actively run tests on mutiple MDTs, at least it
would no longer be skipped when multiple MDSes are configured.  This
will double test coverage, prevent potential regressions, and would
allow future tests to verify DNE functionality more easily.

Clean up code style (tabs for indent, $(...) instead of `...`, etc).

Test-Parameters: alwaysuploadlogs envdefinitions=SLOW=yes \
mdtcount=4 testlist=sanity-sec

Test-Parameters: alwaysuploadlogs envdefinitions=SLOW=yes \
mdtcount=1 testlist=sanity-sec

--- Message (Andreas Dilger, 2014-12-15 21:25:26.000000000) ---
Patch Set 4:

Kit, for some reason I thought the code was configuring the nodemap on all the MDSes, but I see now that all the operations are actually only happening on the MGS (which happens to work in the common test case of MGS+MDT0 on the same node) and not the other MDTs or OSTs (only OST0 in some quota tests).

I thought this patch would at least start testing the code to configure nodemaps on other MDTs, even if files were only being created on MDT0, but that isn't the case, so I'm not sure if there is value to landing this patch as-is or not.

There would be some value for 2.7.0 to configuring nodemaps on all MDTs and OSTs via /proc and then changing the tests to create files on remote MDTs (test_mkdir is used in sanity/sanityn to create remote and/or striped directories automatically).  However, that would also be achieved by finishing the MGS distribution of nodemaps (though that couldn't be landed for 2.7.0), so I'll leave it up to you whether the added coverage of this code for 2.7.0 is worthwhile or not.
