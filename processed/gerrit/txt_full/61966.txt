LU-0000 lov: do-no-harm IO support for EC layouts

Add support for regular IO operations on files with erasure
coded (EC) layouts by transparently skipping parity
components in the IO path.

This implements "do-no-harm" IO where EC parity components
don't interfere with regular IO operations. The IO layer
skips parity components during:
- Mirror write intent selection
- IO iteration and initialization
- Lock initialization
- Page initialization
- Truncate operations
- LRU reservation
- Layout lookups

Also fixes EC file creation in the new EC mirror model by:
- Allowing parity components as first component in mirror
  when starting at offset 0 (new model)
- Setting mirror count to 2 in add_ec_parity_components()

Adds comprehensive test coverage with 3 new tests (4a-4c)
that verify 12 different IO operations work correctly on
EC files: write, read, append, dd, truncate, chmod,
fallocate, and mixed seek operations.
