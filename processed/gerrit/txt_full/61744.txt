LU-0000 llite: add EC mirror read/write ioctl support

Add support for reading and writing directly to EC parity
components via new LL_IOC_FLR_SET_MIRROR_EC ioctl.

This patch implements:

1. Kernel changes:
   - New ioctl LL_IOC_FLR_SET_MIRROR_EC to designate EC parity
     component IO
   - fd_ec_designated flag in ll_file_data to track EC IO mode
   - ci_ec_designated flag in cl_io to propagate EC designation
   - Updated lov_io.c to select parity vs data components based
     on ci_ec_designated flag
   - Allow truncate on parity components when EC designated

2. Userspace API:
   - llapi_ec_mirror_set() to enable EC parity component IO
   - llapi_ec_mirror_clear() to disable EC parity component IO

3. lfs ec commands:
   - Updated lfs ec read/write to use EC ioctl
   - Added validation to reject non-EC files
   - Lease support disabled until EC files support FLR
     state transitions (code preserved for future use)

The EC ioctl allows direct access to parity components for
testing and debugging. When ci_ec_designated is set, IO is
directed to components with LCME_FL_PARITY flag instead of
regular data components.
