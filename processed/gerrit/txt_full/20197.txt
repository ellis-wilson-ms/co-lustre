--- Message (Andreas Dilger, 2016-05-16 18:44:27.000000000) ---
Patch Set 1:

I agree that brw_size was only landed to master in 2.8+, but DDN was previously using it.  In any case, max_pages_per_rpc has been around for ages, and it has accepted arguments of the form "4M" for a long time.

I agree the parameter name was ill chosen, since it directly reflects the internal implementation of the code.  The problem is that specifying the RPC size in terms of client PAGE_SIZE depends on what PAGE_SIZE is, and that may differ between clients.  The ability to uniformly specify the RPC size in units of MB was added after the fact.  

While I agree this is baggage we'd all prefer wasn't there, breaking this tunable (which is one that is changed quite frequently) would be even worse.  I've submitted a patch to disable this test, which should have been done on Friday, and Oleg is working on a patch to fix the actual parsing of this parameter so that it allows units as it previously did.
