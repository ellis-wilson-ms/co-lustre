--- Commit Message ---
LU-16308 llite: wake_up after cl_object_kill

cl_inode_fini() calls cl_object_kill() to set LU_OBJECT_HEARD_BANSHEE,
and then calls cl_object_put_last() to wait for object refcount to
become one, It should wake_up() in the middle in case someone is
waiting on the flag.

--- Message (Neil Brown, 2022-11-14 23:08:30.000000000) ---
Patch Set 1:

To be able to review this properly we need more details of the race that you are trying to close.

Code that waits on this waitq checks for LU_OBJECT_INITED or LU_OBJECT_HEARD_BANSHEE to be set.
According to the documentation for waitqueue_active() we need a memory barrier after the bit is set, and ALSO before the bit is tested.
The test should be changed to test_bit_acquire()

Since Linux 4.5 we've had "wq_has_sleeper()" which performs the smb_mb() and then the waitqueue_active() check.  It would be best to use that.

The comment that is requires before waitqueue_active needs to be more than just "we need a barrier".  It is supposed to tell us what guarantees the safety, both here at the wakeup and also at the waiter.
So a comment before wq_has_sleeper() should say "We might have set LU_OBJECT_HEARD_BANSHEE, waiter uses test_bit_acquire() to avoid races".  And before the wait_event call, "wakeup might use wq_has_sleeper() so we use test_bit_acquire to avoid races".

wake_up_all() is only needed when there are exclusive waiters.
wake_up() will wake ALL non-exclusive waiters, and at most 1 exclusive waiter.
wake_up_all() will also wake all non-exclusive waiter, but also all exclusive waiters.

Seeing wake_up_all() can make the reader think there is an exclusive wait, and this would be wrong.
Please leave it as wake_up().
