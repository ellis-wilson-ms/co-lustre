--- Commit Message ---
LU-20 kernel: make bh_lru_size a kernel parameter

The benefits from bh_lru_size_config.patch can only be seen
with Lustre. So to ease adoption of buffer head per-CPU LRU
size tuning in the upstream kernel, make this tuning be a
kernel parameter instead of a kernel config option.

--- Message (Andreas Dilger, 2014-06-12 15:28:59.000000000) ---
Patch Set 4: Code-Review-1

(1 comment)

The kernel patch looks good, and I would suggest submitting it upstream. 

For landing into Lustre the patch will need to also fix the SLES kernels, so mark this -1 for now. 

Also, there needs to be some mechanism to set notify the admin that it is missing (e.g. a message in mkfs.lustre and mount if the lru size is still the default value and there is no bh_lru_size kernel parameter). Otherwise this would be a performance regression for all users of Lustre that do not know about this parameter. Fortunately, the kernel will ignore parameters that it doesn't understand, so it is possible to set this parameter on the kernel command line even for older kernels that are patched to change it at compile time.
