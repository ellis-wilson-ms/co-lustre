LU-969 debug: reduce stack usage

1, libcfs_debug_vmsg2 to accept libcfs_debug_msg_data struture

   cdls on the stack

   ptlrpc_thread to heap.

Patch Set 7:

> the mask is "*fatal ? D_NETERROR : D_NET", which will
> cause "error: initializer element is not constant"

The number of places that have non-constant masks is very small.  There are 6 callers of CDEBUG_LIMIT, and any callsites that use CDEBUG(mask, ...) that may print to the console (mask = D_ERROR, D_WARNING, D_CONSOLE) should be considered an error because they are not rate-limited, so I'm happy if this turns into a compile-time error.  This will also reduce (but not totally eliminate) the cases where there is a race between one error mask and another.

For those few sites that do this, they should just be split into two separate calls - one using CERROR() and one using CDEBUG() (or appropriate, as needed).

> the independent msgdata is added into CFS_CHECK_STACK 
> for it could change the msg_mask, and CFS_CHECK_STACK
> is used in three places, especially in "debug_req" and
> "ldlm_lock_debug" it will also affect the other functions,
> which make the code look not very good

But the cost of double msgdata structs for every callsite of CDEBUG(), LDLM_DEBUG(), DEBUG_REQ() will be very high.  Passing the mask to CFS_CHECK_STACK() avoids this problem without increasing the stack usage, and the number of times that the stack usage is growing will be very small, and definitely not worth the extra memory usage.
