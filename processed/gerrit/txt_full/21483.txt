--- Commit Message ---
LU-8367 tests: long waiting for multistripe file creation

This test reproduces hang during multistripe file creation.
 1) reserve object for OST0
 2) delay reserve object for OST1 (simulate no object)
 3) failover OST0
As result osp_precreate_cleanup_orphans() hangs until the creation
of all stripes is finished. This leads to impossibility for object
allocation at OST0.
For two multistripe files, when first reserved OST0 and second
reserved OST1, reconnect. Both file creations will sleep OBD_TIMEOUT.
