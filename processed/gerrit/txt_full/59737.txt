LU-0000 flr: add mirror error LVB creation

TODO: The demo code is still present, which is sort of good because
we're only going to put the errors in if we're using DIO, not AIO
or buffered IO.

AIO is going to stink here.

Next item - if you're going to pursue this - is having the server
do something intelligent with this info, ie, parse it and use it
when handling errors.  That means you'll need that new flag too.
----

Add infrastructure to create mirror error LVB data for Active Writer
lock cancellation:

- Add ldlm_mirror_error_lvb structure with magic, mirror_count, and
  flexible array of mirror error codes
- Add ldlm_create_mirror_error_lvb() helper function to create LVB
  data from mirror error arrays
- Integrate LVB creation in ll_file_io_generic() after DIO completion
  for immediate mirror files
- LVB contains simple integer array with one error code per mirror

The LVB data will be sent to the server during Active Writer lock
cancellation to communicate per-mirror I/O results.
