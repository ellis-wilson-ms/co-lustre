--- Message (Andreas Dilger, 2021-10-16 16:37:57.000000000) ---
Patch Set 2: Code-Review-1

One of the reasons for using GFP_HIGHUSER for page allocations is that there may be very little memory pressure on an OSS node if this is not used. Always having __GFP_NOFS means no memory reclaim is happening and can hit OOM. See LU-9728 for details. 

If there is a specific code path that is recursing to call here, it should set the flags to prevent GFP_HIGHUSER from being called, but that shouldn't be disabled for all allocations.
