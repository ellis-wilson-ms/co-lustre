Patch Set 19: I would prefer that you didn't submit this

(7 inline comments)

I agree that this code might be restructured to be less complex, but I don't think it is worthwhile to do that until we have more idea about what requirements the later phases of the code might have.  There are a couple of other minor issues that should be fixed before landing, however.

My thought while looking at this code was to move the iteration entirely into osd_iit_next() so that it handled loading of bitmaps (stored in the "scrub" context) and was only passed the "*pos" and "*dev" as arguments.  That would allow simplifying the callers instead of having the inode iteration multiplex between the functions.

Another option would be to have a callback context that is passed by the caller and run for each inode (calling the "next" and "exec" functions), rather than having osd_inode_iteration() check "if (preload == 0)" each time.

The reason I do NOT ask for this to be restructured again right now, is that I'd rather stick with something working right now, and only change the code if there is a requirement in later phases to do so.  I'm OK with the cleanup that has been done so far, and want to keep moving these patches along.
