LU-16096 recovery: upgrade compatibility for new reply data

As the batched RPC protocol will change the disk format of the
client reply data "REPLY_DATA" for recovery, thus we need to
handle the compatibility during upgrade carefully for this new
format change of the reply data.

The solution is as follows:
Add a flag in the magic number field of the reply data header:
LRH_MAGIC_V1: 0xbdabda01 - magic number for the old reply data.
LRH_MAGIC:    0xbadbda02 - magic number for the new reply data.
LRH_FLAG_BACKUP_DONE: 0x00000004 - indicate that the target has
finished to backup the reply data with the old format.

We finish the upgrade for the client reply data during the target
setup:
1. If found that the "REPLY_DATA" was the old format (according to
   the magic number in the reply data header: LRH_MAGIC_V1), the
   target will start to backup the reply data file "REPLY_DATA"
   into the backup file "REPLY_DATA_BACKUP".
2. After finished the backup, sync the data into the persistent
   disk. And then the target will change the magic number of the
   reply data file with LRH_MAGIC_V1 | LRH_FLAG_BACKUP_DONE, and
   sync the change of the header to the disk.
3. The target starts to convert the old format reply data form
   the backup file "REPLY_DATA_BACKUP" into the original reply
   data file "REPLY_DATA".
4. After finished the conversion, sync the data to the disk. Then
   the target changes the magic number with LRH_MAGIC and the
   reply size @lrh_reply_size in the reply data header with the
   new format and sync the change to the disk.
5. Finally, delete the backup file "REPLY_DATA_BACKUP".

After finshed the upgrade of the reply data, the target can start
the recovery and processes as normal with the new format reply
data.
