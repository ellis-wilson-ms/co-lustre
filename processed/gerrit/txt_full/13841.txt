--- Commit Message ---
LU-6245 libcfs: remove mem wrappers for libcfs

The libcfs layer contains wrappers to handle memory
management for both kernel space and user land. Since
libcfs is no longer built for user land we can remove
the wrappers. For linux-mem.h we kept only what is
needed to deal with different kernel versions. The
rest was moved to libcfs_prim.h.

--- Message (James Simmons, 2015-03-23 15:43:55.000000000) ---
Patch Set 3:

(1 comment)

PAGE_CACHE_MASK is not a replacement for PAGE_CACHE. The idea behind this was to support systems where the page cache uses a different type of memory than normal system ram. For example you could use huge page support for the page cache which would make the page size very different than the normal 4K on x86 machines. Another good example of this is that on our systems when a application does not use the GPU we use the ram on the graphics card as a page cache. Yes the PCI bus is a boot neck but it is still faster than disk. Currently the linux kernel has not implemented this but I do agree Lustre should prepare for the day. A separate ticket should be opened for this.
