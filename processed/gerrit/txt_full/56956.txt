LU-18438 osc: do not osc_extent_wait on conflicting extent

cl_io_loop
  cl_io_lock
    ..
      osc_lock_enqueue <<< ldlm lock gets writer
  cl_io_start
    vvp_io_write_start
      vvp_io_write_commit
        osc_io_commit_async
          osc_queue_async_io
            osc_extent_find
              for (; ext; ext = next_extent(ext)) {
                if (overlapped(ext, cur)) {
                  conflict = osc_extent_get(ext);

              if (conflict != NULL) {
                rc = osc_extent_wait(env, conflict, OES_INV);
  cl_io_end
  cl_io_unlock
    ..
      osc_lock_detach <<< ldlm lock can be cancelled

Using osc_extent_wait() having referenced ldlm lock may lead to client
eviction, as the lock can not be cancelled.

Test to illustrate the issus is added.

The fix is to not to wait, but repeat the operation.
