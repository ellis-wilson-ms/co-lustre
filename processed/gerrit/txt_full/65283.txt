--- Commit Message ---
LU-19497 tests: fix sanity/908a for interop upgrade

When running rolling upgrade testing, if the MDS filesystem was
formatted with pre-2.16.50.151, the change timestamp on "params"
(or any llog) file is not set when it is created.

This causes sanity test_908a to fail when it checks the llog ctime.
Skip this test when the "params" file was created on an old MDS,
using the llog version markers stored in the "params" file.

Fix the test interop version checks in test_908a and test_908b to
accurately reflect the version where these tests were first added.

test_908a should actually be checking the "params" log on the 'mgs'
facet instead of the 'mds1' facet, in case they are separated.

Test-Parameters: trivial fstype=ldiskfs testgroup=rolling-upgrade-client1
Fixes: e33d196b93 ("LU-17309 llog: set timestamp on llog objects at creation time")
