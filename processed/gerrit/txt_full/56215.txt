--- Commit Message ---
LU-18186 libcfs: avoid kmalloc order > MAX_PAGE_ORDER

    WARNING: CPU: 4 PID: 448333 at mm/page_alloc.c:4663
    __alloc_pages_noprof+0x3a7/0x4e0

libcfs calls memdup_user_nul() which uses kmalloc() which can
trigger the above warning.

Limit these excessive allocations a reasonable minimum.

Patch Set 1: Code-Review-1

(7 comments)

I think this is giving too much leeway for something that should never actually work.  We don't expect 8MB values to be written to these files, so handling these large allocations just allows more things to go wrong.

I think a more robust solution is to just check for "reasonable" sized values at the beginning (e.g. PAGE_SIZE or PATH_MAX), and reject them early rather than trying to allocate a GB of RAM first.
