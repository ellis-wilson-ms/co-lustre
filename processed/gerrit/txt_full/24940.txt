--- Commit Message ---
LU-1573 test: replay-ost-single test_9 issue extra write

The proposed changes in the LU-1573 patch,
https://review.whamcloud.com/#/c/16680, changes Lustre in a
way that makes replay-ost-single test 9 hang. A sync write
from the existing call to dd is executed after the fail_loc,
the file system is short on grants and the response is
dropped. To modify replay-ost-single to pass with the new
changes to Lustre, we need to execute a write (dd) before
the replay_barrier so that the necessary grants are
procured before the fail loc and the file system should have
enough grants to allow the sync write to proceed.
