Patch Set 1:

In earlier versions of my original test-suite, I had code that would check the state of the test environment before sanity-lnet was executed, and then restore that environment when the test exited. Do we want to do something like that? You can see the code here https://review.whamcloud.com/#/c/35386/8/lustre/tests/sanity-dlc.sh

In brief:

  restore_mounts=false

  if is_mounted $MOUNT || is_mounted $MOUNT2; then
	  cleanupall || error "Failed cleanup prior to test execution"
	  restore_mounts=true
  fi

  restore_modules=false
  if module_loaded lnet ; then
	  cleanup_dlc || error "Failed to unload modules before test execution"
	  restore_modules=true
  fi

  cleanup_testsuite() {
	  rm -f $TMP/sanity-dlc*
	  cleanup_dlc
	  if $restore_mounts; then
		  setupall || error "Failed to setup Lustre after test execution"
	  elif $restore_modules; then
		  load_modules || error "Couldn't load modules after test execution"
	  fi
	  return 0
  }

I don't like the check_and_setup_lustre call because it'd be nice if you could run this test without waiting for lustre setup/teardown.
