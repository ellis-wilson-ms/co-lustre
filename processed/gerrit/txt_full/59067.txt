LU-18238 llite: deregister HSM agent on close()

This patch automatically unregisters the HSM agent when closing the
dir FD mountpoint (opened by llapi_hsm_copytool_register()).
So it would unregister the agent if the copytool has crashed and
was unable to call llapi_hsm_copytool_unregister().

This should avoid the MDTs to send HSM requests to an already crashed
copytool.

Notes: the MDT unregisters an agent if it returns -EPIPE to an
HSM request (the Lustre client has lost the connection with the
copytool). But this causes the HSM request (and the HSM actions)
to fail.

Add regression test sanity-hsm 106b.
