Patch Set 1: Looks good to me, but someone else must approve

I tend to think this patch is better than http://review.whamcloud.com/#change,2149 .

I think we'd keep the semantics of mdt_object_find(): it just find the object, caller should call mdt_object_exists() to see if the object is existing.

I'm not quite sure the root cause of LU-966, it seems that the LASSERT is only triggered when filesystem (or data) is corrupted by fsck, so this LASSERT looks proper to me (assert on external errors which we can't manage), however, to avoid crash server, we may downgrade this LASSERT into an error message and return error. (though I'm not sure if filesystem can still work properly)
