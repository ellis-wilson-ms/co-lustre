--- Commit Message ---
LU-13799 osc: Update shrink interval per i/o

We currently update the grant shrink interval for every
page; this is surprisingly expensive and there's no reason
to do this because the shrink interval is in seconds.

Move this out to be for each set of submitted pages rather
than for each page.  (There is no benefit to moving this
all the way up to the per i/o level and it makes the code
more complicated.)

NB: I do not have benchmarks for this, but see, eg:
https://review.whamcloud.com/#/c/39437/
for evidence that getting time for each page is costly.
