LU-14950 llite: Remove OBD_MALLOC for pages

The direct i/o code is now efficient enough that tracking
the memory allocation sizes in this code is 5% of the total
CPU time spent.

Since that's a nice-to-have and the cost (as a %) is only
rising as this gets more efficient, let's remove this.
