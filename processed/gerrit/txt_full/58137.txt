--- Commit Message ---
LU-18456 tcu: replicate XATTRs for a dir moving into trash

When remove a directory with trash can enabled, if the directory
named with its FID already existed in the trash dir, then it will
use that directory as a replication of the deleting directory.
In this case, it still needs to replicate XATTRs from the original
deleting directory.

Add sanityn.sh/test_117e to verify that XATTRs are kept when a dir
has moved into trash.
