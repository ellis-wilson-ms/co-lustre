--- Commit Message ---
LU-2613 recovery: free open/close request promptly

- For the non-create open or committed open, the open request
  should be freed along with the close request as soon as the
  close done, despite that the transno of open/close is
  greater than the last committed transno known by client or not.

- Move the committed open request into another dedicated list,
  that will avoid scanning a huge replay list on receiving each
  reply (when there are many open files).

I think the test is too fragile as it currently is.  Better to rebase this patch on http://review.whamcloud.com/6874 and use the rpc_cache to detect uncommitted open/close RPCs.  That makes the test more robust, and also helps get 6874 landed which is also good.

I would be happy to combine this client-side fix with a  server-side fix (maybe in a dependent patch) to store and return the last_committed value from the highest object version accessed by the client.  That actually increases correctness (because the client has seen newer state, it should know this), and can help fix this problem for older clients as well.

A sanityn.sh test for the server-side fix might look something like:

        # get the starting value for last_committed
        sync; sleep 3; sync
        START_LC=$($LCTL get_param -n mdc.*.import |
                   awk '/peer_committed:/ { print $2 }' | sort -u | head -1)
        # f1 will be 1000 transnos behind f999
        createmany -o $DIR1/$tdir/f 1000

        # get the ending value for last_committed
        sync; sleep 3; sync
        END1_LC=$($LCTL get_param -n mdc.*.import |
                   awk '/peer_committed:/ { print $2 }' | sort -u | tail -1)
        # open and close the f1 file repeatedly on second mount.
        # it should only see the first transno at this point
        for i in $(seq 500); do
              cat $DIR2/$tdir/f1
        done

        # get the last committed for client 2
        END2_LC=$($LCTL get_param -n mdc.*.import |
                   awk '/peer_committed:/ { print $2 }' | sort -u | head -1)

        # check that the client 2 end LC ~= the start LC
        # check that accessing $DIR2/$tdir/f999 increases client 2 LC
