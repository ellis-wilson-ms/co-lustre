LU-0000 flr: implement INCONSISTENT flag for immediate mirrors

This patch implements the INCONSISTENT flag infrastructure for FLR
immediate mirror files to mark mirrors that encounter errors during
concurrent write operations.

Key changes:
- Add LCME_FL_INCONSISTENT flag (0x00000800) to lustre_idl.h
- Update wiretest.c to include the new flag constant
- Implement mirror error detection and flag setting in LOD layer
- Add server-side error processing in mdt_handler.c for FLR state
  transitions from WRITE_PENDING to RDONLY
- Extend LVB infrastructure to carry mirror error information
- Add client-side LVB attachment for mirror error data

The INCONSISTENT flag is set on mirror components that experience
errors during immediate mirror write operations, allowing
administrators to identify and handle problematic mirrors.

Mirrors marked with INCONSISTENT flag require explicit intervention
(e.g., lfs mirror resync) to clear the flag and restore the mirror
to a consistent state.
