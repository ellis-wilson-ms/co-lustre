LU-17049 mdt: 'lfs mkdir' check supplementary groups

Check all supplementary groups when checking enable_remote_dir_gid
and enable_pin_gid, instead of only checking the primary GID under
which the RPC was sent.

Add a helper function `mdt_enable_gid_deny()` that checks whether
a particular `mdt_enable_XXX_gid` group matches current credentials
or is overridden by a capability, since this check was getting
increasingly complex.

Add MDT_ENABLE_GID_LUSTRE_RW_ATTR() helper macro to generate the
respective enable_XXX_gid() parameter code to reduce boilerplate.
Use for enable_remote_dir_gid, enable_pin_gid, enable_chprojid_gid.

Use MDT_INVALID_GID instead of '-1' to avoid signed/unsigned issues.
When printing the parameters in sysfs, handle MDT_INVALID_GID as a
special case, so GIDs > 2B print as unsigned instead of negative.

Fix 'lfs mkdir' to report an error if multiple arguments are given,
but there is a failure creating a directory that isn't last.

Add sanity.sh test_300ne to check enable_remote_dir_gid=N for a
GID that is not the primary group for the request.

Add mdt.enable_remote_dir.4 and mdt.enable_remote_dir_gid.4 man pages.

Fixes: 11b2615571 ("LU-2738 mdt: add proc entry to enable lfs mkdir for non-admin")
Fixes: 84100d7f7d ("LU-10499 pcc: tunable GID to pin/unpin files")
