--- Commit Message ---
LU-4085 build: gss/krb5 is disabled despite functions found

The function available check for krb5int_derive_key and
krb5_derive_key by means of AC_CHECK_LIB($gssapi_lib,...)
does not set shell variables HAVE_KRB5_DERIVE_KEY,
HAVE_KRB5INT_DERIVE_KEY so the test
AS_IF([test "x$HAVE_KRB5INT_DERIVE_KEY" = "x1"
-o "x$HAVE_KRB5_DERIVE_KEY" = "x1"] gives false despite
the functions are found. As a side effect this fix
revealed that calls with real prototypes are required.
Thus prototypes of krb5_derive_key(...) and
krb5int_derive_key(...) are provided, and required
structures from krb5-int.h are copied.

So I dug into this a bit more.

It turns out there was generally no prototype for krb5_derive_key() in a public header file; it was a library export, but normally not considered a public function (that's why it was changed to krb5int_derive_key(), which is also exported but apparently has a normal prototype).

I think the only way to correctly resolve this is to put a proper prototype in context.h for krb5_derive_key().

Patch Set 1:

The approach in context.h

#if HAVE_KRB5INT_DERIVE_KEY
extern int krb5int_derive_key();
extern int krb5_k_create_key();
#else /* !HAVE_KRB5INT_DERIVE_KEY */
extern int krb5_derive_key();
#endif

is actually a hack which existed before in code to obtain access to not exported functions.

/usr/include/mit-krb5>grep -r krb5int_derive_key => ""
strings /usr/lib/x86_64-linux-gnu/libkrb5.so | grep krb5int_derive_key => ""

Thats why the function is exported this way.

There is unfortunately a flaw in the autoconf tool.
The statement AC_CHECK_LIB($gssapi_lib, krb5int_derive_key
suggests that krb5int_derive_key can be accessed via 

strings /usr/lib/x86_64-linux-gnu/libgssapi_krb5.so | grep krb5int_derive_key => ""

however, that is not the case. It works however, for
e.g. AC_CHECK_LIB($gssapi_lib, krb5_get_error_message

strings /usr/lib/x86_64-linux-gnu/libgssapi_krb5.so | grep krb5_get_error_message => "krb5_get_error_message"

Regarding the function name, the function name krb5_derive_key was renamed in MIT-Kerberos >= 1.8.X

Before it was (<= 1.7.X):

~/tmp/krb/krb5-1.7.2>grep -r "krb5int_derive_key"
~/tmp/krb/krb5-1.7.2>grep -r "krb5_derive_key"   
src/lib/crypto/vectors.c:    r = krb5_derive_key (enc, in, out, usage);

and with version >= 1.8.X it changed to:
~/tmp/krb/krb5-1.8.6>grep -r "krb5_derive_key"
doc/CHANGES:Rename some lingering krb5_derive_key references.

~/tmp/krb/krb5-1.8.6>grep -r "krb5int_derive_key"
src/lib/crypto/crypto_tests/vectors.c:    r = krb5int_derive_key (enc, in, out, usage);
