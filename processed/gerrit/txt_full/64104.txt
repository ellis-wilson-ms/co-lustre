LU-17901 tests: improve sanityn test_81a error checking

test_81a was missing error checks on the setdirstripe
calls that create striped directories, so if they
fail the test continues and reports a confusing
"create 0 failed" from touch.  Add error checking
after setdirstripe so the next failure identifies
which operation actually gets the EIO.  Also remove
the cd to a relative-path working directory and use
absolute paths throughout, and fix a "filed" typo.
