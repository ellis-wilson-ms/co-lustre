LU-14789 tests: fix sanity 133f/g/h again

sanity test_133f, test_133g, test_133h were changed to use
"lfs list_param" to find the list of parameters, but need to
scan /proc and /sys directly to run badarea_io properly.

The previous version of these tests was spewing a ton of output
but still trying to find filenames based on parameter names,
and running each command with a separate ssh session.

Limit badarea_io to running against writable parameter files,
and run badarea_io commands in parallel to speed up testing.
