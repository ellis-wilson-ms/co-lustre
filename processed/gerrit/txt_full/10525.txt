--- Message (Andreas Dilger, 2014-06-06 21:16:49.000000000) ---
Patch Set 2: Code-Review+1

(58 comments)

Still a bunch of adjacent whitespace that could be cleaned up, but that isn't a requirement for landing and I know it can be tiring to do.  Just a nice-to-have if you are refreshing it again, and the more that gets converted, the less I have to bug people about it...  I tend to edit files (in vim) with "tab" as the search string so that I can easily see which lines near where I'm editing are not indented with tabs.

I'm a bit worried that this patch touches so many files that it is likely going to have conflicts.  If that is the case you may want to consider splitting it into a few smaller chunks.  I don't think there is a strong requirement to convert e.g. all of the uses of one variable over at the same time vs. just splitting it by file, since the list types and iterators are just macros and end up identical at compile time.

A few other comments about potential cleanups while I was looking at the code, but they are probably better as separate patches.
