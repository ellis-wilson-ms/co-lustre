LU-11138 lfs: getstripe display certain mirror(s)

Add [!] --mirror-index=[+-]<index> | [!] --mirror-id=[+-]<id>
option for lfs getstripe to print the components of mirror(s)
relative to <index>-th mirror or components of mirror(s) relative
to the one with mirror ID of <id>.

I think the code looks OK, but the man page can be improved a bit.

Is there a need to allow usage like "--mirror-index=+2" or "! --mirror-id=4" to allow operations on multiple mirrors at one time?  
For example, to change a file to have exactly 2 mirrors you could run "--mirror-index=+2" (or +3 if the index starts at 1).  I'm not sure if this is a requirement, but we need to consider how users will use such commands.
