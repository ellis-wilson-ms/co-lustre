LU-118 clear_inode: BUG_ON(inode->i_data.nrpages)

This patch does a lot of fixes and cleanup:
- grab an inode refcount by top cl_object, so that it can't go away
- cl_object now can be purged by shrinker
- change the way to create cl_object, now it will be created
in ll_file_open and ll_setattr_ost
- cleanup the code by defining cl_object_kill

Future work:
- now pages and locks are holding a refcount against cl_object, this causes
cl_objects can't be purged by shrinker if there exists pages and locks
for an object. This is very bad, it needs fixing.
- lu_object cache algo. The current alogrithm is not good, it should be working
as the following:
* if the object is in use, don't put it into LRU list
* when the last reference of object is put, put it into LRU
* fairness of LRU - LRU is tightly coupled with hash in current
implementation, must fix.
- lu_obj_hop_put needs revising.
