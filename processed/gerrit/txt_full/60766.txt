--- Commit Message ---
LU-19131 utils: 'lfs mirror extend' should use DIO

To reduce memory pressure on the data mover nodes, use O_DIRECT
when opening files.  Otherwise, this can cause issues on OSS
nodes or with highly concurrent data migraton.

Use the same buffer size for all mirror/migrate handling.
Fix lfs_mirror_verify_chunk() to handle a buffer size that
is not an even multiple of the component extent size.

Name the llapi_migration_flags enum and use it in the callers.
