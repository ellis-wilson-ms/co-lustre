Nothing wrong with the patch itself, but with the automatic "ONLY_REPEAT" these modified subtests are running multiple times in a loop, and failing on the second iteration because of the leftover test directory. 

Normally these are cleaned up automatically by the run_one() test wrapper, but these conf-sanity subtests are finishing with the filesystem unmounted, so that is not possible. Instead, it looks like test_35 and test_38 need to clean up the test directory as part of the test (either at the start or end).
