LU-16661 build: improve lustre.spec.in Requires

Add Suggests: bash-completion for lustre-client and lustre for
lctl and lfs sub-command completion.

Move perl from Requires to Recommends, since there are only some
uncommonly used tools (llstat, llobdstat) that are using perl.
Remove a couple of ancient obsolete test scripts that used perl.

lustre-iokit incorrectly Required perl instead of python3.

Set minimum kernel version for client to be 3.10 or later.

Change "netstat" to "ss" in tests to avoid dependency issues.
Fix sanity.sh and conf-sanity.sh tests for sles12sp5 issues.

(1 comment)

> But if /usr/bin/netstat isn't available, the tests will fail.

SLES12 should continue to work without "Requires: netstat" as it always had before the .spec changes, since netstat was already installed on SLES12 test nodes and the tests were working fine.

 > But may it doesn't really matter.  SLE12 is getting a bit old
 > and while it is still supported, we don't need to bend over
 > backwards to make it perfect.

Exactly my thought, ignore the Requires for SLES12 and it should "just work", and it can be handled "properly" for newer distros.

 > netstat is in net-tools-deprecated.  So that needs to be
 > installed or there is no point installing lustre-tests ...

Or we could not Require: netstat and skip_env the few subtests that actually need netstat, and hope that the functionality does break on the distros that do not have it.

That is the same situation with perl.  There are TWO subtests that need perl and it would be no big loss if they were skipped instead of refusing to install lustre-tests, until such a time that those scripts can be rewritten to use bash or python.

 > Maybe we should switch the tests to use "ss" instead of
 > "netstat" - it is the approved and supported replacement.
 > Still - switching to ss might be a good idea. SLE12 has it.

I can never understand the motivation to replace "standard" Unix tools with a plethora of newer alternatives, versus adding some incremental functionality to existing tools as needed. 

Essentially, every time that netstat or ifconfig or sysvinit or whatever gets replaced on SLES or RHEL or Ubuntu with a custom tool, thousands or millions of person-years of common knowledge, experience, and documentation is discarded and needs re-learning, and scripts need conditionals for the distro and version instead of using common tools that have existed since the dawn of time.
 
Those are just musings of a crotchety old Unix nerd I guess.  Those pesky kids should get off my lawn... :-)
