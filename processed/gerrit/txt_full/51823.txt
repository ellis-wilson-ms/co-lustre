LU-17005 obdclass: allow stats header to be disabled

Add a global "enable_stats_header" tunable parameter that can be
set to enable/disable the "start_time" and "elapsed_time" fields
in the standard lprocfs "stats" files.

Default to enabled, since this landed shortly after v2_14_0.

--- Message (Andreas Dilger, 2023-08-21 19:09:01.000000000) ---

> Sorry I mean that this affects ALL stats.

I still don't understand what you are getting at. 

Yes, this new parameter enables/disabled the new "start_time" and "elapsed_time" field in the header of the stats files, but it defaults to "on" so the net result is a no-op compared to 2.15 when these fields were first added. 

However, we had problems with old collectd scrapers because of the new fields in the header, so it makes sense to disable them in all of the places they were added. I'm not sure what else you are expecting here?  I'm not going to add a separate tunable for each stats file to allow enabling and disabling theses fields. This is intended as a temporary workaround on systems until collectd is updated to a newer version (which supports these fields).
