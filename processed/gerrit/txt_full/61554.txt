LU-19426 nodemap: check parent when setting fileset

When a parent nodemap only has alternate filesets set, then child
dynamic nodemaps can set any primary fileset even if it is outside the
parent's namespace restrictions by using "lctl nodemap_set_fileset".

This patch fixes this by repositioning "check_fileset_add_vs_parent()"
when setting a fileset to stay within namespace defined by the
parent's filesets.

Fixes: e8c244e81e ("LU-17431 nodemap: fileset inheritance for dynamic nodemap")
