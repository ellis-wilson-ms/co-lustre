LU-12187 lod: fix write intent for EC files with seek

When writing to an EC file at an offset (e.g., with
dd seek=1), the write was failing with -EALREADY because
lod_declare_update_plain() was not recognizing that
marking parity components as stale is a layout change.

The issue occurred because:
1. EC files start in RDONLY state
2. On write, lod_declare_update_rdonly() transitions to
   WRITE_PENDING and calls lod_declare_update_plain()
3. lod_declare_update_plain() marks parity components as
   stale but doesn't set layout_changed=true
4. With no layout_changed, it returns -EALREADY

Fix by setting layout_changed=true when marking parity
components stale for EC files. This ensures the state
machine progresses correctly: RDONLY to WRITE_PENDING
with stale parity.

Also fixes:
- Duplicate test_2a in sanity-ec-flr.sh (renamed to
  test_2b)
- Test 5a to use $TRUNCATE instead of system truncate
  command
- Test 6a to match actual extent output format
