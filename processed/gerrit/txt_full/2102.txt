--- Commit Message ---
LU-1071 lnet: add config file support for routes and ip2nets params.

Large network requires many config line for routes, but the module
param is limited to 1024 bytes. This patch resolves this limitation.
The routes and ip2nets module parameters accept absolute path names,
the old syntax is supported as well. Example:
routes="/etc/lnet_routes.conf"

--- Message (Isaac Huang, 2012-09-25 00:39:59.000000000) ---
Patch Set 7: I would prefer that you didn't submit this

I believe it belongs to the user space to read and parse config files, rather than the kernel. If I haven't missed something the problems mentioned here could all be solved in the user space:
- ip2nets: Write a script that reads the ip2nets config file and generates a proper "networks=" option in module.conf. This script needs to run for only once on each node unless the config file or the host interfaces changes.
- routes: Write a script that reads the routes config file and adds the routes by the 'lctl add_route' command, after Lustre modules have been loaded into the kernel but before any services gets started or FS gets mounted.

Moreover, once the dynamic lnet configuration feature has landed, these scripts can be easily adapted to use the new interface to configure lnet interfaces and routes. I've seen this approach been used in deployments with complex networks where there's not sufficient room in the kernel module options.

I don't want the kernel to read and parse files unless it can't be done from the user space or it's even messier to do it from the user space, which is not the case here as far as I could see.
