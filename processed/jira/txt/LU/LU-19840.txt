--- Description ---
The current lfs quota output uses fixed-width columns and implicit formatting, which can break automation and parsing tools. When a filesystem mountpoint name exceeds a certain length, the output wraps to a new line, causing quota data to split across multiple lines instead of remaining in a single structured row.

 

This behavior introduces ambiguity and parsing failures for scripts and tools that rely on consistent formatting.

 

Providing a machine-readable format(e.g., JSON, CSV, or YAML) would ensure structured, unambiguous output, improving reliability for automation and tooling.
