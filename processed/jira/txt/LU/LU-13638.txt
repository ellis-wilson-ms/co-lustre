--- Comment (ys, Wed, 3 Sep 2025 08:36:20 +0000) ---
Hi, Chris,

The request out callback could be arrived after reply in callback.

  rq_wait_ctx = 0,
  rq_early = 0,
  rq_req_unlinked = 0,
  rq_reply_unlinked = 1,

The situation can be divided in two cases. One case is the ACK just later arrived. So nothing need to be done. Second case is ACK was lost. The request will be blocked in ptlrpc layer then consistent thread will wait forever. I was thinking to resolve it in LNET layer, but operation is undefined for O2IB. We cannot guarantee o2ib driver doesn't touch the md in such case. The latest patch intents to keep request buffer until lustre shutdown the network. So that we can continue to process the request when reply has been received. Rather than waiting request out ACK pointlessly.
