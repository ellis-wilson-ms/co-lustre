--- Description ---
lnetctl import fails when importing IPv6 configurations, while adding IPv6 NIDs via the `lnetctl net add` command line succeeds. 

Note that `lnetctl import` works correctly for IPv4 configurations.

Environment Information:

# ip addr show dev ens38
4: ens38:  mtu 1500 qdisc fq_codel state UP group default qlen 1000
    link/ether 00:0c:29:dc:8c:00 brd ff:ff:ff:ff:ff:ff
    altname enp2s6
    inet6 2001::211:2233:4455:203/64 scope global tentative
       valid_lft forever preferred_lft forever 

Test showing lnetctl import IPv6 failure:

# lustre_rmmod
# modprobe lnet
# lnetctl lnet configure
# cat lnet.yaml
net:
-     net type: tcp6
      local NI(s):
      -     nid: 2001::211:2233:4455:203@tcp6
            interfaces:
                  0: ens38
# lnetctl import lnet.yaml
---
add:
-     import:
      errno: -22
      descr: ! "\"couldn't query intf ens38\""
... 

Adding IPv6 NID via lnetctl net add command succeeds:

# lnetctl net add --nid 2001::211:2233:4455:203@tcp6
# lnetctl net show
net:
-     net type: lo
      local NI(s):
      -     nid: 0@lo
            status: up
-     net type: tcp6
      local NI(s):
      -     nid: 2001::211:2233:4455:203@tcp6
            status: up
            interfaces:
                  0: ens38
