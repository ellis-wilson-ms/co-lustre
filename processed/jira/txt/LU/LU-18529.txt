--- Description ---
This ticket is for a possible feature called "file level SOM", which would use the file level replication layout IO control mechanisms to allow strict SOM for files not being actively written.

This would apply the SOM after last close, using the FLR mechanisms to avoid the issues with evicted clients which normally prevent safe use of SOM.

It's not clear if this would offer enough performance benefit to be worth the engineering effort, so the first step will be some benchmarking.
