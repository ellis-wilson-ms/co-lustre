--- Description ---
The canmount property should normally be disabled on a lustre zfs dataset to prevent it from being mounted on import. However if canmount gets enabled for some reason mount.lustre will likely fail with EBUSY becase the dataset will get mounted through the ZPL before lustre starts. The tool could print a helpful message in this case like "is the dataset already mounted?"  Perhaps it could also print a warning if the canmount property is enabled to help the system administrator correct the source of the problem.
