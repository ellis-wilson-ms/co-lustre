--- Description ---
sanity-sec:31 fails with error of:

sanity-sec test_31: @@@@@@ FAIL: unable to configure NID on tcp999 for node test01

  Trace dump:

  = /usr/lib64/lustre/tests/test-framework.sh:6646:error()

  = /usr/lib64/lustre/tests/sanity-sec.sh:2561:test_31()

  = /usr/lib64/lustre/tests/test-framework.sh:6981:run_one()

  = /usr/lib64/lustre/tests/test-framework.sh:7031:run_one_logged()

  = /usr/lib64/lustre/tests/test-framework.sh:6869:run_test()

  = /usr/lib64/lustre/tests/sanity-sec.sh:2647:main()

The reason is, Lnet has not been configured yet when the test script tries to run command "/usr/sbin/lnetctl net add --if eth1 --net tcp999".

Maloo does not hit this problem, probably because "options lnet config_on_load=1" is configured for Lnet module.
