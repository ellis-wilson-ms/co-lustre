--- Description ---
The purpose of this new option

--comp-flags-orig=FLAGS

is to set flags on a file which the new mirror is originated from. Currently we can set the flags on the mirrored file, but sometimes we need symmetrically to set flag(s) on the original file. One example is Hot-pools, where files are created in fast pool have to set 'prefer' flag immediately after mirroring them into cold pool. Currently this is done by spawning another lfs setstripe command. This process is suboptimal and also may be delayed.

--- Comment (adilger, Sat, 16 Mar 2024 08:32:01 +0000) ---
I was thinking about this further, and I'm not sure "--comp-flags-orig" will always be correct (even though I suggested this name originally.

If there are two original mirrors and a new one is being created, which component does the "prefer" go on?  I think another option would be:

lfs mirror extend --comp-set [--mirror-id MIRROR_ID|--comp-id COMP_ID|--pool POOL] --comp-flags=FLAGS -N -E MIRROR_OPTIONS

This allows selecting a mirror/component like lfs mirror split and then setting FLAGS on that component before starting the "-N" mirror options.
