--- Description ---
I've been experimenting with LLMs and Lustre for some time and while various code based reviews don't seem to be any good, you can instruct models to find spelling errors and other text inconsistencies and that is actually decently good, including logic errors no spell checker could approach like "this is a xxx_get_yyy function, but description talks about getting a different zzz value which is an error).

The actual doc finding are tracked in LU-18110 and here I am going to document the setup.

Projects currently exist that allow you to run somewhat reduced quality "open weight" models like llama on consumer hardware. I used ollama which seems to be simplest to setup: https://ollama.com/download

this starts a server on a local port that you can query quite easily.

There is a large selection of models available too in various formats.

In general having a GPU or several with lots of VRAM is best, I have two RTX 4090 that together have almost 50G of VRAM, so any model that fits in that space is quite fast, in particular 4-bit converted llama 3.1 70b fits, gemma2 27b also fits and produces great results.

If you have a GPU definitely pick something that fits your VRAM for fast trying of prompts and such even if you plan to use a more elaborate model for real evaluations.

Even if you don't have a GPU, if you have a lot of RAM, you can run models on CPU (much slower than GPU!) as long as they fit into your RAM. There are also single bit model conversions that take less RAM, but they tend to quickly fall apart according to the various forums, so I have not evaluated that.

 

Here's an example script I used for evaluating manpages (run once per file):

import json
import requests
import sys

code = open(sys.argv[1], "r").read()
#model = "mistral-large"
model = "gemma2:27b"if len(sys.argv) > 2:
    model = sys.argv[2]prompt = """You are an expert proofreader and your job is to review Lustre filesystem documentation.
Read the below man page in groff format for spelling and grammar errors, inconsistencies and poor wording:
```
%s
```

Proofread the above man page and provide your feedback. Be specific and short, recommend suggested fixes for misspellings and grammar errors. Limit every suggestion to 50 words or less. Explain inconsistencies.
Respond using JSON like this: {"spelling":[{"original":..., "fix":...}],"grammar":[{"original":...,"fix":...}],"inconsistency":[{"original":...,"fix":...}],"wording":[{"original":...,"fix":...}]}
"""

payload = {
"model":model,
"stream":False,
"format":"json",
#"raw":True,
"options":{"num_ctx": 8192, "temperature": 0.6},
"prompt": prompt % code
}

#print(json.dumps(payload))

reply = requests.post("http://localhost:11434/api/generate", data=json.dumps(payload), headers={'Content-type': 'application/json'})
if reply.status_code == requests.codes.ok:
    res = reply.json()['response']
    try:
        print(json.dumps(json.loads(res), indent=2))
    except json.decoder.JSONDecodeError:
        print("json decoding failed, here's the raw response:")
        print(res)else:
    print(reply.text)

I imagine the prompt could be adjusted to also ask llm to review code, but not for quality, just only the comment and text strings.

 

I also have a separate version of this script that reviews commit comments to find spelling errors and such and one that reviews diffs to manual pages.
