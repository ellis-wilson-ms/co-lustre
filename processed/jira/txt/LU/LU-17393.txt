--- Description ---
Found during code review of LFSCK, it seems only to recreate the LAST_ID file on OSTs and not recreate LAST_ID on MDTs.  This should be checked during "initial scrub" when the filesystem is mounted. 

The LAST_ID file could also be recreated when finding the quota index file, but it will set the content of the file to be the first OID incorrectly. It should also verify and repair the LMA of the LAST_ID file.
