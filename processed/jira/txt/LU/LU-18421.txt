--- Description ---
When I execute the following command to create a DoM file with specific striping settings:

lfs setstripe -E 128k -L mdt -E 2M -c 1 -E 4M -c 2 -E -1 -c 4

I notice that the stripe size for components beyond the component 0 defaults to 128k, which I am not sure (on behavior). Usually, if I create striping without specifying the -s option, the stripe size defaults to 1MB.

My question is whether DoM files should also default to a 1MB stripe size for unspecified components. For instance, in the above example, should Component 0 be 128k, and the rest follow the 1MB default?

Below is the output detailing this behavior. Is this a bug or it should be a document change explaining the behavior

I recall that stripe size of second component should be aligned with DOM size at least. So first I thought that probably next component stripe depends on that somehow, so I set its stripe size manually to greater value and lesser one:

# lfs setstripe -E 128k -L mdt -E 2M -S 1M -c 1 -E 4M -c 2 -E -1 -c 4 /mnt/lustre/test2
# lfs setstripe -E 128k -L mdt -E 2M -S 64k -c 1 -E 4M -c 2 -E -1 -c 4 /mnt/lustre/test3

both commands were successful and all components starting with second have stripe size as 1M and 64k respectively. So it looks like 'lfs' issue which uses last set stripe size value as default one

It is not connected with DoM also, it inherits just any last declared stripe size:

all stripe sizes are 64k in all components

--- Comment (adilger, Mon, 4 Nov 2024 20:02:39 +0000) ---
rajeevm, thanks for raising this question.  I suspect that this is "unintended consequences" for the combined action a number of patches that landed over time.  Can you please fill in the fields for the affected version, and check if this behaviour has changed between releases.

Setting the DoM stripe_size = component size makes sense by itself, and normally layout parameters are inherited from one component to the next (i.e. stripe_size, stripe_count, pool), so this also makes sense in isolation.  I do not think it makes sense to inherit a small (or possibly any?) stripe size from the DoM component to the rest of the components.  The same situation might happen with e.g. a very small flash OST component (e.g. "-E 256K --pool flash -E 1G --pool hdd" as we have seen at some sites with very small hybrid flash pools).

It might make sense to avoid the stripe_size inheritance to the next component when stripe_size == component_size?  I was wondering if that should only apply to the case where stripe_size  or not, but if the DoM component is large (e.g. 16M), then we may not want to inherit a 16MB stripe_size to the rest of the file?

yujian, bobijam, tappro, any thoughts on what should be done here?
