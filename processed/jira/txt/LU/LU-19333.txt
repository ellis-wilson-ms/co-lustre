--- Description ---
Environment: Ubuntu 20.04 running 5.4.0-1145-azure-fips on the server-side (which is where this occurs).  Very large system with billions of files in total.  There are a few directories with high-tens-of-millions of files, but most directories are bucketized such that there are The problem observed is a stall on one of the six MDS nodes (one MDT per node) with that nodes CPU going to 100%.  Operations stop across all server nodes, but that is very likely due to client load piling up stuck on the one MDS that is stalled.  During the CPU spin the kernel drops messages of the following form:

Current suspicion is that the three especially huge directories (approaching 100M entries each) are causing directory entry thrashing, but that's speculative.  The MDS nodes have >100GB of memory and are not near OOM.  The stall with the high CPU usage lasts tens of minutes, but less than an hour.

Advice on further data collection to get or any tunables we can enable to help pinpoint the directory in question when this hits would be greatly appreciated.
