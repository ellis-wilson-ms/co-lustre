--- Description ---
By default (and in autotest) sanity-hsm runs the copytool on $CLIENT2 using $MOUNT. There are several tests in sanity-hsm that also use this mount point (12, 30a, 30b, 57). On a single node run (which nobody cares about but me ) the copytool doesn't have node defined and if you run it on the single node then the CT uses the same mount point as everything. It's find to run the CT on CLIENT1 or CLIENT2 it should just run on a new and obscure mount point. By obscure I mean not $MOUNT and not $MOUNT2.
