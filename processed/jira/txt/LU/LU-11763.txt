--- Description ---
Currently, a network interface in modparam.conf is not necessary to be provided to configure LNet using lctl command - "lctl network configure" or "lctl network up". In this case, a default value ("eth0" for socklnd or "ib0" for o2iblnd) is used to configure network.

With the LNet sysfs implementation, a network interface is necessary to build the sysfs hierarchy. lnetctl utility already has this restriction. 

 This is why it has been decided to impose this restriction with lctl utility as well. Adding a warning message stating this restriction before we implement it will help users to be aware of it beforehand.
