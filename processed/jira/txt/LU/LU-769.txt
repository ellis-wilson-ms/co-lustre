                 * Request in-flight?

Then the current state of this RPC is in RQ_PHASE_RPC state and rq_waiting is 1, so this statment will be true, so continue, so 0 will be returned by ptlrpc_set_next_timeout().

Then take a look at: ptlrpc_set_wait(), it will go into:
