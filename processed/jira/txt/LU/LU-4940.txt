--- Description ---
Currently, the LFSCK fix inconsistent OST-objects via OSP, and if the connection (or the peer) has trouble, related repairing request will return "-ENOTCONN" immediately, then the LFSCK will mark the LFSCK as LF_INCOMPLETE, and move to handle next one.

Such model can make the LFSCK to process the inconsistent OST-objects ASAP, but it may cause the LF_INCOMPLETE to be marked too easy, which may be not the best choice. We can consider to make the LFSCK to wait for a while for the connection or peer to recover.
