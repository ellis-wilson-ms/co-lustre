--- Comment (jay, Tue, 31 Jul 2012 16:17:36 +0000) ---
LU-1442 can definitely cause this issue, let me explain here:

1. from the description of lu-1442, one page can be wrongly marked as Dirty but not be put into OSC's write cache;

2. when the lock is canceled, we only cleanup the region of osc write cache covered by the lock, however, the page is not in osc write cache as I stated in 1;

3. when the page is discarded, we verify that there is no dirty page in the lock region by checking vmpage dirty flag. This will hit assertion.

As a conclusion, I think these are different symptoms of the same issue.
