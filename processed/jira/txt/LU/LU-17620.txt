--- Comment (bzzz, Wed, 3 Apr 2024 14:47:47 +0000) ---
layout refresh is a two-step process: 1) release current configuration with all objects 2) re-initialize a new one. the problem is that group locks are referenced by the objects used in the current configuration, meaning we can't release them technically.

but I guess the most interesting question is what kind of layout changes are acceptable while a grouplock is taken? I guess one more (stale) component should be OK, but what if it changes the components involved in migration itself? say, it just marks the original component stale?

--- Comment (bzzz, Thu, 14 Mar 2024 08:44:39 +0000) ---
only lfs migrate is affected, lfs mirror extend/resync don't use group locks so safe.
