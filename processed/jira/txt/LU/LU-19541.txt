--- Description ---
"lctl set_param -P debug" does not work as expected

[root@vm00 ~]# lctl set_param debug_mb=4096 debug=+vfstrace+dlmtrace+rpctrace
debug_mb=4096
debug=+vfstrace+dlmtrace+rpctrace
[root@vm00 ~]#
[root@vm00 ~]# lctl get_param debug
debug=ioctl neterror warning dlmtrace error emerg ha rpctrace vfstrace config console lfsck
[root@vm00 ~]# lctl get_param debug_mb
debug_mb=4081

--- Comment (emoly.liu, Thu, 30 Oct 2025 08:21:54 +0000) ---
After looking into the code, I know why "debug" and "debug_mb" can't be set with -P by lctl set_param, because they are LNET parameters, not Lustre system parameters.

# find /sys/ -name debug_mb
/sys/kernel/debug/lnet/debug_mb

These parameters of the libcfs kernel module are exposed via symlinks in /sys/kernel/debug/lnet/, not managed by MGS, at a lower layer than Lustre. So, we need to set them at module load time, for example:

modprobe libcfs libcfs_debug_mb=4096

This is not a bug.
