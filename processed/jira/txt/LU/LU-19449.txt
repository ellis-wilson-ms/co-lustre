--- Comment (sebastien, Mon, 22 Dec 2025 08:51:09 +0000) ---
The error comes from this code that is called in sanity-sec test_43 cleanup (cleanup_for_enc_tests -> remount_client_normally -> remove_enc_key):

remove_enc_key() {
	local dummy_key

	$LCTL set_param -n ldlm.namespaces.*.lru_size=clear         /proc/sys/vm/drop_caches
	dummy_key=$(keyctl show | awk '$7 ~ "^fscrypt:" {print $1}')
	if [ -n "$dummy_key" ]; then
		keyctl revoke $dummy_key
		keyctl reap
	fi
}

The first occurrence of 'test_43 returned 2' I could find dates back from 2025-10-04. It does not correspond to any significant change in this cleanup code. So it looks like we can end up with 'leftovers' under /sys/fs/lustre/ldlm/namespaces/ after unmounting then remounting clients. Not sure where this is coming from, but to avoid failing tests because of that I have pushed patch "LU-19449 tests: ignore lru_size error in test cleanup" at https://review.whamcloud.com/63135 .
