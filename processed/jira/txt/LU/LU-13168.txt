--- Description ---
We are using 2.12.3_93_gb75f04d-1 on clients to fix a panic when deleting files and using data on MDT (oe_start >= start && ext->oe_end LU-12462). This has resolved the panic on deleting files, however we are now experiencing 2-3 kernel panics a day between our 6 cluster login machines.

We do not yet know what is triggering these, however they all start from with either a kmalloc-8 freechain corrupt or kmalloc-8 Redzone overwritten, I've reproduced samples of both the vmcore-dmesg file generated by kdump/, this looks similar to me to LU-12581
