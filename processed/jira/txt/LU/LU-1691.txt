--- Comment (chris, Wed, 26 Sep 2012 06:05:38 +0000) ---
Peter and I discussed this and decided that the best plan is for Lustre to be able to support SLES11SP1 and SLES11SP2, this will maintain backward compatibility whilst allowing a transition to the later kernels of SP2.

To do this I'm going to revert the changes in

lustre/kernel_patches/targets/2.6-sles11.target.in

and add a new 

lustre/kernel_patches/targets/3.0-sles11.target.in

We will also need to make a small changes to lbuild and funcs to allow selection between these versions.

By supplying TARGET to lbuild the use should be able to select 2.6 or 3.0 but by default the build will build to the version of the kernel on which lbuild is being executed.
