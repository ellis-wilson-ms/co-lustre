--- Description ---
Although I haven't verify this by looking into a target from a PPC server somewhere, the b1_8, master, and Orion code at the time of writing seems to write and read CATALOGS and lov_objid records in host byte orders.  This prevents users from starting, for example, targets created by PPC servers on x86_64 ones.  Also, a thorough search for similar bugs would be desirable.
