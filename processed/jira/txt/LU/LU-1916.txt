--- Comment (niu, Fri, 22 Mar 2013 02:40:23 +0000) ---
Could you provide the output of "lfs quota -u  cenewma1 -v  /nobackupp1"? lfsck can show the orphan objects and fix them, but I think orphan will be cleared once OST restart and connect to mds, so usually it's not necessary to run lfsck.

--- Comment (niu, Fri, 14 Sep 2012 01:33:15 +0000) ---
I don't think we have any tools to determine which OST has orphan objects.

When some client unlink files, it unlink namespace on MDS first, then issue objects destroy to OSTs, if the client crash (or some other failure) between these two steps, there will be orphan objects on OSTs.

I don't think we need to do a lfsck, the orhpan objects will be cleared automatically once the OSTs restart.
