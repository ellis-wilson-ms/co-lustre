--- Description ---
Explanation from email:

“ rwid=2 is the attach ID, each PCC backend has an attach ID to Uniquely identify itself on the client. For PCC-RW, it is usually same as the archive ID of the copy tool running on the client. The PCC-RO attach ID (roid) is same as PCC-RW attach id (archive ID) by default. “

So there is 2 cases with HSM enabled and without HSM.

	Is it correct that  HSM acrhive ID  will be assigned by copytool?

Second case need additional explanation for me.

	How to choose archive ID without HSM

	What is possible range for archive ID?

	Does user need to know anything else about archive ID? How it used internally?

	It is still unclear for me should be archive  ID the same on all client nodes or it should be different for PCC RO?

--- Comment (qian_wc, Fri, 7 Aug 2020 06:50:19 +0000) ---
"Is it correct that  HSM acrhive ID  will be assigned by copytool?"

Yes, the archive ID will be assigned and set when launch lhsm_posix copytool. It is used to uniquely identify the HSM backend.

Second case need additional explanation for me.

"How to choose archive ID without HSM"

If a PCC backend is used for PCC-RW, then the RW attach ID (rwid) must be same as the archive ID of the copytool running on this client. As archive ID is used to uniquely identify the HSM backend in cluster wide, the archive IDs are usually not same. 

If a PCC backend only provides PCC-RO caching services without HSM, then the RO attach ID (roid) only needs to be unique on the client to identify the HSM backend. Different clients can use same RO attach ID.

"What is possible range for archive ID?"

archive ID in HSM is a 64 bit value, It is large enough, I think.

"It is still unclear for me should be archive  ID the same on all client nodes or it should be different for PCC RO?"

For PCC-RO, the attach ID can be same on all client nodes if each client only has one PCC backend.

But if a client has config two PCC-RO backend, then the attach ID for each PCC-RO backend on this client must be different and unique to identify PCC-RO backend on this client.
