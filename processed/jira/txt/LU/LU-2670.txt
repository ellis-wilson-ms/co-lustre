--- Description ---
We would like to export multi-rail Lustre through routers to one destination network. It looks like it is not allowed in current implementation.

My lnet module option is:

routes="gni0 xxx.xxx.110.xxx@tcp0 xxx.xxx.111.xxx@tcp1" 

I get this error:

LustreError: 5598:0:(router.c:399:lnet_check_routes()) Routes to gni via xxx.xxx.111.xxx@tcp1 and xxx.xxx.110.xxx@tcp not supported

The manual states (p301): 

"It is an error to specify routes to the same destination with routers on different local networks."

It will be very great if this limitation can be removed.
