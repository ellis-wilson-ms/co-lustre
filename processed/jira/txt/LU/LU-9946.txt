--- Description ---
We would like to be able to impose subdirectory mount for clients in the default nodemap.

Initial implementation done in LU-7846 was considering default nodemap should not have fileset information, but there are some use cases where it makes sense to define one.

For instance, it enables restricting mount from clients that do not pertain to any explicitly defined nodemap entry, to a subdirectory with very limited access and content.

It is also a convenient way to prevent mount from such clients: just define fileset on the default nodemap as a directory that does not exist.

I will post a patch with my proposal.

Thanks,

Sebastien.
