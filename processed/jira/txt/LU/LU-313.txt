--- Description ---
The current lustre/tests/lfsck.sh test script is being run by Jenkins/auster/whatever, but in actuality the script is not running the "lfsck" part of the test because SKIP_LFSCK=yes is always set in test-framework.sh due to the old bug 13698.  However, this bug should be fixed 

Also, the lfsck.sh script itself should be changed so that it checks for "SKIP_LFSCK == yes" rather than "SKIP_LFSCK != no" so that without any external specification otherwise it will properly run the distributed lfsck part of the test.
