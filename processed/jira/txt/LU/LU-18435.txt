--- Description ---
when layout creation (open/create) is being replayed it resets layout generation to 0 and this later prevent layout refresh on the client side as the client wants the layout to increase update an update.

this way the client can get into a loop trying to refresh a layout. in write(2) this repeats 1000 times and then the write gives up returning 0 bytes written.
