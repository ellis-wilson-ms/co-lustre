--- Description ---
*at least on a single node setup

On my single node setup, sanity-hsm runs in about 30 minutes, which is too much. Most of the time it is cheaper to identify the tests a patch is most likely to break and only run those rather than run the entire test suite. Ideally tests would run almost instantly, but that is probably too much to ask from a single LU.

From the result.yml file in the test logs: 80% of the time is spent in tests that complete in 10 seconds or more. Assuming each of those tests can be sped up to run under 2 seconds, testing time can be shorten to the 5 minutes advertised in the title.
