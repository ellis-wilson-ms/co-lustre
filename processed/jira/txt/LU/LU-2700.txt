This problem might be related with the recent landing about rename lock

mdt_reint_rename()

   if (!lu_fid_eq(rr->rr_fid1, rr->rr_fid2)) {

                rc = mdt_rename_lock(info, &rename_lh);

                if (rc) 

{
                        CERROR("Can't lock FS for rename, rc %d\n", rc);
                        RETURN(rc);
                }
        }

   }

We probably should remove this eq check, because even though the target and source are in the same directory, we still need serialize the locking order of these two children to avoid deadlock.  In 2.x, there are no other way to do that, we have to rely on this BFL lock.
