--- Description ---
When a remote file or directory are deleted, the stub pFID object may need to be created on another remote MDT rather than the local MDT which is a distributed transaction.

However, it makes the access on Trash Can via ".Trash" easily. When displaying such remote files or directories to the user via the .Trash or MDTXXXX/pFID directory on the client, it  can directly map the access on ".Trash" to the stub pFID object on Trash Can.
