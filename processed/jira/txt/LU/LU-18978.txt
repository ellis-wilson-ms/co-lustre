--- Description ---
With more inflow of new people into Lustre outside of our traditional HPC markets we need to make it more accessible.

As recent posts o nteh lustre-discuss show, people find it challenging to build Lustre esp. using outdated materials on lustre.org wiki

As such we should add repositories for common distros we expect Lustre users to use like rhel/rocku 8.10 and rhel/rocky/9.6 and the like (and some ubuntu?)

Nvidia drivers/cuds is a good example here, they have a repo you just install and then enable and then just "dnf install lustre", reboot and you have a fully functioning system.

See their example here: https://docs.nvidia.com/datacenter/tesla/driver-installation-guide/index.html#network-repository-installation

Once that is in place we should then update the docs to refer to this new easier method

Also for fedora, if we ever decide t osupport that, we could register a copr and then use that for our packages.
