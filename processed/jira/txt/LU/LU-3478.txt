--- Description ---
This is the continuation of work done in LU-2043.

Similarly to obdfilter-survey scripts,

	the output of sgpdd-survey should be fixed to display minimal and maximal bandwidth of sgp_dd instances. This allows to detect if "low" LUNs are present which is critical to get optimal storage performance.

	the plot-sgpdd script should be reorganize (and simplified) to handle read, write, or both operations.

Additionally, the sgpdd-survey script could be enhanced to support NUMA servers. Each sgp_dd instance would need to be binded to the cores local to the device it works on.

I am going to provides patches for these items.
