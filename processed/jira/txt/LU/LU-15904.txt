--- Description ---
Currently, the object on OSD level maps to an inode on Ldiskfs/ZFS directly. It might be useful to add a mapping level between object and Ldiskfs/zfs inode(s). The possible use cases of this new mechanism might be adding features that need to change the layout of file data, like snapshot/reflink/de-duplication.
