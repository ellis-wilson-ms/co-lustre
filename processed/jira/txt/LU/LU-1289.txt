--- Description ---
The module version string printed by Lustre modules is completely useless, usually something like "0.1.0" or similar.

Use the LUSTRE_VERSION_STRING to set the module version to the current Lustre version.  This is OK for the Lustre modules, but some work is needed to get a version string for libcfs/lnet modules, because they should technically not use any code from lustre/.  Probably moving the LUSTRE_VERSION_STRING definition into libcfs would be easiest.
