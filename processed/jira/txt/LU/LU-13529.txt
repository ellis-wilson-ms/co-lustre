--- Description ---
I'm wondering how mkfs.lustre handles --mkfsopts parameters.

We have at least -i -I -N options.

-i 

-I 

-N 

1. How mkfs.lustre behaves if multiple options specified simultaniously , for example: mkfs.lustre --mkfsoptions='-i 131072 -N 1073741824 -I 2048'

2.If mkfs.lustre still handles paramtere when option and value are not space-delimited, for example: --mkfsoptions='-i131072 -N1073741824 -I2048'

--- Comment (adilger, Sat, 9 May 2020 12:15:01 +0000) ---
Using -N is preferred over -i. Using "-I" is independent of the other options, unless it results in an invalid filesystem (eg. -i  device size). 

I don't think it matters if there is a space between the argument and the value or not.
