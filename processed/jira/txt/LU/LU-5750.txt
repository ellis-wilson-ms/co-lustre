--- Description ---
Module parameter lu_cache_nr is always overwritten by lu_htable_order(), so only the default can be used:

[isaac@oyster]% sudo modprobe obdclass lu_cache_nr=-1
[isaac@oyster]% cat /sys/module/obdclass/parameters/lu_cache_nr
-1
[isaac@oyster]% sudo mount -t lustre test/test /mnt/lustre
[isaac@oyster]% cat /sys/module/obdclass/parameters/lu_cache_nr
10240

--- Comment (isaac, Sat, 18 Oct 2014 02:51:34 +0000) ---
It seemed that with the code above, if user sets lu_cache_nr=LU_CACHE_NR_UNLIMITED (e.g. to work around LU-5747), it'd still get overwritten as LU_CACHE_NR_ZFS_LIMIT.

--- Comment (adilger, Fri, 17 Oct 2014 18:01:34 +0000) ---
Looks like this would be enough:

        if (strcmp(top->ld_type->ldt_name, LUSTRE_OSD_ZFS_NAME) == 0) {
