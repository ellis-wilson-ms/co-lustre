--- Comment (morrone, Thu, 15 Mar 2012 21:06:23 +0000) ---
Your approach in http://review.whamcloud.com/1330 of adding new code, rather than compiler directives like Ned used in LU-251, means that you risk introducing new failures:

/g/g0/morrone/src/lustre-bgq/build/conftest.c: In function 'main':
/g/g0/morrone/src/lustre-bgq/build/conftest.c:118: error: implicit declaration of function 'memset'
/g/g0/morrone/src/lustre-bgq/build/conftest.c:118: warning: incompatible implicit declaration of built-in function 'memset'

The above is from the "checking fs_struct use path structure" test, which really should not be doing a memset in the first place.

I think you really need to have a code comment in the test as well any time you are doing something just to work around a compiler warning.  Otherwise the extra code just obfuscates the purpose of the autoconf test.

The 

"__attribute__ ((unused))"

 compiler directive seems pretty self-explanatory to me in contrast.
