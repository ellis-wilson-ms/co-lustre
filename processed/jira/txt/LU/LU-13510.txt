--- Description ---
The commit https://review.whamcloud.com/32770 ("LU-9120 lnet: calculate the lnd timeout") made it so the LND timeouts for ko2iblnd and ksocknal are derived from the lnet_transaction_timeout and retry_count. Thus, they can no longer be set independently. We should restore the ability to set these timeouts via their respective parameters (ko2iblnd timeout; ksocklnd sock_timeout).
