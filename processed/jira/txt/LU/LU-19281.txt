--- Description ---
The ISL library derived code found in lustre/ec Is out of date and does not provide Optimized versions for our two main architectures, x86 and ARM.  This should be replaced with the current version and we should include the optimized versions of the key functions.

It also needs to be possible to build our ISL derived library in both user space and in the kernel, because we will use it from the LFS tool and from the lustre client.

This patch shows how that can be done:

https://review.whamcloud.com/c/fs/lustre-release/+/59485
