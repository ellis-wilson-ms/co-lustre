--- Description ---
If the LNet router is configured to connect an IPv4-based TCP lnet with an IPv6-based TCP lnet, communication between IPv4 and IPv6 endpoints will not work.

C  Router B  A 

The nodes can add the correct routes and successfully ping the router, so the route state is listed as "up" on both sides. However, "lnetctl ping" across the router times out.

Router B shows logs indicating that the IPv6 NID is being truncated. For example, 2006::211:2233:4455:202@tcp6 is truncated to 32.6.0.0@tcp6.

--- Comment (JIRAUSER20209, Mon, 6 Apr 2026 16:21:39 +0000) ---
The issue might be that Router B and Node C negotiated a V2 version connection, and when forwarding IPv6 messages, the message is mapped from the V4 format to V2, causing NID truncation.

Hi, simmonsja ,

The configuration does not use `lnetctl import`. The full setup is as follows:

We can use the following topology for testing:

C  Router B  A 

Node A

lustre_rmmod
modprobe lnet
lnetctl lnet configure

# Configure IP
ip -6 addr add 2006::211:2233:4455:201/64 dev ens38
ip addr show dev ens38

# Configure LNet
lnetctl net add --net tcp6 --if ens38
lnetctl net show

# Add LNet route
lnetctl route add --net tcp --gateway 2006::211:2233:4455:202@tcp6 --hop 1 --priority 0
lnetctl route show 

Router B

lustre_rmmod
modprobe lnet
lnetctl lnet configure

# Configure IPs
ip -6 addr add 2006::211:2233:4455:202/64 dev ens38
ip addr show dev ens38

ip addr add 10.0.2.232/22 dev ens37
ip addr show dev ens37

# Configure both networks
lnetctl net add --net tcp6 --if ens38
lnetctl net add --net tcp  --if ens37

# Enable routing
lnetctl set routing 1

# Verify configuration
lnetctl net show
lnetctl route show
lnetctl peer show 

Node C

lustre_rmmod
modprobe lnet
lnetctl lnet configure

# Configure address
ip addr add 10.0.2.233/22 dev ens37
ip addr show dev ens37

# Configure LNet
lnetctl net add --net tcp --if ens37
lnetctl net show

# Add route to o2ib network
lnetctl route add --net tcp6 --gateway 10.0.2.232@tcp --hop 1 --priority 0
lnetctl route show
