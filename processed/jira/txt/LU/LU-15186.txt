--- Description ---
On kernels that provide global MR we default to using that exclusively even if FMR/FastReg is available. With https://jira.whamcloud.com/browse/LU-15092 this causes an interop issue because FMR/FastReg may be needed to map the higher fragment count. We should change the default map_on_demand to 1 so that FMR/FastReg is used by default. map_on-demand can still be set to 0 if needed.
