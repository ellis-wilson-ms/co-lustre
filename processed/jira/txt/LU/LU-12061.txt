--- Description ---
When a pool is destroyed, it is not removed from existing layouts and layout templates because walking the file system is too expensive.

This is reasonable, but it leads to one odd behavior:

New files created using these old default layouts (ie created in directories where these old layouts are set) will still have the pool name set in their layout.

This doesn't make sense, since the pool is gone and isn't used in their striping.  We should fix this.
