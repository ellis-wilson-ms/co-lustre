--- Comment (johann, Sat, 16 Feb 2013 02:52:14 +0000) ---
To be clear, upgraded filesystems don't have RO_COMPAT_QUOTA by default. The admin has to explicitly run tunefs.lustre --quota to enable it. Once this is done, the admin will indeed have to clear the flag with tune2fs if he wants to downgrade.

As for new filesystems formatted with 2.4, they have the quota feature enabled by default.

Hi, 

During update&downgrade, are you using same linux distro? this error is about quota feature

in "ldiskfs/ldiskfs/ldiskfs.h"

...

#define LDISKFS_FEATURE_RO_COMPAT_QUOTA         0x0100

...

this feature isn't defined in some Linux Distro. e.g. it is defined in RHEL6 but is not in RHEL5.

--- Comment (niu, Tue, 5 Feb 2013 03:56:38 +0000) ---
The 0x100 is quota feature, you'd disable the ext4 quota feature by 'tune2fs -O ^quota' before downgrade.
