--- Description ---
If a client can connect to a server with both @tcp and @o2ib networks to a specific server, then all else being equal (network hops, interface speed) it makes sense for the client to prefer using the @o2ib interface exclusively instead of trying to load-balance across the two network types.  The @o2ib network will have both lower latency as well as lower CPU usage on both the client and server, so using @tcp for some of the traffic will be counter-productive.
