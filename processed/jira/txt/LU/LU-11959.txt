--- Description ---
The 'network' client mount option enables to limit connections from the client to be on the specified network NID.

At the moment, connection to the MGS is out of the scope of this option, which lets a Lustre client use a different LNet network for the mgc.

And when used in a routed environment, the network specified as a restriction has to be the one defined on server side, which is conflicting with the definition given for the 'network' option.

I will push patches to address these two points.
