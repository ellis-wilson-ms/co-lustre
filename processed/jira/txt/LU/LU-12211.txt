--- Comment (adilger, Sat, 20 Apr 2019 22:35:06 +0000) ---
What version of e2fsprogs do you have installed? We recommend using the latest version, which is currently 1.44.5-wc1.

I don't think these messages are harmful, just a new check added since the last time you ran a full e2fsck, or possibly leftovers from the time the filesystem was formatted. 

You can determine which files are affected by running "debugfs -c -R 'ncheck 529673 529713 ...' /dev/XXX" for the listed inode numbers, and/or "debugfs -c -R 'stat  /dev/XXX" to determine the pathname and timestamps/owners of the files. 

Running a fixing e2fsck should be fine.
