--- Description ---
The newer OS (i.e. rhel9.4) is using new CGroup V2 interface and abandoned the CGroup V1.

The memcg testing in sanity/test_411 test cases are still using the old V1 interface.

It results in the. memcg testing on the newer system are skipped:

        local cg_basedir=/sys/fs/cgroup/memory
        # LU-9966
        test -f "$cg_basedir/memory.kmem.limit_in_bytes" ||
                skip "no setup for cgroup"  

This ticket is using to track the testing and performance benchmark with memcg on new CGroup V2 interface on newer version of the OS.
