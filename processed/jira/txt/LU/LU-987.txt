--- Comment (simmonsja, Wed, 18 Jan 2012 08:46:26 +0000) ---
Since gerrit with the git commit hooks hates placing build errors in the comments section I placed them here. If you do make rpms in the ldiskfs directory the error you see is

CONFIGURE_ARGS=$(echo $(eval echo $CONFIGURE_ARGS) | sed -re

's/--with-release=[^ ][^ ]*//'); \

RPMARGS="$RPMARGS --define \"configure_args $CONFIGURE_ARGS\""; \

if ! false; then \

    RPMARGS="$RPMARGS --define \"build_lustre_tests 0\""; \

fi; \

if ! ; then \

    RPMARGS="$RPMARGS --define \"lustre_name lustre-client\""; \

fi; \

echo "Building Lustre RPM with $RPMARGS"; \

eval rpmbuild $RPMARGS -ta lustre-ldiskfs-3.3.0.tar.gz

/bin/sh: -c: line 14: syntax error near unexpected token `;'

/bin/sh: -c: line 14: `if ! ; then \'

make[1]: *** [rpms-real] Error 2

The autoMakefile.am in ldsikfs does not define the BUILD_SERVER flag. 

This is from the autoMakefile in the real-rpms section coming out as

CONFIGURE_ARGS=$$(echo $$(eval echo $$CONFIGURE_ARGS) | sed -re

's/--with-release=[^ ][^ ]*//'); \

RPMARGS="$$RPMARGS --define \"configure_args $$CONFIGURE_ARGS\""; \

if ! $(BUILD_TESTS); then \

    RPMARGS="$$RPMARGS --define \"build_lustre_tests 0\""; \

fi; \

if ! $(BUILD_SERVER); then \

    RPMARGS="$$RPMARGS --define \"lustre_name lustre-client\""; \

fi; \

echo "Building Lustre RPM with $$RPMARGS"; \

eval rpmbuild $$RPMARGS -ta $(distdir).tar.gz

As you can sse no BUILD_SERVER define leads to the above error.
