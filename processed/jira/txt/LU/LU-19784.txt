--- Comment (adilger, Fri, 16 Jan 2026 14:15:06 +0000) ---
nodes could mount the same fs up if they try to mount at the same time and both see EXT4_MMP_SEQ_CLEAN in the beginning?

That is true even if they both wait the same time as well.  What is more important in this case is that they both check after writing the MMP block whether they got back the data they previously wrote.  However, they should have at least some delay between writing and reading, so that one of the two should get back a different value.  Otherwise, there is a tiny chance to have "write A; read A OK; write B; read B OK" succeed in quick succession without actually seeing that the other node updated the MMP block.

What should be happening is "write A; write B; sleep A/B; read A FAIL; read B OK" and B claims ownership of the filesystem, while A refuses to mount it.  It would probably be enough to have the sleep A/B time only a second or two in the "clean" case, enough time to ensure that the initial "clean read + write + re-read" cannot overlap with a second "clean read + write re-read" without one of them seeing the write of the other.

ZFS does this slightly differently, by writing multiple different (über?)blocks in a random order, then immediately reading them all back.  The chance of both nodes writing the same blocks in the same order is very low, so at least some of them would return the other nodes' signature (though this would cause both mounts to fail).

        struct buffer_head *bh = NULL;
        struct mmp_struct *mmp = NULL;
        u32 seq;
        unsigned int mmp_check_interval = le16_to_cpu(es->s_mmp_update_interval);
        unsigned int wait_time = 0;
        int retval;
...
        seq = le32_to_cpu(mmp->mmp_seq);
        if (seq == EXT4_MMP_SEQ_CLEAN)
                goto skip;

        if (seq == EXT4_MMP_SEQ_FSCK) {
                dump_mmp_msg(sb, mmp, "fsck is running on the filesystem");
                retval = -EBUSY;
                goto failed;

        wait_time = min(mmp_check_interval * 2 + 1,
                        mmp_check_interval + 60);
...
skip:
        /*
         * write a new random sequence number.
         */
        seq = mmp_new_seq();
        mmp->mmp_seq = cpu_to_le32(seq); 

        /*
         * On mount / remount we are protected against fs freezing (by s_umount

        retval = write_mmp_block_thawed(sb, bh);

        /*
         * wait for MMP interval and check mmp_seq.

        if (schedule_timeout_interruptible(HZ * wait_time) != 0) {

        retval = read_mmp_block(sb, &bh, mmp_block);
        if (retval)

        mmp = (struct mmp_struct *)(bh->b_data);

        }
