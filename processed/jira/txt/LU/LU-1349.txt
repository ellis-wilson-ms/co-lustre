--- Description ---
For improving the performance of lookup-by-name, FID is stored as part of dirent data in the directory page. Then when lookup the name in the directory page, its FID can be found together, no need to read from related inode LMA.

Unfortunately, such FID in dirent cannot be reserved after MDT restored from file-level backup. So we need to rebuild such FID in dirent to improve the performance after the restoring.
