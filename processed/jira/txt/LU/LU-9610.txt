--- Description ---
LU-7845 introduced a --network option when mounting a client to restrict communication to a specific network. MR could potentially override that, since if we dynamically discover the peer and it has multiple interface on different networks, then MR could will attempt to utilize all those interfaces.

Since there could be multiple interfaces configured under the same network, when the --network option is provided, MR code should restrict message sending only to the interfaces on that network.
