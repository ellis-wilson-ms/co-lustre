--- Comment (bruno.travouillon, Fri, 9 Mar 2018 15:55:16 +0000) ---
We hit the same page allocation failures with mlx5_buf_alloc() for order 8 and order 10. This may be related to this kernel upstream change (also applied to MOFED 4.2.1):

commit 64ffaa2159b752e6c263dc57eaaaed7367d37493
Author: Amir Vadai 
Date:   Thu May 28 22:28:38 2015 +0300

    net/mlx5_core,mlx5_ib: Do not use vmap() on coherent memory

    As David Daney pointed in mlx4_core driver [1], mlx5_core is also
    misusing the DMA-API.

    This patch is removing the code that vmap() memory allocated by
    dma_alloc_coherent().

    After this patch, users of this drivers might fail allocating resources
    on memory fragmeneted systems.  This will be fixed later on.

    [1] - https://patchwork.ozlabs.org/patch/458531/

    CC: David Daney 
    Signed-off-by: Amir Vadai 
    Signed-off-by: David S. Miller 

With previous code, the allocation might be split into multiple pages when the requested size was > PAGE_SIZE * 2. Now, we absolutely need contiguous pages. The commit message is clear about this.
