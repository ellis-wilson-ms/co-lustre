--- Description ---
Currently the LMV of striped directory stored in inode doesn't contain stripe FIDs, and in loading striped directory, it needs to scan subdir entries to get stripe FIDs, and append them to the LMV. Instead, the main body of LMV and stripe FIDs can be put together and save in inode in striped creation, and these FIDs can be appended to stripe LMV as well. This will add redundancy for striped directory layout, which will help LFSCK to verify and fix striped directory upon failure. And this also helps migrating directory LFSCK.
