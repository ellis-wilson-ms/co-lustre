--- Comment (sebastien, Tue, 29 Jun 2021 15:35:19 +0000) ---
In order to address this requirement, we could add the ability to create id maps of a new type projid, in addition to the already existing uid and gid types. That would also require adding a new value to map_mode. Currently it only groks uid_only, gid_only or both, but we could add a projid_only value. Finally, a new property named squash_projid could be used to map all project ID to a default one.

The common case of one projid allowed per nodemap would consist in having a map that just has the projid mapped to itself, the map_mode to projid_only, and deny_unknown set to 1.
