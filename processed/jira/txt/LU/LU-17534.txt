--- Description ---
Andreas's comment during a review(https://review.whamcloud.com/c/fs/lustre-release/+/53796).  Opening an improvement LU so the idea does not get lost.

That makes me wonder if it would be useful for users/scripts to be able to somehow call "lctl get_param llite.$MOUNT.param" and have lctl map that to the actual mount instance internally?  For example, something like:

 # lctl get_param llite./mnt/testfs.pcc

would be mapped internally to be the equivalent of:

 # lctl get_param llite.testfs-ffff9ec607c56678.pcc

The code could key on "./" in the parameter name to do the llapi_getname() call.  That gets tricky if there is a '.' inside the mount path, so it might need to try a few combinations of pathnames.
