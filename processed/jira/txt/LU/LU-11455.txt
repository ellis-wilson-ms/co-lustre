--- Description ---
Hi,

Patch https://review.whamcloud.com/33152 introduced new file named 'lustre/conf/Lustre' (with L at Lustre)

Unfortunately, a file named lustre/conf/lustre (with l at lustre) already existed in Git history. So, on partitions formatted with file systems like MacOS Extended, which is case insensitive by default, Git is confused, saying the file is locally modified whereas it is not.

Would it be possible to rename file 'lustre/conf/Lustre' to something really different, or move it to a different directory?

Alternatively, is there any workaround on MacOS Extended case-insensitive partitions?

Thanks,

Sebastien.
