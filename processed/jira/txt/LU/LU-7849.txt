--- Description ---
We really need to purge the "DLC" and "Dynamic LNet Configuration" terms from the code and from user visible interfaces, and from documentation.  "DLC" was a project to add new functionality, it should not be a name that ever gets embedded anywhere permanent.

For instance, the autoconf function named LN_CONFIG_DLC should not have any reference to "DLC".  The check is whether libyaml exists, and that is all that fuction needs to concern itself with.

The --disable-dlc option should almost certainly have another name as well.  From a user's standpoint, it would seem to primarily be about whether or not the "lnetctl" command gets built, so it might make a great deal more sense to name the option after lnetctl.

In any event, eventually lnet configuration will just by dynamic, and few users will not care that it wasn't in the past, and embedding the DLC project name everywhere will just be an anachronism.
