--- Description ---
In some cases, it's useful to have the ability to start Lustre clients and servers locally using only 0@lo0. Lustre can then be tested without having to setup any networking facilities beforehand.

--- Comment (JIRAUSER18433, Mon, 1 Jul 2024 18:30:36 +0000) ---
This is mostly aimed at QEMU and other local virtual testing environments. If the network gets setup incorrectly, LNet will complain. The traffic will pass over lo0 anyway. So I think it's best to sidestep the issue by avoiding setting up ksocklnd altogether.
