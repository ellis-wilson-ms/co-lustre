--- Description ---
The o2iblnd does not allow sender to overrun receive queue on the receiver - there's a credit mechanism to prevent that. So in theory the o2iblnd should never receive any RNR NAK. Therefore the rnr_retry_count option is not needed at all.

I think:

	rnr_retry_count should be removed as a module option

	Just in case, add some code to issue warnings should RNR NAK is ever received.
