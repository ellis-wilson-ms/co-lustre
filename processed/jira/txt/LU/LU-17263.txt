--- Description ---
The "lfs find -blocks" argument should be in 512-byte units by default to better match the bahevior of "find(1)", to make usage more portable between the two.  If a unit is specified, then this will still use the specified units.

For most cases this change will not be significant as users/scripts typically specify a suffix (e.g. "-blocks 16M"), and this will be unaffected.  At worst, it will find files using 1/2 as much space as before.
