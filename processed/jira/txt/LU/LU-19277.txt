--- Description ---
There are a number of functions in lustre/utils/liblustreapi_fid.c that do not have man pages:

	llapi_fid2path()

	llapi_fid2path_at()

	llapi_fd2fid()

	llapi_get_mdt_index_by_fid()

	llapi_fid_hash()

The existing llapi_path2fid() and llapi_open_by_fid() man pages should also be updated to reference llapi_fd2fid().

--- Comment (JIRAUSER19447, Sun, 31 Aug 2025 09:42:58 +0000) ---
Function llapi_fid2path() returns -EINVAL and -ERANGE.

-EINVAL is returned in case:

	Path give is null pointer or an empty string.

	While parsing fidstr using llapi_fid_parse(), any of the 3 FID fields are zero or strtoull() return EINVAL.

-ERANGE is returned in case:

	While parsing fidstr using llapi_fid_parse(), any of the 3 FID fields are out of range in positive direction.

-EINVAL second point and -ERANGE, both are returned by llapi_fid_parse() which was called inside llapi_fid2path().

I have 2 ways I can include this info in man page:

	Explain the above points in the same detail.

	Document them as wrong syntax for FID string.
