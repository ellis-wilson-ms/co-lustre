--- Environment ---
RHEL5 or SLES11 client

--- Description ---
This is a test script/configuration issue while running recovery-*-scale tests on RHEL5 or SLES11 client. The dbench load description file is installed in "/usr/share/doc/dbench/loadfiles", which is not in the default search path in rundbench script. Although we specified DBENCH_LIB="/usr/share/doc/dbench/loadfiles" to the test script, the value of this variable was not passed to run_dbench.sh in start_client_load().

Running recovery-*-scale tests on RHEL6 client did not hit this issue because the dbench load file was installed in "/usr/share/dbench", which was in the default search path in rundbench script.
