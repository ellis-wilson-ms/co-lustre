--- Description ---
We recently saw what looks like a deadlock on the MDTs of our yuba cluster. When checking /proc/spl/kstat/zfs//txgs, the txg number was not advancing for tens of minutes and the affected nodes had to be rebooted. This happened on several of the MDT nodes all around the same time.

behlendo added a comment - yesterday- edited

Yuba5 encountered the same issue.  The root cause appears to be a Lustre thread holding open a transaction which prevent the ZFS txg sync engine from being able to advance.  This results in all of the Lustre threads waiting to assign new transactions causing a high load average but otherwise idle looking system.

To check if a system has hit this issue the `/proc/spl/kstat/zfs/yuba5/txgs` file can be read to determine if TXGs are advancing.  For example, the following output from yuba5 shows txg 54317256 is held open and is never allowed to close.
