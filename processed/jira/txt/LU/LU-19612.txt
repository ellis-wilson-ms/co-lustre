--- Description ---
Lustre clients mounting with network restrictions (network=tcp or network=tcp6), only the first MDT (MDT0000) becomes accessible. All additional MDTs (MDT0001+) fail to connect, causing remote directory creation commands to fail with "No such device" error.

The bug occurs when a Lustre client mounts a multi-MDT filesystem using explicit network-type NIDs  in a mixed IPv4/IPv6 environment.
