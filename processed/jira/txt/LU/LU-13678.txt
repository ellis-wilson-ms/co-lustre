--- Description ---
The original peer discovery design called for the ability to run discovery in an enabled, disabled or verification mode. Only the enabled and verification modes were implemented. We should be able to implement disabled mode pretty easily. This ticket will track that work.

Note that with the multi-rail routing feature, we now rely on both the lnet monitor thread and the discovery thread to monitor route aliveness. We'll need to decide if we want to also allow disabling of the route checking.
