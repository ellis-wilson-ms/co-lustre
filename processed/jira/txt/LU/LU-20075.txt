--- Description ---
There appears to be some mismatch in the arguments for lfs getstripe.  Running the command "lfs getstripe --ost fileName" results in the following:

usage getstripe [–ost| -O UUID] ....

This conflicts with "man lfs getstripe" which has the followinig ( does not require UUID arg)

--ost|-O

   Print the starting OST index for the file layout

"man lfs getstripe" also has the following

--stripe-index|-i

   Print the starting OST index for the file layout

Shouldn't the "getstripe --ost" report the list of OSTs in the component and not be a duplicate option for --stripe-index, and why does the failed "lfs getstripe" indicate that --ost requires an argument, but "man lfs getstripe" does not.
