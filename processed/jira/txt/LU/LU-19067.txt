--- Comment (sergey, Thu, 29 May 2025 14:14:02 +0000) ---
Hi yujian ,

I'm not sure it is a good solution to clean everything up to be able to start sanity-quota test suite. My arguments are following:

	What if the files that consume quota space are useful for analysis of the previously failed test suite?

	It might be impossible to remove such files because of broken FS. I.e. it was tried to remove these files, but rm failed.

I would propose to fix the previous test suite, if It doesn't do cleanup properly. If the issue reproducible we can add something like "find ." into the check_system_is_clean to define the test case that hasn't cleaned up it's files.

There is also a chance that the problem caused because of delays in ZFS. Probably some timeout could be added in a check_system_is_clean for ZFS case. I.e. If "usage" is detected sleep 20 seconds, try again and fail if the usage is still > 0.
