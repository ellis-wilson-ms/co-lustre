--- Description ---
Periodically when debugging a lustre issue on newer kernels developers run into the default behavior of %p to hash the address. While this safe default hinders bad actors it also frustrates developers.

Lets start by defaulting %p to %pK and slowly decorating pointers with the available pointer decorators. See: https://docs.kernel.org/core-api/printk-formats.html

--- Comment (adilger, Fri, 1 Mar 2024 22:55:46 +0000) ---
Shaun, I just noticed in a test run that this is being set and then cleared for every subtest run, across all of the servers. This would be far better to do only once per test session to avoid adding extra SSH overhead to every subtest that is run.

If libcfs_debug_raw_pointers was made a module parameter then it could automatically be set by the test framework on module load, and no need to even set the parameter at runtime. Alternately, or additionally, it would be possible for test-framework.sh to use:

     do_facet mgs "$LCTL set_param -P debug_raw_pointers=Y"

when the filesystem is formatted so that it is always set at mount time.
