So my question is whether there were any 160l failures on RHEL 9.5 or RHEL 8.10 that cannot be explained by 63516. If such cases exist, that would suggest an additional root cause independent of both RHEL 10 and my patch.

I think the key point is mdd_fix_attr(). If oattr->la_ctime is newer than la->la_ctime (in seconds), atime/mtime/ctime is not updated. 

    if (la->la_valid & LA_CTIME) {
        /**
         * The pure setattr, it has the priority over what is
         * already set, do not drop it if ctime is equal.
         */

            la->la_valid &= ~(LA_ATIME | LA_MTIME | LA_CTIME);
    }
