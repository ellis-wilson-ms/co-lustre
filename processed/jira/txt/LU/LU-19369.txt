--- Description ---
When lustre is compiled with --enable-multiple-lnds the resulting RPM contains the following:

 ❯ rpm -pql --scripts kmod-lustre-client-lnet-socklnd-2.16.57_189_g2eecc80-1.el10.x86_64.rpm

postinstall scriptlet (using /bin/sh):
if [ -e "/boot/System.map-6.12.0-55.22.1.el10_0.x86_64" ]; then
    /usr/sbin/depmod -aeF "/boot/System.map-6.12.0-55.22.1.el10_0.x86_64" "6.12.0-55.22.1.el10_0.x86_64" > /dev/null || :
fi

modules=( $(find /lib/modules/6.12.0-55.22.1.el10_0.x86_64/extra/lustre-client-lnet-socklnd | grep -E '\.ko(\.gz|\.bz2|\.xz|\.zst)?$') )
if [ -x "/usr/sbin/weak-modules" ]; then
    printf '%s\n' "${modules[@]}"     | /usr/sbin/weak-modules --add-modules
fi
preuninstall scriptlet (using /bin/sh):
rpm -ql kmod-lustre-client-lnet-socklnd-2.16.57_189_g2eecc80-1.el10.x86_64 | grep -E '\.ko(\.gz|\.bz2|\.xz|\.zst)?$' > /var/run/rpm-kmod-lustre-client-lnet-socklnd-modules
postuninstall scriptlet (using /bin/sh):
if [ -e "/boot/System.map-6.12.0-55.22.1.el10_0.x86_64" ]; then
    /usr/sbin/depmod -aeF "/boot/System.map-6.12.0-55.22.1.el10_0.x86_64" "6.12.0-55.22.1.el10_0.x86_64" > /dev/null || :
fi

modules=( $(cat /var/run/rpm-kmod-lustre-client-lnet-socklnd-modules) )
rm /var/run/rpm-kmod-lustre-client-lnet-socklnd-modules
if [ -x "/usr/sbin/weak-modules" ]; then
    printf '%s\n' "${modules[@]}"     | /usr/sbin/weak-modules --remove-modules
fi
/lib/modules/6.12.0-55.22.1.el10_0.x86_64/extra/lustre-client/net
/lib/modules/6.12.0-55.22.1.el10_0.x86_64/extra/lustre-client/net/ksocklnd.ko
/usr/lib/.build-id
/usr/lib/.build-id/55/d7b44b52766ee35b69f4a6ae986604e626e0c3

Note that the `find` invocation looks at the `extra/lustre-client-lnet-socklnd` directory whereas the kernel module is installed in `extra/lustre-client`. This is due to the RHEL script `/usr/lib/rpm/redhat/kmodtool` always looking in a directory with the same name as the RPM (`find /lib/modules/${verrel}${dotvariant}/extra/${kmod_name}`).

—

Configure options:

./configure --disable-server --enable-multiple-lnds --with-o2ib=no
