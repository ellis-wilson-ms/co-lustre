--- Description ---
For some reason set -e we have at the start of test framework.sh is not taking effect on centos7, but on rhel8 and other releases it makes llmount.sh and everything else fail like this:

./../tests/test-framework.sh: line 685: echo: write error: Operation not permitted 

or

./../tests/test-framework.sh: line 685: echo: write error: Device or resource busy 

which happens on new kernel when kmemleak is disabled on start and then execution just stops.

So probably need to ignore those failures.
