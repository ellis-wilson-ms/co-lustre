--- Comment (adilger, Tue, 4 Sep 2018 15:09:33 +0000) ---
This problem sounds like the Samba server is keeping the file open after it is unlinked, so that means the file data will not be released until the file is closed.  You could use "lsof" on the Lustre client where Samba is running to see if these files are being kept open.

Note also that Lustre 2.6.0 is quite old and there have been problems with open file references, so you might also try with a 2.7.x or 2.10 client for the SMB server.
