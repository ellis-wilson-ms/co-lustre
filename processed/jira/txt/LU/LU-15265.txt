--- Comment (adilger, Mon, 22 Nov 2021 17:17:14 +0000) ---
The free block count in the group descriptors is likely to be updated if there are any repairs to the filesystem, which is not shown here, so there is no way to know if this information is useful or not. The free block count in the superblock is not kept up to date while the filesystem is mounted, so this is a very common situation. Also, there is absolutely no information here about what happened to the filesystem before e2fsck was run, so there isn't really anything useful that can be done here.
