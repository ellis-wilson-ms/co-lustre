--- Description ---
We need to make sure that older clients do not try to read the parity components. This can probably be handled by setting the layout pattern of the parity components to LOV_PATTERN_PARITY, but this needs to be tested.  It may be desirable to exclude the LOV_PATTERN_PARITY components from the layout that is sent to older clients, since 2.15.x does not (yet) have patch https://review.whamcloud.com/51060 ("LU-16837 llite: handle unknown layout component") to handle FLR-EC layouts properly.

--- Comment (adilger, Fri, 19 Dec 2025 17:10:34 +0000) ---
Patch https://review.whamcloud.com/60266 ("LU-16837 llite: handle unknown layout component") hasn't yet landed to b2_15 as of 2.15.8, which is needed for those clients to skip the unknown components of a mirror. Otherwise, they will get an error trying to read from files with EC components.

For clients older than v2_15_58-44-g14ed4a6f8f23 (when patch 60266 landed on master) opening an EC file for read, it would be necessary for the MDS to remove EC components from the layout, or alternately mark the EC components STALE in memory only when sending it to the old client. If a client is opening an EC file for write, the MDS would always mark the EC components STALE, so they should never be used (regardless of client version).
