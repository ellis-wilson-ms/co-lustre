--- Description ---
With the separation of the of the user land code and server code struct lustre_disk_data was moved to the user land mount utility. This data is stored on the ldiskfs server backend but is never used by the server code which really doesn't make sense. We should make this a standard feature for both ldiskfs and ZFS and take advantage of this data. I could see this being used by the MGS to automount all the other servers with the proper mount options.
