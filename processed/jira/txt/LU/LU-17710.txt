--- Comment (eaujames, Fri, 5 Apr 2024 17:27:31 +0000) ---
This is a 2.12.9 with the "LU-14339 obdclass: add option %H for jobid " backported.

These errors and corrupted jobid are only seen when using "%H".

The race seems to be in "lli_jobid", the field is not protected and have concurrent accesses (read/write) for buffer IOs.

static int jobid_interpret_string(
....
case 'H': /* short hostname. Cut at first dot */           
        l = snprintf(jobid, joblen, "%s",                  
                     init_utsname()->nodename);            
        p = strnchr(jobid, joblen, '.');                   
        if (p) {                                           
                *p = '\0';                                 
                l = p - jobid;                             
        }                                                  
        break;                                             

I think that backporting  the "LU-16251 obdclass: fill jobid in a safe way" (https://review.whamcloud.com/48915) should fix the issue.

But races are still possible with lli_uid/lli_gid/lli_jobid.
