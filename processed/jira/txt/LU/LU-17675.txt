--- Comment (adilger, Sat, 30 Mar 2024 17:05:52 +0000) ---
Is it possible that the el9.x stat command is using statx() and getting the atime only from the MDT and not the OST objects, and the MDT is not updating the atime locally because it isn't old enough?  If stat(1) (the command) is calling a full stat() is done then the client will retrieve the size (and atime) from the OSTs, but AFAIK if it is calling statx() it will only fetch the requested attributes from the MDT inode, and atime is "mostly" uptodate on the MDT.

One way to check this would be to put "sleep $(do_facet mds1 $LCTL -n mdd.*.atime_diff)" before the migrate, so that the MDT inode's atime will be updated on close.
