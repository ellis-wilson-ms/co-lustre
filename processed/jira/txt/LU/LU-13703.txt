--- Description ---
Using the lfs_migrate script with PFL arguments that contain "--stripe-count" or "--stripe-size" (which is very likely) will result in "lfs_migrate" parsing the last incidence of those values as the global stripe count/size and not work properly if different values are specified for different components.

Instead, if a PFL layout is specified on the command-line (even if migrating a file to a PFL layout) is not necessarily the most optimal solution, it should be handled properly.
