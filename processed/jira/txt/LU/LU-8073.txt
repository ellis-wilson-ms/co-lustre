--- Description ---
The lustre-source rpm package should probably be eliminated.  I'm not entirely clear on why it ever existed though.

The canonical Lustre tarball contains the full Lustre source code, and in the rpm world the .src.rpm file contains the canonical tarball.  Also, the full source code probably also appears incidentally in the lustre-debuginfo package.

So it is not clear to me why we also need a binary rpm that contains the lustre source code.  Maybe there was a reason in the past, but I am guessing that it is lost to history.  We should just remove the package now.
