Hi Chakshu,

can you please fix this test case.  It looks like a problem with the aarch64 client with 64KB pages, so the client is always writing full 64KB pages to the server even when only 4KB of data is written by dd.

The test should be changed to use local page_kb=$((PAGE_SIZE / 1024)) instead of hard-coding 4 and ${page_kb}k instead of 4k, and the calculation for expected and whatever else is needed.

For testing you will need to explicitly run on the aarch64 client if using trivial, like:

Test-Parameters: trivial
Test-Parameters: testlist=sanity clientarch=aarch64 env=ONLY=130i,ONLY_REPEAT=10

(and possibly clientdistro=rocky9.5 if autotest complains).
