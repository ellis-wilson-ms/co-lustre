--- Description ---
In order to ease SSK deployments, it would make sense to have the client automatically load the SSK key it finds in the mount point. The key would have been copied ahead of time into the client mountpoint directory with the filename .lgss, so when mounting the client, it would look for this .lgss file and load the SSK key it contains. That would be similar to specifying the -o skpath mount option, but in this case it would be done implicitly.
