--- Description ---
The functions:

lfs_mirror_write
llapi_mirror_copy
llapi_mirror_copy_many
llapi_mirror_resync_many_params 

All share the same mistake, rounding up the number of bytes to write before writing out:

/* round up to page align to make direct IO happy. */
to_write = ((bytes_read - 1) | (page_size - 1)) + 1; 

I noticed this when playing with them - copying a small input file, they round up the size to 4K and put junk padding in.

I'll push a patch and try to include some tests.

--- Comment (paf0186, Wed, 13 Aug 2025 14:18:43 +0000) ---
A write of less than page size works fine if the starting offset is page aligned - DIO has no issue writing out the last few bytes of a file.  No need to bring in buffered IO.  DIO 'just works' for this.

Anyway, it's a relief knowing we have the truncate.  We actually don't have it for mirror write, but that's really only used for testing, so that's probably OK.  We should still fix this but it's not anything scary.

--- Comment (adilger, Wed, 13 Aug 2025 01:19:34 +0000) ---
I don't think this is an actual bug?  In all of those functions, the mirror is truncated before the file is actually used, so it chops off the extra bytes before the mirror becomes visible.  

It might be possible to change the code to check for the end of file and call fcntl() to change over to buffered IO for this last write, but I don't know if this would make any visible performance difference.

That said, I'm a bit sad that we apparently have 4 different functions that are nearly doing the same thing, and would be happy to consolidate these into a single function (possibly structured similar toh `migrate_copy_data()`, but I'm not sure if they can share code or not).
