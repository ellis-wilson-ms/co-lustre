Dear Gabriele, the problem is not related to the  runlevel, but is located in the /etc/init.d/lustremount script that doesn’t respect the rigth skeleton for the init files. According to redhat sample script, after a successuful start the file /var/lock/subsys/ must be created; the same file must be removed after a stop.

See /usr/share/doc/initscripts-/sysvinitfiles

I fixed the script and now the shutdown/reboot works fine.
