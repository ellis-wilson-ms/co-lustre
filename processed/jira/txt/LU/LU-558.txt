--- Environment ---
A separate MGS device hosted on a dedicated MGS.

--- Description ---
On a configuration with a dedicated MGS hosting only the MGS but no MDT or clients, tests that invoke "lctl pool_list" on the "mgs" facet will fail.  For example,

== sanity test 201c: Remove a pool =================================================================== 21:14:28 (1311912868)
client-4-ib: Pool lustre.cea1 not found
client-4-ib: Can't verify pool command since there is no local MDT or client, proceeding anyhow...
Waiting 90 secs for update
Updated after 5 sec: wanted 'foo' got 'foo'
Resetting fail_loc on all nodes...done.
PASS 201c (8s)
client-4-ib: Lustre filesystem 'lustre' not found: No such file or directory (2)

The tests should either start a client on the MGS or use the MDS instead.
