--- Description ---
When multiple filesets are configured that share a directory hierarchy, e.g.,

1. "/dir"
2. "/dir/subdir (ro)" 

whereby read-only is set for fileset 2, mounting subdirectory "/lustre/dir/subdir" only looks for a matching fileset. This could result in fileset 1 being matched which does not have ro flag set, resulting in an rw write mount which is incorrect. Instead, fileset matching should always consider the closest fileset at mount time.
