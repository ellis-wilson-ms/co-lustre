--- Description ---
My name is Bob Torgerson, and I work as an HPC Analyst for the Arctic

Region Supercomputing Center in Fairbanks, Alaska. Recently, we have been

working to build a Lustre file system for our high performance systems and

desktop clients, a file system with a total storage capacity of

approximately 290 TB. We decided to go with Lustre version 1.8.8 for our

initial install on the suggestion of Zhiqi Tao, my instructor in a

WhamCloud training in Alexandria, VA. We have moved forward with the

installation using Lustre 1.8.8  on RHEL 5 servers and have a working file

system able to be exported to our clients. However, we have had some issues

related to hardware such as newer QLogic cards being unsupported in RHEL 5

that appear that they would be resolved by upgrading to RHEL 6, but at this

point, Lustre 1.8.8 is unsupported on RHEL 6 servers.

Before we move further forward with our implementation, I would like to

discuss with WhamCloud support what version of the Lustre software is

recommended for production. We have the 1.8.8 Lustre file system working

now, but have not invited beta testers to try it yet, meaning that we still

have time to make changes including what version of Lustre we use before we

invite users to use the file system.

Any suggestions you could provide to me would be greatly appreciated!

Thank you in advance,

--- Comment (pjones, Wed, 27 Jun 2012 13:33:34 +0000) ---
I would agree that 2.1.x is a sensible approach but I want to clarify that 1.8.x is not due for retirement. We plan to keep supporting it as long as sufficient demand is there, but recognize that this demand will wane over time.

--- Comment (cliffw, Tue, 26 Jun 2012 16:59:17 +0000) ---
As you wish to run Redhat 6, we would advise you moving to Lustre 2.1, our current production release.

http://downloads.whamcloud.com/public/lustre/latest-maintenance-release/el6

This will position you for the future, and eliminate the RHEL6 issue. The 1.8 stream is due for retirement (see the Lustre Roadmap http://wiki.whamcloud.com/display/PUB/Community+Lustre+Roadmap), and especially for a new implementation, we would advise Lustre 2.1
