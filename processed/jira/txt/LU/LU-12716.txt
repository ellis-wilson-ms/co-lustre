--- Description ---
We have increasing occurrences of "text file busy" errors when trying to execute either scripts or binaries that are on our Lustre filesystem. This is blocking productivity.

The only work-around we know of is to copy the file to a new file, then it (sometimes) works. This seems to happen at random, to random scripts and binaries.

The problem was narrowed down to the scenario where files are copied/created on the Lustre file system through an NFS mount.

For example,

(Lustre/OPA network) , NFS server  non-Lustre node

When the non-Lustre node created a file on the Lustre file system through an NFS mount, subsequent accesses to that file from other nodes showed the Test file busy error.

 

I've also upgraded all Lustre to 2.12.2 and will re-open if this becomes a blocker again. For now we have a workaround, that is to not use the NFS share to create files.
