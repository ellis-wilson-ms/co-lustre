--- Description ---
lnet_selftest cannot be used with nids that are not primary.

When it is used with non primary nids, there is nothing pointing to this problem in the logs or in kernel messages, only a timeout error.

There are two possibilities to improve that:

	Print relevant message to explain primary nids have to be used.

	Handle non primary nids.

--- Comment (adilger, Sat, 29 Mar 2025 05:40:27 +0000) ---
My strong preference would be to allow using the non-primary NID transparently. If the code can detect this case and print an error message that tells the user to use the primary NID, it could just use the primary NID directly and just print a message that it is doing so (or not at all if LST is doing MR testing.
