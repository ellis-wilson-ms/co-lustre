--- Description ---
Since the landing of LU-1548 (http://review.whamcloud.com/3561) to move the dynlock from the inode to the iam container, there is no need to have the dynlock code in ldiskfs anymore.  This allows the dynlock code to move into osd-ldiskfs entirely, and remove a patch from the ldiskfs series that otherwise needs to be maintained for each server kernel.
