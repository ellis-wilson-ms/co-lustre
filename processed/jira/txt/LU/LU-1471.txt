--- Description ---
This is a test script issue in setup-nfs.sh and only occurs on SLES11 NFS client:

Exporting Lustre filesystem...
Shutting down NFS daemon: [  OK  ]
Shutting down NFS mountd: [  OK  ]
Shutting down NFS quotas: [  OK  ]
Starting NFS services:  [  OK  ]
Starting NFS quotas: [  OK  ]
Starting NFS mountd: [  OK  ]
Starting NFS daemon: [  OK  ]
client-15-ib: service: no such service rpcidmapd
client-1-ib: service: no such service rpcidmapd
 parallel-scale-nfsv3 : @@@@@@ FAIL: setup nfs failed!

https://maloo.whamcloud.com/test_sets/28c65cbc-ae11-11e1-ae0d-52540035b04c

https://maloo.whamcloud.com/test_sets/295e0c42-ae11-11e1-ae0d-52540035b04c

There is no /etc/init.d/rpcidmapd file on SLES11 nfs-client package. All of the services are in /etc/init.d/nfs.

By only using /etc/init.d/nfs service, the NFS could be set and cleaned up successfully on SLES11 NFS clients.
