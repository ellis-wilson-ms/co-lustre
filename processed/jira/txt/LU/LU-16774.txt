A nice update would be to give warnings when Lustre kernel code uses __uXX values since those are mean only for user land interfaces. Developers tend to use the user land values to this day but its incorrect.

--- Comment (adilger, Fri, 20 Sep 2024 06:02:52 +0000) ---
I don't think it is reasonably possible to implement this in checkpatch.  We need to use __uXX variables when working with any variable/structure related to the network protocol and for variables/structures related to the on-disk storage, config logs, etc. in addition to anything related to the user interface.

Distinguishing between these cases in the code would IMHO be impossible to do in checkpatch.

While it is true that "uXX" is used in the kernel-internal code and "_uXX" is used for disk/userspace, they boil down to the same 32-bit variable type, so there is no real harm or benefit to change them, besides the savings of two "_" characters, which are not in limited supply, and possibly a shortening of a few lines.
