Basic Conclusion

This issue appears to be a C code compatibility problem caused by the GCC version upgrade on RHEL 10. The new GCC version uses the stricter C17 standard by default, which triggers compilation errors (such as implicit function declaration) when building Connectathon source files. Typically, these issues can be resolved by compiling with older standards like C89.

Version Information

[root@localhost ~]# cat /etc/rocky-release

[root@localhost ~]# gcc -v 2>&1 | grep "gcc version"
gcc version 14.3.1 20250617 (Red Hat 14.3.1-2) (GCC)

Causal Path

1. The `test_connectathon()` case executes the test command: `bash ./runtests -N 2 -g -f /mnt/lustre/d0.connectathon`;

2. The test command triggers the compilation of `connectathon/general/stat.c` (and `connectathon/general/large.c`);

3. The new GCC version on RHEL 10 detects issues like implicit function declarations in `stat.c`, leading to compilation errors.
