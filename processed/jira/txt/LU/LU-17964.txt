--- Description ---
Once a file cached on PCC was mmap()ed, we will replace the mapping between the Lustre file and PCC copy for mmap I/O (i.e. fault()).

 

Once the file was detached from PCC backend due to manual detach command or the revocation of the LAYOUT lock which protect the cache validity of PCC cache access, we should reset the mapping of the file handle and new I/O for the file that once cached on PCC should be set with fallback flag.

 

If a Lustre file has some opened file handles with fallback flag, we could not switch the mapping and the mmap() I/O should also fallback to Lustre OSTs.
