--- Description ---
When a router is talking to a non-MR peer, currently the code will assume that it has to send responses out of the same interface it received the message on. This causes a skew in the load on the interfaces. If we declare the peers to be MR to the router (even though they are not) load is spread properly over the interfaces.

We need to handle this without this quirkiness in the configuration.
