--- Comment (adilger, Wed, 27 Mar 2024 22:42:05 +0000) ---
This failed 10x in the past week,  and has been failing frequently since 2024-02-15. 

  

The test is running:

        mkdir_on_mdt0 $DIR/$tdir

        #define OBD_FAIL_OUT_ENOSPC     0x1704
        do_facet mds2 lctl set_param fail_loc=0x80001704
        $LFS setdirstripe -i 0 -c2 $DIR/$tdir/bad_striped_dir

I suspect the issue might be that the MDT QOS allocation is selecting the second MDT stripe on MDT0002 (on mds1 where fail_loc is not set), when it used to always allocate "MDT0000,MDT0001 when "-i 0 -c 2" was used?

I think the test should be considered a success if the MDTs on mds2 are avoided (MDT0001, MDT0003), or possibly changed so that it explicitly tries to create a stripe on MDT0001?  Alternately, change fail_loc=0x1704 so that it checks fail_val==mdt_idx and reports ENOSPC for all but that one MDT and set it on both MDS nodes?

Another option would be to not create a striped directory in this case, and just create an unstriped directory instead of failing?  I think most users would prefer that their application keeps running with one MDT rather than getting an error when there is still space available on one MDT.
