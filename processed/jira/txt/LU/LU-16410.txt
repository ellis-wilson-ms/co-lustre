--- Description ---
It was noticed that after execution sanity-lfsck tests .lustre/lost+found/ folder may be left in non-cleaned state. The script has an ending to do the clean up

 

cleanup the system at last
REFORMAT="yes" cleanup_and_setup_lustre

But probably this sometimes is not executed (test interrupted ?). I tried a few (for example test_1b, test_18c) and if the test is interrupted I see .lustre/lost+found/MDTxxxx/ and a number of files. 

As a result a consequent test if it does not do the complete reformat may fail. Specifically sanity-lipe-find3 tests fail because it finds unexpected files in .lustre/lost+found/
