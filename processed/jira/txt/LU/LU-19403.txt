utopiabound so if I understand you correctly, having a Rust binding to the C llapi using Rust's FFI (and generated bindings) would not be enough? Does it mean you would want a Rust crate wrapping the C llapi?

--- Comment (utopiabound, Fri, 12 Dec 2025 17:39:07 +0000) ---
We experimented with llapi at one point but it caused some problems with the build pipeline because every build target needed lustre libraries/headers to build against, so we currently don't build against llapi and solely use CLI calls.

Pie in the sky request would be a pure rust llapi implementation.

--- Comment (sebastien, Tue, 9 Dec 2025 13:52:57 +0000) ---
The idea is to be able to carry out all 'lctl nodemap*' commands but programmatically via the llapi. Ideally, the associated 'lctl nodemap*' commands would be modified to internally call the corresponding llapi functions. That would make the llapi central and used internally by userspace commands.
