--- Description ---
When choosing a block for multiple mount protection near the end of the file system it can fail.  There is a patch that changes mmp_blk to (long long)mmp_blk  which should prevent the MMP block from overwriting block filesystem data.  A patch has been provided but it has not been inspected from bug24209 mmp_blk64.patch.  We would like this patch to be evaluated.  

We are concerned if we have current customers who have full filesystems and enable MPP they could be putting data at risk.
