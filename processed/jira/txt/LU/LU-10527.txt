The problem exists due to race between 3 threads doing: t1:llog_cat_current_log, t2:llog_cat_current_log and  t3: llog_cat_declare_add_rec.

	t3: llog_cat_declare_add_rec :: chd_next_log = loghandle /* loghandle is not initialized yet */

	t1: llog_cat_current_log :: llog is full thus up_read LLOGH_CAT and try to use next log with write sem

	t2: llog_cat_current_log :: llog is full thus up_read LLOGH_CAT and try to use next log with write sem

	t1: llog_cat_current_log :: holds write LOGH_CAT semaphore /* llog is still full -> try to use next */

	t1: llog_cat_current_log :: chd_current_log = chd_next_log /* chd_next_log is not initialized yet */

	t1: llog_cat_current_log :: up_write LOGH_CAT sem

	t2: llog_cat_currnet_log :: holds write LOGH_CAT sem

	t2: llog_cat_current_log :: LASSERT(llh);

Not doing so should avoid to have llog_cat_declare_add_rec() repeatedly and unnecessarily create new+partially initialized LLOGs/llog_handle and assigned to "cathandle->u.chd.chd_current_log", this without llog_init_handle() never being called to initialize "loghandle->lgh_hdr".
