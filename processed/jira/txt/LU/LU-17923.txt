--- Description ---
After extend a mirror of a file, the new OST object's parent fid points to a non-exist fid. I think the reason is:

1)  A volatile MDT file is created when "lfs mirror extend" command is issued;

2) The new OST object's parent fid to the volatile fid;

3) the volatile file is removed and does not exists on MDT any more after "lfs mirror extend" command finishes.

Having a wrong parent fid in filter_fid is confusing and might causes extra work (if not problem) for lfsck. "lfs mirror extend" should make sure the new parent fid is synced to new objects.

Following are the steps to reproduce:

# FILE=mirror_file
# lfs setstripe --ost 4 $FILE
# lfs mirror extend --mirror-count --ost 0 $FILE
# lfs path2fid $FILE
[0x200000403:0x7:0x0]
# lfs getstripe $FILE
