--- Comment (joshua, Sat, 7 Jun 2014 00:55:49 +0000) ---
I've been doing some debugging on this, and this is what I have uncovered so far:

The recommended RewriteRule is this:

RewriteRule ^/(.*\.git/(?!/?(HEAD|info|objects|refs)).*)?$ /gitweb%{REQUEST_URI}  [L,PT]

But that generates 403 errors when trying to access URLs such as http://git-stage.hpdd.intel.com/fs/lustre-release.git as we know. If I modify the RewriteRule and take out the slash after .git like so:

RewriteRule ^/(.*\.git(?!/?(HEAD|info|objects|refs)).*)?$ /gitweb%{REQUEST_URI}  [L,PT]

The web site works fine, and I can clone via HTTP.

Not sure why the recommended configuration does not work. Does not appear to be a version thing, as git 1.8 generates the same behavior.

 We do not support git clones via HTTP due to not running to post-commit hooks needed to make HTTP clones work. We might want to disable the ability to clone. It does work, but the clone one gets from an HTTP clone can be out of date.
