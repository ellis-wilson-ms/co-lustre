--- Description ---
When running testing on a Lustre configuration with a separate MGS and MDS, we need to mount the file system on the MGS so that the lctl command can look in /proc/fs/lustre/*/lov/pools to get the current state of OST pools. 

We need modify several test scripts or individual tests to mount the file system on the MGS. Tests that manipulate OST pools are:

sanity: 27D, 200, 253, 406

replay-single: 85b

conf-sanity: 82b, 103

ost_pools: All tests
