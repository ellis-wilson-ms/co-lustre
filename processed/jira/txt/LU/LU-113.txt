--- Description ---

---> Package lustre-ldiskfs.x86_64 0:3.3.0-2.6.18_194.17.1.el5_lustre.g94d9119_gd5e659b set to be updated

--> Processing Dependency: ldiskfsprogs >= 1.41.12.2.ora1 for package: lustre-ldiskfs

--> Running transaction check

--> Processing Dependency: e2fsprogs = 1.39-23.el5 for package: infinipath-psm

---> Package e2fsprogs.x86_64 0:1.41.14.wc1-0redhat set to be updated

--> Processing Dependency: libcom_err.so.2 for package: krb5-libs

libcom_err.so.2 is in our e2fsprogs package.

On RedHat, libcom_err.so.2 is in a separate e2fsprogs-lib package.

This may be the result of the packaging mis-match between our e2fsprogs package and 

the RHEL setup.

  Description of problem:

  Some pieces of software that use libcom_err fail to find it  

  during the configure step because the library is in /lib64

  while the header file is in /usr/include/et instead of

  /usr/include.  While there is only one Fedora package at

  present that requires libcom_err-devel (krb5-devel), this

  sometimes causes difficulty in building unpackaged software

  and requires manually setting CFLAGS.

This version should be tested to see if it fixes some of the upgrade problems.
