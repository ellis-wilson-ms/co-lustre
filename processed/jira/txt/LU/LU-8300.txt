--- Description ---
If the oi.16.xxx file lost, then the osd_fid_lookup() will return -115 to the LFSCK when try to locate the object for creating ".lustre/lost+found/MDTxxxx". In fact, we know that such object does not exist, but forget to pass LOC_F_NEW to the OSD. That will cause the LFSCK cannot handle orphan objects under such case.
