--- Description ---
The "enable_crypto" shell variable  used in autoconf/lustre-core.m4 and config/lustre-build.m4 is sometimes used unquoted, so it is unsafe for it to contain special characters.

It can be set to "embedded llcrypt" which contains a space.  This causes errors when running ./configure.

This appears to be the only enable_foo variable that has this problem.
