--- Description ---
Currently the only command option for "lnetctl stats" is "show". At least when debugging connectivity issues, it would be useful to be able to clean up the stats by executing, for instance, "lnetctl stats reset", as it would be easier to detect the changes in stats from the clean state rather than on top of historical values.
