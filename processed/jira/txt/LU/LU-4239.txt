--- Comment (paf, Fri, 13 Jun 2014 21:39:35 +0000) ---
Just put out a patch for this...  Note this is a server patch, not a client one - The limitation is not in LFS, it's actually in some code on the MDS.

As I wrote in the patch description:

LU-4239 mdt: Increase MAX_PATH_DEPTH to PATH_MAX/2

Currently, MAX_PATH_DEPTH is limited to 100.

This is an arbitrary limit, and prevents using lfs fid2path

on very deep directory trees.

More seriously, this same limit prevents HSM from archiving

these files.

Change MAX_PATH_DEPTH to PATH_MAX/2 because the highest

FID/character density is in paths like:

/a/b

which has 4 characters and 2 FIDs.

—

The obvious downside to this patch is it increases the size of path_lookup_info by about 31k.

(Each FID is 16 bytes, 100*16 bytes = 1600 bytes, 2048*16 bytes = 32k bytes)

Here's a quick test showing a file system with this patch on the MDS - Previously, this would give the -75 error mentioned:
