--- Description ---
It would be useful to allow Jenkins to do very limited builds for fortestonly patches with directories in the patch commit message.

Build-Parameters: clientdistro=el7.9
Test-Parameters: fortestonly clientdistro=el7.9 serverversion=2.15.3

Since it isn't possible to do this (it will always start all of the builds) it seems possible to have lbuild check the directives in the commit message and just "exit 0" at the start of the build without actually building anything. 

This would also be useful for cases of patches which are based on an old parent (eg. to give an old build to a customer) that currently fail if the build support for a distro is not in the old code being built and fails the build instead of building for the available distros:

17:19:09 + error 'Target 4.18-rhel8.8 was not found.'
17:19:09 + local 'msg=Target 4.18-rhel8.8 was not found.'

--- Comment (adilger, Thu, 19 Sep 2024 17:43:30 +0000) ---
I added the following code block to only the lustre-reviews and lustre-b_es-reviews in the Jenkins "Execute Steps->Build shell" section.  That means Build-Parameters: in commit messages will affect review builds, but not the release builds for lustre-master and others, which is what we want.

build_params="Build-Parameters:"
echo "### GIT COMMIT BUILD-PARAMETERS CHECK: git log -1 ###"    
if git log -1 | grep "^    $build_params"; then
    echo "### GIT COMMIT BUILD-PARAMETERS CHECK: environment ###"
    echo "arch='$arch'"
    echo "distro='$distro'"
    echo "build_type='$build_type'"

    echo "### GIT COMMIT BUILD-PARAMETERS CHECK: eval ###"
    # Since Build-Parameters set, only listed builds are done, all others skipped.
    # Expect lines with ONE directive per line, possibly comma-separated:
    #    Build-Parameters: clientdistro=el8.7,el9.2 [clientarch=aarch64]
    #    Build-Parameters: serverdistro=el8.8 [serverarch=x86_64]
    #    Build-Parameters: ignore
    # Having both clientdistro and serverdistro, or clientarch and serverarch,
    # on the same line will not work, just use "distro=" or "arch=" in that case.
    skip_for_params=true
    while read build_parameters param1 param2 ignored_params; do
        want_distro=""
        want_arch=""

        [[ "$param1" =~ "ignore" ]] && break # skip all builds, only for review
        [[ "$param1" =~ "client" && "$build_type" != "client" ]] && continue
        [[ "$param1" =~ "server" && "$build_type" == "client" ]] && continue
        [[ "$param1" =~ "distro" ]] && want_distro="$param1" || want_distro="$param2"
        [[ "$param1" =~ "arch" ]] && want_arch="$param1" || want_arch="$param2"
        if [[ -z "$want_arch" || "$want_arch" =~ "$arch" ]] &&
           [[ -z "$want_distro" || "$want_distro" =~ "$distro" ]]; then
            skip_for_params=false
            break
        fi
    done "^    $build_params")

    if $skip_for_params; then
        echo "SKIP: $distro:$arch not found in $build_params line" > artifacts/README
        exit 0
    fi
fi
echo "### GIT COMMIT BUILD-PARAMETERS CHECK: fini ###"
