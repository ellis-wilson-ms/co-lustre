This issue was imported by the layout lock patch on the client side, so it will be fixed in LU-1876(patch http://review.whamcloud.com/4416)

--- Comment (jay, Tue, 6 Nov 2012 01:40:51 +0000) ---
The root cause of this problem is that when a cl_lock belonging to other object is requested(this happens for mmap case), the stripe's refcount may not be greater than 1 if there is no pending IO for it.

Probably this assertion is too restricted here. Anyway, the layout can't go away if there exists locks. So I think we can fix this by removing this assertion.
