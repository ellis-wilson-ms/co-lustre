--- Description ---
sanity test_56a can fail when a composite layout is set on the mount point with 

sanity test_56a: @@@@@@ FAIL: /usr/bin/lfs getstripe --obd wrong: found 6, expected 3

According to Bobi Jam, this looks like a test script issue that needs to be modified to work with PFL; “…PFL file could contains multiple stripes on the same OST, so the found stripe number on the same OST mismatches with the number of file, it presumes that a single file only have one object on the specific OST.”
