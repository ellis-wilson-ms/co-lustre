I suspect the culprit could be a jbd2 bug:

"forever sleeping process in do_get_write_access()" http://www.spinics.net/lists/linux-ext4/msg24689.html

From the stack trace, we can see some ost io threads are waiting in do_get_write_access(), and these threads were holding i_data_sem, which caused some other io thread waiting on i_data_sem, and those never stopped transaction caused lots of other io threads stuck in jbd2_log_wait_commit().

Hi, Yujian

Is it easy to be reproduced? if it's easy to be reproduced, I think we can apply the fix in do_get_write_access() to lustre to see if it can resolve the problem. Thanks.
