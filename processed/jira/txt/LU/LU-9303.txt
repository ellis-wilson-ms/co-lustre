--- Description ---
There are no man pages describing the "lctl lcfg_erase" and "lctl lcfg_fork" commands.  Even if we don't want users to use "lctl lcfg_fork", I think that "lctl lcfg_erase" is useful for deleting a configuration if a filesystem is removed from a shared MGS, or if a filesystem with the same name is being recreated.

Please put this in a separate lctl-lcfg-erase.8 man page, and lctl-lcfg-fork.8 if that is for administrators to use, rather than making lctl.8 even bigger than it already is.  Please also move the snapshot and barrier commands to their own man pages (one per sub-command, maybe with lctl-snapshot.8 and lctl-barrier.8 giving a high-level description of the feature if needed).
