--- Comment (eaujames, Wed, 15 Apr 2026 08:08:02 +0000) ---
Yes, I think this is caused by the  https://review.whamcloud.com/64757 ("LU-5896 mdt: allow clients with old mdc_swap_layouts"):

static const struct req_msg_field *mdt_swap_layouts[] = {         
        &RMF_PTLRPC_BODY,                                         
        &RMF_MDT_BODY,                                            
        &RMF_SWAP_LAYOUTS,                                        
        &RMF_CAPA1,                                               
        &RMF_CAPA2,                                               
        &RMF_DLM_REQ                                              
};                                                                
...
static int mdc_ioc_swap_layouts(struct obd_export *exp,
				struct md_op_data *op_data)
{
...
	rc = mdc_prep_elc_req(exp, req, MDS_SWAP_LAYOUTS, &cancels, count);
	if (rc) {
		ptlrpc_request_free(req);
		RETURN(rc);
	}

	mdc_swap_layouts_pack(&req->rq_pill, op_data);

	payload = req_capsule_client_get(&req->rq_pill, &RMF_SWAP_LAYOUTS);
	LASSERT(payload);
#if LUSTRE_VERSION_CODE 	/* allow client to set msl_dv1/msl_dv2 fields until compat not needed */
	req_capsule_set_size(&req->rq_pill, &RMF_SWAP_LAYOUTS, RCL_CLIENT,
			     sizeof(*msl));
#endif

First we pack the the RMF_DLM_REQ field and then set the size of RMF_SWAP_LAYOUTS. But RMF_SWAP_LAYOUTS is first in the message and RMF_DLM_REQ is last field. So we should set the size of RMF_SWAP_LAYOUTS before packing ELC.

Otherwise, mdc_prep_elc_req() -> req_capsule_client_get(pill, &RMF_DLM_REQ) will return the wrong buffer, the field can be overwritten and then the server will compute the correct offset but the message is corrupted.
