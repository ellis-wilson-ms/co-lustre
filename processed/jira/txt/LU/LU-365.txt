--- Description ---
The copyright headers for files modified by Whamcloud engineers should be updated with a new copyright message in addition to the existing Oracle copyright message (which should not be removed).

Copyright (C) 2011, Whamcloud, Inc.

--- Comment (adilger, Tue, 10 Jan 2012 19:19:56 +0000) ---
This bug needs to be updated to include both the 2011 changes, and now also 2012 changes, like:

	Copyright (c) 2011, 2012 Whamcloud, Inc.

--- Comment (adilger, Tue, 20 Dec 2011 09:56:27 +0000) ---
I was going to close this bug, but it seems there are still quite a number of files that are modified since Lustre 2.1 that do not have Whamcloud copyright messages in them:

git diff v2_1_0_0..HEAD | diffstat | sed -e 's# b/# #' |
     awk '{ print $1 }' | grep "\.[ch]$" | sort --uniq | while read F; do
        grep -q "2011.*Wham" "$F" || echo "$F"
     done

returns 94 files that did not get a copyright, while 98 DID get a copyright.  The list isn't 100% correct, since it may include files that were modified by someone outside Whamcloud but at least provides the list of files that should be checked.
