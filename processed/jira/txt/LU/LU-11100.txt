--- Comment (mhanafi, Wed, 27 Jun 2018 22:16:24 +0000) ---
I got this to reproduce using the following steps.

	Put some load on client.
	
		stress --vm-bytes 8g --vm 5 --vm-keep --cpu 5

	
	

	Run lots of stats. I used recursive ls.
	
		cd /lustrefs

		for i in `ls -1 |tail -40`;do ls -lR ${i} > /dev/null 2>&1 & done

	
	

	Wait a few mins (1 to 2min)

	Drop caches
	
		echo 2 > /proc/sys/vm/drop_caches

		May need to do this a few times

	
	

	kill off the ls commands
	
		pkill ls

	
	

 You will see connection lost to some OSTs and all ptlrpcd* threads will go into %100 run state. Then you will get softlockup stack dumps.
