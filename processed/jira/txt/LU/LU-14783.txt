--- Description ---
When router is offline, the corresponding peer state and route state are both "down", as can be seen using

lnetctl peer show -v 4
lnetctl route show -v 4 

When the router comes back online, the peer is recovered some time before the monitor thread rediscovers the router, so for a while the peer state is "up" but the route is still "down". It should be possible to trigger router discovery on the event of the successful recovery of the router peer to avoid the delay in route availability.
