--- Description ---
Replicate MGS running on MDT0000 to other MDTs/MDSes to improve MGS availability and resiliency in the face of server and/or storage issues.  This includes:

	redundant MGS/MGT configuration logs

	redundant MGS Imperative Recovery service

--- Comment (laisiyao, Mon, 23 Feb 2026 13:11:19 +0000) ---
MGS provides below services:

	generate Lustre system configurations upon target register

	provide configurations to nodes in the system, including both servers and clients

	notify configuration changes to other nodes

	notify server restarts (imperative recovery) to shorten timeout

There should be only one MGS (MGS0) to generate configurations, otherwise it will cause brain split. As a result, MGS0 acts the same as current MGS, and the MGS replicas keep the configurations in sync with MGS0, and only need to provide configurations to clients, and they should notify clients when configuration changes or server restarts.
