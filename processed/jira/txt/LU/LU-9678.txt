It LBUG on:

int lu_site_purge_objects(const struct lu_env *env, struct lu_site *s,
                          int nr, int canblock)
{
        ....
        bnr = (nr == ~0) ? -1 : nr / (int)CFS_HASH_NBKT(s->ls_obj_hash) + 1;
        ....

And s->ls_obj_hash should be NULL at this moment, and this is called arc_prune_func() -> lu_site_purge().

So this looks to be a bug to call lu_site_purge() in arc_prune_func() because site can be released anytime.
