--- Description ---
Currently script only seems to support ipv4 , will need support for ipv6 to help with validation. 

[root@ai400x2-c0-vm00 ~]# lnet_selftest.sh --help

You need to manually load lnet_selftest on all nodes

modprobe lnet_selftest

Invalid option: –

usage: lnet_selftest.sh -c client-nids -s server-nids [-r] [-w] [-n rpcs] [-l size] [-d dist]

-c              client nids

-s              server nids

-r              reads only

-w              writes only

-l size         transfer size, defaults to 1M

-d distribute   distribute, defaults to 1:1

-n rpcs         max_rpcs_in_flight, defaults to 8

example:

lnet_selftest.sh -c "10.10.100.223@o2ib 10.10.100.224@o2ib"  -s "10.10.100.215@o2ib 10.10.100.216@o2ib"

The reason for the delay on this work is that its different than lnetctl. For lnetctl we can push commands and pull data. For lnet selftest it goes both ways at once so it makes it more complicated. I found Netlink not to be too happy when doing these call outs before finishing up processing the group creation.
