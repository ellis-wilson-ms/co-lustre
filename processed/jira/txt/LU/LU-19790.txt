--- Description ---
When creating multiple filesets, it is possible to define alternating rw/ro flags within the same directory hierarchy:

1. /t1
2. /t1/ro_subdir (ro)
3. /t1/ro_subdir/rw_subdir
4. /t1/ro_subdir/rw_subdir/ro_subdir2 (ro) 

While this provides flexibility when setting multiple filesets, it also risks accidentally making read-only directories read-write. In this case, when 3 is mounted, the subdir from 4 can be accessed through a read-write mountpoint. Therefore, the creation of filesets 3 and 4 should not be possible.
