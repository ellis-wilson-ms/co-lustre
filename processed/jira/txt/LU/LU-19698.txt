--- Description ---
This issue is very easy to reproduce by the following steps:

lctl set_param mdd.*.trash_can_enable=1
lctl set_param mdd.lustre-*.trash_can_type=uid
mkdir $mnt/d
chmod 777 $mnt/d
su $user1
touch $mnt/d/f_user1
rm $mnt/d/f_user1
exit
ls $mnt/d/.Trash

No .Trash directory is created under $mnt/d, but file f_user1 can be found under $mnt/.lustre/trash.

I do not think this is a bug.

The reason is that the UID of the parent dir (0) is different from the UID of files (500) under this directory. 

Once the files with UID=500 are removed from the normal Lustre namesake, we can not identify its UID from the parent UID (0).

There is no any easy way to solve this kind problem in TCU. The user itself needs to avoid this kind problem.

The possible solution maybe use the UID of the parent dir to identify pFID, but it may be hard to locate the pFID with UID of files or directories under the parent dir.
