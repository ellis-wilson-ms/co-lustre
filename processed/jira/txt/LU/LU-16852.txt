--- Description ---
Past socklnd code restructuring due to tcp bonding feature removal caused ksni_npeers to be only checked against 0 on shutdown but never ever incremented.

ksni_npeers is used for reporting interface properties via "lctl --net tcp interface_list", but will always be shown as 0. 

It needs to be decided whether it makes more sense to keep ksni_npeers and make it functional again, or get rid of it.
