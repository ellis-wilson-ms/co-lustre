--- Description ---
llapi_dir_create has this code:

        dirpath = strdup(name);
        if (!dirpath) {
                free(lmu);
                return -ENOMEM;
        }

        namepath = strdup(name);
        if (!namepath) {
                free(namepath);
                free(lmu);
                return -ENOMEM;
        }

Of course it makes no sense to free() namepath when it was already NULL.

I suspect it was supposed to be freeing dirpath allocated just above it?
