--- Comment (sebastien, Wed, 8 Apr 2020 15:20:54 +0000) ---
As you can see from the logs of this successful run of sanity-sec test-31 at https://testing.whamcloud.com/test_logs/68aeef43-2eea-4566-95a7-bfa97456c2da/show_text, the error messages appearing when mounting with -o network=tcp999 with LNet Dynamic Discovery enabled are normal, as the test expects failure. Then the test disables LNet Dynamic Discovery, and mount is expected to pass.

I was able to reproduce the error reported here, and it seems to be due to a stalled connection between the client and the MGS:

# lctl dl
  0 UP mgc MGC10.128.11.155@tcp999 702b3c5b-1df0-4 4
# lctl
lctl > cfg_device MGC10.128.11.155@tcp999
lctl > cleanup force

[never returns...]

I was also able to test the tip of master branch (commit 742897a967 "LU-13274 uapi: make lnet UAPI headers C99 compliant") plus a revert of patch 4577410165 "LU-13028 lnet: advertise discovery when toggled", and in this case the issue does not occur.

So I can confirm that this is a regression with the patch 4577410165 "LU-13028 lnet: advertise discovery when toggled", or at least that this patch exposes an incorrect behavior that was previously hidden.

In fact, further testing shows that it breaks the whole -o network option, which is mandatory as soon as you want to implement multi-tenancy for Lustre. So I would prefer that this patch is reverted from master for now, and sanity-sec test_31 removed from the ALWAYS_EXCEPT list.
