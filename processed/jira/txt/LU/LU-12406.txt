--- Description ---
conf-sanity test_111 fails with ‘add mds1 failed with new params’. Looking at the client test_log, we see that the problem is with the --mkfsoptions flag in the mkfs command

--- Comment (adilger, Fri, 30 Aug 2019 18:54:38 +0000) ---
It looks like this problem is at least avoided by patch https://review.whamcloud.com/35358 "LU-11546 tests: enable large_dir support for tests", since that adds the "large_dir" option into the standard mkfs_opts(), so it isn't explicitly added to the end of the other options in test_111().

While that doesn't completely solve the problem (it would still be an error if large_dir was removed from fs_mkfs_opts, it would essentially never be hit again and this ticket could be closed as "Won't fix".

--- Comment (adilger, Fri, 30 Aug 2019 07:18:07 +0000) ---
This seems very similar to LU-12540, which recently landed patch https://review.whamcloud.com/35482 "LU-12540 tests: squash multiply -O -E options". 

I'm not totally satisfied with that solution, since it only fixes the test-framework to merge multiple options and not mkfs.lustre itself. That said, maybe a similar fix is needed for this test as well?  I see this test failing a lot on Oleg's test hardware, so it would be nice to clear that up, even if the solution is sub-optimal.
