--- Description ---
The line in the lustre spec file

Requires: modutils >= 2.4.10

leads to an unresolved rpm dependency in el7/Centos 7.  There is no longer any rpm that provides modutils.

I'm pretty sure this can be fixed by changing the Requires line to module-init-tools.  That is provided by rpms in all current distros as well as new ones like Centos 7.  In all old distros it comes from the same rpms that provided modutils (as far as I can tell).

The dependency shows up in the lustre-osd-ldiskfs rpm, so it's only a server build issue.  It won't be a blocker until we go beyond client-only support of Centos 7.
