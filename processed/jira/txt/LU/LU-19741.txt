--- Comment (paf0186, Thu, 25 Dec 2025 22:45:26 +0000) ---
I got curious about this and fed the issue into Augment, it found a number of other bugs which look real to me, though I'm not an expert in this rea.  The same things are also present in the upstream code.

It looks like large directories really haven't seen a lot of e2fsck action.

Complete Summary of Serious i_size Bugs in e2fsprogs
1. e2fsck/rehash.c - Directory Rehashing (4 bugs)

This file handles rebuilding hash tree directories. All bugs affect directories > 4GB.

Line 138 - fill_dir_block() function:

Should be: EXT2_I_SIZE(fd->inode) Impact: Incorrect bounds check when reading directory blocks. Will fail to detect corruption or incorrectly abort for large directories.

Line 1007 - e2fsck_rehash_dir() function:

Should be: EXT2_I_SIZE(&inode) Impact: Allocates buffer that's too small (only low 32 bits). Will cause buffer overflow when reading large directory.

Line 1011 - e2fsck_rehash_dir() function:

 

Should be: EXT2_I_SIZE(&inode) / 32 Impact: Allocates hash array that's too small. Will cause array overflow when processing large directory.

Line 1023 - e2fsck_rehash_dir() function:

Should be: EXT2_I_SIZE(&inode) / fs->blocksize Impact: Incorrect decision on whether to compress directory index for large directories.

2. e2fsck/pass1.c - Pass 1 Checks (2 bugs)

Line 255 - e2fsck_pass1_check_symlink() function:

Should be: EXT2_I_SIZE(inode) Impact: Symlink validation fails. Mostly harmless - symlinks can't legitimately be > 4GB, and line 182 already rejects symlinks with i_size_high != 0.

Line 3714 - check_blocks() function:

Should be: EXT2_I_SIZE(inode) / fs->blocksize Impact: Large directories won't be scheduled for rehashing when they should be, leading to inefficient directory structure.

3. e2fsck/super.c - Inode Release (1 CRITICAL bug)

Line 213 - release_inode_blocks() function:

Should be: EXT2_I_SIZE(inode) % fs->blocksize Impact: CRITICAL - Wrong truncate offset for files > 4GB. When releasing blocks from deleted files, will calculate wrong offset within the last block, potentially corrupting filesystem or causing incorrect block release.

4. e2fsck/emptydir.c - Empty Directory Processing (1 CRITICAL bug)

Line 170 - empty_pass1() function:

Should be: Use ext2fs_inode_size_set() or manually update both i_size and i_size_high Impact: CRITICAL - Subtraction only affects low 32 bits, doesn't update i_size_high. For a directory that's 4GB + 1 block, freeing 1 block would set i_size to 0xFFFFFFFF but leave i_size_high = 1, resulting in corrupted size of ~8GB instead of 4GB.

5. lib/ext2fs/link.c - Directory Linking (1 CRITICAL bug)

Line 643 - ext2fs_link() function:

Should be: EXT2_I_SIZE(&inode) / fs->blocksize Impact: CRITICAL - When appending to large directories, calculates wrong last block number. Will try to link into wrong block, corrupting directory structure.

6. lib/ext2fs/inline_data.c - Inline Data Conversion (2 bugs)

Line 383 - ext2fs_inline_data_dir_expand() function:

 

Should be: ext2fs_inode_size_set(fs, inode, fs->blocksize) Impact: When converting inline directory to block-based, doesn't clear i_size_high. If inode previously held large file, stale high bits remain.

Line 416 - ext2fs_inline_data_file_expand() function:

Should be: ext2fs_inode_size_set(fs, inode, 0) Impact: When clearing file size, doesn't clear i_size_high. Leaves stale high bits, corrupting file size.

7. lib/ext2fs/punch.c - Block Punching (1 bug)

Line 469 - ext2fs_punch_inline_data() function:

Should be: ext2fs_inode_size_set(fs, inode, 0) Impact: When punching inline data, doesn't clear i_size_high. Leaves stale high bits.

8. lib/ext2fs/mkdir.c - Directory Creation (2 bugs)

Line 111 - ext2fs_mkdir() function (inline data path):

Should be: ext2fs_inode_size_set(fs, &inode, EXT4_MIN_INLINE_DATA_SIZE) Impact: Low risk - creating new directory, unlikely to have stale i_size_high, but should still use proper helper for correctness.

Line 117 - ext2fs_mkdir() function (regular path):

Should be: ext2fs_inode_size_set(fs, &inode, fs->blocksize) Impact: Low risk - same as above.

9. lib/ext2fs/ext_attr.c - Extended Attribute Inode (1 bug)

Line 1292 - ext2fs_xattr_inode_create() function:

Should be: ext2fs_inode_size_set(fs, &inode, 0) Impact: Low risk - creating new EA inode, unlikely to have stale i_size_high.

10. e2fsck/pass3.c - Root/Lost+Found Creation (2 bugs)

Line 214 - e2fsck_get_lost_and_found() creating root:

Should be: ext2fs_inode_size_set(fs, &inode, fs->blocksize) Impact: Low risk - creating new directory.

Line 532 - e2fsck_get_lost_and_found() creating lost+found:

Should be: ext2fs_inode_size_set(fs, &inode, fs->blocksize) Impact: Low risk - creating new directory.
