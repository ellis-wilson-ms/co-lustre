--- Description ---
Current manual page lfs-hsm.1 has minor inconsistent formatting issues such as:

lfs hsm_state ...
lfs hsm_action file

and missing argument in option

[--archive-id]

According to lfs.c the options are:

{"hsm_set", lfs_hsm_set, 0, "Set HSM user flag on specified files.\n"
	 "usage: hsm_set [--norelease] [--noarchive] [--dirty] [--exists] "
	 "[--archived] [--lost] [--archive-id NUM]  ..."}

The provided patch ensure that the manual page and options in lfs.c are in match and consistent.
