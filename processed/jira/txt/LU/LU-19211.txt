--- Description ---
Only ksocklnd is tested via Maloo. Patches that exclusively touch other LNDs (assuming they still build via Jenkins) can not make the Maloo testing fail. We can save testing resources by omitting testing on these patches - and perhaps leaving a message telling developers to perform their own testing on the appropriate hardware.

Yes, AT uses parts of https://github.com/verygreen/lustretester/blob/master/gerrit_build-and-test-new.py to analyze patches for additional testing. I have pushed a patch with the current version of the script.

changed.py has a comment_only feature but it needs work. It was disabled due to ATM-3279. The logic to skip testing when changed.py returns comment_only=true is still active in AT so comment_only could be used for this ticket or a new key could be created to trigger a more relevant message to users.
