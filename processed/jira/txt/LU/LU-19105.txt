--- Description ---

--- Comment (adilger, Fri, 13 Jun 2025 15:06:44 +0000) ---
It would be possible on ldiskfs filesystems to use the "trusted.link" xattr to get the name of the directory entry, which should match the name of the entry to decide which of the entries is correct.  That would properly handle this test case where only the dirent name is corrupted, but it isn't clear if it would fix the original case where duplicate entries were actually added to the directory and the trusted.link xattr also contains the same (duplicate) name.

Probably e2fsck needs to be fixed to take the dirdata field into account when it is "correcting" the duplicate name, so it doesn't clobber the name, and possibly ensures that the replacement name does not get any longer and clobber the dirdata.
