--- Description ---
For the 2.3 release we need to change copyright messages for all Whamcloud copyright statements to be Intel, similar to LU-1146 and LU-365.

There is an existing script build/updatecw.sh which should be modified to do this, instead of doing it by hand.  It needs to leave the existing Oracle copyright statements intact.

--- Comment (adilger, Wed, 19 Sep 2012 18:53:35 +0000) ---
I suspect that e.g. libcfs_cpu.h was recently added, so it is correct to only have a 2012 copyright.

For files with multiple years, at Oracle they only showed "first, last" year, since in-between years are irrelevant for copyright legal purposes, and just clutter up the code.  Of course, Whamcloud itself only exists since late 2010, so nothing should be copyright before that year.  Also, it isn't clear that "first - last" is correct, if there wasn't a change in a middle year.  Please just stick with current format for now.
