--- Description ---
In autotest runs the mmp.sh test_1 through test_7 are always being skipped because check_failover_pair() is always returning false.  However, in a VM test environment it should always be possible to run the MMP tests from two different guests on the same node (so long as the guest VMs are not caching the MMP block updates incorrectly).

It looks like TT-240 was filed and fixed for running 2 OSS nodes for failover, so this is another possibility for running the MMP tests, rather than potentially taking advantage of the fact that e.g. the MDS and OSS VMs run on the same guest.
