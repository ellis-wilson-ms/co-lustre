--- Description ---
MGS NID tables are built when target registers upon start or when target network is changing. But if MGS failed over then NID tables are empty and there is no functionality on targets to notify MGS about their NID after reconnection, so NID tables remains empty until either target remount or target network is changed.

That is unfinished part of IR which was not critical because IR is not mandatory but possible improvement for recovery. But this is critical part to mount clients based only on IR but not config so targets should help MGS to restore NID tables

--- Comment (tappro, Wed, 10 Dec 2025 16:34:57 +0000) ---
Target doesn't notify MGS on reconnect about own NIDs so after MGS failover IR is not valid source of information about target NIDs anymore. 

Possible solution is to register obd_upcall in MGC, so it will be called on import events and call in turn tgt_nid_notify() with list of local node NIDs again, populating NID tables on MGS
