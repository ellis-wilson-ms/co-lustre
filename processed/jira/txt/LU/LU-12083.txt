--- Description ---
conf-sanity test_100 fails in interop testing with 'lshowmount have no output MGS'

Looking at a recent failure, 2.10.7 RC1 clients and 2.7.x servers, at https://testing.whamcloud.com/test_sets/bf2c7fb0-4306-11e9-92fe-52540065bddc, we see the following in the client test_log

setup single mount lustre success
CMD: trevis-54vm12 lshowmount -v
trevis-54vm12: sh: lshowmount: command not found
 conf-sanity test_100: @@@@@@ FAIL: lshowmount have no output MGS 

The patch for LU-8599, https://review.whamcloud.com/17593/, we see that conf-sanity test 100 was added to Lustre version 2.8.59. Thus, it’s clear that we need to add a Lustre version check for this test and not run this test for all interop testing with servers with version less than 2.8.59.
