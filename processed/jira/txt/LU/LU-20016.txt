--- Comment (JIRAUSER19283, Fri, 27 Mar 2026 01:36:53 +0000) ---
Computing parities and recovering a lost stripe is the same code and performs the same : ec_encode_data().

Running tests on the plain C version we currently use, the throughput for computing the parities for --ec 5+2

fluctuates between ~350MB/sec/core to ~450MB/sec/core.

Tests done on a KVM VM on a dell laptop.

The throughput number is measured as such:

Number of input stripes : 5 * stripe size : 1MB.

Testing throughput for EC computations on the upstream hand-optimized assembler versions clocks in at

~35GB/sec/core to ~40GB/sec/core

1, 40GB/sec/core should be plenty fast

2, we should switch to the assembler version of the library.

Note, these numbers are only for the EC computations.
