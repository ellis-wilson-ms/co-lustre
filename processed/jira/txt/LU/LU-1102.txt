--- Comment (bobijam, Mon, 27 Feb 2012 23:40:05 +0000) ---
the lproc capa switch is used to en/disable MDT/OST device's capability, and MDS/OSS always using export's connection flag (OBD_CONNECT_MDS_CAPA/OBD_CONNECT_OSS_CAPA)+ MDT/OST device's capability to decide whether run capa get code path.

And capa_encrypt_id() can only be called with remote client (OBD_CONNECT_RMT_CLIENT) connection, and 1.8 clients do not have these 3 flags in their connection request.

These connection flag values are:

under 1.8
#define OBD_CONNECT_RMT_CLIENT    0x10000ULL /*Remote client */
#define OBD_CONNECT_MDS_CAPA     0x100000ULL /*MDS capability */
#define OBD_CONNECT_OSS_CAPA     0x200000ULL /*OSS capability */

under 2.1
#define OBD_CONNECT_RMT_CLIENT    0x10000ULL /*Remote client */
#define OBD_CONNECT_MDS_CAPA     0x100000ULL /*MDS capability */
#define OBD_CONNECT_OSS_CAPA     0x200000ULL /*OSS capability */

They are consistent.
