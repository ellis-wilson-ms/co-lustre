--- Description ---
A project was reported over quota, after quota enforcement was disabled and sufficient block files were deleted.  It seems that after enforcement is disabled, deleting files won't change the quota reporting state.

Steps to reproduce this problem:

	enable quota enforcement

	define a project quota with soft and hard limits

	create enough data in project to exceed soft limit

	disable quota enforcement

	delete files from project

After these steps, the project quota will report exceeded soft quota even though files were deleted from project and usage is under soft limit.  This state will continue until quota enforcement is re-enabled.

It seems that it should not be necessary to re-enforce project quotas to have the project again display as within quota.

Thanks for your consideration.
