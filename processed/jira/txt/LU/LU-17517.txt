--- Comment (ake_s, Mon, 19 Feb 2024 15:45:13 +0000) ---
Unless I'm mistaken the code should not have been able to go past shift=31 in the first place.

If osfs->os_bsize and lov_sfs->os_bsize are really __u32 and not equal (otherwise it wouldn't end up in this loop in the first place) and strictly power-of-2 values, then the largest diff will be shift=31, (tmp=0x80000001).

The only real way this can happen is if one of osfs->os_bsize of lov_sfs->os_bsize are 0, which I guess should warrant its own check.

(Unless there's a compiler bug causing tmp to be >= 1<<32)
