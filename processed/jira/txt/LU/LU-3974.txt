--- Comment (bergwolf, Tue, 15 Oct 2013 05:57:40 +0000) ---
Bob, I didn't see the ll_read_dir() error message in upstream kernel client but I did see mkdir hung. It is because Lustre always saves the first page of a dir inode mapping at index ~0UL.  And after 3.11 commit 5a72039 (mm: teach truncate_inode_pages_range() to handle non page aligned ranges), truncate_inode_pages_range() NO LONGER truncates the page that is sitting at index ~0UL.

My patch to fix it is at:

https://github.com/bergwolf/linux/commit/28fde12d26d7d4b54cb4689dd6117c2ed791f985

It is not merge in upstream kernel yet because the problem was not there when I tested the code in staging tree before 3.11.
