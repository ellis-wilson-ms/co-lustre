--- Description ---
Manually compiled and installed ("make install")e2fsprogs doesn't install libext2fs.so.2 libraries that needed to mount.lustre and mkfs.lustre utils. As results these symptoms arrised:

 

mount.lustre: /dev/sdb1 has not been formatted with mkfs.lustre or the backend filesystem type is not supported by this tool.

Installing e2fsprogs from rpm solves the problem.

Hi,

I hit this issue but it's not really a make install issue but a configure issue.

I dumbly compiled and installed e2fsprogs with:

# ./configure --prefix=/usr && make && make install
# ls -l /usr/lib64/libext2fs.*
-r--r--r--. 1 root root 5186690 Dec 17 13:11 /usr/lib64/libext2fs.a

The result is that only libext2fs.a was installed in /usr/lib64.

To solve the issue, you must add --enable-elf-shlibs on configure command line

# ./configure --prefix=/usr --enable-elf-shlibs  && make && make install
# ls -l /usr/lib64/libext2fs.*
-r--r--r--. 1 root root 5186690 Dec 17 13:13 /usr/lib64/libext2fs.a
lrwxrwxrwx. 1 root root      14 Dec 17 13:13 /usr/lib64/libext2fs.so -> libext2fs.so.2
lrwxrwxrwx. 1 root root      16 Dec 17 13:13 /usr/lib64/libext2fs.so.2 -> libext2fs.so.2.4
-rwxr-xr-x. 1 root root 2717616 Dec 17 13:13 /usr/lib64/libext2fs.so.2.4
