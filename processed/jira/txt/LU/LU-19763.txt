--- Description ---
It turned out that the change introduced by https://jira.whamcloud.com/browse/LU-12945 impacts network performance significantly because it disables zerocopy in the sending path in the VM. During my LNet throughput test, it reduced the throughput by more than 30%.

 

I'd propose to add a module parameter to optionally disable it because it will hit the issue only for debug kernels. In fact, we should disable it by default.

 

Lustre has been used widely in cloud, we should fix this ASAP. I will work on patch soon.
