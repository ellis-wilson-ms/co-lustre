--- Description ---
colmstea, I noticed on a patch that is adding a test added to "EXCEPT_SLOW" that the subtest is skipped during review-subtest-change sessions as a result.

However, I think review-subtest-change sessions are a great place to run a slow subtest, since it can at least run once and probably several times in a dedicated session without affecting the main session.  It would be great if you could add SLOW=yes to these sessions so that the slow subtests are run.

This happened for patch https://review.whamcloud.com/63654 ("LU-19052 utils: lfs find --skip-rebalance for OSTs"):

I've updated the env configuration values for subtest-change sessions:

{
  "master"  => ["ONLY_MINUTES=30","HONOR_EXCEPT=y","SLOW=yes"],
  "b_es6_0" => ["ONLY_MINUTES=30","HONOR_EXCEPT=y","SLOW=yes"],
  "b_es7_0" => ["ONLY_MINUTES=30","HONOR_EXCEPT=y","SLOW=yes"],
  "default" => ["ONLY_REPEAT=10","HONOR_EXCEPT=y","SLOW=yes"]
}

The change is effective on all new sessions so retests will pick up the change. Re-building is not necessary.
