--- Description ---
"lfs migrate" with only "--pool/-p" argument does not migrate file to pool. It initiate a migration but with the layout of source file (llapi_layout_get_by_path).

When a pool is full and we want to initiate migrations , the behavior is the opposite than expected:

	the migrations create volatiles on the source pool, so the space uses on the full pool increase temporally

	it does not migrate files to the specified pool.
