--- Description ---
Creating hard links to a file on a remote MDT causes an agent inode to be allocated on the target MDT.  However, if multiple hard links are created on the target MDT for the same source inode, a new agent is allocated for each hard link.

It would be better to create only a single agent inode on the target MDT and use it for all of the hard links on that MDT.  That avoids consuming a large number of inodes on the same MDT when there are multiple hard links to a file (e.g. "rsync --link-dest" backups).
