I want to verify that  readcache_max_filesize is object size on OST not actual file size.

--- Comment (adilger, Thu, 29 Jun 2017 17:56:18 +0000) ---
Mahmoud, yes the readcache_max_filesize tunable is the OST object size and not the file size.  The intention of this parameter is that it can be used to limit the read cache on the OSS to small files/objects, and avoid caching large files that are unlikely to be re-used and will overwhelm the cache.
