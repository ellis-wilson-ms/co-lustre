--- Description ---
while checking for lfs find parameters of a PFL file, if at least 1 component matches then the file matches.

however negation of it is !(at least 1 component matches) = no one component matches, 

i.e. not "at least one component does not match” - what is checked now in some cases now.

e.g. having stripe size 1M, 2M for 2 components and searching:

> lfs find ! -S 2M /mnt/lustre

this file should not be found.

presumably all the fp_exclude_* options are involved which are passed to find_value_cmp.
