--- Description ---
Recently I had to add to build environment openjdk 1.8.0 devel as dependency, which is not supported since 2019.

During discussion, adilger stated:

My preference would be to remove the dependency on Java completely, if we can, by setting NO_JVMTI in the kernel.spec file

--- Comment (yujian, Fri, 31 Jan 2025 01:27:07 +0000) ---
It turns out the correct version is java-11-openjdk-devel. Please refer to DCO-10120.

--- Comment (yujian, Thu, 30 Jan 2025 19:47:01 +0000) ---
In RHEL 9.5 kernel.spec:

%if %{with_perf}

BuildRequires: zlib-devel binutils-devel newt-devel perl(ExtUtils::Embed) bison flex xz-devel

BuildRequires: audit-libs-devel python3-setuptools

BuildRequires: java-devel

......

%endif

The java-devel package is required by building perf (Performance monitoring for the Linux kernel) for the kernel.

I'm not sure which version of java-devel is the correct one to meet the requirement. Can we try to find the correct one instead of changing kernel.spec? Or just disabling building the perf package?
