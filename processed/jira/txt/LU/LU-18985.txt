--- Description ---
When files in a striped directory are deleted, each of the stub pFID objects may need to be created on multiple MDTs, so that when there is a file deleted on that MDT it will be possible to move it into the stub directory without a distributed DNE transaction.

When displaying such directories to the user via the .Trash/pFID directory on the client, it would be useful to present these multiple stub directories at a single virtual unified directory. That allows a user to see all of the files in the directory in a single location, even if they are stored in multiple MDTs.
