lfs-performance:test_5a creates up to UNIT * MDSCOUNT * MAXSUBDIR files striped over all OSTs.

For MDSCOUNT=4, UNIT=8192, MAXSUBDIR=4 it fails like:

..
+++++ Start cycle mdscount=3 at: Tue Sep  3 14:26:40 MSK 2024 +++++
..
open(/mnt/lustre_lfsck_1_2/d5a.lfsck-performance/1/2/f7484) error: File too large
..
 lfsck-performance test_5a: @@@@@@ FAIL: (14) Fail to gen 8192 files under /mnt/lustre_lfsck_1_3/d5a.lfsck-performance/1/3

The test should check that there are enough inodes on OSTs.
