--- Description ---
Currently the NID for loopback shows as "0@lo"

# lnetctl net show
net:
    - net type: lo
      local NI(s):
        - nid: 0@lo
          status: up
          interfaces:
              0: lo

It would be better if the  loopback IP address is used instead of 0 in loopback NID.

--- Comment (sharmaso, Wed, 16 Jan 2019 17:52:55 +0000) ---
Amir, James and I were discussing this LNet sysfs patch, wherein we make providing network interface information necessary.

https://review.whamcloud.com/#/c/33583/3

While discussing this, we thought of having the interface name specified for loopback nid as well so as not to special case the loopback NID. In this discussion, the idea came up to have an address for "lo" NID instead of 0.

I just discussed with Amir and James again on this and they agree to your thought of not confusing this with the tcp loopback. So we will keep this as it is. 

I am closing this ticket.

Thanks

--- Comment (adilger, Wed, 16 Jan 2019 07:48:07 +0000) ---
What is the motivation behind this change, and why do you say "better"?  Using e.g. 127.0.0.1 would IMHO add confusion because the @lo interface is not the TCP loopback interface, it is internal to LNet, so if you show 127.0.0.1 here it would give the false impression that it relates to TCP loopback, when in reality it has no relation.
