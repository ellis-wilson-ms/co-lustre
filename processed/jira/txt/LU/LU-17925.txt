Clarification on Stripe Count Logic

The following is the logic for the most confusing part of the code regarding stripe count adjustments:

 

if (stripe_count > lov_desc->ld_active_tgt_count) {

    if (overstriping) {

        if (stripe_count >= LOV_ALL_STRIPES_MIN && stripe_count ld_active_tgt_count;
        }
    }

}

 

Explanation:

    When -C or -c -1 is passed, the stripe count will be 0xffdf, which results in:

(0xffdf - 0xffdf + 1) * ost_count
