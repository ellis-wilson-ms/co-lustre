--- Description ---
multiop_bg_pause under test-framework starts multiop in background and waits for signal if "_" option is provided. On 'verbose' mode the PAUSING string is printed on console which is checked and if not found error is reported by multiop_bg_pause function.

On error, it is required to kill the existing running multiop binary as if not done will eventually timeout and not exit the test. Currently on error multiop_bg_pause function incorrectly sends singlal to wrong PID.
