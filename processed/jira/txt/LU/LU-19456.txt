--- Description ---
There is a line in `contrib/lbuild/lbuild` that looks like: `location="https://linux.mellanox.com/public/repo/doca/${DOCA_VERSION}/SOURCES/MLNX_OFED"`
	
		The MLNX_OFED needs to be lowercase when the doca version is 3.1.0: https://linux.mellanox.com/public/repo/doca/3.1.0/SOURCES/mlnx_ofed/ vs https://linux.mellanox.com/public/repo/doca/3.1.0/SOURCES/MLNX_OFED/ (The second one with capital is giving not found error)
		
			For other current lower versions, MLNX_OFED is capitalized: https://linux.mellanox.com/public/repo/doca/2.9.3/SOURCES/MLNX_OFED/ vs https://linux.mellanox.com/public/repo/doca/2.9.3/SOURCES/mlnx_ofed/ (This time the second one with lower case is giving not found errors)

		
		

	
	

	We need to add `[“3.1.0”]=“0.9.7.0”` to the `doca_version_map` for sles15sp7

	For branch b_es6_0, on line 1896 it goes into the same case block for mlnx and doca, but for the b_es7_0 branch the same code on line 1892 shows the doca case statement has its own block and is causing the build to fail for `sles15sp7` builds. It is changing the sles15sp7 to sles15u7. I think this is correct for rhel builds, but it shouldn't be doing that for sles builds
