--- Description ---
According to current metadata migration implementation, before the old MDT-object is removed, both the new MDT-object and old MDT-object will reference the same LOV layout. Then if the layout LFSCK finds the new MDT-object by race, it will regard related OST-object(s) as multiple referenced case, and will try to create new OST-object(s) for the new MDT-object.
