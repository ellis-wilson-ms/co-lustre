--- Comment (rhenwood, Fri, 5 Dec 2014 17:20:55 +0000) ---
I've spent a bit of time looking into tools to wrap xml. The best I found is 'tidy'. It has a problem, however, which is that in xml mode, --new-pre-tags are ignored. What this means, is that  and  tags, which you typically would want to avoid being parsed and wrapped, get parsed and wrapped.

So you need to go and manually check the  and s after you tidy. For an example of a 'tidy'ied file see here:

http://review.whamcloud.com/#/c/11938/

This was generated with the command below, and then  tags were manually reviewed.

cat ./LustreTuning.xml | ~/tools/tidy/bin/tidy --new-pre-tags screen,programlisting --wrap-script-literals no -utf8 -xml -w 80 -i -c -asxml | vi -
