--- Description ---
There are typos in Chapter 22 of the Lustre Manual, which describes Hierarchical Storage Management.

In section 22.7 on changelogs, all of the HSM events are listed with the same name, HE_STATE.  This should match the enum definition in the code (without the spares, of course):

enum hsm_event {                  
        HE_ARCHIVE      = 0,
        HE_RESTORE      = 1,
        HE_CANCEL       = 2,
        HE_RELEASE      = 3,
        HE_REMOVE       = 4,
        HE_STATE        = 5,
        HE_SPARE1       = 6,
        HE_SPARE2       = 7,
};
