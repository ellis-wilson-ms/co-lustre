--- Description ---
While removing non-ASCII control characters embedded in the Lustre manual ("fancy quotes") I came across the ip2nets examples in the ConfigurationFilesModuleParameters.xml page (https://doc.lustre.org/lustre_manual.xhtml#lnet_ip2nets).

These examples appear to be quite inconsistent, in one place referencing IB and TCP, and in another place referencing the ancient Elan network.

These examples should be updated to be internally consistent (e.g. using the same addresses for each LNet network), and use IB instead of Elan for consistency.
