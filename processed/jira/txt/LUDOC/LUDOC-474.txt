--- Description ---
Lustre documentation (section 25.4.1) provides examples of how to set quotas for users, groups, & projects without needing to explicitly set quotas on each entity.  Great feature, btw!

That section also demonstrates how to disable the default user/group/project quotas.

Given that a u/g/p could have a non-default quota, could we add an example of how to revert a non-default u/g/p quota to the default u/g/p quota?

The following seem to set specific user/group/project quotas to use the default quota limits:

	lfs setquota -d -u user1 /scratch

	lfs setquota -d -g group1 /scratch

	lfs setquota -d -p project1 /scratch

That ability is identified in the lfs setquota usage, and also in the lfs-setquota man page, but thinking it might be helpful to have an example of the -d option somewhere in/around section 25.4.1.

Thanks!
