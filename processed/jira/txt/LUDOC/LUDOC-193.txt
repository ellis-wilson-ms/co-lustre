--- Description ---
Section 5.3.3, page 27 (of the PDF Manual) states that you need to use the tune2fs -O option for large_xattr – when in fact it is ea_inode. Tune2fs -o large_xattr fails because it's an unknown option. I asked James Simmons and the mkfs.lustre option for large_xattr fails as well. We recently had to enable this on our Lustre 2.4 filesystem because we forgot to enable it at format time.

Jason, 

There is room for confusion around the large_xattr/ea_inode feature. Right now, you can use "large_xattr" and "ea_inode" interchangeably in tune2fs and with mkfs.lustre, i.e. "tune2fs -O large_xattr ..." and "tune2fs -O ea_inode" produce the same result and the same is true for mkfs. In either case, you will see "ea_inode" in the filesystem features list with dumpe2fs. I just tested this with the latest 2.5.0 and 2.4.1. 

If your experience differs from what I've said above, please let me know what version of the code you are seeing this in and I will look into it. 

I know that there is a possibility that only large_xattr will be used in the future, but that has not happened yet. Thus, I agree that the documentation should be updated. I will post a patch to the documentation later today.

Thanks,

James
