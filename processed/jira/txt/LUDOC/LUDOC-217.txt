--- Description ---
A drawback of using the jenkins build as the canonical documentation is that Jenkins disables web crawlers from indexing. This means that all Lustre searches return results from the old documentation on lustre.org. It would be nice if there was a search engine friendly mirror somewhere. Or, maybe a better solution is a reverse proxy pointing at the manual on build.hpdd.intel.com so no synchronization step is needed.
