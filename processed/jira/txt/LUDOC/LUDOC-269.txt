--- Description ---
The default build environment for the doc/protocol project was moved from RHEL6 to RHEL7 because the former included a version of GraphViz's 'dot' program (dot-2.26) that segfaults. Since most users of this project will be free to use later working versions of 'dot' this is not a serious issue, for now. It would be good to know why there is a problem and if there is a fix. Documenting the issue so that people aren't surprised is also good.

I do know know the version of 'dot' on RHEL7 (the one that works) but the following have been shown to work:

Ubuntu 12.04 dot-2.36

Mac 10.10.2 (Yosemite) ports installed dot-2.38.0

All of this may be very particular to the mkdir1.gv file, or it may have wider implications.
