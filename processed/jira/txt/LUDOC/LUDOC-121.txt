--- Description ---
The description on Network Interface Credits (section 25.3.4) is missing two critical items:

1- It talks about how to set this tuneable and how it is distributed over the CPT's, but not what effect this tunable has when it is changed.  Does it affect flow control (if so, how), etc.

2- In the case of o2iblnd (IB), this value controls the queue depth for RDMA.  This must by symmetric on all the systems or we will fail connections.  In other words, if this tunable is changed on one system, it needs to be changed on all systems.  That is a very important point to have here!
