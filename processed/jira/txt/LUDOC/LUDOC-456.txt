--- Description ---
The xml:id for each section should have a meaningful name instead of docbook..  This change makes the URLs contain the meaningful section ID which is indicative of the section of the manual to which it links.  It is also nice when editing an xml file to know what section the link is referring to without having to search for the random docbook numeric string to find out.

This ticket should capture all the refactoring needed to update the sections defined in each XML file (one file per patch as a general rule of thumb) and the associated pages that are linking to those section IDs.

An example of a patch already landed for this type of work:  https://review.whamcloud.com/#/c/36015/
