Summary of changes:

Nesting pages is not well-supported in mediawiki.¬† I always forget this.¬† This unfortunately means doing something like "releases/2.12.0/{changelog,release_notes}" is not plausible, as nice as that would be.

¬†

Changelog Pages: While we're not normally supposed to lead with "Lustre" in the name since this is the Lustre Wiki, I think for the changelogs I prefer this parlance.¬† I normalized all pages to the format:

Lustre_[.O]>_Changelog

The early changelog pages contain multiple patch release versions per page, but I prefer the more recent single change per page format.

All pages were added to the Releases category.¬† If they were in the Changelog category before (only a handful) that category tag was eliminated.

Release Pages: These were already titled Release_[.O]>, which I think is great, so I left these alone.¬† Any pages missing the Releases category tag were updated with it.

LUG Pages: These were already named well.¬† Just added a LUG category tag since this is a sizable pile of pages.

LAD Pages:¬†

Added LAD to the one page that's on here.¬† It would be great to get the rest categorized appropriately.

Lustre.org Minutes Pages:

Ensured pre-existing Lustre.org category was used across these pages.¬† A few were missing them.

EOFS, Lustre Fall Workshop, All Hands, and HPC Workshop: were all left alone as they had an Events tag and were named acceptable.¬† Again, it would be stellar to organize these into an misc events directory, but I guess we just tolerate this kind of legacy content for now in the mix with the rest.
