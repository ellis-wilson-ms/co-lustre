--- Description ---
Currently, the manual section on using LFSCK to do the fid-in-dirent and linkEA updates does not include any reference to enabling dirdata on the MDT before doing LFSCK -t namespace. (Command to check and repair fid-in-dirent and linkEA)

My understanding is that this is a necessary step before using lfsck to add/repair fid-in-dirent and linkEA.  Presumably the manual should explain this.

Also, I'm not sure if LFSCK will do the right thing if it is started with -t namespace when dirdata is disabled.  I'm putting a question about this in the LFSCK Phase 1.5 ticket:

https://jira.hpdd.intel.com/browse/LU-1866

See https://jira.hpdd.intel.com/browse/LUDOC-85 for documentation changes already made for LFSCK Phase 1.5.

This may be a partial dupe of https://jira.hpdd.intel.com/browse/LUDOC-68, but this does not reference LFSCK fid-in-dirent and linkEA updates.
