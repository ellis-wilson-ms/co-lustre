--- Description ---
The latest manual discusses displaying changelog user information in

12.1.2.5. Displaying the Changelog Index and Registered Users

mdd.*.changelog_size is also important to document, since unconsumed changelogs can cause problems.

This should also include the UNITS that this changelog size is reporting. It's a number, what does that number mean? This is a pet peeve of mine with various lustre stats and numbers from lctl. Some give an xml-ish output with units, others give a number. Chaos! If units aren't included, let's document them.

--- Comment (adilger, Sat, 31 Oct 2020 00:53:18 +0000) ---

--- Comment (adilger, Sat, 31 Oct 2020 00:51:25 +0000) ---
The units of "changelog_size" are in bytes.  You can use "lctl get_param mdd.*.changelog_users" to get this in "records" units:

mdd.myth-MDT0000.changelog_users=
current index: 75982977
ID    index
cl2   66834025
